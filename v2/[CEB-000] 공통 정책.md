## 목적

이 문서는 CELEBUS 서비스 고도화 전체에 걸쳐 반복 등장하는 **공통 정책, 규칙, 규격**을 한 곳에 정의합니다.

> 각 화면 스펙 문서는 **화면 고유 로직**만 기술하고, 공통 정책은 이 문서를 참조합니다.
> 정책 변경 시 이 문서만 수정하면 됩니다.

**범위:** 고도화 서비스(Quest 스토리, 덕력, 디지털 굿즈, 경품/응모, 기억저장소, 정보/일정/공지) 전체에 적용됩니다. 게임존(PM/ST) 전용 정책은 [CEB-BO-GZ-000] 공통 정책을 참조하세요.

## 3. 재화 체계

### 3.1 유저 인지 재화 4종

유저가 인지해야 할 재화는 4종이다. 이원화 구분(획득/보유 덕력, 메인/이벤트 GP)은 내부 로직으로만 관리하며, 유저 화면에서는 통합 표시한다.

| 재화 | 성격 | 획득 | 소비 | CELB 교환 | 소멸 | 범위 |
| ---- | ---- | ---- | ---- | ---- | ---- | ---- |
| 덕력 | 활동 지표 (XP) | 퀘스트/Trivia/PM/SNS 등 자동 적립 | 서포트 이벤트 응원, 독점 콘텐츠 해금 | 불가 | 랭킹만 연 1회 초기화 (보유 덕력은 소멸 없음) | 아티스트별 별도 |
| 응모권 | 래플 참여권 | 퀘스트 보상, ST 탈락 보상, PM/ST 보상 (BO에서 게임별 응모권 지급 여부 설정), GP로 응모권 구매 (BO에서 비율 설정) | 래플 응모 (1장 = 1회) | 불가 | 사용 시 소모 | **아티스트 무관 통합** (어떤 아티스트 활동으로 획득한 응모권이든 모든 래플에 사용 가능) |

- GP→응모권 교환: BO에서 비율 설정 (기본값: 100GP = 응모권 1장). 구매 한도: BO에서 설정 (기본값: 제한 없음)
| GP | 게임 재화 | 게임 보상, CELB 교환, 프로모션 지급 | 게임 참여, CELB 교환 | 메인 GP만 양방향 | BO에서 특정 기간 전체 삭제 가능 (삭제 1개월 전 공지 필수). Celb 상장 전까지 무료 GP 사용 | 게임존 전체 공통 |
| Celeb Point | 인앱 결제 재화 | 실제 결제 (인앱 구매) | NFT 변환, 프리미엄 기능 (추후 확장) | 불가 | 소멸 없음 | 공통 (아티스트 무관) |

> **Celeb Point**: Coming Soon. 사용처 확정 후 오픈. 유저 UI에 "Coming Soon"으로 노출.

### 3.2 덕력 이원화 원칙

덕력은 **획득 덕력**과 **보유 덕력**으로 이원 관리된다. 유저 화면에서는 통합 표시하고, 상세 페이지에서만 구분한다.

| 구분 | 설명 | 용도 | 초기화 |
| ---- | ---- | ---- | ---- |
| 획득 덕력 | 활동할 때마다 누적 | 개인 랭킹 산출 기준 | 연 1회 초기화 (**BO에서 시즌 종료일 설정**, 종료 시 자동 초기화) |
| 보유 덕력 | 획득 - 사용 | 서포트 이벤트 응원, 독점 콘텐츠 해금 | 소멸 없음, 영구 유지 |
| 사용 덕력 | 서포트 이벤트 + 독점 콘텐츠에 응원한 총량 | 이력 조회용 | - |

- 서포트 이벤트에 덕력을 응원해도 **랭킹에는 영향 없음** (랭킹은 획득 덕력 기준)
- 유저 화면: 통합 표시 ("내 덕력 1,234pt"), 상세에서만 획득/보유/사용 구분

#### 연간 시즌 초기화 절차

| 순서 | 절차 |
| ---- | ---- |
| 1 | BO에서 시즌 종료일 도래 → 랭킹 동결 |
| 2 | 관리자가 TOP 순위 확인 후 보상 수동 지급 |
| 3 | 관리자가 "시즌 초기화" 실행 → 획득 덕력 0으로 리셋 |
| 4 | 새 시즌 자동 시작 |

> 보유 덕력은 초기화되지 않음. 진행 중인 서포트 이벤트 응원분도 영향 없음.
> 시즌 간 공백: 종료~초기화 사이 획득 덕력은 새 시즌에 귀속.

### 3.3 GP 이원화 원칙

> **고도화 런칭 시 이벤트 GP는 이미 소멸 완료** → 메인 GP만 존재한다.

| 구분 | 설명 | CELB 교환 | 소멸 |
| ---- | ---- | ---- | ---- |
| 메인 GP | 게임 보상, CELB 교환 | 양방향 가능 | BO에서 특정 기간 전체 삭제 가능 (삭제 1개월 전 공지 필수). Celb 상장 전까지 무료 GP 사용 |
| 이벤트 GP | 프로모션 지급 (상장 전 한정) | 불가 | 상장 N일 전 전액 소멸 |

- 유저 화면에는 합산 잔고 표시 (내부적으로만 분리 관리)
- 고도화 런칭 시점에서는 메인 GP만 운영

> GP 소멸 시 1개월 전부터 앱 내 공지 + 푸시 알림 발송. BO에서 삭제 일정 설정.

#### 이벤트 GP 소멸 타임라인

| 시점 | 이벤트 | 설명 |
| ---- | ---- | ---- |
| 상장 60일 전 | 소멸 예고 공지 | 이벤트 GP 소멸 예정 안내 |
| 상장 60일 전 ~ 30일 전 | 사용 유도 기간 (30일간) | 응모권 구매, 게임 참가 등으로 소진 유도 |
| 상장 30일 전 | **이벤트 GP 전액 소멸** | 이벤트 GP 잔고 일괄 0 처리 |
| 상장일 | CELB↔GP 교환 오픈 | 메인 GP 활성화, 게임 보상 메인 GP로 전환 |

> GP 상세 정책은 [CEB-BO-GZ-000] 공통 정책 및 [CELEBUS 서비스 고도화 기획서 - GP 시스템.md]을 참조합니다.

### 4.2 {아티스트} 탭 3그룹 구조

{아티스트} 탭은 9개 서비스 카드를 3개 그룹으로 구성한다. 카드 그리드 레이아웃(2열 기본, 홀수 행 1열 풀)으로 배치.

| 그룹 | 포함 서비스 카드 | 성격 |
| ---- | ---- | ---- |
| 미션 (3카드) | V01D 챌린지 (Quest), 일일 미션, {아티스트명} 응원하기 | 유저가 수행할 행동 |
| 내기록 (4카드) | 덕력 + 랭킹, 컬렉션 (디지털 굿즈), Raffle, {아티스트명} 키우기 | 유저가 쌓은 결과 |
| 더보기 (2카드) | V01D 정보/일정/공지, 기억저장소 | 정보/감성 콘텐츠 |

> 상세 레이아웃, 카드별 상태 정보, 탭 액션은 [CEB-ART-101] 참조.

### 5.3 Quest 보상 설정 (BO)

Quest 보상은 원타임(5장 스토리)과 반복 퀘스트 모두 동일한 방식으로 설정한다.

| 보상 유형 | 설명 | 설정 |
| ---- | ---- | ---- |
| 덕력 | N pt 적립 | BO에서 포인트 값 입력 |
| 응모권 | N장 지급 | BO에서 수량 입력 |
| 디지털 굿즈 | 지정한 디지털 굿즈 지급 | BO에서 디지털 굿즈 선택 |

- **복수 선택 가능**: 하나의 퀘스트에 덕력 + 응모권 + 디지털 굿즈를 모두 설정 가능
- 5장 완료 시 추가: **상시 보상 해금** (사인 포카 추첨 응모 자격, 독점 콘텐츠 해금)

#### 서포트 이벤트 상태 전이 트리거

| 전이 | 트리거 |
| ---- | ---- |
| 모집중 → 달성 | 서버 자동 (응원 시 실시간 판정: 누적 >= 목표) |
| 모집중 → 미달성종료 | 서버 자동 (마감일시 경과 + 목표 미달성) |
| 모집중 → 미달성종료 | BO 강제 종료 (응원 덕력 전액 반환) |
| 달성 → 집행중 | BO 수동 전환 |
| 달성 → 집행취소 | BO 수동 전환 (집행 불가 시. 전액 반환) |
| 집행중 → 완료 | BO 수동 전환 (결과물 업로드) |
| 집행중 → 집행취소 | BO 수동 전환 (전액 반환) |

| 규칙 | 정책 |

| ---- | ---- |
| 서사 Quest 구조 | 5장 고정 |
| 각 장 내 서브 퀘스트(미션) 수 | 최대 10개 (BO 설정) |
| 임시 해금 조건 | 현재 장의 모든 미션을 **제출** 완료 시 다음 장 임시 해금 (승인 불필요) |

| 반려 시 영향 | 해당 미션만 재제출 필요. 다른 장 진행에 영향 없음 |

#### 장 상태 정의

| 상태 | 코드 | 설명 |
| ---- | ---- | ---- |
| 잠금 | locked | 이전 장 미완료 |
| 임시해금 | provisional | 이전 장 전체 제출로 해금, 미션 시작 전 |
| 활성 | active | 미션 수행 가능 (1개 이상 시작) |

> 1장은 초기 상태 active (잠금 없음). 반려 시 reviewing → active로 복귀 (반려 미션만 재제출).

### 10.4 캐러셀 배너 → 다른 아티스트 이벤트 탭 시 정책

메인 캐러셀 배너는 아티스트 무관 통합이므로, 현재 선택된 아티스트와 다른 아티스트의 이벤트를 탭할 수 있다.

| 상황 | 정책 |
| ---- | ---- |
| 팔로우한 아티스트 이벤트 탭 | 아티스트 선택 **유지** (전환하지 않음). 이벤트 상세 화면만 이동. 뒤로가기 시 기존 아티스트 콘텐츠로 복귀 |

| 이벤트 상세에서 뒤로가기 | 홈으로 복귀, 기존 선택된 아티스트 콘텐츠 유지 |

> 캐러셀 배너 탭 시 아티스트 선택이 자동 전환되면 유저가 혼란을 느낄 수 있으므로, 아티스트 선택은 항상 유저가 직접 아티스트 로고를 탭해야만 전환된다.
