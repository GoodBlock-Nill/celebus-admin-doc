1. 화면 개요

진행 중인 래플의 상세 정보·혜택·내 응모 현황을 확인하고, 보유 응모권으로 응모하는 화면.

주요 기능
  • Raffle 기본정보 + 보상 + D-day
  • 누적 참여자·누적 응모권 통계
  • 내 누적 응모권 표시
  • 응모 수량 입력(Stepper) + [응모하기]
  • 상태 분기: 진행중 / 집계중 / 종료
접근 경로
  • [CEB-EVT-101] Raffle 메인 > 진행중 카드 탭
  • [CEB-HOM-001] 홈 메인 > 이벤트 카드 탭
  • 딥링크

3. 규칙·로직

3.1 상태 분기

Raffle 상태처리
진행중본 화면 유지
집계중"현재 집계 진행중입니다" 토스트 → 이전 화면 복귀
종료 + 당첨[CEB-EVT-103] Raffle 상세 (당첨) 교체 전환
종료 + 미당첨[CEB-EVT-104] Raffle 상세 (미당첨) 교체 전환

3.2 응모 수량 (Stepper)

항목규칙
기본값1 (보유 ≥ 1)
최소1 (0 입력 시 1로 보정)
최대보유 응모권
1인당 한도없음 (Max Cap 정책 없음)
비활성 조건보유 0장 → Stepper 0 fixed + Disabled

보유 부족 시 "보유 응모권이 부족합니다" 인라인 에러.

3.3 [응모하기] 동작

  • 입력 수량 유효 시 활성
  • 클릭 → [CEB-EVT-102-MD] Raffle 응모 모달 호출 (수량 n 전달)

3.4 체류 중 마감 처리

  • 상세 체류 중 서버에서 진행중 → 집계중으로 전환된 경우, [응모하기] 클릭 시 이벤트 종료 응답 → "이벤트가 종료되었습니다" 안내 후 이전 화면 복귀.
  • 예: D-day가 다음날 00시로 넘어간 직후.

3.5 응모권 거래 코드 (BO-006 §7.6)

응모 시 차감 트랜잭션은 TICKET_RAFFLE_USE 코드로 BO [CEB-BO-FQ-301] 응모권 거래 내역에 기록.

3.6 다국어

  • 아티스트명·타이틀·설명: 서버 데이터 (BO 다국어 설정 의존)
  • 시스템 UI: 앱 다국어 처리 (KO/EN/JP)
