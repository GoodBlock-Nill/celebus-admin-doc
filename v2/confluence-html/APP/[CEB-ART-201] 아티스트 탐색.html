1. 화면 개요

목적

팔로우할 아티스트를 탐색하고 추가/제거하는 화면이다. 런칭 초기 소수 아티스트부터 대규모 확장까지 대응한다.

핵심 원칙: 팬이 좋아하는 아티스트를 쉽게 찾고, 한 탭에 팔로우할 수 있어야 한다.

주요 기능
  • 추천 아티스트 카드로 빠른 탐색
  • 전체 아티스트 리스트에서 검색/팔로우
  • 팔로우 중인 아티스트 관리 (언팔로우)
접근 경로
  • 홈 > 아티스트 선택 영역 > [+] 추가 버튼
  • 최초 가입 시 아티스트 선택 필수 (온보딩에서 진입)
레이아웃 참조

화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

3. 공통 규칙

3.1 팔로우 제한

규칙설명
최대 팔로우 수제한 없음
최소 팔로우 수1명 필수 (마지막 아티스트 언팔로우 차단)
중복 팔로우불가 (이미 팔로우 중이면 버튼 비활성)

3.2 멀티 아티스트 대응 (공통정책 10 참조)

상황동작
첫 팔로우팔로우 완료 → 홈 이동 + 해당 아티스트 자동 선택
추가 팔로우팔로우 완료 → 홈 이동 + 해당 아티스트로 전환
언팔로우 (남은 1명+)내 아티스트에서 제거. 홈은 다른 아티스트로 자동 전환
마지막 아티스트 언팔로우차단. 토스트(Error) "최소 1명의 아티스트를 팔로우해야 해요"

3.3 데이터 갱신

트리거동작
화면 진입아티스트 리스트 + 팔로우 상태 조회
팔로우/언팔로우해당 아티스트 상태 즉시 반영
Pull-to-Refresh전체 재조회 (공통정책 11.2 참조)
