1. 화면 목적

퀘스트 심사 결과가 '반려(REJECTED)'된 구체적인 이유를 유저에게 전달한다. 단순 실패 통보에 그치지 않고, 반려 사유를 개선하여 다시 참여(재도전)하도록 유도하는 모달이다.

주요 기능
  • 반려 사유 텍스트 표시 (Admin에서 선택/입력한 사유)
  • [확인] 버튼으로 모달 닫기
  • [재도전] 버튼으로 퀘스트 제출 화면 이동 유도
호출 경로
  • CEB-FQ-101 Quest 메인 > 타임라인 > 반려(REJECTED) 상태 미션 탭 시 호출

2. UI 설명

2.1 공통 레이아웃

화면 레이아웃은 프로토타입(localhost:3001)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

3. 화면 내 기능설명

3.1 호출 조건

  • 호출처: CEB-FQ-101 Quest 메인 > 타임라인
  • 조건: 타임라인에서 상태가 REJECTED(반려)인 미션을 탭(Tap)했을 때

3.2 데이터 바인딩

서버에서 내려준 reject_reason_code에 매핑된 메시지를 표시한다.

코드명 (Code)운영자 노출명 (Admin Label)유저 노출 메시지 (Client Message)설명 및 가이드
IMG_BLUR식별 불가 (흐림/어두움)이미지가 흐릿하거나 너무 어두워서 내용을 확인할 수 없습니다. 선명한 사진으로 다시 올려주세요!초점이 나갔거나 조명이 부족해 검증이 어려운 경우
WRONG_OBJ대상 불일치 (다른 사물)퀘스트와 관련 없는 사진입니다. 미션 가이드를 다시 확인하고 올바른 대상을 촬영해 주세요.영수증 미션에 풍경 사진을 올리는 등 엉뚱한 이미지인 경우
DUP_ENTRY중복 참여 의심이미 제출되었거나 다른 유저가 사용한 이미지와 동일한 것으로 확인됩니다. 직접 촬영한 새로운 사진으로 참여해 주세요.어뷰징(도용, 재사용) 방지용
DATE_EXP날짜/조건 미달인증 기준(날짜, 시간, 필수 정보 등)이 충족되지 않았습니다. 유효 기간 내의 사진인지 확인해 주세요.영수증 날짜가 지났거나 스트리밍 시간이 부족한 경우
ETC_INPUT기타 (직접 입력){운영자가 입력한 사유 텍스트 노출}위 4가지 프리셋에 해당하지 않는 특수 상황용

반려사유는 어드민 페이지에서 별도로 관리하며, 상단 내용은 참고용이다.

3.3 [재도전] 버튼 분기 처리

퀘스트 상태[재도전] 버튼 동작
퀘스트 진행중모달 닫기 + CEB-FQ-205 Quest 제출 화면으로 이동 (기존 이미지 클리어 상태)
퀘스트 마감버튼 비활성화 처리

4. 규칙/로직

4.1 모달 닫기 규칙

  • [확인] 버튼 클릭 시 모달 닫힘
  • [재도전] 버튼 클릭 시 모달 닫힘 + CEB-FQ-205로 화면 이동
  • Dimmed 영역 탭 시 모달 닫힘 없음 (버튼으로만 닫기 가능)

4.2 다국어 UI 고려사항

  • 기본 모달 너비: 기기 화면 기준 자동 조절
  • 일본어/영어 문구가 한국어보다 길 수 있으므로, 텍스트 줄바꿈이 발생할 경우 모달 너비를 최소 310px로 설정하여 가독성 확보
  • 반려사유 텍스트가 길 경우 최대 3줄까지 표시 후 말줄임(...) 처리

5. 에러 및 예외상황

5.1 사유 데이터 누락

상황처리
서버 오류 등으로 사유 텍스트가 비어있는 경우기본 메시지 "가이드에 맞지 않았어요. 가이드를 확인하고 다시 도전해 주세요!" 노출

5.2 퀘스트 종료 (Expired)

상황처리
반려 확인 시점에 이미 해당 퀘스트 기간이 종료된 경우[재도전] 버튼 비활성화 처리
