1. 화면 개요

아티스트 탭(CEB-ART-101)에서 "정보" 카드를 탭했을 때 진입하는 화면이다. 아티스트의 소식(뉴스)과 일정을 시간순 통합 타임라인으로 보여주는 정보 허브 역할을 한다.

주요 기능
  • 소식(뉴스)과 일정을 시간순 통합 타임라인으로 표시
  • "오늘 / 이번 주 / 지난 주 / 이전" 날짜 그룹핑
  • 아티스트별 🔔 알림 ON/OFF 설정 기능 (개별 일정 알림 아님, 아티스트 단위)
  • IX 선공개 일정/소식에 "CELEBUS 단독" 뱃지 표시
  • 아티스트 공지는 상단 고정 배너 (등록된 공지가 있을 때만 표시). 서비스(플랫폼) 공지는 알림센터(CEB-NTF-510)에서 처리
접근 경로
  • CEB-ART-101 > 정보 카드 탭
  • CEB-HOM-001 > 바로가기 > 오늘의 일정/소식 카드 탭
화면 레이아웃

화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

2. 영역별 상세

2.1 서브페이지 헤더

UI 구성

항목내용
뒤로가기← 탭 → 이전 화면으로 복귀 (네비게이션 스택 기반)
타이틀"{아티스트명} 정보" (예: V01D 정보)
우측🔔 아티스트별 알림 ON/OFF 토글 버튼. ON: 🔔 보라색 / OFF: 🔕 회색. 현재 선택된 아티스트의 일정/소식 알림을 제어
스타일피그마 참조

동작

  • 🔔 토글 ON → 토스트(Success) "{아티스트명} 알림이 설정되었어요. 새로운 일정과 소식을 알려드릴게요"
  • 🔔 토글 OFF → 토스트(Info) "{아티스트명} 알림이 해제되었어요"
  • 알림 ON 시: 해당 아티스트의 모든 새 일정/소식에 대해 푸시 발송 (D-1 20:00 KST + 당일 00:00 KST)
  • 알림 OFF 시: 해당 아티스트의 푸시 미발송 (다른 아티스트 알림 설정에는 영향 없음)
  • 아티스트 전환 시: 전환된 아티스트의 알림 ON/OFF 상태 반영

  • BO에서 아티스트별 공지 등록/삭제로 관리
  • 아티스트 관련 공지만 표시 (예: "V01D 팬미팅 일정 변경", "V01D 앨범 발매 연기")
  • 서비스(플랫폼) 공지 (예: 서버 점검, 앱 업데이트)는 이 화면에 미표시

공지 노출 위치 정리

- 정보 피드(INF-101): 아티스트 공지만 — 해당 아티스트 탭 진입 시 즉시 인지용 상단 배너

- 알림센터(CEB-NTF-510): 아티스트 공지(팔로우 기준) + 서비스 공지 모두 — 전체 알림 통합

상태별 표시

상태처리
공지 0건해당 아티스트에 등록된 공지 없음: 공지 배너 영역 미노출

3. 공통 규칙

3.1 데이터 갱신

3.2 멀티 아티스트 대응

  • 정보 피드는 아티스트별로 분리하여 표시 (artist_id 기반 조회)
  • 아티스트 전환 → CEB-ART-101 재진입 → 해당 아티스트의 정보 피드 표시

3.3 비로그인 공통

아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역(2.1~2.5)의 "비로그인" 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

  • 비로그인도 열람 가능. 타임라인 전체 표시
  • 비로그인 CTA 탭 → 로그인 화면 이동 (공통정책 10.5 참조)
