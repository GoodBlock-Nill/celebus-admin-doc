1. 화면 목적

아티스트 탭(CEB-ART-101)에서 "Raffle" 카드를 탭했을 때 진입하는 화면이다. 해당 아티스트의 래플 이벤트 목록을 필터별로 표시하고, 각 래플 상세/참여/결과 확인으로 이동하는 허브 화면이다.

주요 기능
  • [진행중] / [마감] 2개 필터로 래플 목록 조회
  • 래플 카드에 경품 이미지 + 경품명 + D-day + 참여자 수 + 내 응모 상태 표시
  • 카드 탭 시 상태에 따라 상세 화면(진행중/당첨/미당첨)으로 이동
  • 상단에 보유 응모권 수 표시
접근 경로
  • CEB-ART-101 > Raffle 카드 탭

3. 화면 내 기능설명

3.1 래플 카드 탭 액션

카드 상태탭 동작
진행중 + 미응모→ CEB-FQ-202 (응모 가능)
진행중 + 응모함→ CEB-FQ-202 (추가 응모 가능)
추첨 대기토스트(Info) "추첨 결과를 기다려주세요"
당첨→ CEB-FQ-203 (당첨 결과 및 수령 안내)
미당첨→ CEB-FQ-204 (위로 메시지)
미참여 마감래플 정보 조회 화면 (읽기 전용)

3.2 필터 전환

  • [진행중] 탭: Active 상태 래플만 표시. 정렬: D-day 임박순 (마감이 가까운 순)
  • [마감] 탭: 추첨 완료 및 미참여 마감 래플 표시. 정렬: 마감일 최신순 (최근 마감 먼저)
  • 탭 전환 시 리스트 즉시 갱신

3.3 보유 응모권 표시

  • 헤더 우측에 "응모권: {N}장" 표시
  • 래플 상세(CEB-FQ-202)에서 응모 후 본 화면으로 복귀 시 최신 수량으로 갱신

3.4 데이터 갱신

트리거동작
화면 진입래플 리스트 및 보유 응모권 수 조회
Pull-to-Refresh전체 데이터 재조회 (공통정책 11.2 참조)
상세 화면 복귀래플 상태 및 보유 응모권 수 재조회
필터 전환해당 필터 리스트 재조회

4. 규칙/로직

4.1 래플 정렬 규칙

필터정렬 기준
진행중D-day 임박순 (마감이 가까운 래플이 상단)
마감마감일 최신순 (가장 최근 마감된 래플이 상단)

4.2 응모권 정책

규칙설명
단위1장 = 1회 응모
소비응모 시 즉시 차감. 취소 및 반환 불가
당첨 확률내 응모 수 / 전체 응모 수 = 기본 당첨 확률
획득 경로Quest 보상, ST 탈락 보상, PM 보상, GP 구매 (100GP = 1장, BO 설정 기준)
아티스트별 분리응모권은 아티스트별 분리 (artist_id 기반)

4.3 D-day 표시 규칙

조건표시
마감까지 2일 이상"D-{N}" (예: D-5, D-12)
마감 당일"오늘 마감"
마감일 지남D-day 미표시 (마감 탭으로 이동됨)

4.4 멀티 아티스트 대응

  • 래플 이벤트는 아티스트별 분리
  • artist_id 기반으로 데이터 조회
  • 아티스트 전환 시 CEB-ART-101 재진입 → 해당 아티스트의 래플 목록 표시

5. 에러 및 예외상황

상태처리
Loading카드 리스트 영역에 Skeleton UI 표시
Default래플 리스트 정상 표시
Error전체 조회 실패 시 에러 안내 메시지 + 재시도 버튼
Empty (진행중 0건)"현재 진행 중인 래플이 없습니다" 안내 + "새 래플이 열리면 알려드릴게요" + [알림 설정] CTA
Empty (마감 0건)"아직 참여한 래플이 없습니다" 안내 + "진행중인 래플에 참여해보세요"
응모권 0장헤더에 "응모권: 0장" 표시. 응모 시도 시 CEB-FQ-202에서 응모 불가 처리
비로그인본 화면은 로그인 상태에서만 접근 가능. CEB-ART-101에서 Raffle 카드 탭 시 로그인 유도 처리됨 (공통정책 10.5 참조)
