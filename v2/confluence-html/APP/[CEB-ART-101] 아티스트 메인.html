1. 화면 목적

유저가 하단 탭에서 아티스트 탭을 탭했을 때 진입하는 메인 화면이다. 아티스트 관련 모든 서비스(미션/내기록/더보기)의 진입점 역할을 한다.

주요 기능
  • 3그룹(미션/내기록/더보기) 카드 그리드로 서비스 한눈에 파악
  • 각 카드에 현재 상태 정보 표시 (Quest 진행률, 덕력 순위, 응모권 수, 팬덤 레벨 등)
  • 카드 탭 시 해당 서비스 화면으로 이동
접근 경로
  • 하단 탭 > {아티스트명} 탭 (예: V01D, iKON)

참고: 홈에서 아티스트를 선택하면 홈 화면 내 콘텐츠만 전환되며, 이 화면(CEB-ART-101)으로 이동하지 않는다. 이 화면은 하단 탭을 통해서만 접근한다.

2. UI 설명

2.1 공통 레이아웃

화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

3. 화면 내 기능설명

3.1 카드 탭

  • 각 카드 탭 시 해당 서비스의 메인 화면으로 이동
  • 이동 화면 목록은 2.2 카드 구성 표의 "탭 액션" 참조
  • 비로그인 유저가 카드 탭 시 → 로그인 화면으로 이동 (공통정책 10.5 참조)
  • Coming Soon 서비스 카드 탭 시 → "곧 만나요! 기대해 주세요" 토스트(Info) 노출, 화면 이동 없음

3.2 상태 정보 갱신

  • 진입 시: 화면 진입 시 각 카드의 상태 정보를 서버에서 조회하여 최신 데이터 표시
  • Pull-to-Refresh: 유저가 화면을 아래로 당기면 전체 카드 상태 정보를 재조회 (공통정책 11.2 참조)
  • 복귀 시 재조회: 서브 화면(카드 탭으로 이동한 화면)에서 이 화면으로 돌아올 때 모든 카드 상태 정보를 재조회하여 최신 상태 반영
  • Quest 진행률, 덕력 순위, 래플 진행 수, 팬덤 레벨 등은 위 시점에 갱신

3.3 그룹 라벨

  • "미션", "내기록", "더보기" 라벨은 정보 구분용으로만 사용
  • 라벨 자체에 탭 이벤트 없음

3.4 스크롤

  • 화면 전체가 세로 스크롤 가능 (카드가 많을 경우)
  • 총 9개 카드(미션 3 + 내기록 4 + 더보기 2)이므로 디바이스에 따라 스크롤 필요

3.5 온보딩 안내 오버레이

  • 신규 유저(활동 데이터가 전혀 없는 첫 진입 시) 대상으로 온보딩 안내 오버레이 표시
  • 오버레이는 각 카드의 기능을 간략히 설명하는 가이드 UI (피그마 참조)
  • 유저가 오버레이를 탭하거나 "닫기" 버튼을 탭하면 오버레이 해제
  • 오버레이 해제 후 재진입 시에는 표시하지 않음 (1회성)

4.2 정렬 기준

  • 그룹 순서: 미션 → 내기록 → 더보기 (고정)
  • 그룹 내 카드 순서: 위 표 순서 그대로 (고정)
  • 미션 그룹 카드 배치: Row 1(챌린지 | 일일미션), Row 2(서포트 — 1열 풀)
  • 내기록 그룹 카드 배치: Row 1(덕력 | 컬렉션), Row 2(응모 | 팬덤 레벨)
  • 더보기 그룹 카드 배치: Row 1(정보 | 기억저장소)

4.3 멀티 아티스트 대응

  • 이 화면은 현재 선택된 아티스트 기준으로 표시
  • 아티스트 전환 시 이 화면의 모든 카드 상태 정보가 해당 아티스트 데이터로 갱신
  • artist_id 기반으로 모든 데이터 조회
