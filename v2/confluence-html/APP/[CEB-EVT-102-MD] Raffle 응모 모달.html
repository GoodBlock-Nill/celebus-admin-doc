1. 화면 개요

Raffle 응모 최종 확인 모달. 응모권 차감 수량을 명확히 고지하여 실수로 인한 재화 소모를 방지하고, [확인] 시 응모 API 호출.

2. UI 설명

+------------------------------+
|  응모하기                     |
|                              |
|  응모권 {N}장이 차감됩니다.    |
|  진행하시겠습니까?             |
|                              |
|  사용된 응모권은              |
|  취소·반환되지 않습니다.       |
|                              |
|  [취소]         [확인]        |
+------------------------------+
항목표시
Container시스템 모달, Dimmed 배경 (클릭 닫힘 차단)

[취소]좌측

시각적 스타일은 피그마 참조

3. 규칙·로직

3.1 호출 조건

  • 호출처: [CEB-EVT-102] Raffle 상세(진행중) — [응모하기] 클릭 시
  • 입력 수량 ≥ 1, 보유량 이하

3.2 [확인] 클릭

  1. 버튼 비활성 + 로딩 (중복 클릭 방지)
  2. 서버 응모 API 호출
결과처리
Success모달 닫힘 → "응모가 완료되었습니다" Success 토스트 → 상세 화면 데이터(내 응모권·통계) 갱신
래플 마감됨모달 본문 변경: "이벤트가 종료되었습니다" + [확인]만 → 이전 화면 복귀
Network Error"네트워크 오류가 발생했습니다. 다시 시도해주세요" Error 토스트 (모달 유지)
기타 실패Error 토스트 + 모달 닫힘

3.3 거래 코드

응모 성공 시 BO [CEB-BO-FQ-301] 응모권 거래 내역에 TICKET_RAFFLE_USE 코드로 기록 ([CEB-BO-006] §7.6).

3.4 [취소] / Dimmed

  • [취소] → 모달 닫힘, 상세 화면 유지
  • Dimmed 배경 → 닫힘 차단
