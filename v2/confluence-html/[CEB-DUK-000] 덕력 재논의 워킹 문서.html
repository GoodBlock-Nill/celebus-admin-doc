0. 배경 및 목적

CELEBUS v2 덕력 시스템은 2026.04.14~23 9일간 집중 정의되었으나 5개 영역에서 정책 미숙성·불일치·미결사항이 식별됨. 본 문서는 5대 영역별 현황 → 옵션 → 결정 → 후속 액션을 한 곳에서 추적하기 위한 워킹 문서.

각 영역마다 의사결정이 완료되면 본 문서에 기록하고, 모든 영역 완료 후 공통 정책 v5.0 및 영향 화면 명세에 일괄 반영한다.

3. 랭킹 동순위 정책 안정화

3.1 현황

  • 2026.04.14: 동률 + 순번 스킵 (1, 1, 3, 4, …)
  • 2026.04.21 (commit 82f348e): "먼저 도달 우선" — 동률 없음 (1, 2, 3, …)
  • CEB-DUK-101 9개 버전 (4/14~21) — 정책 미숙성 신호
  • 동시성 처리 미명시: 밀리초 vs 마이크로초 vs DB 트랜잭션 순서

3.2 옵션

Option A — 현 정책(먼저 도달 우선) 유지 + 정밀 명문화

  • 서버 타임스탬프 기준: 밀리초 (ms)
  • 동일 ms 내 충돌 시: DB 자동 증가 ID 우선
  • 적립 시점 = 이벤트 발생 시점이 아니라 DB 트랜잭션 커밋 시점
  • 운영자 승인이 필요한 활동(Quest 등)은 승인 시각 기준
  • 장점: 결정론적, 구현 명확
  • 단점: 운영자 승인 시점에 의존하는 활동의 공정성 우려 (승인 순서가 영향)

Option B — 동률 처리 (4/14 정책 복구)

  • 동률 시 동순위 부여, 다음 순위 스킵 (1, 1, 3, 4)
  • 장점: 사용자 정서에 부합 (실제로 같은 점수면 같은 순위)
  • 단점: 보상 지급 룰 복잡 (1위 보상을 2명 모두에게? 분할?)

Option C — 동률 처리 + 시즌 보상 분할

  • 동률 시 동순위 + 보상 동률 인원수만큼 분할
  • 장점: 정서적 공정 + 보상 룰 명확
  • 단점: 분할 단위가 작은 경우 보상 의미 약화

3.4 운영 SOP 고려사항

  • 운영자 승인 시각이 순위 결정에 영향 → 승인 처리 SOP 표준화 필수
  • 권장: FIFO(선착순) 승인 처리 — 유저 제출 순서가 자연스럽게 적립 시점에 반영
  • 비정상 지연(시스템 장애 등) 시: 운영팀이 일괄 승인 + 별도 시각 보정 검토 (예외 SOP)
  • 승인 SLA: 영업일 1~2일 (V01D 기존 운영 기준)

3.5 사용자 안내 문구 (UI)

4. 소비 엣지케이스 정의

4.1 현황

다음 6개 시나리오에 대해 시스템 동작 + UI 안내가 미정의.

4.2 6개 케이스 검토

Case (a) 보유 덕력 0DUK에서 응원 시도

  • 현재: 미정의
  • 권장: 응원 버튼 비활성화 + 툴팁 "덕력이 부족해요"
  • 결정 ☐ 권장 / ☐ 기타

Case (b) 응원 처리 중 시즌 초기화 발생

  • 현재: 미정의 (시즌 초기화는 획득 덕력만, 보유 덕력은 영구)
  • 권장: 시즌 초기화는 사전 안내된 시각에 자동. 시각 직전 5분간 응원 입력 잠금 (lock)
  • 결정 ☐ 권장 / ☐ 기타

Case (c) 서포트 응원 → 미달성 → 반환 → 재사용 (이력 트레이스)

  • 현재: "전액 반환" 명시. 이력에 "+500DUK (반환)" 기록
  • 권장: 반환된 덕력은 보유 덕력에 즉시 합산, 다음 응원에 사용 가능. 이력에 "반환 사유: 미달성" 명시
  • 결정 ☐ 권장 / ☐ 기타

Case (d) 독점 콘텐츠 해금 후 환불 정책

  • 현재: "영구 해금" 명시, 환불 정책 없음
  • 권장: 환불 정책 도입 안 함 (해금은 즉시 소비 인정). UI에 "해금 후 환불 불가" 사전 고지
  • 결정 ☐ 권장 (환불 없음) / ☐ 환불 도입 / ☐ 기타

Case (e) 응원 도중 BO에서 덕력 단가 변경

  • 현재: 미정의
  • 권장: 단가 변경은 새 응원부터 적용. 진행 중 응원은 변경 시점의 단가로 잠금
  • 결정 ☐ 권장 / ☐ 기타

Case (f) 다중 응원 동시 클릭 (race condition)

  • 현재: 미정의
  • 권장: 클라이언트 디바운스(0.5초) + 서버 트랜잭션 lock. 동시 요청은 선착 1건만 처리, 나머지 거부 → 토스트 "잠시 후 다시 시도"
  • 결정 ☐ 권장 / ☐ 기타
