1. 화면 개요

기존 에피소드 그룹의 기본 정보를 수정. 상태 전환은 그룹 상세 화면의 헤더 액션으로 분리되어 있어 본 화면에서는 다루지 않음 (다만 정책 안내는 표시).

항목값
화면 경로/sq/groups/{gid}/edit
Breadcrumb홈 / 에피소드 / 그룹 리스트 / {그룹명} / 수정
페이지 제목"에피소드 그룹 수정"
진입 경로[CEB-BO-SQ-201] 그룹 상세의 [그룹 수정] 버튼
접근 권한[관리자 > 권한관리] 설정에 따른다

2. 화면 구성

2-1. 헤더

  • 좌측: ← 뒤로가기 → 그룹 상세 복귀
  • 페이지 제목 "에피소드 그룹 수정"

2-2. 상태별 알림 박스 (조건부)

그룹 상태톤메시지
종료회색"종료된 그룹입니다. 수정 후 상태 전환은 불가합니다."
진행중노랑"진행중 그룹입니다. 수정 사항이 즉시 회원 앱에 반영됩니다."
임시저장(배너 없음)—

2-3. 안내 박스

회색 톤: "한 아티스트당 진행중 그룹은 1개만 허용됩니다. 다른 그룹을 진행중 상태로 두려면 현재 그룹을 종료해주세요."

2-4. 입력 폼 — 기본 정보 (수정 가능)

필드입력 방식옵션·검증
아티스트드롭다운V01D / iKON / CELEBUS (변경 시 진행중 정책 재검증)
타이틀텍스트한국어, 최대 80자

상태는 본 화면에서 직접 수정 불가. 그룹 상세의 [진행중 전환] / [종료 전환] 액션으로만 변경 가능.

2-5. 입력 폼 — 기간

필드입력 방식검증
시작일자날짜 + 시간종료일자보다 이전. 진행중 그룹은 시작일자 변경 시 추가 검증 필요
종료일자날짜 + 시간시작일자보다 이후. 현재보다 과거 종료일은 저장 시 자동 종료 처리 가능

2-6. 관리 정보 (read-only)

생성 관리자 / 생성 일시 / 최근 수정자 / 최근 수정 일시 — 4컬럼 표시.

2-7. 하단 액션 (Sticky 푸터)

  • [취소] → 그룹 상세 (변경사항 있으면 확인 다이얼로그)
  • [저장]

3. 화면에 노출되는 정보 / 입력되는 정보

영역정보의미
상태 알림 박스상태별 안내 메시지운영자 액션 시 영향도 안내
기본 정보아티스트·타이틀수정 가능 핵심 필드

4. 상태별 동작 분기

상황동작
데이터 로딩 중Skeleton 폼
종료 그룹 진입모든 필드 read-only, [저장] 비활성
진행중 그룹 진입노랑 알림 박스 노출, 모든 필드 수정 가능
임시저장 그룹 진입알림 박스 없음, 모든 필드 수정 가능
저장 성공그룹 상세로 복귀 + "그룹 정보가 수정되었습니다." 토스트
권한 없음 진입[관리자 > 권한관리] 설정에 따라 화면 미노출 또는 [저장] 비활성

5. 검증 규칙

케이스동작
시작일자 ≥ 종료일자[저장] 비활성 + 인라인 에러
아티스트 변경 + 새 아티스트에 진행중 그룹 존재 + 현재 상태 진행중경고 박스 + [저장] 차단
그룹이 종료 상태모든 필드 read-only
타이틀 80자 초과입력 차단

5-1. 자동 동작

  • 저장 성공 → 최근 수정자·최근 수정 일시 자동 갱신
  • 종료일자를 현재 시점 이전으로 설정 시 추가 안내: "종료일이 현재 시점보다 이전입니다. 저장 시 그룹이 자동 종료 처리됩니다."
