관련 화면: [CEB-BO-SQ-202] 에피소드 상세

1. 모달 개요

에피소드와 그 안의 모든 하위 미션을 영구 삭제하는 확인 모달. 임시저장 상태에서만 가능.

항목값
트리거에피소드 상세의 [삭제] 액션 (임시저장 상태에서만 노출)
진입 조건에피소드 상태가 "임시저장"
차단 조건진행중·종료 상태는 액션 자체 미노출
접근 권한[관리자 > 권한관리] 설정에 따른다
영향 범위에피소드 + 하위 모든 미션 (cascade) + 팬퀘스트 매핑 자동 해제

2. 화면 구성

2-1. 헤더

  • 모달 제목: "에피소드 삭제" (h3)
  • 우상단: X 닫기 버튼

2-2. 본문

메인 메시지:

이 에피소드와 포함된 미션이 모두 영구 삭제됩니다. 복구할 수 없습니다.

영향 통계 박스 (회색 배경):

  • 이 에피소드에 속한 미션: N개

그룹 영향 안내:

  • 부모 그룹의 에피소드 수 자동 감소
  • 메인 / 반복 슬롯 1개 비워짐 → 다음 에피소드 추가 가능

경고 박스 (조건부):

  • 미션이 1개 이상 있을 때 노란 박스:

> ⚠️ 이 에피소드에 등록된 미션이 있습니다. 모두 함께 삭제됩니다.

버튼색상동작
취소회색 외곽선모달 닫기
삭제빨강 (위험)확정. 외부 클릭으로 닫히지 않도록 차단

3. 화면에 노출되는 정보

위치정보의미
메인 메시지영구 삭제 안내운영자 의도 확인
영향 통계 박스하위 미션 카운트삭제 범위 미리 안내
그룹 영향 안내슬롯 비워짐 안내후속 운영 가이드
경고 박스미션 존재 안내 (조건부)의도치 않은 삭제 방지

4. 상태별 동작 분기

상황동작
임시저장 에피소드 진입모달 정상 노출
진행중·종료 에피소드트리거 액션 자체 미노출 → 모달 진입 불가
삭제 성공부모 그룹 상세로 이동 + "에피소드가 삭제되었습니다." 토스트
삭제 실패 (서버 오류)모달 유지 + 에러 안내
취소모달 닫힘
권한 없음트리거 액션 미노출 또는 [삭제] 비활성

5. 검증 규칙

검증동작
에피소드 상태 ≠ 임시저장모달 진입 불가 (액션 미노출)
미션에 팬퀘스트 매핑 존재모달 본문 추가 안내: "팬퀘스트 매핑이 자동 해제됩니다. (팬퀘스트 Quest 자체는 보존)"

6. 삭제 영향 (cascade)

에피소드 (삭제 대상, 유형: 메인 또는 반복)
  └─ 하위 미션 N개          ← 함께 삭제
       └─ 팬퀘스트 매핑      ← 함께 해제 (팬퀘스트 자체는 보존)

부모 에피소드 그룹
  └─ 에피소드 수 자동 감소
  └─ 메인 또는 반복 슬롯 1개 비워짐

삭제된 에피소드가 점유하던 표시 순서는 슬롯이 비고, 다음 에피소드 추가 시 비어 있는 슬롯을 이어받음.

⚠️ v2.2부터 완료 조건 계층은 폐기되어 미션이 완료 판정을 직접 흡수. 별도 완료 조건 삭제 cascade 없음.

7. 라이프사이클 대안

  • 운영 중인 에피소드(진행중) 정리하려면 → 에피소드 상세의 [종료] 액션으로 종료 전환
  • 임시 보관 → 임시저장 상태로 둠
