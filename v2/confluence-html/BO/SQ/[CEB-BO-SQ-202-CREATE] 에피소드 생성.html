1. 화면 개요

기존 에피소드 그룹에 새 에피소드를 추가하는 폼 화면. 그룹 상세에서 [새 메인 에피소드] 또는 [반복 에피소드 추가] 버튼으로 진입하며, URL 쿼리로 그룹 식별값과 유형(메인/반복)이 전달된다.

2. 화면 구성

2-1. 헤더

  • 좌측: ← 뒤로가기 (취소 확인 모달 후 그룹 리스트로)
  • 페이지 제목 "새 에피소드 생성" + 유형 배지 (메인=인디고 / 반복=바이올렛)
  • 보조 텍스트: "{그룹 타이틀} 그룹에 추가 (메인 N/5)"

2-2. 만석 차단 박스 (조건부 노출)

유형조건메시지

반복그룹에 이미 반복 1개 등록됨"본 그룹은 이미 반복 1개 최대치에 도달했습니다."

2-3. 정책 안내 박스 (인디고 톤)

본 화면에서는 마스터 메타데이터(아티스트·다국어 타이틀·다국어 설명·메인 이미지·표시 순서)만 입력합니다.

기간은 상위 에피소드 그룹에서 관리하므로 별도 기간 입력 필드가 없습니다.

다국어 입력은 한국어/영어/일본어 3개 언어 모두 필수입니다.

  • 드롭 시: 파일 타입 검증 (이미지 외 차단) → 자동 업로드 URL 생성

2-7. 에피소드 완료 보상 (메인 에피소드 전용)

에피소드 안의 모든 미션이 완료된 회원에게 자동 지급되는 보상. 미션 보상과 별도로 추가 지급됩니다. 메인 에피소드에서만 노출되며, 반복 에피소드에서는 본 섹션이 숨겨집니다.

필드입력 방식검증필수
응모권 보상정수 입력 (단위: 장)0 이상 정수. 0이면 응모권 미지급—
덕력 보상정수 입력0 이상 정수. 0이면 덕력 미지급—
BIVE 보상토글 ON/OFFOFF 시 NFT 보상 없음—
민팅 캠페인드롭다운 (BIVE 토글 ON일 때만 노출)운영 중인 BIVE 민팅 캠페인 중 1종 선택BIVE ON 시 ★

모든 값이 0 또는 OFF여도 저장 가능합니다 (보상 없는 에피소드 허용). 보상 지급 시점·정책은 [CEB-BO-011] §5 정책 상수의 "메인 에피소드 완료 보상" 항목 참조.
