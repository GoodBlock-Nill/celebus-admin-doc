1. 화면 개요

기존 미션의 정보를 수정하는 폼 화면. [CEB-BO-SQ-204] 미션 상세의 [수정하기] 버튼을 통해 진입한다. 기존 데이터가 모든 필드에 prefill 되며, 부모 에피소드 상태에 따라 일부 필드가 read-only로 잠긴다 (회원 기대권 보호).

2. 화면 구성

2-1. 헤더

  • 좌측: ← 뒤로가기 (수정 취소 확인 → 미션 상세)
  • 페이지 제목 "미션 수정" + 현재 미션 한국어 타이틀
  • 아티스트 배지 + 상위 에피소드 타이틀
  • 우측: 유형 배지 (팬퀘스트=분홍 / 예측 마켓=앰버 / 서바이벌 트리비아=파랑)

2-2. 상태 배너 (조건부)

부모 에피소드 상태에 따라 상단에 안내 배너가 노출된다.

부모 에피소드 상태배너 톤안내 메시지
임시저장 (DRAFT)미노출—
진행중 (ACTIVE)노랑"진행중 에피소드의 미션입니다. 회원 기대권 보호를 위해 보상 필드(응모권·덕력·BIVE) 는 수정할 수 없습니다."
종료 (CLOSED)회색"종료된 에피소드의 미션은 수정할 수 없습니다." + 모든 필드 read-only + [저장] 비활성

정합 근거: [CEB-BO-SQ-202-EDIT] v3.1 §4 "진행중 진입 시 완료 보상 섹션 read-only 잠금" 정책 차용.

2-3. 팬퀘스트 메타 안내 박스 (유형이 팬퀘스트일 때만 노출)

미션 유형이 팬퀘스트면 상단에 인디고 박스로 안내한다.

이 미션은 팬퀘스트 자체 데이터를 상속받습니다.

- 타이틀·다국어 타이틀·이미지·보상은 [팬퀘스트 영역](/fanquest/{팬퀘스트 식별값})에서만 수정 가능합니다.

- 본 화면에서는 반복 여부·반복 주기·기간·BIVE 토글·민팅 캠페인 등 미션 고유 메타만 수정할 수 있습니다.

- 팬퀘스트 매핑(fanQuestId) 자체는 본 화면에서 변경할 수 없습니다 (재매핑은 미션 삭제 후 재생성).
