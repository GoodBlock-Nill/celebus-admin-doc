1. 화면 개요

검수가 필요한 제출이 1건 이상 있는 팬퀘스트(Quest)를 모아 보여주는 운영 대시보드. SQ 영역에서 팬퀘스트 검수 처리를 빠르게 시작하는 진입점.

항목값
화면 경로/sq/pending
Breadcrumb홈 / 에피소드 / 팬퀘스트 대기내역
페이지 제목"팬퀘스트 대기내역"
진입 경로사이드바 "에피소드 > 팬퀘스트 대기내역"
접근 권한[관리자 > 권한관리] 설정에 따른다
데이터 소스팬퀘스트(FQ) 영역의 검수 대기 수 1건 이상 Quest 목록

ℹ️ 본 화면은 SQ 영역 메뉴에 있지만 데이터는 FQ 도메인. SQ 운영자가 에피소드 운영 관점에서 검수 흐름을 통합 관리할 수 있도록 한다.

2. 화면 구성

2-1. 헤더

  • 페이지 제목 "팬퀘스트 대기내역"
  • 상단 안내 텍스트: "대기 중인 제출이 있는 팬퀘스트를 모아 빠르게 검수할 수 있습니다."

2-2. 필터

필터옵션
아티스트 드롭다운전체 / V01D / iKON / CELEBUS

상태 필터는 자동 적용 — 검수 대기 수 > 0인 팬퀘스트만 노출.

2-3. 테이블

컬럼너비표시 내용
상태100pxQuest 상태 배지
타이틀flexQuest 한국어 타이틀
아티스트100pxV01D / iKON / CELEBUS
진행 방식90px일반 / 반복 배지
반복 주기90px일간 / 주간 / 월간 또는 "—"
검토 필요100px노란 pill 배지로 강조 (검수 대기 수)

행 정렬: 검토 필요 내림차순 (대기 건수 많은 Quest 우선 노출).

행 클릭 동작: 행 전체가 클릭 가능. 클릭 시 해당 팬퀘스트 상세의 대기내역 탭(/fanquest/{식별값}?tab=pending)으로 이동. 별도 "검수" 액션 컬럼·버튼은 두지 않음.

2-4. 페이지네이션

  • 페이지당 20건 (필터 결과 기준)
  • 항상 노출 — 결과가 20건 미만이라 1페이지일 때도 비활성 형태로 노출 ([CEB-BO-011] §5 공통 페이지네이션 정합)
  • 윈도잉 규칙 — 총 페이지 ≤ 7이면 전체 페이지 번호 노출, 8 이상이면 윈도우 + 양 끝 ... 표시
  • 아티스트 필터 변경 시 자동으로 1페이지로 리셋

2-5. 빈 상태

모든 Quest의 검수 대기 수가 0일 때: "검수가 필요한 팬퀘스트가 없습니다." 메시지 노출.
