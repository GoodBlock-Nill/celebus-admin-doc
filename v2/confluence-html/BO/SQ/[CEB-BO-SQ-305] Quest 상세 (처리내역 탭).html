1. 화면 개요

특정 Quest의 검수 처리 이력(승인·반려)을 표 형식으로 노출하고 필터·검색을 제공하는 화면. Quest 상세 3탭 중 처리내역 탭.

2. 화면 구성

2-1. 헤더 (Quest 상세 공통)

[CEB-BO-SQ-303] §2-1·§2-2와 동일.

2-2. 필터·검색 바 (한 줄 가로 배치)

위치요소옵션
좌상태 드롭다운"상태(전체)" (기본) / 승인 / 반려
우검색 대상 드롭다운"닉네임" (기본) / "제출ID"
우키워드 입력부분 일치 검색. placeholder는 검색 대상별 분기
우[초기화] (어두운 회색)모든 필터·검색 리셋

2-3. 처리내역 테이블 (8개 컬럼)

컬럼너비표시 내용
상태80px배지 — 승인=에메랄드 / 반려=빨강
제출ID90px제출 식별값 (강조)
유저닉네임160px회원 닉네임
반려사유120px반려인 경우 사유명, 승인은 "—"
이미지70px (중앙 정렬)확대 아이콘 버튼 (라이트박스용)
제출일시140px회원 제출 시각
처리일시140px운영자 처리 시각
처리자90px처리한 운영자 닉네임

2-4. 페이지네이션

  • 페이지당 10건 (필터·검색 결과 기준)
  • 필터·검색·키워드 변경 시 자동으로 1페이지로 리셋

2-5. 빈 상태

검색 결과 0건 시: "처리내역이 없습니다." 메시지 노출.
