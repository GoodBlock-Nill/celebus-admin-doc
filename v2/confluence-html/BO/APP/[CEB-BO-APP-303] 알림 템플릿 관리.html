비고재사용 템플릿 개념 자체 폐지 — 운영자는 매번 직접 작성. 운영 시작 후 니즈 확인 시 별도 화면 형태로 재도입 검토

작성자: @Nill Yoo · 작성일: 2026-05-20 · 버전: v3.1 (개념 폐지)

1. 폐지 사유

UI/UX 검증 결과 재사용 템플릿 개념 자체가 현재 단계에 오버스펙으로 판단되어 폐지한다.

항목검토 결과
별도 화면 (v3.0까지)사이드바 액션 1개 + 라우트 1개 + CRUD 화면 — 사용 빈도 미검증
폼 안 통합 시도 (v1.6 NotificationForm)폼 상단 [템플릿 불러오기]·[저장] 바를 두면 저장은 가능한데 편집·삭제는 불가한 비정상 상태 — 저장된 템플릿이 누적되어도 운영자가 정리 불가
→ 결론"별도 화면 폐지 + 폼 통합"은 모순. 재사용 템플릿 개념 자체를 가져가지 않고, 운영자는 매번 직접 작성하거나 외부 메모에서 복사·붙여넣기

3. 재도입 조건

운영 시작 후 다음 조건이 검증되면 별도 화면 형태로 재도입을 검토한다.

  • 운영자가 동일·유사 메시지를 주기적으로 (월 N회 이상) 재발송하는 패턴이 확인됨
  • 외부 메모 관리(노션·구글 닥스 등)로는 협업·버전 관리에 한계가 분명함
  • 다국어 3개 언어 묶음을 매번 재작성하는 비효율이 운영자의 명시적 요구로 올라옴
