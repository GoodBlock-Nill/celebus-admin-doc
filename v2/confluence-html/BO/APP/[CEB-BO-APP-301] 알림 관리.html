1. 화면 개요

운영자가 회원 앱(PWA)에 발송되는 알림(기본알림 + 푸시알림 Web Push)을 조회·발송·관리하는 인벤토리 화면. 수동 알림(운영자 캠페인)과 자동 트리거 알림(시스템 발송 로그)의 라이프사이클이 본질적으로 다르므로 같은 사이드바 메뉴 하위에서 2개 탭으로 분리한다.

2-5. 자동 트리거 탭 — 트리거 정책 카탈로그 (조회 전용)

자동 트리거는 시스템 이벤트 발생 시 백엔드가 자동 발송하는 알림으로, 운영자가 신규 등록·메시지 수정·활성 변경을 모두 할 수 없다. 본 탭은 현재 백엔드에 등록된 트리거 정책 목록을 조회 전용으로 표시한다 ([CEB-BO-013] §12·§14-2 정합).

A. 안내 띠

상단 보라색 안내: "자동 트리거는 시스템 이벤트 발생 시 시스템이 자동으로 발송하는 알림입니다. 메시지·대상·활성 상태 변경은 개발팀에 별도 요청해 주세요. 이 화면은 현재 등록된 트리거 정책 목록을 조회만 할 수 있습니다."

5-3. 자동 트리거 탭 제약

자동 트리거는 백엔드 코드/설정에 정의되어 있어 백오피스에서 다음을 할 수 없다. 변경은 모두 개발팀 요청 채널을 통해서만 가능하다 ([CEB-BO-013] §12-1).

제약내용

활성/미사용 토글백오피스에서 불가. 개발팀에 활성/비활성 요청
삭제백오피스에서 불가
가능 액션[상세 미리보기] (다국어 메시지 모달 read-only)만. 자동 트리거 메시지는 변수 토큰을 포함하므로 수동 알림으로 복제하지 않으며, 유사 캠페인이 필요하면 운영자가 [+ 새 알림]에서 빈 폼으로 새로 작성한다
