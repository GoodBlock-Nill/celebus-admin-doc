비고게임존(BO-GAM)은 휴면 deprecation 상태이므로 게임 생성 폼 정정 명세를 RFT 영역에 별도 보관. PM·ST 운영 재개 시점에 BO-GAM-201-CREATE에 흡수

작성자: @Nill Yoo · 작성일: 2026-05-14 · 버전: v1.0 (신규 작성)

본 문서는 PM·ST 게임 생성 화면(BO-GAM-201-CREATE)에 추가될 응모권 지급 토글 운영 명세이다. 게임존 영역이 현재 휴면(PM·ST 운영 0건)이므로 본 정정 명세를 RFT 영역에 임시 보관하며, PM·ST 운영 재개 시점에 게임 생성 폼 본 명세에 흡수한다.

1. 화면 목적

PM·ST 게임마다 운영자가 응모권 지급 여부와 수량을 직접 결정할 수 있도록 게임 생성·수정 폼에 토글을 추가한다. 본 토글은 [CEB-BO-RFT-301] §2-7에 정의된 PM·ST 게임별 응모권 지급 기본 정책 매트릭스의 게임별 오버라이드 수단이다.

항목값
대상 화면BO-GAM-201-CREATE (게임 생성), BO-GAM-202-EDIT (게임 수정)
추가 위치보상 설정 섹션 하단
적용 시점PM·ST 운영 재개 시점 (현재 휴면)
SSOT 기본 정책[CEB-BO-RFT-301] §2-7 PM·ST 게임별 응모권 지급 기본 정책
흡수 시점BO-GAM 휴면 해제 + 게임존 v2 작성 시점

2. 추가 항목 명세

게임 생성 폼 §보상 설정 하단에 다음 영역을 추가한다.

2-1. 응모권 지급 영역 (PM)

요소내용
라벨"응모권 지급"
토글"[● ON] 지급 / [○ OFF] 미지급" (라디오 또는 스위치, 기본값 ON — RFT-301 §2-7 매트릭스 정합)
수량 입력 (ON일 때만 활성)정수 0~100, 기본 0매. 라벨: "정답자 1인당 응모권 수량"
안내 박스"기본값은 ON·0매이므로 운영자가 수량을 입력하지 않으면 실질 지급이 없습니다. 정답자에게 응모권을 지급하려면 1매 이상으로 변경하세요."

2-2. 응모권 지급 영역 (ST)

ST는 생존자·탈락자를 분리해서 입력.

대상토글수량 입력
생존자 응모권"[● ON] / [○ OFF]" (기본 ON)정수 0~100, 기본 0매 (라벨: "생존자 1인당 응모권 수량")
탈락자 응모권"[● ON] / [○ OFF]" (기본 ON)정수 0~100, 기본 0매 (라벨: "탈락자 1인당 응모권 수량")

2-3. 정책 안내 박스 (PM·ST 공통)

ⓘ 본 게임의 응모권 지급 정책
   기본 정책 (모든 게임 공통): [CEB-BO-RFT-301] §2-7 매트릭스 따름
   본 게임 적용 정책: 위 토글·수량으로 오버라이드
   변경 시 활동 로그에 자동 기록됩니다.

3. 검증 규칙

규칙내용
수량 입력정수 0~100. 소수·음수·문자 입력 불가
토글 ON + 수량 0저장 시 안내 토스트 ("0매 지급은 OFF와 동일합니다. 진행하시겠습니까?") + [확인]·[취소] 분기
토글 OFF수량 입력 비활성·회색 처리
게시 후 수정게시 후(Active 상태) 본 토글·수량 수정 가능. 단 종료 시점 이후로는 잠금 (이미 결과 산출 시작)
게시 후 OFF → ON 전환"본 게임에 응모한 회원에게 결과 발표 시 응모권을 지급합니다." 안내 + 변경 사유 입력 (필수)
게시 후 ON → OFF 전환"본 게임의 응모권 지급이 취소됩니다. 이미 안내된 회원에게 혼란을 줄 수 있습니다. 변경 사유를 입력하세요." 안내 + 변경 사유 입력 (필수)

4. 활동 로그

게임 생성·수정 시 본 영역의 변경은 게임존 활동 로그에 자동 기록.

액션로그 메시지
응모권 지급 정책 신규 설정"게임 '{타이틀}'의 응모권 지급 정책을 설정했습니다. (PM 정답자: {지급 여부 + 수량} / ST 생존자: {지급 여부 + 수량} / ST 탈락자: {지급 여부 + 수량})"
응모권 지급 정책 수정"게임 '{타이틀}'의 응모권 지급 정책을 변경했습니다. (변경 항목: {필드 목록}, 사유: {요약})"

5. 응모권 발급 시점

게임 종료 시 시스템이 본 정책을 조회해 응모권을 자동 발급한다.

게임유형발급 시점발급 대상
Prediction Market결과 발표 + 보상 지급 시점토글 ON일 때 정답자에게 설정 수량 지급
Survival Trivia게임 종료 시점생존자 토글 ON이면 생존자에게, 탈락자 토글 ON이면 탈락자에게 각각 설정 수량 지급

응모권 발급 시 출처는 "PM·ST 보상" ([CEB-BO-006] §7.6 정합).

7. 마이그레이션 전략 (PM·ST 휴면 해제 시)

PM·ST 운영이 재개되어 BO-GAM 영역이 v2로 작성될 때 본 정정 명세를 흡수하는 절차:

  1. BO-GAM 신규 작성 시 게임 생성 폼 §보상 설정 하단에 본 §2 항목 그대로 통합
  2. 본 정정 명세([CEB-BO-RFT-302])는 흡수 완료 후 deprecation 처리
  3. v1 휴면 BO-GAM-201-CREATE 배너의 RFT-302 참조 링크 제거
  4. RFT-301 §2-7 안내 박스 갱신: "게임별 오버라이드는 게임 생성 폼에서 설정" → "[CEB-BO-GZ-신규ID] 참조"
