본 문서는 응모권(RFT) 영역의 전체 범위·정책·라이프사이클을 정의한다.

1. 영역 개요

응모권(RFT)은 회원이 래플에 응모하기 위해 사용하는 자산이다. 회원은 퀘스트 보상·PM·ST 보상·BIVE 보유혜택·덕력 랭킹 보상·GP로 응모권 구매 5가지 경로로 응모권을 발급받고, 래플 응모 시 사용한다. 본 영역은 응모권의 발급·보유·사용 흐름을 모니터링하는 운영자용 영역이며, 회원에게 응모권을 직접 지급하거나 회수하는 기능은 포함하지 않는다.

2. 화면 구성

본 영역은 운영 화면 3개 + 정정 명세 1개로 구성된다.

사이드바 배치: UX 편의 사유로 사이드바는 "래플" 영역 하위 메뉴 위치에 배치된다 (래플 응모와 직접 연계되는 자산이므로). 사이드바 메뉴 라벨은 응모권 현황 / 변동 내역 / 응모권 발급 정책 3종. 정정 명세(RFT-302)는 별도 메뉴를 두지 않고 RFT-301 본문에서 링크로 연결한다. 관리·정책 측면에서는 독립 영역으로 운영된다.

3. 권한

본 영역의 모든 화면·액션 권한은 [관리자 > 권한관리]에서 설정된 권한을 따른다. 권한이 없는 운영자가 진입한 경우 해당 화면·버튼은 미노출 또는 비활성 상태로 처리한다. 본 문서에서는 세부 역할 명세를 별도로 두지 않는다.

4. 출처 정의

응모권은 다음 6종 출처 중 하나로 변동 기록된다 (정규 발급 5종 + 사용 1종). 운영 카테고리(환불·보정·회수)는 별도 분류로 §4-3에서 다룬다.

4-1. 정규 발급 출처 5종 (응모권 증가, 회원 노출 카드·필터)

출처의미변동 트리거SSOT 문서
퀘스트 보상회원이 Quest 제출 후 운영자 승인 시 보상 발급Quest 검수 승인[CEB-BO-FQ-203] 처리 내역 §승인
PM·ST 보상게임별 응모권 지급 토글이 켜져 있을 때 종료 시 자동 발급PM 결과 발표 / ST 게임 종료 (생존자·탈락자)[CEB-BO-RFT-301] §4 + [CEB-BO-RFT-302]
BIVE 보유혜택회원이 보유한 BIVE 등급 기반 정기 발급매일 자정 직후 자동[CEB-BO-BIVE-104] §혜택 관리
덕력 랭킹 보상덕력 시즌 정산 시 티어·랭킹 보상으로 자동 발급시즌 종료·정산 시점[CEB-BO-DUK-201] §2.2-C·D + [CEB-BO-DUK-000] §6.2
GP로 응모권 구매회원이 앱 응모권 구매 화면(Buy-RF-Ticket-001)에서 GP를 사용해 직접 구매회원 구매 시점 (앱내 구매 운영 토글이 켜져 있을 때만)[CEB-BO-RFT-301] §3 (앱내 구매 운영 토글)

⚠️ 일일 미션은 응모권을 발급하지 않음 — 일일 미션 보상은 덕력 25로만 지급된다. [CEB-BO-FQ-501] 일일 미션 관리 §2.4 정합. 따라서 응모권 정규 발급 출처에서 제외.

4-2. 사용 출처 1종 (응모권 차감)

출처의미변동 트리거
래플 응모 사용회원이 진행중인 래플에 응모권 사용[CEB-BO-RFL-202] 응모내역 발생 시점

4-3. 운영 카테고리 3종 (정규 출처와 별도 분류)

운영자의 사후 처리·시스템 자동 처리로 발생하는 변동. 회원 노출 분포·필터에서는 "운영" 카테고리로 묶음.

변동 유형의미
래플 취소 환불래플 강제 종료·취소 시 응모자 자동 환불
운영자 보정 적립최고 권한 운영자가 CS 대응으로 수동 환불
운영자 회수최고 권한 운영자가 부정 이용 회수 ([CEB-BO-USR] §7.4 연동)

"만료 소멸"은 운영 카테고리에 포함되지 않는다 — 본 영역은 만료 없음 (영구 보유) 정책(§5). 한 번 발급된 응모권은 사용·소진 전까지 영구 유지된다.

4-4. 발급량·환율·운영 토글

  • 퀘스트 보상 Quest당 발급량, BIVE 혜택 등급별 발급량, 덕력 시즌 티어·랭킹별 보상량은 각 SSOT 문서(§4-1)에서 별도 운영 설정값으로 관리. 본 영역에서는 변경할 수 없음.
  • PM·ST 게임별 응모권 지급 여부 + 수량은 [CEB-BO-RFT-301] §4 발급 정책의 기본값 매트릭스로 SSOT를 두고, 게임별 오버라이드는 [CEB-BO-RFT-302]가 정의하는 게임 생성 폼(BO-GAM-201-CREATE) 토글에서 설정.
  • GP → 응모권 환율, 회원당 1일 교환 한도, 앱내 구매 운영 토글(ON/OFF + 점검 사유)은 [CEB-BO-RFT-301] 응모권 발급 정책 화면에서 운영자가 직접 조정. 변경 즉시 회원 앱에 반영.

5. 정책 상수

응모권 영역 운영 정책은 다음과 같다.

  • 만료 없음 (영구 보유) — 한 번 발급된 응모권은 회원이 사용·소진하기 전까지 영구 유지. 시간 경과로 자동 소멸하지 않음
  • 환수 기능 없음 — 부정 사용·CS 회수 등으로 응모권을 회수해야 하는 경우 운영자가 백오피스에서 직접 처리하지 않고, 데이터팀의 백엔드 직접 보정으로 처리
  • 관리자 수동 지급 없음 — 응모권은 §4-1 정규 발급 출처 5종으로만 발급된다. 이벤트·보상 등의 사유로 운영자가 회원에게 직접 응모권을 지급할 수 없음
  • 변동 데이터 영구 보존 (append-only) — 응모권 변동 로그는 수정·삭제되지 않는다. 모든 발급·사용 이력은 시계열로 영구 보존
  • 공통 페이지네이션 패턴: 모든 리스트 테이블에 페이지당 20건, 항상 노출(1페이지일 때도 ‹ 1 › 비활성). 윈도잉 규칙: 총 페이지 ≤ 7이면 전체 노출, 8 이상이면 1 ... cur-1 cur cur+1 ... last 윈도우 + .... 필터·검색 변경 시 1페이지 자동 리셋 ([CEB-BO-012] §5-2 공통 페이지네이션 정합)
  • 1일 기준 = KST 자정 (00:00 ~ 23:59:59). 회원당 1일 교환 한도는 매일 자정에 자동 리셋. 환율·한도 모두 [CEB-BO-RFT-301]에서 운영자가 조정
  • 앱내 구매 운영 토글 — [CEB-BO-RFT-301] §3에서 운영자가 즉시 ON/OFF. OFF 시 앱 Buy-RF-Ticket-001 화면의 [구매하기] 버튼이 [점검중] 텍스트로 비활성. 변경 즉시 회원 앱에 반영(캐시 TTL 60초)

6. 라이프사이클

응모권은 다음 3상태를 거친다.

[정규 발급 출처 5종]
   ↓ 발급
발급 ───── 회원 보유 ─────→ 보유 ───── 래플 응모 ─────→ 사용
상태의미
발급정규 발급 출처 5종 중 하나가 트리거되어 회원에게 응모권이 지급된 시점
보유발급 후 회원 잔액으로 누적되어 있는 상태 (만료 없음, 영구 유지)
사용회원이 래플 응모에 응모권을 차감한 시점

상태 머신 보조 정책: 정규 발급·사용 모두 시스템 자동 트리거다. 운영자가 상태를 강제로 전환할 수 없다. 운영 카테고리(환불·보정·회수)는 §4-3 별도 트리거. 부정 사용 회수가 필요한 경우 §5 정책 참조.

7. 활동 로그

응모권 변동은 시스템 자동 기록과 운영자 토글 변경 로그 2종으로 구성된다. 회원 자산 변동 로그는 [CEB-BO-RFT-201] 응모권 변동 내역에서 시계열로 조회 가능.

7-1. 자산 변동 로그 (시스템 자동 + 운영 카테고리)

정규 발급 5종 + 사용 1종은 시스템 자동, 운영 카테고리 3종(§4-3)은 운영자·시스템 트리거. 활동 로그는 모두 동일한 변동 로그 스트림에 기록되며 [CEB-BO-RFT-201]에서 조회.
