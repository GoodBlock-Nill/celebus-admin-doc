1. 개요

기억저장소 운영(MEM) 영역은 회원이 작성한 기억(Memory) 게시물의 신고·차단·위반 게시물을 관리. 회원 작성·게시는 v2 APP에서, BO는 사후 운영 (신고 처리·차단 결정).

4. 영역 고유 정책

4.1 신고 라이프사이클

회원 신고 (앱) → Pending (운영자 검토 대기) → 처리 완료 (Approved/Rejected) → 자동 보관 (Archived)
상태의미
Pending신고 접수 후 운영자 검토 대기
Approved신고 인정 — 게시물 차단 + 작성자 제재
Rejected신고 반려 — 게시물 유지
Archived30일 후 자동 보관

4.2 신고 사유 유형

회원 앱에서 선택 가능한 신고 사유 (POLICY 정의):

코드사유
MEM_REPORT_INSULT욕설·비방
MEM_REPORT_SPAM스팸·광고
MEM_REPORT_PORNO음란·선정
MEM_REPORT_PRIVACY개인정보 노출
MEM_REPORT_COPYRIGHT저작권 침해
MEM_REPORT_OTHER기타 (자유 입력)

4.3 신고 처리 정책

OPERATOR가 [신고 인정] 시:

  • 게시물 즉시 차단 (앱 미노출)
  • 작성자에게 제재 검토 알림 ([CEB-BO-USR-201-MD-SANCTION])
  • 신고자에게 처리 완료 푸시 알림
  • 활동 로그 기록

[신고 반려] 시:

  • 게시물 유지
  • 신고자에게 반려 사유 안내
  • 활동 로그 기록

4.4 자동 트리거 ([CEB-BO-002] §7.3 연동)

  • 7일 이내 동일 회원 5건 이상 신고 → 운영자 푸시 알림 ("회원 '{닉네임}' 신고 누적 — 검토 필요")
  • 회원 USR-201 [기본정보] 탭에 "⚠️ 검토 필요" 뱃지 노출

4.5 DUK·EVT 차감 정책

기억 게시 차단 시 회원 DUK·EVT 적립 분 차감 여부:

  • DUK: 차감 X (영구 누적 정책 — 회수는 SUPER 자산 회수 별도)
  • EVT: 차감 X (레벨 다운 없음 — [CEB-BO-EVT-000] §6.4)
  • 단, SUPER가 어뷰징 판단 시 자산 회수 (DUK_RECLAIM, EVT_RESET) 가능
