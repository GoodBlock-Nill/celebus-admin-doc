1. 영역 개요

본 문서는 BIVE 영역의 운영 플로우 SSOT다. [CEB-BO-004] 영역 가이드가 정책·구조를 정의한다면, 본 문서는 실제 운영 흐름(생성→활성→자동 민팅→종료)을 구체적인 단계로 명세한다. 화면 명세에서 반복되는 상태 전이·자동 지급·외부 영역 연동 로직을 본 문서에서 한 번 정의하고, 개별 화면 명세는 본 문서를 참조한다.

3. 운영 계층 요약

3-1. 에디션 (Edition)

항목정의
정의BIVE NFT 묶음의 큐레이션 단위 (예: V01D 시즌 1 에디션)
입력 데이터다국어 명칭 (한국어/영어/일본어 3종) — 그 외 없음
수정·삭제 조건등록된 BIVE 0건 + 민팅 기록 0건일 때만 가능
상태별도 상태 없음 (존재 / 삭제)

3-2. BIVE (NFT)

항목정의
정의실제 NFT 단위. 아티스트 × 등급 × 등급번호로 식별
명칭{아티스트명} {등급}-{등급번호} 자동 생성 (예: 하시모토 칸나 Normal-001)
등급 6종Event / Normal / Special / Wicked / Next / Final (고정)
등급번호최대 5자리 숫자 (예: 001, 12345)
고유성동일 에디션 내 (아티스트, 등급, 등급번호) 조합 유니크
미디어PNG·JPG·GIF·MP4 / 최대 20MB / 필수
기능 토글 3종보내기(Send) / Mix / Pick — 기본 ON, 개별 ON/OFF
상태 3종Draft / Active / Inactive

3-3. 민팅 이벤트 (Mint Campaign)

항목정의
정의회원 액션 발생 시 자동으로 BIVE를 발행하는 트리거 단위
유형 4종Event / Ticket / Mix / Pick (고정)
연결 기능캠페인 ↔ 연결 기능 = 1:1. 개발 프리셋만 사용 (예: 회원가입 보상 받기)
BIVE 보상N:M 관계 — 캠페인이 보상으로 여러 BIVE 선택. 가중치 1~999 정수
비중 (확률)가중치 / 가중치합 × 100% — 추첨 시 확률 분포
상태 4종초안 / 활성 / 중지 / 종료
활성화 선행 조건기본정보 + BIVE 보상 저장 완료 + BIVE 보상 ≥ 1건

3-4. 혜택 (Benefit)

항목정의
정의BIVE 보유자에게 별도로 지급되는 추가 보상
유형 2종Boost Point (BP) / Raffle Ticket (RT)
지급주기 3종일일 / 주간 / 1회(획득 시)
지급 시점일일·주간 = 매일 00:05 배치. 1회(획득 시) = 사용자가 대상 BIVE 획득 즉시 (또는 짧은 지연 배치)
멱등성동일 혜택/사용자/일(또는 지정 주기 키) 중복 지급 방지
상태 5종초안 / 대기 / 활성 / 중지 / 종료
활성/대기 전환 선행 조건등록된 BIVE ≥ 1 + 수량 ≥ 1 + 지급주기별 필수 필드

4. 상태 머신

4-1. 에디션 상태 (2상태)

존재 ────────────────► 삭제
       (BIVE 0 + 민팅 0일 때만)

4-2. BIVE 상태 (3상태)

┌────────┐   활성화    ┌────────┐  비활성   ┌──────────┐
│ Draft  │ ─────────► │ Active │ ────────►│ Inactive │
│ (초안) │            │ (활성) │           │ (비활성) │
└────────┘            └────────┘           └──────────┘
    ↓                                          │
   삭제                                  ┌─────┘
                                         ▼ 재활성
                                    ┌────────┐
                                    │ Active │
                                    └────────┘

조건:
- Draft → Active: 운영자 활성화 (필수 정보 모두 입력)
- Active → Inactive: 연결된 민팅 이벤트가 없거나 모두 종료 상태일 때만
- Inactive → Active: 운영자 재활성화
- Active 후 기본정보·기능 수정 불가 (모두 잠금)
- Inactive 상태 BIVE는 민팅 이벤트 보상으로 신규 선택 불가

4-3. 민팅 이벤트 상태 (4상태)

┌────────┐   활성화    ┌────────┐  일시중지  ┌────────┐
│  초안  │ ─────────► │  활성  │ ─────────► │  중지  │
│        │            │        │            │        │
└────────┘            └────────┘ ◄───────── └────────┘
    ↓                                 활성화
   삭제                                    
                                     │
                                     ▼ (개발팀 수동)
                                ┌────────┐
                                │  종료  │
                                └────────┘

조건:
- 초안 → 활성: 활성화 (BIVE 보상 ≥ 1 + 저장 완료)
- 활성 → 중지: 일시중지 (회원 액션 발생해도 민팅 안 됨)
- 중지 → 활성: 재활성화
- 활성/중지 → 종료: 개발팀 수동 처리 (자동 도달 아님)
- 초안 → 삭제: 가능
- 활성 → 초안: 불가 (되돌리기 금지)
- 종료: 완전 읽기 전용

4-4. 혜택 상태 (5상태)

┌────────┐ 활성대기   ┌────────┐  자동활성   ┌────────┐  일시중지 ┌────────┐
│  초안  │ ─────────► │  대기  │ ──────────► │  활성  │ ────────► │  중지  │
│        │ ◄───────── │        │             │        │ ◄──────── │        │
└────────┘  대기취소  └────────┘             └────────┘   활성화   └────────┘
    ↓                                            │                     │
   삭제                                           │ 종료일시            │ 종료일시
                                                  ▼ 자동 도달          ▼ 자동 도달
                                            ┌──────────────────────────┐
                                            │           종료           │
                                            └──────────────────────────┘

조건:
- 초안 → 대기: 활성대기 버튼 (필수 정보 모두 입력)
- 대기 → 초안: 대기취소 버튼
- 대기 → 활성: 시작일시 도달 시 자동
- 활성 → 중지: 일시중지 버튼
- 중지 → 활성: 활성화 버튼
- 활성/중지 → 종료: 종료일시 자동 도달
- 중지 상태에서 종료일·종료시간만 수정 가능
- 종료: 완전 읽기 전용

5. 정책

5-1. 민팅 이벤트 SSOT 정책

본 영역 [CEB-BO-BIVE-103]에서 모든 민팅 이벤트를 마스터 관리한다. 외부 영역(SQ/RFL/FQ)은 다음을 따른다:

  • 외부 영역에서 캠페인을 신규 정의할 수 없음
  • 외부 영역은 본 영역에 등록된 "활성" 상태 캠페인만 드롭다운으로 참조
  • 캠페인 상태가 "중지"·"종료"로 전환되면 외부 영역에서도 자동으로 사용 중지

5-2. 자동 민팅 트리거 메커니즘

트리거발생 시점동작
SQ 에피소드 완료회원이 미션 완료 시 (biveRewardYn ON일 때)SQ-202 지정 캠페인의 BIVE 보상에서 가중치 기반 추첨 → 1개 BIVE 발행
RFL 당첨 확정추첨 완료 후 즉시RFL-201 지정 캠페인의 BIVE 보상에서 가중치 기반 추첨 → 당첨자에게만 발행
FQ Quest 승인운영자 승인 시(정책 미정 — 후속 정의)
Event 캠페인 연결 기능회원의 등록된 연결 기능 발생 (예: 회원가입)즉시 자동 민팅
Ticket 캠페인 연결 기능공연 종료 다음날 자동 에어드랍배치 처리
Mix/Pick 캠페인회원이 BIVE를 Mix/Pick 액션 수행 시즉시 자동 민팅

5-3. 자동 민팅 추첨 로직

1. 캠페인 활성 상태 + 회원 액션이 연결 기능과 일치 → 민팅 대상 회원 확정
2. 캠페인 BIVE 보상 목록 조회
3. 각 BIVE의 가중치를 합산 (가중치합)
4. 0 < r ≤ 가중치합 사이 난수 r 생성
5. 가중치 누적합 기준으로 r이 속한 BIVE 선택 → 발행
6. 회원 지갑주소로 BIVE NFT 트랜잭션 발행 (BSC)
7. 활동 로그 기록 + 발행 카운트 +1

5-4. 혜택 자동 지급 배치

[매일 00:05]
1. 활성 상태 혜택 모두 조회
2. 각 혜택의 지급주기별 처리:
   - 일일: 모든 활성 혜택 → 등록된 BIVE 보유 회원에게 1회 지급
   - 주간: 오늘이 설정 요일이면 → 등록된 BIVE 보유 회원에게 1회 지급
3. 멱등성 체크: 동일 혜택/사용자/일(또는 주기 키) 중복 지급 방지
4. BP 또는 RT 지급 트랜잭션 발행
5. 활동 로그 기록

[1회(획득 시)]
1. 회원이 활성 혜택의 등록된 BIVE를 획득하는 이벤트 발생
2. 즉시 또는 짧은 지연 배치로 1회 지급
3. 이미 지급 완료된 사용자에 대해서는 재지급 금지

5-5. BIVE 비활성화 조건

활성 상태 BIVE를 비활성으로 전환하려면:

  • 본 BIVE가 보상으로 연결된 모든 민팅 이벤트가 "종료" 상태여야 함
  • 활성 캠페인 또는 중지 캠페인에 연결되어 있으면 비활성 불가
  • 비활성된 BIVE는 신규 캠페인의 BIVE 보상 추가 모달에서 선택 불가 (단, 기존 활성 캠페인에는 계속 사용 가능)

5-6. 캠페인 BIVE 보상 제거 정책

활성 → 일시중지 후 BIVE 보상에서 특정 BIVE를 제거할 때:

BIVE의 본 캠페인 민팅 수동작
1 이상보상 내역 테이블에서 행 유지 + 가중치/비중 "-" 표시 + 가중치합 계산에서 제외
0보상 내역 테이블에서도 행 제거

6. 사용자 플로우

6-1. 에디션 생성 → BIVE 등록 → 활성화 플로우

1. 운영자 사이드바 → BIVE > 에디션 관리 → [CEB-BO-BIVE-101]
2. [+ 에디션 생성] 클릭 → 생성 모달 → 다국어 명칭 3종 입력 → 생성하기
   → 에디션 초기 상태로 리스트 추가
3. 에디션 행 클릭 → [CEB-BO-BIVE-102] BIVE 관리 진입
4. [+ BIVE 등록] 클릭 → [CEB-BO-BIVE-202-CREATE] BIVE 등록 진입
   → 아티스트·아티스트·등급·등급번호·미디어·기능 토글 입력 → 등록
   → Draft 상태로 BIVE 리스트 추가
5. BIVE 행 클릭 → [CEB-BO-BIVE-202] BIVE 상세 진입 → 기본정보·기능 확인
6. [활성화] 버튼 클릭 → 활성화 모달 → Active 상태 전환
   → 이 시점부터 민팅 이벤트 BIVE 보상으로 등록 가능

6-2. 민팅 이벤트 생성 → BIVE 보상 추가 → 활성화 플로우

1. 운영자 사이드바 → BIVE > 민팅 관리 → [CEB-BO-BIVE-103]
2. Event 탭에서 [+ 캠페인 생성] 클릭 → [CEB-BO-BIVE-203-CREATE] 진입
3. 기본정보 탭에서 캠페인명 + 연결 기능 입력 → BIVE 보상 탭 이동
4. [+ 추가하기] 클릭 → [CEB-BO-BIVE-203-MD-ADD] 모달 오픈
   → 에디션·아티스트·아티스트·등급 필터 선택 → 활성 BIVE 다중 선택 → 추가하기
   → 본 화면 테이블에 BIVE 행 추가
5. 각 BIVE의 가중치 1~999 정수 입력 → 비중 자동 계산
6. [생성하기] 클릭 → 캠페인 초안 상태로 [CEB-BO-BIVE-103] 리스트 추가
7. 캠페인 행 클릭 → [CEB-BO-BIVE-203] 상세 진입 → [활성화] 버튼
   → 활성화 모달 확인 → Active 상태 전환
   → 이 시점부터 회원의 연결 기능 발생 시 자동 민팅 발생

6-3. 자동 민팅 발생 플로우 (회원 액션 → BIVE 발행)

1. 회원이 앱에서 캠페인의 "연결 기능" 발생
   (예: 회원가입 / 출석체크 5일 / 공연 티켓 검증 등)
2. 시스템이 활성 캠페인 조회 → 연결 기능 일치 캠페인 확정
3. 캠페인 BIVE 보상 목록 + 가중치 조회
4. 가중치 기반 추첨 → 1개 BIVE 선택
5. 회원 지갑주소로 BIVE NFT 트랜잭션 발행 (BSC)
6. 활동 로그 기록: "민팅 이벤트 '{캠페인명}' 자동 민팅 — 트리거: {연결 기능명}, 대상 회원: {닉네임}, 발행 BIVE: '{BIVE 명칭}'"
7. BIVE 상세 §민팅이력 탭에 신규 행 추가 (Token ID·닉네임·지갑주소·민팅일시)

6-4. 혜택 생성 → 활성대기 → 자동 활성 플로우

1. 운영자 사이드바 → BIVE > 혜택 관리 → [CEB-BO-BIVE-104]
2. Boost Point 탭에서 [+ 혜택 생성] 클릭 → [CEB-BO-BIVE-204-CREATE] 진입
3. 기본정보 탭에서 혜택명·BP 수량·지급주기·요일(주간)·시작/종료 일시 입력
4. BIVE 추가 탭에서 [+ 추가하기] 모달 → 활성 BIVE 다중 선택 → 추가
5. [생성하기] 클릭 → 혜택 초안 상태로 [CEB-BO-BIVE-104] 리스트 추가
6. 혜택 행 클릭 → [CEB-BO-BIVE-204] 상세 진입 → [활성대기] 버튼
   → 활성대기 모달 확인 → 대기 상태 전환
7. 시작일시 도달 시 → 자동으로 활성 상태 전환
   → 매일 00:05 배치 (일일/주간) 또는 회원 BIVE 획득 시 (1회)에 자동 지급 발생

6-5. 혜택 자동 지급 배치 플로우 (매일 00:05)

1. 매일 00:05 배치 스케줄러 트리거
2. 활성 상태 혜택 모두 조회
3. 각 혜택별 처리:
   a. 일일 혜택: 등록 BIVE 보유 회원 → BP/RT 지급
   b. 주간 혜택: 오늘이 설정 요일이면 → BP/RT 지급
4. 멱등성 키 체크: 동일 혜택/사용자/일 중복 지급 방지
5. BP/RT 지급 트랜잭션 + 활동 로그 기록
6. 종료일시 도달한 혜택은 자동으로 종료 상태 전환

6-6. 캠페인 종료 플로우 (개발팀 수동 처리)

1. 운영자가 캠페인을 더이상 사용하지 않을 때
2. 운영자 → 개발팀에 종료 요청 (별도 절차)
3. 개발팀이 캠페인 상태를 종료로 수동 전환
4. 종료 후:
   - 회원 액션 발생해도 민팅 안 됨
   - 캠페인 정보 완전 읽기 전용
   - 연결된 BIVE는 다른 활성 캠페인에서 계속 사용 가능
   - 본 캠페인 보상 내역은 그대로 보존 (이력 추적용)
