1. 화면 개요

신규 민팅 이벤트을 생성한다. 기본정보 + BIVE 보상 2탭 구조이며, 모두 입력 후 [생성하기] 클릭 시 초안 상태로 [CEB-BO-BIVE-103]에 추가된다.

2. 화면 구성

2-1. 헤더 (공통)

  • 좌측: 페이지 타이틀 "{Event/Ticket} 캠페인 생성"
  • 우측 액션:
  • [취소하기]: [CEB-BO-BIVE-103]으로 이동 (입력 폐기)
  • [생성하기] (Primary): 검증 통과 시 토스트 "생성 완료" → [CEB-BO-BIVE-103]으로 이동 + 초안 상태로 행 추가
  • 탭 네비게이션: "기본정보 | BIVE 보상" (현재 탭 활성)

2-2. 기본정보 탭

  • 안내 문구: "캠페인 식별을 위한 캠페인 명을 입력하고, 연결기능을 선택해 주세요."
필드입력 방식검증·옵션필수

2-3. BIVE 보상 탭

보상 방식 세그먼티드 컨트롤 (v1.3 신규, 탭 최상단):

  • 가중치 보상 (WEIGHTED) — 기본. 트리거 → 가중치 분포 추첨 → BIVE 1개 발행
  • 지정 보상 (FIXED) — 트리거 → 등록된 모든 BIVE(1~10종) 동시 발행
  • 캠페인당 1개 선택 (XOR). 생성 화면에서는 자유롭게 전환 가능 (캠페인 생성 후 §2-3 상태별 매트릭스에 따라 잠금)

안내 분기:

  • 가중치: "보상으로 지급되는 BIVE를 추가하고 각 항목에 가중치를 입력하세요."
  • 지정: "캠페인 트리거 시 등록된 모든 BIVE가 동시에 지급됩니다. 최대 10종까지 등록 가능합니다."

우측 상단:

  • 가중치 보상: 카운터 "가중치 합: n"
  • 지정 보상: 카운터 "등록 BIVE: n / 10"
  • [+ 추가하기]: 생성 화면 단계에서는 비활성 ("캠페인 생성 후 BIVE 보상을 추가할 수 있습니다." 안내). 캠페인 생성 직후 [CEB-BO-BIVE-203] §2-3에서 보상 방식에 맞춰 추가

테이블 컬럼 (보상 방식에 따라 분기):

  • 공통: BIVE 명칭 / 아티스트 그룹 / 아티스트 / 등급 / 등급번호 / 관리 (X)
  • 가중치 보상에만 추가: 가중치 (입력) / 비중 (읽기)

가중치 입력 (가중치 보상 전용):

  • 정수만, 1~999 허용 범위
  • placeholder "1-999"
  • 입력 즉시 가중치 합·각 행 비중 재계산
  • 비중 표기: 가중치 / 가중치합 분수 (예: 10/55)
  • (선택) 백분율 보조 표기: (가중치 ÷ 가중치합 × 100)%

행 삭제 (X 아이콘): 즉시 행 제거 + 가중치 합/비중 재계산

빈 상태: "+ 추가 버튼을 눌러 BIVE를 등록하세요."
