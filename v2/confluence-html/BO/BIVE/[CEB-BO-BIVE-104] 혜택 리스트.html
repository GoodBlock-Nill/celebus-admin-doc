1. 화면 개요

BIVE 보유 조건에 따라 회원에게 지급되는 혜택(Boost Point + Raffle Ticket)을 유형별로 조회·생성·상태 관리하는 화면. 일일·주간·1회(획득 시) 형태의 지급 정책을 운영자가 정의하고, 매일 00:05 배치로 자동 지급된다.

2. 화면 구성

2-1. 상단 탭

탭의미
Boost Point (BP, 기본)회원에게 Boost Point 지급. SQ/RFL 등에서 보상 배수 강화 등에 사용
Raffle Ticket (RT)회원에게 응모권(Raffle Ticket) 지급. RFL 영역에서 응모 시 사용

2-2. 필터·검색 영역

  • 상태 필터 (드롭다운): 전체(기본) / 초안 / 대기 / 활성 / 중지 / 종료
  • 지급주기 필터 (드롭다운): 전체(기본) / 일일 / 주간 / 1회(획득 시)
  • 검색 입력: placeholder "혜택 명칭 입력" — 키워드 부분 일치
  • [조회] 버튼: 현재 탭 + 조건으로 재조회
  • [+ 혜택 생성] 버튼 (우측 상단): 현재 탭의 혜택 생성 페이지로 이동
  • Boost Point 탭 → [CEB-BO-BIVE-204-CREATE] (BP 변형)
  • Raffle Ticket 탭 → [CEB-BO-BIVE-204-CREATE] (RT 변형)

4-1. 혜택 상태 정의 + 전이

상태의미전이 가능
