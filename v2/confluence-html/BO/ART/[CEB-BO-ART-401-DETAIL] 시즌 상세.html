1. 화면 목적

운영자가 시즌(1년) 내 매월 별도의 덕력 랭킹 보상을 설정하는 화면이다. 12개월 각각에 대해 보상 구간을 자유롭게 구성하고, 한 구간 안에 5종(배송 수령·현장 수령·BIVE NFT·응모권·덕력) 상품을 자유롭게 배치할 수 있다.

  • 누가: 덕력 운영 권한이 있는 백오피스 관리자
  • 왜: 매월 말일 자동 정산 시 회원에게 지급될 보상을 사전 설정해야 함
  • 결과: 미정산 월의 구간·상품 설정이 다음 정산 시점부터 자동 적용됨

핵심 규칙

  • 편집 가능 조건은 단 두 가지: ① 예정 시즌의 모든 월 OR ② 진행중 시즌의 이번 달·미래 월
  • 그 외 모든 경우는 잠금 (조회만)
  • 진행중 시즌의 지난 월은 정산 완료 여부와 무관하게 잠금 — 자동 정산이 지연되더라도 운영자 임의 변경을 차단하는 안전망
  • 종료 시즌은 정산 완료·미정산 모두 잠금 — 신규 적립·소비가 일어나지 않으므로 수정 무의미 ([CEB-BO-ART-401-MD-CLOSE] §4.4 정합)

잠금 상태 컨트롤 동작 (위 표에서 "조회만"인 모든 행)

  • [+ 구간 추가]·[구간 삭제]·[+ 상품 추가]·[상품 삭제]·[월 보상 저장] 모두 미노출
  • 기존 입력 컨트롤(선택 메뉴·입력란·날짜·시간·숫자 입력)은 회색 처리 + 수정 불가
  • 다국어 입력 영역도 회색 처리. 탭 전환은 조회 목적으로 허용하되 입력은 차단
  • 보존된 기존 구간·상품 데이터는 그대로 노출되어 운영자가 정책 이력을 확인할 수 있도록 함

2.6 구간(대상 기준) 입력

각 월 카드 내부에 구간을 자유롭게 N개 추가한다. 한 구간 = 대상 기준 1개 + 보상 상품 N개 묶음.

본 절의 모든 추가·수정·삭제 액션은 §2.5 매트릭스에서 편집 가능한 월 카드 안에서만 노출된다. 잠금 월 카드는 §2.5 끝의 잠금 상태 컨트롤 동작 규칙을 따른다.

2.8 다국어 입력 정책

  • 모든 상품의 상품명·장소·지참물은 [CEB-000] §5 다국어 정책에 따라 한국어·영어·일본어 3언어 필수 입력
  • 다국어 입력 영역은 백오피스 공통 컴포넌트를 그대로 사용 (래플·기억저장소·아티스트 화면 통일)
  • 미입력 언어는 다국어 탭 우상단에 빨간 점 알림으로 표시
  • 저장 시 모든 언어 입력이 완성되어 있어야 함

2.9 시각적 스타일

  • 잠금 월 카드 (§2.5 매트릭스 "조회만" 행): 잠금 아이콘 + 회색 음영 (수정 불가 인지)
  • 편집 가능 월 카드: 흰 배경 + "수정 가능" 배지
  • 아코디언 헤더 hover 시 강조
  • 빈 상태 (구간 0건, 편집 가능 카드): "보상 구간이 없습니다. [+ 구간 추가]로 시작하세요."
  • 빈 상태 (구간 0건, 잠금 카드): "설정된 보상 구간이 없습니다."

3. 기능 설명

운영자가 이 화면에서 할 수 있는 동작:

  1. 시즌 정보 조회 — 그룹·시즌명·기간·상태·정산 완료 카운트 확인
  2. 시즌 수정·종료 — 상단 액션으로 시즌 자체 정보·상태 변경 (시즌 상태별 노출 분기)
  3. 월별 보상 펼침/접기 — 12개월 아코디언 자유 토글 (§2.4 자동 펼침 정책으로 1개월 자동 펼침)
  4. 편집 가능 월 보상 편집 — §2.5 매트릭스의 "편집 + [월 보상 저장]" 행에 해당하는 월(예정 시즌 전 월 OR 진행중 시즌의 이번 달·미래 월)에서 구간·상품 추가·수정·삭제·종류 전환
  5. 월 보상 저장 — 한 월 단위로 구간·상품 일괄 저장
  6. 잠금 월 조회 — 정산 완료 월·진행중 시즌의 지난 월·종료 시즌의 전 월은 조회만 가능
  7. 목록으로 복귀 — 시즌 리스트로 돌아가기

4. 로직·플로우

4.1 화면 진입 → 초기 표시

  1. 시즌 리스트에서 시즌 행 클릭으로 본 화면 진입
  2. 시즌 정보 카드 + 12개월 아코디언 표시
  3. §2.4 자동 펼침 정책에 따라 1개월만 자동 펼침, 나머지 11개월은 접힌 상태
  • 예정 시즌 → 시즌 첫 월
  • 진행중 시즌 → 이번 달
  • 종료 시즌 → 정산 완료된 마지막 월 (정산 완료 0개월이면 시즌 첫 월)
  1. §2.5 매트릭스에 따라 각 월 카드의 잠금·편집 가능 표시 분기

4.2 월 카드 펼침/접기

  1. 접힌 헤더 클릭 → 본문 펼침
  2. 펼친 헤더 클릭 → 본문 접힘
  3. 접힌 상태에서도 헤더 요약 ("구간 N개 · 상품 M개")으로 현황 파악 가능

4.3 편집 가능 월 — 구간 추가

§2.5 매트릭스 "편집 + [월 보상 저장]" 행에 해당하는 월에서만 동작.

  1. [+ 구간 추가] 클릭 → 빈 구간(상품 0개) 1건 추가
  2. 대상 기준 미선택 + 값 빈 상태로 추가됨
  3. 한 구간 안 최소 1개 상품이 필수이므로, 빈 구간만 추가하고 저장하면 차단됨

4.4 편집 가능 월 — 구간 안 상품 추가

  1. 구간 내부 [+ 상품 추가] 클릭 → 빈 상품 1건 추가 (기본 종류: 배송 수령)
  2. 종류 선택 메뉴 변경 시 종류별 폼이 분기되어 노출 (이전 입력값은 폐기)

4.5 편집 가능 월 — 저장

  1. [월 보상 저장] 클릭
  2. 해당 월의 모든 구간·상품 검증 (대상 기준·값 형식·상품 종류·종류별 필드·다국어 완성)
  3. 검증 통과 시 토스트 "월별 보상이 저장되었습니다." + 월 카드 갱신
  4. 검증 실패 시 첫 오류 위치로 스크롤 + 인라인 에러 메시지

4.6 잠금 월 — 조회만

§2.5 매트릭스 "조회만" 행에 해당하는 모든 월(정산 완료 / 진행중 시즌 지난 월 / 종료 시즌의 전 월).

  1. 잠금 상태로 기존 구간·상품 노출 (모든 입력 컨트롤 수정 불가)
  2. [+ 구간 추가]·[+ 상품 추가]·[구간 삭제]·[상품 삭제]·[월 보상 저장] 모두 미노출
  3. 잠금 사유에 따른 안내 노출:
  • 정산 완료: "정산 완료 (yyyy.mm.dd HH:mm)"
  • 진행중 시즌 + 지난 월 + 미정산: "지난 월 — 수정 불가"
  • 종료 시즌 + 미정산: "시즌 종료로 인해 수정 불가"

4.7 상단 [수정] 클릭

  1. 시즌 생성·수정 모달 [CEB-BO-ART-401-MD-SEASON] 열림 (수정 모드)
  2. 시즌명·시작일시 변경 → [저장] 시 토스트 "시즌이 저장되었습니다." + 본 화면 갱신
  3. 진행중·종료 시즌은 시작일시 수정 불가 (시즌명만 변경 가능)

4.8 상단 [종료] 클릭

  1. 시즌 종료 확인 모달 [CEB-BO-ART-401-MD-CLOSE] 열림
  2. [종료 확인] 시 시즌 상태 "종료"로 전환 + 토스트 "시즌이 종료되었습니다." + 본 화면 갱신

4.9 상단 [목록으로] 클릭

  1. 랭킹 시즌 설정 [CEB-BO-ART-401-SEASON]로 이동
  2. 본 화면의 아코디언 펼침 상태는 보존하지 않음 (재진입 시 §2.4 자동 펼침 정책 재적용)

5. 검증·예외 상황

5.1 입력 검증

항목룰
구간 — 대상 기준등수 / 퍼센트 / 등수범위 중 1종 필수
구간 — 값등수: 양의 정수 / 퍼센트: 1~100 / 등수범위: N-M(N
구간 — 상품 수한 구간 최소 1개 필수. 0개면 [월 보상 저장] 차단
상품 — 종류배송 수령 / 현장 수령 / BIVE NFT / 응모권 / 덕력 중 1종 필수
상품 — 상품명한국어·영어·일본어 모두 입력 (각 50자 이하)
배송 수령 — URL올바른 URL 형식. 잘못된 형식 시 인라인 에러 "올바른 URL 형식이 아닙니다"
현장 수령 — 기간수령 시작일 ≤ 수령 종료일
BIVE NFT — 민팅 이벤트디폴트 미선택. 미선택 상태 저장 시 차단 ("민팅 이벤트를 선택해주세요")
응모권 / 덕력 — 수량양의 정수

5.2 비즈니스 규칙

  • 편집 가능 조건: ① 예정 시즌의 모든 월 OR ② 진행중 시즌의 이번 달·미래 월. 이 조건에 해당하는 월만 구간·상품 추가·수정·삭제 + [월 보상 저장] 가능 (§2.5 매트릭스)
  • 지난 월 잠금 (안전망): 진행중 시즌에서 현재 시각보다 과거 년월은 정산 완료 여부와 무관하게 모두 조회만 가능. 자동 정산이 지연되더라도 운영자 임의 변경 차단
  • 정산 완료 월 잠금: [CEB-000] §5.2.1 월간 정산 SSOT — 매월 말일 23:59 서버 자동 정산. 정산 완료 후에는 보상 변경 불가 (이력 무결성)
  • 종료 시즌 전체 잠금: 시즌이 종료되면 모든 월(정산 완료·미정산 무관) 조회만 가능. 미정산 월의 보상 설정은 데이터로 보존되나 신규 적립·소비가 일어나지 않음 ([CEB-BO-ART-401-MD-CLOSE] §4.4 정합)
  • 빈 보상 (구간 0건): 보상 미설정 월은 정산 시 보상 미지급
  • 저장 즉시 적용: 저장 성공 시 다음 정산 시점부터 새 구간·상품 자동 적용

5.3 빈 상태

상황표시
편집 가능 월 구간 0건"보상 구간이 없습니다. [+ 구간 추가]로 시작하세요."
잠금 월 구간 0건"설정된 보상 구간이 없습니다." (수정 불가)

5.4 권한별 동작

  • 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다
  • 쓰기 권한 없으면 [수정]·[종료]·[+ 구간 추가]·[+ 상품 추가]·[구간 삭제]·[상품 삭제]·[월 보상 저장] 모두 미노출 (조회만 가능)
  • 다음 잠금 조건에 해당하는 월 카드는 쓰기 권한 보유 여부와 무관하게 [+ 구간 추가]·[+ 상품 추가]·[구간 삭제]·[상품 삭제]·[월 보상 저장] 모두 미노출 (잠금 조건 자체가 우선):
  • 시즌 상태가 종료
  • 진행중 시즌의 지난 월 (정산 완료 여부 무관)
  • 정산 완료 월
  • 외부 협력사 권한자는 본인 매핑 소속사 산하 그룹의 시즌만 조회 가능 (쓰기 불가)

5.5 활동 로그

동작로그 메시지
월별 보상 저장덕력 시즌 '{그룹명} - {시즌명}' {yyyy.mm} 보상을 저장했습니다. (구간: {N}개, 상품: {M}개)

시즌 자체 수정·종료 로그는 각 모달에서 기록.
