1. 화면 개요

회원이 아티스트 그룹별로 적립·소비하는 덕력(DUK) 을 운영자가 시즌·랭킹·내역 차원에서 관리·조회하는 단일 화면. 사이드바 진입 1회 후 상단 탭 3개로 분기.

항목값
화면 경로/artists/duk (기본 탭 = 시즌 설정 / 탭은 URL query `?tab={seasonrankinghistory}`)
Breadcrumb홈 › 아티스트 › 덕력관리
진입 경로사이드바 [아티스트] > [덕력관리]
관리 단위아티스트 그룹 단위 (V01D, iKON, MADEIN 등 — 멤버 단위 제외)
시즌 단위그룹별 독립 시즌 (동일 시기에 그룹별로 다른 시즌 운영 가능)
데이터 모델시즌 마스터 + 변동 ledger 2종. 랭킹은 ledger의 (그룹, 시즌, 회원) 누적 합산으로 동적 도출
접근 권한[CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다 (쓰기 권한 없으면 모달 진입 차단 + [신규 시즌]·[수정]·[종료] 버튼 미노출)
사용처 화면 (응원)회원이 보유 덕력을 사용하는 화면(응원하기 등)은 본 사이클 범위 외 — ledger에 출처 "응원하기"로 기록만, UI 정의는 별도 사이클

주요 기능

  • 그룹별 랭킹 시즌 CRUD (생성·수정·종료)
  • 시즌·그룹 기준 회원별 누적 덕력 순위 조회
  • 획득·사용 ledger 전체 조회 (그룹·유형·기간·회원 필터 결합)

5-1. 입력 검증

5-2. 비즈니스 규칙

  • 그룹별 진행중 시즌은 동시 1개 제한 — 생성·수정 시 동일 그룹의 다른 진행중 시즌과 기간이 겹치면 차단
  • 시즌 상태는 시작·종료 일시 기준 자동 전이: 시작 전 = 예정 / 시작 ~ 종료 = 진행중 / 종료 이후 = 종료. 운영자가 [종료] 클릭 시 종료 일시를 현재로 조정하여 즉시 종료
  • 종료된 시즌은 수정·종료 액션 미노출 (조회만 가능)
  • 종료된 시즌의 랭킹은 변동 없이 보존 — 신규 적립·소비는 다음 시즌에 귀속
  • 시즌 삭제 액션 없음 — ledger 무결성 보존
  • 덕력 변동은 적립·소비 시점에 ledger 1행 신규 생성 (수정·삭제 불가 — 운영자 보정이 필요한 경우 "운영자 지급" 출처로 보정 행 추가)

랭킹·내역 조회는 활동 로그 대상 아님 (조회만).

5-4. 권한

  • 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다 (R/W 분리)
  • 쓰기 권한 없으면 [신규 시즌]·[수정]·[종료] 버튼 미노출 (조회만 가능)
  • EXTERNAL_ARTIST 권한자(외부 협력사) — [CEB-BO-003] §4 정합: 본인 매핑 소속사 산하 그룹의 시즌·랭킹·내역만 조회 가능 (쓰기 불가)
