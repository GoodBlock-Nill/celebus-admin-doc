5. 검증 규칙

5-1. 교환 전/후 GP 잔액 정합

거래 유형정합 공식
GP 충전 (CELB → GP)교환 후 GP 잔액 = 교환 전 GP 잔액 + GP 수량
GP 출금 (GP → CELB)교환 후 GP 잔액 = 교환 전 GP 잔액 − GP 수량
실패교환 후 GP 잔액 = 교환 전 GP 잔액 (변동 없음)

5-2. 실패 사유 (운영 BO 실패 케이스 추가 검증 시 보강)

명세 v1.1 시점의 사유 코드 표는 백엔드 식별자로 본 화면 정합 외. 사용자 메시지(한국어)만 운영 정합으로 보강 예정:

운영 메시지 (예시)
잔액 부족
1일 한도 초과
최소 교환 수량 미만
최대 교환 수량 초과
교환 기능 비활성화
시스템 오류

실패 사유 메시지의 정확한 운영 표기는 실패 케이스 발생 시 chrome-devtools 캡처 후 보강.

5-3. 적용 비율

  • 거래 시점에 적용된 GP ↔ CELB 비율이 기록되며, [CEB-BO-GZ-502] 교환 설정 변경 후에도 과거 거래는 당시 비율 유지 (소급 적용 없음)
  • 운영 표기 형태: "1 CELB = X GP"(충전 시 의미 부각) / "1 GP = X CELB"(출금 시 의미 부각)

5-4. 권한 및 활동 로그

  • 권한은 부모 화면 [CEB-BO-GZ-501]과 동일 ([관리자 > 권한관리] 설정)
  • 모달 조회 자체는 활동 로그 미기록 (조회 로그 정책 — [CEB-BO-GZ-000] 정합)
  • 모달은 조회 전용 — 수정·취소 액션 없음
