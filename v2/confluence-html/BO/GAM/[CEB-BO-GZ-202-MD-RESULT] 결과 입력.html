1. 모달 개요

PM 게임의 결과를 입력하여 Closed 전이시키는 모달. 결과제목·결과(YES/NO)·결과설명·결과링크를 입력하고 [결과 확정] 시 보상 계산 및 분배 가능 상태로 전환된다.

항목값

표시 조건게임유형=PM + 상태=Pending
영향 범위게임 상태 → Closed / 결과 정보 저장 / [보상 지급] 버튼 활성
접근 권한[관리자 > 권한관리] 설정 (쓰기 권한)

2-4. 결과 선택 (필수)

버튼표기설명
YES"YES 예상 정답자: {N}명" + 분배 미리보기YES 선택자 수 미리보기 + 그 아래 회색 보조 텍스트로 분배 미리보기 노출 (아래 분배 미리보기 규칙 참조)
NO"NO 예상 정답자: {N}명" + 분배 미리보기NO 선택자 수 미리보기 + 분배 미리보기 노출 (NO 선택 시 정답자 = NO 선택자로 계산)

분배 미리보기 규칙 (YES/NO 선택 후 예상 정답자 수 아래에 회색 보조 텍스트로 노출):

  • "응모권 {ticketReward}장 × 정답자 {N}명 = {N}장"
  • 게임 생성 시 입력한 응모권 보상(정답자 1인당)이 0이면 줄 미노출
  • "덕력 {dukReward} DUK × 참여자 {M}명 = {N} DUK"
  • 게임 생성 시 입력한 덕력 보상(참여자 1인당)이 0이면 줄 미노출
  • 덕력은 정답·오답 무관 참여자 전원에게 균등 분배 ([CEB-BO-GZ-000] §5 참조)

실제 분배는 [결과 확정] → [202-MD-REWARD] [지급] 클릭 시점에 트리거된다. 본 미리보기는 운영자가 확정 전에 분배될 보상 총량을 확인하는 용도다.

2-6. 결과링크 (선택)

  • 다국어 탭: KO / EN / JP
  • 입력 2개: 링크 텍스트 (placeholder "링크 텍스트를 입력하세요.") + URL (placeholder "https://example.com/result-proof")
