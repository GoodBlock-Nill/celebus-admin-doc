1. 화면 목적

Active 상태 Raffle을 관리자가 수동으로 조기 종료. Closed 상태로 전환 (Ended 아님 — 추첨이 남아있으므로). 신규 응모는 즉시 차단되며, 추첨은 [CEB-BO-FQ-206-MD-DRAW]에서 진행.

4.2 종료 프로세스

  • [확인] → 서버 종료 API 호출
  • 상태: Active → Closed ([CEB-BO-006] §7.2 — Quest와 다름. Raffle은 Closed 거쳐 Ended)
  • 응모 차단: 신규 응모 시도 시 회원 앱에 마감 안내
  • 추첨 대기: [CEB-BO-FQ-206]에서 추첨 진행

마감일 도래 시 시스템 자동 종료도 동일 결과 (Active → Closed).

4.3 종료 후 응모 데이터

  • 응모 스냅샷 동결 (이후 응모 불가)
  • 추첨 시점에 사용 ([CEB-BO-FQ-206-MD-DRAW] §4.2 시드 기반 추첨)
