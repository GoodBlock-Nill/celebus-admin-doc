1. 화면 목적

개별 소식·일정·공지의 내용을 읽기 전용으로 조회하고 라이프사이클 액션(게시·보관·재게시·삭제)을 수행하는 화면. 내용 수정은 [CEB-BO-INF-201-EDIT] 작성·수정으로 진입한다.

진입 경로

  • [CEB-BO-INF-101] 소식·일정 리스트 > Row 클릭

2. UI 설명

본 화면은 읽기 전용 상세이며, 내용 수정은 [CEB-BO-INF-201-EDIT] 작성·수정에서 수행한다.

2.1 헤더

  • Breadcrumb + 타입 뱃지 + 상태 뱃지
  • 타이틀: {제목(한국어)}
  • 우측 상단 액션 (상태별):
상태노출 액션
임시저장[수정] / [게시] / [삭제]
게시[수정] / [보관]
보관[재게시] / [삭제]

2.2 콘텐츠 표시 (타입별, 읽기 전용)

  • 제목·본문(소식·공지)·일정명(일정)은 언어 전환 탭(한국어/영어/일본어)으로 입력된 다국어 값을 모두 조회할 수 있다. 미입력 언어는 "({언어} 미입력)"으로 표시.
타입표시 항목
소식다국어 제목·본문 / 이미지 썸네일(등록 이미지 전체) / 연관 콘텐츠(콘텐츠 타입 + 콘텐츠명 + URL 링크) / 좋아요·조회수
일정다국어 일정명 / 일시 / 장소
공지다국어 제목·본문 / 대표 이미지 / CTA 버튼(라벨 → 링크)
  • 이미지는 등록한 파일을 썸네일 그리드로 표시한다(개수 텍스트가 아님). 미등록 시 "미등록".
  • 연관 콘텐츠·CTA의 URL은 새 탭으로 열린다.

3. 규칙·로직

3.1 권한

  • 권한 그룹별 읽기·쓰기·삭제 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.
  • [수정]·[게시]·[보관]·[재게시]·[삭제]는 쓰기 권한 보유 그룹만 사용한다.

3.2 상태별 액션

액션동작
[수정][CEB-BO-INF-201-EDIT] 작성·수정으로 이동
[게시] / [재게시]게시 확인 모달 → 게시(앱 노출) + 완료 토스트
[보관]보관 확인 모달 → 앱 노출 종료(데이터 보존) + 완료 토스트
[삭제]삭제 확인 모달 → 영구 삭제 + 토스트 → 리스트 복귀
