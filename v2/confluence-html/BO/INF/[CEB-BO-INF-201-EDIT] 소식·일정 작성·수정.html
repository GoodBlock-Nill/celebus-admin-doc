1. 화면 목적

소식·일정·공지를 작성(신규)하거나 수정하는 입력 폼. 운영자가 직접 작성·게시한다. 조회는 [CEB-BO-INF-201] 상세.

진입 경로

  • [CEB-BO-INF-101] 리스트 > [작성하기] (신규)
  • [CEB-BO-INF-201] 상세 > [수정] (수정)

2. UI 설명

2.1 헤더

  • Breadcrumb + 타이틀(작성 / {제목} 수정)
  • 우측 상단 액션: [취소] / [삭제](수정 시) / [임시저장] / [게시]
  • [게시] 버튼은 다국어 필수(한국어·영어·일본어)와 타입별 필수 항목이 모두 충족될 때만 활성된다. 미충족 시 비활성 처리하며, 미충족 사유 안내 문구를 노출한다. [임시저장]은 미완성 상태에서도 항상 가능하다.

2.2 공통 메타 (사이드)

항목표시
타입Dropdown — 소식 / 일정 / 공지 (선택에 따라 §2.3 입력 필드 분기)
연결 아티스트Dropdown — 그룹
공식 출처토글 — 공식 제공 뱃지 표기 (소식·일정만, 공지 미해당)

2.3 타입별 입력 폼

A. 소식

항목표시
제목다국어(한국어/영어/일본어), 각 50자 필수
본문다국어 위지윅, 각 5,000자 필수
게시일날짜 필수
이미지최대 10장
연관 콘텐츠최대 3건 — 각 행: 콘텐츠 타입(유튜브/틱톡/스포티파이/스레드/인스타그램/X/웹) + 콘텐츠명(50자) + URL. 3건 초과 추가 불가

B. 일정

항목표시
일정명다국어, 각 50자 필수
일시날짜 + 시간 (필수)
장소구글 장소 검색 입력 — 검색어 입력 시 장소 제안, 선택 시 장소명 입력 (선택, 100자)

일정에는 부제·썸네일 입력이 없다.

C. 공지

항목표시
제목다국어, 각 50자 필수
본문다국어 위지윅, 각 5,000자 필수
게시일날짜 필수
대표 이미지1장 (선택)
CTA 버튼라벨(30자) + 링크 URL (선택). 입력 시 앱 공지 상세에 버튼 노출

2.4 모달

모달트리거동작
작성/수정 취소 확인[취소]"작성을 취소할까요?" → 나가기 시 미저장
게시 확인[게시]"게시할까요?" → 게시 즉시 앱 노출
삭제 확인[삭제]"삭제할까요?" → 영구 삭제(복구 불가)

시각적 스타일은 피그마 참조

3. 규칙·로직

3.1 권한

  • 권한 그룹별 읽기·쓰기·삭제 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.

3.2 다국어 게시 검증

  • [게시] 시 소식·공지의 제목·본문, 일정의 일정명은 한국어·영어·일본어 모두 입력되어야 한다. 1개라도 빈 칸이면 [게시] 버튼이 비활성이며, 비활성 상태에서 게시 시도 시 차단 Toast: "한국어·영어·일본어를 모두 입력해야 게시할 수 있습니다."

3.3 타입별 필수 검증

타입게시 필수
소식제목·본문(3언어), 게시일
일정일정명(3언어), 일시
공지제목·본문(3언어), 게시일
  • 연관 콘텐츠는 최대 3건(콘텐츠 타입·콘텐츠명·URL). CTA는 선택(라벨·링크 동시 입력 시 유효).
