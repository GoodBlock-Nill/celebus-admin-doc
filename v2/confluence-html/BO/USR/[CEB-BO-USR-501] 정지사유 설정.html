1. 화면 목적

회원 제재 처리 시 운영자가 선택할 정지사유 데이터베이스를 통합 관리. [CEB-BO-FQ-101-REJETSET] 의 패턴을 회원 제재용으로 변형 — 코드(SUS_{n})로 정규화하여 일관된 회원 안내 메시지 보장.

주요 기능

  • 정지사유 리스트 조회 (운영자 노출명 + 코드 + 상태 + 사용 횟수)
  • 상태 필터 (사용/미사용)
  • 신규 정지사유 추가
  • 기존 정지사유 수정

진입 경로

  • [CEB-BO-USR-101] 회원 리스트 > [정지사유 설정] 버튼 클릭

4.7 멀티아티스트 정책 — 전역 단일 + 발견 컨텍스트 메타 ⭐ v1.1 신규

본 마스터는 [CEB-BO-100] §3 결정 ①에 따라 전역 단일로 운영한다. 아티스트별 분리하지 않는다.

4.7.1 결정 근거

검토 관점채택
정지사유 본질회원 행동 판정 (욕설·도배·악성 활동 등) — 회원 단위
식별 일관성같은 위반 1개 코드로 통합 통계
운영 복잡도단일 마스터 — 1벌만 관리
차별 룰 실수요거의 없음 (대부분 위반은 공통 카테고리)
결과전역 단일 채택 ([CEB-BO-100] §3.2 v1.0 → v1.1 변경 근거 인용)

4.7.2 발견 컨텍스트 메타

정지 처리 시점에 어느 아티스트 활동 중 발견되었는지 메타로 기록 → 통계·감사 가능.

필드타입설명
discoveryContextID NULL발견 컨텍스트 — 아티스트 그룹 ID 또는 NULL(전역 활동 중)
discoveredFeatureenumFANQUEST / RAFFLE / MEM / INF_COMMENT / SUPPORT / OTHER

메타는 본 화면(사유 마스터)에서는 입력하지 않는다. 회원 제재 처리 화면([CEB-BO-USR-201-MD-SANCTION])에서 운영자가 정지 처리 시 함께 입력한다.

4.7.3 발견 컨텍스트별 통계

본 화면 §2.3 테이블의 "사용 횟수" 컬럼은 발견 컨텍스트별로도 분리 통계 가능 (Phase 2+ 추가):

  • 컬럼 추가 옵션: 사용 횟수 (V01D 출처 / iKON 출처 / 전역 출처)
