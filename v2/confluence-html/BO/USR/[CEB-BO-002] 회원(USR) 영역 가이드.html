1. 개요

회원(USR) 영역은 CELEBUS 가입자 정보 조회·검색·필터·제재·자산 회수 등 회원 라이프사이클 전반을 관리한다. v1에서 회원 상세 4탭(기본·참여내역·게임존·BIVE)을 운영 중이며, v2에서 신규 4탭(덕력·서포트·기억·팬덤레벨)을 추가하여 8탭 구조로 확장한다.

7. 영역 고유 정책

7.1 회원 상세 9탭 구조 (v1.1 갱신 — 실제 운영 정렬)

분석 리포트 [CEB-BO-001-A] §3.2.2 운영 검증: 실제 BO에 9탭 구현됨. v2 신규 4탭(덕력·서포트·기억·팬덤레벨)은 별도 영역(Phase 9)에서 운영자가 영역별 화면으로 진입.

탭 순서 (실제 운영 그대로):

#탭명핵심 표시 항목
1기본정보닉네임·실명·전화·이메일·국가·로그인 방식·가입일·최근 로그인·DID 발급·Set Approval·지갑주소·소개·운영자 메모·제재 상태
2참여내역 (Quest·Raffle)Quest 제출 이력 (검수 상태별 카운트), Raffle 응모·당첨 이력, 응모권 잔액·만료 일정
3게임존PM·ST 참여 이력, GP 잔액·변동, CELB 교환 이력, 랭킹
4BIVE보유 BIVE 목록 (에디션·획득 경로·획득일), 외부 지갑 연동 상태, 혜택 사용 이력
5프로젝트(구현 대기 — 운영 시작 시 회원별 프로젝트 참여 이력)
6Fans(구현 대기 — 회원별 Fans 서비스 활동 이력)
7티켓회원이 보유한 응모권·이벤트 티켓 통합 조회 ([CEB-BO-FQ-301] §7.6 거래 코드 9종 회원 단위 필터링)
8지갑회원 외부 지갑 연동 상태, BIVE 보유 추적 ([CEB-BO-000] §15)
9활동내역회원 행위 통합 시계열 (로그인·제재·자산 회수·응모·민팅 등)

각 탭의 정렬·필터·표시 상세는 [CEB-BO-USR-201] 명세서에서 확정.

탭 전환 정책:

  • 탭 클릭 시 비동기 로딩 (탭별 API 분리)
  • 탭 진입 시 Skeleton UI ([CEB-BO-000] §7)
  • 탭별 빈 데이터 시 Empty State 메시지 (예: "해당 회원은 아직 Quest 참여 이력이 없어요")
  • 미구현 placeholder 탭(프로젝트/Fans): "해당 영역 운영 시작 시 활성화됩니다." 안내 노출

v2 신규 영역과의 분리

v2 BO-002 v1.0에서 정의했던 신규 4탭(덕력·서포트·기억·팬덤레벨)은 별도 영역 화면으로 분리:

v2 신규 영역영역 가이드 (Phase 9)회원별 진입 방식
덕력DUK 영역DUK-101 → 회원 검색 또는 회원 상세 [활동내역] 탭에서 딥링크
서포트SUP 영역SUP-101 → 회원 단위 응원 이력
기억MEM 영역MEM-101 신고 관리 → 회원별 신고당한·신고한 이력
팬덤레벨EVT 영역EVT-201 → 회원별 아티스트 팬덤 레벨 추이

→ 회원 상세 화면의 UX 부담을 낮추고, 영역별 운영 책임자(POLICY 등)가 자기 영역에 집중 가능.

탭 전환 정책:

  • 탭 클릭 시 비동기 로딩 (탭별 API 분리)
  • 탭 진입 시 Skeleton UI ([CEB-BO-000] §7)
  • 탭별 빈 데이터 시 Empty State 메시지 (예: "해당 회원은 아직 Quest 참여 이력이 없어요")
