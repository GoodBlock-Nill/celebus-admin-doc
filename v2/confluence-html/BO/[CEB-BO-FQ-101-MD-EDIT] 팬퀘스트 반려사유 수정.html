4.2 데이터 바인딩

  • 목록에서 클릭한 Row의 상세 데이터 API 조회
  • 운영자 노출명·유저 노출 메시지(3언어 모두)·상태값 모두 바인딩
  • 라디오 전환 시 해당 언어의 기존 메시지 노출

4.3 상태 변경 로직

[CEB-BO-006] §7.4 적용:

  • 사용(Active) → 미사용(Inactive) 변경 시: 저장 즉시 [CEB-BO-202-MD-REJECT] 검수 모달의 사유 목록에서 해당 항목 제거. 기존 반려 처리된 Quest 내역에는 영향 없음
  • 미사용(Inactive) → 사용(Active) 변경 시: 검수 모달 목록에 다시 노출
  • 메시지 수정 시: 신규 반려부터 새 메시지 적용. 과거 반려 건의 알림 메시지는 그대로

4.4 유효성 검사

  • 운영자 노출명: 1~30자
  • 유저 노출 메시지: KO/EN/JP 3언어 모두 1~200자

4.5 [저장하기] 활성화 조건

  • 모든 필수값 입력 완료
  • 변경 사항이 1건 이상 있어야 활성화 (변경 없음 시 비활성)
