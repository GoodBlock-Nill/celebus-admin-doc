1. 화면 목적

팬퀘스트 검수 시 운영자가 선택할 반려사유 데이터베이스를 통합 조회·관리. [CEB-BO-006] §7.4 정책 적용 — 코드(REJ_{n})로 정규화되어 일관된 회원 안내 메시지 보장.

주요 기능

  • 반려사유 리스트 조회 (운영자 노출명 + 상태 + 코드)
  • 상태 필터 (사용/미사용)
  • 신규 반려사유 추가 진입
  • 기존 반려사유 수정 진입

진입 경로

  • [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > [반려사유 설정] 버튼 클릭
