2. 화면 목적

전체 BO에서 발생한 운영자 활동 로그를 통합 단일 테이블로 조회한다. 멀티아티스트 운영을 반영해 artistContext 컬럼·필터를 추가하여, 특정 아티스트와 관련된 운영 활동을 분리 검색 가능하다.

본 화면은 SUPER 전용 감사 도구다. 일반 운영자는 본인 활동 로그만 [CEB-BO-SYS-201] 관리자 상세에서 조회.

3. 멀티아티스트 정책 적용

3.1 정책 결정 — 전역 + 컨텍스트 컬럼

[CEB-BO-100] §3 결정:

측면채택
감사 통합성모든 활동을 단일 테이블에 적재. 영역·아티스트별 분리 X
검색 분기artistContext 컬럼 + 필터로 아티스트 단위 분기 가능
EXTERNAL_ARTIST 접근❌ 차단 ([CEB-BO-100] §4)

3.2 artistContext 자동 기록 룰

활동 로그 적재 시 다음 룰로 artistContext 자동 기록:

작업artistContext 값
콘텐츠 생성·수정·삭제 (Quest/BIVE/모바일티켓 등)콘텐츠 소속 아티스트 ID
회원 정지 처리discoveryContext ([CEB-BO-USR-501])와 동일 — 발견 컨텍스트 아티스트 ID
헤더 글로벌 토글이 단일 아티스트인 상태에서 작업토글 아티스트 ID
글로벌 토글이 "전체" + 콘텐츠 무관 작업 (관리자 초대·권한 변경 등)NULL (전역)
마스터 데이터 변경 (그룹 등록·곡선 변경 등)변경 대상 아티스트 ID

시스템이 자동 추론. 운영자가 수동으로 입력하지 않음.

6. 규칙·로직

6.1 정렬 기준

  • 기본: 일시 내림차순 (최신순).
  • 일시 컬럼 헤더 클릭으로 정렬 토글.

6.2 컨텍스트 필터 옵션

옵션동작

전역 (NULL)컨텍스트 NULL 로그만 (관리자 초대·권한 변경 등)

6.3 데이터 보관

  • [CEB-BO-000] §6 표준 — append-only, 3년 보관.
  • 3년 초과 로그는 콜드 스토리지 자동 이관 (BO 화면 미노출. 별도 아카이브 도구 필요 시 후속 트랙).

6.4 EXTERNAL_ARTIST 차단

[CEB-BO-100] §4 적용:

  • 본 화면 진입 차단.
  • 자기 아티스트 활동만이라도 보고 싶다는 요구가 있을 경우 별도 정책 검토 (현재 본 가이드 범위 밖).

6.5 헤더 글로벌 토글 연동

  • 토글 = 단일 아티스트일 때 본 화면 진입 시 컨텍스트 필터 자동 적용.
  • 운영자가 페이지에서 "전체"로 덮어쓰기 가능 ([CEB-BO-100] §2.2).

11. 도입 시점

[CEB-BO-100] §6 로드맵:

  • Phase 1: 데이터 모델에 artistContext 필드만 추가 (NULL 디폴트). 기존 활동 로그 그대로.
  • Phase 2: 본 화면 활성화 — artistContext 자동 기록·필터·컬럼 노출. 기존 로그는 NULL로 backfill.
  • Phase 3: 헤더 글로벌 토글과 자동 연동.
