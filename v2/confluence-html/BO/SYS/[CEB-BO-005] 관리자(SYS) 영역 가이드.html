1. 개요

관리자(SYS) 영역은 BO 사용자 계정·권한·활동 로그를 관리한다. v1에서 관리자 초대·리스트·권한 관리(권한 그룹·권한 생성·권한 상세 3탭)를 운영 중이며, v2에서 4단계 권한 + 그룹 커스터마이징 하이브리드를 신규 정의한다. 본 영역은 SUPER 전용 (활동 로그 조회는 자기 활동만 다른 권한도 R 가능).

7. 영역 고유 정책

7.1 4단계 권한 + 그룹 하이브리드

[CEB-BO-000] §3.1·§3.2 기반. 하이브리드 모델:

4개 기본 프리셋 그룹 (시스템 정의, 수정 불가)
├── SUPER          — 모든 영역 R/W/D + 권한 부여·회수
├── OPERATOR       — 일상 운영 (Quest 검수·콘텐츠 관리·결과 입력 등)
├── POLICY         — 정책 매개변수 (덕력 캡·팬덤 곡선·교환 비율 등)
└── EXTERNAL_ARTIST— 외부 협력사 (자기 아티스트 한정)

+ 운영자 정의 커스텀 그룹 (SUPER가 추가)
├── 예: "QA Reviewer"  — Quest 검수만 가능, 다른 영역 R only
├── 예: "콘텐츠 매니저"  — INF·HOM 영역 W, 다른 영역 차단
└── 예: "임시 운영자"   — OPERATOR 권한 + 90일 만료 강제

그룹 ↔ 관리자 N:N

항목정책
한 관리자 = 여러 그룹 가능권한은 OR 합집합 (가장 권한이 큰 액션 적용)
그룹 = N명의 관리자 매핑한 그룹 변경 시 매핑된 모든 관리자에게 즉시 반영
기본 프리셋 그룹시스템 정의, 수정·삭제 불가 (이름·설명만 SUPER가 변경 가능)
커스텀 그룹SUPER가 자유 생성·수정·삭제 (가드레일 적용)
충돌 방지한 관리자가 여러 그룹에 매핑되어 권한 충돌 시 더 큰 권한 우선 (Union)
