4.2 추첨 알고리즘 ([CEB-BO-006] §7.5 적용)

시드 기반 결정론적 추첨:

seed = SHA256(raffleId + drawTimestamp + adminUserId + entryListSnapshot)
shuffled = Fisher-Yates(entries, seed)
winners = shuffled[0:winnerCount]

Step 1: 데이터 준비

각 회원의 (전화번호, 응모권 수)를 매핑.

식별자 (Phone)사용 응모권 (Weight)
821011112222 (User A)1장
821033334444 (User B)5장
819055556666 (User C)2장
합계8장

Step 2: 가중치 구간 할당

회원구간
User A1 ~ 1 (1구간)
User B2 ~ 6 (5구간)
User C7 ~ 8 (2구간)

→ User B 당첨 확률 = 5/8 = 62.5%

Step 3: 시드 기반 Random + 중복 방지

  • 시드로 1~8 사이 정수 생성
  • 매칭된 회원 당첨
  • 당첨 회원은 풀에서 제외 (응모권 모두 회수)
  • 목표 당첨자 수만큼 반복

4.5 확정 프로세스

[추첨결과 확정하기] 클릭 → 다음을 단일 트랜잭션으로 처리:

  1. 가결과를 DB 영구 저장 (Commit, 번복 불가)
  2. Raffle 상태: Closed → Ended
  3. 응모 스냅샷 동결 (이후 응모 시도 자동 차단 — 사실 Closed 시점에 이미 차단됨)
  4. Win 회원: 보상 지급 자동 트리거
  • 현장/배송/온라인 타입에 따라 분기
  • BIVE 보상이 있으면 [CEB-BO-004] §4.3 자동 민팅 큐 추가
  1. 모든 응모 회원: 추첨 결과 푸시 알림 ([CEB-BO-000] §15) + 결과 화면 딥링크
  2. 활동 로그 기록 ([CEB-BO-006] §6) — 시드 hex 포함

4.6 재추첨 정책

  • 확정 전 무제한 재추첨 가능
  • 재추첨 시 새로운 시드 생성 (timestamp 갱신)
  • 재추첨 시도는 활동 로그 미기록 (확정 시점만 기록)

4.7 응모자 < 목표 당첨자 처리

  • 실제 당첨자 = 응모자 전체
  • 모달 상단 안내: "응모자 N명, 목표 당첨 M명 → 전원 당첨 처리됩니다."
