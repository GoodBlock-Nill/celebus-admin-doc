1. 화면 목적

등록된 전체 Raffle(추첨형 퀘스트) 현황을 목록 형태로 조회·관리. 운영자는 본 화면에서 Raffle의 상태별 분포와 누적 응모 추이를 파악하고, 신규 Raffle 생성·개별 Raffle 상세로 진입한다.

주요 기능

  • 등록된 전체 Raffle 목록 조회 및 검색
  • 상태별 필터링 (진행중 / 임시저장 / 추첨대기 / 종료)
  • 신규 Raffle 생성 진입
  • Raffle 상세 진입

진입 경로

  • [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > [Raffle] 탭 클릭

4.3 상태 정의

4.4 상태별 데이터 노출

  • 임시저장(Draft) → 관리자 화면 전용. 앱(유저 화면) 미노출
  • 추첨대기(Closed) → 모집 기간 종료 또는 관리자 강제 종료 진입. 추첨(Draw) 프로세스 대기
  • 추첨 진입 조건은 [CEB-BO-006] §4.2 SUPER 가드레일 적용 (응모권 10K+ 또는 BIVE 100+ 시 SUPER 2FA)

4.5 검색 조건

  • 상태 필터 + 키워드 검색 AND 조건
  • 키워드 검색은 부분 일치(LIKE)
  • 검색어 미입력 시 해당 조건 무시
  • 검색 대상 "아티스트" 시: 그룹·멤버명 통합 검색
