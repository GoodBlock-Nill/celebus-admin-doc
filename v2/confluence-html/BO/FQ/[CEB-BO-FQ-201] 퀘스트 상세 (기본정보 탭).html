1. 화면 목적

등록된 개별 Quest의 기본 정보·설정·생애 주기를 관리. 운영자는 본 화면에서 Quest 현황을 확인하고 게시/수정/종료/삭제 액션을 수행한다. 검수 작업은 형제 탭(대기내역·처리내역)에서.

주요 기능

  • Quest 기본 정보 및 설정 상태 확인 (다국어 텍스트 / 대표 이미지 / 보상 / 일정 / 연관 링크)
  • 실시간 제출 현황 대시보드 (총 제출 / 승인 / 반려 / 대기)
  • Quest 라이프사이클 관리 ([CEB-BO-006] §7.1)
  • 형제 탭 진입 (대기내역 / 처리내역)

진입 경로

  • [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > Quest 리스트 Row 클릭

4.2 상태 전환 로직 ([CEB-BO-006] §7.1)

  • 게시하기 (Draft → Active): [게시하기] → [MD-OPEN] 확인 → 서버 시간(KST)을 시작 일시로 자동 기록 + 상태 Active. 앱 즉시 노출.
  • 종료하기 (Active → Ended): [종료하기] → [MD-CLOSE] 확인 → 참여 차단 + 상태 Ended. 마감일 도래 시 시스템 자동 종료도 동일 결과.
  • 삭제하기 (Draft 한정): [삭제하기] → [MD-DELETE] 확인 → 영구 삭제. 진행중/종료 상태는 [삭제하기] 미노출.

4.3 데이터 표시 규칙

  • 대기 카운트: 종료 상태가 되어도 미처리 검수 건이 있으면 숫자 잔존. 운영자는 처리내역 탭에서 정리
  • 날짜 포맷: YYYY.MM.DD HH:mm KST 통일 ([CEB-BO-000] 표준)
  • 다국어 콘텐츠: 임시저장 상태에서 일부 언어 비어있을 수 있음. 게시 시점에 3언어 검증
