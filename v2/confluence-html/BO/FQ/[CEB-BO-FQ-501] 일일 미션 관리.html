1. 화면 목적

APP 일일 미션 화면([CEB-FQ-501])의 운영 관리. 미션 풀 8종 CRUD, 보상량 정책 설정, 스트릭 보너스 정책, 일·주 캡 관리, 회원별 미션 회수·재지급.

핵심 운영 항목:

  • 미션 풀 8종 등록·수정·삭제 (텍스트·아이콘·딥링크)
  • 출석 보상 / 일일 미션 일괄 보상 / 스트릭 4단계 보상량 설정
  • 일 500DUK · 주 3,000DUK 상한 정책
  • 푸시 알림 시각·문구 설정 (23:00 KST 스트릭 끊김 알림)
  • 회원별 미션 회수·재지급 (이슈 발생 시)
  • 활동 로그·통계

보상량·캡·스트릭 정책은 POLICY 권한 전용 (OPERATOR는 조회만).

3.2 미션 풀 활성화 정책

  • 활성 미션 ≥ 1건 필요 (0건이면 APP 카드 숨김)
  • 비활성 전환 시 당일 이미 선택된 미션은 유지, 익일부터 풀 제외
  • 삭제 시 활동 로그 보관, 회원 데이터에는 영향 없음 (이미 적립된 DUK 유지)

3.3 보상 정책 변경 트랜잭션

  1. POLICY가 보상량 변경 → 활동 로그 기록
  2. 변경 시점 이후 발생하는 적립부터 신규 값 적용
  3. 이미 지급된 DUK는 회수하지 않음 (회수가 필요하면 회원 보정 별도 사용)

3.4 회원 보정 적립·회수

  • OPERATOR/SUPER가 [회원 상세 → 수동 보정] 클릭
  • 사유 입력 필수 (200자, KO)
  • 일·주 캡 영향 없음 (보정은 캡 우회)
  • DUK 거래 코드: DUK_OPERATIONAL_ADJUST ([CEB-BO-DUK-000] §운영 2종)

3.5 스트릭 보너스 1회성 정책

  • 7/14/30/100일 보너스는 회원당 각 1회만 지급
  • 시즌 변경에도 보상 이력은 유지 (스트릭은 회원 계정 단위)
  • 4개 마일스톤 모두 달성 후 추가 마일스톤은 BO에서 정책 추가 시 적용

3.6 일·주 캡 우선순위

  • 출석 → 일일 미션 → 스트릭 보너스 순으로 적립 시도
  • 캡 도달 시 후속 적립 0 (회원에게 안내 — APP 책임)
  • 보정 적립은 캡 우회
