비고Quest·Raffle 검수·반려 시 선택 가능한 사유 마스터. 아티스트별 분리 정책. 멀티아티스트 마스터 = [CEB-BO-100] v1.2 §3 (반려사유 = 아티스트별, 🔴 Critical 등급). FQ-501은 일일 미션 관리로 사용 중이므로 본 화면은 FQ-502 ID 부여.

2. 화면 목적

Quest·Raffle 콘텐츠 검수 단계에서 운영자가 반려 처리 시 선택할 표준 사유 마스터를 아티스트별로 일괄 관리한다. V01D, iKON 등 각 아티스트마다 별도 사유 마스터를 보유.

본 화면은 마스터 데이터 관리다. 실제 반려 처리는 [CEB-BO-FQ-202-MD-REJECT] 등 검수 화면에서 진행한다.

3. 정책 결정 — 아티스트별 분리

검토 관점채택
반려사유 본질콘텐츠 스타일·검수 기준 (예: "아티스트 사진 무단 합성 금지")
콘텐츠 가이드아티스트마다 다름 — 한 아티스트의 합법 콘텐츠가 다른 아티스트에게는 위반일 수 있음
운영 책임아티스트사(EXTERNAL_ARTIST)가 자기 아티스트 사유 가이드 정의 가능
결과아티스트별 분리 채택 ([CEB-BO-100] §3)

6. 규칙·로직

6.1 코드 정책

  • 코드는 영문 대문자 + 언더바 (예: V01D_PHOTO_EDIT).
  • 아티스트 접두어 권장 (예: V01D_, IKON_) — 통계 분리용.
  • 등록 후 수정 불가 (감사 추적 보존).
  • 동일 아티스트 내 코드 중복 금지. 다른 아티스트 간 같은 코드 허용 (단, 권장 X).

6.2 카테고리

코드표시명예시
CONTENT콘텐츠 위반무단 합성 사진·저작권 침해·아티스트 부정적 묘사
FORMAT형식 위반해시태그 누락·이미지 품질 미달·필수 항목 누락
POLICY정책 위반단순 좋아요·RT만으로 인증 시도·계정 비공개
OTHER기타위 분류 외 사유

6.3 적용 단계 (applyTo)

반려사유는 적용 단계를 다중 선택 — 검수 화면에서 해당 단계의 사유만 노출.

단계적용 화면
Quest 검수[CEB-BO-FQ-202-MD-REJECT]
Raffle 응모 검수(Raffle 응모 검수 화면)
Raffle 추첨 후 회수(Raffle 회수 화면)

6.4 비활성화 영향

  • 비활성된 사유는 검수 화면 dropdown에서 제외.
  • 기존 반려 이력에 사용된 사유는 그대로 노출 (감사).
  • 재활성화 가능.

6.5 신규 아티스트 추가 시 템플릿 복제

[CEB-BO-100] §5 + [CEB-BO-ART-501] 마법사 연계.

  • 신규 아티스트 등록 시 템플릿 아티스트의 반려사유 마스터 전체 자동 복제.
  • 운영자가 항목별 옵트아웃 가능.
  • 코드는 신규 아티스트 접두어로 자동 변환 (예: 템플릿 TPL_PHOTO_EDIT → 신규 IKON_PHOTO_EDIT).
  • 복제된 사유는 즉시 편집 가능.

6.6 EXTERNAL_ARTIST 권한 적용

[CEB-BO-100] §4 + [CEB-BO-006] 적용:

  • 자기 아티스트 사유 마스터만 R/W (추가·수정·활성·비활성 모두 가능).
  • 다른 아티스트 사유 노출 차단.
  • 헤더 컨텍스트 토글 자기 아티스트 고정.

6.7 헤더 글로벌 컨텍스트 토글 영향

  • 토글이 "전체"이면 아티스트 선택 dropdown 활성. 운영자가 자유 선택.
  • 토글이 단일 아티스트면 dropdown 잠금 (해당 아티스트 고정).
