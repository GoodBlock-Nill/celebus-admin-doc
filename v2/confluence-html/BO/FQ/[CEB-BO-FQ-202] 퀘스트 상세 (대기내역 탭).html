1. 화면 목적

유저가 Quest 미션을 수행하고 제출한 결과물 중 검수 미처리(Pending) 상태의 제출 건을 운영자가 승인/반려한다. 검수 워크플로우의 핵심 진입점. [CEB-BO-006] §7.3 워크플로우 참조.

주요 기능

  • 대기 상태 유저 제출물 카드 그리드 조회
  • 개별 승인 (즉시 처리, BIVE 자동 민팅 + 응모권 지급 + 푸시 알림)
  • 개별 반려 (반려사유 모달 호출 → [CEB-BO-FQ-202-MD-REJECT])
  • 현재 페이지 일괄 승인

진입 경로

  • [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) > [대기내역] 탭 클릭

2. UI 설명

2.1 상단 영역

  • 탭 메뉴 (3탭): 기본정보 / 대기내역 (현재) / 처리내역
  • [현재 페이지 모두 승인] 버튼 (우측 상단)
  • 현재 페이지(Pagination된) 전체 대기 건 일괄 승인
  • 페이지에 대기 건 없을 시 비활성화

2.2 카드 그리드 (Grid List)

  • 정렬: 제출일시 오름차순 (오래된순)
  • 페이지당 최대 10건
  • 데이터 없음: "제출내역이 없습니다." 문구

각 카드 구성:

항목표시
이미지 썸네일유저 업로드 원본 (클릭 시 lightbox 확대)
유저 정보{유저닉네임} / User ID
제출 정보Submit ID / 제출일시 (YYYY.MM.DD HH:mm KST)
액션 버튼[반려] [승인]

2.3 페이지네이션

  • 하단 중앙. 10페이지 단위 이동.

시각적 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 승인 처리 (Approve)

[CEB-BO-006] §7.3 검수 워크플로우 참조.

  • 개별 승인 ([승인] 버튼 클릭)
  • 별도 확인 절차 없이 즉시 처리
  • 상태 → Approved. 카드는 처리내역 탭으로 이동
  • 트리거 (자동):
  • 응모권 지급 (Quest 보상 설정 따라)
  • BIVE 자동 민팅 ([CEB-BO-004] §4.3, 보상 NFT 캠페인 연결 시)
  • 회원 푸시 알림 ([CEB-BO-000] §15. 알림 본문은 피그마 참조)
  • 회원 앱: Event 화면(CEB-FQ-201)에서 응모권 / 컬렉션에서 BIVE 확인 가능
  • 일괄 승인 ([현재 페이지 모두 승인] 클릭)
  • 현재 페이지의 모든 대기 건 한 번에 승인 처리
  • 부분 실패 가능: 결과 토스트에 성공/실패 건수 표시

3.2 반려 처리 (Reject)

  • [반려] 버튼 클릭 → [CEB-BO-FQ-202-MD-REJECT] 모달 호출
  • 모달에서 반려사유 코드 선택 + 운영자 메시지 (선택, 200자) 입력
  • 최종 확인 시: 상태 → Rejected. 카드는 처리내역 탭 이동. 회원 푸시 알림 + 재참여 가이드

3.3 네비게이션

  • [기본정보] 탭 → [CEB-BO-FQ-201]
  • [처리내역] 탭 → [CEB-BO-FQ-203]

4.2 정렬 기준

정렬 기준정렬 방식비고
제출일시오름차순 (오래된순)기본. FIFO 검수 원칙

4.3 승인 시 자동 트리거

승인 1건당 다음 4개 트리거가 단일 트랜잭션으로 처리:

  1. 응모권 잔액 +N (Quest 보상 설정 수량)
  2. BIVE 자동 민팅 큐에 추가 ([CEB-BO-004] §4.3) — 보상에 NFT 포함 시
  3. 활동 로그 기록 ([CEB-BO-006] §6)
  4. 회원 푸시 알림 ([CEB-BO-000] §15)

부분 실패 처리: 응모권 지급 실패 시 전체 롤백. BIVE 민팅 실패는 별도 큐 재시도 (응모권은 정상 지급).

4.4 일괄 승인 한도

  • 1회당 최대 페이지 단위 (10건)
  • 대량 승인 시 단계별 처리 (1건씩 순차)
  • 처리 중 페이지 이탈 시 백그라운드 큐 처리 + 완료 시 토스트
