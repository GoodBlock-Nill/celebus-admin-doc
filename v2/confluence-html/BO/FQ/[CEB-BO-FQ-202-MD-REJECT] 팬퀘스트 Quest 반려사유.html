4.2 반려 프로세스

[CEB-BO-006] §7.3 검수 워크플로우 적용:

  1. 반려사유 코드 선택 + (선택) 운영자 메시지 입력
  2. [저장하기] → 서버 API 호출
  3. 제출 건 상태: Pending → Rejected
  4. 회원 푸시 알림 ([CEB-BO-000] §15):
  • 본문: 반려사유 코드의 유저 노출 메시지(KO/EN/JP, 회원 언어 설정 우선) + 운영자 메시지(있을 경우)
  • 재참여 가이드 (앱 내 딥링크)
  1. 활동 로그 기록 ([CEB-BO-006] §6)
  2. [CEB-BO-FQ-203] 처리내역 탭으로 데이터 이동

4.3 유효성 검사

  • 반려사유: 필수 (선택 안 한 채 [저장하기] → Toast Error)
  • 운영자 메시지: 선택. 입력 시 200자 제한

4.4 [저장하기] 활성화 조건

  • 반려사유 선택 완료
