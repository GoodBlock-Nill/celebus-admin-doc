1. 영역 개요

  • 운영 메뉴 (v1.1): 좌측 사이드바 에피소드 › 팬퀘스트 → /sq/quests 진입. Quest 리스트 단일 페이지
  • Raffle: 별도 메뉴 래플 › 래플 → /raffle (본 문서 범위 외)
  • 권한: Super Admin 확인. 세부 권한 매트릭스는 [CEB-BO-006](v2/BO/FQ/[CEB-BO-006]%20Quest%C2%B7Raffle(FQ)%20영역%20가이드.md) 참조

⚠️ v1.1 변경: ~~사이드바 탑레벨 "팬퀘스트" 단일 항목~~ + ~~/fanquest Quest/Raffle 두 탭~~ 폐기. Quest는 SQ 도메인 메뉴 안 페이지로, Raffle은 응모권과 묶인 "래플" 그룹으로 이전. 기존 /fanquest는 /sq/quests로 redirect 처리.

3. Quest 상태 머신

        [생성 폼] 작성
            ↓ "임시저장" (임시저장 버튼은 운영 미실측 — 추정)
        ┌──────────┐
        │ 임시저장   │ Draft (게시 전, 비공개)
        └──────────┘
            │
       (게시하기)
            ↓
        ┌──────────┐
        │ 진행중    │ Active (시작 일시 도달 → 마감 일시 전까지 참여 가능)
        └──────────┘
            │
   ┌────────┴───────────┐
(자동: 마감일시 도달)  (수동: 종료하기)
            ↓                ↓
        ┌──────────┐
        │ 종료     │ Ended (read-only)
        └──────────┘
  • 상태 값: 임시저장 (DRAFT) / 진행중 (ACTIVE) / 종료 (ENDED) (운영 코피 기준)
  • 자동 전이: 마감일시 도달 시 → 종료 (서버 잡)
  • 수동 전이:
  • 임시저장 → 진행중: 게시하기 (FQ-201-MD-OPEN)
  • 진행중 → 종료: 종료하기 (FQ-201-MD-CLOSE) — 조기 종료
  • 삭제: 임시저장 상태에서만 삭제 가능 (추정 — 실측 필요)
  • 시작 일시: 게시 시점에 자동 설정 (운영 안내 텍스트 명시)

5. 제출(Submission) 처리 플로우

[유저 앱] 인증 사진 업로드 (관련 BO 외부)
    ↓
[BO 대기내역 탭] 카드 노출 (이미지·닉네임·User ID·Submit ID·제출일시)
    ↓
운영자 액션:
  ├── (행 단위) 승인 → 상태 = 승인, 처리일시·처리자 기록
  ├── (행 단위) 반려 → 반려사유 모달 → 사유 선택 후 반려하기 → 상태 = 반려
  └── (일괄) 현재 페이지 모두 승인 → 페이지 내 모든 대기 행 일괄 승인
    ↓
[처리내역 탭] 처리된 카드가 표시 (상태, 반려사유, 처리일시, 처리자 표시)
