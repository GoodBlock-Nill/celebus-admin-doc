1. 화면 개요

사이드바 "래플 › 래플" 진입 시 노출되는 래플 영역의 메인 리스트 화면. 운영자는 본 화면에서 전체 래플의 상태별 분포를 한눈에 파악하고 신규 래플 생성·개별 래플 상세 진입을 수행한다.

항목값
화면 경로/raffle
Breadcrumb래플 / 래플
페이지 제목"래플"
진입 경로사이드바 "래플 › 래플"
접근 권한[관리자 > 권한관리]에 설정된 권한을 따름

2. 화면 구성

2-1. 헤더

  • Breadcrumb (홈 → 래플 → 래플)
  • 페이지 제목 "래플"

2-2. 필터·검색 바 (한 줄 가로 배치)

위치요소동작
좌상태 드롭다운"전체 / 임시저장 / 진행중 / 추첨대기 / 종료" (기본 "전체")
우검색 대상 드롭다운"타이틀 / 아티스트" (기본 "타이틀")
우키워드 입력 칸부분 일치 검색. 입력 즉시 필터 적용
우[초기화] 버튼모든 필터·검색 조건을 리셋하고 1페이지로 복귀
우[생성하기] 버튼래플 생성 화면으로 이동 ([CEB-BO-RFL-101-CREATE])

2-3. 래플 목록 테이블 (7개 컬럼)

컬럼정렬·정렬값표시 내용
상태좌측상태 배지 (임시저장 / 진행중 / 추첨대기 / 종료)
타이틀좌측래플 한국어 타이틀
아티스트좌측V01D / iKON / CELEBUS
기간좌측"시작일 ~ 마감일" (임시저장은 "— ~ 마감일")
누적 참여자우측응모 회원 수 (중복 제외)
누적 응모권우측사용된 응모권 총합
숨김중앙종료 후 운영자가 숨김 처리한 경우 "숨김" 배지 노출. 그 외 "—"

각 행을 클릭하면 래플 상세 화면([CEB-BO-RFL-201])으로 이동.

2-4. 페이지네이션

  • 페이지당 20건 (필터·검색 결과 기준)
  • 항상 노출 — 결과가 20건 미만이라 1페이지일 때도 ‹ 1 › 비활성 형태로 노출 ([CEB-BO-012] §5-2 공통 페이지네이션 정합)
  • 윈도잉 규칙 — 총 페이지 ≤ 7이면 전체 페이지 번호 노출, 8페이지 이상이면 1 ... cur-1 cur cur+1 ... last 윈도우 + 양 끝(...) 표시
  • 필터·검색·키워드 변경 시 자동으로 1페이지로 리셋

3. 화면에 노출되는 정보

위치정보설명
테이블 컬럼 1상태래플의 현재 운영 단계
테이블 컬럼 2타이틀회원에게 노출되는 래플 제목 (한국어)
테이블 컬럼 3아티스트래플이 속한 아티스트
테이블 컬럼 4기간응모 시작일 ~ 마감일 (날짜만 표시, 시각은 생략)
테이블 컬럼 5누적 참여자응모한 회원 수. 한 회원이 N장 응모해도 1명으로 계상
테이블 컬럼 6누적 응모권실제 사용된 응모권 총량. 한 회원이 N장 응모하면 N으로 계상
테이블 컬럼 7숨김종료 후 앱에서 숨김 처리된 래플 여부

추가 보상(BIVE) 정보는 본 리스트에 노출되지 않는다. 래플 상세 화면([CEB-BO-RFL-201])에서 확인.

4. 상태별 동작 분기

본 화면은 읽기 전용 리스트이므로 상태별 분기가 없다. 모든 상태의 래플이 동일하게 노출된다.

다만 다음 정책이 적용된다.

정책동작
권한별 접근 제한[관리자 > 권한관리] 설정에 따라 일부 운영자에게 특정 아티스트의 래플만 노출되도록 자동 필터링될 수 있음
검색 결과 0건"래플이 없습니다." 빈 상태 메시지 노출
기본 정렬마감일 임박순 (임박한 진행중 래플이 가장 위). 단, 종료·임시저장은 마지막에 배치되어 운영 관심사를 우선 노출
