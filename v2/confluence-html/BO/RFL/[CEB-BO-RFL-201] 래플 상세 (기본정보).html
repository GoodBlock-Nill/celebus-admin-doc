1. 화면 개요

특정 래플 한 건의 모든 운영 정보를 통합 노출하는 운영자용 허브 화면. 상단 3탭(기본정보 / 응모내역 / 추첨내역) 중 기본정보 탭은 마스터 정보·통계·다국어 콘텐츠·추가 보상(BIVE)·수령 가이드·유의사항을 한 화면에 노출한다.

항목값
화면 경로/raffle/{id} (기본정보 탭 활성)
Breadcrumb래플 / 래플 상세
페이지 제목래플의 한국어 타이틀
진입 경로[CEB-BO-RFL-101] 리스트에서 행 클릭
자매 화면[CEB-BO-RFL-202] 응모내역 / [CEB-BO-RFL-203] 추첨내역

2. 화면 구성

2-1. 헤더

  • 좌측: 뒤로가기 버튼 (리스트로 복귀)
  • 중앙: 래플의 한국어 타이틀 + 상태 배지 + (해당 시) 숨김 배지
  • 우측: 상태별 액션 버튼 (§4 상세)

2-2. 탭 메뉴

상단에 가로 배치된 3개 탭:

  • 기본정보 (본 문서)
  • 응모내역 → [CEB-BO-RFL-202]
  • 추첨내역 → [CEB-BO-RFL-203]

2-3. 통계 카드 (가로 4개)

카드의미
총 응모자본 래플에 응모한 회원 수 (중복 제외)
총 응모권사용된 응모권 누적 합 (한 회원이 N장 응모 시 N으로 계상)
목표 당첨자생성 시 설정한 당첨 추첨 수
지급 완료추첨내역 탭의 "보상 지급" 토글로 갱신되는 카운트. 임시저장·진행중·추첨대기에서는 0으로 고정, 종료 상태에서만 운영자 토글 조작에 따라 증감

2-5. 다국어 콘텐츠 섹션 (각 1 카드)

래플 타이틀 — 한국어/영어/일본어 3행 노출. 입력되지 않은 언어는 회색 안내 "(미입력)".

래플 설명 — 한국어/영어/일본어 3행.

경품 상세 — 한국어/영어/일본어 3행. 경품 단위(장/개)는 별도 표기 없이 자연어로 결합.

2-6. 추가 보상 (BIVE) 카드

BIVE 보상 토글 상태에 따라 분기 노출:

토글 상태표시
ON4개 항목 2열 배치 — "BIVE 보상 ON" / 민팅 이벤트명 / 캠페인 ID / 지급 대상 ("당첨자 자동 민팅")
OFF회색 안내 — "BIVE 보상 미지정 — 당첨자에게 경품만 지급됩니다."

운영 동작: BIVE 보상이 ON이면 추첨 확정 즉시 당첨자 전원에게 자동 민팅 큐 적재된다 ([CEB-BO-004] 민팅 영역으로 위임). 경품 수령 여부와 무관하게 추첨 확정 시점에 일괄 발송.

2-7. 수령 가이드 / 배송 가이드 카드 (deliveryType 분기)

배송 타입에 따라 카드 제목과 본문 필드가 동적으로 전환된다.

2-7-1. 현장 수령일 때 — 카드 제목 "수령 가이드"

상단 2열:

  • 보상수령 기간 ("시작일 ~ 마감일")
  • 운영 시간 ("오픈 시각 ~ 마감 시각")

하단 3열 (다국어):

  • 한국어 — 수령장소 + 지참물
  • 영어 — 수령장소 + 지참물
  • 일본어 — 수령장소 + 지참물

각 항목 미입력 시 회색 안내 "(미입력)".

2-7-2. 배송 수령일 때 — 카드 제목 "배송 가이드"

상단 행 2개 (전체 너비):

  • 배송지 입력 마감일시 ("YYYY.MM.DD HH:mm")
  • 배송지 입력 폼 URL (외부 폼 링크, 클릭 시 새 탭 오픈)

다국어 영역 없음 (회원 측 외부 폼에서 입력받는 흐름).

2-8. 유의사항 카드

한국어/영어/일본어 3행 (줄바꿈 그대로 유지).

4-2. [게시하기] 버튼 활성화 조건

임시저장 상태에서 노출되는 [게시하기] 버튼은 다음 조건이 모두 충족될 때만 활성화된다. 1개라도 미충족이면 회색 처리 + 클릭 불가 상태로 표시된다.

#조건
1아티스트 선택
2대표 이미지 등록 (URL 또는 업로드)
3마감일시 입력
4당첨 인원 ≥ 1
5수령 방식 선택 (현장 수령 / 배송 수령, 운영 SSOT 2종)
6경품 단위 선택 (장 / 개)
7다국어 KO·EN·JA 세 언어 모두 입력: 타이틀(title) · 설명(desc) · 경품(prize) · 유의사항(notice) — 현장 수령일 때 추가: 수령 위치(pickup.location) · 수령 시 지참물(pickup.items)
8수령 가이드 (deliveryType 분기) — 현장 수령: pickup.startDt·endDt·openTime·closeTime 4종 모두 / 배송 수령: pickup.deliveryDeadlineDt·deliveryDeadlineTime·deliveryFormUrl 3종 모두
9BIVE 보상 조건: BIVE 보상 토글 OFF, 또는 ON + 민팅 이벤트 선택

비활성 상태 안내 UX: 별도 툴팁·경고 메시지 없음. 운영자는 [수정하기]로 이동하여 누락 항목을 직접 확인한다.

임시저장 vs 게시 다국어 정책 차이:

  • 임시저장 가능 조건 ([CEB-BO-RFL-101-CREATE] §4): 다국어 필드 1개 언어 이상 입력
  • 게시 가능 조건: 다국어 필드 KO·EN·JA 3개 언어 모두 입력
