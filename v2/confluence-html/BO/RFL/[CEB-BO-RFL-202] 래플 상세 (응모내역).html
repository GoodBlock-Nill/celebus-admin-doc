1. 화면 개요

래플 상세 페이지의 두 번째 탭. 응모한 회원의 누적 응모 현황을 운영자가 모니터링하고, 추첨대기 상태에서는 특정 응모자를 추첨에서 제외할 수 있다. 추첨 확정 전까지 제외는 자유롭게 토글 가능하며, 응모권 환급은 발생하지 않는다.

2. 화면 구성

2-1. 통계 카드 (가로 3개)

카드의미
누적 참여자 수응모한 회원 수 (중복 제외)
누적 사용 응모권 합계사용된 응모권 총합 (한 회원이 N장 응모하면 N으로 계상)
추첨 제외추첨 풀에서 제외 처리된 응모자 수. 추첨대기 상태에서 운영자가 토글한 결과

응모권 거래 자체는 [CEB-BO-010] 응모권(RFT) 영역에서 별도로 트래킹된다. 추첨 제외 처리 시 응모권 환급은 발생하지 않는다 (이미 사용된 응모권은 그대로 소진 처리).

2-2. 일괄 처리 액션 바 (추첨대기 + 선택 1건 이상에서만 노출)

체크박스로 응모자 1명 이상 선택 시 테이블 상단에 노출:

요소설명
선택 카운터"N건 선택됨" 텍스트
[선택 N건 추첨 제외]선택 응모자를 추첨 제외로 일괄 전환. 이미 제외 상태인 응모자는 변화 없음
[선택 N건 제외 해제]선택 응모자를 정상 상태로 일괄 전환. 이미 정상 상태인 응모자는 변화 없음
[선택 해제]선택 초기화 (상태 변경 없음)

노출 조건:

  • 래플 상태 = 추첨대기일 때만
  • 선택 응모자 1명 이상일 때만
  • 그 외 상태(임시저장·진행중·종료)에서는 액션 바 자체 미노출

2-3. 정렬·검색 바

위치요소옵션
좌정렬 드롭다운"응모일시 최근 순" (기본) / "응모권 많은 순" — 동률 정렬: "응모권 많은 순"에서 누적 응모권이 같은 회원은 최근 응모일시가 최신인 회원이 위에 위치
우검색 대상 드롭다운"닉네임" (기본) / "전화번호"
우키워드 입력 칸부분 일치 검색
우[초기화] 버튼정렬·검색 초기화 (선택 상태는 별도, 초기화 영향 없음)

컬럼정렬표시 내용
체크박스좌측 첫 컬럼헤더 = 현재 보이는 응모자 전체 선택 토글. 각 행 = 개별 선택. 추첨대기 상태에서만 활성, 그 외 상태에서는 비활성
유저 닉네임좌측응모 회원의 닉네임
전화번호좌측회원 전화번호 (예: 01012345678 형식)
최근 사용 응모권우측가장 최근 응모 1건에서 사용한 응모권 수량
최근 응모일시우측가장 최근 응모 시각 (YYYY.MM.DD HH:mm)
누적 사용 응모권우측본 래플에 응모한 응모권 총합 (강조)
상태우측 끝"정상" (회색 배지) / "추첨 제외" (rose 배지). 모든 상태에서 표시

행 클릭 동작:

  • 체크박스 영역 클릭 → 선택 토글만, 모달 미오픈
  • 그 외 빈 영역 클릭 → 응모내역 상세 모달 ([CEB-BO-RFL-202-MD-DETAIL]) 노출

제외된 응모자 시각 표시: "추첨 제외" 행은 배경에 옅은 rose 톤이 적용되어 정상 응모자와 구분된다.

2-5. 페이지네이션

  • 페이지당 20건 (필터·검색 결과 기준)
  • 항상 노출 — 응모자가 20건 미만이라 1페이지일 때도 ‹ 1 › 비활성 형태로 노출 (운영 사이트 정합)
  • 윈도잉 규칙 — 총 페이지 ≤ 7이면 전체 페이지 번호 노출, 8페이지 이상이면 1 ... cur-1 cur cur+1 ... last 윈도우 + 양 끝(...) 표시
  • 정렬·검색·키워드 변경 시 자동으로 1페이지로 리셋
  • 체크박스 "현재 페이지 전체 선택"은 현재 페이지에 보이는 응모자만 대상 (페이지 외 응모자는 선택에서 제외)

2-6. 빈 상태

  • 검색 결과 0건: "응모 내역이 없습니다." 메시지 노출
  • 응모자 자체가 없을 때 (임시저장 등): 동일 메시지

5. 검증 규칙 — 추첨 제외 정책

규칙내용
토글 가능 상태추첨대기 상태에서만 토글 가능. 진행중·종료에서는 잠금
사유 입력없음 — 토글 1회 클릭(또는 일괄 처리)으로 즉시 반영. 사후 감사 추적은 활동 로그로 대체
제외 해제가능 — 추첨 확정 전까지 자유롭게 정상 ↔ 제외 전환 가능
응모권 환급없음 — 이미 사용된 응모권은 그대로 소진 처리. 향후 별도 환급이 필요한 경우 응모권 영역([CEB-BO-RFT-201])에서 별도 흐름으로 처리
정렬 영향없음 — 제외 회원도 누적 응모권 많은 순/응모일시 최근 순 정렬에 정상 포함. 상태 컬럼·배경 톤으로만 구분
검색 영향없음 — 닉네임·전화번호 검색에 정상 노출
동결 시점추첨 확정 시 ([CEB-BO-RFL-203-MD-DRAW] 추첨 확정 클릭 시점) 모든 응모자의 제외 상태가 동결되어 종료 상태에서 토글 비활성

6. 활동 로그

액션메시지 형식

단건 해제래플 '{타이틀}'의 응모자 '{닉네임}'의 추첨 제외를 해제했습니다.
다중 제외래플 '{타이틀}'에서 응모자 {N}명을 추첨에서 제외했습니다.
다중 해제래플 '{타이틀}'에서 응모자 {N}명의 추첨 제외를 해제했습니다.

분류: 대=래플 / 중=추첨 / 소=추첨 제외(또는 추첨 제외 해제). 자세한 활동 로그 정책은 [CEB-BO-012] 영역 가이드 §7 참조.
