본 문서는 사이드바 "래플" 그룹의 래플 화면 영역 전체를 다룬다. 응모권 현황·변동 내역·응모권 발급 정책은 별도 [CEB-BO-010] RFT 영역 참조.

1. 영역 정의

래플(Raffle, 이하 RFL) 은 회원이 응모권을 사용해 응모하고, 마감 후 운영자가 추첨을 실행하여 당첨자에게 경품(필요시 BIVE NFT)을 지급하는 추첨형 콘텐츠 영역이다.

항목값
영역 코드RFL (Raffle)
영역 가이드본 문서 [CEB-BO-012]
화면 ID 패턴CEB-BO-RFL-{번호}[-{유형}]
사이드바 메뉴좌측 "래플" 그룹
분리 배경메뉴 재구성으로 기존 팬퀘스트 영역에서 분리. 운영 흐름상 응모권·BIVE와 함께 묶이는 것이 자연스러움

⚠️ 운영 용어 주의: 모든 한국어 라벨·본문에서 "래플"로 통일한다. 화면 명세 본문·헤더·Breadcrumb 등 사용자 가시 영역에 "Raffle" 영문을 혼용하지 않는다.

3. 접근 권한

본 영역의 모든 화면·액션에 대한 권한은 [관리자 > 권한관리]에서 설정된 권한을 따른다. 세부 권한 매트릭스(역할별 조회·생성·수정·삭제·추첨·숨김 등의 가능 여부)는 권한관리 화면에서 운영자가 직접 설정하며, 본 문서는 각 화면·모달에서 어떤 액션이 가능한지에 대한 정책만 정의한다.

권한이 없는 운영자가 진입한 경우 해당 액션 버튼은 미노출되거나 비활성 상태로 처리한다.

5-1. 추가 보상(BIVE) 선택 옵션

[CEB-BO-004] BIVE 영역에 등록된 민팅 캠페인 중에서 선택. 예시 (시연용):

캠페인 ID캠페인명
23V01D Welcome ED
24V01D Trivia Master
25V01D Final Boss
26V01D Prophet
27V01D 100 Days
28iKON FOUREVER Commemorative

5-2. 정책 상수

  • 1 래플 = 1 BIVE 매핑 (팬퀘스트 정책 준용)
  • BIVE 지급 대상 = 당첨자 한정 (미당첨자에게는 BIVE 미지급)
  • BIVE 자동 민팅 시점 = 추첨 확정 즉시 (당첨자 전원에게 일괄 민팅 큐 적재)
  • 다국어 게시 검증: 한국어/영어/일본어 3개 모두 필수
  • 임시저장 검증: 1개 언어 이상 입력이면 가능 (한국어 필수 아님)
  • 추첨 가중치: 회원의 사용 응모권 수에 정확 비례 (응모권 10장 = 1장 사용 회원보다 10배 높은 당첨 확률)
  • 재추첨 횟수: 무제한 (확정 전까지 자유롭게 시드 재계산)
  • 추첨 확정 후 되돌리기: 완전 불가 (한 번 확정은 최종)
  • 다른 래플 중복 당첨: 허용 (각 래플은 독립적)
  • 미당첨자 위로 보상: 없음 (응모권 환급·위로 BIVE 모두 없음)
  • 공통 페이지네이션 패턴: 모든 리스트·내역 테이블에 페이지당 20건(모달 내 시계열은 10건), 항상 노출(1페이지일 때도 ‹ 1 › 비활성). 윈도잉 규칙: 총 페이지 ≤ 7이면 전체 노출, 8 이상이면 1 ... cur-1 cur cur+1 ... last 윈도우 + .... 정렬·검색·필터 변경 시 1페이지로 자동 리셋. 운영 사이트와 동일

6. 상태 머신

래플은 다음 4상태를 순차적으로 거친다.

[생성 폼 작성]
   ↓ 임시저장
임시저장 ───── 게시 ─────→ 진행중 ───── 종료 ─────→ 추첨대기 ───── 추첨 확정 ───→ 종료
   │                          │                       │                               │
   삭제 가능                  수정 가능              추첨 가능                     숨김 처리·해제
상태의미운영자 가능 액션
임시저장미게시. 작성·검토 중수정 / 삭제 / 게시
진행중앱에 노출 + 회원 응모 가능수정 (응모자 보호 잠금 적용) / 종료
추첨대기응모 마감 + 추첨 대기추첨내역 탭에서 "추첨하기" / 응모내역 탭에서 추첨 제외 명단 토글 ([CEB-BO-RFL-202])
종료추첨 확정 + 결과 공개숨김 처리 / 숨김 해제 (앱 노출 토글). 추첨 제외 명단은 동결되어 변경 불가

자동 종료: 마감일시 도달 시 시스템이 진행중 → 추첨대기로 자동 전환. 운영자 수동 "종료"와 결과 동일. 자동 종료 발생 시 관리자 대시보드에 알림 노출 + 활동 로그 기록.

추첨 제외 명단 동결: 추첨대기 → 종료 전환(추첨 확정) 시점에 [CEB-BO-RFL-202]의 추첨 제외 토글 상태가 동결되어 종료 상태에서는 변경 불가. 추첨 풀(응모자 − 제외) 기준으로 당첨자·미당첨자가 산출되며, 제외 회원은 어느 명단에도 포함되지 않는다 (응모권 환급도 없음).

6-1. 응모자 0건 종료의 특수 처리

진행중·추첨대기 상태에서 응모자가 한 명도 없을 때 운영자가 "종료하기" 또는 마감일시 도래로 자동 종료 시:

  • 추첨대기 단계를 거치지 않고 즉시 종료 상태로 전환
  • 추첨 시드 미생성, 당첨자·미당첨자 데이터 없음
  • 추첨내역 탭에 "추첨 없이 자동 종료 — 입상자 없음" 표기 노출
  • BIVE 보상이 설정되어 있어도 민팅 발생하지 않음
