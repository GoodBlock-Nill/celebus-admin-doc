1. 화면 목적

운영자가 아티스트(그룹)별 팬덤 레벨을 수동 생성·관리하는 화면. 운영자는 [생성]으로 아티스트·시즌 단위 팬덤레벨을 직접 생성하고(준비), 상태(준비/진행중/종료)별로 조회하며, 개별 팬덤레벨 상세로 진입한다. 자동 생성되지 않는다.

진입 경로

  • 사이드 메뉴 > 아티스트 > 팬덤레벨

2.4 생성 모달

항목표시
아티스트Dropdown — 그룹 선택
시즌 연도Dropdown — 시즌 연도 선택 (기본: 올해). 기간은 해당 연도 1월 1일 ~ 12월 31일 자동 설정
시즌명Text Input — "{아티스트} 팬덤 N기 (YYYY)" 자동 제안(수정 가능)
액션[취소] / [생성] — 생성 시 준비 상태로 리스트 추가
  • 시즌 기간은 캘린더 연도(1.1~12.31)로 자동 설정된다. 동일 아티스트·동일 연도 중복 생성은 차단된다.
  • 레벨 곡선·보상 등 상세 항목은 생성 후 [CEB-BO-EVT-201] 상세에서 설정한다.

4. 규칙·로직

4.1 권한

  • 권한 그룹별 읽기·쓰기 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.

4.2 정렬·필터

  • 기본 정렬: 누적 덕력 내림차순.
  • 상태 필터 + 시즌 필터 + 그룹명 검색 = AND 조합. 그룹명 검색은 한국어·영어 통합 부분 일치.

4.3 상태 정의

상태의미앱 노출
준비생성 후 게시 전 (상세에서 곡선·보상 설정)미노출
진행중앱 노출·운영 중노출
종료시즌 마감미노출

운영자 상태 전환은 [게시](준비→진행중)만 수행하며, 종료는 12.31 자동(또는 [강제 종료])이다. 상세 정의는 [CEB-BO-EVT-000] §4.1.1 참조.

6. 활동 로그

목록 조회는 로그를 기록하지 않는다. 단, 팬덤레벨 생성은 로그를 기록한다. 곡선·보상·추첨·상태 전환 액션은 [CEB-BO-EVT-201]에서 발생한다.

액션로그 메시지 (한국어)
팬덤레벨 생성'{시즌명}' 팬덤레벨을 생성했습니다. (그룹: {그룹명})
