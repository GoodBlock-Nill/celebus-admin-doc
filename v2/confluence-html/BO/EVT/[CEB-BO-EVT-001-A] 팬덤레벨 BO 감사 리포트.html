5.1 보상 모델 재설계 (기존 SSOT 연계·단순화) ✅

추가 심층 검토 결과, 보상 설정이 기존 BO 보상 SSOT를 파악하지 못한 채 재발명됐음이 확인됐다 (운영 플로우 = 🔴 곤란). 근본 원인:

  • BIVE 민팅 이벤트의 "연결 기능"에 "아티스트 키우기 보상" 트리거가 이미 존재 ([CEB-BO-BIVE-203]) → 팬덤 레벨업 → BIVE 자동 민팅 아키텍처가 인프라에 이미 있음. 그런데 EVT는 "참조 연결: 바이브"를 라벨로만 둠.
  • RFL 래플 = 추첨 SSOT인데 EVT가 자체 [래플 추첨 실행]을 신설 → 추첨 이원화.

사용자 결정(2026-05-29)으로 보상 모델을 대폭 단순화·SSOT 연계 재설계:
