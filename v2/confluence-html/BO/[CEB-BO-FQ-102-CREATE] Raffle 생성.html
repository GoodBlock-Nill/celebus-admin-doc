1. 화면 목적

유저가 응모권을 사용해 응모 후 추첨(Draw)을 통해 보상을 받는 Raffle(추첨형 퀘스트)를 신규 생성. Quest와 달리 응모 정원·당첨 인원·배송 등 추첨 이벤트 특화 정보 입력. [CEB-BO-006] §7.2 라이프사이클 + §7.5 추첨 알고리즘 적용.

주요 기능

  • 응모 기간·당첨 인원·보상 지급 방식 등 추첨 특화 정보 설정
  • Split View UI (좌측 항목 리스트 + 우측 입력 패널)
  • Raffle 기본 정보 입력 (아티스트·마감·대표 이미지·타이틀·설명) — KO/EN/JP 3언어
  • Raffle 보상 입력 (당첨 인원·배송 타입·보상 상세·수령 가이드·유의사항)
  • 임시저장(Draft) 상태로 즉시 저장. 게시는 [CEB-BO-FQ-204-MD-OPEN] 통해 별도 진행

진입 경로

  • [CEB-BO-FQ-102] 팬퀘스트 메인 (Raffle 탭) > [생성하기] 버튼 클릭

4.5 기간·추첨 로직

  • 마감일: 현재 시점 +1일 이후만 허용 (Quest 생성과 동일 원칙)
  • 당첨 추첨 수: 1 이상의 양의 정수
  • 응모 정원 무제한 (응모권 보유자는 누구나 응모 가능, 단 추첨 시 [CEB-BO-006] §7.5 가중치 적용)
  • 시드 기반 결정론적 추첨 ([CEB-BO-006] §7.5) — 추첨 실행 시점에 시드 자동 생성

4.7 응모권 정책

[CEB-BO-006] §7.6 응모권 거래 코드 적용:

  • 응모권 1장 = 1엔트리
  • 다중 응모자(N장 사용)는 N엔트리 ([CEB-BO-006] §7.5 가중치)
  • 1명당 최대 1회 당첨 (다중 응모해도 1회만)
  • 응모권 차감 코드: TICKET_RAFFLE_USE

4.8 SUPER 추첨 가드레일 (생성 시점에는 영향 없음)

[CEB-BO-006] §4.2 — 생성 시점이 아닌 추첨 실행 시점에 적용. 본 화면에서는 가드레일 발동 조건만 안내:

  • 응모권 사용 누적 ≥ 10,000장 또는 보상 BIVE 발행 ≥ 100장 → 추첨 시 SUPER 2FA 재인증 필요
