1. 개요

BIVE 영역은 CELEBUS의 NFT/온체인 디지털 굿즈 (BIVE)를 관리한다. v1에서 4서브 영역(에디션관리·BIVE관리·민팅관리 캠페인·혜택관리)을 운영 중이며, v2에서 Quest·Raffle 보상에 BIVE 자동 민팅 연결을 신규 정의한다. v2 BO-000 §1.2 정의 COL (디지털 굿즈)과 분리하여 BIVE는 NFT 발행이 수반되는 자산을 다루고, COL은 비-NFT 디지털 자산(포토카드 이미지·배지 등)을 다룬다.

7. 영역 고유 정책

7.1 4서브 영역 구조

BIVE 영역
├── 1. 에디션관리 (101·201·101-Create)
│   └── 시즌별 에디션 발행 단위 정의 (예: V01D Genesis, V01D Season 2)
│   └── 발행 수량·이미지·메타데이터·아티스트 매핑
│
├── 2. BIVE관리 (102·202·202의 4탭)
│   └── 개별 BIVE 토큰 단위 조회 (에디션 산하)
│   └── 보유자 추적·기능 설정·민팅 이력
│
├── 3. 민팅관리 캠페인 (103·103-Create·203)
│   └── EVENT 탭 (캠페인 단위 일괄 민팅)
│   └── Ticket 탭 (Quest·Raffle 보상 자동 민팅)  ⭐ v2 신규
│
└── 4. 혜택관리 (104·204)
    └── BIVE 보유자 대상 혜택 (팬덤레벨업·독점 콘텐츠·할인 등)

1기본토큰 ID·에디션·이미지·메타데이터·발행일·민팅 트리거(수동/Quest/Raffle)
2기능설정토큰별 기능 토글 (혜택 활성화·전송 가능 여부·거래 잠금 등)
3민팅관리보유 이력 (현재 보유자·이전 보유자 추적·전송 이력)
4민팅이력트랜잭션 로그 (민팅 트랜잭션·전송 트랜잭션·소각 트랜잭션 — 가스비·해시 포함)

7.3 Quest·Raffle 보상 연결 (v2 핵심 신규)

Quest 완료자 또는 Raffle 당첨자에게 BIVE를 자동 민팅하여 지급하는 흐름.

연결 UI 위치

위치설정 방법
Quest 생성 화면 ([CEB-BO-FQ-101-Create])보상 설정 섹션에 [BIVE 자동 민팅] 토글 추가 → ON 시 캠페인 선택 드롭다운 노출
Raffle 추첨 화면 ([CEB-BO-FQ-204-Edit] 또는 추첨 모달)당첨자 보상 섹션에 [BIVE 자동 민팅] 토글 추가 → ON 시 캠페인 선택
민팅 캠페인 생성 ([CEB-BO-BIVE-103-Create])트리거 유형 선택: 수동(EVENT) / Quest(Ticket) / Raffle(Ticket) — 트리거에 따라 연결 Quest·Raffle 선택

양방향 호환: Quest·Raffle에서 시작하든, 캠페인에서 시작하든 동일하게 매핑됨.

자동 민팅 흐름

[Quest 완료 또는 Raffle 당첨]
    ↓
시스템: 보상 설정에서 [BIVE 자동 민팅] ON 확인
    ↓
민팅 캠페인 큐에 적재 (회원 ID·에디션·메타데이터)
    ↓
온체인 트랜잭션 발송 (회원 외부 지갑 또는 회원 매핑 지갑)
    ↓
[성공] 민팅 완료 → 회원 푸시 알림 + 활동 로그 + BIVE 보유 갱신
[실패] 트랜잭션 실패 → 운영자 푸시 알림 + 자동 재시도(3회) → 실패 시 운영자 수동 재시도(BIVE-203-MD-RETRY)

트랜잭션 실패·재시도 정책

상황처리
가스비 부족 (지갑 잔고 부족)자동 알림 → SUPER가 가스비 충전 후 재시도
회원 지갑 미연동회원에게 "지갑 연동 필요" 알림 → 연동 완료 시 재시도
블록체인 노드 오류자동 재시도 (10분 간격, 3회) → 실패 시 운영자 알림
메타데이터 IPFS 업로드 실패자동 재시도 (2회) → 실패 시 운영자 알림
영구 실패 (의도치 않은 오류)운영자 수동 재시도 ([BIVE-203-MD-RETRY]) — 사유 입력 필수, 활동 로그

자동 재시도 3회 실패 시 캠페인 [CEB-BO-BIVE-203] 상세에 ⚠️ 트랜잭션 실패 뱃지 노출 + 운영자 푸시 알림.
