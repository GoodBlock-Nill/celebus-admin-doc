# Sync Matrix — 기획서 ↔ 프로토타입 정합 매트릭스

각 화면별로 (기획서 §섹션·필드 ↔ 프로토타입 파일·컴포넌트·라인) 매핑을 명시한 운영 문서. 명세 또는 프로토타입 중 한쪽만 수정되어도 매트릭스를 통해 누락을 즉시 발견할 수 있다.

본 폴더는 `[feedback-spec-prototype-sync]` 메모리 룰의 **L4 단계**다.

## 파일 명세

- **위치**: `v2/sync-matrix/<화면 ID>.md` (예: `CEB-BO-APP-201.md`)
- **HTML 빌드 대상 외**: confluence-html 미러 미생성 (내부 운영 문서)
- **버전 관리**: git에 포함. 명세/프로토타입 변경 시 매트릭스도 같은 PR에 포함

## 상태 기호

| 기호 | 의미 |
|---|---|
| ✅ | 명세 ↔ 프로토타입 정합 (라벨·옵션·검증 모두 일치) |
| ⚠️ 누락 | 한쪽에만 존재 — 정정 필요 |
| 🔄 불일치 | 양쪽 모두 존재하나 라벨·옵션·검증 차이 |
| 🚫 폐기 | 명세 또는 프로토타입에서 의도적으로 제거된 필드 (이력 보존) |
| ⏸ 보류 | 사용자 결정 대기 |

## 운영 정책

### 신규 화면 작업 시
- 매트릭스 동시 신규 작성 (작업 PR에 포함)
- 명세 모든 입력 필드 → 매트릭스 행 1개씩 생성
- 프로토타입의 모든 input/select/checkbox 라벨 → 매트릭스 검증

### 기존 화면 수정 시
- 추가된 필드/라벨 → 신규 행
- 제거된 필드/라벨 → 행 삭제 또는 🚫 폐기 표시
- 옵션값 변경 → 행 수정 + 변경 이력 추가
- 누락 발견 시 → `⚠️ 누락` 상태로 명시 (향후 정정 작업 우선순위 추적)

### 작성 시점
- 매트릭스가 없는 화면을 작업할 때는 **본 작업에서 즉시 매트릭스 신규 작성**
- 별도 작업으로 미루지 않음 (메모리 룰 강제 사항)

### 검증 활용
- `v2/scripts/sync-check.sh <화면 ID>` 실행 시 본 매트릭스의 (명세 파일, 프로토타입 파일) 매핑을 자동 조회
- 매트릭스가 있으면 정확한 파일 매칭으로 검증
- 매트릭스가 없으면 휴리스틱 탐색 (정확도 낮음)
