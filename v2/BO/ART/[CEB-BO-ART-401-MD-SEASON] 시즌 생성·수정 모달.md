## 1. 화면 목적

운영자가 **시즌을 신규로 등록하거나 기존 시즌의 시즌명·시작일시를 수정**하는 모달이다. 생성·수정 두 모드를 한 모달에서 처리하며, 모드별로 입력 가능 필드와 기본값이 다르다.

- **누가**: 덕력 운영 권한이 있는 백오피스 관리자
- **왜**: 그룹별 1년 운영 단위 시즌 정보를 등록·갱신해야 정산·랭킹·보상이 시작됨
- **결과**: 저장 시 시즌 리스트·시즌 상세 화면에 즉시 반영

## 2. UI 설명

### 2.1 모달 레이아웃

```
┌───────────────────────────────────────────────┐
│ [모달 타이틀]                          [X]   │
│  신규 시즌 등록  (또는)  시즌 정보 수정       │
├───────────────────────────────────────────────┤
│                                               │
│  그룹                                         │
│  [V01D ▾]   (수정 모드는 회색·수정 불가)     │
│                                               │
│  시즌명 *                                     │
│  [예: V01D 2026 시즌                       ]  │
│  (인라인 에러: 시즌명을 입력하세요)           │
│                                               │
│  시작일시 *                                   │
│  [2026.06.01]  [00:00]                        │
│  (진행중·종료 시즌은 회색·수정 불가)          │
│                                               │
│  종료일시 (자동 계산)                         │
│  2027.05.31 23:59 — 시작일시 + 1년 자동       │
│                                               │
├───────────────────────────────────────────────┤
│                       [취소]   [저장]         │
└───────────────────────────────────────────────┘
```

### 2.2 진입 모드

| 모드 | 진입 경로 | 모달 타이틀 |
| --- | --- | --- |
| 생성 | 랭킹 시즌 설정 [CEB-BO-ART-401-SEASON]의 [+ 신규 시즌] | "신규 시즌 등록" |
| 수정 | 시즌 상세 [CEB-BO-ART-401-DETAIL] 상단 [수정] | "시즌 정보 수정" |

### 2.3 입력 필드 — 모드별 동작

| 필드 | 컨트롤 | 생성 모드 | 수정 모드 |
| --- | --- | --- | --- |
| 그룹 | 선택 메뉴 (활성 그룹 5종 중 1종) | 활성 — 운영자가 선택 | 수정 불가 (회색 텍스트로 표시) |
| 시즌명 | 입력란 ("예: V01D 2026 시즌" 안내) | 활성 — 1~50자 | 활성 — 1~50자 |
| 시작일시 | 날짜 + 시간 선택 | 활성 — 운영자가 선택 | 시즌 상태가 "예정"일 때만 활성. "진행중"·"종료"는 수정 불가 |
| 종료일시 | 자동 계산 안내 | 수정 불가 — "{시작일시 + 1년} (자동)" 형식으로 표시 | 동일 |

### 2.4 시즌 1년 운영 단위 정책

- 종료일시 자유 입력은 폐지. 시즌은 운영 단위 일관성을 위해 시작일시로부터 **정확히 1년** 운영
- 모달 하단에 "종료: {시작 + 1년} (자동)" 안내가 표시되어 운영자가 종료 시점을 사전 확인 가능
- 시작일시 변경 시 종료일시 안내도 즉시 갱신

### 2.5 수정 모드 — 시작일시 변경 제약

- 시즌이 "진행중" 또는 "종료" 상태일 때 시작일시를 변경하면, 시즌 시작일시 기준으로 자동 생성된 12개월 보상 시퀀스(년월)가 기존 보상 데이터와 매칭되지 않아 운영 무결성에 영향을 준다
- 따라서 "진행중"·"종료" 시즌은 시즌명만 변경 가능, 시작일시는 회색 처리로 수정 불가 안내
- "예정" 시즌은 시작일시 자유 변경 허용 (보상 설정이 아직 운영되지 않음)

### 2.6 인라인 검증 메시지

운영자가 어떤 필드가 실패인지 즉시 파악할 수 있도록 필드별 인라인 에러 메시지를 표시한다. 각 필드는 **포커스 후 첫 이탈 시점부터만** 에러를 노출하여 모달 진입 직후 빈 폼에서 에러가 폭주하는 현상을 회피한다.

| 필드 | 인라인 에러 메시지 |
| --- | --- |
| 시즌명 | 빈 값: "시즌명을 입력하세요" / 50자 초과 시 입력 자체가 차단 + "50자 이하로 입력하세요" |
| 시작일시 | 빈 값: "시작일시를 선택하세요" |
| 기간 충돌 | "해당 기간이 같은 그룹의 다른 시즌과 겹칩니다" |

### 2.7 푸터

| 액션 | 동작 |
| --- | --- |
| [취소] | 모달 닫기. 입력값 폐기. 부모 화면 상태 유지 |
| [저장] | 검증 통과 시 저장 + 모달 닫기 + 토스트. 검증 실패 시 첫 오류 필드로 포커스 이동 |

## 3. 기능 설명

운영자가 이 모달에서 할 수 있는 동작:

1. **신규 시즌 등록** — 그룹·시즌명·시작일시 입력 후 저장
2. **시즌명 수정** — 모든 상태 시즌에서 가능
3. **시작일시 수정** — "예정" 시즌에 한해 가능
4. **종료일시 자동 확인** — 시작일시 + 1년이 자동 계산되어 표시
5. **저장·취소** — 푸터 액션으로 처리

## 4. 로직·플로우

### 4.1 생성 모드 진입

1. 랭킹 시즌 설정에서 [+ 신규 시즌] 클릭 → 본 모달 열림
2. 모달 타이틀 "신규 시즌 등록" + 모든 필드 빈 상태
3. 그룹·시즌명·시작일시 입력 가능 / 종료일시는 시작일시 입력 시 자동 갱신

### 4.2 수정 모드 진입

1. 시즌 상세 상단 [수정] 클릭 → 본 모달 열림
2. 모달 타이틀 "시즌 정보 수정" + 기존 값 자동 채움
3. 그룹은 회색·수정 불가
4. 시작일시는 시즌 상태별 분기:
   - 예정 시즌: 활성
   - 진행중·종료 시즌: 회색·수정 불가
5. 시즌명은 모든 상태에서 활성

### 4.3 입력 → 검증 → 저장

1. 운영자가 필드 입력
2. 각 필드 포커스 이탈 시 인라인 검증 메시지 노출 (touched 이후만)
3. [저장] 클릭 시 전체 검증 재실행:
   - 시즌명 1~50자
   - 시작일시 유효 일시
   - 같은 그룹·기간 중복 시즌 없음
4. 검증 통과 시:
   - 저장 처리
   - 모달 닫기
   - 토스트 "시즌이 저장되었습니다."
   - 부모 화면 자동 갱신
5. 검증 실패 시:
   - 첫 오류 필드로 포커스 이동
   - 인라인 에러 메시지 노출

### 4.4 취소

1. [취소] 클릭 또는 모달 외부 클릭 또는 [X] 클릭
2. 입력값 모두 폐기 후 모달 닫기 (확인 모달 없음 — 시즌 저장 모달은 가벼운 입력 단위)

## 5. 검증·예외 상황

### 5.1 입력 검증

| 필드 | 룰 |
| --- | --- |
| 그룹 (생성 모드) | 활성 그룹 5종 중 1종 필수 |
| 시즌명 | 1~50자 필수. 공백만 입력 불가 |
| 시작일시 | 유효한 일시 필수 |
| 종료일시 | 입력 불가 — 시작일시 + 1년 자동 산출 |

### 5.2 동일 그룹 진행중 1개 제약

- 같은 그룹에 진행중 시즌은 1개만 운영 가능
- 생성 시 진행중 시즌이 이미 있으면 토스트 "해당 그룹에 이미 진행중인 시즌이 있습니다." + 모달 유지

### 5.3 기간 충돌

- 같은 그룹의 다른 시즌(예정·진행중·종료 무관)과 기간이 겹치면 시즌명 필드 하단에 인라인 에러 "해당 기간이 같은 그룹의 다른 시즌과 겹칩니다" 노출 + [저장] 차단

### 5.4 권한별 동작

- 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다
- 쓰기 권한 없으면 모달 자체가 호출되지 않음 (부모 화면의 [+ 신규 시즌]·[수정] 버튼 미노출)

### 5.5 활동 로그

| 동작 | 로그 메시지 |
| --- | --- |
| 시즌 신규 등록 | 덕력 시즌 '{그룹명} - {시즌명}'을(를) 등록했습니다. (기간: {시작일시} ~ {종료일시}) |
| 시즌 정보 수정 | 덕력 시즌 '{그룹명} - {시즌명}'의 정보를 수정했습니다. |
