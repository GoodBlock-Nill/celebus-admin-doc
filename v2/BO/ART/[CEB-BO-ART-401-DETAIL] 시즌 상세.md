## 1. 화면 목적

운영자가 **시즌(1년) 내 매월 별도의 덕력 랭킹 보상을 설정**하고, **정산이 완료된 월의 지급 결과를 조회·관리**하는 2탭 화면이다. 12개월 각각에 대해 보상 구간을 자유롭게 구성하고, 한 구간 안에 5종(배송 수령·현장 수령·BIVE NFT·응모권·덕력) 상품을 자유롭게 배치할 수 있다. 정산 완료된 월은 회원·상품 단위로 지급 결과를 확인하고 수동 지급 상품(배송·현장)의 진행 상태를 관리할 수 있다.

- **누가**: 덕력 운영 권한이 있는 백오피스 관리자
- **왜**:
  - 매월 말일 자동 정산 시 회원에게 지급될 보상을 사전 설정해야 함
  - 정산 완료 후 누가 어떤 등수로 어떤 상품을 받았는지 확인하고 수동 지급 상품의 진행 상태를 관리해야 함
- **결과**:
  - 미정산 월의 구간·상품 설정이 다음 정산 시점부터 자동 적용됨
  - 정산 완료 월의 지급 내역에서 운영자가 수동 지급 상품의 상태를 갱신하고 메모를 기록

## 월별 보상 설정 탭 (§2.5 ~ §2.10)

### 2.5 월별 보상 12개월 — 아코디언

시즌 시작일시 기준 12개월의 년월 시퀀스를 자동으로 만든다.

| 시즌 시작일시 | 12개월 시퀀스 |
| --- | --- |
| 2026.01.01 | 2026.01 ~ 2026.12 |
| 2026.05.01 | 2026.05 ~ 2027.04 |

**가독성 정책**: 12개월 카드는 매월 다구간·다상품으로 페이지가 매우 길어진다. 아코디언 패턴으로 접고 펼친다. 항상 1개월만 자동 펼침되며 나머지 11개월은 접힘. 헤더 클릭으로 토글한다.

**자동 펼침 정책 — 시즌 상태별 분기**

| 시즌 상태 | 이번 달이 시즌 12개월 범위 안 | 이번 달이 시즌 12개월 범위 밖 |
| --- | --- | --- |
| 예정 | (해당 없음 — 예정 시즌은 시작 전이라 이번 달이 시즌 안일 수 없음) | 시즌 첫 월 자동 펼침 |
| 진행중 | 이번 달 자동 펼침 | (해당 없음) |
| 종료 | 정산 완료된 마지막 월 자동 펼침 (정산 완료 0개월이면 시즌 첫 월) | 정산 완료된 마지막 월 자동 펼침 (정산 완료 0개월이면 시즌 첫 월) |

**아코디언 헤더 표시**

| 상태 | 표시 |
| --- | --- |
| 접힌 헤더 요약 | "년월 · 상태 배지 · 구간 N개 · 상품 M개" — 클릭으로 펼침 |
| 펼친 헤더 | 동일 요약 + 본문 노출 — 클릭으로 접기 |

### 2.6 월 카드 상태별 표시 — 시즌 상태 × 월 시점 × 월 정산 매트릭스

각 월 카드는 시즌 상태·월 시점(현재 시각 기준)·월 정산 여부의 조합에 따라 다음과 같이 분기한다.

**용어 정의**

| 용어 | 정의 |
| --- | --- |
| 지난 월 | 현재 시각 기준 년월보다 이전 년월 (예: 오늘 2026.05.15 → 지난 월 = 2026.04 이하) |
| 이번 달 | 현재 시각 년월 (예: 2026.05) |
| 미래 월 | 현재 시각 년월보다 이후 년월 (예: 2026.06 이상) |

**상태 매트릭스 (7행)**

| 시즌 상태 | 월 시점 | 월 정산 | 표시 | 가능 액션 |
| --- | --- | --- | --- | --- |
| 예정 | 미래 (시즌 시작 전이므로 모든 월이 미래) | 미정산 | "수정 가능" 배지 | 편집 + [월 보상 저장] |
| 진행중 | 지난 월 | 정산 완료 | 🔒 + "정산 완료 (yyyy.mm.dd HH:mm)" + 회색 음영 | 조회만 |
| 진행중 | 지난 월 | 미정산 (정산 지연) | 🔒 + "지난 월 — 수정 불가" + 회색 음영 | 조회만 |
| 진행중 | 이번 달 | 미정산 | "수정 가능" 배지 | 편집 + [월 보상 저장] |
| 진행중 | 미래 월 | 미정산 | "수정 가능" 배지 | 편집 + [월 보상 저장] |
| 종료 | 모든 월 | 정산 완료 | 🔒 + "정산 완료 (yyyy.mm.dd HH:mm)" + 회색 음영 | 조회만 |
| 종료 | 모든 월 | 미정산 | 🔒 + "시즌 종료로 인해 수정 불가" + 회색 음영 | 조회만 (보존된 설정 표시) |

**핵심 규칙**

- 편집 가능 조건은 단 두 가지: ① 예정 시즌의 모든 월 OR ② 진행중 시즌의 이번 달·미래 월
- 그 외 모든 경우는 잠금 (조회만)
- 진행중 시즌의 **지난 월은 정산 완료 여부와 무관하게 잠금** — 자동 정산이 지연되더라도 운영자 임의 변경을 차단하는 안전망
- 종료 시즌은 정산 완료·미정산 모두 잠금 — 신규 적립·소비가 일어나지 않으므로 수정 무의미 ([CEB-BO-ART-401-MD-CLOSE] §4.4 정합)

**잠금 상태 컨트롤 동작 (위 표에서 "조회만"인 모든 행)**

- [+ 구간 추가]·[구간 삭제]·[+ 상품 추가]·[상품 삭제]·[월 보상 저장] 모두 미노출
- 기존 입력 컨트롤(선택 메뉴·입력란·날짜·시간·숫자 입력)은 회색 처리 + 수정 불가
- 다국어 입력 영역도 회색 처리. 탭 전환은 조회 목적으로 허용하되 입력은 차단
- 보존된 기존 구간·상품 데이터는 그대로 노출되어 운영자가 정책 이력을 확인할 수 있도록 함

### 2.7 구간(대상 기준) 입력

각 월 카드 내부에 구간을 자유롭게 N개 추가한다. **한 구간 = 대상 기준 1개 + 보상 상품 N개 묶음**.

> 본 절의 모든 추가·수정·삭제 액션은 §2.6 매트릭스에서 **편집 가능한 월 카드 안에서만** 노출된다. 잠금 월 카드는 §2.6 끝의 **잠금 상태 컨트롤 동작** 규칙을 따른다.

**구간 상단 — 대상 기준 필드**

| 컬럼 | 컨트롤 | 검증 |
| --- | --- | --- |
| 대상 기준 | 선택 메뉴 (등수 / 퍼센트 / 등수범위) | 필수 |
| 값 | 입력란 — 등수: 양의 정수(예: 1) / 퍼센트: 1~100(예: 10) / 등수범위: `N-M` 형식(예: 2-10, N<M) | 필수, 형식 정합 |
| 액션 | [구간 삭제] | 구간 1건 + 그 안의 상품 모두 함께 제거 |

**구간 푸터 — 상품 관리**

- [+ 상품 추가] — 빈 상품 1건 추가 (기본 종류: 배송 수령)
- 한 구간 안 최소 1개 상품 필수. 0개면 [월 보상 저장] 차단

### 2.8 보상 상품 종류 5종 분기 입력

| 배송 마감 시간 | 시간 선택 (HH:mm) | 필수 |
| 배송 폼 URL | 입력란 (외부 폼 주소) | 필수, 올바른 URL 형식 |

| 필드 | 컨트롤 | 검증 |

### 2.9 다국어 입력 정책

- 모든 상품의 상품명·장소·지참물은 [CEB-000] §5 다국어 정책에 따라 한국어·영어·일본어 3언어 필수 입력
- 다국어 입력 영역은 백오피스 공통 컴포넌트를 그대로 사용 (래플·기억저장소·아티스트 화면 통일)
- 미입력 언어는 다국어 탭 우상단에 빨간 점 알림으로 표시
- 저장 시 모든 언어 입력이 완성되어 있어야 함

### 2.10 시각적 스타일

- 잠금 월 카드 (§2.6 매트릭스 "조회만" 행): 잠금 아이콘 + 회색 음영 (수정 불가 인지)
- 편집 가능 월 카드: 흰 배경 + "수정 가능" 배지
- 아코디언 헤더 hover 시 강조
- 빈 상태 (구간 0건, 편집 가능 카드): "보상 구간이 없습니다. [+ 구간 추가]로 시작하세요."
- 빈 상태 (구간 0건, 잠금 카드): "설정된 보상 구간이 없습니다."

## 월별 보상 내역 탭 (§2.11 ~ §2.13)

### 2.11 필터·통계

정산이 완료된 월의 지급 결과(누가 어떤 등수로 어떤 상품을 받았는지)를 회원·상품 단위로 조회한다.

**월 선택 드롭다운**

| 항목 | 동작 |
| --- | --- |
| 옵션 | 본 시즌의 정산 완료된 월만 (최신순) |
| 기본값 | 가장 최근 정산 완료 월 |
| 빈 상태 | 정산 완료 월이 0개면 드롭다운 비활성화 + 본문에 "정산 완료된 월이 없습니다." 안내 (탭 본문 전체 빈 상태로 처리) |

**통계 카드 (3개 — 선택한 월 기준)**

| 카드 | 표시 |
| --- | --- |
| 정산 회원 수 | 해당 월에 1건 이상 받은 회원 수 |
| 총 지급 상품 수 | 해당 월의 모든 회원·상품 조합 합계 (테이블 총 행 수) |
| 지급 상태 분포 | "지급완료 N · 지급대기 N · 지급실패 N · 재지급대기 N" 한 줄 표시 |

### 2.12 내역 테이블

**컬럼 (8개)**

| 컬럼 | 표시 |
| --- | --- |
| 순위 | 1·2·3위는 메달 색상 강조 (금·은·동) / 4위 이하는 회색 숫자. 기본 오름차순 정렬 |
| 회원 | 회원 닉네임 — 클릭 시 회원 상세 [CEB-BO-USR-201]로 새 창 이동 |
| 등수 기준 | 대상 기준·값을 사람 친화적 표현으로 — 등수: "1위" / 등수범위: "2위~10위" / 퍼센트: "상위 10%" |
| 상품 종류 | 5종 배지 — 배송 수령 / 현장 수령 / BIVE NFT / 응모권 / 덕력 |
| 상품명 | 한국어 상품명 (영어·일본어는 hover 시 툴팁으로 노출) |
| 지급 일시 | 정산 완료 시각 또는 운영자 처리 시각 (YYYY.MM.DD HH:mm). 지급 대기 상태면 "—" |

| 액션 | **수동 지급 상품(배송 수령·현장 수령)에만** [상태 변경] 버튼 노출. 자동 지급 상품(BIVE NFT·응모권·덕력)은 액션 컬럼 비워둠 |

**행 단위**

- 한 행 = **한 상품 지급 건**. 한 회원이 한 구간 안의 여러 상품을 받으면 행이 여러 개 생성됨
- **한 회원은 §5.2 중복 지급 금지 규칙에 따라 매칭되는 여러 구간 중 가장 상위 1개 구간의 모든 상품만 받는다**. 따라서 본 컬럼의 "등수 기준"은 그 회원에게 적용된 단일 구간을 표시하며, 한 회원의 여러 행은 모두 같은 등수 기준을 공유한다 (구간 안에 상품이 N개일 때만 행이 N개로 분리)

**페이지네이션**

- 20건/페이지
- 페이지 이동 시 정렬·필터 상태 유지

### 2.13 액션 — 지급 상태 변경

[상태 변경] 클릭 → [CEB-BO-ART-401-MD-PAYOUT] 지급 상태 변경 모달 진입.

- 모달에서 새 상태와 메모를 입력하고 저장
- 저장 성공 시 토스트 "지급 상태가 변경되었습니다." + 해당 행 즉시 갱신 + 통계 카드 재계산
- 자동 지급 상품(BIVE NFT·응모권·덕력)은 [상태 변경] 버튼 자체가 미노출이므로 모달 진입 불가

## 3. 기능 설명

운영자가 이 화면에서 할 수 있는 동작:

1. **시즌 정보 조회** — 그룹·시즌명·기간·상태·정산 완료 카운트 확인
2. **시즌 수정·종료** — 상단 액션으로 시즌 자체 정보·상태 변경 (시즌 상태별 노출 분기)
3. **탭 전환** — 월별 보상 설정 ↔ 월별 보상 내역 (§2.4)
4. **월별 보상 펼침/접기** (설정 탭) — 12개월 아코디언 자유 토글 (§2.5 자동 펼침 정책으로 1개월 자동 펼침)
5. **편집 가능 월 보상 편집** (설정 탭) — §2.6 매트릭스의 "편집 + [월 보상 저장]" 행에 해당하는 월(예정 시즌 전 월 OR 진행중 시즌의 이번 달·미래 월)에서 구간·상품 추가·수정·삭제·종류 전환
6. **월 보상 저장** (설정 탭) — 한 월 단위로 구간·상품 일괄 저장
7. **잠금 월 조회** (설정 탭) — 정산 완료 월·진행중 시즌의 지난 월·종료 시즌의 전 월은 조회만 가능
8. **월별 지급 내역 조회** (내역 탭) — 정산 완료 월의 회원·상품 단위 지급 결과 + 통계 카드 확인
9. **수동 지급 상품 상태 변경** (내역 탭) — 배송 수령·현장 수령 상품의 지급 상태를 운영자가 갱신 + 메모 입력 (§2.13)
10. **회원 상세 진입** (내역 탭) — 회원 닉네임 클릭 시 새 창으로 회원 상세 진입
11. **목록으로 복귀** — 시즌 리스트로 돌아가기

## 4. 로직·플로우

### 4.1 화면 진입 → 초기 표시

1. 시즌 리스트에서 시즌 행 클릭으로 본 화면 진입
2. 시즌 정보 카드 + 12개월 아코디언 표시
3. §2.5 자동 펼침 정책에 따라 1개월만 자동 펼침, 나머지 11개월은 접힌 상태
   - 예정 시즌 → 시즌 첫 월
   - 진행중 시즌 → 이번 달
   - 종료 시즌 → 정산 완료된 마지막 월 (정산 완료 0개월이면 시즌 첫 월)
4. §2.6 매트릭스에 따라 각 월 카드의 잠금·편집 가능 표시 분기

### 4.2 월 카드 펼침/접기

1. 접힌 헤더 클릭 → 본문 펼침
2. 펼친 헤더 클릭 → 본문 접힘
3. 접힌 상태에서도 헤더 요약 ("구간 N개 · 상품 M개")으로 현황 파악 가능

### 4.3 편집 가능 월 — 구간 추가

§2.6 매트릭스 "편집 + [월 보상 저장]" 행에 해당하는 월에서만 동작.

1. [+ 구간 추가] 클릭 → 빈 구간(상품 0개) 1건 추가
2. 대상 기준 미선택 + 값 빈 상태로 추가됨
3. 한 구간 안 최소 1개 상품이 필수이므로, 빈 구간만 추가하고 저장하면 차단됨

### 4.4 편집 가능 월 — 구간 안 상품 추가

1. 구간 내부 [+ 상품 추가] 클릭 → 빈 상품 1건 추가 (기본 종류: 배송 수령)
2. 종류 선택 메뉴 변경 시 종류별 폼이 분기되어 노출 (이전 입력값은 폐기)

### 4.5 편집 가능 월 — 저장

1. [월 보상 저장] 클릭
2. 해당 월의 모든 구간·상품 검증 (대상 기준·값 형식·상품 종류·종류별 필드·다국어 완성)
3. 검증 통과 시 토스트 "월별 보상이 저장되었습니다." + 월 카드 갱신
4. 검증 실패 시 첫 오류 위치로 스크롤 + 인라인 에러 메시지

### 4.6 잠금 월 — 조회만

§2.6 매트릭스 "조회만" 행에 해당하는 모든 월(정산 완료 / 진행중 시즌 지난 월 / 종료 시즌의 전 월).

1. 잠금 상태로 기존 구간·상품 노출 (모든 입력 컨트롤 수정 불가)
2. [+ 구간 추가]·[+ 상품 추가]·[구간 삭제]·[상품 삭제]·[월 보상 저장] 모두 미노출
3. 잠금 사유에 따른 안내 노출:
   - 정산 완료: "정산 완료 (yyyy.mm.dd HH:mm)"
   - 진행중 시즌 + 지난 월 + 미정산: "지난 월 — 수정 불가"
   - 종료 시즌 + 미정산: "시즌 종료로 인해 수정 불가"

### 4.7 상단 [수정] 클릭

1. 시즌 생성·수정 모달 [CEB-BO-ART-401-MD-SEASON] 열림 (수정 모드)
2. 시즌명·시작일시 변경 → [저장] 시 토스트 "시즌이 저장되었습니다." + 본 화면 갱신
3. 진행중·종료 시즌은 시작일시 수정 불가 (시즌명만 변경 가능)

### 4.8 상단 [종료] 클릭

1. 시즌 종료 확인 모달 [CEB-BO-ART-401-MD-CLOSE] 열림
2. [종료 확인] 시 시즌 상태 "종료"로 전환 + 토스트 "시즌이 종료되었습니다." + 본 화면 갱신

### 4.9 상단 [목록으로] 클릭

1. 랭킹 시즌 설정 [CEB-BO-ART-401-SEASON]로 이동
2. 본 화면의 아코디언 펼침 상태는 보존하지 않음 (재진입 시 §2.5 자동 펼침 정책 재적용)

### 4.10 탭 전환

1. 탭 메뉴에서 [월별 보상 설정] 또는 [월별 보상 내역] 클릭
2. URL 쿼리 `?tab=setting` 또는 `?tab=history`로 갱신
3. 본문 영역이 해당 탭 콘텐츠로 교체. 시즌 정보 카드와 페이지 헤더는 두 탭 공통 유지
4. 새로고침·공유 시 URL 쿼리 기준으로 동일 탭 복원. 쿼리 누락·잘못된 값이면 기본 탭(`setting`)으로 복귀

### 4.11 내역 탭 — 진입 + 월 선택

1. 내역 탭 진입 시 본 시즌의 정산 완료 월 목록을 최신순으로 드롭다운에 채움
2. 정산 완료 월이 1개 이상이면 가장 최근 월 자동 선택
3. 선택 월의 통계 카드 + 내역 테이블 + 페이지네이션 로드 (1페이지)
4. 운영자가 다른 월 선택 시 즉시 재로드 + 페이지 1 복귀
5. 정산 완료 월이 0개이면 드롭다운 비활성화 + 본문 빈 상태 "정산 완료된 월이 없습니다."

### 4.12 내역 탭 — 상태 변경

1. 수동 지급 상품(배송 수령·현장 수령)의 [상태 변경] 버튼 클릭
2. 지급 상태 변경 모달 [CEB-BO-ART-401-MD-PAYOUT] 열림 (현재 상태·메모 prefill)
3. 새 상태와 메모 입력 후 [저장]
4. 검증 통과 시 토스트 "지급 상태가 변경되었습니다." + 해당 행 상태·일시·메모 즉시 갱신 + 통계 카드 재계산
5. 자동 지급 상품(BIVE NFT·응모권·덕력)은 [상태 변경] 버튼 자체 미노출 — 본 플로우 진입 불가

### 4.13 내역 탭 — 회원 닉네임 클릭

1. 새 창으로 회원 상세 [CEB-BO-USR-201] 진입 (`/members/{회원ID}?tab=basic`)
2. 현재 내역 탭 상태(선택 월·페이지)는 유지

## 5. 검증·예외 상황

### 5.1 입력 검증

| 항목 | 룰 |
| --- | --- |
| 구간 — 대상 기준 | 등수 / 퍼센트 / 등수범위 중 1종 필수 |
| 구간 — 값 | 등수: 양의 정수 / 퍼센트: 1~100 / 등수범위: `N-M`(N<M) |
| 구간 — 상품 수 | 한 구간 최소 1개 필수. 0개면 [월 보상 저장] 차단 |
| 상품 — 종류 | 배송 수령 / 현장 수령 / BIVE NFT / 응모권 / 덕력 중 1종 필수 |
| 상품 — 상품명 | 한국어·영어·일본어 모두 입력 (각 50자 이하) |
| 배송 수령 — URL | 올바른 URL 형식. 잘못된 형식 시 인라인 에러 "올바른 URL 형식이 아닙니다" |
| 현장 수령 — 기간 | 수령 시작일 ≤ 수령 종료일 |
| BIVE NFT — 민팅 이벤트 | 디폴트 미선택. 미선택 상태 저장 시 차단 ("민팅 이벤트를 선택해주세요") |
| 응모권 / 덕력 — 수량 | 양의 정수 |

### 5.2 비즈니스 규칙

- **편집 가능 조건**: ① 예정 시즌의 모든 월 OR ② 진행중 시즌의 이번 달·미래 월. 이 조건에 해당하는 월만 구간·상품 추가·수정·삭제 + [월 보상 저장] 가능 (§2.6 매트릭스)
- **지난 월 잠금 (안전망)**: 진행중 시즌에서 현재 시각보다 과거 년월은 정산 완료 여부와 무관하게 모두 조회만 가능. 자동 정산이 지연되더라도 운영자 임의 변경 차단
- **정산 완료 월 잠금**: [CEB-000] §5.2.1 월간 정산 SSOT — 매월 말일 23:59 서버 자동 정산. 정산 완료 후에는 보상 변경 불가 (이력 무결성)
- **종료 시즌 전체 잠금**: 시즌이 종료되면 모든 월(정산 완료·미정산 무관) 조회만 가능. 미정산 월의 보상 설정은 데이터로 보존되나 신규 적립·소비가 일어나지 않음 ([CEB-BO-ART-401-MD-CLOSE] §4.4 정합)
- **빈 보상 (구간 0건)**: 보상 미설정 월은 정산 시 보상 미지급
- **저장 즉시 적용**: 저장 성공 시 다음 정산 시점부터 새 구간·상품 자동 적용
- **중복 지급 금지**: 한 회원이 정산 월에 여러 구간에 동시 매칭되어도, **가장 상위 1개 구간의 모든 상품만** 지급받는다. 매칭 우선순위:
  - ① 타입 우선순위: 등수 > 등수범위 > 퍼센트
  - ② 같은 타입 안에서는 값이 작은 것 우선 (등수 1 > 등수 2 / 등수범위 2-10 > 등수범위 11-50 / 퍼센트 5 > 퍼센트 10)
  - 예: 1위 회원은 [등수 1]만 받고 [상위 10%] 중복 안 받음. 5위 회원은 [등수범위 2-10] 받고 [상위 20%] 중복 안 받음
  - 본 규칙은 정산 시점에 시스템이 자동 적용 — 운영자가 별도로 처리할 액션 없음

### 5.4 권한별 동작

- 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다
- 쓰기 권한 없으면 [수정]·[종료]·[+ 구간 추가]·[+ 상품 추가]·[구간 삭제]·[상품 삭제]·[월 보상 저장] 모두 미노출 (조회만 가능)
- 다음 잠금 조건에 해당하는 월 카드는 쓰기 권한 보유 여부와 무관하게 [+ 구간 추가]·[+ 상품 추가]·[구간 삭제]·[상품 삭제]·[월 보상 저장] 모두 미노출 (잠금 조건 자체가 우선):
  - 시즌 상태가 종료
  - 진행중 시즌의 지난 월 (정산 완료 여부 무관)
  - 정산 완료 월
- 외부 협력사 권한자는 본인 매핑 소속사 산하 그룹의 시즌만 조회 가능 (쓰기 불가)
