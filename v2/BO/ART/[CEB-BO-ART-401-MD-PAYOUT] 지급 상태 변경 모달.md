## 1. 화면 목적

운영자가 **수동 지급 상품(배송 수령·현장 수령)의 지급 상태를 갱신하고 메모/사유를 기록**하는 모달이다. 자동 지급 상품(BIVE NFT·응모권·덕력)은 정산 시점에 즉시 지급 완료 처리되므로 본 모달의 대상이 아니다.

- **누가**: 덕력 운영 권한이 있는 백오피스 관리자
- **왜**: 배송·현장 수령 상품은 실제 발송·수령 진행 상황에 따라 운영자가 수동으로 상태를 갱신해야 함
- **결과**: 지급 상태가 갱신되고 활동 로그에 변경 이력 기록 + 본 화면 내역 테이블·통계 카드 즉시 반영

- 시즌 상세 [CEB-BO-ART-401-DETAIL] 내역 탭의 수동 지급 상품(배송 수령·현장 수령) [상태 변경] 버튼 클릭 시 노출

## 3. 기능 설명

운영자가 이 모달에서 할 수 있는 동작:

1. **지급 상태 변경** — 4가지 상태 중 하나로 갱신
2. **메모 입력** — 변경 사유·내용 자유 입력 (지급실패·재지급대기는 필수)
3. **저장·취소** — 푸터 액션으로 처리

---

## 4. 로직·플로우

### 4.1 모달 진입

1. 내역 탭에서 수동 지급 상품의 [상태 변경] 클릭
2. 본 모달 열림
3. 대상 안내·현재 상태·라디오·메모 필드가 현재 데이터로 prefill

### 4.2 입력 → 검증 → 저장

1. 운영자가 라디오로 상태 선택
2. 메모 입력 (선택, 단 지급실패·재지급대기 시 필수)
3. [저장] 클릭 시 검증:
   - 상태 미선택 차단 (라디오 1개 필수)
   - 지급실패·재지급대기 선택 + 메모 빈 값이면 차단
   - 메모 입력 시 50~200자 범위
4. 검증 통과 시:
   - 지급 상태 갱신 + 지급 일시는 현재 시각으로 자동 기록 (지급완료·지급실패·재지급대기 선택 시) 또는 빈 값으로 되돌림 (지급대기로 되돌릴 시)
   - 처리자는 운영자명으로 자동 기록
   - 모달 닫기
   - 토스트 "지급 상태가 변경되었습니다."
   - 내역 테이블 해당 행 + 통계 카드 즉시 갱신
   - 활동 로그 기록
5. 검증 실패 시:
   - 첫 오류 필드로 포커스 이동
   - 인라인 에러 메시지 노출

### 4.3 취소

1. [취소] 또는 모달 외부 클릭 또는 [X] 클릭
2. 입력값 폐기 후 모달 닫기 (확인 모달 없음 — 입력 분량이 적음)

## 5. 검증·예외 상황

### 5.1 입력 검증

| 항목 | 룰 |
| --- | --- |
| 지급 상태 | 4가지 중 1종 필수 |
| 메모 | 지급실패·재지급대기 선택 시 필수. 입력 시 50~200자 범위. 빈 값은 지급완료·지급대기에서만 허용 |

### 5.2 인라인 에러 메시지

| 상황 | 메시지 |
| --- | --- |
| 지급실패·재지급대기 선택 + 메모 빈 값 | "메모를 입력해주세요. 사유 기록이 필요합니다." |
| 메모 50자 미만 (지급실패·재지급대기 선택 시) | "메모는 50자 이상으로 입력하세요." |
| 메모 200자 초과 | 입력 차단 + "200자 이하로 입력하세요." |

### 5.3 권한별 동작

- 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다
- 쓰기 권한 없으면 내역 탭의 [상태 변경] 버튼이 미노출이므로 본 모달은 호출되지 않음

### 5.4 활동 로그

| 동작 | 로그 메시지 |
| --- | --- |
| 지급 상태 변경 | 덕력 시즌 '{그룹명} - {시즌명}' {yyyy.mm} {회원닉네임} '{상품명}'의 지급 상태를 '{이전 상태}' → '{신규 상태}'(으)로 변경했습니다. 메모: '{메모 요약}' |
