## 1. 화면 목적

운영자가 **진행중 시즌을 종료하기 전 마지막 확인을 받고, 종료 후 데이터 처리 정책 3행을 안내**하는 모달이다. 종료는 되돌릴 수 없으므로 신중한 의사결정을 유도한다.

- **누가**: 덕력 운영 권한이 있는 백오피스 관리자
- **왜**: 시즌 종료는 회원 적립·소비 흐름에 영향을 주는 비가역 행위이므로 명시적 확인 필요
- **결과**: 시즌 상태가 "진행중" → "종료"로 전환되고, 정산 완료 월·미정산 월·랭킹이 정책대로 보존됨

## 2. UI 설명

### 2.1 모달 레이아웃

```
┌───────────────────────────────────────────────┐
│ [모달 타이틀]                          [X]   │
│  시즌을 종료하시겠습니까?                     │
├───────────────────────────────────────────────┤
│                                               │
│  V01D - V01D 2026 시즌 시즌을 종료합니다.     │
│                                               │
│  · 정산 완료된 월의 보상은 확정 보존됩니다.   │
│  · 미정산 월의 보상 설정은 보존되나 신규      │
│    적립·소비가 발생하지 않습니다.             │
│  · 시즌 랭킹은 확정 보존되며 신규 적립·소비   │
│    는 다음 시즌에 귀속됩니다.                 │
│                                               │
│  ⚠ 이 작업은 되돌릴 수 없습니다.              │
│                                               │
├───────────────────────────────────────────────┤
│                       [취소]   [종료하기]     │
└───────────────────────────────────────────────┘
```

### 2.2 노출 조건

- 시즌 상세 [CEB-BO-ART-401-DETAIL] 상단 [종료] 액션 클릭 시 노출
- 진행중 시즌에만 노출 (예정·종료 시즌은 [종료] 버튼 자체가 미노출)

### 2.3 본문 구성

| 영역 | 내용 |
| --- | --- |
| 타이틀 | "시즌을 종료하시겠습니까?" |
| 대상 시즌 | "{그룹명} - {시즌명} 시즌을 종료합니다." (모달 첫 줄) |
| 정책 안내 (불릿 3행) | (아래 표) |
| 경고 줄 | "이 작업은 되돌릴 수 없습니다." — 빨간 강조 |

**정책 안내 3행** ([CEB-000] §5.2.1 정합):

| 행 | 내용 | 의미 |
| --- | --- | --- |
| 1 | 정산 완료된 월의 보상은 확정 보존됩니다. | 이미 정산된 월(매월 말일 23:59 자동 정산 완료분)은 그대로 유지 — 회원이 받은 보상에 영향 없음 |
| 2 | 미정산 월의 보상 설정은 보존되나 신규 적립·소비가 발생하지 않습니다. | 아직 정산되지 않은 월의 보상 구간 설정은 데이터로 남되, 종료된 시즌이므로 회원 활동·정산이 추가 일어나지 않음 |
| 3 | 시즌 랭킹은 확정 보존되며 신규 적립·소비는 다음 시즌에 귀속됩니다. | 회원이 누적한 덕력 랭킹은 그대로 보존. 종료 시점 이후의 적립·소비는 다음 시즌으로 이관 |

| [종료하기] | 위험 강조 색상. 클릭 시 시즌 상태 "진행중" → "종료" 전환 |

## 3. 기능 설명

운영자가 이 모달에서 할 수 있는 동작:

1. **종료 정책 확인** — 정산 완료 월·미정산 월·랭킹 처리 안내 3행 확인
2. **종료 확정** — [종료하기] 클릭으로 시즌 종료
3. **취소** — [취소] 또는 [X]로 모달 닫고 시즌 상태 유지

---

## 4. 로직·플로우

### 4.1 모달 진입

1. 시즌 상세에서 [종료] 클릭
2. 본 모달 열림
3. 본문에 대상 시즌의 그룹명·시즌명 표시 + 정책 3행 안내

### 4.2 종료 확정

1. [종료하기] 클릭
2. 시즌 상태 "진행중" → "종료" 전환 처리
3. 모달 닫기
4. 토스트 "시즌이 종료되었습니다."
5. 시즌 상세 화면 자동 갱신 (상태 배지 변경 + [종료] 버튼 미노출 + [수정] 버튼 시즌명만 활성)

### 4.3 취소

1. [취소] 또는 모달 외부 클릭 또는 [X] 클릭
2. 모달 닫기. 시즌 상태 그대로 유지

### 4.4 종료 후 후속 처리 (시스템)

| 데이터 | 처리 |
| --- | --- |
| 정산 완료된 월의 보상 | 그대로 보존 |
| 미정산 월의 보상 설정 | 데이터 보존, 신규 적립·소비 발생 안 함 |
| 시즌 랭킹 | 종료 시점 기준 확정 보존 |
| 신규 적립·소비 | 다음 시즌에 귀속 |

## 5. 검증·예외 상황

### 5.1 사전 조건

- 종료할 수 있는 시즌은 "진행중" 상태만 가능
- 예정·종료 시즌은 시즌 상세 상단에 [종료] 버튼 자체가 미노출이므로 본 모달도 호출되지 않음

### 5.2 권한별 동작

- 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다
- 쓰기 권한 없으면 시즌 상세 상단 [종료] 버튼이 미노출이므로 본 모달은 호출되지 않음

### 5.3 활동 로그

| 동작 | 로그 메시지 |
| --- | --- |
| 시즌 종료 | 덕력 시즌 '{그룹명} - {시즌명}'을(를) 종료했습니다. |
