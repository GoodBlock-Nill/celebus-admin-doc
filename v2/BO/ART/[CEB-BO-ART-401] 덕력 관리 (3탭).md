## 1. 화면 개요

회원이 아티스트 그룹별로 적립·소비하는 **덕력(DUK)** 을 운영자가 시즌·랭킹·내역 차원에서 관리·조회하는 단일 화면. 사이드바 진입 1회 후 상단 탭 3개로 분기.

**시즌 vs 랭킹 관계 (v1.1 정정)**: 시즌은 **1년 단위 운영 컨텍스트** (트로피·정산 단위 등). 랭킹은 시즌과 무관하게 **년/월 단위 ledger 누적**으로 집계. 시즌 설정 탭은 마스터 관리, 랭킹 탭은 기간 단위 조회로 책임 분리.

**시즌 상세 페이지 (v1.5 신규)**: 시즌(1년) 내 매월 별도 보상을 설정하는 페이지. 시즌 리스트 탭의 row 클릭 또는 [상세] 액션으로 진입. 라우트 `/artists/duk/seasons/{id}`. [CEB-000] §5.2.1 월간 정산 SSOT 정합 — 매월 말일 23:59 서버 자동 정산 + 정산 완료 월 잠금 정책 위임.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/artists/duk` (기본 탭 = 시즌 설정 / 탭은 URL query `?tab={season|ranking|history}`) |
| Breadcrumb | 홈 › 아티스트 › 덕력관리 |
| 진입 경로 | 사이드바 [아티스트] > [덕력관리] |
| 관리 단위 | **아티스트 그룹** 단위 (V01D, iKON, MADEIN 등 — 멤버 단위 제외) |
| 시즌 단위 | 그룹별 독립 시즌 (동일 시기에 그룹별로 다른 시즌 운영 가능) |
| 데이터 모델 | 시즌 마스터 + 변동 ledger 2종. 랭킹은 ledger의 (그룹, 시즌, 회원) 누적 합산으로 동적 도출 |
| 접근 권한 | [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다 (쓰기 권한 없으면 모달 진입 차단 + [신규 시즌]·[수정]·[종료] 버튼 미노출) |
| 사용처 화면 (응원) | 회원이 보유 덕력을 사용하는 화면(응원하기 등)은 **본 사이클 범위 외** — ledger에 출처 "응원하기"로 기록만, UI 정의는 별도 사이클 |

### 주요 기능

- 그룹별 랭킹 시즌 CRUD (생성·수정·종료)
- 시즌·그룹 기준 회원별 누적 덕력 순위 조회
- 획득·사용 ledger 전체 조회 (그룹·유형·기간·회원 필터 결합)

### 2-1. 탭 1 — 랭킹 시즌 설정 (`?tab=season`)

그룹별 시즌의 마스터 관리.

#### A. 헤더 액션

| 위치 | 요소 | 동작 |
|---|---|---|
| 우상단 | [신규 시즌] 버튼 (Primary) | 시즌 생성 모달 열기 |

#### B. 시즌 리스트 테이블

| 컬럼 | 표시 |
|---|---|
| 그룹 | 그룹명 (한국어 표시명) |
| 시즌명 | 시즌명 (1~50자) |
| 기간 | 시작일시 ~ 종료일시 (YYYY.MM.DD HH:mm) |
| 상태 | 배지 — 예정(회색) / 진행중(초록) / 종료(검정) |
| 액션 | [상세] (모든 상태 — 월별 보상 설정 페이지로 이동, v1.5 신규) / [수정] (예정·진행중) / [종료] (진행중) / [상세]만 (종료) |

- 정렬: 기본 시작일시 내림차순. 그룹·상태 컬럼 정렬 가능
- 페이지네이션: 20건/페이지

#### C. 시즌 생성·수정 모달 (`SeasonFormModal`)

| 필드 | 입력 | 검증 |
|---|---|---|
| 그룹 | Dropdown (활성 그룹 5종 중 1종 — 생성 모드만 노출, 수정 모드는 readonly 표시) | 필수 |
| 시즌명 | Text Input (placeholder "예: V01D 2026 시즌") | 필수, 1~50자 |
| 시작일시 | Date + Time | 필수 |
| 종료일시 | **입력 부재 — 시작일시 + 1년 자동 산출하여 readonly 안내 표시** | 시스템 계산 |

푸터: [취소] / [저장]

**v1.1 변경 — 시즌 1년 고정**: 종료일시 자유 입력 폐지. 시즌은 운영 단위 일관성을 위해 시작일시로부터 **정확히 1년** 동안 운영. 모달에 "종료: {시작 + 1년} (자동)" 형태로 표시되어 운영자가 종료 시점을 사전 확인 가능.

#### D. 종료 확인 모달

진행중 시즌의 [종료] 클릭 시 확인 모달:

- 타이틀: "시즌을 종료하시겠습니까?"
- 본문: "{그룹명} - {시즌명} 시즌을 종료합니다. 종료 후 본 시즌의 랭킹은 확정 보존되며 신규 적립/소비는 다음 시즌에 귀속됩니다."
- 푸터: [취소] / [종료하기] (Danger)

#### E. 시즌 상세 페이지 — 월별 보상 설정 (v1.5 신규)

시즌(1년) 내 매월 별도 보상을 설정하는 페이지. 시즌 리스트 row 클릭 또는 [상세] 액션으로 진입.

##### E-1. 페이지 정보

| 항목 | 값 |
|---|---|
| 라우트 | `/artists/duk/seasons/{id}` (별도 페이지) |
| Breadcrumb | 홈 › 아티스트 › 덕력관리 › 랭킹 시즌 설정 › {그룹명} - {시즌명} |
| 진입 경로 | §2-1-B 시즌 리스트 — row 클릭 / [상세] 액션 |
| 상단 액션 | [목록으로] — 시즌 리스트(/artists/duk?tab=season)로 복귀 |

##### E-2. 시즌 정보 카드

| 라벨 | 값 |
|---|---|
| 그룹 | {그룹명} |
| 시즌명 | {시즌명} |
| 기간 | 시작일시 ~ 종료일시 (YYYY.MM.DD HH:mm) |
| 상태 | 배지 — 예정 / 진행중 / 종료 |
| 정산 완료 | "{N}개월 ({yyyy.mm}~{yyyy.mm})" — 정산 완료 월 카운트 |

##### E-3. 월별 보상 섹션 (12개)

시즌 시작일시 기준 12개월 년월 시퀀스 자동 생성. 예: 시작일시 `2026.01.01` → `2026.01`~`2026.12` / 시작일시 `2026.05.01` → `2026.05`~`2027.04`.

각 월 카드는 다음 상태별 분기:

| 월 상태 | 표시 | 액션 |
|---|---|---|
| 정산 완료 | 🔒 + "정산 완료 (yyyy.mm.dd HH:mm)" + 회색 음영 | 구간 조회만 (편집 불가) |
| 미정산 (수정 가능) | ✏️ "수정 가능" 배지 | 구간 추가·수정·삭제·[저장] |

##### E-4. 구간(보상 행) 입력 폼

각 월 카드 내부 구간 N개. 행 자유 추가/삭제.

| 컬럼 | 컨트롤 | 검증 |
|---|---|---|
| 대상 기준 | Dropdown (등수 / 퍼센트 / 등수범위) | 필수 |
| 값 | Text Input — 등수=숫자(예: 1) / 퍼센트=숫자(예: 10) / 등수범위=`N-M`(예: 2-10) | 필수, 형식 정합 |
| 상품 | Text Input — 자유 입력 (예: "사인 앨범 + 디지털 포카") | 필수, 1~200자 |
| 액션 | [삭제] — 구간 행 제거 | — |

하단 [+ 구간 추가] — 빈 구간 행 1건 추가.

푸터 [저장] — 월별 구간 일괄 저장.

##### E-5. 비즈니스 규칙

- **정산 완료 월 잠금**: [CEB-000] §5.2.1 월간 정산 SSOT — 매월 말일 23:59 서버 자동 정산. 정산 완료 후에는 보상 변경 불가 (이력 무결성).
- **미정산 월 자유 편집**: 진행중·예정 시즌 모두 미정산 월은 구간 추가·수정·삭제 가능.
- **빈 보상 (구간 0건)**: 보상 미설정 월은 정산 시 보상 미지급 (정책 위임 — [CEB-000] §5.2.1).
- **보상 변경 즉시 적용**: 저장 시 토스트 + 카드 갱신. 다음 정산 시점부터 새 구간 적용.
- **상품 자유 입력 사유**: [CEB-DUK-101] §2.5 SSOT — 복수 상품(실물·디지털·뱃지) 자유 조합. 본 명세는 텍스트 자유 입력으로 위임.

### 2-2. 탭 2 — 덕력랭킹 (`?tab=ranking`)

그룹별 **년/월 단위** 회원 누적 덕력 순위 (v1.1 정정 — 시즌 단위 폐지).

#### A. 필터 바

| 필터 | 컨트롤 | 기본값 |

| 단위 | 토글 [년 단위 / 월 단위] | 년 단위 |

**단위 토글 동작**: "년 단위" 선택 시 → 년 Dropdown만 노출. "월 단위" 선택 시 → 년-월 Dropdown으로 교체. 두 단위 모두 그룹별 ledger에서 해당 기간 누적 합산.

#### B. 랭킹 테이블

| 컬럼 | 표시 |
|---|---|
| 순위 | 1·2·3위 메달 컬러 / 4위 이하 회색 숫자 |
| 회원 | 닉네임 (회원 상세로 새 탭 진입) |
| 누적 덕력 | {N} DUK (천 단위 콤마) — 해당 기간 획득 - 사용 합산 |
| 최근 변동일 | YYYY.MM.DD HH:mm (해당 기간 내) |

- 정렬: 누적 덕력 내림차순 고정
- 페이지네이션: 20건/페이지
- 빈 결과: "해당 필터 조건에 일치하는 회원이 없습니다." 안내

## 5. 검증 규칙

### 5-1. 입력 검증

| 필드 | 필수 | 검증 |
|---|---|---|
| 시즌 - 그룹 | O | 활성 그룹 1종 선택 (생성 모드만 변경 가능) |
| 시즌 - 시즌명 | O | 1~50자. 공백만 입력 불가 |
| 시즌 - 시작일시 | O | 유효한 일시 |
| 시즌 - 종료일시 | (자동) | **입력 없음 — 시작일시 + 1년으로 시스템 자동 산출**. 운영자 변경 불가 |
| 월별 보상 - 대상 기준 (v1.5) | O | 등수 / 퍼센트 / 등수범위 중 1종 |
| 월별 보상 - 값 (v1.5) | O | 등수=양의 정수 / 퍼센트=1~100 / 등수범위=`N-M` (N<M) 형식 |
| 월별 보상 - 상품 (v1.5) | O | 1~200자. 공백만 입력 불가 |

### 5-2. 비즈니스 규칙

- **시즌 기간 1년 고정** (v1.1) — 시작일시로부터 정확히 +1년. 자유 기간 입력 폐지
- **그룹별 진행중 시즌은 동시 1개 제한** — 생성·수정 시 동일 그룹의 다른 시즌과 기간이 겹치면 차단 (1년 고정으로 충돌 검증 단순화)
- 시즌 상태는 시작·종료 일시 기준 자동 전이: 시작 전 = 예정 / 시작 ~ 종료 = 진행중 / 종료 이후 = 종료. 운영자가 [종료] 클릭 시 종료 일시를 현재로 조정하여 즉시 종료
- 종료된 시즌은 수정·종료 액션 미노출 (조회만 가능)
- 종료된 시즌의 랭킹은 변동 없이 보존 — 신규 적립·소비는 다음 시즌에 귀속 (랭킹 조회는 시즌과 무관하게 년/월 단위로 집계되므로 시즌 상태가 랭킹 노출에는 영향 없음)
- 시즌 삭제 액션 없음 — ledger 무결성 보존
- 덕력 변동은 적립·소비 시점에 ledger 1행 신규 생성 (수정·삭제 불가 — 운영자 보정이 필요한 경우 [CEB-000] 정책 개정으로 매트릭스 활동 신설 절차)
- **랭킹 집계 단위** (v1.1) — 년 단위는 1월 1일 ~ 12월 31일 / 월 단위는 매월 1일 ~ 말일 ledger의 발생 일시 기준
- **월별 보상 정산** (v1.5) — [CEB-000] §5.2.1 SSOT 정합. 매월 말일 23:59 서버 자동 정산 + 정산 완료 월 잠금. 정산 완료 후 보상 변경 불가 (이력 무결성)
- **보상 미설정 월** (v1.5) — 구간 0건인 월은 정산 시 보상 미지급 (정책 위임 — 운영 알림은 별도)
- **보상 변경 즉시 적용** (v1.5) — 저장 시 다음 정산 시점부터 새 구간 적용. 구간 추가·수정·삭제 모두 동일

### 5-3. 활동 로그
