## 1. 화면 개요

회원이 아티스트 그룹별로 적립·소비하는 **덕력(DUK)** 을 운영자가 시즌·랭킹·내역 차원에서 관리·조회하는 단일 화면. 사이드바 진입 1회 후 상단 탭 3개로 분기.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/artists/duk` (기본 탭 = 시즌 설정 / 탭은 URL query `?tab={season|ranking|history}`) |
| Breadcrumb | 홈 › 아티스트 › 덕력관리 |
| 진입 경로 | 사이드바 [아티스트] > [덕력관리] |
| 관리 단위 | **아티스트 그룹** 단위 (V01D, iKON, MADEIN 등 — 멤버 단위 제외) |
| 시즌 단위 | 그룹별 독립 시즌 (동일 시기에 그룹별로 다른 시즌 운영 가능) |
| 데이터 모델 | 시즌 마스터 + 변동 ledger 2종. 랭킹은 ledger의 (그룹, 시즌, 회원) 누적 합산으로 동적 도출 |
| 접근 권한 | [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다 (쓰기 권한 없으면 모달 진입 차단 + [신규 시즌]·[수정]·[종료] 버튼 미노출) |
| 사용처 화면 (응원) | 회원이 보유 덕력을 사용하는 화면(응원하기 등)은 **본 사이클 범위 외** — ledger에 출처 "응원하기"로 기록만, UI 정의는 별도 사이클 |

### 주요 기능

- 그룹별 랭킹 시즌 CRUD (생성·수정·종료)
- 시즌·그룹 기준 회원별 누적 덕력 순위 조회
- 획득·사용 ledger 전체 조회 (그룹·유형·기간·회원 필터 결합)

### 2-0. 상단 공통 영역

### 2-1. 탭 1 — 랭킹 시즌 설정 (`?tab=season`)

그룹별 시즌의 마스터 관리.

#### A. 헤더 액션

| 위치 | 요소 | 동작 |
|---|---|---|
| 우상단 | [신규 시즌] 버튼 (Primary) | 시즌 생성 모달 열기 |

#### B. 시즌 리스트 테이블

| 컬럼 | 표시 |
|---|---|
| 그룹 | 그룹명 (한국어 표시명) |
| 시즌명 | 시즌명 (1~50자) |
| 기간 | 시작일시 ~ 종료일시 (YYYY.MM.DD HH:mm) |
| 상태 | 배지 — 예정(회색) / 진행중(초록) / 종료(검정) |
| 액션 | [수정] (예정·진행중) / [종료] (진행중) / 액션 없음 (종료) |

- 정렬: 기본 시작일시 내림차순. 그룹·상태 컬럼 정렬 가능
- 페이지네이션: 20건/페이지

#### C. 시즌 생성·수정 모달 (`SeasonFormModal`)

| 필드 | 입력 | 검증 |
|---|---|---|
| 그룹 | Dropdown (활성 그룹 5종 중 1종 — 생성 모드만 노출, 수정 모드는 readonly 표시) | 필수 |
| 시즌명 | Text Input (placeholder "예: V01D 2026 1Q 시즌") | 필수, 1~50자 |
| 시작일시 | Date + Time | 필수 |
| 종료일시 | Date + Time | 필수, 시작일시 초과 |

푸터: [취소] / [저장]

#### D. 종료 확인 모달

진행중 시즌의 [종료] 클릭 시 확인 모달:

- 타이틀: "시즌을 종료하시겠습니까?"
- 본문: "{그룹명} - {시즌명} 시즌을 종료합니다. 종료 후 본 시즌의 랭킹은 확정 보존되며 신규 적립/소비는 다음 시즌에 귀속됩니다."
- 푸터: [취소] / [종료하기] (Danger)

### 2-2. 탭 2 — 덕력랭킹 (`?tab=ranking`)

시즌·그룹 기준 회원별 누적 덕력 순위.

#### A. 필터 바

| 필터 | 컨트롤 | 기본값 |
|---|---|---|
| 시즌 | Dropdown (그룹 선택에 종속 — 해당 그룹의 시즌 목록만 노출, "전체 시즌(누적)" 옵션 포함) | 진행중 시즌 우선 |
| 그룹 | Dropdown (활성 그룹 5종) | 첫 번째 그룹 |
| 회원 검색 | Text Input (닉네임 부분 일치) | 빈 값 |

#### B. 랭킹 테이블

| 컬럼 | 표시 |
|---|---|
| 순위 | 1·2·3위 메달 컬러 / 4위 이하 회색 숫자 |
| 회원 | 닉네임 (회원 상세로 새 탭 진입) |
| 누적 덕력 | {N} DUK (천 단위 콤마) |
| 최근 변동일 | YYYY.MM.DD HH:mm |

- 정렬: 누적 덕력 내림차순 고정
- 페이지네이션: 20건/페이지
- 빈 결과: "해당 필터 조건에 일치하는 회원이 없습니다." 안내

### 2-3. 탭 3 — 덕력내역 (`?tab=history`)

회원의 덕력 획득·사용 ledger 전체 조회.

#### A. 필터 바

| 필터 | 컨트롤 | 기본값 |
|---|---|---|
| 그룹 | Dropdown (전체 + 활성 그룹 5종) | 전체 |
| 유형 | 라디오 (전체 / 획득 / 사용) | 전체 |
| 기간 시작 | Date Picker | 최근 30일 전 |
| 기간 종료 | Date Picker | 오늘 |
| 회원 검색 | Text Input (닉네임 부분 일치) | 빈 값 |
| 초기화 | [초기화] 버튼 | 모든 필터 기본값 복원 |

#### B. 요약 카드 2종

| 카드 | 값 |
|---|---|
| 필터 결과 건수 | 현재 필터 적용 후 ledger 총 건수 |
| 합계 (유형별) | 획득 합계 +N DUK / 사용 합계 -N DUK / 전체 선택 시 둘 다 표시 |

#### C. ledger 테이블

| 컬럼 | 표시 |
|---|---|
| 일시 | YYYY.MM.DD HH:mm |
| 회원 | 닉네임 (새 탭 진입) |
| 그룹 | 그룹명 |
| 유형 | 배지 — 획득(초록) / 사용(분홍) |
| 출처 | 한국어 — 획득: 퀘스트 보상·래플 환급·운영자 지급·이벤트 보상 / 사용: 응원하기·래플 응모 |
| 변동 | +N DUK (획득, 초록) / -N DUK (사용, 분홍) |
| 잔액 | 변동 후 잔액 (해당 그룹 기준, {N} DUK) |
| 시즌 | 해당 시점 진행중 시즌명 (시즌 없을 시 "—") |

- 정렬: 일시 내림차순 고정
- 페이지네이션: 20건/페이지

## 5. 검증 규칙

### 5-1. 입력 검증

| 필드 | 필수 | 검증 |
|---|---|---|
| 시즌 - 그룹 | O | 활성 그룹 1종 선택 (생성 모드만 변경 가능) |
| 시즌 - 시즌명 | O | 1~50자. 공백만 입력 불가 |
| 시즌 - 시작일시 | O | 유효한 일시 |
| 시즌 - 종료일시 | O | 유효한 일시, 시작일시보다 미래 |

### 5-2. 비즈니스 규칙

- **그룹별 진행중 시즌은 동시 1개 제한** — 생성·수정 시 동일 그룹의 다른 진행중 시즌과 기간이 겹치면 차단
- 시즌 상태는 시작·종료 일시 기준 자동 전이: 시작 전 = 예정 / 시작 ~ 종료 = 진행중 / 종료 이후 = 종료. 운영자가 [종료] 클릭 시 종료 일시를 현재로 조정하여 즉시 종료
- 종료된 시즌은 수정·종료 액션 미노출 (조회만 가능)
- 종료된 시즌의 랭킹은 변동 없이 보존 — 신규 적립·소비는 다음 시즌에 귀속
- 시즌 삭제 액션 없음 — ledger 무결성 보존
- 덕력 변동은 적립·소비 시점에 ledger 1행 신규 생성 (수정·삭제 불가 — 운영자 보정이 필요한 경우 "운영자 지급" 출처로 보정 행 추가)

### 5-3. 활동 로그

`[CEB-BO-000] §6 활동 로그` 양식 정합:

| 액션 | 로그 메시지 |
|---|---|
| 시즌 생성 | 덕력 시즌 '{그룹명} - {시즌명}'을(를) 생성했습니다. (기간: {시작} ~ {종료}) |
| 시즌 수정 | 덕력 시즌 '{그룹명} - {시즌명}'을(를) 수정했습니다. (변경 항목: {필드명 리스트}) |
| 시즌 종료 | 덕력 시즌 '{그룹명} - {시즌명}'을(를) 종료했습니다. |

> 랭킹·내역 조회는 활동 로그 대상 아님 (조회만).

### 5-4. 권한

- 권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다 (R/W 분리)
- 쓰기 권한 없으면 [신규 시즌]·[수정]·[종료] 버튼 미노출 (조회만 가능)
- `EXTERNAL_ARTIST` 권한자(외부 협력사) — [CEB-BO-003] §4 정합: 본인 매핑 소속사 산하 그룹의 시즌·랭킹·내역만 조회 가능 (쓰기 불가)
