## 1. 개요

소식·일정(INF) 영역은 아티스트의 **소식·일정·공지**를 운영자가 등록·관리하는 영역이다. 등록한 콘텐츠는 앱 [CEB-INF-101] 정보 피드(통합 타임라인 + 캘린더)와 [CEB-INF-201] 소식 상세에 노출된다. 운영자가 직접 작성·게시한다.

| 항목 | 위치 |
| --- | --- |
| 마스터 가이드 | [CEB-BO-001] 마이그레이션·재구성 마스터 가이드 |
| 사이드 메뉴 | 아티스트 > 소식/일정 |
| 권한 | [CEB-BO-SYS-102] 권한 관리 그룹 설정 |
| 활동 로그 | [CEB-BO-000] 백오피스 공통 정책 |
| 앱 연동 | [CEB-INF-101] 정보 피드 · [CEB-INF-201] 소식 상세 |

## 4. 영역 고유 정책

### 4.1 콘텐츠 타입 (3종)

| 타입 | 설명 | 앱 노출 |
| --- | --- | --- |
| 소식 | 아티스트 뉴스·소식 | 정보 피드 타임라인 + 소식 상세 |
| 일정 | 콘서트·팬미팅·방송 등 일정 | 정보 피드 캘린더 + 타임라인 + 일정 상세 |
| 공지 | 운영 공지·중요 안내 | 정보 피드 상단 고정 배너 + 공지 상세 |

### 4.2 라이프사이클

```
임시저장 → 게시 → 보관
```

| 상태 | 의미 | 앱 노출 |
| --- | --- | --- |
| 임시저장 | 작성 중 (게시 전) | 미노출 |
| 게시 | 앱 노출 중 | 노출 |
| 보관 | 노출 종료 (데이터 보존, 삭제 아님) | 미노출 |

### 4.3 타입별 입력 필드

| 구분 | 소식 | 일정 | 공지 |
| --- | --- | --- | --- |
| 제목 | 필수(다국어) | 필수(일정명) | 필수(다국어) |
| 본문 | 필수(다국어, 위지윅) | — | 필수(다국어) |
| 일시 | — | 필수(날짜+시간) | — |
| 장소 | — | 선택(구글 장소 검색 연동) | — |
| 게시일 | 필수 | — | 필수 |
| 이미지 | 선택(최대 10장) | — | 대표 이미지 선택(1장) |
| 연관 콘텐츠 | 선택(최대 3건, 콘텐츠 타입+콘텐츠명+URL) | — | — |
| CTA 버튼 | — | — | 선택(라벨+링크) |
| 공식 출처 | 선택(플래그) | 선택(플래그) | — |

- 공통 필드: 연결 아티스트(그룹), 상태.
- 연관 콘텐츠의 콘텐츠 타입: 유튜브·틱톡·스포티파이·스레드·인스타그램·X·웹.
- 일정의 장소는 구글 장소 검색으로 조회한다. 일정에는 부제·썸네일 입력이 없다.
- 일정의 장소·소식의 연관 콘텐츠·공지의 대표 이미지·CTA 버튼은 앱 디자인 정합 필드다.

### 4.4 공식 출처 플래그

- 소속사/아티스트 공식 채널을 통해 제공된 정보임을 표기하는 플래그.
- 게시 시 앱에서 "공식 제공" 뱃지(일정) / "[공식 제공]" 태그(소식)로 노출된다.
- 출처 신뢰성 강조 목적이며 독점 의미는 없다. (앱 [CEB-INF-101] 정합)

### 4.5 다국어 정책

| 필드 | 다국어 | 글자수 |
| --- | --- | --- |
| 제목 | 한국어/영어/일본어 | 50자 |
| 본문 (소식·공지) | 한국어/영어/일본어 | 5,000자 (위지윅) |
| 일정 장소 | 단일 | 100자 |
| 연관 콘텐츠명 | 단일 | 50자 |
| CTA 라벨 | 단일 | 30자 |

- 게시 시 소식·공지 제목·본문은 한국어·영어·일본어를 모두 입력해야 한다.

### 4.6 푸시 알림 (아티스트별)

- 게시 시 해당 아티스트 알림을 켠 회원에게 푸시 발송 옵션을 제공한다.
- 발송 대상: 전체 / 해당 아티스트 팔로워 / 무발송.
- 발송 시점: 즉시 / 예약.
- 알림 메시지는 다국어(80자). 앱 [CEB-INF-101] §2.4 아티스트별 알림 정책과 정합한다.
