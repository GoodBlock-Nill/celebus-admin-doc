## 1. 화면 개요

배너 1건을 신규 작성·조회·수정·노출 제어하는 단일 화면. 신규(`/home/banners/new`)와 상세(`/home/banners/{id}`)를 동일 폼으로 통합한다. 슬롯 컨텍스트(위치 + 아티스트)는 [CEB-BO-APP-202] 슬롯 상세에서 [+ 배너 추가]로 진입할 때 자동 prefill되며, 본 화면에서는 read-only로 표기한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/home/banners/new?slot={slotKind}&artist={artist}` (신규) · `/home/banners/{id}` (조회) · `/home/banners/{id}?mode=edit` (수정) |
| Breadcrumb | 홈 / 앱 / 배너 관리 / {위치 · 아티스트} / {제목} |
| 진입 경로 | [CEB-BO-APP-202] 슬롯 상세 [+ 배너 추가] (신규) · 슬롯 상세 배너 Row 클릭 (조회) |
| 용도 | 배너 1건의 다국어·미디어·소스·딥링크·노출 기간 운영. **노출 시작은 공개일 도달 시 시스템 자동 전이** (운영자 수동 트리거 없음). 비상 시 [노출 종료]만 운영자 수동 |

> **v6.7 정책 변경 핵심**: "즉시 노출 시작" 운영자 수동 트리거를 폐기한다. FQ Raffle·게임존 게임과 동일하게 **2단계 흐름**으로 통일 — 생성 폼에서는 [생성하기](DRAFT 진입)만, 상세에서는 다음 단계 액션. **공개일은 항상 필수**, 도달 시 검증 통과 자동 ACTIVE 전이. 운영자가 "지금 노출"을 원하면 공개일을 현재 시각으로 설정하면 동일 효과. [노출 종료]만 비상 차단 권한으로 수동 트리거 유지.

### 2-2. 입력 폼 (5섹션)

#### A. 슬롯 컨텍스트 (모드별 분기 — v6.8)

| 모드 | 위치 | 아티스트 |
|---|---|---|
| 신규 작성 (슬롯 상세 [+ 배너 추가] 진입) | read-only Badge | read-only Badge |
| **신규 작성 (복제 진입, `?from={원본 배너ID}`)** | **편집 가능 Select** — 메인 / 오늘의 할일 / 다함께 / 미션 (원본 슬롯 기본값) | **편집 가능 Select** — 전역 / 그룹 (원본 아티스트 기본값) |
| 조회·수정 모드 | read-only Badge | read-only Badge |

각 슬롯의 정책 보조 표시는 동적으로 노출 ("캐러셀 · 최대 8개" / "단일 배너 1개 고정"). 아티스트 메인 슬롯(다함께/미션)은 전역이 없고 항상 단일 아티스트만 옵션.

> **복제 모드 슬롯 변경 시 미디어 권장 비율 동적 분기** — 캐러셀(**메인** 1:1) ↔ 단일(오늘의 할일·**다함께**·미션 16:9) 비율 불일치 시 §C 미디어 패널에 경고 Toast "권장 비율이 변경되었습니다. 이미지를 재업로드하세요." 노출. 자동 변환은 하지 않음 (운영자 책임).

> 일반 신규 작성 모드 (복제가 아닌 경우)는 슬롯 상세에서 prefill된 슬롯 컨텍스트가 read-only로 고정. 변경하려면 다른 슬롯 상세에서 새로 [+ 배너 추가] 진입.

#### D. 딥링크 (선택)

[CEB-BO-013] §9 공통 딥링크 정책 적용. **소스 타입 + URL 2개 자유 텍스트박스**. 운영자가 분류 라벨과 도달 URL을 직접 입력.

| 필드 | 입력 방식 | 규격 |
|---|---|---|
| 소스 타입 | 텍스트박스 (좌, 너비 160px) | 자유 텍스트, placeholder `"소스 타입 입력 (예: 래플, 프로모션, 공지)"`. 형식 제약 없음. 빈 값 허용 |
| 링크 URL | 텍스트박스 (우, flex) | placeholder `"https:// 또는 /로 시작하는 URL (비우면 이동 없음)"`. 형식 검증 `https://`·`http://`·`/`. 그 외는 인라인 에러 |

- **URL 빈 값**: 이동 없음 — 클릭 비활성 (표시 전용 배너)
- **외부 URL** (`https://`·`http://`): 새 탭에서 열림
- **내부 URL** (`/`로 시작): 회원 앱 SPA 라우팅 (`/raffle/123`·`/quest/45` 등)

> 소스 타입은 운영자가 분류·집계용으로 입력하는 자유 라벨. 표기 표준은 운영팀 정책으로 별도 정의.

#### E. 노출 기간 (v6.7 정책 정정)

[CEB-BO-013] §5 적용. **"즉시 노출 시작" 운영자 트리거 폐기**. 공개일 도달 시 시스템 자동 활성화 전용.

| 필드 | 입력 방식 | 규격 |
|---|---|---|
| 공개일시 | Date + Time Picker (KST, 분 단위) | **항상 필수**. 도달 시 시스템 자동 ACTIVE 전이 (검증 통과 시) |
| 종료일시 | 라디오 "무기한 / 사용자 지정" + 사용자 지정 시 Date + Time Picker | **무기한**: 종료 시각 없음 — 운영자 [노출 종료] 비상 트리거로만 종료. **사용자 지정**: 도달 시 시스템 자동 CLOSED 전이 |

운영자가 "지금 바로 노출"을 원하면 공개일을 현재 시각(또는 ±몇 분)으로 설정하면 시스템이 즉시 자동 활성화. 별도 [즉시 시작] 버튼 불필요.

> **v6.7 변경 요약** — v6.6 이전: "무기한" 모드는 운영자 [즉시 노출 시작]으로 활성화 / "사용자 지정" 모드는 공개일 자동 활성화 (2중 흐름). v6.7부터: **단일 흐름** — 공개일 항상 필수 + 도달 시 시스템 자동 활성화. 종료일만 무기한 옵션 유지.

> **캐러셀 슬롯(메인 한정)** 의 슬라이드 순서는 본 폼에서 다루지 않는다 — 저장 시 마지막 순서로 자동 배치되며, 슬롯 상세([CEB-BO-APP-202])에서 드래그로 변경한다. 단일 슬롯(오늘의 할일·**다함께**·미션)은 정렬 개념 없음. E 섹션 하단에 안내 부기로 노출.

>
> **슬롯 한도 검증**: 노출 재개도 신규 노출 시작과 동일한 슬롯 한도 검증 적용. 캐러셀 8개 초과 → 모달 안에서 차단 토스트 + 모달 유지 (운영자가 다른 배너를 먼저 종료할 수 있도록).
>
> **단일 슬롯 자동 교체**: 노출 재개 시 동일 슬롯에 기존 ACTIVE 1건이 있으면 모달 내부에 "기존 배너 '{기존 제목}'이(가) 자동 종료됩니다" 안내 + 확인 시 자동 처리.
>
> **CLOSED [수정] 허용 (v6.8 신규)**: 종료 후 콘텐츠 수정 → 재개 흐름 자연스러움. 다국어 타이틀·이미지·딥링크 모두 수정 가능. 단, 슬롯 컨텍스트는 read-only (변경하려면 [복제] 사용).

### 5-5. 복제 동작 (v6.8 정정)

[복제] 클릭 → 현재 배너의 모든 입력값을 신규 작성 폼에 프리필 → `/home/banners/new?slot={원본 slotKind}&artist={원본 artist}&from={원본 배너ID}`로 이동.

- 공개일·종료일·상태·통계는 초기화
- `?from` 쿼리가 있으면 **복제 모드 식별** — §2-2 A 슬롯 컨텍스트가 read-only Badge 대신 **편집 가능 Select**로 노출 (위치·아티스트 변경 허용)
- 슬롯 변경 시 §C 미디어 권장 비율 동적 분기 — 캐러셀(메인 1:1) ↔ 단일(다함께·미션·오늘의 할일 16:9) 비율 불일치 시 경고 Toast

### 5-6. 수정 모드 변경 추적 (v6.8 신규)

운영자가 [수정] 모드에서 입력 후 [취소] 시 변경 사항 폐기 경고를 제공한다.

| 시점 | 검증 | 동작 |
|---|---|---|
| 수정 모드 진입 | `initialSnapshot` = 현재 배너의 입력값 JSON 직렬화 (다국어 타이틀·이미지 URL·기간·딥링크 핵심 필드) | snapshot 저장 |
| 입력 변경 | `currentSnapshot` 실시간 갱신 | `hasChanged` 플래그 갱신 |
| [취소] 클릭 | `hasChanged === true` | **`BannerEditCancelModal`** 호출 → 확정 시 view 전환 |
| [취소] 클릭 | `hasChanged === false` | 즉시 view 전환 (모달 없음) |
| [저장] 클릭 | `hasChanged === false` | Toast "변경 사항이 없습니다" + 저장 차단 |
| [← 슬롯으로] 클릭 (수정 모드) | `hasChanged === true` | `BannerEditCancelModal` 호출 → 확정 시 슬롯 상세로 복귀 |

JSON 직렬화 기반이므로 객체 순서·undefined 등에서 false positive 가능. 단순화 위해 핵심 필드만 비교 (다국어 타이틀·이미지·기간·딥링크 URL). **신규 입력 필드 추가 시 `initialSnapshot`·`currentSnapshot` 양쪽 모두에 필드를 함께 추가**해야 false negative가 발생하지 않는다 (v6.9 코드 코멘트로 명시).

### 5-7. CLOSED [수정] 후 [노출 재개] 통합 흐름 (v6.9 신규)

CLOSED 상태에서 콘텐츠 정정 후 재개하는 흐름을 운영자가 끊김 없이 수행할 수 있도록 정책 명시.

| 시나리오 | 권장 흐름 |
|---|---|
| CLOSED 콘텐츠만 정정 (재개 없음) | [수정] → 입력 → [저장] → view 모드 (CLOSED 유지) |
| CLOSED 정정 후 재개 (운영자 선의도) | [수정] → 입력 → [저장] → view 모드 → [노출 재개] → 모달 확정 → ACTIVE |
| **종료일 과거 + 재개** (자동 분기, v6.9) | [노출 재개] → 모달 → [수정 후 재개] 선택 → 수정 모드 자동 전환 + `pendingResumeAfterEdit` 플래그 set → [저장] 후 **자동으로 노출 재개 모달이 재오픈** → 확정 시 ACTIVE |
| 수정 도중 [취소]·[← 슬롯으로] | `BannerEditCancelModal` 호출 → 확정 시 변경 사항 폐기 + `pendingResumeAfterEdit` 플래그 cleanup (의도 폐기) |

**핵심**: 운영자가 "수정 후 재개" 의도를 한 번 선언하면 시스템이 추적하여 누락 없이 ACTIVE까지 안내. 운영자가 명시적으로 취소하지 않는 한 재개 모달이 자동 재오픈된다.
