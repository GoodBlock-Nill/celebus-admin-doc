## 1. 화면 개요

배너 1건을 신규 작성·조회·수정·노출 제어하는 단일 화면. 신규(`/home/banners/new`)와 상세(`/home/banners/{id}`)를 동일 폼으로 통합한다. 슬롯 컨텍스트(위치 + 아티스트)는 [CEB-BO-APP-202] 슬롯 상세에서 [+ 배너 추가]로 진입할 때 자동 prefill되며, 본 화면에서는 read-only로 표기한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/home/banners/new?slot={slotKind}&artist={artist}` (신규) · `/home/banners/{id}` (조회) · `/home/banners/{id}?mode=edit` (수정) |
| Breadcrumb | 홈 / 앱 / 배너 관리 / {위치 · 아티스트} / {제목} |
| 진입 경로 | [CEB-BO-APP-202] 슬롯 상세 [+ 배너 추가] (신규) · 슬롯 상세 배너 Row 클릭 (조회) |
| 용도 | 배너 1건의 다국어·미디어·소스·딥링크·노출 기간·노출 ON/OFF 운영 |

### 2-1. 헤더

| 요소 | 설명 |

| 상태 Badge | 임시저장 (회색) / 노출중 (초록) / 노출 종료 (진회색) |
| 기간 Badge | "무기한" 또는 기간 표시 |
| 액션 영역 | 상태별 가능 액션 (§4 참조) |

### 2-2. 입력 폼 (5섹션)

#### A. 슬롯 컨텍스트 (read-only)

| 항목 | 표시 |
|---|---|
| 배너 위치 | 슬롯의 위치 Badge + 정책 보조 표시 ("캐러셀 · 최대 8개" / "단일") |
| 아티스트 | 슬롯의 아티스트 Badge ("전역" 또는 단일 아티스트) |

> 슬롯 컨텍스트는 슬롯 상세에서 prefill되어 본 화면에서 변경할 수 없다. 변경이 필요하면 다른 슬롯 상세에서 신규 작성을 시작한다. 아티스트 메인 슬롯(다함께/미션)은 전역이 없고 항상 단일 아티스트.

#### B. 기본 정보 (다국어)

| 필드 | KO | EN | JP | 글자수 |
|---|---|---|---|---|
| 메인 타이틀 | 필수 | 필수 | 필수 | 30자 |
| 서브 타이틀 | 필수 | 필수 | 필수 | 60자 |
| 이미지 대체 텍스트 | 선택 | 선택 | 선택 | 100자 |

#### C. 미디어 (슬롯별 분기)

[CEB-BO-013] §8 적용. 슬롯의 위치에 따라 권장 비율·사이즈가 자동 분기되어 폼에 표기.

| 위치 | 이미지 형태 | 권장 비율 | 권장 사이즈 |
|---|---|---|---|
| 메인 | 메인이미지 | 1:1 (정사각) | 1080×1080 |
| 오늘의 할일 | 배경이미지 | 16:9 (가로) | 1920×1080 |
| 다함께 | 배경이미지 | 16:9 (가로) | 1920×1080 |
| 미션 | 배경이미지 | 16:9 (가로) | 1920×1080 |

| 공통 항목 | 값 |
|---|---|
| 최대 용량 | 50MB ([CEB-BO-000] §16) |
| 허용 형식 | JPG / PNG / WEBP |
| 미리보기 | 업로드 즉시 미리보기 + 회원 앱 시뮬레이션 (슬롯별 다른 자리·비율 표시) |

#### D. 딥링크 (선택)

[CEB-BO-013] §9 공통 딥링크 정책 적용. **소스 타입 + URL 2개 자유 텍스트박스**. 운영자가 분류 라벨과 도달 URL을 직접 입력.

| 필드 | 입력 방식 | 규격 |
|---|---|---|
| 소스 타입 | 텍스트박스 (좌, 너비 160px) | 자유 텍스트, placeholder `"소스 타입 입력 (예: 래플, 프로모션, 공지)"`. 형식 제약 없음. 빈 값 허용 |
| 링크 URL | 텍스트박스 (우, flex) | placeholder `"https:// 또는 /로 시작하는 URL (비우면 이동 없음)"`. 형식 검증 `https://`·`http://`·`/`. 그 외는 인라인 에러 |

- **URL 빈 값**: 이동 없음 — 클릭 비활성 (표시 전용 배너)
- **외부 URL** (`https://`·`http://`): 새 탭에서 열림
- **내부 URL** (`/`로 시작): 회원 앱 SPA 라우팅 (`/raffle/123`·`/quest/45` 등)

> 소스 타입은 운영자가 분류·집계용으로 입력하는 자유 라벨. 표기 표준은 운영팀 정책으로 별도 정의.

#### E. 노출 기간

[CEB-BO-013] §5 적용.

| 모드 | 입력 |
|---|---|
| 무기한 | 일정 입력 없음. 저장 후 운영자 [즉시 노출 시작]으로 ON, [즉시 노출 종료]로 OFF |
| 사용자 지정 | 공개일시 + 종료일시 (Date+Time Picker, KST, 분 단위). 도래 시 자동 노출/종료 + [즉시 시작/종료] 오버라이드 가능 |

라디오 선택 후 "사용자 지정" 시 공개·종료일 입력 필드 동적 노출.

> **캐러셀 슬롯**의 슬라이드 순서는 본 폼에서 다루지 않는다 — 저장 시 마지막 순서로 자동 배치되며, 슬롯 상세([CEB-BO-APP-202])에서 드래그로 변경한다. 단일 슬롯(오늘의 할일·미션)은 정렬 개념 없음. E 섹션 하단에 안내 부기로 노출.

### 4-2. 데이터·동작 분기

| 신규 진입 (쿼리 prefill) | A 슬롯 컨텍스트 read-only 표시 + 빈 폼 |
| 신규 진입 (슬롯 쿼리 없음) | **안전망 리다이렉트** — `/home/banners` 배너 관리로 이동 + 안내 토스트 "슬롯을 먼저 선택해 주세요. 배너 관리에서 슬롯 행을 클릭한 뒤 [+ 배너 추가]로 진입합니다." 운영자가 의도 없이 첫 슬롯으로 진입하는 사고 방지 |
| 조회 실패 | "배너 정보를 불러올 수 없습니다." + [재시도] |
| 검증 실패 ([노출 시작]) | 토스트 (Error) + 누락 필드 빨강 강조 |
| 저장 성공 | 토스트 (Success) + 슬롯 상세로 복귀 |
| 캐러셀 한도 초과 | 토스트 (Error) "이 슬롯의 동시 노출 8개를 이미 채웠습니다." |
| 단일 슬롯 자동 교체 안내 | [노출 시작] 클릭 시 확인 모달 "{기존 배너 제목}이(가) 자동으로 노출 종료됩니다. 진행하시겠습니까?" |
| 권한 없음 진입 | [관리자 > 권한관리] 설정에 따라 화면 자체 미노출 또는 액션 버튼 비활성 |

## 5. 검증 규칙

### 5-1. 다국어 검증

| 검증 시점 | 검증 항목 |
|---|---|
| 임시저장 | 메인 타이틀 KO 최소 입력 |
| [노출 시작] (즉시 또는 자동) | 메인·서브 타이틀 KO/EN/JP 모두 입력 + 이미지 업로드 + 공개일·종료일 (사용자 지정 시) + 딥링크 (이동 없음 허용) |

미충족 시 토스트 + 누락 필드 빨강 강조. 자동 노출의 경우 공개일 도래 시점에 검증 실패하면 자동 노출 차단 + 운영자 알림.

### 5-2. 슬롯 한도·교체

| 슬롯 유형 | 규칙 |
|---|---|
| 캐러셀 (메인/다함께) | [노출 시작] 시 슬롯 ACTIVE 8개 초과 차단 ([CEB-BO-013] §5-3) |
| 단일 (오늘의 할일/미션) | [노출 시작] 시 기존 ACTIVE 있으면 자동 OFF (즉시 교체) ([CEB-BO-013] §5-4) |

### 5-3. 딥링크 검증

| 필드 | 검증 |
|---|---|
| 소스 타입 | 형식 제약 없음. 운영자 자유 입력. 빈 값 허용 |
| 링크 URL | `https://` · `http://` · `/` 중 하나로 시작해야 함. 그 외 형식은 인라인 에러 메시지. 빈 값 = 이동 없음 (표시 전용 배너) |
| 길이 제한 | 별도 제한 없음 (운영자 책임) |

### 5-4. 활동 로그

[CEB-BO-013] §18 정합.

| 액션 | 로그 메시지 |
|---|---|
| 배너 등록 | 배너 '{제목}' 등록 (슬롯: {위치} · {아티스트}, 기간: {무기한/시작~종료}) |
| 배너 수정 | 배너 '{제목}'의 정보를 수정했습니다. (변경 항목: {필드 목록}) |

| 즉시 노출 종료 | 배너 '{제목}'의 노출을 즉시 종료했습니다. |

### 5-5. 복제 동작

[복제] 클릭 → 현재 배너의 모든 입력값을 신규 작성 폼에 프리필 → `/home/banners/new?slot={slotKind}&artist={artist}`로 이동. 단, 공개일·종료일·상태·통계는 초기화.
