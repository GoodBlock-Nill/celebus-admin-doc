## 1. 화면 개요

슬롯 1개에 들어있는 배너들을 조회·추가·정렬하는 화면. 위치별 정책(캐러셀 vs 단일)에 따라 UI가 자동 분기된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/home/banners/slot/{slotKind}/{artist}` (artist=GLOBAL 또는 그룹명) |
| Breadcrumb | 홈 / 앱 / 배너 관리 / {위치} · {아티스트} |
| 진입 경로 | [CEB-BO-APP-101] 배너 관리 Row 클릭 |
| 용도 | 슬롯 1개의 배너 인벤토리 관리 (추가·정렬·진입) |

## 2. 화면 구성

### 2-1. 헤더

| 요소 | 설명 |
|---|---|
| 타이틀 | "{위치명} — {아티스트명}" (예: "메인 — 전역", "다함께 — V01D") |
| 위치 Badge | 메인 / 오늘의 할일 / 다함께 / 미션 |
| 아티스트 Badge | "전역" (회색) 또는 그룹명 (인디고) |
| 정책 Badge | "캐러셀 · 최대 동시 8개" / "단일 배너 1개 고정" |
| [+ 배너 추가] | 항상 활성. 클릭 시 [CEB-BO-APP-201] 신규 작성 진입 (슬롯 컨텍스트 쿼리 전달) |

### 2-2. 슬롯 상태 안내 띠

슬롯 유형 + 노출중/대기 조합 6시나리오.

| 슬롯 유형 | 상황 | 안내 텍스트 |
|---|---|---|
| 캐러셀 | 한도 미달 | "노출중 N / 8" + 대기 건수 |
| 캐러셀 | 한도 도달 | 강조 색 + "한도 도달. 신규 노출 시작 시 기존 1건을 종료해 주세요." |
| 단일 | 비어 있음 | "이 슬롯은 비어 있습니다. [배너 추가]로 등록해 주세요." |
| 단일 | 노출중 1건만 | "노출중 1 / 1" |
| 단일 | 노출중 1건 + 대기 N건 | "노출중 1 / 1 · 대기 N건 · 대기 배너 [노출 시작] 시 기존이 자동 종료됩니다." |
| 단일 | 노출중 0건 + 대기 N건 | "노출중 0 / 1 · 대기 N건" (운영자가 [노출 시작] 필요) |

### 2-3. 배너 리스트 테이블

캐러셀 슬롯과 단일 슬롯의 컬럼 구성이 다르다.

#### 캐러셀 슬롯 (메인 / 다함께)

| 컬럼 | 표시 내용 |
|---|---|
| 정렬 핸들 | 드래그로 순서 변경 |
| 순서 | 1, 2, 3 … |
| 상태 | Badge (임시저장 / 노출중 / 노출 종료) |
| 소스 타입 | 운영자가 입력한 URL 텍스트 그대로 표시. 빈 값이면 "—" (이동 없음). 너무 긴 URL은 말줄임 + 호버 시 풀 URL 툴팁. [CEB-BO-013] §9 정합 |
| 타이틀 (KO) | 메인 + 서브 타이틀 2줄 |
| 노출 기간 | "무기한" Badge 또는 `YYYY.MM.DD HH:mm ~ YYYY.MM.DD HH:mm` |
| 최근 수정 | 수정 시각 + 수정자 |

#### 단일 슬롯 (오늘의 할일 / 미션)

캐러셀과 동일 컬럼에서 **정렬 핸들·순서 컬럼만 제거**.

Row 클릭 → [CEB-BO-APP-201] 배너 상세·작성 진입.

### 2-4. 빈 상태

배너 0건일 때: Empty 아이콘 + "등록된 배너가 없습니다." + [+ 첫 배너 추가] CTA.

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 테이블 |
| 배너 0건 | Empty CTA "등록된 배너가 없습니다." + [+ 첫 배너 추가] |
| 캐러셀 노출중 8개 도달 | 안내 띠 강조 색 + 신규 노출 시작 시 차단 토스트 |
| 단일 슬롯 대기 [노출 시작] | 기존 ACTIVE 있으면 confirm 모달 "{기존 배너 제목}이(가) 자동 종료됩니다" |
| 드래그 정렬 (캐러셀) | 즉시 서버 PATCH + 활동 로그 + 성공 토스트 |
| 정렬 저장 실패 | 행 원위치 복귀 + 에러 토스트 |
| 잘못된 (위치, 대상) 조합 URL 직접 접근 | 가드 화면: "메인 위치는 전역만 지원합니다." / "오늘의 할일 위치는 아티스트만 지원합니다." + [배너 리스트로] 복귀 버튼 |
| 권한 없음 진입 | [관리자 > 권한관리] 설정에 따라 화면 자체 미노출 또는 [+ 배너 추가]·드래그 비활성 |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 슬롯 자체 등록·삭제 | **불가**. 슬롯은 시스템 자동 생성 ([CEB-BO-013] §4-3) |
| [+ 배너 추가] 활성 | 항상 활성 — 캐러셀은 8개 한도와 무관하게 등록 가능, 단일은 DRAFT 다수 등록 가능 |
| 캐러셀 한도 차단 | [노출 시작]에서만 차단. 등록 자체는 무제한 |
| 단일 슬롯 ACTIVE | 항상 1건만. 대기 DRAFT가 [노출 시작] 시 기존 자동 OFF |
| 드래그 정렬 | 캐러셀 슬롯만. 단일은 정렬 개념 없음 |
| 활동 로그 | 정렬 변경만 본 화면에서 발생. 등록·수정·노출 ON/OFF·삭제는 [CEB-BO-APP-201]에서 |
