## 1. 화면 개요

운영자가 회원 앱(PWA)에 발송되는 알림(기본알림 + 푸시알림 Web Push)을 조회·발송·관리하는 인벤토리 화면. **수동 알림(운영자 캠페인)**과 **자동 트리거 알림(시스템 발송 로그)**의 라이프사이클이 본질적으로 다르므로 같은 사이드바 메뉴 하위에서 **2개 탭으로 분리**한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/app/notifications` (탭 쿼리 동기화 — `?tab=manual` / `?tab=auto`) |
| Breadcrumb | 홈 / 앱 / 알림 관리 |
| 진입 경로 | 사이드바 "앱 > 알림 관리" |
| 용도 | 탭 1 수동 알림 캠페인 조회·작성·발송 / 탭 2 자동 트리거 발송 로그 조회 |

### 2-5. 자동 트리거 탭 — 트리거 정책 카탈로그 (조회 전용)

자동 트리거는 시스템 이벤트 발생 시 백엔드가 자동 발송하는 알림으로, **운영자가 신규 등록·메시지 수정·활성 변경을 모두 할 수 없다**. 본 탭은 현재 백엔드에 등록된 트리거 정책 목록을 조회 전용으로 표시한다 ([CEB-BO-013] §12·§14-2 정합).

#### A. 안내 띠

상단 보라색 안내: "자동 트리거는 시스템 이벤트 발생 시 시스템이 자동으로 발송하는 알림입니다. **메시지·대상·활성 상태 변경은 개발팀에 별도 요청**해 주세요. 이 화면은 현재 등록된 트리거 정책 목록을 조회만 할 수 있습니다."

#### B. 필터 바 (최소화)

| 필터 | 옵션 |
|---|---|
| 트리거 카테고리 | 전체 / 래플 / 응원·퀘스트 / BIVE / 회원 활동 / 시스템 공지 |
| 활성 상태 | 전체 / 사용중 / 미사용 |
| 키워드 | 트리거명 부분 일치 |
| [초기화] | 모든 필터 해제 |

**기간 필터·대상 필터·발송 인스턴스 상태 필터 모두 없음** (피드백 #2·#3·#7 반영).

#### C. 트리거 정책 테이블 (수동 알림 탭과 동일 시각 통일감)

| 컬럼 | 표시 내용 |
|---|---|
| 활성 | "**사용중**" (초록) 또는 "**미사용**" (회색) 배지. read-only |
| 카테고리 | 트리거 카테고리 배지 — 보라 톤 |
| 채널 | "기본" / "기본 + 푸시" 배지 |
| 트리거 | 트리거명 1줄 + 트리거 조건 1줄 미리보기 (호버 시 풀 텍스트 툴팁) |
| 제목 (KO) | 메시지 제목 1줄 + 본문 1줄 (말줄임) |
| 최근 발송 | YYYY.MM.DD HH:mm 또는 "—" |

Row 클릭 시 **상세 미리보기 모달** 노출 (다국어 메시지·딥링크·트리거 조건 read-only 조회). 자동 트리거 메시지는 변수 토큰(`{닉네임}`·`{래플명}` 등)을 포함하므로 수동 알림으로 복제하지 않는다 — 운영자가 유사 캠페인이 필요하면 [+ 새 알림]에서 빈 폼으로 새로 작성한다.

> **딥링크 표시 정책 (자동 트리거 전용)**: 자동 트리거 미리보기의 딥링크는 **단일 라벨 텍스트**로 표시한다 (예: "응모권 받기" → `/raffle/{래플식별값}`). 운영자가 직접 입력하는 수동 알림·배너의 "소스 타입 + URL" 2필드 자유 입력 구조와 다르며, 트리거 정책이 백엔드에 사전 정의되어 있으므로 단일 라벨이면 충분하다. 변수 토큰을 포함한 URL은 운영자 편집 대상이 아니므로 분리 입력 의미가 없다.

## 5. 검증 규칙

### 5-1. 탭 분리 규칙

| 규칙 | 내용 |
|---|---|

| 탭별 필터 격리 | 탭 전환 시 모든 필터 자동 초기화 (다른 탭의 필터가 의미 없으므로) |
| 자동 탭 Row 액션 | Row 클릭 시 상세 미리보기 모달 노출 (다국어 메시지 read-only). 별도 상세 페이지 진입 없음 |
| 기간 필터 적용 범위 | 수동 탭에만 적용. 자동 탭은 트리거 정책 카탈로그이므로 기간 무관 |

| 도달 수 집계 | 발송 완료 후 5분 캐시. 푸시(Web Push) + 기본알림 도달 회원 합집합 |

### 5-3. 자동 트리거 탭 제약

자동 트리거는 백엔드 코드/설정에 정의되어 있어 백오피스에서 다음을 할 수 없다. **변경은 모두 개발팀 요청 채널을 통해서만 가능**하다 ([CEB-BO-013] §12-1).

| 제약 | 내용 |
|---|---|
| 신규 등록 | 백오피스에서 불가. 개발팀에 트리거 추가 요청 |
| 메시지·대상·트리거 조건 수정 | 백오피스에서 불가. 개발팀에 변경 요청 |
| 활성/미사용 토글 | 백오피스에서 불가. 개발팀에 활성/비활성 요청 |
| 삭제 | 백오피스에서 불가 |
| 가능 액션 | [상세 미리보기] (다국어 메시지 모달 read-only)만. 자동 트리거 메시지는 변수 토큰을 포함하므로 수동 알림으로 복제하지 않으며, 유사 캠페인이 필요하면 운영자가 [+ 새 알림]에서 빈 폼으로 새로 작성한다 |
