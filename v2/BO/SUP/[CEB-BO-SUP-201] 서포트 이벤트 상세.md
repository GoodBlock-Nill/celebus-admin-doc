## 1. 화면 목적

개별 서포트 이벤트의 정보 조회·수정·라이프사이클 전환을 한 화면에서 수행한다. 기본 정보·응원자 내역·결과물·반환 현황을 3개 탭으로 분리하고, 상태 전환은 헤더 액션 버튼 + 확인 모달로 처리한다.

| 항목 | 값 |

| 화면 경로 | `/artists/support/{id}` |
| 진입 경로 | [CEB-BO-SUP-101] 리스트 Row 클릭 |

### 2.2 탭1 — 기본 정보

조회 모드(기본)와 수정 모드를 분기한다. 진행 상태에 따라 수정 가능 범위가 달라진다(§3.1).

#### 통계 카드 (상단)

| 카드 | 값 |
| --- | --- |
| 응원자 수 | 1 덕력 이상 응원한 회원 수 |
| 누적 응원 | 누적 응원 덕력 |
| 평균 응원량 | 누적 응원 / 응원자 수 |
| 달성률 | 누적 응원 / 목표 응원량 (%) |

#### 기본 정보 (조회/수정 모드)

| 항목 | 표시 | 수정 |
| --- | --- | --- |
| 이벤트명 (한국어/영어/일본어) | 50자 | 수정 가능 |
| 설명 (한국어/영어/일본어) | 500자 | 수정 가능 |
| 그룹 | 응원 대상 | 읽기 전용 (변경 불가) |
| 목표 응원량 | 덕력 | 임시저장에서만 수정 가능 (모집중 이후 잠금) |
| 기간 | 시작·마감 (KST) | 임시저장에서만 수정 가능 (모집중 이후 잠금) |
| 대표 이미지 | 운영자 업로드 | 수정 가능 |

> 수정 모드는 [수정] → [저장]/[취소] 분기. **[수정]·[저장]·[취소] 버튼은 탭 상단 우측(통계 카드 아래)에 배치**해 조회 중에도 쉽게 찾을 수 있게 한다. 변경 사항 없으면 [저장] 비활성. 수정 중 탭 전환·이탈 시 미저장 확인.

### 2.3 탭2 — 응원자 내역

한 회원은 동일 이벤트에 **여러 번 응원**할 수 있다(앱 [CEB-DUK-201] §2.4 누적 응원). 따라서 회원별로 응원을 합산해 보여주되, 회차별 개별 응원 내역까지 확인할 수 있도록 구성한다.

#### 요약 (상단)

| 항목 | 값 |
| --- | --- |
| 응원자 수 | 1 덕력 이상 응원한 회원 수 (중복 제거) |
| 총 응원 횟수 | 이벤트 전체 응원 건수 (회원별 다회 합산) |
| 누적 응원 | 누적 응원 덕력 |

#### 회원별 목록 테이블

| 컬럼 | 설명 |
| --- | --- |
| 닉네임 | 응원 회원 |
| 누적 응원량 | 회원의 전체 응원 합산 (덕력) |
| 응원 횟수 | 회원이 응원한 횟수 (2회 이상 강조) |
| 최초 응원 | 회원의 첫 응원 시점 (KST) |
| 최근 응원 | 회원의 마지막 응원 시점 (KST) |
| 반환 여부 | 미달성종료·집행취소 시 "반환 완료" 표시 |

- 누적 응원량 많은 순 정렬, 페이지네이션 10건 단위
- 읽기 전용 (응원은 회원 앱에서만 발생)

#### 응원 상세 모달 (행 클릭)

회원 행 클릭 시 해당 회원의 **회차별 응원 내역**을 모달로 표시한다.

| 항목 | 값 |
| --- | --- |
| 헤더 | {닉네임} 응원 내역 — 총 {N}회 · 누적 {N} 덕력 (반환 시 "전액 반환 완료 ({반환 시점})") |
| 회차 테이블 | 회차 / 응원량(덕력) / 응원 시점 (응원 시점 오름차순) |
| 합계 | 누적 응원량 |

### 2.4 탭3 — 결과물·반환

#### 결과물 등록 영역

| 항목 | 값 |
| --- | --- |
| 결과 메시지 (한국어/영어/일본어) | 각 200자 |
| 결과 사진·영상 | 최대 10장 (회원 앱 갤러리) |
| 노출 시점 | 완료 상태 시 회원 앱 [CEB-DUK-201]에 공개 |

> 결과물 등록은 집행중 상태에서 [결과물 등록·완료] 액션으로 수행하며, 완료 전이와 함께 처리된다.

#### 반환 현황 (읽기 전용)

미달성종료·집행취소 상태일 때 ① 요약 ② 회원별 반환 내역으로 구성한다. 정상 진행·완료 이벤트는 "반환 없음" 안내만 표시한다.

**요약**

| 항목 | 값 |
| --- | --- |
| 반환 사유 | 목표 미달성 / 집행 취소 / 강제 종료 |
| 반환 방식 | 서버 자동 · 즉시 전액 |
| 반환율 | 100% |
| 반환 대상 | 응원 회원 수 |
| 반환 총 덕력 | 반환된 총 덕력 |
| 반환 완료 시점 | 상태 전이 시각 |

**회원별 반환 내역 테이블**

| 컬럼 | 설명 |
| --- | --- |
| 닉네임 | 반환 대상 회원 |
| 반환 덕력 | 회원의 누적 응원 = 반환액 |
| 원 응원 횟수 | 반환 전 응원한 횟수 |
| 반환 시점 | 회원별 반환 처리 시각 |
| 상태 | "반환 완료" |

- 반환 덕력 많은 순 정렬. 회차별 응원 내역은 [응원자 내역] 탭의 응원 상세 모달 참조.

> 반환은 서버 자동 처리(§3.3). 운영자 수동 반환 버튼은 없다.

## 3. 규칙·로직

### 3.1 상태별 수정 범위

| 상태 | 수정 가능 |
| --- | --- |
| 임시저장 | 전 항목 (이벤트명·설명·그룹 제외·목표 응원량·기간·이미지) |
| 모집중 | 이벤트명·설명·대표 이미지만 (목표 응원량·기간·그룹 잠금 — 응원 진행 중 기준 변경 방지) |
| 달성·집행중·완료·미달성종료·집행취소 | 수정 불가 (읽기 전용) |

### 3.2 라이프사이클 전환

[CEB-BO-SUP-000] §4.1 적용. 각 전환은 §2.5 액션 버튼 + 확인 모달. 모집중→달성, 모집중→미달성종료(마감+미달성)는 서버 자동 전이로 운영자 버튼 없이 발생한다.

### 3.3 반환 (전액·자동)

미달성종료·집행취소·강제 종료 시 응원 회원 전원에게 응원 덕력을 **즉시 전액 자동 반환**한다([CEB-BO-SUP-000] §4.4). 운영자 수동 반환·추가 인증 절차는 없다.

### 3.4 권한

권한은 [CEB-BO-SYS-102] 권한 관리 그룹 설정에 따른다. 외부 아티스트 계정은 자기 아티스트 이벤트만 조회 가능하며 상태 전환·수정은 불가하다.
