## 1. 화면 개요

진행중 상태의 래플을 추첨대기 상태로 전환하는 확인 모달. 응모가 차단되고 운영자가 추첨내역 탭에서 추첨을 실행할 수 있게 된다.

| 항목 | 값 |
|---|---|
| 모달 트리거 | 래플 상세 헤더의 [종료하기] 버튼 |
| 진입 조건 | 래플 상태 = 진행중 |
| 영향 | 상태가 진행중 → 추첨대기로 전환 (응모 차단) |
| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "종료하시겠어요?"

### 2-2. 본문 메시지

```
'{래플 타이틀}'을(를) 종료합니다.
응모가 차단되고 추첨대기 상태로 전환됩니다.
```

### 2-3. 액션 버튼

| 버튼 | 색상 | 동작 |
|---|---|---|
| 취소 | 회색 외곽선 | 모달 닫기 |
| **종료** | 빨강 (위험 강조) | 상태 전환 → 즉시 응모 차단 |

## 3. 화면에 노출되는 정보 / 종료 후 변화

**응모자 1명 이상의 일반 종료:**

| 항목 | 종료 전 | 종료 후 |
|---|---|---|
| 상태 | 진행중 | 추첨대기 |
| 최근 수정자·일시 | (이전 값) | 현재 운영자 / 현재 시각 |
| 회원 응모 가능 여부 | 가능 | 차단 |

**응모자 0건 종료 (특수 분기):**

| 항목 | 종료 전 | 종료 후 |
|---|---|---|
| 상태 | 진행중 | **종료** (추첨대기 단계 건너뜀) |
| 추첨 시드 | — | 미생성 |
| 당첨자·미당첨자 데이터 | — | 없음 |
| 추첨내역 탭 표기 | — | "추첨 없이 자동 종료 — 입상자 없음" |
| BIVE 보상 (설정되어 있던 경우) | — | 민팅 발생하지 않음 |

추가 영향:
- 회원이 앱에서 응모 시도 시 시스템이 차단
- 마감일시 도달 시 시스템 자동 종료와 동일한 결과. 자동 종료 시 **관리자 대시보드에 알림 노출 + 활동 로그 기록**

## 4. 검증 규칙

| 검증 조건 | 동작 |
|---|---|
| 래플 상태 ≠ 진행중 | 헤더 액션 자체가 미노출 |
| 응모자 0건 종료 | **허용** — 추첨대기 단계 건너뛰고 즉시 종료. 추첨내역 탭에 "추첨 없이 자동 종료 — 입상자 없음" 표기 |
| 종료 후 되돌리기 | 불가 — 추첨대기·종료는 일방향 전환 |
