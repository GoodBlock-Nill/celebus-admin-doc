## 1. 화면 개요

임시저장 상태의 래플을 영구 삭제하는 확인 모달. 진행중·추첨대기·종료 상태는 응모자·당첨자 보호를 위해 삭제 불가.

| 항목 | 값 |
|---|---|
| 모달 트리거 | 래플 상세 헤더의 [삭제] 버튼 |
| 진입 조건 | 래플 상태 = 임시저장 |
| 영향 | 래플 한 건이 **Hard 삭제** (DB에서 즉시 제거, 복구 불가). 활동 로그만 3년간 보존 (개인정보 보호법 준용) |
| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "삭제하시겠어요?"

### 2-2. 본문 메시지

```
'{래플 타이틀}'을(를) 삭제합니다. 복구할 수 없습니다.
```

### 2-3. 액션 버튼

| 버튼 | 색상 | 동작 |
|---|---|---|
| 취소 | 회색 외곽선 | 모달 닫기 |
| **삭제** | 빨강 (위험 강조) | 삭제 실행 + 리스트로 복귀 |

## 3. 화면에 노출되는 정보 / 삭제 후 변화

| 데이터 | 변화 |
|---|---|
| 래플 1건 | **Hard 삭제 — DB에서 즉시 영구 제거** (복구 불가, soft delete 없음) |
| 응모 / 당첨 / 추첨 데이터 | 임시저장 상태이므로 존재하지 않음 |
| 활동 로그 | **3년간 보존** (개인정보 보호법 준용) — "래플 '{타이틀}'을(를) 삭제했습니다. (임시저장 상태만 가능)" |

## 4. 검증 규칙

| 검증 조건 | 동작 |
|---|---|
| 래플 상태 ≠ 임시저장 | 헤더의 [삭제] 액션 자체가 노출되지 않음 |
| 삭제 권한이 없는 운영자 | 헤더의 [삭제] 액션 자체가 노출되지 않음 ([관리자 > 권한관리] 설정에 따름) |

> **삭제 정책 배경**: 진행중 상태부터는 회원이 응모권을 사용하여 응모하므로 회원 자산에 영향을 준다. 따라서 진행중 이후는 영구 삭제 차단. 종료된 래플을 앱에서만 가리고 싶다면 [CEB-BO-RFL-201-MD-HIDE] 숨김 처리 사용.
