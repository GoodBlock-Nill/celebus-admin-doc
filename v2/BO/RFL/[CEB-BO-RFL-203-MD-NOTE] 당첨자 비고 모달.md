> 모달 트리거: [CEB-BO-RFL-203] 추첨내역 테이블의 비고 아이콘 클릭 (당첨자만)

## 1. 화면 개요

특정 당첨자에 대한 운영 내부 메모(비고사항)를 입력·수정하는 모달. CS 대응·배송 메모·특이사항 기록 용도. **당첨자에 한정**, 미당첨자에서는 비고 아이콘이 비활성된다.

| 항목 | 값 |
|---|---|
| 모달 트리거 | 추첨내역 탭 종료 테이블의 비고 아이콘 |
| 진입 조건 | 래플 상태 = 종료 + 해당 행이 당첨자 |
| 영향 | 당첨자 비고 메모 수정 (한국어 200자 이내) |
| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "당첨자 비고사항"
- 보조 정보: 당첨자 닉네임 · 전화번호
- 우상단: 모달 닫기 버튼

### 2-2. 본문 — 당첨자/미당첨자 분기

#### 당첨자

- 안내문: "CS 대응·배송 메모 등 운영 내부 노트 (한국어, 200자)"
- 텍스트 입력 영역 (여러 줄, 5행 정도) — 기존 비고 내용 미리 채워짐
- 입력 즉시 200자 자동 잘림
- 우측 하단: 글자수 카운터 "N/200"

#### 미당첨자

- 안내 문구: "미당첨자는 비고를 작성할 수 없습니다."
- 저장 버튼 미노출 (취소만)

### 2-3. 액션 버튼

| 버튼 | 색상 | 표시 조건 | 동작 |
|---|---|---|---|
| 취소 | 회색 외곽선 | 항상 | 모달 닫기 (변경사항 폐기) |
| **저장** | 인디고 (강조) | 당첨자만 | 비고 메모 저장 + "비고가 저장되었습니다" 토스트 |

## 3. 화면에 노출되는 정보 / 저장 후 변화

| 항목 | 저장 전 | 저장 후 |
|---|---|---|
| 비고 메모 | (이전 값 또는 빈 문자열) | 입력값 (최대 200자) |
| 마지막 작성자 | (이전 운영자) | 현재 운영자 닉네임으로 갱신 |
| 마지막 수정 일시 | (이전 일시) | 현재 시각으로 갱신 |
| 활동 로그 | (변화 없음) | "래플 '{타이틀}' 당첨자 '{닉네임}' 비고를 수정했습니다." 기록 (3년 보존) |

본 메모는 **운영 내부 전용**:
- 회원 앱에 노출되지 않음
- CS 인계 메모, 배송 처리 메모, 부정 행위 의심 기록 등 활용
- 한국어 단일 (다국어 불필요 — 운영자만 조회)
- 추적 범위: **마지막 작성자 + 수정 일시만 기록**. 중간 수정 이력은 보존하지 않음 (대신 활동 로그에서 수정 이벤트 자체는 추적 가능)

## 4. 검증 규칙

| 검증 조건 | 동작 |
|---|---|
| 행의 상태 ≠ 당첨 | 추첨내역 테이블의 비고 아이콘이 비활성 (모달 진입 불가) |
| 입력 길이 > 200자 | 입력 즉시 200자로 자동 잘림 |
| 변경사항 없는 채 저장 | 토스트 "변경사항이 없습니다" 노출 후 모달 유지. 활동 로그·마지막 수정자 갱신 없음 |
| 보상 지급 토글 상태 | 본 모달과 무관 — 비고와 보상 지급은 독립적으로 운영 |
