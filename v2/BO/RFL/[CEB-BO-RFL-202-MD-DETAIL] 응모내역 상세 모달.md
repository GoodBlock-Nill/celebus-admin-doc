## 1. 화면 개요

특정 응모자 한 명의 시계열 응모 이력을 조회하는 모달. CS 대응·어뷰징 분석 시 운영자가 개별 응모 분포를 확인하는 용도. **읽기 전용**.

| 항목 | 값 |

| 모달 트리거 | 응모내역 탭 테이블의 행 클릭 |
| 진입 조건 | 제한 없음 (모든 래플 상태에서 가능) |

| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "응모내역 상세"
- 보조 정보: 응모자 닉네임 · 전화번호
- 우상단: 모달 닫기 버튼

### 2-2. 누적 요약 박스

회색 박스에 한 줄로:
- "누적 응모권 — N장"

### 2-3. 시계열 응모 이력 테이블

| 컬럼 | 표시 내용 |
|---|---|
| # | 순번 (1, 2, 3, …) |
| 사용 응모권 | 응모 1건당 사용한 응모권 수량 |
| 응모일시 | 응모가 발생한 시각 |

테이블 하단 합계 행:
- "합계" + 사용 응모권 총합
- 상단 "누적 응모권" 값과 동일해야 함 (운영자가 시각적으로 정합성 확인)

**페이지네이션**:
- **페이지당 10건** (응모 이력 건수 기준)
- **항상 노출** — 1페이지일 때도 `‹ 1 ›` 비활성 형태로 노출. 응모 이력 0건일 때만 미노출 (운영 사이트 정합)
- **윈도잉 규칙** — 총 페이지 ≤ 7이면 전체 페이지 번호 노출, 8페이지 이상이면 `1 ... cur-1 cur cur+1 ... last` 윈도우 + 양 끝(`...`) 표시

빈 상태: "응모 이력이 없습니다." (이론적으로 발생하지 않음, 안전 대비)

## 3. 화면에 노출되는 정보

| 위치 | 정보 | 의미 |
|---|---|---|
| 헤더 보조 | 닉네임 | 응모자 닉네임 |
| 헤더 보조 | 전화번호 | 응모자 전화번호 |
| 요약 박스 | 누적 응모권 | 본 래플에 응모한 응모권 총합 |
| 테이블 컬럼 1 | # | 응모 순번 |
| 테이블 컬럼 2 | 사용 응모권 | 각 응모 1건의 사용 수량 |
| 테이블 컬럼 3 | 응모일시 | 응모 시각 |
| 테이블 합계 행 | 합계 | 모든 응모 건의 사용 응모권 총합 |

**참고**: 본 모달은 한 회원의 응모 이력 1건 1건을 시계열 순으로 노출한다. 한 회원이 여러 번 나누어 응모한 경우 그 분포를 확인할 수 있다.

## 4. 검증 규칙

| 검증 조건 | 동작 |
|---|---|
| 응모 이벤트 0건 | "응모 이력이 없습니다." 빈 상태 노출. 페이지네이션 미노출 |
| 합계 vs 누적 불일치 | 데이터 정합성 문제 — 시스템 별도 알람 처리 (회원에게는 노출 안 됨) |
| 응모 이력 1건 이상 | **10건/페이지 페이지네이션 항상 노출** — 1페이지일 때도 비활성 `‹ 1 ›` 형태로 노출. 11건 이상에서는 활성 페이지 번호 + 좌우 화살표로 페이지 이동 |
