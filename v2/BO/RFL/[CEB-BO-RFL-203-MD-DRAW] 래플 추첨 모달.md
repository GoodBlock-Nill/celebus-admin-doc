## 1. 화면 개요

추첨대기 상태의 래플에 대해 **응모권 가중치 기반 무작위 추첨**을 실행하는 모달. 가추첨 미리보기 → 재추첨(시드 변경) → 확정의 3단계 흐름으로 운영된다. 확정 시 래플 상태가 종료로 전환되고 당첨자에게 자동 알림 + (BIVE 보상이 설정되어 있다면) NFT 자동 민팅 요청이 발송된다.

| 항목 | 값 |
|---|---|
| 모달 트리거 | 추첨내역 탭의 큰 파란 [추첨하기] 버튼 |
| 진입 조건 | 래플 상태 = 추첨대기 |
| 영향 | 상태 추첨대기 → 종료. 당첨자·미당첨자 명단 영구 저장. 추첨 시드 기록. 보상 지급 시작 |
| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "래플 추첨하기"
- 보조 정보: 래플의 한국어 타이틀
- 우상단: 모달 닫기 버튼

### 2-2. 추첨 시드 안내 박스

- 좌측: "가추첨 시드" 라벨 + 시드값 (분쟁 시 재현 가능한 고유 식별값)
- 우측: **[재추첨]** 버튼 — 클릭 시 새 시드를 생성하여 가추첨 결과를 다시 계산
- 하단 안내: "응모권 가중치를 반영한 결정론적 추첨. 재추첨 시 새 시드로 다시 뽑습니다."

### 2-3. 추첨 요약 행

`예상 당첨 N명 / 목표 M명 · 총 응모자 K명 · 추첨 풀 P명 (제외 X명 차감)`

- **총 응모자(K)** — 본 래플에 응모한 전체 회원 수
- **추첨 풀(P)** — 추첨 대상 응모자 수 = `K − X`
- **제외(X)** — 응모내역 탭([CEB-BO-RFL-202])에서 "추첨 제외" 처리된 회원 수
- 제외 0명이면 "(제외 X명 차감)" 표시 생략
- **풀 0명(전원 제외) 시**: 안내문 "제외 처리로 추첨 가능한 응모자가 0명입니다. 응모내역 탭에서 제외를 해제하거나 자동 종료를 진행해주세요." + [추첨 확정] 버튼 비활성

### 2-4. 당첨자 미리보기 테이블

| 컬럼 | 표시 내용 |
|---|---|
| # | 순번 |
| 닉네임 | 당첨 후보의 닉네임 |
| 전화번호 | 회원 전화번호 |
| 응모권 | 누적 응모권 (가중치 반영 기준) |

**페이지네이션** ([CEB-BO-012] §5-2 공통 정책 — 모달 10건/페이지 정합):
- **모달 내 페이지당 10건** (RFL-202-MD-DETAIL과 동일)
- **항상 노출** — 목표 당첨자 ≤ 10명이면 `‹ 1 ›` 비활성 형태로 노출. 11명 이상이면 활성 페이지 번호 + 좌우 화살표
- **윈도잉 규칙** — 총 페이지 ≤ 7이면 전체 페이지 번호 노출, 8 이상이면 `1 ... cur-1 cur cur+1 ... last` 윈도우 + `...` 표시
- 재추첨(시드 변경) 시 페이지 1로 자동 리셋

응모자 0건일 때는 안내문 "응모자가 없어 추첨이 불가능합니다.". 페이지네이션 미노출.

|---|---|---|

## 3. 화면에 노출되는 정보 / 추첨 동작 원리

### 3-1. 추첨 알고리즘 (기획 관점)

운영자가 [추첨하기]를 누르면 시스템은 **사용 응모권 수에 정확 비례한 가중치 무작위 추첨**을 수행한다.

원칙:
- **추첨 풀 = 총 응모자 − 추첨 제외 회원**. 응모내역 탭에서 "추첨 제외"로 토글된 회원은 가중치 풀에 포함되지 않음 (응모권 수와 무관하게 0%의 당첨 확률)
- **가중치 = 사용 응모권 수**. 추첨 풀 내에서 회원 A가 응모권 10장, 회원 B가 응모권 1장 사용했다면, A는 B보다 **정확히 10배 높은 확률**로 추첨됨
- **본 래플 내에서 중복 당첨 불가** — 동일 회원이 같은 래플에서 1회만 당첨됨
- **다른 래플에서는 중복 당첨 허용** — 각 래플은 독립적으로 운영. 한 회원이 여러 래플에 응모해 모두 당첨되어도 무방
- **시드 기반 결정론적 추첨** — 동일 시드로 다시 추첨하면 동일 결과 재현 가능 (분쟁 시 검증용)
- **동률 응모권 회원 간 우선순위는 시드 기반 무작위로 결정** (별도 정렬 기준 없음)
- **시드 변경(재추첨)해도 제외 명단은 그대로 유지** — 시드는 무작위성만 다시 뽑을 뿐, 풀 구성에는 영향 없음

### 3-2. 시드의 의미

추첨 시드는 다음 정보를 조합해 생성되어 영구 기록된다:
- 래플 식별자
- 추첨 실행 시각
- 추첨을 실행한 운영자
- 추첨 직전의 응모 데이터 스냅샷

이후 응모 시도는 자동 차단된다.

### 3-3. 추첨 확정 후 변화

| 항목 | 변화 |
|---|---|
| 래플 상태 | 추첨대기 → 종료 |
| 당첨자 수 | 추첨 결과값으로 확정 |
| 보상 지급 상태 | 초기값 "미지급" (운영자가 추첨내역 탭에서 토글로 지급 완료 처리) |
| 미당첨자 수 | **추첨 풀 수 − 당첨자 수** (추첨 풀 = 응모자 − 제외). 제외된 회원은 미당첨자·당첨자 어디에도 포함되지 않음 |
| 추첨 제외 명단 | **확정 시점에 동결** — 종료 상태에서는 [CEB-BO-RFL-202] 응모내역 탭의 체크박스·일괄 처리 액션 바·상태 토글이 모두 비활성화되어 더 이상 변경 불가 |
| 당첨자·미당첨자 명단 | 영구 저장 |
| 추첨 시드 | 영구 기록 |
| BIVE 보상이 설정된 경우 | **추첨 확정 즉시 당첨자 전원의 외부 지갑으로 자동 민팅 큐 일괄 적재** ([CEB-BO-004] BIVE 영역으로 위임). 경품 수령 여부와 무관 |
| 회원 알림 | **당첨자·미당첨자 모두에게 앱 푸시 + 인앱 알림 센터 발송** (이메일·SMS 없음). **추첨 제외 회원에게는 별도 안내 없음** (응모 자체는 정상 처리된 상태이므로) |
| 미당첨자 위로 보상 | **없음** — 응모권 환급·위로 BIVE 등 별도 보상 미지급 |
| 추첨 제외 회원 응모권 | **환급 없음** — 사용된 응모권은 그대로 소진 처리 (응모내역 탭 정책과 동일) |

## 4. 검증 규칙

| 검증 조건 | 동작 |
|---|---|
| 래플 상태 ≠ 추첨대기 | [추첨하기] 버튼 자체가 노출되지 않음 |
| 응모자 0건 | 추첨대기 단계 자체에 도달하지 않음 — [CEB-BO-RFL-201-MD-CLOSE] §3에 따라 진행중 → 종료로 즉시 전환되며, 추첨내역 탭에 "추첨 없이 자동 종료 — 입상자 없음" 표기 노출 |
| **추첨 풀 0명 (전원 제외)** | **[추첨 확정] 버튼 비활성**. 본문 영역에 안내문 노출. 운영자는 [CEB-BO-RFL-202] 응모내역 탭으로 돌아가 일부 회원의 제외를 해제하거나, [CEB-BO-RFL-201-MD-CLOSE]의 0건 종료 흐름과 동일하게 "추첨 없이 자동 종료" 처리 가능 |
| 재추첨 횟수 | **무제한** — 확정 전까지 자유롭게 시드 변경 가능. 각 재추첨 시도는 활동 로그에 기록. 재추첨 시 제외 명단은 그대로 유지 |
| 한 번 확정 후 되돌리기 | **완전 불가** — 종료 상태에서는 추첨 재실행 불가 (시드 영구 기록, 운영자 실수·부정 발견 시에도 되돌릴 수 없음). 제외 명단도 동결 |
