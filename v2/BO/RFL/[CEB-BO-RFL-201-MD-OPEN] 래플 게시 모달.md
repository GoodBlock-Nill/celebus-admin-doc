## 1. 화면 개요

임시저장 상태의 래플을 진행중 상태로 전환하는 확인 모달. 게시 시점에 시작일시가 자동 기록되며 회원 앱에 즉시 노출된다.

> **모달 진입 시점에는 게시 조건이 모두 충족된 것이 보장됨**. [CEB-BO-RFL-201] §4-2에 명시된 [게시하기] 버튼 활성화 조건 9개가 모두 통과한 후에만 모달이 노출된다. 누락 필드가 있으면 버튼이 비활성 상태로 표시되어 모달 자체에 진입할 수 없다.

| 항목 | 값 |
|---|---|
| 모달 트리거 | 래플 상세 헤더의 [게시하기] 버튼 (활성 상태일 때만) |
| 진입 조건 | 래플 상태 = 임시저장 + [CEB-BO-RFL-201] §4-2의 게시 조건 9개 모두 충족 |
| 영향 | 상태가 임시저장 → 진행중으로 전환되고, 시작일시가 현재 시각으로 자동 기록됨 |
| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 3. 화면에 노출되는 정보 / 게시 후 변화

게시 모달 자체는 단순 확인 모달로 정보 노출이 거의 없다. 게시 확정 후 다음 정보가 변경된다.

| 항목 | 게시 전 | 게시 후 |
|---|---|---|
| 상태 | 임시저장 | 진행중 |
| 시작일시 | 비어 있음 | 현재 시각 자동 기록 |
| 최근 수정자 | (이전 값) | 현재 운영자 |
| 최근 수정 일시 | (이전 값) | 현재 시각 |
| 앱 노출 | 노출되지 않음 | 즉시 노출, 회원 응모 가능 |

## 4. 검증 규칙

본 모달은 **사전 검증 통과 후에만 진입**하는 단순 확인 모달이므로 모달 자체의 검증 분기는 최소화된다. 모든 게시 조건은 [CEB-BO-RFL-201] §4-2에서 정의되며, 1개라도 미충족이면 [게시하기] 버튼이 비활성 상태로 표시되어 본 모달에 진입할 수 없다.

| 검증 조건 | 동작 |
|---|---|
| 게시 조건 9개 모두 충족 ([CEB-BO-RFL-201] §4-2) | 버튼 활성 → 본 모달 진입 가능 |
| 게시 조건 1개라도 미충족 | 버튼 비활성(회색) → 클릭 불가, 모달 진입 차단. 운영자는 [수정하기]에서 누락 항목 직접 확인 |
| 게시 권한 없는 운영자 | 래플 상세 헤더에서 [게시하기] 버튼 자체가 노출되지 않음 ([관리자 > 권한관리] 설정에 따름) |
| 래플 상태 ≠ 임시저장 | 헤더 액션 자체가 미노출되어 모달 진입 불가 |

> **다국어 검증 정책**: 임시저장 단계에서는 **한국어/영어/일본어 중 1개 언어 이상**만 입력해도 저장 허용 (한국어가 비어 있고 영어만 채워져도 가능). 다만 **게시 시점에는 한국어/영어/일본어 3개 모두 필수**. 누락된 언어가 있으면 [게시하기] 버튼이 비활성 상태로 표시되어 본 모달에 진입할 수 없으며, 운영자가 [CEB-BO-RFL-201-EDIT] 수정 화면에서 보완해야 한다.
