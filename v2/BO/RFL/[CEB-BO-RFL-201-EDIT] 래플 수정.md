## 1. 화면 개요

기존 래플 한 건의 마스터 정보를 수정한다. 래플 생성 화면([CEB-BO-RFL-101-CREATE])과 **동일한 3열 입력 폼**을 재사용하되, 진행중·종료·추첨대기 상태에 따라 일부 항목을 잠근다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/raffle/{id}/edit` |
| Breadcrumb | 래플 / 래플 상세 / 수정 |
| 페이지 제목 | "래플 수정" + 래플 타이틀 + 상태 배지 |
| 진입 경로 | [CEB-BO-RFL-201] 헤더의 [수정하기] 버튼 (임시저장·진행중에서만 노출) |

## 2. 화면 구성

### 2-1. 헤더

- Breadcrumb (3단계)
- 페이지 제목 "래플 수정" + 래플 타이틀 + 현재 상태 배지
- 우측 액션 2개:
  - **[취소하기]** — 변경사항 있을 시 확인 다이얼로그 노출 후 상세 화면으로 복귀
  - **[저장하기]** — 모든 필수 항목 입력 완료 시 활성

### 2-2. 상태별 안내 배너 (조건부 노출)

| 래플 상태 | 안내 배너 | 톤 |
|---|---|---|
| 진행중 | "**진행중 상태**에서는 응모자 보호를 위해 **아티스트 · 마감일시 · 당첨 추첨 수 · 보상지급 타입 · 추가 보상(BIVE)** 변경이 차단됩니다. 이미지·타이틀·설명·경품 상세·수령 가이드·유의사항만 수정 가능합니다." | 노랑 |
| 추첨대기·종료 | "**{상태}** 상태의 래플은 수정할 수 없습니다. 본 페이지는 읽기 전용입니다." | 분홍 |
| 임시저장 | (배너 없음) | — |

### 2-3. 입력 폼

[CEB-BO-RFL-101-CREATE] §2-2의 3열 폼과 동일한 구성. 차이점:

- 초기값이 현재 래플 정보로 미리 채워져 있음
- 상태별로 일부 항목 잠금 (§3 잠금 매트릭스)
- 추첨대기·종료 상태에서는 전체 폼이 읽기 전용

## 3. 화면에 노출되는 정보 / 입력 잠금 매트릭스

| 항목 | 임시저장 | 진행중 | 추첨대기·종료 |
|---|---|---|---|
| 아티스트 | 수정 가능 | **잠금** | 읽기 전용 |
| 마감일시 | 수정 가능 | **잠금** | 읽기 전용 |
| 대표 이미지 | 수정 가능 | 수정 가능 | 읽기 전용 |
| 타이틀 (다국어) | 수정 가능 | 수정 가능 | 읽기 전용 |
| 설명 (다국어) | 수정 가능 | 수정 가능 | 읽기 전용 |
| 당첨 추첨 수 | 수정 가능 | **잠금** | 읽기 전용 |
| 보상지급 타입 | 수정 가능 | **잠금** | 읽기 전용 |
| 경품 단위·내역 | 수정 가능 | 수정 가능 | 읽기 전용 |
| **추가 보상 (BIVE)** | 수정 가능 | **잠금** | 읽기 전용 |
| 수령 가이드 / 배송 가이드 (deliveryType별 필드) | 수정 가능 | 수정 가능 | 읽기 전용 |
| 유의사항 (다국어) | 수정 가능 | 수정 가능 | 읽기 전용 |

> **정책 배경**: 진행중 상태에서 잠긴 5개 항목(아티스트/마감일시/당첨 추첨 수/보상지급 타입/추가 보상 BIVE)은 이미 응모한 회원에게 직접 영향을 주는 핵심 조건이므로 응모자 보호 차원에서 차단된다. 마감일시는 단축·연장 모두 불가. BIVE 보상은 응모자의 "BIVE 받을 기대권" 보호를 위해 게시 후 변경 자체 차단(OFF→ON, ON→OFF, 캠페인 교체 모두 불가).

## 4. 검증 규칙 (저장 가능 조건)

[CEB-BO-RFL-101-CREATE] §4와 동일한 필수 입력 조건. 추가 조건:

| 조건 | 동작 |
|---|---|
| 래플 상태 = 추첨대기 또는 종료 | [저장하기] 비활성 (전체 읽기 전용) |
| 진행중 상태에서 잠긴 항목 변경 시도 | 입력 자체가 비활성 |
| 변경사항이 전혀 없는 채 저장 시도 | 토스트 "변경사항이 없습니다" 노출 후 화면 유지 |
