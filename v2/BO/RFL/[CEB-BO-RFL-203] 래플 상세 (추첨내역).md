래플 상세 페이지의 세 번째 탭. 운영자가 추첨을 실행하고, 추첨 완료 후 당첨자 명단·보상 지급 상태를 관리하는 화면. 래플 운영 흐름의 종착점.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/raffle/{id}` (탭 전환) |
| Breadcrumb | 래플 / 래플 상세 / 추첨내역 |
| 진입 경로 | [CEB-BO-RFL-201] 상세 화면에서 "추첨내역" 탭 클릭 |
| 핵심 액션 | 추첨대기 → 추첨 실행 → 종료 / 종료 → 당첨자별 보상 지급 토글·비고 입력 |

## 2. 화면 구성

### 2-1. 상태별 분기

본 탭은 래플 상태에 따라 **3가지 화면**으로 분기된다.

| 래플 상태 | 노출 화면 |
|---|---|
| 임시저장 / 진행중 | §2-2 안내 메시지만 노출 |
| 추첨대기 | §2-3 추첨하기 버튼 노출 |
| 종료 | §2-4 통계 + 추첨 시드 + 결과 테이블 |

### 2-2. 안내 메시지 (임시저장·진행중)

중앙 정렬 회색 안내 박스:
- "래플 상태가 추첨대기일 때만 추첨이 가능합니다."

추첨하기 버튼 미노출.

### 2-3. 추첨하기 버튼 (추첨대기)

중앙 정렬:
- 안내문: "추첨대기 상태에서는 하단 추첨하기 버튼을 눌러 추첨을 진행해주세요"
- 큰 **[추첨하기]** 버튼 → 클릭 시 추첨 모달([CEB-BO-RFL-203-MD-DRAW]) 노출

### 2-4. 종료 상태 화면

#### 2-4-1. 통계 카드 (가로 4개)

| 카드 | 의미 |
|---|---|
| 보상지급 | 보상이 지급 완료된 당첨자 수 |
| 총 응모 유저 | 본 래플에 응모한 회원 수 |
| 당첨 | 추첨 결과 당첨자 수 |
| 미당첨 | 응모했으나 미당첨된 인원 수 |

#### 2-4-2. 추첨 시드 안내 영역

추첨 실행 시 생성된 시드 정보를 1줄로 노출 (분쟁 시 추첨 재현 검증용):
- "추첨 시드 {시드값} · {추첨 일시} · {추첨 운영자}"

운영 환경에서는 시드값을 통해 추첨이 결정론적으로 재현 가능하며, 부정 추첨 의혹 발생 시 검증 가능.

#### 2-4-3. 필터·검색 바

| 위치 | 요소 |
|---|---|
| 좌 | 상태 드롭다운 — "전체 / 당첨 / 미당첨" |
| 우 | 검색 대상 드롭다운 — "닉네임 / 전화번호" |
| 우 | 키워드 입력 칸 |
| 우 | [초기화] 버튼 |
| 우 | **[당첨자 명단 다운로드] 버튼** — CSV 다운로드 (§2-4-3-1 참조) |

##### 2-4-3-1. 당첨자 명단 CSV 다운로드

| 항목 | 값 |
|---|---|
| 버튼 라벨 | "당첨자 명단 다운로드" + 다운로드 아이콘 |
| 노출 조건 | 종료 상태 + 당첨자 ≥ 1명. 필터 바 자체가 종료 상태에서만 렌더링되므로 응모자 0건 자동 종료에서는 자연 차단 |
| 필터 영향 | **없음** — 화면 상단 상태·검색 필터와 무관하게 항상 **전체 당첨자** 다운로드 |
| CSV 컬럼 3종 | 유저 닉네임 / 전화번호 / 누적 응모권 |
| 정렬 | 누적 응모권 내림차순 (다수 응모자 우선) |
| 인코딩 | UTF-8 with BOM (엑셀에서 한글 깨짐 방지) |
| 줄바꿈 | `\r\n` (윈도우/엑셀 호환) |
| 셀 형식 | 모든 값 큰따옴표 감싸기 (전화번호 앞자리 0 보호) |
| 파일명 | `raffle-{래플ID}-winners-{YYYYMMDD-HHmm}.csv` |
| 클릭 동작 | 브라우저 다운로드 즉시 트리거. 모달 없음 |

#### 2-4-4. 추첨 결과 테이블 (7개 컬럼)

|---|---|---|
| 상태 | 좌측 | "당첨" 배지(노랑) 또는 "미당첨" 배지(회색) |
| 유저 닉네임 | 좌측 | 응모자 닉네임 |
| 전화번호 | 좌측 | 회원 전화번호 |
| 누적 응모권 | 우측 | 추첨 당시의 누적 응모권 (가중치 기준) |
| 추첨 일시 | 우측 | 추첨이 실행된 시각 |
| 보상 지급 | 우측 | **토글 스위치 — 즉시 토글 (확인 다이얼로그 없음)**. 운영자가 클릭하여 지급/미지급 상태를 한 번의 클릭으로 전환. **당첨자만 활성**, 미당첨자는 토글 비노출. 토글 변경 시 통계 카드의 "보상지급" 수치와 [CEB-BO-RFL-201] 기본정보 탭의 "지급 완료" 카드가 실시간 반영됨 |

### 4-1. 운영 정책

- **다른 래플에서 동일 회원 중복 당첨 허용** — 각 래플은 독립적으로 추첨되며, 회원이 여러 래플에 응모해 모두 당첨되어도 무방
- **자동 종료 알림** — 마감일시 도래로 자동 종료되어 추첨대기 상태로 전환될 때 관리자 대시보드에 알림 노출 + 활동 로그 기록
- **응모자 0건의 자동 종료** — 응모자가 없는 채로 자동 종료된 경우 본 탭은 "추첨 없이 자동 종료 — 입상자 없음" 표기를 노출. 추첨하기 버튼 미노출

### 4-2. 페이지네이션 (종료 상태 결과 테이블)

- **페이지당 20건** (상태 필터·검색 결과 기준)
- **항상 노출** — 결과가 20건 미만이라 1페이지일 때도 `‹ 1 ›` 비활성 형태로 노출 (운영 사이트 정합)
- **윈도잉 규칙** — 총 페이지 ≤ 7이면 전체 페이지 번호 노출, 8페이지 이상이면 `1 ... cur-1 cur cur+1 ... last` 윈도우 + 양 끝(`...`) 표시
- 상태 필터·검색·키워드 변경 시 자동으로 1페이지로 리셋
