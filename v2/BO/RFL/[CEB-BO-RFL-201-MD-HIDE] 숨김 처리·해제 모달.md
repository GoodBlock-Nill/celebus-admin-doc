## 1. 화면 개요

종료된 래플을 앱에서 노출하지 않도록 처리하거나, 다시 노출하도록 해제하는 토글 모달. 데이터와 추첨내역은 보존되며 운영 화면에서는 계속 조회 가능하지만, 회원 앱 측 노출만 제어한다.

| 항목 | 값 |
|---|---|
| 모달 트리거 | 래플 상세 헤더의 [숨김 처리] 또는 [숨김 해제] 버튼 |
| 진입 조건 | 래플 상태 = 종료 |
| 영향 | 앱 노출 여부 토글 (숨김 ↔ 노출) |
| 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |

## 2. 화면 구성

### 2-1. 헤더

현재 상태에 따라 제목이 분기된다.

| 현재 노출 상태 | 모달 제목 |
|---|---|
| 노출 중 (숨김 아님) | "숨김 처리하시겠어요?" |
| 숨김 중 | "숨김 해제하시겠어요?" |

### 2-2. 본문 메시지

**숨김 처리 시:**
```
본 래플이 앱에서 숨겨집니다.
다시 노출하려면 [숨김 해제]로 전환하세요.
```

**숨김 해제 시:**
```
앱에서 다시 노출됩니다.
```

### 2-3. 액션 버튼

| 버튼 | 색상 | 동작 |
|---|---|---|
| 취소 | 회색 외곽선 | 모달 닫기 |
| **숨김 처리** / **숨김 해제** | 빨강 (숨김 처리) / 인디고 (숨김 해제) | 노출 상태 토글 |

## 3. 화면에 노출되는 정보 / 토글 후 변화

| 항목 | 변화 |
|---|---|
| 앱 노출 여부 | 토글됨 (노출 ↔ 숨김) |
| 최근 수정자·일시 | 현재 운영자 / 현재 시각 |
| 활동 로그 | 보존 — "래플 '{타이틀}'을(를) 숨김 처리/해제했습니다." |

추가 영향:
- **숨김 적용 범위는 앱 "래플 리스트"에서만**. 앱의 래플 메인 목록에서 노출 제외
- 이미 응모한 회원은 앱 "내 래플 이력" 화면에서 자신의 응모·당첨/미당첨 결과를 그대로 조회 가능 (숨김의 영향을 받지 않음)
- 운영 리스트([CEB-BO-RFL-101])에서는 "숨김" 배지로 시각 구분
- 회원 자산(응모권·BIVE 보유)에는 영향 없음

## 4. 검증 규칙

| 검증 조건 | 동작 |
|---|---|
| 래플 상태 ≠ 종료 | 헤더의 [숨김 처리/해제] 액션 미노출 (임시저장·진행중·추첨대기는 액션 없음) |
| 숨김 처리 권한이 없는 운영자 | 헤더의 [숨김 처리/해제] 액션이 노출되지 않음 ([관리자 > 권한관리] 설정에 따름) |
