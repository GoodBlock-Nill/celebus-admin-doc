## 1. 화면 개요

신규 래플 한 건을 임시저장 상태로 생성하는 입력 화면. 3열 카드 레이아웃으로 입력 영역을 분리하여 운영자가 항목군별로 작성하기 쉽게 구성한다. **생성 시점에는 항상 임시저장 상태로 저장**되며, 회원에게 노출되려면 별도 게시 작업이 필요하다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/raffle/create` |
| Breadcrumb | 래플 / 래플 생성 |
| 페이지 제목 | "래플 생성" |
| 진입 경로 | [CEB-BO-RFL-101] 리스트의 [생성하기] 버튼 |
| 저장 결과 | 임시저장 상태로 생성. 게시는 [CEB-BO-RFL-201-MD-OPEN] 모달에서 별도 진행 |

## 2. 화면 구성

### 2-1. 헤더

- 페이지 제목 "래플 생성"
- 우측 액션 2개:
  - **[취소하기]** — 변경사항이 있으면 확인 다이얼로그 노출 후 리스트로 복귀
  - **[생성하기]** — 모든 필수 항목 입력 완료 시 활성. 클릭 시 임시저장 상태로 저장

### 2-2. 3열 입력 폼

#### 좌측 — 래플 정보

| 입력 항목 | 입력 방식 | 글자수·제약 | 필수 |
|---|---|---|---|
| 아티스트 | 드롭다운 선택 (V01D / iKON / CELEBUS) | — | ★ |
| 마감일시 | 날짜 + 시간 선택 (기본 시간 23:59) | — | ★ |
| 대표 이미지 | 드래그앤드롭 또는 URL 직접 입력 | 권장 686×332px (가로 2:1) — 안내만, 강제 검증 없음. **파일 크기 5MB 초과만 차단**. 형식 jpeg/png/jpg/webp/gif/svg | ★ |
| 타이틀 | 한국어/영어/일본어 탭으로 전환 입력 | 각 100자 이내 | ★ (게시 시 3개 언어 모두 필수, 임시저장은 **1개 언어 이상**이면 가능) |
| 설명 | 한국어/영어/일본어 탭으로 전환 입력 | 각 200자 이내 | ★ (위와 동일) |

#### 중앙 — 보상 기본 설정

| 입력 항목 | 입력 방식 | 비고 | 필수 |
|---|---|---|---|
| 당첨 추첨 수 | 숫자 입력 | 최소 1, 기본 1 | ★ |
| 보상지급 타입 | 드롭다운 선택 (**현장 수령 / 배송 수령** 2종) | 운영 정합 — 온라인 옵션 없음 | ★ |
| 경품 단위 | 토글 (장 / 개) | 기본 "장" | — |
| 경품 내역 | 한국어/영어/일본어 탭으로 전환 입력 | 각 50자 이내 | ★ (한국어 필수) |

**추가 보상 설정 (BIVE)** — 같은 카드 하단에 분리선과 함께 배치:

| 입력 항목 | 입력 방식 | 비고 |
|---|---|---|
| BIVE 보상 지급 | 토글 체크박스 | 기본 OFF |
| 민팅 이벤트 | 드롭다운 선택 (등록된 BIVE 캠페인 중 1종) | 토글 OFF 시 선택 불가 |
| 안내 문구 | (참고 안내) | "추첨 확정 시 당첨자에게 BIVE NFT가 자동 민팅됩니다. 경품과 함께 지급됩니다." |

> **검증 규칙**: BIVE 토글 ON + 민팅 이벤트 미선택 시 [생성하기] 비활성.
> **정책**: 1 래플 = 1 BIVE 매핑 (팬퀘스트 정책 준용). 등록 가능한 캠페인 목록은 [CEB-BO-004] BIVE 영역의 운영 상황에 따른다.

#### 우측 — 수령 가이드 및 유의사항

**수령 가이드 박스 — 보상지급 타입에 따라 조건부 렌더링**:

박스 제목과 입력 필드가 위 §2-3 보상지급 타입 드롭다운 선택값에 따라 동적으로 전환된다.

##### 현장 수령 선택 시 — 박스 제목 "수령 가이드"

| 입력 항목 | 입력 방식 | 글자수 | 필수 |
|---|---|---|---|
| 수령 시작일·마감일 | 날짜 선택 (2개) | — | ★ |
| 운영 시작 시간·마감 시간 | 시간 선택 (2개) | 기본 09:00 ~ 18:00 | ★ |
| 수령 장소 | 한국어/영어/일본어 탭 입력 | 각 100자 이내 | ★ (1개 언어 이상) |
| 지참물 | 한국어/영어/일본어 탭 입력 | 각 50자 이내 | ★ (1개 언어 이상) |

##### 배송 수령 선택 시 — 박스 제목 "배송 가이드"

| 입력 항목 | 입력 방식 | 비고 | 필수 |
|---|---|---|---|
| 배송지 입력 마감일 | 날짜 선택 | — | ★ |
| 배송지 입력 마감 시간 | 시간 선택 | 기본 23:59 | ★ |
| 배송지 입력 폼 URL | URL 입력 (https://) | 외부 폼 링크 (Google Forms 등) | ★ |

> **다국어 입력 없음**: 배송 수령은 회원에게 외부 폼 URL을 통해 배송지를 입력받는 흐름이므로, 다국어 입력이 필요하지 않다. 배송 안내는 §유의사항 영역에 다국어로 작성한다.

**유의사항** (별도 입력 박스, 보상지급 타입 무관 공통):
- 한국어/영어/일본어 탭 입력, 각 500자 이내, 1개 언어 이상 필수

## 3. 화면에 노출되는 정보 / 입력되는 정보

본 화면은 **신규 데이터 입력 화면**이므로 노출보다는 입력이 중심.

| 영역 | 입력 정보 | 의미 |
|---|---|---|
| 좌측 카드 | 래플 정보 | 래플의 기본 식별 정보와 회원이 보게 될 콘텐츠 |
| 중앙 카드 | 보상 기본 설정 | 당첨자 수, 경품 지급 방식, 경품 상세, 추가 BIVE 보상 |
| 우측 카드 | 수령 가이드·유의사항 | 당첨 후 수령 절차와 안내 메시지 |

입력 완료 시 임시저장 상태로 저장되며 다음 정보가 자동 기록된다:
- 신규 ID 발번
- 상태 = "임시저장"
- 시작일시 = 비어 있음 (게시 시점에 기록)
- 누적 참여자 / 누적 응모권 = 0
- 생성자 / 생성 일시 / 최근 수정자 / 최근 수정 일시 = 현재 운영자 / 현재 시각

## 4. 검증 규칙 (저장 가능 조건)

다음 항목이 모두 충족되어야 [생성하기] 버튼이 활성된다.

| 항목 | 조건 |
|---|---|
| 아티스트 | 선택 |
| 마감일시 | 날짜 입력 |
| 대표 이미지 | 등록 (URL 또는 업로드), 파일 크기 5MB 이하 |
| 다국어 텍스트 (타이틀·설명·경품·수령장소·지참물·유의사항) | **임시저장 단계에서는 한국어/영어/일본어 중 1개 언어 이상 입력**이면 통과 (한국어가 비어 있고 영어만 채워져도 가능) |
| 당첨 추첨 수 | 1 이상 |
| 수령 시작일·마감일 | 둘 다 입력 |
| BIVE 보상 토글 ON | 민팅 이벤트 선택 필수 |

> **임시저장 vs 게시 검증의 차이**: 본 화면은 임시저장 단계이므로 한국어/영어/일본어 중 **1개 언어 이상**이면 저장 가능. 게시([CEB-BO-RFL-201-MD-OPEN])할 때는 한국어/영어/일본어 3개 언어 모두 입력되어야 한다.
