## 1. 개요

관리자(SYS) 영역은 BO 사용자 계정·권한·활동 로그를 관리한다. v1에서 관리자 초대·리스트·권한 관리(권한 그룹·권한 생성·권한 상세 3탭)를 운영 중이며, v2에서 **4단계 권한 + 그룹 커스터마이징 하이브리드**를 신규 정의한다. 본 영역은 **SUPER 전용** (활동 로그 조회는 자기 활동만 다른 권한도 R 가능).

## 7. 영역 고유 정책

### 7.1 4단계 권한 + 그룹 하이브리드

[CEB-BO-000] §3.1·§3.2 기반. **하이브리드 모델**:

```
4개 기본 프리셋 그룹 (시스템 정의, 수정 불가)
├── SUPER          — 모든 영역 R/W/D + 권한 부여·회수
├── OPERATOR       — 일상 운영 (Quest 검수·콘텐츠 관리·결과 입력 등)
├── POLICY         — 정책 매개변수 (덕력 캡·팬덤 곡선·교환 비율 등)
└── EXTERNAL_ARTIST— 외부 협력사 (자기 아티스트 한정)

+ 운영자 정의 커스텀 그룹 (SUPER가 추가)
├── 예: "QA Reviewer"  — Quest 검수만 가능, 다른 영역 R only
├── 예: "콘텐츠 매니저"  — INF·HOM 영역 W, 다른 영역 차단
└── 예: "임시 운영자"   — OPERATOR 권한 + 90일 만료 강제
```

#### 그룹 ↔ 관리자 N:N

| 항목 | 정책 |
| --- | --- |
| 한 관리자 = 여러 그룹 가능 | 권한은 OR 합집합 (가장 권한이 큰 액션 적용) |
| 그룹 = N명의 관리자 매핑 | 한 그룹 변경 시 매핑된 모든 관리자에게 즉시 반영 |
| 기본 프리셋 그룹 | 시스템 정의, 수정·삭제 불가 (이름·설명만 SUPER가 변경 가능) |
| 커스텀 그룹 | SUPER가 자유 생성·수정·삭제 (가드레일 적용) |
| 충돌 방지 | 한 관리자가 여러 그룹에 매핑되어 권한 충돌 시 더 큰 권한 우선 (Union) |

### 7.2 관리자 초대 흐름

```
[CEB-BO-SYS-101-Invite] 관리자 초대 화면
    ↓
입력: 이메일 + 이름 + 권한 그룹 (1~N개) + 만료일 (선택, EXTERNAL_ARTIST 필수) + IP 화이트리스트 (SUPER·EXTERNAL_ARTIST 필수)
    ↓
저장 → 초대 메일 발송 (KO/EN/JP — 외부 권한자는 EN 우선)
    ↓
초대받은 관리자가 링크 클릭 → 비밀번호 설정 → 2FA 등록 → 활성화
    ↓
활동 로그 기록 (초대 발송·수락 각각)
```

#### 초대 만료 정책

| 항목 | 정책 |
| --- | --- |
| 초대 링크 유효 기간 | 7일 (만료 시 SUPER가 재발송) |
| 비밀번호 미설정 | 초대 만료 시 자동 폐기. SUPER 재초대 필요 |
| 2FA 미등록 | 초대 만료 시 비밀번호 설정 무효화 + 재초대 필요 |
| EXTERNAL_ARTIST 만료일 | [CEB-BO-000] §3.2 — 기본 6개월, 갱신 가능 (SYS-201-MD-EXTEND) |

### 7.3 권한 그룹 상세 3탭

[CEB-BO-SYS-202]에서 사용. v1 3탭 유지 + v2 12영역 매트릭스 적용.

| # | 탭명 | 표시 항목 |
| --- | --- | --- |
| 1 | **기본** | 그룹명·설명·생성일·수정일·생성자·매핑 관리자 수·시스템 정의 여부 |
| 2 | **권한 설정** | **12영역 × R·W·D 매트릭스 (체크박스 토글)** + 영역별 SUPER 전용 액션 표시 |
| 3 | **관리자** | 본 그룹에 매핑된 관리자 리스트 (이메일·이름·매핑일·만료일·계정 상태) — 추가·해제 액션 |

```

SUP (서포트)          ☐    ☐    ☐
```

> **SUPER 전용 액션** (영구 정지·자산 회수·Genesis 민팅·Private Key 조회 등)은 매트릭스 토글로 부여 불가 — SUPER 프리셋 그룹에 매핑된 관리자만 가능.

#### 그룹 삭제 가드레일

[CEB-BO-SYS-202-MD-DELETE]:
- 매핑된 관리자가 1명 이상이면 삭제 차단 → "이 그룹에 매핑된 관리자 {N}명을 다른 그룹으로 재할당 후 다시 시도하세요" 토스트
- 시스템 정의 4개 프리셋 그룹은 삭제 불가 ([삭제] 버튼 자체 비활성)

### 7.4 권한 부여·회수 정책

[CEB-BO-000] §3.2 적용 + §3.3 IP 제한 호환.

| 항목 | 정책 |
| --- | --- |
| **만료일** | EXTERNAL_ARTIST 필수 (기본 6개월). 다른 권한은 옵션 (영구 가능) |
| **IP 화이트리스트** | SUPER·EXTERNAL_ARTIST 필수. OPERATOR·POLICY는 권장 (옵션) |
| **시간대 제한** | EXTERNAL_ARTIST 평일 09:00~22:00 KST (협의 후 조정). 다른 권한은 24/7 |
| **90일 비활성 자동 잠금** | 모든 권한 공통 (시스템 자동) — SUPER 재승인 후 잠금 해제 |
| **2FA** | 모든 권한 필수 (앱 OTP 또는 SMS) |
| **회수 즉시 효과** | SUPER가 권한 회수 시 → 즉시 세션 강제 종료 + 데이터 접근 차단 |
| **만료 연장** | SUPER만 [CEB-BO-SYS-201-MD-EXTEND]에서 연장. 사유 필수 |

### 7.5 활동 로그 (CEB-BO-SYS-401)

[CEB-BO-000] §6 표준 적용.

| 항목 | 정책 |
| --- | --- |
| 보관 기간 | 3년 (5년 권장 — 운영 결정) |
| 로그 형식 | append-only (수정·삭제 불가) |
| 필수 필드 | 시각·관리자 ID·영역·액션·대상·사유 (해당 시)·이전 값·신규 값·IP·User-Agent |
