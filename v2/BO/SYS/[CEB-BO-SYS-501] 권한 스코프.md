## 2. 화면 목적

각 역할(Role)에 **"운영 가능 아티스트" 다중 선택 메타필드(`scopeArtistIds`)**를 부여하여, 같은 역할이라도 운영자마다 접근 가능한 아티스트 범위를 다르게 설정할 수 있도록 한다.

> 본 화면은 **정책 마스터**다. 실제 역할 정의·운영자 매핑은 [CEB-BO-SYS-202] 권한 그룹 상세에서 수행한다.

---

## 3. 정책 결정 — 전역 + 스코프 메타

[CEB-BO-100] §3 결정:

| 측면 | 채택 |
| --- | --- |
| 역할 정의 | **전역** — 모든 아티스트 공유 (예: "Quest 검수자" 1개 역할 정의로 모든 아티스트 사용) |
| 적용 범위 | **스코프 메타** — 역할·운영자 단위로 `scopeArtistIds` 부여 |
| 결과 | 역할 정의는 1벌, 적용 범위는 매핑 시점에 분기 |

---

## 4. 데이터 구조

### 4.1 역할(Role) 테이블 확장

```text
Role {
  id: ID
  name: string                   // 예: "Quest 검수자"
  description: string
  permissions: {                 // 영역×CRUD 매트릭스
    FQ: { R, W, D },
    BIVE: { R, W, D },
    ...
  }
  scopeArtistIds: ID[] | null    // ⭐ v1.0 신규 — 운영 가능 아티스트 ID 배열. NULL = 전역(모든 아티스트)
  createdAt, updatedAt, ...
}
```

### 4.2 운영자 매핑 (`AdminRoleAssignment`)

```text
AdminRoleAssignment {
  adminId: ID
  roleId: ID
  scopeArtistIds: ID[] | null    // ⭐ v1.0 신규 — 운영자별 추가 스코프 좁힘. NULL = Role 기본 스코프 사용
  expiresAt: timestamp | null
}
```

> **두 단계 스코프**: ① Role 자체 `scopeArtistIds` (역할 디폴트), ② 운영자 매핑 시 `scopeArtistIds` (개별 좁힘). 운영자 실제 접근 범위 = Role 스코프 ∩ 매핑 스코프.

---

## 5. 권한별 스코프 룰

| 권한 | Role `scopeArtistIds` | 매핑 시 좁힘 가능 |
| --- | --- | --- |
| **SUPER** | NULL (전역 강제) | ❌ — 좁힘 불가 |
| **OPERATOR** | NULL (디폴트 전역) 또는 ID 배열 | ✅ — 매핑 시 추가 좁힘 가능 |
| **POLICY** | NULL (디폴트 전역) 또는 ID 배열 | ✅ — 좁힘 가능 |
| **EXTERNAL_ARTIST** | 자기 아티스트 ID 1개로 **자동 강제** | ❌ — 변경 불가 |

> EXTERNAL_ARTIST는 [CEB-BO-100] §4에 따라 자기 아티스트만 R/W. 다른 아티스트 ID를 스코프에 추가 시도 시 차단.

## 8. 규칙·로직

### 8.1 스코프 검증

- SUPER 권한 단계의 역할은 디폴트 스코프 = 전역 강제. 다중 선택 시도 시 차단.
- EXTERNAL_ARTIST 권한 단계의 역할은 디폴트 스코프 = 자기 아티스트 1개 강제. 다른 아티스트 추가 시도 시 차단.
- OPERATOR/POLICY는 자유롭게 전역 또는 다중 선택.

### 8.2 디폴트 스코프 변경 영향

- 디폴트 스코프 변경 시 매핑된 모든 운영자의 접근 범위 즉시 재계산.
- 매핑 시 별도 좁힘이 있는 운영자는 영향 없음 (Role 스코프 ∩ 매핑 스코프 그대로 유지).
- 변경 시 매핑 운영자 N명에게 시스템 알림 (선택).

### 8.3 권한 분기

| POLICY | ❌ 차단 |

## 13. 도입 시점

[CEB-BO-100] §6 로드맵 — **Phase 2** 운영 정책 분리 단계.

- Phase 1: Role 데이터 모델에 `scopeArtistIds` 필드만 추가 (NULL 디폴트).
- Phase 2: 본 화면 활성화 + 운영자 매핑 시 좁힘 가능.
- Phase 3: 헤더 글로벌 토글과 연동하여 EXTERNAL_ARTIST 사이드바 자동 가림.
