## 1. 화면 개요

사용자의 **모든 GP 증감 내역**을 통합 조회하는 화면. 게임 참여·부스팅·환급·보상·GP 충전·GP 출금에 더해 게임존 외 영역(출석체크·응모권 구매 등)의 GP 변동까지 포함한 단일 인벤토리. 운영자는 변동 유형·기간·닉네임 필터로 좁힌 뒤 행 클릭으로 [CEB-BO-GZ-601-MD-DETAIL] 변동 상세 모달을 열어 변동 ID·지갑·잔액을 확인한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/gamezone/gp-history` |
| Breadcrumb | 홈 › 게임존 › GP 변동 내역 |
| 페이지 제목 | "GP 변동 내역" |
| 진입 경로 | 사이드바 "게임존 › GP 변동 내역" |
| 접근 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |
| 용도 | 통합 GP 변동 조회·검색, 변동 상세 진입 |

> **포괄 범위**: 게임존(PM/ST 보상·참여·부스팅·환급) + GP 교환소(GP 충전·GP 출금) + 게임존 외(GP 출석체크·응모권 구매 등) 모든 GP 증감을 단일 인벤토리로 집계. 변동 유형 라벨로 출처를 구분.

## 5. 검증 규칙

### 5-1. GP 변동 유형 분류

[CEB-BO-GZ-000] 공통 정책 §GP 변동 유형 정합:

| 유형 | 부호 | 게임유형 | 비고 |
|---|---|---|---|
| 참여 | − | PM·ST | 게임 참여 시 차감 |
| 부스팅 | − | PM 전용 | 부스팅 사용 시 차감 |
| 환급 | + | PM 전용 | 결과 발표 후 참여 GP 반환 (ST는 환급 없음) |
| 보상 | + | PM·ST | 정답/생존 시 보상 지급 (PM: 비율 배분, ST: 균등 분배) |
| 환불 | + | PM·ST | 게임 취소 시 환불 |
| GP 충전 | + | (게임 외) | CELB → GP 교환 ([GZ-501] 거래로 발생) |
| GP 출금 | − | (게임 외) | GP → CELB 교환 |
| 기타 (게임존 외) | + / − | (게임 외) | GP 출석체크·응모권 구매 등. 비고 텍스트로 출처 표기 |

> 명세 v1.4의 "GP 가져오기 / CELB으로 보내기" → 운영 표기 "GP 충전 / GP 출금"으로 환원 ([GZ-501·502·503 v1.x] 동일 방침. [project_gam_terminology_decision] 정합)

### 5-2. 게임유형 분류

| 게임유형 | 적용 변동 |
|---|---|
| PM (Prediction Market) | 참여 / 부스팅 / 환급 / 보상 / 환불 |
| ST (Survival Trivia) | 참여 / 보상 / 환불 (부스팅·환급 없음) |
| (게임 외) | GP 충전 / GP 출금 / 기타 (출석·응모권 등) → "-" 표기 |

### 5-3. 권한 및 활동 로그

- 권한은 [관리자 > 권한관리]에 설정된 그룹별 권한을 따름 ([CEB-BO-SYS-102])
- 조회 자체는 활동 로그 미기록 (조회 로그 정책 — [CEB-BO-GZ-000] 정합)
- 변동 내역은 삭제·수정 불가 (이력 보존)
