## 1. 화면 개요

새로운 게임을 생성하는 **풀페이지 폼 화면**. 관리자는 게임 유형(PM/ST)을 라디오로 선택한 뒤 기본정보·보상설정·참여설정·일정설정·(PM)결과설정 또는 (ST)퀴즈설정을 입력하여 [임시저장] 또는 [생성하기]를 실행한다. 게임유형에 따라 폼 항목이 조건부로 표시되며, 우측 사이드에 **앱 미리보기 패널**이 상시 노출되어 입력값을 실시간 반영한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/gamezone/games/create?mode=create` |
| Breadcrumb | 홈 › 게임존 › 게임 관리 › 게임 생성 |
| 페이지 제목 | "게임 생성" |
| 진입 경로 | [CEB-BO-GZ-201] 게임 리스트의 [+ 게임 생성] 클릭 |
| 접근 권한 | [관리자 > 권한관리]에 설정된 권한을 따른다 (쓰기 권한 필요) |
| 용도 | 신규 PM/ST 게임 생성 (임시저장 또는 즉시 게시 대기) |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 타이틀 글자수 | 최대 35자 (운영 BO 표기) |
| 결과 확인 기준 글자수 (PM) | 최대 500자 |
| 이미지 비율 (PM 배너 / 상세) | 3:2 / 1:1 |
| 이미지 비율 (ST 배너) | 3:2 |
| 이미지 사양 | jpeg·png·jpg·webp, 각 최대 5MB |
| 다국어 입력 | KO/EN/JA 3종. 운영 BO 실측은 필수 검증을 KO 위주로 적용 (EN/JA 검증 정책은 [CEB-BO-GZ-000] 참조) |
| PM 부스팅 | 부스팅 GP × 배수만큼 보상 분배 가중치 적용 (보상 계산 공식은 [CEB-BO-GZ-000]) |
| ST 보상 유형 | "단계별 보상" 또는 "비율별 보상" 중 1개 선택. 단계별 = 모집인원 구간별 비율 테이블, 비율별 = 단일 비율 분배 |
| ST 적용 상금풀 공식 | 적용 상금풀 = 실제 모집인원 / 최대 모집인원 × 최대 상금풀 (게임 시작 시 자동 결정) |
| ST 부스팅 | 미지원 (안내문 노출) |
| 게임 시작 시각 (ST) | 최소 1시간 이전 권장 (강제 아님, 안내문만) |
| 응모권 토글 | 운영 BO 미적용. 정정 명세 [CEB-BO-RFT-302] 참조 (재개 시점 도입) |
| 권한 | 쓰기 권한 필요. 권한 없으면 [+ 게임 생성] 진입 자체 차단 |
