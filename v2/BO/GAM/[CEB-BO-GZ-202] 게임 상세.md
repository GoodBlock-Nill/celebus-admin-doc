## 1. 화면 개요

게임의 상세 정보를 **3개 탭**으로 조회하고, 게임 상태에 따른 액션(수정·게시·결과 입력·강제 종료·보상 지급 등)을 실행하는 화면. PM/ST 두 게임유형을 지원하며, 상태별로 헤더 우측 액션 버튼이 동적 분기된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/gamezone/games/{gameId}?tab={basic\|entry-history\|result-reward}` (기본 `tab=basic`) |
| Breadcrumb | 홈 › 게임존 › 게임 관리 › 게임 상세 |
| 페이지 제목 | "{게임 타이틀(KO)} {상태배지}" (h1 — 타이틀과 상태배지가 같은 라인에 인라인) |
| 진입 경로 | [CEB-BO-GZ-201] 게임 리스트의 행 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 (읽기 권한). 상태 변경 액션은 쓰기 권한 |
| 용도 | 게임 정보 조회, 참여자 내역 조회, 결과/보상 정보 조회, 상태별 액션 실행 |

| 권한 | 읽기 권한으로 진입. 상태 변경 액션은 쓰기 권한. 권한 없으면 액션 버튼 미노출 |

### 5-1. 결과 입력 사전 검증 (PM Pending)

[결과 입력] 클릭 시 [202-MD-RESULT] 모달에서 YES/NO 선택 + 결과 정보(KO/EN/JP) 입력 후 저장하면 상태가 Pending → **Closed**로 전이. 이후 [보상 지급] 가능. (자세한 동작은 사이클 #2-3c에서 [202-MD-RESULT]·[202-MD-REWARD] 명세에 보강)

### 5-2. 게임 종료 자동 전이 (Ended)

PM/ST 공통 — Closed 상태(PM) 또는 Active 종료 시점(ST)에서 일정 시간 경과 후 **Ended**로 자동 전이. 정확한 경과 시간 정책은 [CEB-BO-GZ-000] 참조.
