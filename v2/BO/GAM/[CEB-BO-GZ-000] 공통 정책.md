### 게임유형별 상태 흐름

- **Prediction Market (PM)**: Draft → Ready → Active → Pending → Closed → Ended
  - Pending: 참여 마감 후 관리자 수동 결과 입력 대기 상태
  - Closed: 결과 확정 후 관리자 수동 보상 지급 대기 상태
- **Survival Trivia (ST)**: Draft → Ready → Active → Ended
  - Pending / Closed 상태 없음 (게임 종료 즉시 서버 자동 결과 확정 및 보상 지급)

## 5. 보상 계산 로직

### Prediction Market 타입1 (유료 참가 — 비율 배분)

- 정답자 보상 = 총 상금 GP × (개인 배분 값 / 전체 합)
- 기본: 개인 배분 값 = 참여 GP
  - 부스팅 구매시: 개인 배분 값 = 참여 GP × 부스팅 배수
- 전체 합 = ∑(각 참여자의 배분 값)
- 참여 GP는 정답/오답 관계없이 전액 환급
- 부스팅 GP는 환급 없음, 배분 가중치로만 사용
- 소수점 8자리 이하 절삭, 단수는 시스템 귀속

> ⚠️ **보류**: PM 타입2 (무료 참가)는 정책 변경으로 보류되었습니다. 이벤트 GP 지급(출석체크/가입보상)으로 타입1(유료) 참가를 유도하는 방식으로 변경되었습니다.

### Prediction Market 타입2 (무료 참가 — 균등 보상)

- 참여비: 0 GP (무료)
- 부스팅: 없음
- 정답자 보상 = 승리 보상 GP (게임 생성 시 설정한 고정값, 정답자 전원 동일 금액)
- 총 상금 GP = 승리 보상 GP × 참여 정원 (게임 생성 시 자동 계산)
- 오답자 보상: 없음
- 참여 GP 환급: 해당 없음 (무료 참가이므로 차감된 GP 없음)
- 소수점 처리: 해당 없음 (고정 금액이므로 소수 발생 안 함)

### Survival Trivia (동적 상금풀 + 균등 분배)

- 참여비 = floor(최대 상금풀 / 최대 모집인원 × 배수)
- 적용 상금풀 결정: 게임 시작 시 실제 모집인원 기준
  - 모집률 = 실제 모집인원 / 최대 모집인원
  - 단계 사이 모집률은 하위 단계 적용 (보수적)
  - 적용 상금풀 = 최대 상금풀 × 해당 단계의 상금 비율
- 생존자 1인당 보상 = floor(적용 상금풀 / 최종 생존자 수)
- 참여비 환급 없음
- 부스팅 없음
- 탈락자 보상: 모든 탈락자에게 팬퀘스트 응모권 지급 (관전 여부 무관)
- 관전 완료 탈락자: 하트 1개 + 응모권 병행 지급
- 전원 탈락 시: 적용 상금풀이 시스템에 귀속
- 소수점 8자리 이하 절삭, 단수는 시스템 귀속
- 보상은 서버에서 자동 지급 (관리자 수동 지급 불필요)

> 응모권 관리: BO 조회 화면 CEB-BO-FQ-301, 앱 유저 확인 화면 CEB-FQ-301

### 응모권·덕력 추가 보상 (PM·ST 공통, v2.4 신규 / v2.5 정책 정정)

게임 생성 시 보상 설정에서 응모권·덕력 1인당 고정 개수를 입력. 결과 확정 후 자동 분배.

**입력**:

| 보상 | PM | ST | 분배 기준 |
|---|---|---|---|
| 응모권 보상 (정답자 1인당) | ✅ `ticketReward` | — | **결과 기반** (정답자) |
| 응모권 보상 (생존자 1인당) | — | ✅ `ticketReward` | **결과 기반** (최종 생존자) |
| 응모권 보상 (탈락자 1인당) | — | ✅ `eliminatedTicketReward` (v2.5 선택화·통일) | **결과 기반** (탈락자 전원) |
| 덕력 보상 (참여자 1인당) | ✅ `dukReward` | ✅ `dukReward` | **참여 기반** (참여자 전원) |

모두 0 이상 정수, **0 = 미지급** (분배·로그 건너뜀). 결과 확정 후 잠금.

**분배 공식 (균등 분배)**:
- PM 응모권: 총 지급 = `ticketReward × 정답자 수`
- ST 응모권 (생존자): 총 지급 = `ticketReward × 최종 생존자 수`
- ST 응모권 (탈락자): 총 지급 = `eliminatedTicketReward × 탈락자 수`
- **PM 덕력**: 총 지급 = `dukReward × 참여자 수` (참여자 전원, 정답/오답 무관)
- **ST 덕력**: 총 지급 = `dukReward × 참여자 수` (참여자 전원, 생존/탈락 무관)

> **v2.5 정정 핵심**: 덕력을 "정답자/생존자 1인당"에서 **"참여자 1인당"**으로 변경. DUK 영역의 `PM_PARTICIPATION` / `TRIVIA_PARTICIPATION` enum이 "참여" 의미인 점과 정합. 참여 자체에 대한 보상이므로 결과 무관.

> **v2.5 정정 핵심 2**: ST "탈락자 응모권 수량"을 **선택**으로 변경 (기본 0 = 미지급). 라벨 "응모권 보상 (탈락자 1인당)"으로 통일 (생존자·정답자 응모권 라벨과 일관).

**변동 출처**:
- 응모권: `[CEB-BO-RFT-201]` 변동 내역에 출처 `"PM 보상"` / `"ST 보상"` 라벨
- 덕력: `[CEB-BO-DUK-401]` 변동 내역에 출처 `PM_PARTICIPATION` / `TRIVIA_PARTICIPATION` enum

**활동 로그** (지급 트리거 시):
- "게임 '{타이틀}'의 보상을 지급했습니다. (GP {총}, 응모권 정답/생존 {총}장, 응모권 탈락 {총}장, 덕력 {총} DUK, 대상 GP {N}명·응모권 {N}명·덕력 참여자 {N}명)"
- 0 항목은 로그에서 생략

> **회원 앱 표시 정합은 별도 사이클** — 본 정책은 BO 한정. APP-GAM 명세 갱신은 후속 task.

| 항목 | 값 |
| ---- | ---- |
| 위치 | 화면 상단 중앙 |
| 상단 여백 | 20px |
| Z-index | 9999 (최상위 레이어) |
| 자동 닫힘 | 3초 |
| 닫기 버튼 | [X] 아이콘 버튼 (우측) |
| 페이드인 | 0.2초 (opacity 0→100%, translateY -10→0px) |
