## 1. 모달 개요

PM 게임의 **결과를 입력하여 Closed 전이**시키는 모달. 결과제목·결과(YES/NO)·결과설명·결과링크를 입력하고 [결과 확정] 시 보상 계산 및 분배 가능 상태로 전환된다.

| 항목 | 값 |
|---|---|
| 트리거 | [CEB-BO-GZ-202] PM Pending 상태의 [결과 입력] 버튼 |
| 표시 조건 | 게임유형=PM + 상태=Pending |
| 영향 범위 | 게임 상태 → Closed / 결과 정보 저장 / [보상 지급] 버튼 활성 |
| 접근 권한 | [관리자 > 권한관리] 설정 (쓰기 권한) |

## 2. 화면 구성

### 2-1. 모달 헤더

- 타이틀 (h3): **"결과 입력"** (운영 정합)

### 2-2. 게임 정보 (상단)

| 라벨 | 값 |
|---|---|
| 타이틀 | {게임 타이틀} |
| 총 참여자 | {N}명 |
| 총 상금 GP | {N} GP |

### 2-3. 결과제목 (필수)

- 다국어 탭: **KO / EN / JP**
- 텍스트박스: placeholder "결과 제목을 입력하세요."
- 글자수 카운터: 0/50

### 2-4. 결과 선택 (필수)

| 버튼 | 표기 | 설명 |
|---|---|---|
| YES | "YES 예상 정답자: {N}명" + 분배 미리보기 | YES 선택자 수 미리보기 + 그 아래 회색 보조 텍스트로 분배 미리보기 노출 (아래 분배 미리보기 규칙 참조) |
| NO | "NO 예상 정답자: {N}명" + 분배 미리보기 | NO 선택자 수 미리보기 + 분배 미리보기 노출 (NO 선택 시 정답자 = NO 선택자로 계산) |

**분배 미리보기 규칙** (YES/NO 선택 후 예상 정답자 수 아래에 회색 보조 텍스트로 노출):

- "응모권 {ticketReward}장 × 정답자 {N}명 = {N}장"
  - 게임 생성 시 입력한 응모권 보상(정답자 1인당)이 0이면 줄 미노출
- "덕력 {dukReward} DUK × 참여자 {M}명 = {N} DUK"
  - 게임 생성 시 입력한 덕력 보상(참여자 1인당)이 0이면 줄 미노출
  - 덕력은 정답·오답 무관 참여자 전원에게 균등 분배 ([CEB-BO-GZ-000] §5 참조)

> 실제 분배는 [결과 확정] → [202-MD-REWARD] [지급] 클릭 시점에 트리거된다. 본 미리보기는 운영자가 확정 전에 분배될 보상 총량을 확인하는 용도다.

### 2-5. 결과설명 (필수)

- 다국어 탭: KO / EN / JP
- 텍스트박스(multiline): placeholder "결과에 대한 설명을 입력하세요."
- 글자수 카운터: 0/500

### 2-6. 결과링크 (선택)

- 다국어 탭: KO / EN / JP
- 입력 2개: 링크 텍스트 (placeholder "링크 텍스트를 입력하세요.") + URL (placeholder "https://example.com/result-proof")
- 안내문: "결과를 확인할 수 있는 링크를 입력하세요. (선택)"

### 2-7. 푸터

| 버튼 | 스타일 | 상태 |
|---|---|---|
| [취소] | Secondary | 항상 활성. 모달 닫기 (입력값 폐기) |
| [결과 확정] | Primary | 필수값(결과제목 KO + 결과 선택 + 결과설명 KO) 충족 전 disabled |

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 모달 오픈 | 필수값 미입력 → [결과 확정] disabled |
| 결과 선택 (YES/NO) | 선택 시 강조 표시 + [결과 확정] 활성 조건 갱신 |
| 필수값 충족 | [결과 확정] 활성 |
| [결과 확정] 클릭 | API 호출 → Closed 전이 + [202] 결과/보상 탭에 결과 정보 표시 + 헤더 액션 [결과 수정] [보상 지급]으로 변경 |
| [취소] / ESC | 모달 닫기 (입력값 폐기) |
| 중복 클릭 | 첫 클릭 후 [결과 확정] disabled |
| API 실패 | 토스트 "결과 저장에 실패했습니다." + 입력값 보존 |
