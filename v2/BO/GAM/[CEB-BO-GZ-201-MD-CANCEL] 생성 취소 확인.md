## 1. 모달 개요

게임 생성 화면에서 [취소] 버튼 클릭 시, 작성 중인 데이터가 저장되지 않음을 경고하고 최종 확인을 받는 **확인 모달**. 실수로 인한 데이터 손실 방지가 목적.

| 항목 | 값 |
|---|---|
| 트리거 | [CEB-BO-GZ-201-CREATE]의 [취소] 버튼 클릭 |
| 차단 조건 | 없음 (항상 모달 표시) — 운영 BO 실측: 입력 유무와 무관하게 모달 노출 |
| 영향 범위 | [취소하기] 선택 시 입력 데이터 폐기 + 게임 리스트로 이동 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 모달 헤더

- 타이틀 (h3): **"생성 취소 확인"** (운영 BO 정합)

### 2-2. 본문

| 줄 | 내용 |
|---|---|
| 본문 1 | "게임 생성을 취소하시겠습니까?" |
| 본문 2 | "작성 중인 내용은 저장되지 않습니다." |

### 2-3. 모달 푸터 (하단 우측)

| 버튼 | 스타일 | 의미 |
|---|---|---|
| [계속 작성] | Secondary | 안전 — 모달 닫고 작성 화면 유지 |
| [취소하기] | Danger | 위험 — 입력 데이터 폐기 + 게임 리스트로 이동 |

### 2-4. 오버레이

- 배경: 반투명 Dim (opacity 0.5)
- 모달 외부 스크롤 비활성화

| 상황 | 동작 |

| 모달 오픈 | 배경 Dim + 모달 중앙 정렬 + 외부 스크롤 잠금 + [계속 작성]에 기본 포커스 |
