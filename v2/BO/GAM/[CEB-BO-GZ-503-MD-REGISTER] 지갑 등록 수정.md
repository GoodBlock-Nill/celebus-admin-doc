### 2-2. 모달 본문 (충전용 2필드 / 출금용 3필드 + 안내)

운영 + 테스트 BO 실측 결과 본문은 지갑 유형에 따라 동적 분기. 충전용은 2필드, 출금용은 3필드 + 본문 하단 안내 박스.

#### 2-2-1. 공통 필드 (충전용·출금용 모두)

| # | 라벨 | 컨트롤 | 운영 정합 |

| 2 | **지갑 주소 *** | Text Input + placeholder `0x...` + 하단 helper "블록체인 지갑 주소를 입력하세요." | 등록 모드 = 빈 값 / 수정 모드 = 기존 주소 전체 prefill (`0x` + 40자) |

본문 하단(푸터 직전)에 다음 안내 1줄 노출:

> 출금용 지갑은 가스비(BNB) 잔고가 있어야 출금이 가능합니다.

**[Private Key 필드 — 출금용 전용]**

- 충전용: PK 필드 미사용 — 모달에 부재. 충전 흐름은 사용자 → 지갑 입금이므로 운영자 키 불필요.
- 출금용: PK 필드 필수 — 운영자가 등록하는 출금 트랜잭션 서명 키. 서버에서 암호화하여 저장하며 BO 응답에는 마스킹된 prefix·suffix만 노출됨(리스트 컬럼: `PK: 94d2....97ef` 형태).
- v1.4·v1.5의 "PK 비노출" 단정은 충전용 모달 기본값 상태만 보고 내린 잘못된 결론 — 본 v1.6에서 운영 + 테스트 BO 모달 출금용 분기 재실측으로 정정.
- 운영자가 BO에서 직접 PK를 입력·교체 가능. 백엔드 단독 관리 단정은 폐기.

## 5. 검증 규칙

### 5-1. 입력 검증

| 필드 | 필수 | 검증 |

| 지갑 유형 | O | 충전용 / 출금용 2종 옵션 중 1종 (Dropdown 기본 선택으로 충족) |
