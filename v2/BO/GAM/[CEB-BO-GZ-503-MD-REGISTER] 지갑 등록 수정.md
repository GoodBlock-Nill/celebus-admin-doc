## 1. 화면 개요

운영용 블록체인 지갑(충전용/출금용)을 새로 등록하거나 기존 지갑을 수정하는 모달. 등록·수정 모드는 동일한 폼 레이아웃을 사용하며, 모달 제목과 CTA 버튼 라벨만 분기된다. Private Key 입력은 운영 BO 모달에 비노출 — 백엔드 키 관리 정책에 따라 별도 처리(운영 정책 추가 검증 시 보강).

| 화면 경로 | `/gamezone/exchange/wallets` (모달, 부모 화면 내 오픈) |
| Breadcrumb | 홈 › 게임존 › GP 교환소 › 지갑 관리 (모달) |
| 모달 제목 | "지갑 등록" (등록 모드) / "지갑 수정" (수정 모드) |

### 2-2. 모달 본문 (2필드)

운영 BO 실측 결과 본문은 2필드 단일 컬럼 구성. 명세 v1.3의 Private Key 필드는 운영 BO에 미노출 — 본 명세에서 제거.

| # | 라벨 | 컨트롤 | 운영 정합 |
|---|---|---|---|
| 1 | **지갑 유형** | **Dropdown** (충전용 / 출금용 2종 옵션) | 명세 v1.3의 Toggle Button → 운영 Dropdown 정정. 기본 선택: 등록 모드 = "충전용", 수정 모드 = 기존 지갑 유형 prefill |
| 2 | **지갑 주소 *** | Text Input + placeholder `0x...` + 하단 helper "블록체인 지갑 주소를 입력하세요." | 등록 모드 = 빈 값 / 수정 모드 = 기존 주소 전체 prefill (`0x` + 40자) |

**[Private Key 필드 제거 — 운영 정합]**

- 명세 v1.3은 Private Key 필드를 등록·수정 모달에서 필수 노출했으나, 운영 BO 실측 결과 PK 입력 UI 자체가 모달에 **비노출**
- 백엔드 키 관리 정책으로 별도 처리되는 것으로 추정 — 정확한 PK 입력·교체 흐름은 운영팀 정책 확인 시 보강

## 5. 검증 규칙

### 5-1. 입력 검증

| 필드 | 필수 | 검증 |
|---|---|---|
| 지갑 유형 | O | 충전용 / 출금용 2종 옵션 중 1종 (Dropdown 기본 선택으로 충족) |
| 지갑 주소 | O | `0x`로 시작하는 42자 hex 문자열. 형식 불일치 시 인라인 에러 "올바른 지갑 주소 형식이 아닙니다." |

### 5-2. 비즈니스 규칙

- 같은 유형 다중 등록 허용 (충전용 N개 가능)
- 동일 주소 중복 등록 불가 — API 응답 시 "이미 등록된 지갑입니다." 토스트
- 유형·주소 모두 수정 가능 (수정 모드)
- 모달 닫기 시 입력값 완전 폐기 (로컬 저장소 미사용)

### 5-3. 활동 로그

`[CEB-BO-GZ-000] 공통 정책 §활동 로그` 정합:

| 액션 | 로그 메시지 |
|---|---|
| 지갑 등록 | 지갑을 등록했습니다. (유형: {유형}, 주소: {주소}) |
| 지갑 수정 | 지갑을 수정했습니다. (유형: {유형}, 주소: {주소}) |

> Private Key는 운영 BO 모달에 비노출이므로 로그 메시지에 PK 등록 여부(Y/N) 표기 제거. PK 흐름이 별도 화면 또는 API로 명확히 정의되면 별도 명세로 보강.

### 5-4. 권한

- 권한은 부모 화면 [CEB-BO-GZ-503]과 동일 ([관리자 > 권한관리] 설정)
- 쓰기 권한 없으면 부모 화면에서 [+ 지갑 등록] / 행 [수정] 버튼 자체가 미노출
