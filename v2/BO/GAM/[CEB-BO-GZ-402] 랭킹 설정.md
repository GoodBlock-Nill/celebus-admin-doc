## 1. 화면 개요

게임존 랭킹 시스템의 **운영 설정** 화면. 관리자는 TOP 10 공개 여부, 집계 기준, 업데이트 주기, 최소 참여 횟수 4가지 설정을 변경한다. [CEB-BO-GZ-401] 랭킹 조회의 산출 정책과 앱 노출 범위가 본 화면 설정에 의해 결정된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/gamezone/ranking/settings` |
| Breadcrumb | 홈 › 게임존 › 랭킹 › 랭킹 설정 |
| 페이지 제목 | "랭킹 설정" |
| 진입 경로 | [CEB-BO-GZ-401] 랭킹 조회 헤더의 [랭킹 설정] 버튼 |
| 접근 권한 | [관리자 > 권한관리]에 설정된 권한을 따름 |
| 용도 | 랭킹 산출 정책 4종(공개 여부·기준·주기·최소 참여) 변경 |

## 2. 화면 구성

### 2-1. 헤더

| 위치 | 요소 | 동작 |
|---|---|---|
| 상단 | Breadcrumb (4단) | 홈 › 게임존 › 랭킹 › 랭킹 설정 |
| 좌 | 페이지 제목 (h1) | "랭킹 설정" |
| 우 | (액션 없음) | — |

> 401과 달리 본 화면 우측에는 별도 액션 버튼이 없다. 저장·초기화는 화면 하단 고정 영역에 배치.

### 2-2. TOP 10 공개

| 항목 | 값 |
|---|---|
| 섹션 헤더 | "TOP 10 공개" |
| 컨트롤 | Toggle Switch + 우측 라벨 (운영 표기 "OFF" / "ON") |
| 기본값 | OFF (운영 BO 실측 정합 — 명세 v1.0 "기본 ON" 정정) |
| 효과 | ON: 회원 앱 게임존에 TOP 10 섹션 노출 / OFF: 앱 섹션 숨김 (백오피스에서는 항상 조회 가능) |
| 반영 | 저장 즉시 회원 앱 캐시 갱신 |

### 2-3. 랭킹 기준

| 항목 | 값 |
|---|---|
| 섹션 헤더 | "랭킹 기준" |
| 컨트롤 | Radio Button 그룹 (단일 선택) |
| 기본 선택 | 누적 GP |

**옵션 3종**:

| 옵션 | 설명 |
|---|---|
| 누적 GP | (보상 + 환급) − (참여 + 부스팅). 내림차순. **기본값** |
| 승률 | (성공 게임 ÷ 참여 완료 게임) × 100. 내림차순. 동률 시 참여 횟수 많은 순 |
| 참여 횟수 | 결과 확정된 게임 참여 횟수. 내림차순. 동률 시 누적 GP 많은 순 |

> 산출 공식 상세는 [CEB-BO-GZ-401] §5-1~5-3 참조.

### 2-4. 업데이트 주기

| 항목 | 값 |
|---|---|
| 섹션 헤더 | "업데이트 주기" |
| 컨트롤 | Dropdown Select |
| 기본값 | 1시간마다 (권장) |

**옵션 3종 (운영 표기 정합)**:

| 옵션 | 설명 | 부하 |
|---|---|---|
| 실시간 | 게임 완료 시 즉시 재계산 | 높음 |
| **1시간마다** | 매시 정각 KST 배치 (권장) | 중간 |
| **1일마다** | 매일 00:00 KST 배치 | 낮음 |

> 운영 BO 라벨 정합 — 명세 v1.0 "1시간 / 1일" → "1시간마다 / 1일마다". 적용 시점: 저장 후 다음 갱신 사이클부터.

### 2-5. 최소 참여 횟수

| 항목 | 값 |
|---|---|
| 섹션 헤더 | "최소 참여 횟수" |
| 컨트롤 | Number Input + 우측 단위 "회" |
| 기본값 | 1 |
| 최솟값 | 1 |
| 최댓값 | 100 |
| 정수 검증 | 정수만 허용 |

**효과**: 결과 확정된 게임 참여 횟수가 본 값 미만인 사용자는 랭킹에서 제외된다. 우연한 1~2회 참여자를 거를 때 1~5회로 설정 권장.

### 2-6. 액션 버튼 (하단 고정)

| 버튼 | 위치 | 스타일 | 동작 |
|---|---|---|---|
| [초기화] | 우하단(저장 좌측) | Secondary (Outline) | 모든 설정을 기본값으로 복원 — 확인 모달 노출 후 적용 (저장은 별도) |
| [저장] | 우하단 | Primary (Solid) | 변경값 저장 + Toast "랭킹 설정이 저장되었습니다." + 활동 로그 |

> **[초기화] 확인 모달**: "모든 설정을 기본값으로 복원하시겠습니까? 변경사항은 저장되지 않습니다." [취소] [복원]. 복원 후 [저장]을 별도 클릭해야 실제 반영.

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton (각 섹션) |
| 데이터 로딩 실패 | 에러 토스트 "랭킹 설정을 불러오지 못했습니다" + [다시 시도] |
| 권한 없음 (읽기) | 사이드바 미노출 + URL 직접 접근 시 권한 오류 페이지 |
| 권한 없음 (쓰기) | 입력 컨트롤 disabled + 액션 버튼 disabled (조회만 가능) |
| 변경 후 [저장] 성공 | Toast "랭킹 설정이 저장되었습니다." + 활동 로그 기록 + 화면 유지 |
| 변경 후 [저장] 실패 | 에러 토스트 + 잘못된 필드에 인라인 에러 메시지 |
| [초기화] 클릭 | 확인 모달 → [복원] 시 화면 값만 기본값 복원 (저장 X) |
| 다른 관리자 동시 변경 | 마지막 저장이 우선 (별도 충돌 안내 없음) |

## 5. 검증 규칙

### 5-1. 입력 검증

| 필드 | 검증 |
|---|---|
| TOP 10 공개 | Toggle OFF/ON — 형식 오류 없음 |

### 5-2. 저장 시 처리

| 항목 | 처리 |
|---|---|
| TOP 10 공개 변경 | 회원 앱 캐시 즉시 갱신 |
| 랭킹 기준 변경 | 다음 갱신 사이클부터 적용 |
| 업데이트 주기 변경 | 저장 후 다음 갱신 시점부터 적용 |
| 최소 참여 횟수 변경 | 다음 갱신 사이클부터 적용 |

### 5-3. 활동 로그

`[CEB-BO-GZ-000] 공통 정책 §활동 로그` 정합. 본 화면에서 발생하는 로그:

| 액션 | 로그 메시지 |
|---|---|
| 랭킹 설정 저장 | 랭킹 설정을 변경했습니다. (TOP 10 공개: {ON/OFF}, 기준: {기준}, 주기: {주기}, 최소 참여: {N}회) |

> 로그는 변경 후 값만 기록한다. 변경 전 값과의 비교 표기는 운영 표준 로그 형식 외 (별도 진단 화면에서 처리).

### 5-4. 권한

- 권한은 [관리자 > 권한관리]에 설정된 그룹별 권한을 따름 ([CEB-BO-SYS-102])
- 읽기 권한: 본 화면 조회만 가능 (입력·저장 disabled)
- 쓰기 권한: 모든 설정 변경 및 저장 가능
