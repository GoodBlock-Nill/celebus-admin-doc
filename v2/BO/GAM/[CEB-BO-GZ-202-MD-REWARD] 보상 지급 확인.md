## 1. 모달 개요

PM 게임의 결과 확정(Closed) 후 정답자에게 **보상 GP를 분배 지급**하는 확인 모달. 보상 계산식(비율 배분)에 따라 정답자별 지급 GP가 산출되며, [지급]으로 즉시 회원 GP 계정에 환산 적립된다.

| 항목 | 값 |
|---|---|
| 트리거 | [CEB-BO-GZ-202] PM Closed 상태의 [보상 지급] 버튼 |
| 표시 조건 | 게임유형=PM + 상태=Closed + 보상 미지급 |
| 영향 범위 | 정답자별 보상 GP 적립 / 게임 상태 → Ended (보상 지급 완료 후 일정 시간 경과 시) |
| 접근 권한 | [관리자 > 권한관리] 설정 (쓰기 권한) |

## 2. 화면 구성

> 본 섹션은 v2.1 본문 보존. 운영 실측 정합 시 갱신 예정.

### 2-1. 모달 헤더

- 타이틀 (h3): "보상을 지급하시겠습니까?" (운영 실측 미확정)

### 2-2. 본문 (예상 구조)

- 안내: "보상 지급 후에는 되돌릴 수 없습니다."
- 게임 정보: 타이틀 / 총 상금 GP / 정답자 수 / 미지급 보상 GP / 지분당 보상 GP
- 응모권 보상 (신규):
  - "응모권 총 지급 N장 (1인당 N장 × 정답자 N명)"
  - 게임 생성 시 입력한 응모권 보상(정답자 1인당)이 0이면 줄 미노출
- 덕력 보상 (신규):
  - "덕력 총 지급 N DUK (1인당 N DUK × 참여자 N명)"
  - 게임 생성 시 입력한 덕력 보상(참여자 1인당)이 0이면 줄 미노출
  - 덕력은 정답·오답 무관 참여자 전원에게 균등 분배 ([CEB-BO-GZ-000] §5 참조)

### 2-3. 보상 분배 미리보기 (예상)

테이블 또는 요약: 정답자별 분배 GP (참여 GP × 가중치 × 부스팅 배수 기반).

응모권·덕력은 단일 총량으로 노출(개별 분배 X). [지급] 클릭 시 응모권은 RFT 영역(출처: PM·ST 보상)으로 자동 발급, 덕력은 DUK 영역(PM 참여 보상)으로 자동 적립된다.

### 2-4. 푸터

| 버튼 | 스타일 | 의미 |
|---|---|---|
| [취소] | Secondary | 모달 닫기 (보상 미지급 유지) |
| [지급] | Primary | 즉시 보상 GP 적립 + Closed → Ended 전이 |
