### 2-2. 교환 비율 (섹션 1)

| 항목 | 컨트롤 | 형식 |
|---|---|---|
| GP → CELB 비율 | "1 GP = [N] CELB" Number Input | 정수 또는 소수 1자리 |
| CELB → GP 비율 | "1 CELB = [N] GP" Number Input | 정수 또는 소수 1자리 |

- 두 비율은 독립 설정 (서로 역수 강제 아님 — 운영자가 의도적으로 비대칭 설정 가능)
- 운영 BO 실측 시점 값: 양쪽 모두 1

### 2-3. GP 충전 (CELB → GP) — 한도 (섹션 2)

| 항목 | 컨트롤 | 단위 | 운영 실측 시점 값 |
|---|---|---|---|
| 최소 금액 | Number Input | CELB | 5 |
| 최대 금액 | Number Input | CELB | 10000 |
| 일일 한도 (금액) | Number Input | CELB | 10000 |
| 일일 한도 (횟수) | Number Input | 회 | 20 |

> 운영 실측 값은 운영자가 수정 가능한 현재 설정값. 명세는 검증 룰만 보존(§5-1).

### 2-4. GP 출금 (GP → CELB) — 한도 (섹션 3)

| 항목 | 컨트롤 | 단위 | 운영 실측 시점 값 |
|---|---|---|---|

| 최대 금액 | Number Input | GP | 10000 |
| 일일 한도 (금액) | Number Input | GP | 10000 |
| 일일 한도 (횟수) | Number Input | 회 | 20 |

### 2-5. 교환 기능 (섹션 4)

| 항목 | 컨트롤 | 형식 |
|---|---|---|
| GP 충전 활성화 | Toggle Switch + 우측 라벨 "(ON)" / "(OFF)" | 회원 앱의 CELB → GP 교환 가능 여부 |
| GP 출금 활성화 | Toggle Switch + 우측 라벨 "(ON)" / "(OFF)" | 회원 앱의 GP → CELB 교환 가능 여부 |

- 비활성화(OFF) 시 회원 앱에서 해당 방향의 [교환] 시도 자체가 차단됨
- 진행 중 거래는 영향 없음 (이미 시작된 트랜잭션은 완료)

### 2-6. 액션 버튼 (하단 고정)

| 버튼 | 위치 | 스타일 | 동작 |
|---|---|---|---|

| [저장] | 우하단 | Primary (Solid) | 변경값 저장 + Toast "교환 설정이 저장되었습니다." + 활동 로그 |

## 5. 검증 규칙

### 5-1. 입력 검증

| 필드 | 검증 |
|---|---|
| GP → CELB 비율 / CELB → GP 비율 | 정수 또는 소수 1자리, 0보다 큰 양수 |
| 최소 금액 / 최대 금액 | 정수, 0보다 큰 양수, 최소 ≤ 최대 |
| 일일 한도 (금액) | 정수, 0보다 큰 양수. 운영 BO 입력 요소는 **`<input type="text">` + min/max/required 속성 미적용** — JS 단 검증으로 추정. **무제한 정책(0 = 무제한 여부)은 운영팀 핑 대기** (운영자 가변 값. 2026-05-26 실측: 충전 일일 10000, 횟수 100 / 출금 동일) |
| 일일 한도 (횟수) | 정수, 0보다 큰 양수 |
| 활성화 토글 | OFF / ON 2상태 |

### 5-2. 저장 시 처리

| 항목 | 처리 |
|---|---|
| 교환 비율 변경 | 저장 후 새 거래부터 적용 (소급 적용 없음 — [GZ-501] §5-2 정합) |
| 한도 변경 | 저장 후 새 거래부터 적용. 진행 중 거래는 영향 없음 |
| 활성화 토글 변경 | 회원 앱 캐시 즉시 갱신 |
| 동시 변경 | 마지막 저장이 우선 (별도 충돌 안내 없음) |
