### 2-2. 운영 지갑 현황

페이지 상단의 큰 카드 영역 — 섹션 제목 **"운영 지갑 현황"**.

| 카드 | 표시 정보 |
|---|---|
| **충전 지갑** | [대표] Badge + CELB 잔액(소수 8자리, 굵은 본문, 보라 톤) + BNB 잔고(소수 8자리, 보조 텍스트) + 지갑 주소(축약 0xXXXX...XXXX) + "상태: 활성/비활성" |
| **출금 지갑** | [대표] Badge + 동일 구성 (CELB·BNB·주소·상태) |

- 운영 BO에서 [대표] 지갑이 지정되지 않은 경우 첫 번째 활성 지갑의 잔액을 표시한다.
- BNB 잔고가 임계값(0.01 BNB) 미만이면 빨간색(red-600) + ⚠ 경고 아이콘 (가스비 부족 경고).
- 지갑 주소는 축약 표기. 호버 시 풀 주소 툴팁 (선택 — 운영 BO 동작 추가 검증 시 보강).

### 2-3. 통계 카드 (5종)

운영 지갑 현황 아래에 가로 5열 카드.

| 카드 | 라벨 (운영 정합) | 의미 |
|---|---|---|
| 1 | **오늘 GP 충전** | 오늘 00:00 ~ 현재까지 GP 충전(CELB→GP) 누계 GP |
| 2 | **오늘 GP 출금** | 오늘 GP 출금(GP→CELB) 누계 GP |
| 3 | **오늘 교환 건수(건)** | 충전+출금 합산 건수 (성공 기준) |
| 4 | **전체 유저 보유 GP** | 활성 유저가 보유한 GP 총합 (탈퇴 유저 제외) |
| 5 | **탈퇴 유저 GP** | 탈퇴 유저의 보유 GP 총합 (회수·소각 정책에 따라 별도 관리) |

- 숫자는 천 단위 콤마.
- 통계 카드는 기간 필터와 무관하게 항상 **오늘(KST)** 기준 (1·2·3 카드) / 현재 시점 기준 (4·5 카드)으로 표시한다.
- 실시간성 — 운영 BO 동작 기준. 필요 시 [CEB-BO-GZ-502] 교환 설정 통해 갱신 정책 변경.

## 5. 검증 규칙

### 5-1. 통계 카드 집계 (오늘 = KST 00:00 ~ 현재)

| 카드 | 집계 기준 |
|---|---|
| 오늘 GP 충전 | 오늘 성공한 "GP 충전" 거래의 GP 수량 합계 (실패 제외) |
| 오늘 GP 출금 | 오늘 성공한 "GP 출금" 거래의 GP 수량 합계 |
| 오늘 교환 건수(건) | 충전 + 출금 성공 거래 총 건수 |
| 전체 유저 보유 GP | 활성 유저 보유 GP 총합 (현재 시점) |
| 탈퇴 유저 GP | 탈퇴 유저 보유 GP 총합 (회수·소각 정책 별도) |

### 5-2. 교환 비율 표기

- 운영 표기 형태: "1 CELB = 1 GP" (CELB → GP 충전 시 의미) 또는 "1 GP = 1 CELB" (GP → CELB 출금 시 의미)
- 교환 시점의 비율이 기록되며, [CEB-BO-GZ-502] 교환 설정 변경 후에도 과거 거래는 당시 비율 유지 (소급 적용 없음)

### 5-3. 운영 지갑 정책

- 운영 지갑은 충전용·출금용 각 1개의 [대표] 지갑이 활성 상태로 동작
- [대표] 미지정 시 첫 번째 활성 지갑의 잔액 표시
- 출금 지갑 BNB 잔고 < 0.01 BNB 시 가스비 부족 경고 (텍스트 강조 + 아이콘)
- 지갑 등록·수정·삭제는 [CEB-BO-GZ-503] 지갑 관리에서 처리

### 5-4. 권한 및 활동 로그

- 권한은 [관리자 > 권한관리]에 설정된 그룹별 권한을 따름 ([CEB-BO-SYS-102])
- 활동 로그는 [CEB-BO-GZ-000] 공통 정책 §활동 로그 정합
- 조회 자체는 활동 로그 미기록 (조회 로그 정책)
- 거래 내역은 삭제·수정 불가 (이력 보존)
