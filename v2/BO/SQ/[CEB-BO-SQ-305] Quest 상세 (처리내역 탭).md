## 1. 화면 개요

특정 Quest의 검수 처리 이력(승인·반려)을 표 형식으로 노출하고 필터·검색을 제공하는 화면. Quest 상세 3탭 중 **처리내역 탭**.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/fanquest/{id}?tab=history` |
| Breadcrumb | 홈 / 에피소드 / 팬퀘스트 / Quest 상세 / 처리내역 |
| 진입 경로 | [CEB-BO-SQ-303] Quest 상세에서 "처리내역" 탭 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 자매 탭 | [CEB-BO-SQ-303] 기본정보 / [CEB-BO-SQ-304] 대기내역 |
| 용도 | 검수 처리 결과 조회, 통계 분석, CS 대응 |

## 2. 화면 구성

### 2-1. 헤더 (Quest 상세 공통)

[CEB-BO-SQ-303] §2-1·§2-2와 동일.

### 2-2. 필터·검색 바 (한 줄 가로 배치)

| 위치 | 요소 | 옵션 |
|---|---|---|
| 좌 | 상태 드롭다운 | "상태(전체)" (기본) / 승인 / 반려 |
| 우 | 검색 대상 드롭다운 | "닉네임" (기본) / "제출ID" |
| 우 | 키워드 입력 | 부분 일치 검색. placeholder는 검색 대상별 분기 |
| 우 | [초기화] (어두운 회색) | 모든 필터·검색 리셋 |

### 2-3. 처리내역 테이블 (8개 컬럼)

| 컬럼 | 너비 | 표시 내용 |
|---|---|---|
| 상태 | 80px | 배지 — 승인=에메랄드 / 반려=빨강 |
| 제출ID | 90px | 제출 식별값 (강조) |
| 유저닉네임 | 160px | 회원 닉네임 |
| 반려사유 | 120px | 반려인 경우 사유명, 승인은 "—" |
| 이미지 | 70px (중앙 정렬) | 확대 아이콘 버튼 (라이트박스용) |
| 제출일시 | 140px | 회원 제출 시각 |
| 처리일시 | 140px | 운영자 처리 시각 |
| 처리자 | 90px | 처리한 운영자 닉네임 |

### 2-4. 페이지네이션

- **페이지당 10건** (필터·검색 결과 기준)
- 필터·검색·키워드 변경 시 자동으로 1페이지로 리셋

### 2-5. 빈 상태

검색 결과 0건 시: "처리내역이 없습니다." 메시지 노출.
