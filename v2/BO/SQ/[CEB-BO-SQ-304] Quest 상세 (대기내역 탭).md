## 1. 화면 개요

특정 Quest의 검수 대기 제출을 카드 그리드로 노출하고 운영자가 행별 승인·반려 또는 일괄 승인을 수행하는 화면. Quest 상세 3탭 중 **대기내역 탭**.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/fanquest/{id}?tab=pending` |
| Breadcrumb | 홈 / 에피소드 / 팬퀘스트 / Quest 상세 / 대기내역 |
| 진입 경로 | [CEB-BO-SQ-303] Quest 상세에서 "대기내역" 탭 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 자매 탭 | [CEB-BO-SQ-303] 기본정보 / [CEB-BO-SQ-305] 처리내역 |
| 핵심 액션 | 행별 승인·반려, 페이지 일괄 승인 |

## 2. 화면 구성

### 2-1. 헤더 (Quest 상세 공통)

[CEB-BO-SQ-303] §2-1·§2-2와 동일. 탭 메뉴에서 "대기내역" 활성 시 검수 대기 카운트 ≥ 1이면 우측에 앰버 배지 표시.

### 2-2. 상단 액션 바

- 우측 정렬 [현재 페이지 모두 승인] (에메랄드 톤) — 클릭 시 현재 페이지의 모든 제출 일괄 승인

### 2-3. 제출 카드 그리드

CSS Grid `repeat(auto-fill, minmax(240px, 1fr))` 4px gap.

각 카드 구성:

| 영역 | 내용 |
|---|---|
| 이미지 영역 (상단) | 정사각형 비율 (aspect-square). 회원 제출 이미지. 우상단 [확대] 아이콘 버튼 (라이트박스용) |
| 정보 영역 (중간) | 닉네임 (큰 글씨 굵게) + 작은 글씨로 User ID·Submit ID·제출일시 |
| 액션 영역 (하단, 가로 2분할) | [반려] (빨강) · [승인] (에메랄드) |

### 2-4. 페이지네이션

- **페이지당 10건**
- 카드 그리드 결과 기준
- "현재 페이지 모두 승인"은 현재 페이지 카드만 영향

### 2-5. 빈 상태

검수 대기 0건 시: "검수할 제출이 없습니다." 중앙 정렬 안내 (py-16 회색 텍스트).

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 검수 대기 0건 | 빈 상태 메시지만 노출. 액션 바·페이지네이션 미노출 |
| 데이터 로딩 중 | Skeleton 카드 그리드 |
| 행별 [승인] 클릭 | 해당 제출 제거 + "승인 처리되었습니다." 토스트 (2.5초) |
| 행별 [반려] 클릭 | [CEB-BO-SQ-304-MD-REJECT] 반려 사유 선택 모달 노출 |
| [현재 페이지 모두 승인] 클릭 | 현재 페이지의 모든 카드 일괄 승인 + "{N}건 일괄 승인되었습니다." 토스트 |
| 권한 없음 진입 | [관리자 > 권한관리] 설정에 따라 화면 미노출 또는 액션 버튼 비활성 |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 일괄 승인 범위 | 현재 페이지에 노출된 카드만 대상. 다른 페이지는 영향 없음 |
| 즉시 처리 | 승인·반려 모두 즉시 반영. 카드 그리드에서 제거 + 다음 카드 자동 정렬 |
| 토스트 시간 | 처리 결과 토스트는 화면 상단 중앙, 2.5초 후 자동 닫힘 |
| 부모 Quest 상태 | 본 탭은 Quest 상태에 관계없이 노출되나, 실제 운영은 진행중·종료에서 의미 있음 |
