> 작성자: @Nill Yoo · 작성일: 2026-05-20 · 버전: **v3.1** (반복 주기 표시 옵션 4종으로 확장 — 일간/주간/월간/없음)
> 화면 경로: `/sq/{id}/quests/{qid}` · Breadcrumb: `홈 › 에피소드 › {그룹명} › {에피소드명} › 미션 N`

## 1. 화면 개요

특정 에피소드 내 미션의 정보·완료 판정·통계를 조회·관리하는 화면. 미션 유형별로 표시되는 정보가 분기된다. **v2.2부터 미션 수정 진입점이 본 화면으로 통합** — 기존 별도 모달은 폐기.

> 🚫 **v2.2 변경**: 완료 조건 별도 계층은 폐기. **미션 1개 = 완료 판정 1개**로 단순화. 완료 판정 필드(판정 유형·기준값·소스 참조값·반복 주기·시작·종료일)는 미션에 직접 흡수된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/{id}/quests/{qid}` |
| Breadcrumb | 홈 / 에피소드 / {그룹명} / {에피소드명} / 미션 N |
| 페이지 제목 | 미션의 한국어 타이틀 |
| 진입 경로 | [CEB-BO-SQ-202] 에피소드 상세의 미션 카드 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 → 에피소드 상세
- 페이지 제목: 미션 한국어 타이틀
- 상위 에피소드명 소제목
- 우측 영역:
  - 유형 배지 (팬퀘스트=분홍 / 예측 마켓=앰버 / 서바이벌 트리비아=파랑)
  - [수정하기] 버튼
  - [삭제하기] 버튼

### 2-2. 기본 정보 카드 (5행)

| 라벨 | 값 |
|---|---|
| 순서 | 1~10 |
| 유형 | 팬퀘스트 / 예측 마켓 / 서바이벌 트리비아 |
| 반복 여부 | 반복 시 "반복" 배지 노출 |
| 응모권 보상 | 지급 응모권 수량 |
| 덕력 보상 | 지급 덕력 수량 |

### 2-3. BIVE 보상 카드 (토글형)

| 상태 | 표시 |
|---|---|
| ON | 민팅 캠페인 명 + 캠페인 식별값 |
| OFF | "BIVE 보상 비활성" 안내 |

### 2-4. 완료 판정 카드 (v2.2 신규 — 흡수 필드)

미션 자체에 흡수된 완료 판정 정보를 좌·우 2열로 표시.

| 좌측 | 값 |
|---|---|
| 판정 유형 | 미션 유형별 분기 매트릭스 (라벨 · 색상) |
| 기준 수치 | "트리비아 N회 정답" 선택 시에만 노출 — "{기준값}회" |
| 조건 내용 | 예측 마켓·서바이벌 트리비아 — 소스 참조값 / 팬퀘스트 — 매핑된 팬퀘스트 식별값 |

| 우측 | 값 |
|---|---|
| 반복 주기 | **일간 / 주간 / 월간 / 없음** (4종). 단일 미션이거나 팬퀘스트 유형이면 팬퀘스트 자체 설정 상속 또는 "없음" |
| 시작 | 시작일자 (없으면 "그룹 기간 상속") |
| 종료 | 종료일자 (없으면 "그룹 기간 상속") |

> 푸터 안내: "v2.2부터 **완료 조건은 미션에 직접 흡수**됩니다. 미션 1개 = 완료 판정 1개."

#### 미션 유형별 판정 유형 매트릭스

| 미션 유형 | 가능한 판정 유형 | 기준값 사용 |
|---|---|---|
| 팬퀘스트 | "관리자 승인" 고정 | 없음 |
| 예측 마켓 | "예측 참여" / "예측 정답" 중 선택 | 없음 |
| 서바이벌 트리비아 | "트리비아 참여" / "트리비아 N회 정답" 중 선택 | "N회 정답" 선택 시 횟수 입력 |

### 2-5. 진행 통계 카드 (가로 3개, 파스텔 톤)

v2.2부터 "완료 조건 수" 카드는 폐기 (완료 조건 계층 폐기 정합).

| 카드 | 값 | 톤 |
|---|---|---|
| 진행 회원 | 현재 진행중인 회원 수 | 활성 (에메랄드) |
| 완료 회원 | 완료한 회원 수 | 활성 (에메랄드) |
| 완료율 | (완료 회원 / 진행 회원) × 100% | 커스텀 (에메랄드 막대 + 에메랄드 배지) |

### 2-6. 팬퀘스트 참조 패널 (유형이 팬퀘스트일 때만 노출)

- 연결된 팬퀘스트 Quest 정보 노출 (매핑된 식별값으로 조회)
- 안내: "본 미션의 타이틀·설명·이미지·보상은 [팬퀘스트 영역](/fanquest/{팬퀘스트 식별값})에서 관리됩니다."
- 클릭 시 팬퀘스트 상세로 이동

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 카드 |
| 부모 에피소드·그룹이 종료 상태 | 액션 버튼 비활성 또는 read-only |
| [수정하기] 클릭 | 본 화면을 수정 모드로 전환 (또는 별도 페이지 — 후속 정의). 미션 + 완료 판정 + 보상 통합 수정 |
| [삭제하기] 클릭 | 확인 모달 → 미션 삭제, 부모 에피소드의 미션 수 감소 |
| 권한 없음 진입 | [관리자 > 권한관리] 설정에 따라 화면 미노출 또는 액션 버튼 비활성 |

## 5. 검증 규칙

|---|---|
| 팬퀘스트 유형 미션의 보상·이미지·타이틀 수정 | 본 화면에서 수정 불가 → 팬퀘스트 영역에서만 가능. 본 화면에서는 매핑(팬퀘스트 식별값)만 관리 |

| 시작·종료일 | 비우면 부모 그룹 기간 상속. 입력 시 부모 그룹 기간 안에 포함 권장 |
