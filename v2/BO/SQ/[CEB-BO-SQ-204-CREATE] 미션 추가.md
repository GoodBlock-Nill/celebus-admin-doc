## 1. 화면 개요

에피소드 안에 새 미션을 추가하는 폼 화면. 미션 유형(팬퀘스트 / 예측 마켓 / 서바이벌 트리비아)별로 입력 항목이 분기된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/{id}/quests/create` |
| Breadcrumb | 홈 / 에피소드 / {그룹명} / {에피소드명} / 새 미션 추가 |
| 페이지 제목 | "새 미션 추가" + 현재 미션 수 표기 "(N/10)" |
| 보조 정보 | 아티스트 배지 + 상위 에피소드명 |
| 진입 경로 | [CEB-BO-SQ-202] 에피소드 상세의 [미션 추가] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 정책 상수 | 에피소드당 미션 합계 상한 10개 ([CEB-BO-011] §5 정합) |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 (취소 확인 → 에피소드 상세)
- 페이지 제목 "새 미션 추가" + "(N/10)" 현재 미션 수
- 아티스트 배지 + 상위 에피소드 타이틀

### 2-2. 만석 차단 박스 (조건부)

미션 합계 10개 도달 시: 로즈색 경고 박스
> "본 에피소드는 이미 미션 10개 최대치에 도달했습니다."
+ [미션 추가] 버튼 영구 비활성

### 2-3. 부모 에피소드 유형 안내 (조건부)

부모 에피소드 유형에 따라 안내 박스가 분기된다.

| 부모 에피소드 유형 | 박스 톤 | 안내 메시지 |
|---|---|---|
| **메인** | 인디고 | "메인 에피소드는 **단일 미션만** 추가 가능합니다. 반복 미션이 필요하면 반복 에피소드에서 추가하세요. 본 화면에서는 반복 여부 토글이 표시되지 않습니다." |
| **반복** | 바이올렛 | "반복 에피소드 미션 정책: ① 팬퀘스트 유형은 **반복 종류의 팬퀘스트만** 매핑 가능. ② 예측 마켓·서바이벌 트리비아는 단일·반복 모두 가능. 반복 ON 시 **반복 주기·기간 설정 필수**입니다." |

> 정합 근거: [CEB-BO-011] §5 정책 상수 "메인 에피소드 미션 유형 / 반복 에피소드 미션 유형 / 반복 미션 반복 주기 / 반복 미션 운영 기간" 4행 참조.

### 2-4. 정책 안내 (인디고 박스)

> **미션 유형별 정책**:
> - **팬퀘스트**: 팬퀘스트 영역에서 관리되는 Quest를 참조만 합니다. 보상·이미지·타이틀은 자동 상속.
> - **예측 마켓**: 예측 마켓 콘텐츠 참조 없음. 판정 유형·보상을 본 화면에서 직접 입력.
> - **서바이벌 트리비아**: 서바이벌 트리비아 콘텐츠 참조 없음. 판정 유형·보상을 본 화면에서 직접 입력.

### 2-5. 기본 정보

| 필드 | 입력 방식 | 옵션 |
|---|---|---|
| 표시 순서 | read-only | 다음 빈 슬롯 (자동) |
| 유형 | 드롭다운 | 팬퀘스트 / 예측 마켓 / 서바이벌 트리비아 (필수 ★) |

### 2-6. 유형이 "팬퀘스트"일 때

| 필드 | 동작 |
|---|---|
| 팬퀘스트 참조 | 드롭다운 — "[{상태}] {타이틀} (#{식별값})" 형식 |
| 옵션 소스 | 부모 에피소드가 반복이면 반복 종류 팬퀘스트만, 아니면 일반 종류만 |
| 중복 방지 | 이미 다른 미션에 매핑된 팬퀘스트는 드롭다운에서 자동 제외 (1:1 정책) |
| 옵션 없음 | 앰버색 안내 — "먼저 [팬퀘스트 영역](/fanquest)에서 Quest를 생성하세요." |
| 선택 후 | 핑크색 "참조 완료" 안내 — "모든 설정(타이틀·이미지·보상)은 팬퀘스트 영역에서 관리됩니다." |
| 반복 주기·기간 | **본 화면에서 별도 설정 불가**. 팬퀘스트 자체 설정(반복 주기·기간)을 그대로 상속함. 매핑된 팬퀘스트가 반복 종류일 경우 팬퀘스트 영역에서 정의된 반복 주기·기간이 적용됨 |

### 2-7. 유형이 "예측 마켓" 또는 "서바이벌 트리비아"일 때

**판정 유형 섹션** (v2.2부터 미션 자체 필드에 직접 저장):
- 판정 유형 드롭다운:
  - 예측 마켓: "예측 참여" / "예측 정답" 중 선택
  - 서바이벌 트리비아: "트리비아 참여" / "트리비아 N회 정답" 중 선택
- 기준 횟수: "트리비아 N회 정답" 선택 시에만 활성 (정수 입력)
- 소스 참조값 (선택, 예: "예측 마켓 #34 — 다음 컴백 시점")

> v2.2부터 별도 완료 조건 모달 없이 미션 생성 시 1:1로 완료 판정 필드가 같이 저장된다.

**보상 섹션**:
- 응모권: 정수 입력 (단위: 장)
- 팬덤 포인트: 정수 입력

**반복 여부 섹션** (부모 에피소드 유형에 따라 분기):

| 부모 에피소드 유형 | UI 노출 | 동작 |
|---|---|---|
| **메인** | 반복 여부 토글 **자체 미노출** | 자동 단일 미션으로 저장. 반복 주기·기간 설정 영역 없음 |
| **반복** | 반복 여부 토글 노출 (기본 OFF) | OFF면 단일 미션. ON 시 아래 "반복 주기·기간 설정" 영역 추가 노출 |

**반복 주기·기간 설정** (부모 = 반복 에피소드 + 반복 토글 ON일 때만 노출):

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 반복 주기 | 버튼 그룹 3종 — **일간 / 주간 / 월간** (팬퀘스트 생성과 동일 UI) | 1개 선택 | ★ |
| 시작일자 | 날짜 + 시간 입력 | 비우면 그룹 시작일 자동 상속 | — |
| 종료일자 | 날짜 + 시간 입력 | 비우면 그룹 종료일 자동 상속. **그룹 기간 안에서만 허용** (벗어나면 인라인 에러) | — |

> 정합 근거: [CEB-BO-011] §5 정책 상수 "반복 미션 반복 주기 / 반복 미션 운영 기간" 2행. 팬퀘스트 생성 화면의 반복 주기·기간 입력 UI와 동일 패턴.

**BIVE 보상 섹션** (토글):
- OFF: 비활성 안내
- ON: 민팅 캠페인 드롭다운 (등록된 캠페인 6종 — V01D Welcome ED / Trivia Master / Final Boss / Prophet / 100 Days 등)

| 판정 영역 | 판정 유형·기준값·소스 참조값 (예측 마켓·서바이벌 트리비아일 때) | 미션 흡수 필드 |

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 폼 |
| 만석 도달 진입 | 만석 차단 박스 노출 + 모든 입력 비활성 |
| 반복 에피소드 + 팬퀘스트 유형 선택 | 반복 종류 팬퀘스트만 드롭다운에 노출 |
| 메인 에피소드 + 팬퀘스트 유형 | 일반 종류 팬퀘스트만 드롭다운에 노출 |
| 메인 에피소드 + PM/ST 선택 | 반복 여부 토글 자체 미노출. 단일 미션으로 자동 저장 |
| 반복 에피소드 + PM/ST 선택 | 반복 여부 토글 노출 (기본 OFF). ON 시 반복 주기·시작·종료일자 추가 노출 |
| 반복 주기·기간 입력 + 그룹 기간 초과 | 종료일자 입력란에 인라인 에러 "그룹 기간 안에서만 허용" + [미션 추가] 비활성 |
| 유형 변경 | 하단 입력 필드 동적 전환 |
| 추가 성공 | 에피소드 상세로 복귀 + 토스트, 부모 에피소드의 미션 수 +1 |
| 권한 없음 진입 | 화면 미노출 또는 [미션 추가] 비활성 |

## 5. 검증 규칙

| 검증 | 동작 |
|---|---|
| 만석 (미션 10개) | 모든 입력 비활성 |
| 유형 미선택 | [미션 추가] 비활성 |
| 팬퀘스트 유형 + 팬퀘스트 미선택 | [미션 추가] 비활성 |
| 팬퀘스트 유형 + 이미 사용 중인 팬퀘스트 | 드롭다운에 노출되지 않음 |
| 예측 마켓·서바이벌 트리비아 + 판정 유형/기준값 미입력 | [미션 추가] 비활성 |
| BIVE ON + 민팅 캠페인 미선택 | [미션 추가] 비활성 |
| 메인 에피소드 + PM/ST 미션 | 반복 여부 자동 OFF·잠금. 단일 미션으로만 저장 가능 |
| 반복 에피소드 + PM/ST + 반복 ON | 반복 주기(일간/주간/월간) 1개 선택 필수 |
| 반복 ON + 시작일자 > 종료일자 | 종료일자 인라인 에러 + [미션 추가] 비활성 |
| 반복 ON + 시작·종료일자가 그룹 기간 밖 | "그룹 기간 안에서만 허용" 인라인 에러 + [미션 추가] 비활성 |
| 팬퀘스트 유형 미션 | 반복 주기·기간 본 화면 검증 없음 (팬퀘스트 자체 설정 상속) |
