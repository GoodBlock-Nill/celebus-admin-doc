## 1. 화면 개요

특정 에피소드의 메인 이미지·다국어 정보·통계·하위 미션 그리드를 통합 노출하는 운영자 화면.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/{id}` |
| Breadcrumb | 홈 / 에피소드 / {그룹명} / {에피소드명} |
| 페이지 제목 | 에피소드의 한국어 타이틀 |
| 진입 경로 | [CEB-BO-SQ-201] 그룹 상세의 에피소드 카드 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 → 그룹 상세 복귀
- 페이지 제목: 에피소드 한국어 타이틀
- **유형 배지 (v2.2)**: 메인 / 반복 — 에피소드 종류 노출 (상태 배지는 미노출, 그룹 상태 상속)
- 우측 액션 (상태별):
  - 임시저장: [수정] · [삭제] · [게시]
  - 진행중: [수정] · [종료]
  - 종료: (없음 — read-only)

### 2-2. 3열 그리드 — 핵심 정보

| 좌 (메인 이미지) | 중 (기본 정보) | 우 (관리 정보) |
|---|---|---|
| 3:4 비율 카드, 등록된 메인 이미지 | 표시 순서 · 유형(메인/반복) · 미션 수 · 그룹 기간(상속) | 생성 관리자 · 생성 일시 · 최근 수정자 · 최근 수정 일시 |

### 2-3. 다국어 콘텐츠 블록

다음 두 항목을 한국어/영어/일본어 3행으로 노출 (미입력 언어는 회색 "(미입력)"):

- 에피소드 타이틀
- 에피소드 설명

### 2-4. 통계 카드 (가로 4개, 파스텔 톤)

| 카드 | 의미 | 톤 |
|---|---|---|
| 총 미션 | 본 에피소드 미션 수 (최대 10) | 기본 (회색) |
| 진행 회원 | 현재 참여 중인 회원 누적 | 활성 (에메랄드) |
| 미션 완료 | 미션 완료 누적 회원 수 | 활성 (에메랄드) |
| 검수 대기 | 검수 대기 미션 수 | 대기 (앰버, 0이면 비활성 회색) |

### 2-5. 미션 그리드

- 섹션 헤더: "미션 (N/10)" + 우측 [미션 추가] 버튼
  - 10개 도달 시 [미션 추가] 비활성 + 안내 툴팁
- 카드 그리드 — 표시 순서 오름차순 정렬
- 각 카드 (v2.2부터 "완료 조건 수" 행 제거):
  - 미션 썸네일
  - 유형 배지 (팬퀘스트 / 예측 마켓 / 서바이벌 트리비아)
  - 미션 한국어 타이틀
  - 보상 행: 응모권 · 팬덤 포인트 · BIVE 보상 ON/OFF
  - 팬퀘스트 유형일 때 매핑된 팬퀘스트 타이틀 표시
  - **우상단 ↑↓ 화살표 버튼** — 순서 변경, 임시저장 상태에서만 활성
- 카드 본문 클릭 → 미션 상세 화면([CEB-BO-SQ-204], `/sq/{id}/quests/{qid}`) 이동

### 2-6. 순서 변경 동작

| 동작 | 결과 |
|---|---|
| ↑ 클릭 | 현재 미션 순서 −1, 위 미션 순서 +1 (스왑) |
| ↓ 클릭 | 반대 방향 스왑 |
| 첫 카드의 ↑ / 마지막 카드의 ↓ | 비활성 |
| 부모 에피소드 상태 ≠ 임시저장 | 모든 ↑↓ 비활성 + 툴팁 "임시저장 상태에서만 순서 변경 가능" |

## 4. 상태별 동작 분기

| 에피소드 상태 | 헤더 액션 | 미션 영역 |
|---|---|---|
| 임시저장 | [수정] · [삭제] · [게시] | 추가·수정·순서 변경 가능 |
| 진행중 | [수정] · [종료] | 추가 가능. 순서 변경은 정책 검토 (일반적으로 제한) |
| 종료 | (없음) | read-only |

### 4-1. 게시 (임시저장 → 진행중) 흐름

1. 부모 그룹이 진행중 상태인지 검증 (임시저장 그룹에서는 에피소드 게시 불가)
2. 다국어 한국어/영어/일본어 3개 언어 입력 완료 검증
3. 메인 이미지 등록 검증
4. 확인 모달 → 상태 = 진행중

### 4-2. 종료 (진행중 → 종료) 흐름

1. 확인 모달 "에피소드를 종료합니다. 모든 하위 미션도 종료됩니다."
2. 확인 → 상태 = 종료

### 4-3. 삭제 (임시저장만)

1. 확인 모달 "에피소드를 삭제합니다. 복구할 수 없습니다."
2. 확인 → 그룹 상세로 복귀, 토스트

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 미션 한도 | 10개 초과 시 [미션 추가] 비활성 |
