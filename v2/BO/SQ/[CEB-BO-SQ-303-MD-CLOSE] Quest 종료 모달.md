## 1. 모달 개요

진행중 Quest를 종료 상태로 조기 전환하기 위한 확인 모달. 위험 액션이므로 빨강 톤.

| 항목 | 값 |
|---|---|
| 트리거 | Quest 상세의 [종료하기] (진행중 상태에서만 노출) |
| 진입 조건 | Quest 상태가 "진행중" |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 영향 범위 | Quest 상태 진행중 → 종료. 회원 앱 노출 중단. 새 제출 차단 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "Quest 종료"

### 2-2. 본문

> 확인 버튼을 누르면 Quest가 조기 종료됩니다.
> 계속 진행하시겠습니까?

### 2-3. 에피소드 사용 현황 패널 (조건부 추가 노출)

본 Quest를 사용 중인 에피소드 미션이 있으면 본문 아래 "사용 중인 에피소드" 패널 추가 노출. 모달 사이즈는 md로 확대.

### 2-4. 하단 액션

| 버튼 | 색상 | 동작 |
|---|---|---|
| 취소 | 회색 외곽선 | 모달 닫기 |
| 확인 | **빨강 (danger)** | Quest 상태 진행중 → 종료 전환 + "Quest가 종료되었습니다." 토스트 |

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 진행중 Quest 진입 | 모달 정상 노출 |
| 임시저장·종료 Quest | 트리거 액션 자체 미노출 |
| 사용 중인 에피소드 있음 | 모달 사이즈 md + 에피소드 패널 추가 |
| 사용 중인 에피소드 없음 | 모달 사이즈 sm (기본) |
| [확인] 클릭 | 상태 전환 + 토스트 + 모달 닫힘 |
| [취소] / 외부 클릭 | 모달 닫힘 (상태 변경 없음) |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| Quest 상태 ≠ 진행중 | 모달 진입 불가 |
| 종료 후 효과 | Quest는 read-only, 새 제출 차단, 회원 앱에서 비활성 |
| 에피소드 사용 영향 | 종료된 Quest와 매핑된 미션은 그대로 유지 (회원 진행은 차단). 매핑 해제는 별도 작업 |
