## 1. 화면 개요

신규 에피소드 그룹을 생성하는 폼 화면. 아티스트·상태·타이틀·기간만 입력하며, 메인 이미지·다국어·설명 등은 그룹 레벨에 없음 (에피소드 레벨에서만 입력).

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/groups/create` |
| Breadcrumb | 홈 / 에피소드 / 그룹 리스트 / 새 그룹 생성 |
| 페이지 제목 | "새 에피소드 그룹 생성" |
| 진입 경로 | [CEB-BO-SQ-101] 그룹 리스트의 [새 그룹 생성] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 버튼 → 그룹 리스트로 복귀
- 페이지 제목 "새 에피소드 그룹 생성"

### 2-2. 안내 박스

회색 톤 안내: "그룹은 아티스트별 시즌·큐레이션 묶음입니다. 진행중 상태는 아티스트당 1개만 허용됩니다."

### 2-3. 경고 박스 (조건부 노출)

상태를 **진행중**으로 선택했을 때, 같은 아티스트에 이미 진행중인 그룹이 존재하면:
- 빨강 톤 박스: "현재 {아티스트}에 이미 진행중인 그룹이 있습니다. 진행중 상태로 저장할 수 없습니다."

### 2-4. 입력 폼 — 기본 정보

| 필드 | 입력 방식 | 옵션·검증 | 필수 |
|---|---|---|---|
| 아티스트 | 드롭다운 | V01D / iKON / CELEBUS | ★ |
| 상태 | 드롭다운 | 임시저장 / 진행중 / 종료 (기본 "임시저장") | ★ |
| 타이틀 | 텍스트 | 한국어, 최대 80자 | ★ |

### 2-5. 입력 폼 — 기간

| 필드 | 입력 방식 | 검증 | 필수 |
|---|---|---|---|
| 시작일자 | 날짜 + 시간 선택 | 종료일자보다 이전 | ★ |
| 종료일자 | 날짜 + 시간 선택 | 시작일자보다 이후 | ★ |

### 2-6. 하단 액션 (Sticky 푸터)

- [취소] → 그룹 리스트로 복귀 (입력값 있을 시 확인 다이얼로그)
- [저장] — 필수 항목 모두 입력 + 진행중 중복 검증 통과 시 활성

## 3. 화면에 노출되는 정보 / 입력되는 정보

본 화면은 신규 데이터 입력 화면이므로 운영자가 입력하는 정보 중심.

| 영역 | 입력 정보 | 의미 |
|---|---|---|
| 기본 정보 | 아티스트·상태·타이틀 | 그룹의 식별·상태·표시 정보 |
| 기간 | 시작일자·종료일자 | 그룹 기간 (하위 에피소드는 본 기간 상속) |
| 자동 기록 | 식별값·에피소드 수·생성자·생성일시·최근 수정자·최근 수정일시 | 저장 시 시스템 자동 부여 |

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 폼 |
| 신규 진입 | 빈 폼 (상태 "임시저장" 기본 선택) |
| 진행중 중복 발생 | 경고 박스 노출 + [저장] 비활성 |
| 시작일자 ≥ 종료일자 | 인라인 에러 + [저장] 비활성 |
| 저장 성공 | 그룹 상세 페이지([CEB-BO-SQ-201])로 이동 + "그룹이 생성되었습니다." 토스트 |
| 권한 없음 진입 | [관리자 > 권한관리] 설정에 따라 화면 미노출 또는 [저장] 비활성 |

## 5. 검증 규칙

| 케이스 | 동작 |
|---|---|
| 필수 미입력 | [저장] 버튼 비활성 |
| 시작일자 ≥ 종료일자 | 인라인 에러 + [저장] 비활성 |
| 진행중 중복 (같은 아티스트의 다른 그룹이 이미 진행중) | 경고 박스 노출 + [저장] 차단 |
| 타이틀 80자 초과 | 글자수 표시 + 초과 입력 차단 |

### 5-1. 동적 검증 동작

- 아티스트 변경 시: 진행중 중복 검증 재실행
- 상태 변경 시: "진행중" 선택하면 경고 박스 즉시 토글
- 취소: 입력값 있으면 확인 다이얼로그 → 그룹 리스트로 복귀
