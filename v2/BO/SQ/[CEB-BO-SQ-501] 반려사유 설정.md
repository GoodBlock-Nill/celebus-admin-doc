## 1. 화면 개요

검수 반려 시 사용되는 반려사유 목록을 관리하는 정책 설정 화면. 운영자가 사유를 신규 추가·수정하며, 활성/미사용 상태 관리. **[CEB-BO-SQ-304-MD-REJECT] 제출 반려 모달에서 본 화면의 활성 사유만 노출됨**.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/fanquest/reject-reasons` (목표 `/sq/reject-reasons`) |
| Breadcrumb | 홈 / 에피소드 / 반려사유 설정 |
| 페이지 제목 | "반려사유 설정" |
| 진입 경로 | [CEB-BO-SQ-302] 팬퀘스트 리스트의 [반려사유 설정] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 페이지 제목 "반려사유 설정"
- Breadcrumb (홈 → 에피소드 → 반려사유 설정)

### 2-2. 필터·액션 바

| 위치 | 요소 | 옵션·동작 |
|---|---|---|
| 좌 | 상태 드롭다운 | "상태(전체)" (기본) / 사용 (Active) / 미사용 (Inactive) |
| 우 | [+ 새 사유 추가] (인디고) | 클릭 시 [CEB-BO-SQ-501-MD-ADD] 반려사유 추가 모달 노출 |

### 2-3. 반려사유 목록 테이블 (2개 컬럼)

| 컬럼 | 너비 | 표시 내용 |
|---|---|---|
| 상태 | 120px | 배지 — 사용=에메랄드 / 미사용=회색 |
| 운영자 노출명 | flex | 사유 이름 (예: "이미지 품질 미달") |

각 행을 클릭하면 [CEB-BO-SQ-501-MD-EDIT] 반려사유 수정 모달 노출.

### 2-4. 페이지네이션

- **페이지당 10건** (필터 결과 기준)
- 상태 필터 변경 시 자동으로 1페이지로 리셋

### 2-5. 빈 상태

검색 결과 0건 시: "검색 결과가 없습니다." 메시지 노출.
