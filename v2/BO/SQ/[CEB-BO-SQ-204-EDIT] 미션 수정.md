## 1. 화면 개요

기존 미션의 정보를 수정하는 폼 화면. [CEB-BO-SQ-204] 미션 상세의 [수정하기] 버튼을 통해 진입한다. 기존 데이터가 모든 필드에 prefill 되며, **부모 에피소드 상태에 따라 일부 필드가 read-only로 잠긴다** (회원 기대권 보호).

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/{id}/quests/{qid}/edit` |
| Breadcrumb | 홈 / 에피소드 / {그룹명} / {에피소드명} / 미션 N / 수정 |
| 페이지 제목 | "미션 수정" + 미션 한국어 타이틀 |
| 보조 정보 | 아티스트 배지 + 상위 에피소드명 |
| 진입 경로 | [CEB-BO-SQ-204] 미션 상세의 [수정하기] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 (수정 취소 확인 → 미션 상세)
- 페이지 제목 "미션 수정" + 현재 미션 한국어 타이틀
- 아티스트 배지 + 상위 에피소드 타이틀
- 우측: 유형 배지 (팬퀘스트=분홍 / 예측 마켓=앰버 / 서바이벌 트리비아=파랑)

### 2-2. 상태 배너 (조건부)

부모 에피소드 상태 + 미션 유형에 따라 상단에 안내 배너가 노출된다.

| 상태 | 유형 | 배너 톤 | 안내 메시지 |
|---|---|---|---|
| **임시저장 (DRAFT)** | 전 유형 | 미노출 | — |
| **진행중 (ACTIVE)** | **PM/ST** | 노랑 | "**진행중 에피소드의 미션**입니다. 회원 기대권 보호를 위해 **다국어 타이틀(KO·EN·JA)만 수정 가능**합니다. 판정 유형·보상·반복 설정은 모두 read-only입니다." |
| **진행중 (ACTIVE)** | **팬퀘스트** | 회색 + 자물쇠 | "**팬퀘스트 유형 미션은 진행중 상태에서 수정할 수 없습니다.** 본 화면은 직접 URL 진입을 통한 방어 표시이며 모든 필드가 read-only입니다." (정상 흐름에서는 미션 상세의 [수정하기] 버튼이 비활성화되어 본 화면에 진입할 수 없음) |
| **종료 (CLOSED)** | 전 유형 | 회색 + 자물쇠 | "**종료된 에피소드의 미션은 수정할 수 없습니다.**" + 모든 필드 read-only + [저장] 비활성 (정상 흐름에서는 [수정하기] 버튼이 비활성화되어 본 화면에 진입할 수 없음) |

> 정합 근거: v1.0의 "진행중 보상만 read-only" 정책을 v1.1에서 사용자 정정 — **진행중+PM/ST는 다국어 타이틀만 수정 / 진행중+팬퀘스트와 종료는 [수정하기] 버튼 자체 비활성** (UI에서 진입 차단). 본 화면은 정상 흐름 진입 차단 케이스에서 직접 URL 진입에 대한 방어 표시만 담당.

### 2-3. 팬퀘스트 메타 안내 박스 (유형이 팬퀘스트일 때만 노출)

미션 유형이 팬퀘스트면 상단에 인디고 박스로 안내한다.

> **이 미션은 팬퀘스트 자체 데이터를 상속받습니다.**
> - 타이틀·다국어 타이틀·이미지·보상은 [팬퀘스트 영역](/fanquest/{팬퀘스트 식별값})에서만 수정 가능합니다.
> - 본 화면에서는 **반복 여부·반복 주기·기간·BIVE 토글·민팅 캠페인** 등 미션 고유 메타만 수정할 수 있습니다.
> - 팬퀘스트 매핑(fanQuestId) 자체는 본 화면에서 변경할 수 없습니다 (재매핑은 미션 삭제 후 재생성).

### 2-6. 유형이 "예측 마켓" 또는 "서바이벌 트리비아"일 때

v1.1 정정 — 상태별 필드 잠금 매트릭스 (✏️ = 수정 가능 / 🔒 = read-only):

| 필드 | DRAFT | **ACTIVE (정정)** | CLOSED |
|---|---|---|---|
| 다국어 타이틀 (KO/EN/JA) | ✏️ | **✏️ (유일 수정 가능)** | 🔒 |
| 판정 유형 | ✏️ | **🔒 (정정)** | 🔒 |
| 기준 횟수 | ✏️ | **🔒 (정정)** | 🔒 |
| 소스 참조값 | ✏️ | **🔒 (정정)** | 🔒 |
| 응모권 (장) | ✏️ | 🔒 (v1.0 유지) | 🔒 |
| 덕력 | ✏️ | 🔒 (v1.0 유지) | 🔒 |
| BIVE 토글 | ✏️ | 🔒 (v1.0 유지) | 🔒 |
| 민팅 캠페인 | ✏️ | 🔒 (v1.0 유지) | 🔒 |
| 반복 여부 토글 | ✏️ | **🔒 (정정)** | 🔒 |
| 반복 주기 | ✏️ | **🔒 (정정)** | 🔒 |
| 시작·종료일자 | ✏️ | **🔒 (정정)** | 🔒 |

> Prefill: 기존 `titleKO`·`titleEN`·`titleJA`·판정·보상·반복 모두 prefill 후 노출. 진행중 상태에서는 운영자가 누락된 언어 보강·오타 정정만 가능.

> **v1.1 정정 근거**: v1.0의 "진행중 = 보상만 read-only" 정책을 사용자 정정으로 **다국어 타이틀만 유일하게 수정 가능**으로 강화. 회원이 이미 기대권을 가진 판정·보상·반복 메타도 진행중 상태에서는 변경 불가. 다국어 타이틀만 예외 — 잘못 입력된 번역 보강·오타 정정 등 회원 기대권에 영향이 적은 텍스트 정정만 허용.

## 4. 상태별 동작 분기

v1.1 정정 — 부모 에피소드 상태 + 미션 유형별 진입·수정 가능 매트릭스:

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 폼 |
| 부모 에피소드 미존재 / 미션 미존재 | "미션을 찾을 수 없습니다." 안내 + [뒤로가기] |
| **DRAFT + 팬퀘스트** | EDIT 진입 가능. 타이틀·이미지·보상 read-only (자체 상속) + 반복·BIVE 메타만 수정 |
| **DRAFT + PM/ST** | EDIT 진입 가능. 전체 수정 가능 (CREATE와 동일 UX) |
| **ACTIVE + 팬퀘스트** | **미션 상세에서 [수정하기] 비활성** → EDIT 진입 차단. 직접 URL 진입 시 회색 차단 배너 + 전체 read-only + [저장] 비활성 |
| **ACTIVE + PM/ST** | EDIT 진입 가능. **다국어 타이틀(KO·EN·JA)만 수정** + 노랑 배너 노출. 판정·보상·반복은 모두 read-only |
| **CLOSED + 전 유형** | **미션 상세에서 [수정하기] 비활성** → EDIT 진입 차단. 직접 URL 진입 시 회색 차단 배너 + 전체 read-only + [저장] 비활성 |
| 반복 주기·기간 입력 + 그룹 기간 초과 | 종료일자 입력란에 인라인 에러 "그룹 기간 안에서만 허용" + [저장] 비활성 (DRAFT 한정 — ACTIVE는 read-only로 입력 자체 불가) |
| 변경 사항 없음 + [저장] 시도 | 토스트 "변경된 항목이 없습니다" + [저장] 무시 |
| [취소하기] 클릭 | 변경 사항 폐기 확인 모달 → 미션 상세로 복귀 |
| 저장 성공 | 미션 상세로 복귀 + 토스트 "수정 완료" |
| 권한 없음 진입 | 화면 미노출 또는 모든 필드 read-only + [저장] 비활성 |
