## 1. 화면 개요

기존 미션의 정보를 수정하는 폼 화면. [CEB-BO-SQ-204] 미션 상세의 [수정하기] 버튼을 통해 진입한다. 기존 데이터가 모든 필드에 prefill 되며, **부모 에피소드 상태에 따라 일부 필드가 read-only로 잠긴다** (회원 기대권 보호).

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/{id}/quests/{qid}/edit` |
| Breadcrumb | 홈 / 에피소드 / {그룹명} / {에피소드명} / 미션 N / 수정 |
| 페이지 제목 | "미션 수정" + 미션 한국어 타이틀 |
| 보조 정보 | 아티스트 배지 + 상위 에피소드명 |
| 진입 경로 | [CEB-BO-SQ-204] 미션 상세의 [수정하기] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 (수정 취소 확인 → 미션 상세)
- 페이지 제목 "미션 수정" + 현재 미션 한국어 타이틀
- 아티스트 배지 + 상위 에피소드 타이틀
- 우측: 유형 배지 (팬퀘스트=분홍 / 예측 마켓=앰버 / 서바이벌 트리비아=파랑)

### 2-2. 상태 배너 (조건부)

부모 에피소드 상태에 따라 상단에 안내 배너가 노출된다.

| 부모 에피소드 상태 | 배너 톤 | 안내 메시지 |
|---|---|---|
| **임시저장 (DRAFT)** | 미노출 | — |
| **진행중 (ACTIVE)** | 노랑 | "**진행중 에피소드의 미션**입니다. 회원 기대권 보호를 위해 **보상 필드(응모권·덕력·BIVE)** 는 수정할 수 없습니다." |
| **종료 (CLOSED)** | 회색 | "**종료된 에피소드의 미션**은 수정할 수 없습니다." + 모든 필드 read-only + [저장] 비활성 |

> 정합 근거: [CEB-BO-SQ-202-EDIT] v3.1 §4 "진행중 진입 시 완료 보상 섹션 read-only 잠금" 정책 차용.

### 2-3. 팬퀘스트 메타 안내 박스 (유형이 팬퀘스트일 때만 노출)

미션 유형이 팬퀘스트면 상단에 인디고 박스로 안내한다.

> **이 미션은 팬퀘스트 자체 데이터를 상속받습니다.**
> - 타이틀·다국어 타이틀·이미지·보상은 [팬퀘스트 영역](/fanquest/{팬퀘스트 식별값})에서만 수정 가능합니다.
> - 본 화면에서는 **반복 여부·반복 주기·기간·BIVE 토글·민팅 캠페인** 등 미션 고유 메타만 수정할 수 있습니다.
> - 팬퀘스트 매핑(fanQuestId) 자체는 본 화면에서 변경할 수 없습니다 (재매핑은 미션 삭제 후 재생성).

### 2-4. 기본 정보

| 필드 | 입력 방식 | 옵션 | 잠금 조건 |
|---|---|---|---|
| 표시 순서 | read-only | 기존 순서 표시 (자동) | 항상 read-only |
| 유형 | read-only 배지 | 팬퀘스트 / 예측 마켓 / 서바이벌 트리비아 | **항상 read-only** — 유형 변경 불가 (변경 필요 시 미션 삭제 후 재생성) |

### 2-5. 유형이 "팬퀘스트"일 때

| 필드 | 동작 |
|---|---|
| 팬퀘스트 참조 | read-only 표시 — "[{상태}] {타이틀} (#{식별값})" 형식 |
| 안내 | 핑크색 "참조 완료" 박스 — "모든 설정(타이틀·이미지·보상)은 팬퀘스트 영역에서 관리됩니다." |
| 반복 주기·기간 | 본 화면에서 별도 설정 불가. 팬퀘스트 자체 설정(반복 주기·기간)을 그대로 상속함 |

### 2-6. 유형이 "예측 마켓" 또는 "서바이벌 트리비아"일 때

**다국어 타이틀 섹션** (KO/EN/JA 3종):

| 필드 | 입력 방식 | 검증·옵션 | 필수 | 잠금 조건 |
|---|---|---|---|---|
| 타이틀 (KO/EN/JP) | 다국어 탭 전환 입력 (KO/EN/JA 3종) | 각 50자. **KO/EN/JA 3개 언어 모두 입력 필수** ([CEB-BO-011] §5 다국어 필수 정합) | ★ ★ ★ | 종료 상태에서 read-only |

> Prefill: 기존 `titleKO`·`titleEN`·`titleJA` 입력값을 그대로 노출. 운영자는 누락된 언어를 보강하거나 오타를 정정할 수 있다.

**판정 유형 섹션**:

| 필드 | 입력 방식 | 잠금 조건 |
|---|---|---|
| 판정 유형 | 드롭다운 (CREATE와 동일) | 종료 상태에서 read-only |
| 기준 횟수 | 정수 입력 ("N회 정답" 선택 시에만 활성) | 종료 상태에서 read-only |
| 소스 참조값 | 선택 입력 | 종료 상태에서 read-only |

**보상 섹션** (응모권·덕력·BIVE):

| 필드 | 입력 방식 | 잠금 조건 |
|---|---|---|
| 응모권 (장) | 정수 입력 | **진행중 + 종료 상태에서 read-only** (회원 기대권 보호) |
| 덕력 | 정수 입력 | **진행중 + 종료 상태에서 read-only** (회원 기대권 보호) |
| BIVE 토글 | 토글 ON/OFF | **진행중 + 종료 상태에서 read-only** (회원 기대권 보호) |
| 민팅 캠페인 | 드롭다운 (BIVE ON일 때만 활성) | **진행중 + 종료 상태에서 read-only** |

> 정합 근거: [CEB-BO-SQ-202-EDIT] v3.1 §4 진행중 진입 시 완료 보상 섹션 read-only 잠금 정책과 동일 원칙. 회원이 이미 기대하고 있는 보상이 운영자 임의로 변경되어선 안 된다.

**반복 여부·반복 주기·기간 섹션**:

| 필드 | 입력 방식 | 잠금 조건 |
|---|---|---|
| 반복 여부 토글 | 토글 (반복 에피소드만 노출) | 종료 상태에서 read-only |
| 반복 주기 | 버튼 그룹 (일간/주간/월간) | 종료 상태에서 read-only |
| 시작·종료일자 | 날짜 + 시간 입력 | 종료 상태에서 read-only |

### 2-7. 하단 액션 (Sticky 푸터)

- [취소하기] (빨강) — 변경 사항 폐기 확인 모달 후 미션 상세로 복귀
- [저장] (인디고) — 검증 통과 + 종료 상태 아님 + 변경 사항 존재 시 활성

## 5. 검증 규칙

| 검증 | 동작 |
|---|---|
| 부모 에피소드 = 종료 (CLOSED) | 전체 필드 read-only + [저장] 영구 비활성 |
| 부모 에피소드 = 진행중 (ACTIVE) | 보상 섹션(응모권·덕력·BIVE) read-only. 그 외 필드는 정상 수정 가능 |

| 팬퀘스트 유형 + 타이틀·보상 수정 시도 | UI에서 입력 자체 차단 (read-only) |
| 예측 마켓·서바이벌 트리비아 + 판정 유형/기준값 미입력 | [저장] 비활성 |
| BIVE ON + 민팅 캠페인 미선택 | [저장] 비활성 |
| 반복 ON + 시작일자 > 종료일자 | 종료일자 인라인 에러 + [저장] 비활성 |
| 반복 ON + 시작·종료일자가 그룹 기간 밖 | "그룹 기간 안에서만 허용" 인라인 에러 + [저장] 비활성 |
| 유형 변경 시도 | UI에서 차단 (유형은 항상 read-only — 변경 필요 시 삭제 후 재생성) |
| 팬퀘스트 매핑(fanQuestId) 변경 시도 | UI에서 차단 (재매핑은 미션 삭제 후 재생성) |
