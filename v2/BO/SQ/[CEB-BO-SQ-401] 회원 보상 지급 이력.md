## 1. 화면 목적

운영자가 에피소드 완료 보상 지급 결과를 추적·검증. 응모권·덕력·BIVE 민팅 3종 보상을 통합 조회.

## 3. 데이터 소스

- 응모권: `rft_ledger.source_feature = 'STORY_QUEST_REWARD'` (RFT-201과 통합 조회 가능)
- 덕력: `user_fan_power_history.source_type = 'STORY_QUEST'`
- BIVE 민팅: `user_fandom_level_reward` 또는 별도 SQ 보상 로그 (개발팀 ERD 추가 자료 필요)

## 4. 지급 상태

| 상태 | 의미 |
| --- | --- |
| READY | 에피소드 완료 직후 지급 작업 큐 진입 |
| PROCESSING | 지급 처리 중 |
| COMPLETED | 지급 성공 |
| FAILED | 지급 실패 (재시도 큐) |
