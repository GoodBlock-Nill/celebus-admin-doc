특정 에피소드 그룹의 기본 정보·통계·메인/반복 에피소드 카드를 한 화면에서 관리하는 운영자 허브.

| 항목 | 값 |

| 화면 경로 | `/sq/groups/{gid}` |
| Breadcrumb | 홈 / 에피소드 / 그룹 리스트 / {그룹명} |
| 페이지 제목 | 그룹의 한국어 타이틀 |
| 진입 경로 | [CEB-BO-SQ-101] 그룹 리스트의 행 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

| 카드 | 의미 | 톤 |
|---|---|---|

| 반복 에피소드 | 반복 에피소드 수 (최대 1) | 활성 (에메랄드) |
| 진행 회원 | 진행중인 회원 누적 합계 | 활성 (에메랄드) |

### 2-4. 메인 에피소드 섹션

- 섹션 헤더: "메인 에피소드 (N/5)" + 우측 [새 메인 에피소드] 버튼
  - 5개 도달 시 버튼 비활성 (안내 툴팁: "그룹당 메인 에피소드는 5개까지")
- 카드 그리드 — 표시 순서 오름차순 정렬
- 각 카드 (v2.2부터 **상태 배지 미노출** — 에피소드는 그룹 상태를 그대로 상속):
  - 메인 이미지 (3:4 비율)
  - 에피소드 한국어 타이틀
  - 미션 수 (`{현재}/10`), 진행 회원 / 완료 회원, 검수 대기
  - 임시저장 상태일 때 순서 ↑↓ 버튼 활성
- 카드 클릭 → 에피소드 상세 화면([CEB-BO-SQ-202], `/sq/{id}`) 이동

### 2-5. 반복 에피소드 섹션

- 섹션 헤더: "반복 에피소드" + 우측 [반복 에피소드 추가] 버튼
  - 이미 1개 있으면 비활성 (안내 툴팁: "반복 에피소드는 1개만 가능")
- 카드 1개 (있는 경우, 메인과 동일 양식)

## 4. 상태별 동작 분기

| 그룹 상태 | 헤더 액션 | 에피소드 카드 액션 |
|---|---|---|
| 임시저장 | [그룹 수정] · [진행중 전환] | 카드 순서 변경 가능 (↑↓), 신규 추가 가능 |
| 진행중 | [그룹 수정] · [종료 전환] | 신규 추가 가능. 순서 변경은 정책 검토 |
| 종료 | (없음, read-only) | 카드 read-only |

### 4-1. 진행중 전환 버튼 클릭 흐름

1. 같은 아티스트의 다른 그룹이 이미 진행중인지 검증
2. 다른 진행중 그룹 있음 → "다른 그룹이 진행중입니다. 진행할 수 없습니다." 토스트, 차단
3. 없음 → 확인 모달 "이 그룹을 진행중으로 전환합니다. 계속하시겠습니까?"
4. 확인 → 그룹 상태 = 진행중, "그룹이 진행중으로 전환되었습니다." 토스트

### 4-2. 종료 전환 버튼 클릭 흐름

1. 확인 모달 "이 그룹을 종료합니다. 모든 하위 에피소드도 종료됩니다."
2. 확인 → 그룹 상태 = 종료, 모든 하위 에피소드 자동 종료, 토스트

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 메인 한도 | 5개 초과 시 [새 메인 에피소드] 비활성 |
| 반복 한도 | 1개 초과 시 [반복 에피소드 추가] 비활성 |
| 에피소드 상태 상속 | 에피소드의 상태 배지는 노출하지 않음. 그룹 상태로 일관 표시 |
| 미션 순서 변경 가능 조건 | 임시저장 상태에서만 ↑↓ 활성 |
| 진행중 전환 검증 | 같은 아티스트의 다른 진행중 그룹 존재 시 차단 |
