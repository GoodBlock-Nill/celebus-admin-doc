## 1. 모달 개요

검수 대기 제출 1건을 반려할 때 반려사유를 선택하는 모달. 운영자는 [CEB-BO-SQ-501] 반려사유 설정에서 등록된 활성 사유 중 1개를 라디오로 선택.

| 항목 | 값 |
|---|---|
| 트리거 | 대기내역 탭의 카드 [반려] 버튼 |
| 진입 조건 | 검수 대기 제출 카드 존재 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 영향 범위 | 해당 제출 1건 반려 처리 + 사유 기록. 회원에게 반려 사유 노출 메시지 전달 |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "반려 사유 선택"
- 우상단: X 닫기 버튼

### 2-2. 본문

- 보조 정보: "Submit ID: {제출 식별값}" (회색 텍스트)
- 반려 사유 라디오 그룹 — [CEB-BO-SQ-501] 등록된 **활성(사용)** 사유만 노출:
  - 각 행: 라디오 + 운영자 노출명
  - 선택된 사유는 인디고 배경 + 인디고 보더 강조
- 사유 0건일 때: "사용 가능한 반려사유가 없습니다." 중앙 정렬 안내

### 2-3. 하단 액션

| 버튼 | 색상 | 동작 |
|---|---|---|
| 취소 | 회색 외곽선 | 모달 닫기 |
| 반려하기 | **빨강** | 선택한 사유로 제출 반려 처리 + "반려 처리되었습니다. ({사유명})" 토스트 |

> [반려하기]는 사유가 선택되어야 활성. 미선택 시 비활성.

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 활성 사유 0건 | 빈 상태 메시지 + [반려하기] 영구 비활성 |
| 사유 미선택 | [반려하기] 비활성 |
| 사유 선택 후 [반려하기] | 반려 처리 + 카드 그리드에서 제거 + 토스트 |
| [취소] / X 클릭 | 모달 닫기 (제출 상태 변경 없음) |
| 외부 클릭 | [취소]와 동일 (닫기) |
| 권한 없음 | 트리거 버튼 자체 미노출 |
