## 1. 화면 개요

신규 Quest 1건을 임시저장 상태로 생성하는 입력 화면. 3섹션(기본 정보 / 퀘스트 설정 / 보상 설정)으로 구성되며, 생성 시점에는 항상 임시저장 상태로 저장된다. 회원에게 노출되려면 별도 게시 작업 필요.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/fanquest/create` (목표 라우트 `/sq/quests/create`) |
| Breadcrumb | 홈 / 에피소드 / 팬퀘스트 / Quest 생성 |
| 페이지 제목 | "Quest 생성" |
| 진입 경로 | [CEB-BO-SQ-302] 팬퀘스트 리스트의 [생성하기] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 저장 결과 | 임시저장 상태로 생성. 게시는 [CEB-BO-SQ-303] 상세 화면의 [게시하기] 액션으로 별도 진행 |

## 2. 화면 구성

### 2-1. 헤더

- 좌측: ← 뒤로가기 버튼 (취소 확인 모달 노출 후 리스트로 복귀)
- 페이지 제목 "Quest 생성"
- 우측 액션 2개:
  - [취소하기] (빨강) — 변경사항 확인 후 리스트로 복귀
  - [생성하기] (인디고) — 모든 필수 항목 입력 완료 시 활성. 클릭 시 완료 모달 노출 후 임시저장 상태로 저장

### 2-2. 입력 폼 (3섹션)

#### A. 기본 정보 섹션

| 필드 | 입력 방식 | 글자수·검증 | 필수 |
|---|---|---|---|
| 타이틀 | 한국어/영어/일본어 탭 입력 | 각 100자 이내. 미입력 언어는 탭 우상단 빨강 점 표시 | ★ (3개 언어 모두) |
| 설명 | 한국어/영어/일본어 탭 입력 (4행) | 각 200자 이내 | ★ (3개 언어 모두) |
| 유저 가이드 | 한국어/영어/일본어 탭 입력 (6행) | 각 500자 이내 | ★ (3개 언어 모두) |
| 대표 이미지 | 드래그앤드롭 또는 파일 업로드 | 권장 1:1 비율, 최대 5MB, jpeg/png/jpg/webp/gif/svg | ★ |

#### B. 퀘스트 설정 섹션

| 필드 | 입력 방식 | 옵션·검증 | 필수 |
|---|---|---|---|
| 아티스트 | 드롭다운 | "아티스트 선택" / V01D / iKON / CELEBUS | ★ |
| 퀘스트 타입 | 드롭다운 | "이미지 촬영 및 업로드" (기본) | ★ |
| 참여 방식 | 라디오 (2종, 가로 분할) | "일반 (1회 참여)" / "반복 (주기 반복)" | ★ |
| 반복 주기 | 버튼 그룹 3종 (참여 방식이 반복일 때만 노출) | ☀️ 일간 (하루 1회) / 🗓️ 주간 (주 1회) / 📅 월간 (월 1회) | ★ (반복일 때) |
| 운영 종료일 / 마감 일시 | 날짜 + 시간 선택 (2열) | 시간 기본 23:59, 생성 모드는 오늘 +1일 이후만 선택 가능 | ★ |
| 연관 링크 | 동적 행 추가 (다국어 라벨 20자 + URL) | 선택, 다국어 탭 + URL | — |

> 시작 일시는 게시 시점에 자동 설정. 반복형은 운영 종료일까지 반복 후 자동 종료.

#### C. 보상 설정 섹션

지급할 보상을 1개 이상 선택 필수. 선택한 항목만 회원에게 지급.

| 보상 | 입력 방식 | 비고 |
|---|---|---|
| 응모권 | 체크박스 + 수량 입력 (장) | 체크 시 수량 ≥ 1 |
| 덕력 | 체크박스 + 수량 입력 (DUK) | 체크 시 수량 ≥ 1 |
| NFT | 체크박스 + 민팅 캠페인 드롭다운 | 현재 mock: "선택 가능한 NFT 이벤트가 없습니다. BIVE 메뉴에서 이벤트를 먼저 생성해주세요." 안내 |

> 모든 보상이 비활성이면 "보상 항목을 1개 이상 선택해주세요." 빨강 안내 노출 + [생성하기] 비활성.
