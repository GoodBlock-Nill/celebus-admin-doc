## 1. 화면 개요

사이드바 "에피소드 > 팬퀘스트" 진입 시 노출되는 SQ 영역 메인 리스트 화면. 운영자가 팬퀘스트(Quest) 전체를 한 화면에서 조회·필터·검색하고 신규 생성·반려사유 설정 진입을 수행한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/sq/quests` |
| Breadcrumb | 홈 / 에피소드 / 팬퀘스트 |
| 페이지 제목 | "팬퀘스트" |
| 진입 경로 | 사이드바 "에피소드 > 팬퀘스트" |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더

- 페이지 제목 "팬퀘스트"
- Breadcrumb (홈 → 에피소드 → 팬퀘스트)

### 2-2. 필터·액션 바 (한 줄 가로 배치)

| 위치 | 요소 | 동작 |
|---|---|---|
| 좌 | 상태 드롭다운 | "상태(전체)" (기본) / 진행중 / 임시저장 / 종료 |
| 우 | 검색 대상 드롭다운 | "타이틀" (기본) / "아티스트" |
| 우 | 키워드 입력 | 부분 일치 검색. placeholder는 검색 대상별 분기 ("타이틀 입력" / "아티스트 입력") |
| 우 | [초기화] 버튼 (어두운 회색) | 모든 필터·검색·페이지 리셋 |
| 우 | [반려사유 설정] 버튼 (빨강 톤) | 클릭 시 [CEB-BO-SQ-501] 반려사유 설정 화면으로 이동 |
| 우 | [생성하기] 버튼 (인디고 톤) | 클릭 시 [CEB-BO-SQ-302-CREATE] Quest 생성 화면으로 이동 |

### 2-3. 팬퀘스트 목록 테이블 (7개 컬럼)

| 컬럼 | 너비 | 표시 내용 |
|---|---|---|
| 상태 | 100px | 상태 배지 (진행중=에메랄드 / 임시저장=회색 / 종료=진한 회색) |
| 타이틀 | flex | Quest 한국어 타이틀 |
| 아티스트 | 110px | V01D / iKON / CELEBUS |
| 진행 방식 | 90px | 배지 — 반복(인디고 톤) / 일반(회색 톤) |
| 반복 주기 | 90px | 반복인 경우 일간 / 주간 / 월간. 일반인 경우 "—" |
| 기간 | 200px | "시작일 ~ 마감일" 또는 임시저장이면 "— ~ 마감일" |
| 검토 필요 | 90px (우측 정렬) | 검수 대기 제출 건수 (강조) |

각 행을 클릭하면 Quest 상세 화면([CEB-BO-SQ-303], `/sq/quests/{id}?tab=info`)으로 이동.

### 2-4. 페이지네이션

- **페이지당 10건** (필터·검색 결과 기준)
- **항상 노출** — 결과가 10건 미만이라 1페이지일 때도 비활성 형태로 노출
- 필터·검색·키워드 변경 시 자동으로 1페이지로 리셋

### 2-5. 빈 상태

검색 결과 0건 시: "팬퀘스트가 없습니다." 메시지 노출.

## 4. 상태별 동작 분기

본 화면은 **상태 변경 액션이 없는 조회·진입 전용 화면**이므로 데이터 가용성 분기만 존재.

| 데이터 조건 | 동작 |
|---|---|
| 검색 결과 0건 | "팬퀘스트가 없습니다." 빈 상태 메시지 |
| 데이터 로딩 중 | Skeleton 테이블 |
| 권한 없음 진입 | [관리자 > 권한관리] 설정에 따라 화면 미노출 또는 [생성하기]·[반려사유 설정] 비활성 |
| 기본 정렬 | 최근 수정 일시 내림차순 (프로토타입 정합) |
