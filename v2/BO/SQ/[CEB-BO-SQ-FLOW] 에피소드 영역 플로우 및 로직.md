## 1. 영역 개요

- **메뉴**: 사이드바 "에피소드" 그룹
- **계층 (v2.2부터, 3단계)**: 에피소드 그룹 → 에피소드 → 미션 (미션이 완료 판정을 직접 흡수)
- **운영 용어**: 에피소드 그룹 / 에피소드 / **미션**
- **권한**: [관리자 > 권한관리]에서 설정된 권한을 따른다 ([CEB-BO-011] §3 정합)

## 3. 운영 계층 요약

본 영역은 3계층(에피소드 그룹 → 에피소드 → 미션)으로 구성된다. v2.2부터 "완료 조건"이 미션에 흡수되어 미션 1개 = 완료 판정 1개로 단순화.

### 3-1. 에피소드 그룹

| 항목 | 정의 |
|---|---|
| 아티스트 | V01D / iKON / CELEBUS 중 선택 |
| 상태 | 임시저장 / 진행중 / 종료 |
| 타이틀 | 한국어 (필수) |
| 기간 | 시작일·종료일 (그룹의 SSOT — 하위 에피소드는 본 기간을 상속) |
| 에피소드 수 | 하위 에피소드 합계 (자동 계산) |
| 생성·수정 메타 | 생성자 / 생성일시 / 최근 수정자 / 최근 수정일시 |

### 3-2. 에피소드 (그룹 하위)

| 항목 | 정의 |
|---|---|
| 부모 그룹 | 1:N 관계 |
| 유형 | 메인 / 반복 (그룹당 메인 5개 + 반복 1개 한도) |
| 표시 순서 | 메인 1~5, 반복은 6번 슬롯 고정 |
| 아티스트 | 부모 그룹에서 상속 |
| 상태 | 부모 그룹에서 상속 |
| 다국어 타이틀·설명 | 한국어/영어/일본어 모두 필수 (게시 시) |
| 메인 이미지 | 3:4 비율 권장 |
| 미션 수 | 하위 미션 합계 (자동 계산, 최대 10개) |
| 에피소드 완료 보상 (v3.2 신규, 메인 전용) | 응모권 수량 + 덕력 수량 + BIVE 보상 토글 + 민팅 캠페인. 에피소드 안 모든 미션 완료 회원에게 자동 지급. 진행중 상태 진입 시 read-only |
| 통계 | 진행 회원 수 / 완료 누적 수 / 검수 대기 수 |

### 3-3. 미션 (에피소드 하위)

| 항목 | 정의 |
|---|---|
| 부모 에피소드 | 1:N 관계 |
| 순서 | 1~10 (↑↓ 화살표로 변경, 임시저장 상태에서만 활성) |
| 유형 | 팬퀘스트 / 예측 마켓 / 서바이벌 트리비아 중 선택 |
| 다국어 타이틀 (v3.4 신규, v3.5 정정, PM·ST 한정) | **KO/EN/JP 3종, 각 50자**. **KO/EN/JA 3개 언어 모두 입력 필수** ([CEB-BO-011] §5 "다국어 필수" 표준 정합). 팬퀘스트는 매핑된 팬퀘스트 타이틀 자동 상속 (본 입력 미노출) |
| 팬퀘스트 매핑 | 유형이 "팬퀘스트"일 때, 팬퀘스트 Quest와 1:1 연결 |
| 보상 | 응모권 지급 수량 / 덕력 지급 수량 / BIVE 보상 토글 + 민팅 캠페인 |
| 반복 여부 (v3.2 정정) | **부모 에피소드 유형에 따라 분기**. 메인 에피소드: 자동 단일(반복 토글 미노출). 반복 에피소드: 반복 토글 노출 (기본 OFF) |
| 반복 주기 (v3.2 신규, PM·ST 한정) | **일간 / 주간 / 월간** 3종 (반복 토글 ON일 때 1개 선택 필수). 팬퀘스트는 팬퀘스트 자체 설정 상속 (본 영역에서 별도 설정 불가) |
| 반복 운영 기간 (v3.2 신규) | 시작·종료일자 자유 입력. 비우면 그룹 기간 상속. **그룹 기간 안에서만 허용** (벗어나면 인라인 에러) |
| 완료 판정 (v2.2 흡수) | 완료 판정 유형 / 완료 판정 기준값 / 소스 참조값 / 기간 (비우면 부모 기간 상속) |

#### 미션 완료 판정 유형 매트릭스

| 미션 유형 | 가능한 완료 판정 유형 | 기준값 사용 |
|---|---|---|
| 팬퀘스트 | "관리자 승인" 고정 | 없음 |
| 예측 마켓 | "예측 참여" / "예측 정답" 중 선택 | 없음 |
| 서바이벌 트리비아 | "트리비아 참여" / "트리비아 N회 정답" 중 선택 | "N회 정답" 선택 시 횟수 입력 |

> **소스 참조값**: 예측 마켓·서바이벌 트리비아 유형일 때만 사용. 팬퀘스트는 팬퀘스트 매핑으로 대체.

## 4. 상태 머신

### 4-1. 에피소드 그룹 상태

```
[그룹 생성]
   ↓ 초기 상태
임시저장 ───── 진행중 전환 ─────→ 진행중 ───── 종료 전환 ─────→ 종료 (read-only)
   │
   삭제 가능
```

| 상태 | 의미 | 트리거 |
|---|---|---|
| 임시저장 | 운영자 작성 중. 회원 미노출 | 그룹 생성 시 초기 상태 |
| 진행중 | 회원 앱에 공개 — 아티스트당 1개만 가능 | 운영자 액션: "진행중 전환" 버튼 |
| 종료 | read-only. 수정·삭제·하위 신규 추가 불가 | 운영자 액션 "종료 전환" 또는 자동: 종료일 도래 |

### 4-2. 에피소드·미션 상태 상속

에피소드와 미션은 부모 그룹의 상태를 그대로 상속한다.

- 그룹이 임시저장이면 모든 에피소드·미션은 임시저장
- 그룹이 진행중일 때만 회원 앱에 노출
- 그룹이 종료되면 모든 하위 에피소드·미션은 자동 종료

### 4-3. 응모자 0건 또는 미션 0건의 처리

- 그룹·에피소드 모두 비어 있는 상태로 종료해도 무방
- 삭제는 임시저장 상태에서만 가능

## 5. 정책

### 5-1. 아티스트당 진행중 그룹 1개 제한

- 그룹 생성·수정에서 "진행중 전환" 시 검증
- 같은 아티스트의 다른 그룹이 이미 진행중이면 차단 ("이미 {아티스트}에 진행 중인 그룹이 있습니다" 토스트)
- 운영자는 기존 그룹을 종료 전환한 뒤 새 그룹을 진행할 수 있음
- 운영 시점에 아티스트별 진행중 그룹 1개 → 회원 앱 UX 일관성 확보

### 5-2. 메인/반복 구분 및 표시 슬롯

- 그룹당 메인 5개 + 반복 1개 = 최대 6개 에피소드
- 반복 에피소드는 항상 표시 슬롯 6번 (메인 5개 다음)
- 반복 에피소드의 미션은 팬퀘스트 유형 중 "반복" 종류의 Quest만 매핑 가능 (팬퀘스트-에피소드 매핑 정책)

### 5-2-1. 메인/반복 에피소드 미션 유형 분기 정책 (v3.2 신규)

- **메인 에피소드 미션 유형**: **단일 미션만** 추가 가능 (반복 토글 자동 OFF·잠금). 반복 미션이 필요하면 반복 에피소드에서 추가
- **반복 에피소드 미션 유형**: 단일·반복 미션 모두 추가 가능. 단, 팬퀘스트 유형은 기존 정책대로 반복 종류 팬퀘스트만 매핑 가능
- **PM·ST 반복 미션 반복 주기**: 일간 / 주간 / 월간 3종. 반복 토글 ON일 때 1개 선택 필수. 팬퀘스트 유형 미션은 팬퀘스트 자체 설정 상속 (본 영역에서 별도 설정 불가)
- **반복 미션 운영 기간**: 시작·종료일자 자유 입력. 비우면 그룹 기간 상속. 그룹 기간 안에서만 허용 (벗어나면 인라인 에러)
- 정합 문서: [CEB-BO-SQ-204-CREATE] v3.1 §2-3·§2-7

### 5-2-2. 메인 에피소드 완료 보상 정책 (v3.2 신규)

- **노출 위치**: 메인 에피소드 생성·수정·상세 화면에서만 노출. 반복 에피소드에서는 본 섹션이 숨겨진다
- **구성**: 응모권 보상(정수) + 덕력 보상(정수) + BIVE 보상 토글 + 민팅 캠페인 드롭다운 (BIVE ON일 때 필수)
- **지급 시점**: 에피소드 안의 **모든 미션이 완료된 회원에게 자동 지급**. 미션 보상과 별도 추가 지급
- **저장 허용**: 모든 값이 0/OFF여도 저장 가능 (보상 없는 에피소드 허용)
- **진행중 잠금**: 진행중 상태 진입 시 회원 기대권 보호 차원에서 보상 필드 전체 read-only. 변경 필요 시 종료 후 재게시 또는 별도 운영 절차
- 정합 문서: [CEB-BO-SQ-202-CREATE] v3.1 §2-7, [CEB-BO-SQ-202-EDIT] v3.1 §2-7, [CEB-BO-SQ-202] v3.1 §2-3

### 5-3. 그룹 기간 = 에피소드 기간 단일 SSOT

- 그룹의 시작일·종료일이 단일 SSOT
- 에피소드별 기간 필드 없음
- 그룹 기간 외에는 에피소드도 자동 비활성

### 5-4. 메인 이미지

- 그룹: 메인 이미지 없음
- 에피소드: 3:4 비율 권장. 회원 앱 카드 노출용
- 미션: 별도 썸네일 이미지 1장

### 5-5. 다국어 (한국어/영어/일본어)

- 에피소드 레벨: 타이틀·설명 3개 언어 필수 (게시 시 검증)
- 미션 레벨: 현재 한국어 우선 (다국어 정책은 운영 협의)

### 5-6. 팬퀘스트 ↔ 미션 1:1 매핑

- 미션 유형이 "팬퀘스트"일 때만 팬퀘스트 Quest와 연결
- 하나의 팬퀘스트는 정확히 하나의 미션에만 매핑 가능
- 미션 추가 화면에서 이미 다른 미션에 매핑된 팬퀘스트는 드롭다운에 노출되지 않음
- 팬퀘스트 상세 화면에서는 본 매핑을 역참조해 "사용 중인 에피소드" 패널로 노출

### 5-7. 미션 완료 판정 정책 (v2.2 — 완료 조건 흡수 후)

- **미션 1개 = 완료 판정 1개**. 별도 완료 조건 목록 없음
- 완료 판정 필드는 미션 자체에 직접 저장: 완료 판정 유형 / 완료 판정 기준값 / 소스 참조값 / 반복 주기 / 기간
- 완료 판정 유형 옵션은 §3-3 매트릭스 참조
- 완료 판정 기준값은 "트리비아 N회 정답" 선택 시에만 의미 있음
- 소스 참조값은 예측 마켓·서바이벌 트리비아일 때만 사용
- 반복 주기는 반복 에피소드의 PM·ST 미션에서만 의미 있음 (일간 / 주간 / 월간). 팬퀘스트 유형은 팬퀘스트 자체 설정 상속
- 기간은 비우면 부모 그룹의 기간을 상속 (개별 오버라이드 가능)

### 5-8. 삭제 정책 (cascade · v2.2)

| 대상 | 허용 상태 | cascade 영향 |
|---|---|---|
| 에피소드 그룹 | 임시저장만 | 하위 에피소드·미션 모두 삭제 + 팬퀘스트 매핑 자동 해제 |
| 에피소드 | 임시저장만 | 하위 미션 모두 삭제 + 팬퀘스트 매핑 자동 해제 + 부모 그룹의 에피소드 수 자동 감소 |
| 미션 | 부모가 임시저장일 때만 | 단순 삭제 + 팬퀘스트 매핑 자동 해제 (완료 판정 필드는 미션과 함께 즉시 소멸) |

> 진행중·종료 상태는 삭제 대신 라이프사이클 전환(종료 전환)으로 종료. 팬퀘스트 Quest 자체는 어떤 cascade에도 삭제되지 않음 — 매핑만 해제됨.

### 5-9. 미션 순서 변경 (v2.1)

- 에피소드 상세 화면의 미션 카드에 **↑↓ 화살표 버튼** 제공
- 부모 에피소드가 임시저장 상태일 때만 활성
- 첫 카드의 ↑, 마지막 카드의 ↓는 비활성
- 클릭 시 인접 미션과 순서 스왑 → 즉시 화면 재렌더 (토스트 없음, 빠른 반복 작업)
- 진행중·종료 상태에서는 ↑↓ 둘 다 비활성 + 툴팁 "임시저장 상태에서만 순서 변경 가능"

## 6. 사용자 플로우

### 6-1. 그룹 생성 → 에피소드 추가 → 게시

```
1. /sq/groups/list 진입 → "새 그룹 생성"
2. /sq/groups/create — 아티스트·상태·타이틀·시작·종료일자 입력
   - 진행중 선택 시 같은 아티스트에 이미 진행중 그룹 있으면 경고·차단
3. 저장 → /sq/groups/{gid} 그룹 상세
4. 그룹 상세에서 "새 메인 에피소드" 또는 "반복 에피소드 추가" → /sq/create
5. 에피소드 생성 후 그룹 상세로 복귀 — 메인·반복 섹션에 카드 노출
6. 에피소드 상세에서 "수정" → /sq/{id}/edit
7. 그룹 상태도 "진행중 전환" 버튼으로 운영 시작
```

### 6-2. 팬퀘스트-에피소드 연동 플로우

```
1. 팬퀘스트 운영자가 /fanquest/create로 Quest 생성 (반복 또는 일반)
2. 에피소드 운영자가 /sq/{id}/quests/create에서 미션 유형 "팬퀘스트" 선택
3. 아티스트 + 반복 여부로 필터된 팬퀘스트 Quest 드롭다운에서 선택
   - 이미 다른 미션에 묶인 팬퀘스트는 제외 (1:1 정책)
   - 반복 에피소드의 미션에는 반복 종류의 팬퀘스트만 노출
4. 저장 → 미션에 팬퀘스트 매핑 기록
5. 이후 팬퀘스트 상세 화면에서 "사용 중인 에피소드" 패널에 자동 노출
```

### 6-3. 검수 대기 처리

```
1. 회원이 앱에서 팬퀘스트 미션 제출 → 팬퀘스트 Quest의 검수 대기 수 증가
2. 운영자가 /sq/pending(팬퀘스트 대기내역) 진입
3. 검수 대기 수 > 0인 Quest 목록 노출 (검토 필요 내림차순)
4. 행 클릭 → 팬퀘스트 상세 대기내역 탭으로 이동
5. 팬퀘스트 대기내역 탭에서 행별 승인·반려 또는 일괄 승인
6. 처리 후 검수 대기 수 감소
```

### 6-4. 종료 플로우

```
1. 그룹 상태가 종료로 전환 (자동 종료일 도달 또는 운영자 수동)
2. 모든 하위 에피소드 자동 종료
3. read-only 상태 — 수정·게시 진입 불가
4. 통계(진행 회원 / 완료 누적 / 검수 대기) 보존
```

### 6-5. 미션 추가 플로우 (v3.2 — 메인/반복 분기 명시)

```
1. 운영자가 /sq/{id} (에피소드 상세) 진입
2. 미션 그리드 우측 "미션 추가" 버튼 클릭
   - 에피소드 미션 수 = 10이면 버튼 비활성
3. /sq/{id}/quests/create (미션 추가 페이지) 진입
4. 부모 에피소드 유형 안내 박스 노출:
   - 메인 에피소드 진입 → "단일 미션만 추가 가능" 인디고 박스 (반복 토글 미노출)
   - 반복 에피소드 진입 → "단일·반복 모두 가능, 팬퀘스트는 반복 종류만" 바이올렛 박스
5. 미션 유형(팬퀘스트 / 예측 마켓 / 서바이벌 트리비아) + 보상 + 완료 판정 입력
   - 팬퀘스트: 반복 주기·기간 본 화면에서 설정 불가 (팬퀘스트 자체 설정 상속)
   - 예측 마켓·서바이벌 트리비아: 미션 자체 필드(완료 판정 유형·기준값·소스 참조값)에 직접 저장
     · 부모=메인: 반복 토글 미노출. 자동 단일 미션
     · 부모=반복: 반복 토글 노출 (기본 OFF). ON 시 반복 주기(일간/주간/월간 1개) + 시작·종료일자 추가 입력
       그룹 기간 밖이면 인라인 에러
6. 추가하기 → 에피소드 상세 복귀 + 카드 즉시 갱신 + 토스트
```

### 6-6. 미션 수정 플로우 (v3.0 정정 — 모달 폐기, 상세 화면 진입)

```
1. 에피소드 상세의 미션 카드 또는 행 클릭
   - 부모 에피소드·그룹이 종료면 클릭 비활성 또는 read-only 진입
2. 미션 상세 [CEB-BO-SQ-204] 노출 (현재 값 prefill)
3. 소스 참조값 · 완료 판정 유형 · 기준값 · 반복 주기 · 기간 수정 가능
   - 미션 유형·식별값·부모 에피소드는 read-only
4. 저장하기 → 행 갱신 + 토스트
```

> **v3.0 정정**: v2.2 이전에는 "미션 수정 모달"이 존재했으나 v2.2부터 미션 상세 화면([CEB-BO-SQ-204])에서 진입·수정한다. v2.2 시점 본 문서가 모달 ID를 잘못 표기하던 부분 정정.

### 6-7. 그룹·에피소드 삭제 플로우 (임시저장 한정)

```
1. 그룹 또는 에피소드 상세에서 "삭제" CTA 클릭
   - 임시저장 상태에서만 CTA 노출
2. 삭제 확인 모달 노출
   - 영향 통계 표시 (에피소드 N개 / 미션 N개 등)
   - 미션에 팬퀘스트 매핑 있으면 "매핑 해제됨" 안내
3. 삭제 (적색 위험) → cascade 삭제 + 토스트 + 상위 화면 이동
   - 그룹 삭제 → 그룹 리스트
   - 에피소드 삭제 → 그룹 상세 (에피소드 수 자동 감소)
```

## 7. 에피소드 ↔ 팬퀘스트 연동 (단방향 + 역참조)

| 방향 | 운영 동작 | 사용처 |
|---|---|---|
| 에피소드 → 팬퀘스트 (조회) | 아티스트별 팬퀘스트 Quest 목록 조회 | 미션 추가 폼 드롭다운 |
| 에피소드 → 팬퀘스트 (반복 한정) | 반복 종류 팬퀘스트만 조회 | 반복 에피소드의 미션 드롭다운 |
| 에피소드 → 팬퀘스트 (중복 방지) | 이미 매핑된 팬퀘스트 식별값 조회 | 드롭다운에서 제외 |
| 팬퀘스트 → 에피소드 (역참조) | 특정 팬퀘스트의 사용 미션 조회 | 팬퀘스트 상세 "사용 중인 에피소드" 패널 |
| 데이터 보존 | 미션-팬퀘스트 1:1 매핑 테이블 | 운영자 액션 시 자동 갱신 |
