## 1. 모달 개요

신규 반려사유를 등록하는 모달. 운영자 노출명 + 회원 노출 메시지(한국어/영어/일본어 다국어)를 입력. 등록 시 자동으로 "사용(Active)" 상태로 추가.

| 항목 | 값 |
|---|---|
| 트리거 | 반려사유 설정의 [+ 새 사유 추가] 버튼 |
| 진입 조건 | 항상 가능 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 영향 범위 | 반려사유 신규 1건 등록 (Active 상태) |

## 2. 화면 구성

### 2-1. 헤더

- 모달 제목: "반려사유 추가"
- 우상단: X 닫기 버튼

### 2-2. 입력 폼

| 필드 | 입력 방식 | 글자수·검증 | 필수 |
|---|---|---|---|
| 운영자 노출명 | 텍스트 + 글자수 카운터 | 최대 100자 | ★ (한국어 1자 이상) |
| 유저 노출 메시지 | 다국어 라디오 + 텍스트 영역 (4행) + 진행률 표시 | 각 200자 | ★ (한국어 1자 이상) |

#### 다국어 라디오 그룹

| 라디오 | 라벨 |
|---|---|
| KO | "한국어(기본값)" — 기본 활성 |
| EN | "영어" |
| JA | "일본어" |

#### 다국어 진행률 표시

각 언어별로 입력 상태를 시각적으로 표시:
- 입력됨: 에메랄드 톤 + "✓"
- 미입력: 회색

### 2-3. 하단 액션

| 버튼 | 색상 | 동작 |
|---|---|---|
| 취소하기 | 회색 외곽선 | 모달 닫기 (입력값 폐기) |
| 추가하기 | 인디고 | 신규 사유 등록 + 모달 닫기 |

> [추가하기]는 운영자 노출명 + 한국어 메시지 모두 입력 시 활성. 영어·일본어는 선택 (다국어 매칭 폴백 정책에 따름).

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 운영자 노출명 미입력 | [추가하기] 비활성 |
| 한국어 메시지 미입력 | [추가하기] 비활성 |
| 한국어만 입력 + 영어·일본어 비어있음 | [추가하기] 활성 (선택 다국어이므로 통과) |
| [추가하기] 클릭 | 사유 등록 (자동 Active 상태) + 모달 닫기 + 리스트 갱신 |
| [취소하기] / X / 외부 클릭 | 모달 닫기 (입력값 폐기) |
| 권한 없음 | 트리거 버튼 자체 미노출 |
