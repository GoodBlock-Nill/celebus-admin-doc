## 1. 영역 개요

**스토리 퀘스트(이하 SQ, 운영 명칭 "에피소드")** 는 아티스트별 시즌·큐레이션 묶음을 운영하는 영역이다. v2.2부터 다음 **3계층**으로 구성된다.

```
에피소드 그룹                   ← 아티스트별 시즌·큐레이션 묶음
  └─ 에피소드                  ← 그룹 안의 메인 5개 + 반복 1개
       └─ 미션                ← 에피소드 안의 콘텐츠 단위, 최대 10개 (완료 판정 직접 흡수)
```

| 항목 | 값 |
|---|---|
| 영역 코드 | SQ (Story Quest, 운영 명칭 "에피소드") |
| 영역 가이드 | 본 문서 `[CEB-BO-011]` |
| 화면 ID 패턴 | `CEB-BO-SQ-{번호}[-{유형}]` |
| 사이드바 메뉴 | 좌측 "에피소드" 그룹 |
| 운영 화면 수 | 활성 30개 + 통합 플로우 문서 1개 + 폐지된 7개 |

> ⚠️ **운영 용어 주의**: 사이드바·페이지 제목·본문 모두 "에피소드"로 통일한다. 화면 본문에 영문 인터페이스명("StoryQuest" 등)이나 코드 식별자(`fanQuestId`·`displayOrder` 등)를 사용하지 않는다.

## 2. 메뉴 구조

사이드바 "에피소드" 그룹은 다음 4개 메뉴로 구성된다. (v1.1 메뉴 재구성 — 팬퀘스트 Quest 리스트가 본 영역 하위로 이전, v3.1 — 반려사유 설정도 본 영역으로 흡수)

```
에피소드  📖
  ├─ 그룹 리스트          → /sq/groups/list
  ├─ 팬퀘스트 대기내역    → /sq/pending
  ├─ 팬퀘스트            → /sq/quests       (v1.1 신규 — 구 /fanquest Quest 탭에서 이전. v3.1 — SQ-302로 SQ 영역 컨텍스트 명세 신규 작성)
  └─ 반려사유 설정       → /sq/reject-reasons  (v3.1 신규 흡수 — 구 /fanquest/reject-reasons)
```

기본 확장 상태로 설정되어 페이지 로드 시 자동 펼침.

> **팬퀘스트 메뉴 이전·흡수(v1.1·v3.1)**: 팬퀘스트(Quest)는 SQ 미션의 1:1 소스이므로 운영 흐름상 본 영역 안에서 관리한다. v3.1부터 Quest 관련 17개 화면 명세도 모두 SQ 영역(SQ-302~SQ-305 + SQ-501·관련 모달)으로 이전됨. 기존 [CEB-BO-FQ-101] 등 FQ 원본은 DEPRECATED 표기로 이력 보존. 래플은 별도 "래플" 그룹으로 분리.

## 3. 권한

본 영역의 모든 화면·액션 권한은 **[관리자 > 권한관리]에서 설정된 권한을 따른다**. 권한이 없는 운영자가 진입한 경우 해당 화면·버튼은 미노출 또는 비활성 상태로 처리한다. 본 문서에서는 세부 역할 명세를 별도로 두지 않는다.

## 5. 정책 상수

본 영역 운영 정책 및 한도는 다음과 같다.

| 정책 | 값·정의 |
|---|---|

| 반복 미션 반복 주기 | **일간 / 주간 / 월간** 3종. 예측 마켓·서바이벌 트리비아 미션이 반복 ON일 때 본 화면에서 설정. 팬퀘스트 유형은 팬퀘스트 자체 설정 상속(본 영역에서 별도 설정 불가) |

| 미션 순서 변경 | 에피소드 상세에서 ↑↓ 화살표로 변경. **임시저장 상태에서만 활성** |
| 에피소드 카드 상태 배지 | v2.2부터 미노출 — 에피소드는 그룹 상태를 그대로 상속하므로 별도 배지 불필요 |
| 공통 페이지네이션 | **페이지당 20건** (모달 내 시계열은 10건), **항상 노출** (1페이지일 때도 비활성 형태). 윈도잉 규칙: 총 페이지 ≤ 7이면 전체 노출, 8 이상이면 `1 ... cur-1 cur cur+1 ... last` 윈도우 |

## 6. 상태 머신

에피소드 그룹은 다음 3상태를 순차적으로 거친다. 에피소드(그룹 내)와 미션(에피소드 내)은 그룹의 상태를 상속한다.

```
[그룹 작성]
   ↓ 임시저장
임시저장 ───── 진행중 전환 ─────→ 진행중 ───── 종료 전환 ─────→ 종료
   │                                                              │
   삭제 가능 (DRAFT 한정)                                        read-only
```

| 상태 | 의미 | 운영자 가능 액션 |
|---|---|---|
| 임시저장 | 미게시. 작성·검토 중 | 수정 / 삭제 / 진행중 전환 / 하위 에피소드·미션 자유롭게 추가·수정·순서 변경 |
| 진행중 | 회원 앱에 노출 + 회원 참여 가능 | 수정 (응모자 보호 잠금 적용) / 종료 전환 / 하위 신규 추가 가능 |
| 종료 | 정산 완료 | read-only (수정·삭제·하위 신규 추가 모두 불가) |

> **진행중 전환 조건**: 같은 아티스트의 다른 그룹이 이미 "진행중"이면 차단된다. 운영자는 기존 그룹을 종료 전환한 뒤 새 그룹을 진행할 수 있다.

> **종료 전환 영향**: 그룹을 "종료"로 전환하면 모든 하위 에피소드도 자동으로 종료 상태가 된다.

### 6-1. 응모자 0건 또는 미션 0건의 처리

- 그룹·에피소드 모두 비어 있는 상태로 종료해도 무방
- 삭제는 임시저장 상태에서만 가능 (진행중·종료 상태는 read-only)

## 7. 외부 영역 연동

| 외부 영역 | 연결 방식 | 핵심 정의 |
|---|---|---|
| 팬퀘스트(FQ) | 미션의 팬퀘스트 매핑 (1:1) | 미션 유형이 "팬퀘스트"인 경우. 이미 다른 미션에 매핑된 팬퀘스트는 드롭다운 후보에서 제외. 팬퀘스트 상세에서는 본 매핑을 역참조해 "사용 중인 에피소드" 패널로 노출 |
| BIVE 민팅 | 미션의 BIVE 보상 토글 | 미션의 BIVE 보상이 켜져 있을 때 추첨·완료 확정 시 자동 민팅 큐 적재 |
| 예측 마켓(PM) | 미션 유형이 "예측 마켓"일 때 | 완료 판정·완료 기준값·소스 참조값을 미션이 직접 흡수 (별도 모달 없음) |
| 서바이벌 트리비아(ST) | 미션 유형이 "서바이벌 트리비아"일 때 | 위와 동일 |
| 회원 통계 | (조회 전용) | 에피소드 단위 "진행 회원·완료 누적·검수 대기" 카운트 |
| 활동 로그 | [CEB-BO-000] §6 표준 양식 (3년 보관) | 그룹·에피소드·미션의 생성·수정·게시·종료·삭제 모두 기록 |

## 8. 활동 로그

본 영역의 운영자 액션은 모두 활동 로그에 자동 기록된다. 자세한 정책은 [CEB-BO-000] §6 참조.

| 분류 | 메시지 형식 |
|---|---|
| 그룹 생성 | 에피소드 그룹 '{타이틀}'을(를) 생성했습니다. (아티스트: {아티스트}) |
| 그룹 수정 | 에피소드 그룹 '{타이틀}'의 정보를 수정했습니다. (변경 항목: {필드 목록}) |
| 그룹 진행중 전환 | 에피소드 그룹 '{타이틀}'을(를) 진행중으로 전환했습니다. |
| 그룹 종료 전환 | 에피소드 그룹 '{타이틀}'을(를) 종료했습니다. (하위 에피소드 {N}건 동시 종료) |
| 그룹 삭제 | 에피소드 그룹 '{타이틀}'을(를) 삭제했습니다. (임시저장 상태만 가능) |
| 에피소드 생성 | 그룹 '{그룹명}'에 에피소드 '{타이틀}'을(를) 추가했습니다. (유형: {메인/반복}) |
| 에피소드 수정 | 에피소드 '{타이틀}'의 정보를 수정했습니다. (변경 항목: {필드 목록}) |
| 에피소드 삭제 | 에피소드 '{타이틀}'을(를) 삭제했습니다. (하위 미션 {N}건 동시 삭제, 팬퀘스트 매핑 자동 해제) |
| 미션 추가 | 에피소드 '{에피소드명}'에 미션 '{타이틀}'을(를) 추가했습니다. (유형: {팬퀘스트/예측마켓/서바이벌트리비아}, 반복: {단일/반복}, 반복 주기: {일간/주간/월간/—}) |
| 미션 수정 | 미션 '{타이틀}'의 정보를 수정했습니다. (변경 항목: {필드 목록}) |
| 미션 순서 변경 | 에피소드 '{에피소드명}'의 미션 순서를 변경했습니다. ('{타이틀}' {기존 순서} → {신규 순서}) |
