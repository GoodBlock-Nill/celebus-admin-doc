## 2. UI 설명

### 2.1 상태 대시보드 (4카드)

| 카드 | 의미 |
| --- | --- |
| Pending | 처리 대기 (우선) |
| Approved | 신고 인정 누적 |
| Rejected | 반려 누적 |
| Archived | 30일 경과 보관 |

### 2.2 검색·필터

| 구분 | 요소 | 설명 |
| ---- | ---- | ---- |
| 필터 | 상태 | 전체 / Pending / Approved / Rejected / Archived |
| 필터 | 신고 사유 | 6종 코드 ([CEB-BO-MEM-000] §4.2) |
| 필터 | 기간 | Date Picker |
| 입력 | 신고 대상 닉네임 | 부분 일치 |
| 버튼 | 초기화 | |

### 2.3 목록 테이블

| 컬럼 | 설명 |
| --- | --- |
| 상태 | Badge |
| 신고 사유 | 6종 코드 매핑 (KO 표시) |
| 신고 대상 게시물 | 제목 일부 발췌 |
| 작성자 | 닉네임 |
| 신고자 | 닉네임 |
| 신고 일시 | KST |
| 신고 누적 | "이 게시물에 누적된 신고 건수" (3건+ 시 빨간 강조) |

- Row 클릭 → MEM-201 상세

> 시각적 스타일은 피그마 참조

## 3. 권한·로직

### 3.1 권한별 분기

| 권한 | 진입 | 조회 | Row 클릭 |
| ---- | ---- | ---- | -------- |
| SUPER / OPERATOR / POLICY | ✅ | ✅ | ✅ |
| EXTERNAL_ARTIST | ❌ | — | — |

### 3.2 정렬

| 기본 | Pending 우선 → 신고 누적 많은 순 → 신고 일시 오래된 순 |

### 3.3 누적 신고 강조

동일 게시물에 신고 3건 이상 → 빨간 강조 + 카드 상단에 "누적 신고 게시물" 추가 카운트.
