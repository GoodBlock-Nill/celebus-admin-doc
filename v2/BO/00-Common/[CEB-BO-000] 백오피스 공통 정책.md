## 목적

이 문서는 CELEBUS v2 시스템 백오피스(BO) 전반에 걸쳐 반복 등장하는 **공통 정책, 규칙, 규격**을 한 곳에 정의한다. 화면 명세 작성 전에 BO 운영 표준을 먼저 확정하여, 모든 BO 화면이 일관된 권한·다국어·로그·UI 상태·검수 워크플로우를 따르게 한다.

> 각 BO 화면 스펙 문서는 **화면 고유 로직만** 기술하고, 공통 정책은 본 문서를 참조한다.
> 공통 정책 변경 시 본 문서만 수정한다.

> v2 앱 공통 정책 [CEB-000]과 별도지만 호환되는 BO 정책 체계이다. 사용자 노출 정책(다국어 라벨·토스트·UI 상태)은 [CEB-000]과 동일/하위 호환 유지.

## 2. 화면 ID 체계

### 2.1 패턴

```
CEB-BO-{서비스}-{번호}[-{유형}]
```

| 구성 | 설명 | 예시 |
| --- | --- | --- |
| `CEB-BO-` | 백오피스 고정 접두어 | — |
| `{서비스}` | 8개 영역 약자 (FQ·DUK·EVT·SUP·INF·MEM·COL·HOM) | `FQ` |
| `{번호}` | 영역 내 화면 번호 (3자리) | `101` |
| `{유형}` | (선택) 화면 유형 — `Create` / `Edit` / `MD-{action}` | `MD-DELETE` |

**전체 예시**:
- `CEB-BO-FQ-101` — Quest 메인 (List)
- `CEB-BO-FQ-101-Create` — Quest 생성
- `CEB-BO-FQ-201-MD-PUBLISH` — Quest 게시 모달
- `CEB-BO-DUK-101` — 덕력 시즌 설정
- `CEB-BO-MEM-101` — 기억저장소 신고 관리

### 2.2 번호 분배 규칙

| 번호대 | 용도 |
| --- | --- |
| `100` | 메인 List (서비스 진입 화면) |
| `101~199` | 메인 List 하위 화면 (Create·List 변형 등) |
| `201~299` | Detail (탭별 또는 단일) |
| `301~399` | 부속 관리 화면 (예: 반려사유 설정·이력) |
| `401~499` | 내역 조회 (제출 이력·변동 내역 등) |
| `501~599` | 정책 설정 (시즌·캡·곡선 등) |
| `601~699` | 외부 연동·API 관리 (필요 시) |
| `MD-{action}` | Modal — 부모 화면 ID 뒤에 붙임 |

### 2.3 v1 시스템과의 관계

| 항목 | v1 (BO-GAM·BO-FQ) | v2 (BO) |
| --- | --- | --- |
| 화면 ID | `CEB-BO-GZ-*` (게임존), `CEB-BO-FQ-*` (팬퀘스트) | `CEB-BO-{서비스}-*` (8개 영역) |
| 운영 | v1 그대로 유지 (게임존 운영 지속) | v2 신규 영역만 적용 |
| 마이그레이션 | 단계적 — v1·v2 병행 운영. 통합 시점은 별도 결정 | — |

> v1·v2 화면 ID는 서로 다른 네임스페이스이므로 충돌 없음. 동일 영역 약자(`FQ`)이지만 화면 번호 체계가 분리되어 있어 호환 가능.

## 3. 권한 체계 (4단계)

v2 BO는 **4단계 권한 체계**를 적용한다.

### 3.1 권한 정의

| 권한 | 한국어 | 역할 | 인증 |
| --- | --- | --- | --- |
| `SUPER` | **슈퍼관리자** | CELEBUS 본사 최상위. 모든 영역 R/W/D + 권한 부여·회수 | 이메일+비밀번호+2FA + IP 화이트리스트 |
| `OPERATOR` | **운영자** | 일상 운영. Quest·콘텐츠 검수·승인·반려·결과 입력·배송 처리 | 이메일+비밀번호+2FA |
| `POLICY` | **정책관리자** | 기획·마케팅 정책. 덕력 캡·티어 보상·시즌 곡선·팬덤 레벨 곡선 R/W | 이메일+비밀번호+2FA |
| `EXTERNAL_ARTIST` | **아티스트사 외부 권한** | 외부 협력사 (예: IX Entertainment). **자기 아티스트 한정** 콘텐츠·소식·디지털 굿즈 등록 R/W. 검수 권한 없음 | 이메일+비밀번호+2FA + 아티스트 ID 매핑 |

### 3.2 권한 부여·회수

| 항목 | 정책 |
| --- | --- |
| 부여 | 슈퍼관리자만 가능 |
| 회수 | 슈퍼관리자만 가능. 회수 즉시 세션 강제 종료 |
| 권한 변경 | 슈퍼관리자만 가능. 변경 시 활동 로그 자동 기록 |
| 외부 권한 만료 | 아티스트사 권한자는 **만료일 필수 설정** (기본 6개월, 갱신 가능) |
| 비활성 자동 잠금 | 90일 미접속 시 자동 비활성화 — 슈퍼관리자 재승인 후 재활성화 |

### 3.3 IP·접속 제한

| 권한 | IP 제한 | 시간대 제한 |
| --- | --- | --- |
| SUPER | 화이트리스트 필수 | 제한 없음 |
| OPERATOR | 권장 (옵션) | 제한 없음 (24/7 운영 가능) |
| POLICY | 권장 (옵션) | 제한 없음 |
| EXTERNAL_ARTIST | 화이트리스트 필수 | 평일 09:00~22:00 KST (협의 후 조정 가능) |

## 5. 다국어 입력 정책

### 5.1 KO/EN/JP 3언어 동시 필수

**모든 사용자 노출 텍스트**(타이틀·설명·라벨·CTA·안내 문구)는 KO/EN/JP 3개 언어 모두 입력해야 한다.

| 항목 | 정책 |
| --- | --- |
| 필수 입력 | KO·EN·JP **3개 모두** |
| 빈 칸 검증 | 1개 이상 빈 칸 → 저장 차단 + 토스트(Error) "모든 언어 필드를 입력해주세요" |
| 저장 시점 | 임시저장(Draft)도 3언어 모두 입력 권장. 단, **임시저장은 1개 언어만 입력해도 허용** (게시 시점에 3언어 강제 검증) |
| 게시 차단 | 게시(Publish) 시 3언어 빈 칸 검증 → 미충족 시 게시 불가 |
| 글자수 제한 | 필드별로 화면 명세서에 명시 (예: 타이틀 50자, 설명 500자) — 모든 언어 동일 적용 |

### 5.2 글자수 제한 필드 기준

다국어 입력 필드의 **표준 글자수 제한**. 화면 명세 작성 시 본 표를 기준으로 일관 적용. 모든 언어(KO/EN/JP) 동일 적용.

| 필드 유형 | 글자수 제한 | 비고 |
| --- | --- | --- |
| **타이틀** (Quest·Raffle·이벤트·소식 등) | **50자** | 카드·리스트 노출용 짧은 제목 |
| **상세설명** | **500자** | 본문 설명. 줄바꿈 포함 |
| **결과·정답 텍스트** (PM 결과·SETLIST 정답 등) | **50자** | 단답형 |
| **반려사유 텍스트** (운영자 추가 메시지) | **200자** | 코드 외 자유 입력 보충 |
| **카드 라벨·뱃지명** | **20자** | 짧은 표기 |
| **CTA 버튼 텍스트** | **15자** | 행동 유도 짧은 문구 |
| **링크 텍스트** | **30자** | 링크 표시 텍스트 |
| **알림·푸시 메시지** | **80자** | 모바일 알림 절단 방지 |
| **약관·안내문** | **2,000자** | 긴 본문. 줄바꿈·단락 포함 |
| **운영자 메모** (내부 관리용) | **500자** | KO만, 비공개 |

**화면 명세에서 다른 글자수가 필요하면**: 해당 화면 스펙 문서 §UI 구성 표에 명시. 본 표는 기본값.

**검증 정책**:
- 글자수 초과 입력 차단 + 입력 필드 카운터 빨간색 표시 (`{현재}/{제한}`)
- 글자수 90% 도달 시 카운터 주황색 경고
- 줄바꿈도 1자로 카운트
- 이모지·특수문자도 1자로 카운트 (multi-byte 무관)

### 5.4 다국어 라벨 일관성

- KO/EN/JP 라벨은 [CEB-000] §1 다국어 라벨 매핑을 따름 (덕력=Fan Power=推し力 등 공식 채택 어휘).
- BO에서 신규 라벨 추가 시 [CEB-000]에도 동시 등록.
- 모순 발생 시 **앱 [CEB-000] 정책 우선**.

## 10. 검수·승인 워크플로우

### 10.1 Quest 제출 검수 (대표 흐름)

```
유저 제출 (SUBMITTED)
    ↓
[CEB-BO-FQ-202] 검수 대기 리스트
    ↓
운영자 검토
    ↓
승인 (APPROVED) ─→ 보상 자동 지급 (응모권·덕력·디지털 포토카드)
    또는
반려 (REJECTED) ─→ 반려사유 코드 + 텍스트 입력 → 유저 알림
```

| 항목 | 정책 |
| --- | --- |
| 검수 SLA | 평일 09:30~18:30 KST 내 24시간 처리 목표 |
| 유저 안내 | 제출 직후 "하루 안에 결과를 알려드릴게요" 안내 |
| 검수자 권한 | OPERATOR 이상 |
| 동시 검수 방지 | 한 운영자가 검수 중인 항목은 다른 운영자에게 잠금 (5분 idle 후 자동 해제) |
| 반려사유 코드 | `[CEB-BO-FQ-101-REJECTSET]` 별도 화면에서 관리 |
| 재제출 | 반려 후 유저 재제출 가능 (제한 횟수: 별도 정책) |

### 10.2 콘텐츠 등록 (소식·디지털 굿즈)

```
EXTERNAL_ARTIST 등록 (DRAFT)
    ↓
운영자(OPERATOR) 검수
    ↓
승인 (PUBLISHED) ─→ 앱 노출 시작
    또는
반려 (REJECTED) ─→ 사유 → 외부 권한자 알림
```

> EXTERNAL_ARTIST가 직접 게시(PUBLISH)하는 경우는 **사전 승인 받은 카테고리에 한정**. 그 외는 운영자 검수 필수.

### 10.3 Raffle 추첨

```
응모 마감 (자동 상태 전환)
    ↓
[CEB-BO-FQ-206] 추첨 화면
    ↓
운영자 [추첨 실행] → BO 알고리즘 자동 추첨
    ↓
당첨자 명단 확정 → SNS 발표 + 당첨자 개별 안내
    ↓
배송 처리 (실물) 또는 자동 지급 (디지털)
```

### 10.4 서포트 이벤트 상태 전환

```
생성 (DRAFT)
    ↓
게시 (ACTIVE) — 응원 수집 중
    ↓
달성 (ACHIEVED) — 자동 판정. 운영자 [집행중] 수동 전환
    또는
미달성 (FAILED) — 자동 종료 + 덕력 자동 반환 (3회 재시도)
    ↓
결과물 등록 → 종료 (CLOSED)
```

### 10.5 반려사유 코드 체계

검수·반려 처리 시 사용하는 **표준 반려사유 코드**. `[CEB-BO-FQ-101-REJECTSET]` 별도 화면에서 관리.

#### 표준 반려사유 코드 (전 영역 공통)

| 코드 | 사유 (KO) | EN | JP | 비고 |

#### 검수 품질 관리

- **샘플링 감사**: SUPER가 OPERATOR 검수 결과를 무작위 샘플링 감사. 잘못 승인·반려 발견 시 시정 + 활동 로그 기록.
- **반려율 모니터링**: OPERATOR별 반려율 통계. 비정상적으로 높거나 낮으면 검토.
- **유저 이의 제기**: 반려 후 유저가 이의 제기 가능 (CS 채널). SUPER가 재검토 후 결정.

## 14. 회원 제재 정책

### 14.2 제재 단계

| 제재 | 효과 | 권한 |
| --- | --- | --- |
| **경고** (Warning) | 안내 메시지 발송. 제재 이력에만 기록. 서비스 이용 가능 | OPERATOR |
| **부분 정지** (Partial Restriction) | 특정 기능만 차단 (예: 기억저장소 작성 금지, 댓글 금지) | OPERATOR |

| **영구 정지** (Permanent Ban) | 로그인 불가. 모든 활동 차단. 보유 자산 소멸 | SUPER만 |
| **당첨 취소** | 진행 중·완료된 당첨 권리 회수 | SUPER만 |

### 14.3 제재 처리 절차

```
위반 적발 (자동 탐지·신고·운영자 인지)
    ↓
[CEB-BO-USR-201] 회원 상세 → [제재 처리] 버튼
    ↓
모달: 위반 코드 선택 + 제재 단계 선택 + 사유 텍스트 (필수, 200자)
    ↓
저장 → 즉시 적용 + 회원에게 알림 + 활동 로그 기록
    ↓
회원이 이의 제기 시: SUPER 재검토 → 제재 해제 또는 유지 결정
```

### 14.4 제재 해제·복구

| 항목 | 정책 |
| --- | --- |
| 자동 해제 | 임시 정지는 기간 만료 시 자동 해제 |
| 수동 해제 | SUPER가 [제재 해제] 버튼으로 즉시 해제. 사유 필수 |
| 영구 정지 해제 | SUPER만 가능. 매우 신중하게 — 활동 로그에 해제 사유 상세 기록 |
| 당첨 권리 복구 | 잘못된 제재로 당첨 취소된 경우 SUPER가 수동 복구 |
| 보유 자산 복구 | 영구 정지 시 소멸된 응모권·덕력은 해제 시 자동 복구되지 않음 (별도 운영 결정) |

### 14.5 제재 이력 조회

- 모든 제재는 회원 상세 화면(CEB-BO-USR-201)의 [제재 이력] 탭에서 조회
- SUPER는 전체 회원 제재 이력 통합 조회 가능 (CEB-BO-USR-301)
- 활동 로그와 별도로 제재 이력 전용 보관 (3년)

## 15. 외부 지갑·정산 정책

CELB ↔ GP 교환·외부 지갑 관리 정책. v1 BO-GAM/[CEB-BO-GZ-501·502·503] 패턴 차용.

### 15.1 지갑 유형

| 유형 | 용도 | 비고 |
| --- | --- | --- |
| **충전용 지갑** | 사용자 → CELB 입금 받음 (GP 전환의 입구) | 1개 이상 등록 가능 |
| **출금용 지갑** | 사용자에게 CELB 송금 (GP → CELB 출금) | 1개 이상 등록 가능 |
| **대표 지갑** | 각 유형별 1개. 신규 거래 기본 사용 | 변경 가능, 활동 로그 기록 |

### 15.2 지갑 등록·관리

| 항목 | 정책 |
| --- | --- |
| 등록 권한 | SUPER만 가능 |
| 입력 정보 | 이름·유형(충전/출금)·주소·Private Key (암호화 저장) |
| Private Key 저장 | **암호화 저장 + DB 분리 보관**. BO 화면에서는 마스킹 (`0x...****...`) |
| Private Key 조회 | SUPER만 가능. 조회 시 2FA 추가 인증 + 활동 로그 기록 |
| 잔액 표시 | 실시간 BNB·CELB 잔고 표시. 5분마다 자동 갱신 |
| 비활성화 | 거래 중단 시 비활성화 가능. 잔액 0 또는 다른 지갑으로 이체 후 |

### 15.3 잔고 경고

| 시나리오 | 경고 |
| --- | --- |
| BNB 잔고 < 0.01 BNB | 빨간 경고 + 슈퍼관리자에게 푸시 알림 (가스비 부족) |
| 출금용 CELB 잔고 < 일일 평균 출금량의 3배 | 주황 경고 + 운영자 노티 |
| 충전용 지갑 폭증 (시간당 거래 수 평균 ×3 초과) | 의심 거래 알림 |

### 15.4 교환 정책

| 항목 | 기본 정책 | 조정 권한 |
| --- | --- | --- |
| 교환 비율 (CELB↔GP) | 1 CELB = 1,000 GP (예시) | POLICY 변경 가능 |
| 일일 출금 한도 (1인당) | 100,000 GP 상당 | POLICY 변경 가능 |
| 주간 출금 한도 (1인당) | 500,000 GP 상당 | POLICY 변경 가능 |
| 최소 출금 단위 | 10,000 GP | POLICY 변경 가능 |
| 출금 수수료 | 1% (가스비 별도) | POLICY 변경 가능 |
| 교환 검증 | KYC 인증 회원만 출금 가능 | 정책 고정 |

### 15.5 정산 워크플로우

```
사용자 출금 요청 (앱)
    ↓
BO 자동 검증 (잔고·한도·KYC)
    ↓
검증 통과 → 출금 큐에 적재
    ↓
일괄 출금 실행 (일 1회·UTC 02:00 기준 / SUPER 수동 트리거 가능)
    ↓
트랜잭션 발송 → 블록 컨펌 대기
    ↓
완료 → 사용자 알림 + 활동 로그
```

### 15.6 변동 내역 조회

- CEB-BO-GZ-601 패턴 차용. 모든 GP·CELB 거래 내역 시계열 조회.
- 필터: 회원·유형·기간·금액 범위
- 내보내기: CSV (SUPER 전용, 감사 대비)

## 16. BO 푸시·알림 정책

BO에서 회원에게 발송하는 푸시·앱 알림 운영 정책.

### 16.1 발송 권한

| 발송 유형 | 권한 | 비고 |
| --- | --- | --- |
| 자동 푸시 (시스템 트리거) | 자동 | 검수 승인·당첨·보상 지급 등 — 시스템이 자동 발송 |
| 수동 푸시 (캠페인) | SUPER + POLICY | 캠페인 안내·이벤트 알림 등 — 운영자 수동 발송 |
| 긴급 공지 푸시 | SUPER만 | 점검·장애·보안 등 |

OPERATOR는 **자동 푸시만** (검수 승인 시 자동 발송). 임의 캠페인 푸시 발송 권한 없음.

### 16.2 푸시 메시지 템플릿

| 시나리오 | 템플릿 (KO) | 영역 |
| --- | --- | --- |
| Quest 검수 승인 | "Quest '{타이틀}' 승인! 응모권 {N}장 + 디지털 포토카드 1장이 지급됐어요 🎁" | FQ |
| Quest 검수 반려 | "Quest 제출이 반려됐어요. 사유: {반려사유 메시지}. 다시 제출해보세요!" | FQ |
| Raffle 당첨 | "🎉 {Raffle 명} 당첨을 축하드려요! 자세한 안내는 메일을 확인하세요" | FQ |
| 덕력 적립 | "덕력 +{N}DUK 획득! 현재 잔액: {잔액}DUK" | DUK |
| 팬덤 레벨업 | "🎊 {아티스트명} 팬덤이 Lv.{N}에 도달했어요! 보상을 확인하세요" | EVT |
| 서포트 이벤트 달성 | "🎉 '{이벤트명}' 응원 목표 달성! 결과물이 곧 공개돼요" | SUP |
| 응모권 만료 안내 | "보유 응모권 {N}장이 곧 만료돼요. ({만료일}까지)" | FQ |

각 템플릿은 KO/EN/JP 3언어 동시 작성 (`§5` 적용). 변수 `{N}·{타이틀}` 등은 발송 시점에 치환.

### 16.3 수동 푸시 발송 절차

```
[CEB-BO-PUSH-101] 푸시 발송 화면
    ↓
대상 선정 (전체·아티스트별·등급별·기간 활동·태그 등)
    ↓
메시지 입력 (KO/EN/JP 3언어 + 클릭 시 이동 링크)
    ↓
미리보기 (모바일 화면 시뮬레이션)
    ↓
발송 시점 (즉시 발송 / 예약 발송)
    ↓
[발송] → 2차 확인 모달 → 최종 발송
    ↓
발송 결과 자동 집계 (수신·표시·클릭 수)
```

### 16.4 발송 한도·SPAM 방지

| 항목 | 정책 |
| --- | --- |
| 회원당 푸시 한도 | 1일 최대 5회 (시스템 자동 + 캠페인 합산) |
| 캠페인 푸시 한도 | 1일 1회 (전체 발송 기준) |
| 야간 발송 차단 | 22:00~08:00 KST 발송 차단 (긴급 공지 제외) |
| 수신 거부 | 회원이 푸시 알림 끄기 가능 (시스템 알림은 강제 발송) |
| 다국어 매칭 | 회원 앱 언어 설정에 따라 자동 매칭 발송 |

### 16.5 발송 로그

- 모든 푸시 발송은 로그 보관 (3년)
- 로그 항목: 발송 시각·발송자(시스템 또는 운영자 이메일)·대상 수·수신 성공/실패·클릭률
- SUPER는 전체 발송 로그 조회 가능 (CEB-BO-PUSH-401)

### 16.6 발송 실패 재시도

| 실패 유형 | 재시도 |
| --- | --- |
| 일시적 네트워크 오류 | 자동 재시도 (3회, 지수 백오프) |
| 토큰 만료 | 자동 토큰 갱신 후 재시도 |
| 회원 비활성·탈퇴 | 재시도 안 함. 로그에 "수신 불가" 기록 |
| FCM/APNs 오류 | 운영자 알림 + SUPER 검토 |

## 17. BO 대시보드·진입 화면

운영자가 BO 로그인 후 처음 보는 화면. 권한별로 다른 KPI·할 일·알림이 표시.

### 17.1 공통 상단 영역

| 영역 | 내용 |
| --- | --- |
| 헤더 | 로고·로그인 사용자명·권한·로그아웃 |
| 알림 벨 | 신규 검수 대기·시스템 경고·SUPER 메시지 등 카운트 |
| 글로벌 검색 | 회원·Quest·Raffle·콘텐츠 통합 검색 |

### 17.2 권한별 대시보드 구성

#### SUPER 대시보드

- **시스템 KPI**: 전체 회원 수·MAU·신규 가입·활동 로그 이상 징후
- **운영 현황**: 검수 대기·반려율·SLA 초과 항목·OPERATOR별 부하
- **재무**: 일일 거래량·지갑 잔고·BNB 가스비
- **할 일**: SUPER 결재 필요 항목 (영구 정지·정책 변경·환불 등)
- **알림**: 시스템 장애·보안 경고·이상 거래

#### OPERATOR 대시보드

- **검수 큐**: 대기 중 항목 수 (영역별·우선순위별)
- **내가 잠금 중인 항목**: idle 시간 표시
- **오늘의 처리량**: 본인이 승인·반려한 건수
- **할 일**: 마감 임박 검수·재제출·인계 받은 항목
- **공지**: SUPER 공지·팀 메시지

#### POLICY 대시보드

- **정책 현황**: 현재 시즌·캡·곡선 요약
- **이번 달 변경 이력**: 정책 변경 사항·영향 범위
- **KPI 분석**: 덕력 발급량·팬덤 레벨업 추이·서포트 이벤트 성과
- **할 일**: 정책 검토 일정·예약된 변경

#### EXTERNAL_ARTIST 대시보드

- **자기 아티스트 현황**: 콘텐츠 노출 수·반응·팬덤 활동
- **등록한 콘텐츠**: 검수 대기·승인·반려 상태
- **이번 주 성과**: 신규 게시물·반응·팬덤 레벨업
- **할 일**: 반려된 콘텐츠 재등록·일정 등록

### 17.3 KPI 위젯 표준

| 위젯 | 데이터 갱신 | 클릭 동작 |
| --- | --- | --- |
| 숫자 카드 | 실시간 (5분 캐시) | 상세 화면 이동 |
| 그래프 (선·막대·도넛) | 1시간 캐시 | 기간 변경 가능 |
| 리스트 (할 일·최근 활동) | 실시간 | 항목 클릭 → 상세 |

### 17.4 사용자 정의

- 운영자는 위젯 순서 드래그로 변경 가능 (개인 설정 저장)

## 18. 미디어 업로드 규격

BO에서 등록하는 이미지·영상·파일의 표준 규격.

### 18.1 이미지 업로드

| 항목 | 정책 |
| --- | --- |
| 지원 포맷 | JPG / PNG / WebP / SVG (벡터) |
| 최대 용량 | 10 MB (배너·포스터) / 5 MB (일반) / 2 MB (썸네일·아이콘) |
| 권장 해상도 | 화면 명세서별 명시. 일반: 1080×1080 (정사각) / 1080×1920 (세로) / 1920×1080 (가로) |
| 자동 리사이징 | 업로드 시 썸네일 자동 생성 (300×300, 600×600) |
| 자동 압축 | JPG 품질 85, PNG 무손실 최적화 |
| 알파 채널 | PNG·WebP·SVG는 투명 배경 지원 |
| 메타데이터 | EXIF 자동 제거 (개인정보 보호) |

### 18.2 영상 업로드

| 항목 | 정책 |
| --- | --- |
| 지원 포맷 | MP4 (H.264 + AAC) / MOV |
| 최대 용량 | 100 MB (홍보영상·릴스용) |
| 최대 길이 | 60초 (BO 직접 업로드). 더 긴 영상은 외부 CDN URL 등록 권장 |
| 권장 해상도 | 1080×1920 (세로 9:16) / 1920×1080 (가로 16:9) |
| 자동 썸네일 | 영상 첫 프레임 자동 캡처 |
| 자동 트랜스코딩 | 다양한 해상도 변환 (480p·720p·1080p) |

### 18.3 파일 업로드 (문서·CSV 등)

| 항목 | 정책 |
| --- | --- |
| 지원 포맷 | PDF·CSV·XLSX·TXT·JSON |
| 최대 용량 | 50 MB |
| 보안 검사 | 업로드 시 자동 바이러스 스캔 |
| 보관 기간 | 영구 (또는 운영자 수동 삭제까지) |

### 18.4 업로드 동작

| 시나리오 | 처리 |
| --- | --- |
| 업로드 중 | 진행률 바 + 취소 버튼 |
| 업로드 실패 | 토스트(Error) "업로드 실패. 다시 시도하세요" + 자동 재시도 1회 |
| 용량 초과 | 토스트(Error) "최대 {N}MB까지 가능합니다" + 업로드 차단 |
| 포맷 불일치 | 토스트(Error) "지원하지 않는 파일 형식입니다 ({지원 목록})" |
| 보안 검사 실패 | 토스트(Error) "보안 위험이 감지됐습니다. 운영진에 문의하세요" + 운영진 자동 알림 |
| 중복 업로드 | "동일한 파일이 이미 등록돼 있어요. 그래도 등록할까요?" 확인 모달 |

### 18.5 미디어 자산 관리

- **공통 자산 라이브러리**: 자주 사용하는 이미지·로고는 공통 자산으로 등록 (CEB-BO-MEDIA-101)
- **태그 분류**: 아티스트·캠페인·등급별 태그
- **사용 추적**: 자산이 사용된 화면 목록 자동 추적
- **삭제 차단**: 사용 중 자산 삭제 차단 (강제 삭제 시 SUPER 권한 + 영향 범위 경고)
