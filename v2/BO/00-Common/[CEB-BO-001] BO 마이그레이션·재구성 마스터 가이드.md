## 0. 목적

기존 운영 중인 v1 백오피스(BO-GAM·BO-FQ·회원·아티스트·BIVE·관리자)를 **v2 고도화 BO 체계로 흡수·재구성**한다. 본 문서는 작업 전체의 마스터 가이드로:

1. **현황 분석** — 기존 BO 영역과 v2 BO-000 정의의 갭
2. **마이그레이션 전략** — v1 내용을 v2/BO/로 단계적 이전 (deprecation 태그)
3. **v2 통합 영역 매핑** — 8개 영역 → 12개 영역으로 확장
4. **작업 우선순위·일정**
5. **싱크 정책** — 누락 없이 모든 문서 일관성 유지

> **사용자 결정 사항** (1·2·3차 인터뷰):
> - 마이그레이션: v1 흡수 → v2 통합 재구성 (권장)
> - BIVE: NFT/온체인 실제 발행 + Quest/Raffle 보상 연결
> - 우선순위: 기존 영역 v2 호환 검토 먼저 (회원·아티스트·BIVE·관리자)
> - 회원 상세: 기존 4탭 + v2 신규 4탭 = **8탭**
> - 관리자 권한: **하이브리드** (4단계 기본 + 그룹 커스터마이징)
> - v1 폴더: **v2/BO/로 세계 이전** (deprecation 태그)
> - 산출물 구조: **메인 가이드 + 영역별 조각 문서**

## 3. 마이그레이션 전략

### 3.1 v1 → v2/BO/ 세계 이전 (Deprecation 모델)

```
[현재] v1 폴더                 [목표] v2/BO/ 통합
BO-GAM/      ─┐
BO-FQ/        ├──→  v2/BO/ (12개 영역)
(BO-USR/)     │     ├── 화면별 .md 명세
(BO-ART/)     │     ├── confluence-html/ 변환본
(BO-BIVE/)    │     └── 영역별 가이드 (CEB-BO-002~005)
(BO-SYS/)    ─┘
                        + v1 폴더 deprecation 태그
                          (Page Properties에 "deprecated 2026.MM.DD")
```

### 3.2 단계별 이전 절차

각 화면 명세서별로 다음 4단계를 거친다.

| 단계 | 작업 | 산출물 |
| --- | --- | --- |
| **1. 검토** | v1 화면 명세서 읽기 + v2 정책(BO-000) 호환성 점검 | 갭 노트 |
| **2. 재구성** | v2/BO/에 신규 명세서 작성 (v2 정책 적용) | `v2/BO/[CEB-BO-{영역}-{번호}] {제목}.md` |
| **3. Deprecate** | v1 원본 Page Properties에 `상태: deprecated (이전: v2/BO/...)` 추가 | v1 원본 (수정 후) |
| **4. HTML 변환** | v2/convert-to-html.js에 매핑 추가 → 실행 | `v2/confluence-html/BO/...html` |

### 3.3 V1 폴더 deprecation 태그 표준

기존 v1 화면 명세서 상단 Page Properties에 다음 행 추가:

```markdown
| 상태 | **deprecated 2026.MM.DD** (이전: `v2/BO/[CEB-BO-{영역}-{번호}] {제목}.md`) |
```

v1 원본은 **삭제하지 않음** (히스토리 보존). 신규 작업·검색은 v2/BO/로 진행.

## 8. 싱크 정책 — 누락 없이 모든 문서 일관성 유지

### 8.1 문서 간 동기화 원칙

| 변경 발생 위치 | 동기화 대상 |
| --- | --- |
| **v2 정책 변경** ([CEB-000] 또는 [CEB-BO-000]) | ① 영향 받는 화면 명세서 changelog 동기화 ② confluence-html 재변환 |
| **화면 명세서 변경** (예: CEB-BO-USR-201) | ① 해당 영역 가이드 (CEB-BO-002) 변경 이력 ② HTML 재변환 |
| **새 화면 추가** | ① 영역 가이드의 화면 목록 ② 본 마스터 가이드 §5 화면 ID 표 ③ HTML 매핑 |
| **이전(deprecation)** | ① v1 원본 Page Properties 태그 추가 ② v2/BO/ 신규 명세 ③ 본 마스터 가이드 §1.1 표 |

### 8.2 변경 이력 (changelog) 표준

모든 BO 명세서는 변경 이력 표를 가지며, 다음 패턴으로 작성:

```markdown
## N. 변경 이력

| 버전 | 일자 | 작성자 | 변경 내용 |
| ---- | ---- | ------ | --------- |
| v1.0 | YYYY.MM.DD | @작성자 | 초안 작성. 핵심 정책 ① ② ③ ... |
| v1.1 | YYYY.MM.DD | @작성자 | (변경 동기) — ① ② ③ |
```

영향 받는 다른 문서가 있을 경우 changelog 마지막에 명시:
> `동기화: [CEB-BO-000] v1.2, [CEB-BO-001] §5 화면 ID 표 갱신`

### 8.3 HTML 변환 매핑 정책

`v2/convert-to-html.js`에 BO 폴더의 모든 .md 파일을 매핑 추가. 신규 명세서 추가 시 매핑도 동시 추가 (체크리스트 §10).

### 8.4 검색·인덱스 유지

`v2/BO/`에 새 명세서 추가 시:
1. 본 마스터 가이드 §5 화면 ID 표 갱신
2. 해당 영역 가이드 (CEB-BO-002~005) 화면 목록 갱신
3. 기존 영역(FQ 등)의 영역 가이드도 동일

### 8.5 다국어 라벨 일관성

[CEB-000] §1 다국어 라벨 매핑이 단일 진실 원천(SSOT). BO 명세서에서 라벨 신규 추가 시 [CEB-000] 동시 등록 필수.
