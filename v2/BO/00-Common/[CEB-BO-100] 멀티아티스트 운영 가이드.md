## 목적

CELEBUS는 단일 아티스트(V01D) 운영에서 여러 아티스트를 동시 운영하는 멀티아티스트 구조로 확장한다. 본 가이드는 백오피스(BO) 전 영역에 적용되는 **멀티아티스트 운영 메탈 모델·UX 패턴·정책 분리 매트릭스·도입 로드맵**을 정의한다.

> 본 가이드는 [CEB-BO-000] 백오피스 공통 정책의 멀티아티스트 정책 확장이며, 모든 영역별 가이드(USR·ART·BIVE·FQ·EVT·DUK·SYS·RFT 등)는 본 가이드의 정책을 따른다.

## 1. APP vs BO 메탈 모델 차이

| 측면 | APP (`[CEB-000]` 적용) | BO (본 가이드 적용) |
| --- | --- | --- |
| 사용자 시점 | 한 명의 팬이 한 번에 한 아티스트를 본다 | 한 명의 운영자가 동시에 여러 아티스트를 비교·감독한다 |
| 컨텍스트 | 1개 (현재 선택 아티스트) | 0~N개 (전체 비교, 단일 집중 모두 필요) |
| 권한 모델 | 단일 사용자 | 4단계 (SUPER / OPERATOR / POLICY / EXTERNAL_ARTIST) |
| 콘텐츠 귀속 | "현재 선택 아티스트" 또는 "콘텐츠 아티스트" | 콘텐츠는 항상 명시적 아티스트 ID에 귀속, 운영 컨텍스트는 별도 |

**결론**: APP의 컨텍스트 패턴을 BO에 그대로 복제하면 운영자가 비교·감독을 못한다. BO는 **권한별로 다른 컨텍스트 메탈 모델**을 적용한다.

## 2. UX 패턴 — 하이브리드 컨텍스트

### 2.1 두 축의 결합

BO 멀티아티스트 운영은 두 축을 동시에 사용한다.

| 축 | 위치 | 역할 |
| --- | --- | --- |
| **헤더 글로벌 컨텍스트 토글** | 상단 헤더 (사이드바 옆) | 전체 BO의 **기본 운영 시야** 결정. "전체 / V01D / iKON / …" |
| **페이지별 필터** | 각 콘텐츠 리스트 상단 필터 행 | 헤더 컨텍스트 위에 **덮어쓰기**. 운영자가 페이지에서 일시적으로 다른 아티스트로 전환 가능 |

### 2.2 권한별 적용 룰

| 권한 | 헤더 토글 동작 | 페이지 필터 동작 |
| --- | --- | --- |
| SUPER / OPERATOR / POLICY | 기본값 "전체". 변경 가능. 토글 변경 시 페이지 필터 기본값이 따라 바뀜 (사용자가 페이지에서 덮어쓰기 가능) | "전체" + 모든 아티스트 옵션 노출 |
| EXTERNAL_ARTIST | 자기 아티스트로 **고정·비활성** (변경 불가) | 다른 아티스트 옵션 미노출. 서버 측에서 본인 아티스트 외 데이터 차단 |

### 2.3 화면 영향 매트릭스 (사이드바 메뉴별 4단계 분류)

본 매트릭스는 멀티아티스트 도입 시 **각 사이드바 메뉴의 영향도**를 분류한다.

| 등급 | 정의 | 메뉴 |
| --- | --- | --- |
| 🔴 **Critical** | 헤더 토글 + 페이지 필터 모두 적용. 콘텐츠가 명시적으로 아티스트에 귀속 | **팬퀘스트 / BIVE / 아티스트 / 모바일티켓 / 스토리 퀘스트(SQ)** ⭐ v1.5 |
| 🟠 **High** | 페이지 필터 또는 보조 탭으로 아티스트별 분리 필요 | **회원 상세 / Fans / 프로젝트 / 대시보드** |
| 🟡 **Info** | 메타필드·라벨·통계만 영향. 본질은 통합 운영 | **응모권(RFT) / 회원 리스트 / 권한관리 / 관리자 활동 로그** |
| ⚪ **None** | 멀티아티스트와 무관한 전역 운영 | **앱 / 게임존 / 재무 / 관리자 리스트 / 정지사유 (회원)** |

#### 등급별 적용 패턴

| 등급 | 헤더 토글 | 페이지 필터 | 정책 분리 | EXTERNAL_ARTIST |
| --- | --- | --- | --- | --- |
| 🔴 Critical | ✅ | ✅ (덮어쓰기) | 가이드별 (반려사유=아티스트별 등) | 자기 아티스트만 R/W |
| 🟠 High | ⚠️ 부분 | ✅ 또는 보조 탭 | 보통 전역 | 회원 영역은 차단, Fans·프로젝트는 자기 아티스트만 |
| 🟡 Info | ❌ | ❌ 또는 메타 표시 | 메타필드·통계 분기만 | 본인 출처 통계만 R |
| ⚪ None | ❌ | ❌ | 전역 단일 | 진입 차단 |

### 2.4 URL 동기화

- 헤더 컨텍스트 변경 시 URL에 `?artistContext={그룹ID}` 또는 `?artistContext=all` 자동 반영.
- 페이지 필터 변경 시 URL에 `?artistFilter={그룹ID|멤버ID}` 추가.
- 새로고침·공유 링크에서도 컨텍스트 유지.
- EXTERNAL_ARTIST가 자기 아티스트 외 ID로 직접 URL 접근 시 [CEB-BO-000] §7.2 권한 부족 표시.

### 2.5 도입 시점 (결정사항 ③ 적용)

- **Phase 1**: 헤더 토글 UI는 **placeholder만 노출** (비활성, 회색, 툴팁 "Phase 3에서 활성화"). SUPER 운영자가 미리 메탈 모델을 학습하도록 위치 확보.
- **Phase 2**: 페이지별 필터·정책 분리만 우선 동작.
- **Phase 3**: 헤더 토글 활성화 + URL 동기화 + 권한별 잠금.

### 3.1 아티스트별 정책 데이터 구조 (반려사유 전용)

반려사유 등 **아티스트별 분리 정책**은 다음 필드를 갖는다.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `artistId` | ID | 적용 대상 아티스트 그룹 ID (필수) |
| `code` | string | 내부 식별 코드 (영문, 등록 후 수정 불가) |
| `displayName` | string | 운영자 노출명 (다국어 KO/EN/JA 지원) |
| `description` | string | 적용 기준 |
| `status` | enum | `Active` / `Inactive` |

> 정지사유는 v1.1부터 2단 구조 폐기 → **전역 단일 + `discoveryContext` 메타** (정지 처리 시점에 어느 아티스트 활동 중 발견되었는지 기록)로 변경.

### 3.2 정지사유 변경 근거 (v1.0 → v1.1)

| 결과 | 폐기 | **채택** |

## 4. EXTERNAL_ARTIST 권한 적용 범위 (결정사항 ② 적용)

EXTERNAL_ARTIST는 **콘텐츠 영역에만 접근 가능**하다. 회원 데이터·재무·전역 정책 등은 차단.

### 4.1 접근 매트릭스

| 영역 | EXTERNAL_ARTIST 접근 | 비고 |
| --- | --- | --- |
| Quest·Raffle (자기 아티스트) | R/W (생성·수정) | 검수·삭제·추첨은 OPERATOR 이상 |
| BIVE 캠페인·등록 (자기 아티스트) | R/W | 등급 정의·민팅 정책은 SUPER 한정 |
| **모바일티켓 공연·팬미팅 (자기 아티스트)** ⭐ v1.1 | R/W (등록·수정) | 환불·취소는 OPERATOR 이상 |
| **스토리 퀘스트(SQ, 자기 아티스트)** ⭐ v1.5 | R/W (생성·챕터 추가·미션 정의) | 챕터 미션 검수(ADMIN_APPROVAL)는 OPERATOR 이상. 챕터 보상(BIVE 민팅·덕력)은 자기 아티스트 한정 |
| 콘텐츠·소식 (INF, 자기 아티스트) | R/W | 게시·공지·홈 배너 |
| 디지털 굿즈 (COL, 자기 아티스트) | R/W | |
| 서포트 이벤트 (자기 아티스트) | R | 생성·종료는 OPERATOR 이상 |
| **응모권 (RFT)** ⭐ v1.1 | R (자기 아티스트 출처 통계만) | 회원 보유·거래 내역 차단 |
| 회원 리스트·상세 | ❌ 차단 | 본인 아티스트 팬 회원이라도 접근 불가 |
| 회원 정지내역·자산 회수 | ❌ 차단 | |
| 팬덤 레벨 통계 (자기 아티스트) | R (집계만) | 개별 회원 식별 정보 미노출 |
| 게임존·재무·교환소 | ❌ 차단 | |
| 권한 관리·관리자 활동 로그 | ❌ 차단 | |

### 4.2 통계 화면의 비식별 처리

EXTERNAL_ARTIST가 자기 아티스트 팬덤 레벨 분포·응모권 발급 통계 등 통계 화면을 볼 때:

- 집계 수치(예: "Lv5 이상 회원 1,234명", "이번 달 자사 아티스트 출처 응모권 5,678장 발급")만 노출.
- 개별 회원 ID·닉네임·이메일·전화번호 등 식별 정보 모두 마스킹.
- 다운로드·CSV 내보내기 비활성화.

### 4.3 직접 URL 접근 시도

- EXTERNAL_ARTIST가 `/members/123` 등 차단 영역에 직접 URL로 접근 시 [CEB-BO-000] §7.2 권한 부족 표시 → "권한이 없습니다" + [홈으로].
- 자기 아티스트 외 콘텐츠 ID로 접근 시도 시 동일 처리.

## 5. 신규 아티스트 추가 정책 (결정사항 ④ 적용)

신규 아티스트 추가 시 **템플릿 아티스트로부터 기본 정책을 1-click 복제**한다.

### 5.1 템플릿 아티스트 개념

- "템플릿 아티스트"는 실제 운영되지 않는 가상 아티스트로, BO에 1개만 존재.
- 운영팀이 합의한 **표준 정책 묶음**(곡선·시즌·반려사유·BIVE 등급 매핑·기본 Quest 템플릿 등)을 보관.
- SUPER만 수정 가능. 일반 콘텐츠 등록·앱 노출 대상이 아님.
- ID: 예약값 (`group_id=0` 또는 별도 플래그 `is_template=true`).

### 5.2 복제 항목

신규 아티스트 추가 마법사에서 다음 항목을 자동 복제한다.

| 항목 | 출처 | 복제 후 편집 |
| --- | --- | --- |
| 팬덤 레벨 곡선 | 템플릿의 곡선 | 즉시 편집 가능 ([CEB-BO-EVT-501]) |
| 덕력 시즌 정책 | 템플릿의 시즌 캘린더 | 시즌 시작일만 조정 |
| 반려사유 마스터 | 템플릿의 반려사유 전체 | 추가·삭제·수정 모두 가능 |
| BIVE 등급 매핑 | 템플릿의 Event/Ticket/Mix/Pick 등급 정의 | 신규 아티스트에서는 일반적으로 그대로 유지 |
| 기본 Quest 템플릿 | 템플릿의 "환영 Quest", "SNS 팔로우 Quest" 등 | 즉시 게시 또는 수정 후 게시 |
| 응모권 발급 정책 (자기 아티스트 출처) ⭐ v1.1 | 템플릿의 출처별 발급 룰 | 즉시 편집 가능 |
| 정지사유 (전역) | 복제 안 함 (전역 단일 적용) | 신규 아티스트 추가와 무관 |

### 5.3 신규 아티스트 추가 워크플로우

상세 명세는 [CEB-BO-ART-501] 신규 아티스트 추가 체크리스트에서 정의. 본 가이드는 정책 원칙만 제시:

1. ART 영역에서 그룹 등록 → 신규 아티스트 추가 마법사 자동 트리거.
2. 마법사 1단계: 그룹 기본 정보 (이름·로고·이미지·SNS) 입력.
3. 마법사 2단계: 멤버 등록 (선택, 이후 별도 가능).
4. 마법사 3단계: **초기화 체크리스트** — 위 §5.2 항목 자동 복제 (체크박스로 항목별 옵트아웃 가능).
5. 마법사 4단계: 운영 가능 권한자 매핑 (EXTERNAL_ARTIST 권한자 연결).
6. 활성화 → 앱 노출 시작.

## 6. 도입 로드맵 (Phase 1~3)

### 6.1 Phase 1 — 기반 (hardcoded 제거 + 공통 컴포넌트 + 응모권 영역 신설)

1. `<ArtistSelector />` 공통 컴포넌트 — 그룹·멤버 2단계, 마스터에서 동적 로드, 권한별 옵션 필터.
2. 콘텐츠 생성 모달의 hardcoded 아티스트 dropdown 제거 → `<ArtistSelector />` 적용.
3. 콘텐츠 리스트 (Quest·BIVE Edition·BIVE Minting·모바일티켓) 아티스트 필터 추가.
4. 회원 상세 BIVE 탭·참여 내역 탭에 아티스트별 보조 탭 분리.
5. 헤더 글로벌 토글 placeholder 노출 (비활성).
6. mock·DB 데이터 정합성 — 모든 아티스트 필드를 그룹·멤버 ID 참조로 정규화.
7. **응모권(RFT) 영역 신설** ⭐ v1.1, MVP 스코프 v1.2 — 사이드바 BIVE 직후 메인 메뉴 추가. MVP 화면 3종(RFT-101/201/301)만 작성. RFT-401·RFT-501은 APP 고도화 완료 후 별도 트랙. [CEB-BO-FQ-301] 거래 내역 → [CEB-BO-RFT-201] 변동 내역으로 이전·확장.

### 6.2 Phase 2 — 정책 분리

1. 정지사유 화면에 `discoveryContext` 메타 컬럼 추가 ([CEB-BO-USR-501]). ⭐ v1.1 변경
2. 반려사유 마스터 신설 ([CEB-BO-FQ-501]).
3. 팬덤 레벨 곡선 페이지 신설 ([CEB-BO-EVT-501]).
4. 신규 아티스트 추가 마법사 ([CEB-BO-ART-501]).
5. 권한 관리에 역할별 운영 가능 아티스트 다중 선택 필드 ([CEB-BO-SYS-501]).
6. 콘텐츠 리스트 다중 선택 → 일괄 이관·복제 도구.
7. 관리자 활동 로그에 `artistContext` 컬럼 + 필터 ([CEB-BO-SYS-601]).
8. **응모권 발급 정책 마스터** ([CEB-BO-RFT-301]) — 출처별 발급 룰. ⭐ v1.1

### 6.3 Phase 3 — 고도화

1. 헤더 글로벌 토글 활성화 + URL 동기화 + 권한별 잠금.
2. EXTERNAL_ARTIST 진입 시 사이드바 메뉴 자동 숨김.
3. 팬덤 레벨 달성 → 독점 콘텐츠 자동 해금 룰 빌더 (아티스트별).
4. 아티스트 팔로우 해제·재팔로우 시 데이터 보존·복구 정책 콘솔.
5. 응모권 환율 차등 옵션 (아티스트별 프로모션 보너스).
6. 데이터 마이그레이션 정책 콘솔 ([CEB-BO-SYS-701]).
7. 대시보드에 아티스트별 KPI 비교 위젯 (헤더 컨텍스트=전체일 때만).
