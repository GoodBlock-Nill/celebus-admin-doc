## 1. 화면 목적

개별 시즌의 정책(기간·캡·티어 보상·랭킹 보상·이월) 조회·수정. POLICY가 시즌 시작 전 정책을 확정하고, Active 중에는 캡·티어 일부만 수정 가능.

### 진입 경로

- [CEB-BO-DUK-101] > Row 클릭

URL: `/duk/seasons/{id}`

---

## 2. UI 설명

### 2.1 상단 헤더

- Breadcrumb: `홈 > 덕력 > 시즌 관리 > {시즌명(KO)}`
- 뒤로가기 (←) → [CEB-BO-DUK-101]
- 타이틀: `{시즌명(KO)} [상태 Badge]`
- 우측 상단 액션:
  - [수정하기] (Primary, Draft·Ready 상태)
  - [캡 조정] (Active 상태, POLICY 사유 입력 필수)
  - [게시하기] (Draft 상태)
  - [강제 종료] (Active 상태, SUPER 전용 + 사유)
  - [삭제하기] (Draft 한정)

### 2.2 정보 그리드

#### A. 시즌 기본 정보

| 항목 | 표시 |
| --- | --- |
| 시즌명 (KO/EN/JP) | 다국어 |
| 시즌 설명 (KO/EN/JP) | 200자 |
| 시작일 | YYYY.MM.DD 00:00 KST |
| 종료일 | YYYY.MM.DD 23:59 KST |

#### B. 캡 설정

| 항목 | 표시 |
| --- | --- |
| 회원당 누적 적립 캡 | 100,000 DUK (기본) |
| 캡 도달 시 동작 | 추가 적립 차단 + 회원 알림 ("이번 시즌 캡에 도달했어요") |
| 캡 조정 이력 | (활동 로그 링크) |

#### C. 티어 보상 설정

POLICY가 티어별 잔액 구간 + 보상 정의:

| 티어 | 잔액 조건 | 응모권 보상 | BIVE 보상 | 추가 보상 |
| --- | --- | --- | --- | --- |
| Bronze | 0 ~ 999 | 1 | — | — |
| Silver | 1,000 ~ 4,999 | 3 | — | — |
| Gold | 5,000 ~ 19,999 | 10 | 1 (Genesis Edition) | — |
| Platinum | 20,000 ~ 49,999 | 30 | 3 | 독점 콘텐츠 |
| Diamond | 50,000+ | 100 | 10 | VIP 뱃지 + 독점 |

- [티어 추가] / [티어 삭제] (Draft 한정)
- [티어 수정] (Draft·Ready 자유, Active 제한적 — 보상량만)

> **응모권 지급 정합**: 티어별 응모권 보상은 시즌 정산 시 자동 발급되며, [CEB-BO-RFT-201] 변동 내역에서 출처 **"덕력 랭킹 보상"**으로 조회 가능 ([CEB-BO-006] §7.6 + [CEB-BO-010] §4-1 정합).

#### D. 랭킹 보상 설정

| 항목 | 표시 |
| --- | --- |
| TOP N | 100명 (POLICY 설정) |
| 1등 보상 | 한정 BIVE + 응모권 1,000장 |
| 2~10등 보상 | BIVE + 응모권 500장 |
| 11~100등 보상 | 응모권 100장 |
| 동률 처리 | 시즌 마지막 적립 시점 기준 (선착순) |

> **응모권 지급 정합**: 랭킹 보상으로 발급되는 응모권은 §C 티어 보상과 동일하게 출처 **"덕력 랭킹 보상"**으로 변동 기록.

#### E. 정산 정책

| 항목 | 표시 |
| --- | --- |
| 잔액 이월률 | 50% (기본, 0~100% 조정) |
| 정산 시점 | 종료일 자정 자동 |
| 자동 알림 | 종료 7일·1일 전 회원 푸시 |
| 정산 실패 재시도 | 3회 자동 → 운영자 수동 |

### 2.3 통계 섹션 (Active·Settling·Ended 상태)

| 항목 | 표시 |
| --- | --- |
| 참여 회원 | 1,234명 |
| 누적 적립 / 소비 | 12,345,678 / 5,432,100 DUK |
| 회원당 평균 잔액 | 5,615 DUK |
| 캡 도달 회원 | 12명 (1.0%) |

### 3.3 게시 검증

[게시하기] 클릭 시 검증:
- 다국어 KO/EN/JP 3언어 모두 입력
- 시작일·종료일·캡·티어 1개 이상·랭킹 보상 모두 설정
- Active 시즌 미존재 ([CEB-BO-DUK-101] §4.3)

### 3.4 강제 종료 (SUPER 전용)

운영 사고 등 비상 상황에서 SUPER가 즉시 종료. 사유 입력 + 2FA 재인증. 즉시 정산 트리거.
