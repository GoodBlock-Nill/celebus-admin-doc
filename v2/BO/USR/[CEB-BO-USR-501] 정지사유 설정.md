## 1. 화면 목적

회원 제재 처리 시 운영자가 선택할 **정지사유 데이터베이스**를 통합 관리. [CEB-BO-FQ-101-REJETSET] 의 패턴을 회원 제재용으로 변형 — 코드(`SUS_{n}`)로 정규화하여 일관된 회원 안내 메시지 보장.

### 주요 기능

- 정지사유 리스트 조회 (운영자 노출명 + 코드 + 상태 + 사용 횟수)
- 상태 필터 (사용/미사용)
- 신규 정지사유 추가
- 기존 정지사유 수정

### 진입 경로

- [CEB-BO-USR-101] 회원 리스트 > [정지사유 설정] 버튼 클릭

### 4.2 정렬

| 기준 | 방식 |
| --- | --- |
| 등록일시 | 내림차순 (최신순) |

### 4.3 상태 정의

[CEB-BO-002] §7.3 적용 ([CEB-BO-FQ-101-REJETSET] 와 동일 패턴):

| 상태 | 의미 |
| --- | --- |
| 사용 (Active) | [CEB-BO-USR-201-MD-SANCTION] 제재 모달의 사유 목록에 노출 |
| 미사용 (Inactive) | 제재 모달에 미노출. 과거 제재 건은 영향 없음 |

### 4.4 코드 표준

- 자동 채번: `SUS_{n}` (4자리)
- 코드 변경 불가 (NFT 메타데이터 일관성과 동일 원칙)
- 메시지(다국어)·운영자 노출명·상태는 변경 가능 (신규 제재부터 적용)

### 4.5 적용 가능 단계 매핑

[CEB-BO-002] §7.3 회원 제재 단계와 사유의 매핑:

| 사유 예시 | 적용 가능 단계 |
| --- | --- |
| "약관 단순 위반 (경미)" | 경고만 |
| "약관 반복 위반" | 경고 / 부분정지 / 1·7일 정지 |
| "어뷰징 의심" | 1·7일 / 30·90일 정지 (SUPER) |
| "심각한 위반·다중 계정" | 영구 정지 (SUPER) / 당첨 취소 (SUPER) |

각 사유는 **다중 단계** 적용 가능하도록 체크박스로 설정.

### 4.6 회원 알림 메시지 (다국어)

[CEB-BO-002] §5 적용. 사유별 KO/EN/JP 3언어 메시지 입력. 회원 제재 시 회원 언어 설정 우선 안내.

### 4.7 멀티아티스트 정책 — 전역 단일 + 발견 컨텍스트 메타 ⭐ v1.1 신규

본 마스터는 [CEB-BO-100] §3 결정 ①에 따라 **전역 단일**로 운영한다. 아티스트별 분리하지 않는다.

#### 4.7.1 결정 근거

| 검토 관점 | 채택 |
| --- | --- |
| 정지사유 본질 | 회원 행동 판정 (욕설·도배·악성 활동 등) — 회원 단위 |
| 식별 일관성 | 같은 위반 1개 코드로 통합 통계 |
| 운영 복잡도 | 단일 마스터 — 1벌만 관리 |
| 차별 룰 실수요 | 거의 없음 (대부분 위반은 공통 카테고리) |
| 결과 | **전역 단일 채택** ([CEB-BO-100] §3.2 v1.0 → v1.1 변경 근거 인용) |

#### 4.7.2 발견 컨텍스트 메타

정지 처리 시점에 어느 아티스트 활동 중 발견되었는지 메타로 기록 → 통계·감사 가능.

| 필드 | 타입 | 설명 |
| --- | --- | --- |
| `discoveryContext` | ID NULL | 발견 컨텍스트 — 아티스트 그룹 ID 또는 NULL(전역 활동 중) |
| `discoveredFeature` | enum | `FANQUEST` / `RAFFLE` / `MEM` / `INF_COMMENT` / `SUPPORT` / `OTHER` |

> 메타는 본 화면(사유 마스터)에서는 입력하지 않는다. **회원 제재 처리 화면**([CEB-BO-USR-201-MD-SANCTION])에서 운영자가 정지 처리 시 함께 입력한다.

#### 4.7.3 발견 컨텍스트별 통계

본 화면 §2.3 테이블의 "사용 횟수" 컬럼은 발견 컨텍스트별로도 분리 통계 가능 (Phase 2+ 추가):

- 컬럼 추가 옵션: 사용 횟수 (V01D 출처 / iKON 출처 / 전역 출처)
