## 1. 개요

회원(USR) 영역은 CELEBUS 가입자 정보 조회·검색·필터·제재·자산 회수 등 **회원 라이프사이클 전반**을 관리한다. v1에서 회원 상세 4탭(기본·참여내역·게임존·BIVE)을 운영 중이며, v2에서 신규 4탭(덕력·서포트·기억·팬덤레벨)을 추가하여 **8탭 구조**로 확장한다.

| 항목 | 위치 |
| --- | --- |
| 마스터 가이드 | [CEB-BO-001] §2 12영역 매핑 (#9 USR), §4.1 회원(USR) 영역 |
| 영역 ID 체계 | [CEB-BO-001] §5 — `CEB-BO-USR-{번호}` |
| 권한 매트릭스 | [CEB-BO-000] §3·§4 (4단계 권한 정책) |

| 활동 로그 | [CEB-BO-000] §6 |

## 7. 영역 고유 정책

### 7.1 회원 상세 9탭 구조 (v1.1 갱신 — 실제 운영 정렬)

분석 리포트 [CEB-BO-001-A] §3.2.2 운영 검증: 실제 BO에 9탭 구현됨. v2 신규 4탭(덕력·서포트·기억·팬덤레벨)은 별도 영역(Phase 9)에서 운영자가 영역별 화면으로 진입.

탭 순서 (실제 운영 그대로):

| # | 탭명 | 핵심 표시 항목 |
| --- | --- | --- |
| 1 | **기본정보** | 닉네임·실명·전화·이메일·국가·로그인 방식·가입일·최근 로그인·DID 발급·Set Approval·지갑주소·소개·운영자 메모·제재 상태 |
| 2 | **참여내역** (Quest·Raffle) | Quest 제출 이력 (검수 상태별 카운트), Raffle 응모·당첨 이력, 응모권 잔액·만료 일정 |
| 3 | **게임존** | PM·ST 참여 이력, GP 잔액·변동, CELB 교환 이력, 랭킹 |
| 4 | **BIVE** | 보유 BIVE 목록 (에디션·획득 경로·획득일), 외부 지갑 연동 상태, 혜택 사용 이력 |
| 5 | **프로젝트** | (구현 대기 — 운영 시작 시 회원별 프로젝트 참여 이력) |
| 6 | **Fans** | (구현 대기 — 회원별 Fans 서비스 활동 이력) |
| 7 | **티켓** | 회원이 보유한 응모권·이벤트 티켓 통합 조회 ([CEB-BO-FQ-301] §7.6 거래 코드 9종 회원 단위 필터링) |
| 8 | **지갑** | 회원 외부 지갑 연동 상태, BIVE 보유 추적 ([CEB-BO-000] §15) |
| 9 | **활동내역** | 회원 행위 통합 시계열 (로그인·제재·자산 회수·응모·민팅 등) |

각 탭의 정렬·필터·표시 상세는 [CEB-BO-USR-201] 명세서에서 확정.

**탭 전환 정책**:
- 탭 클릭 시 비동기 로딩 (탭별 API 분리)
- 탭 진입 시 Skeleton UI ([CEB-BO-000] §7)
- 탭별 빈 데이터 시 Empty State 메시지 (예: "해당 회원은 아직 Quest 참여 이력이 없어요")
- 미구현 placeholder 탭(프로젝트/Fans): "해당 영역 운영 시작 시 활성화됩니다." 안내 노출

#### v2 신규 영역과의 분리

v2 BO-002 v1.0에서 정의했던 신규 4탭(덕력·서포트·기억·팬덤레벨)은 **별도 영역 화면으로 분리**:

| v2 신규 영역 | 영역 가이드 (Phase 9) | 회원별 진입 방식 |
| --- | --- | --- |
| 덕력 | DUK 영역 | DUK-101 → 회원 검색 또는 회원 상세 [활동내역] 탭에서 딥링크 |
| 서포트 | SUP 영역 | SUP-101 → 회원 단위 응원 이력 |
| 기억 | MEM 영역 | MEM-101 신고 관리 → 회원별 신고당한·신고한 이력 |
| 팬덤레벨 | EVT 영역 | EVT-201 → 회원별 아티스트 팬덤 레벨 추이 |

→ 회원 상세 화면의 UX 부담을 낮추고, 영역별 운영 책임자(POLICY 등)가 자기 영역에 집중 가능.

**탭 전환 정책**:
- 탭 클릭 시 비동기 로딩 (탭별 API 분리)
- 탭 진입 시 Skeleton UI ([CEB-BO-000] §7)
- 탭별 빈 데이터 시 Empty State 메시지 (예: "해당 회원은 아직 Quest 참여 이력이 없어요")

#### 자동 트리거 (운영자 인지 보조)

자동 트리거는 회원 자동 제재가 아닌 **운영자에게 검토 요청 알림**.

| 트리거 | 조건 | 알림 |
| --- | --- | --- |
| 기억저장소 N회 신고 | 7일 이내 동일 회원이 5건 이상 신고 받음 | OPERATOR 푸시: "회원 '{닉네임}' 신고 누적 — 검토 필요" |
| Quest 반복 반려 | 30일 이내 5건 이상 반려 (동일 사유) | OPERATOR 푸시: "회원 '{닉네임}' 검수 반복 반려 — 어뷰징 의심" |
| 다중 계정 의심 | 동일 IP·기기에서 N개 계정 가입 (24시간 이내 3개) | SUPER 푸시: "다중 계정 의심 — 검토 요청" |
| 비정상 응모 패턴 | 1시간 내 100건 이상 응모 시도 | OPERATOR 푸시: "비정상 응모 패턴 — 자동 차단됨" |

자동 트리거 발생 시 회원 상세 [기본] 탭에 "**⚠️ 검토 필요**" 뱃지 노출.

#### 제재 이력 화면 (CEB-BO-USR-301)

전체 회원 제재 이력 통합 조회 (SUPER 전용). 필터: 위반 코드·제재 단계·기간·처리 운영자.

### 7.4 자산 회수·강제 처리

부정 이용·당첨 취소 등으로 회원 보유 자산을 강제 회수하는 정책. **SUPER 전용**.

#### 회수 가능 자산

| 자산 | 회수 방식 | 비고 |
| --- | --- | --- |
| 덕력 | 차감 (수량 지정) | [CEB-BO-000] §11 자산 정의 차감 처리 |
| 응모권 | 회수 (수량 지정, 만료 기준 오래된 것부터) | 진행 중 Raffle 응모분도 회수 시 응모 취소 처리 |
| BIVE (NFT) | 강제 소각 또는 운영 지갑 이전 | 온체인 트랜잭션 발생 ([CEB-BO-000] §15) |
| GP | 차감 (수량 지정) | 게임존 영역 (v1 BO-GAM 호환) |
| CELB (외부 자산) | 회수 불가 (회원 외부 지갑 자산) | 출금 차단으로만 가능 |

#### 회수 절차

```
[CEB-BO-USR-201] 회원 상세 → [자산 회수] 버튼 (SUPER만 노출)
    ↓
[CEB-BO-USR-201-MD-RECLAIM] 자산 회수 모달
   - 회수 자산 종류 선택 (체크박스, 다중)
   - 자산별 수량 입력
   - 회수 사유 텍스트 (필수, 200자, KO)
   - 위반 코드 매핑 (선택, [CEB-BO-000] §14.1)
   - 2FA 추가 인증 (SUPER 자산 회수 가드레일)
    ↓
저장 → 즉시 차감 + 회원 알림 + 활동 로그 (각 자산별 1행씩 상세 기록)
    ↓
온체인 자산(BIVE)인 경우: 트랜잭션 큐 → 블록 컨펌 → 완료 알림
```

회수 직후 회원 상세 [기본] 탭 상단에 "**⚠️ 자산 회수 처리됨 ({날짜}, 사유: {요약})**" 배너 30일간 노출.

#### Deprecation 절차

v1 회원 영역 화면(현재 별도 명세서 폴더 미존재 — 운영 BO 직접 구현)은 **명세서가 없어 deprecation 태그 적용 불필요**. 단, BO-001 §3 절차에 따라 v2 화면 명세서 작성 시 v1 출처 명시 필수:

```markdown
| 비고 | v1 회원 영역(BO 실구현)을 v2 정책으로 흡수·확장. 기존 4탭 + 신규 4탭 |
```
