## 1. 화면 목적

팬퀘스트 검수 시 사용할 신규 반려사유(Rejection Reason)를 시스템에 등록. 운영자 식별 명칭 + 유저 안내 메시지(KO/EN/JP) 입력. [CEB-BO-006] §7.4 반려사유 DB 정책 적용.

### 진입 경로

- [CEB-BO-FQ-101-REJETSET] 반려사유 설정 > [+ 새 사유 추가] 버튼 클릭

---

## 2. UI 설명

### 모달 구조

| 영역 | 내용 |
| --- | --- |
| 타이틀 | "반려사유 추가" |
| 닫기 버튼 (X) | 우측 상단. 작성 내용 미저장 닫기 |

### 입력 폼

| 요소명 | 타입 | 필수 | 글자수 | 설명 |
| ------ | ---- | ---- | ----- | ---- |
| 운영자 노출명 | Text Input | Y | **30자** | [CEB-BO-006] §5.1 표준 (v1 100자 → v2 30자 정정). KO 단일, 내부 식별용 |
| 유저 노출 메시지 | Textarea + Radio(KO/EN/JP) | Y | 200자 | KO/EN/JP 3언어 모두 필수. 회원 알림 메시지 ([CEB-BO-006] §5.1) |

- 운영자 노출명: KO 단일 (관리자 페이지 리스트·드롭다운 표시용)
- 유저 노출 메시지: 언어 라디오 버튼 (KO 기본 선택). 언어 전환 시 입력 데이터는 언어별 보존
- 미입력 언어 → 라디오 옆 빨간 점, 입력 완료 → 체크마크

### 하단 액션 버튼

| 버튼 | 타입 | 동작 |
| ---- | ---- | ---- |
| [취소하기] | Secondary | 모달 닫기 (작성 내용 폐기) |
| [추가하기] | Primary | 필수 입력 검증 후 서버 전송 → 신규 반려사유 등록 |

> 시각적 스타일은 피그마 참조

## 4. 규칙·로직

### 4.1 권한별 분기

[CEB-BO-006] §4 적용:

| 권한 | 본 모달 사용 |
| --- | --- |
| SUPER | ✅ |
| OPERATOR | ✅ |
| POLICY | ❌ |
| EXTERNAL_ARTIST | ❌ (반려사유는 본사 운영 영역) |

### 4.2 초기값 및 상태 설정

- 신규 등록되는 모든 반려사유는 기본 **Active(사용)** 상태로 등록 ([CEB-BO-006] §7.4)
- 기본 언어: KO 라디오 선택 상태로 진입
- 코드 형식: 자동 채번 `REJ_{n}` (4자리)

### 4.3 다국어 입력 로직

[CEB-BO-000] §5 적용:

- 라디오 전환 시 각 언어별 입력 데이터 보존 (서버 저장 전 클라이언트 메모리 보존)
- KO/EN/JP **3언어 모두 필수** (반려사유 메시지는 회원 알림용 — 임시저장 개념 없음)
- 1개 이상 빈 칸 → 인라인 에러 + [추가하기] 비활성화

### 4.4 [추가하기] 활성화 조건

- 운영자 노출명: 입력 완료 (KO, 30자 이내)
- 유저 노출 메시지: KO + EN + JP 모두 입력 완료 (각 200자 이내)
