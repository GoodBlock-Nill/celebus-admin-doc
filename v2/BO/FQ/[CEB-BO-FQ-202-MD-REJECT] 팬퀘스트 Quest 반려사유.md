## 1. 화면 목적

대기내역 탭([CEB-BO-FQ-202])에서 [반려] 클릭 시 호출. 운영자가 반려사유 코드를 선택하여 반려 처리를 확정. 선택된 사유의 다국어 메시지가 회원에게 푸시 전달.

### 진입 경로

- [CEB-BO-FQ-202] 대기내역 > 특정 제출 건 카드 [반려] 버튼 클릭

| 운영자 메시지 (선택) | Textarea | N | 반려사유 외 운영자 추가 메시지. **200자** ([CEB-BO-006] §5.1 반려사유 텍스트 표준), KO 단일 |

## 4. 규칙·로직

### 4.1 권한별 분기

[CEB-BO-006] §4 적용:

| 권한 | 사용 |
| --- | --- |
| SUPER | ✅ |
| OPERATOR | ✅ |
| POLICY | ❌ |
| EXTERNAL_ARTIST | ❌ (검수 권한 없음) |

### 4.2 반려 프로세스

[CEB-BO-006] §7.3 검수 워크플로우 적용:

1. 반려사유 코드 선택 + (선택) 운영자 메시지 입력
2. [저장하기] → 서버 API 호출
3. 제출 건 상태: Pending → Rejected
4. **회원 푸시 알림** ([CEB-BO-000] §15):
   - 본문: 반려사유 코드의 유저 노출 메시지(KO/EN/JP, 회원 언어 설정 우선) + 운영자 메시지(있을 경우)
   - 재참여 가이드 (앱 내 딥링크)
5. 활동 로그 기록 ([CEB-BO-006] §6)
6. [CEB-BO-FQ-203] 처리내역 탭으로 데이터 이동

### 4.3 유효성 검사

- 반려사유: 필수 (선택 안 한 채 [저장하기] → Toast Error)
- 운영자 메시지: 선택. 입력 시 200자 제한

### 4.4 [저장하기] 활성화 조건

- 반려사유 선택 완료

| Loading (반려사유 목록 조회) | Select 영역 Skeleton |

| Error | Toast (Error) "선택한 퀘스트의 반려 처리에 실패하였습니다." → 모달 유지 (재시도 가능) |

## 7. 에러·예외 상황

| 상황 | 메시지 |
| --- | --- |
| 반려 성공 | Toast (Success): "선택한 퀘스트의 반려 처리가 완료되었습니다." |
