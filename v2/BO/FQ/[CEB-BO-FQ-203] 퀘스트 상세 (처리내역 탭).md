## 1. 화면 목적

해당 Quest에서 검수 완료(승인 또는 반려)된 모든 제출 건의 처리 이력을 시계열로 조회. 운영팀 검수 활동 추적·CS 대응·어뷰징 분석에 활용.

### 주요 기능

- 승인/반려 처리된 제출 건의 상태·처리 일시·처리자(관리자) 확인
- 이미지보기 아이콘으로 유저 제출 원본 이미지 조회
- 상태별 필터링 + 닉네임/제출 ID 검색

### 진입 경로

- [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) > [처리내역] 탭 클릭

---

## 2. UI 설명

### 2.1 상단 영역

- 탭 메뉴 (3탭): 기본정보 / 대기내역 / **처리내역 (현재)**
- 검색·필터:
  - 상태 Dropdown (Default 전체): 전체 / Approved / Rejected
  - 검색 대상 Dropdown: 닉네임 / 제출 ID
  - 키워드 입력 Text Input
  - [초기화] 버튼

### 2.2 목록 영역 (List Table)

| 컬럼명 | 설명 |
| ------ | ---- |
| 상태 | Approved (Green Badge) / Rejected (Red Badge) |
| 제출 ID | 유일 식별 코드 (예: `Q00000003A`) |
| 유저닉네임 | 제출자 닉네임 |
| 반려사유 | 반려 시 사유명 (KO 운영자 노출명). 승인 시 "-" |
| 이미지보기 | 아이콘 버튼. 클릭 시 lightbox 팝업으로 원본 이미지 |
| 제출일시 | 유저 제출 시각 (YYYY.MM.DD HH:mm KST) |
| 처리일시 | 검수 완료 시각 (YYYY.MM.DD HH:mm KST) |
| 처리자 | 검수한 관리자명 |

- 데이터 없음: "처리내역이 없습니다." 또는 "검색 결과가 없습니다."
- 페이지네이션: 하단 중앙, 10페이지 단위
- Row 클릭은 미제공 (이미지보기 아이콘만)

> 시각적 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 이미지보기

- 이미지보기 컬럼 아이콘 클릭 → lightbox 팝업으로 유저 업로드 원본 이미지 표시
- ESC 또는 외부 클릭으로 닫기

### 3.2 네비게이션

- [기본정보] 탭 → [CEB-BO-FQ-201]
- [대기내역] 탭 → [CEB-BO-FQ-202]

## 4. 규칙·로직

### 4.1 권한별 분기

[CEB-BO-006] §4 적용:

| 권한 | 화면 진입 | 조회 |
| --- | --- | --- |
| SUPER | ✅ | ✅ (전체) |
| OPERATOR | ✅ | ✅ (전체) |
| POLICY | ✅ | ✅ (조회 전용) |
| EXTERNAL_ARTIST | ✅ (자기 아티스트만) | ✅ |

본 화면은 조회 전용이므로 액션 권한 매트릭스 별도 없음.

### 4.2 정렬·필터

| 정렬 기준 | 정렬 방식 | 비고 |
| --------- | --------- | ---- |
| 처리일시 | 내림차순 (최신순) | 기본 정렬. 최근 작업 상단 |

- 상태 필터 + 키워드 검색 AND 조건
- 키워드 검색: 닉네임/제출ID 부분 일치(LIKE)

### 4.3 처리자 표기

- 검수자(승인 또는 반려 처리한 관리자) 닉네임 표기
- 활동 로그 기록 시점의 처리자 정보 ([CEB-BO-006] §6)
