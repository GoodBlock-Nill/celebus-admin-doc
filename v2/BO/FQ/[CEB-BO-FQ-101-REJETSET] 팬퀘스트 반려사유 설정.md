## 1. 화면 목적

팬퀘스트 검수 시 운영자가 선택할 **반려사유 데이터베이스**를 통합 조회·관리. [CEB-BO-006] §7.4 정책 적용 — 코드(`REJ_{n}`)로 정규화되어 일관된 회원 안내 메시지 보장.

### 주요 기능

- 반려사유 리스트 조회 (운영자 노출명 + 상태 + 코드)
- 상태 필터 (사용/미사용)
- 신규 반려사유 추가 진입
- 기존 반려사유 수정 진입

### 진입 경로

- [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > [반려사유 설정] 버튼 클릭

## 4. 규칙·로직

### 4.1 권한별 분기

[CEB-BO-006] §4 적용:

| 권한 | 화면 진입 | 조회 | [+ 새 사유 추가] | Row 클릭 (수정) |
| --- | --- | --- | --- | --- |
| SUPER | ✅ | ✅ | ✅ | ✅ |
| OPERATOR | ✅ | ✅ | ✅ | ✅ |
| POLICY | ✅ | ✅ (조회 전용) | ❌ | ❌ |
| EXTERNAL_ARTIST | ❌ | — | — | — |

> EXTERNAL_ARTIST 차단 — 반려사유는 본사 운영 영역.

### 4.2 정렬·필터

| 정렬 기준 | 정렬 방식 | 비고 |
| --------- | --------- | ---- |
| 등록일시 | 내림차순 (최신순) | 기본 |

상태 필터만 적용. 텍스트 검색 미제공 (DB 규모 작음).

### 4.3 상태 정의 ([CEB-BO-006] §7.4)

| 상태 | 의미 |
| ---- | ---- |
| 사용 (Active) | [CEB-BO-FQ-202-MD-REJECT] 검수 모달 사유 목록에 노출 |
| 미사용 (Inactive) | 검수 모달에 미노출 (과거 반려 건 영향 없음) |

### 4.4 사용 횟수 카운트

- 해당 코드로 반려 처리된 누적 건수 (read-only)
- Inactive 변경 후에도 과거 누적은 보존
