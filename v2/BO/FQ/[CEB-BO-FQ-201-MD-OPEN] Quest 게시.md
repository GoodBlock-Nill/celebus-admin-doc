## 1. 화면 목적

Draft 상태 Quest를 Active로 전환하기 전 게시 의사 최종 확인. 게시 시점부터 Quest가 즉시 시작되어 앱에 노출됨을 인지시킴.

### 진입 경로

- [CEB-BO-FQ-201] (Draft 상태) > [게시하기] 클릭

## 4. 규칙·로직

### 4.1 권한별 분기

| 권한 | 사용 |
| --- | --- |
| SUPER | ✅ |
| OPERATOR | ✅ |
| POLICY | ❌ |
| EXTERNAL_ARTIST | ❌ (검수 권한 없음 — [CEB-BO-006] §4) |

### 4.2 게시 전 검증

[CEB-BO-006] §5.2 다국어 게시 검증:

- KO/EN/JP 3언어 모두 비어있지 않아야 함
- 1개라도 빈 칸 → 게시 차단 + Toast (Error) "모든 언어 필드를 입력해주세요"
- 백엔드 추가 검증: 필수 항목 누락 시 차단

### 4.3 게시 프로세스

- [확인] 클릭 → 서버 API 호출
- 시작 일시: 서버 시간(KST UTC+9) 자동 기록
- 상태: Draft → Active
- 앱 즉시 노출

## 5. UI 상태별 처리

| 상태 | 표시 |
| --- | --- |
| Loading | [확인] 버튼 스피너 + 비활성 |
| Success | Toast (Success) "Quest가 게시 되었습니다." → 모달 닫기 + 상세 화면 새로고침 (상태 Active, [종료하기] 버튼 노출) |
| Error | Toast (Error) "Quest 게시에 실패하였습니다. 다시 시도하세요." → 모달 유지 (재시도 가능) |
