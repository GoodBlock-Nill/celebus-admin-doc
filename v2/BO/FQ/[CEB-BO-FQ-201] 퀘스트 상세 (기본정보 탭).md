## 1. 화면 목적

등록된 개별 Quest의 기본 정보·설정·생애 주기를 관리. 운영자는 본 화면에서 Quest 현황을 확인하고 게시/수정/종료/삭제 액션을 수행한다. 검수 작업은 형제 탭(대기내역·처리내역)에서.

### 주요 기능

- Quest 기본 정보 및 설정 상태 확인 (다국어 텍스트 / 대표 이미지 / 보상 / 일정 / 연관 링크)
- 실시간 제출 현황 대시보드 (총 제출 / 승인 / 반려 / 대기)
- Quest 라이프사이클 관리 ([CEB-BO-006] §7.1)
- 형제 탭 진입 (대기내역 / 처리내역)

### 진입 경로

- [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > Quest 리스트 Row 클릭

## 2. UI 설명

### 2.1 상단 네비게이션·헤더

- Breadcrumb: `홈 > 팬퀘스트 > Fan Quest 관리 > Quest > {Quest 타이틀}`
- 뒤로가기 화살표 (←) 아이콘 → [CEB-BO-FQ-101] 이동
- 타이틀: `{Quest 타이틀(KO)} [상태 Badge]`
- 탭 메뉴 (3탭, 상호 배타):
  - **기본정보** (현재 화면)
  - **대기내역** → [CEB-BO-FQ-202] 이동
  - **처리내역** → [CEB-BO-FQ-203] 이동

### 2.2 상태별 액션 버튼 (우측 상단)

[CEB-BO-006] §7.1 라이프사이클 적용. 권한 매트릭스는 [CEB-BO-006] §4 참조.

| Quest 상태 | 노출 버튼 | 호출 모달 |
| ---- | --------- | --------- |
| 임시저장(Draft) | [삭제하기] [수정하기] [게시하기] | MD-DELETE / EDIT 화면 / MD-OPEN |
| 진행중(Active) | [수정하기] [종료하기] | EDIT 화면 (제한적) / MD-CLOSE |
| 종료(Ended) | (버튼 없음) | — |

> EXTERNAL_ARTIST: 자기 아티스트 한정 [수정하기]만. [게시하기]·[종료하기]·[삭제하기]는 OPERATOR 이상 본사 권한.

### 2.3 현황 대시보드 카드 (상단)

4개 카드, 실시간 제출 현황:

| 카드 | 아이콘 | 의미 |
| --- | --- | --- |
| 총 제출 수 | people | 전체 참여 건수 |
| 승인 | check | 검수 완료 + 보상 지급 확정 (BIVE 자동 민팅 포함) |
| 반려 | X | 검수 반려 |
| 대기 | clock | 검수 미처리 |

### 2.4 상세 정보 그리드

#### A. 대표 이미지 (좌측 상단)

- Quest 대표 이미지를 큰 썸네일로 표시
- 이미지 클릭 → 원본 보기 모달 (lightbox)

#### B. Quest 정보 (우측 상단)

| 항목 | 값 표시 |
| ---- | ---- |
| 아티스트 | 그룹명 / 멤버명 |
| 타입 | 미션 타입 (예: 이미지 촬영 및 업로드) |
| 보상 | 보상 종류 + 수량 (예: 응모권 1매 + BIVE [V01D Event-005]) |
| 시작 일시 | 진행중·종료 상태에서만 표시. 게시 시점 KST |
| 마감 일시 | 모든 상태 표시. 임시저장 상태에서는 단독 표시 |
| 연관 링크 | KO 라벨 + URL (최대 2개). 미등록 "-" |

> 임시저장(Draft): 마감 일시만 표시. 진행중·종료: 시작·마감 둘 다 표시.

#### C. 관리 정보 (우측 하단)

| 항목 | 값 |
| ---- | ---- |
| 상태 | Status Badge (임시저장 Gray / 진행중 Green / 종료 Black) |
| 생성 관리자 | 생성자 표기 (현재 운영 데이터 부재 — 분석 리포트 §3.4 활동 로그 부재 이슈 참조) |
| 생성 일시 | YYYY.MM.DD HH:mm KST |
| 최근 수정자 | 수정자 표기 |
| 최근 수정 일시 | YYYY.MM.DD HH:mm KST |

#### D. Quest 다국어 콘텐츠 (하단)

- 타이틀 KO / EN / JP
- 설명 KO / EN / JP
- 유저 가이드 KO / EN / JP

> 시각적 스타일은 피그마 참조

## 4. 규칙·로직

### 4.1 권한별 분기

[CEB-BO-006] §4 적용:

| 권한 | 화면 진입 | 조회 | [수정] | [게시] | [종료] | [삭제] |
| --- | --- | --- | --- | --- | --- | --- |
| SUPER | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ (Draft만) |
| OPERATOR | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ (Draft만) |
| POLICY | ✅ | ✅ | ❌ | ❌ | ❌ | ❌ |
| EXTERNAL_ARTIST | ✅ (자기 아티스트만) | ✅ | ✅ (자기 아티스트만, 제한적) | ❌ | ❌ | ❌ |

### 4.2 상태 전환 로직 ([CEB-BO-006] §7.1)

- **게시하기 (Draft → Active)**: [게시하기] → [MD-OPEN] 확인 → 서버 시간(KST)을 시작 일시로 자동 기록 + 상태 Active. 앱 즉시 노출.
- **종료하기 (Active → Ended)**: [종료하기] → [MD-CLOSE] 확인 → 참여 차단 + 상태 Ended. 마감일 도래 시 시스템 자동 종료도 동일 결과.
- **삭제하기 (Draft 한정)**: [삭제하기] → [MD-DELETE] 확인 → 영구 삭제. 진행중/종료 상태는 [삭제하기] 미노출.

### 4.3 데이터 표시 규칙

- 대기 카운트: 종료 상태가 되어도 미처리 검수 건이 있으면 숫자 잔존. 운영자는 처리내역 탭에서 정리
- 날짜 포맷: `YYYY.MM.DD HH:mm` KST 통일 ([CEB-BO-000] 표준)
- 다국어 콘텐츠: 임시저장 상태에서 일부 언어 비어있을 수 있음. 게시 시점에 3언어 검증

### 4.4 수정 가능 범위 (Active 상태)

[CEB-BO-006] §7.1·§7.4 적용. 진행중 상태에서는 핵심 정보 수정 제한:

| 항목 | Draft 수정 | Active 수정 |
| --- | --- | --- |
| 타이틀·설명·유저 가이드 (다국어) | ✅ | ✅ (오타 수정 등) |
| 대표 이미지 | ✅ | ✅ |
| 아티스트 | ✅ | ❌ |
| 퀘스트 타입 | ✅ | ❌ |
| 마감 일시 | ✅ | ✅ (연장만 가능, 단축 불가) |
| 보상 종류·수량 | ✅ | ❌ |
| 연관 링크 | ✅ | ✅ |

상세 제약은 [CEB-BO-FQ-201-EDIT] 참조.

## 6. 활동 로그

[CEB-BO-006] §6 적용:

| 액션 | 발생 화면 | 로그 메시지 (KO) |
| --- | --- | --- |
| Quest 게시 | [MD-OPEN] | Quest '{타이틀}'을(를) 게시했습니다. (보상: {보상 BIVE}, 종료: {종료일}) |
| Quest 종료 | [MD-CLOSE] | Quest '{타이틀}'을(를) 조기 종료했습니다. (참여자: {n}명, 사유: {요약}) |
| Quest 삭제 | [MD-DELETE] | Quest '{타이틀}'을(를) 삭제했습니다. (Draft 상태만 가능) |

본 화면 자체(조회)는 로그 미기록 (빈도 과다).

## 7. 에러·예외 상황

| 상황 | 처리 |
| --- | --- |
| 삭제 불가 (Active/Ended 상태) | [삭제하기] 버튼 미노출. 백엔드 추가 검증 |
| 수정 제한 항목 변경 시도 (Active 상태) | EDIT 화면에서 핵심 항목 비활성화 처리 |
| 종료 후 미처리 검수 건 잔존 | 대시보드 대기 카운트 잔존 + 운영자 안내 배너 (Active 종료 시) |
| 데이터 조회 실패 | Toast (Error) + [재시도] 버튼 |
| EXTERNAL_ARTIST 자기 아티스트 외 진입 | 서버 차단 + Toast (Error) "접근 권한이 없는 Quest입니다." |
