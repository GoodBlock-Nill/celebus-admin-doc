## 1. 화면 목적

전체 회원의 응모권(티켓) 거래 이력을 관리자가 통합 조회. Fan Quest 보상, Game Zone 보상, BIVE 보유 보상, GP 구매, Raffle 응모/반환, 만료 소멸, 운영자 환불·회수 등 **모든 응모권 변동 내역**을 단일 화면에서 모니터링.

### 주요 기능

- 전체 응모권 변동 내역 통합 조회 (read-only)
- 거래 코드별 필터링 ([CEB-BO-006] §7.6 6종)
- 적립경로별 필터링 (Fan Quest / Game Zone / BIVE / GP 구매)
- 기간·유저(닉네임/유저ID) 검색
- 어뷰징 분석·CS 대응 활용

### 진입 경로

- 사이드 메뉴 > 팬퀘스트 > 응모권 거래 내역

## 2. UI 설명

### 2.1 상단 영역

- Breadcrumb: `홈 > 팬퀘스트 > 응모권 거래 내역`

### 2.2 검색·필터 영역

| 구분 | 요소명 | 타입 | 설명 |
| ---- | ------ | ---- | ---- |
| 필터 | 거래 유형 | Dropdown | 전체 / 적립 / 사용 / 반환 / 만료 / 회수 (코드 매핑은 §4.3) |
| 필터 | 적립경로 | Dropdown | 전체 / Fan Quest / Game Zone / BIVE / GP 구매 |
| 필터 | 기간 (시작) | Date Picker | 조회 시작일 |
| 필터 | 기간 (종료) | Date Picker | 조회 종료일 |
| 입력 | 검색 | Text Input | 닉네임 또는 유저ID. Enter 또는 검색 아이콘 |
| 버튼 | 초기화 | Dark Button | 필터 리셋 |

기본값: 전체 / 전체 / (전체 기간) / (빈 값)

### 2.3 목록 영역 (List Table)

| 컬럼명 | 데이터 타입 | 설명 |
| ------ | ----------- | ---- |
| 변동 유형 | Badge | [CEB-BO-006] §7.6 정합 (퀘스트 보상 / PM·ST 보상 / BIVE 보유혜택 / 덕력 랭킹 보상 / GP로 응모권 구매 / 래플 응모 사용 / 래플 취소 환불 / 만료 소멸 / 운영자 보정 적립 / 운영자 회수) |
| 카테고리 | Badge | 적립 (Green) / 사용 (Red) / 반환 (Blue) / 만료 (Gray) / 회수 (Red Dark) |
| 적립경로 | Text | 퀘스트 보상 / PM·ST 보상 / BIVE 보유혜택 / 덕력 랭킹 보상 / GP로 응모권 구매 / 래플 |
| 회원 | Text | 닉네임 + 회원 식별값 (2행). **닉네임 클릭 시 [CEB-BO-USR-201] 회원 상세 새 탭 진입** |
| 관련 이벤트 | Text | 퀘스트 / 래플 / 게임 / BIVE 캠페인 타이틀 |
| 변동 수량 | Number | +N (적립/반환) / −N (사용/만료/회수) |
| 변동 후 잔여 | Number | 거래 후 회원 응모권 잔액 |
| 일시 | DateTime | YYYY.MM.DD HH:mm:ss KST |

- Row 클릭 → 별도 동작 없음 (조회 전용 통합 화면)
- 닉네임 셀만 클릭 가능 — 회원 상세 새 탭 진입
- 페이지네이션: 하단 중앙, 10건 단위

> 시각적 스타일은 피그마 참조

> ⚠️ **일일 미션은 응모권을 발급하지 않음** — 덕력 25만 적립 ([CEB-BO-FQ-501] 정합).
