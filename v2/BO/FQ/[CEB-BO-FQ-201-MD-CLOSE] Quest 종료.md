## 1. 화면 목적

Active 상태 Quest를 관리자가 수동으로 조기 종료(Early Termination). 실수 방지를 위한 최종 확인. 종료 후 더 이상 유저 참여 불가.

### 진입 경로

- [CEB-BO-FQ-201] (Active 상태) > [종료하기] 클릭

## 4. 규칙·로직

### 4.1 권한별 분기

| 권한 | 사용 |
| --- | --- |
| SUPER | ✅ |
| OPERATOR | ✅ |
| POLICY | ❌ |
| EXTERNAL_ARTIST | ❌ |

### 4.2 종료 프로세스

- [확인] → 서버 종료 API 호출
- 상태: Active → Ended
- 마감 일시: 종료 시점(서버 KST)으로 갱신
- 유저 앱: 신규 참여(제출) 차단
- **미처리 검수 건은 잔존** ([CEB-BO-FQ-201] §4.3): 운영자는 처리내역 탭에서 정리 가능

### 4.3 종료 후 미처리 검수 안내

종료 시점에 [CEB-BO-FQ-202] 대기내역에 건이 남아있으면 모달에 추가 경고:

> "현재 검수 대기 중인 제출 {n}건이 있습니다. 종료 후에도 검수는 가능하나, 신규 참여는 즉시 차단됩니다."
