## 1. 화면 목적

신규 Quest를 생성하기 위한 입력 폼 화면. 기본 정보(다국어)·설정(아티스트·타입·마감)·보상(응모권/NFT)·연관 링크를 입력하고 임시저장(Draft) 상태로 저장한다. 게시(Active 전환)는 [CEB-BO-FQ-201] 상세에서 [CEB-BO-FQ-201-MD-OPEN] 모달을 통해 진행.

### 주요 기능

- Quest 기본 정보 입력 (타이틀·설명·유저 가이드·대표 이미지) — KO/EN/JP 3언어 지원
- Quest 설정 입력 (아티스트·타입·마감 일시)
- Quest 보상 설정 (응모권 / 응모권+NFT / NFT)
- 연관 링크 입력 (선택, 최대 2개)
- 임시저장(Draft) 상태로 즉시 저장

### 진입 경로

- [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > [생성하기] 버튼 클릭

|  대표 이미지 *  [Upload]   |  연관 링크     [+ 링크 추가]|                            |
|                            |                            |                            |
|                            |  * 시작 일시는 게시 시점에  |                            |
|                            |    자동으로 설정됩니다.     |                            |
|                            |    마감일은 오늘 기준 최소  |                            |
|                            |    1일 이후부터 선택 가능   |                            |

> 시각적 스타일은 피그마 참조

### 2.3 기본 정보 섹션 (좌측)

- 임시저장은 1개 언어만 입력해도 허용. **게시 시점**(별도 화면)에 3언어 검증 ([CEB-BO-006] §5.2)

### 2.4 퀘스트 설정 섹션 (중앙)

| 요소명 | 타입 | 필수 | 설명 |
| ------ | ---- | ---- | ---- |

| [+ 링크 추가] | Text Button | — | — | 새 링크 행 추가 (최대 2개, 도달 시 비활성화) |
| [삭제] | Icon Button (X) | — | — | 해당 링크 행 삭제 |

## 4. 규칙·로직

### 4.1 권한별 기능 분기

[CEB-BO-006] §4 적용. 본 화면 한정:

| 권한 | 화면 접근 | 입력 | 아티스트 선택 | NFT 보상 연결 | [생성하기] |
| ---- | --------- | ---- | ------------ | ------------ | ---------- |
| SUPER | ✅ | ✅ | 전체 | ✅ | ✅ |
| OPERATOR | ✅ | ✅ | 전체 | ✅ | ✅ |
| POLICY | ❌ | — | — | — | — |
| EXTERNAL_ARTIST | ✅ | ✅ | **자기 아티스트 자동 고정** | ✅ | ✅ |

### 4.2 다국어 입력 유효성 (KO/EN/JP)

[CEB-BO-006] §5 적용:

| 상태 | 표시 |
| ---- | ---- |
| 미입력 언어 존재 | 해당 언어 탭에 빨간 점(dot) |
| 모든 언어 입력 완료 | 모든 언어 탭에 체크마크 |
| 임시저장 시 | 1개 언어만 입력해도 허용 |
| **게시 시 (별도 화면)** | KO/EN/JP 3언어 모두 비어있지 않아야 함. 미충족 시 게시 차단 |

### 4.3 기간 설정 로직

- 시작 일시: 별도 입력 없음. [생성하기] → 임시저장 → [게시하기] 시점의 서버 시간(KST UTC+9)이 자동 설정
- 마감 일시: 현재 시점 +1일 이후만 선택 가능
  - 캘린더에서 +1일 미만 비활성화
  - 시간 입력: 시(00~23), 분(00~59) 정수만

### 4.4 보상 설정 로직

- 응모권 수량: 직접 입력 X, Dropdown 1~10장
- NFT 이벤트 목록: BIVE 시스템 API 연동 — 활성 상태(`Active`) + 보상 미할당 캠페인만 노출 ([CEB-BO-004] §4.3)
- 보상 종류 변경 시 하위 입력 필드 동적 노출/숨김

### 4.5 연관 링크 유효성

- 0~2개 입력 가능 (선택)
- 링크 한 행마다 라벨(KO/EN/JP) + URL 쌍 필수
  - 라벨만 입력 + URL 미입력 → 인라인 에러: "URL을 입력해주세요"
  - URL만 입력 + 라벨 미입력 → 인라인 에러: "라벨을 입력해주세요"
- URL 형식: `https://` 또는 `http://` 시작 → 미준수 시 "올바른 URL 형식을 입력해주세요 (예: https://example.com)"
- 미입력 시 [생성하기] 활성화에 영향 없음

### 4.6 [생성하기] 활성화 조건

모두 충족해야 활성화:
- 타이틀: 1개 이상 언어 입력완료 (임시저장 기준 — [CEB-BO-006] §5.2)
- 설명: 1개 이상 언어 입력완료
- 유저 가이드: 1개 이상 언어 입력완료
- 대표 이미지: 업로드 완료
- 아티스트: 선택 완료
- 퀘스트 타입: 선택 완료
- 마감 일시: 설정 완료
- 보상 설정: 종류 + 하위 항목 입력 완료
- 연관 링크: 입력 시 유효성 검증 통과 (미입력 시 영향 없음)
