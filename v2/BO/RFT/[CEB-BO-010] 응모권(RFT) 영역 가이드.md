> 본 문서는 응모권(RFT) 영역의 전체 범위·정책·라이프사이클을 정의한다.

## 1. 영역 개요

응모권(RFT)은 회원이 래플에 응모하기 위해 사용하는 자산이다. 회원은 **퀘스트 보상·PM·ST 보상·BIVE 보유혜택·덕력 랭킹 보상·GP로 응모권 구매** 5가지 경로로 응모권을 발급받고, 래플 응모 시 사용한다. **본 영역은 응모권의 발급·보유·사용 흐름을 모니터링하는 운영자용 영역**이며, 회원에게 응모권을 직접 지급하거나 회수하는 기능은 포함하지 않는다.

| 항목 | 값 |
|---|---|
| 영역 범위 | 응모권 발급·보유·사용 모니터링 + GP 교환·앱 구매 운영 토글 |
| 미포함 항목 | 응모권 만료, 환수(부정 사용·CS 회수), 관리자 수동 지급 |
| 발급 출처 | **5종** (퀘스트 보상 / PM·ST 보상 / BIVE 보유혜택 / 덕력 랭킹 보상 / GP로 응모권 구매) |
| 사용 출처 | 1종 (래플 응모) |
| 운영 화면 수 | 3개 ([CEB-BO-RFT-101] 응모권 현황, [CEB-BO-RFT-201] 응모권 변동 내역, [CEB-BO-RFT-301] 응모권 발급 정책) + 정정 명세 1개 ([CEB-BO-RFT-302] PM·ST 게임 생성 응모권 토글 정정 명세) |

## 2. 화면 구성

본 영역은 운영 화면 **3개 + 정정 명세 1개**로 구성된다.

| 화면 ID | 화면명 | 용도 |
|---|---|---|
| [CEB-BO-RFT-101] | 응모권 현황 | 발급·보유·사용 총량 + 출처별 발급·사용 분포 모니터링 |
| [CEB-BO-RFT-201] | 응모권 변동 내역 | 시계열 변동 로그. 회원·출처·아티스트 컨텍스트별 필터링·검색 |
| [CEB-BO-RFT-301] | 응모권 발급 정책 | GP↔응모권 환율 + 회원당 1일 교환 한도 + 앱내 구매 운영 토글 + PM·ST 게임 보상 기본 정책 |
| [CEB-BO-RFT-302] | (정정 명세) PM·ST 게임 생성 응모권 토글 | RFT-301 §2-7 기본 매트릭스의 게임별 오버라이드 — PM·ST 운영 재개 시점에 BO-GAM 게임 생성 폼에 흡수 예정 |

**사이드바 배치**: UX 편의 사유로 사이드바는 "래플" 영역 하위 메뉴 위치에 배치된다 (래플 응모와 직접 연계되는 자산이므로). 사이드바 메뉴 라벨은 **응모권 현황 / 변동 내역 / 응모권 발급 정책** 3종. 정정 명세(RFT-302)는 별도 메뉴를 두지 않고 RFT-301 본문에서 링크로 연결한다. **관리·정책 측면에서는 독립 영역**으로 운영된다.

## 3. 권한

본 영역의 모든 화면·액션 권한은 **[관리자 > 권한관리]에서 설정된 권한을 따른다**. 권한이 없는 운영자가 진입한 경우 해당 화면·버튼은 미노출 또는 비활성 상태로 처리한다. 본 문서에서는 세부 역할 명세를 별도로 두지 않는다.

## 4. 출처 정의

응모권은 다음 6종 출처 중 하나로 변동 기록된다 (정규 발급 5종 + 사용 1종).

### 4-1. 정규 발급 출처 5종 (응모권 증가, 회원 노출 카드·필터)

| 출처 | 의미 | 변동 트리거 | SSOT 문서 |
|---|---|---|---|
| 퀘스트 보상 | 회원이 Quest 제출 후 운영자 승인 시 보상 발급 | Quest 검수 승인 | [CEB-BO-FQ-203] 처리 내역 §승인 |
| PM·ST 보상 | 게임별 응모권 지급 토글이 켜져 있을 때 종료 시 자동 발급 | PM 결과 발표 / ST 게임 종료 (생존자·탈락자) | [CEB-BO-RFT-301] §4 + [CEB-BO-RFT-302] |
| BIVE 보유혜택 | 회원이 보유한 BIVE 등급 기반 정기 발급 | 매일 자정 직후 자동 | [CEB-BO-BIVE-104] §혜택 관리 |
| 덕력 랭킹 보상 | 덕력 시즌 정산 시 티어·랭킹 보상으로 자동 발급 | 시즌 종료·정산 시점 | [CEB-BO-DUK-201] §2.2-C·D + [CEB-BO-DUK-000] §6.2 |
| GP로 응모권 구매 | 회원이 앱 응모권 구매 화면(Buy-RF-Ticket-001)에서 GP를 사용해 직접 구매 | 회원 구매 시점 (앱내 구매 운영 토글이 켜져 있을 때만) | [CEB-BO-RFT-301] §3 (앱내 구매 운영 토글) |

> ⚠️ **일일 미션은 응모권을 발급하지 않음** — 일일 미션 보상은 **덕력 25**로만 지급된다. [CEB-BO-FQ-501] 일일 미션 관리 §2.4 정합. 따라서 응모권 정규 발급 출처에서 제외.

### 4-2. 사용 출처 1종 (응모권 차감)

| 출처 | 의미 | 변동 트리거 |
|---|---|---|
| 래플 응모 사용 | 회원이 진행중인 래플에 응모권 사용 | [CEB-BO-RFL-202] 응모내역 발생 시점 |

### 4-3. 발급량·환율·운영 토글

- **퀘스트 보상 Quest당 발급량**, **BIVE 혜택 등급별 발급량**, **덕력 시즌 티어·랭킹별 보상량**은 각 SSOT 문서(§4-1)에서 별도 운영 설정값으로 관리. 본 영역에서는 변경할 수 없음.
- **PM·ST 게임별 응모권 지급 여부 + 수량**은 **[CEB-BO-RFT-301] §4 발급 정책의 기본값 매트릭스**로 SSOT를 두고, **게임별 오버라이드는 [CEB-BO-RFT-302]가 정의하는 게임 생성 폼(BO-GAM-201-CREATE) 토글**에서 설정.
- **GP → 응모권 환율**, **회원당 1일 교환 한도**, **앱내 구매 운영 토글(ON/OFF + 점검 사유)**은 **[CEB-BO-RFT-301] 응모권 발급 정책 화면에서 운영자가 직접 조정**. 변경 즉시 회원 앱에 반영.

## 5. 정책 상수

응모권 영역 운영 정책은 다음과 같다.

- **만료 없음 (영구 보유)** — 한 번 발급된 응모권은 회원이 사용·소진하기 전까지 영구 유지. 시간 경과로 자동 소멸하지 않음
- **환수 기능 없음** — 부정 사용·CS 회수 등으로 응모권을 회수해야 하는 경우 운영자가 백오피스에서 직접 처리하지 않고, 데이터팀의 백엔드 직접 보정으로 처리
- **관리자 수동 지급 없음** — 응모권은 §4-1 정규 발급 출처 5종으로만 발급된다. 이벤트·보상 등의 사유로 운영자가 회원에게 직접 응모권을 지급할 수 없음
- **변동 데이터 영구 보존 (append-only)** — 응모권 변동 로그는 수정·삭제되지 않는다. 모든 발급·사용 이력은 시계열로 영구 보존
- **공통 페이지네이션 패턴**: 모든 리스트 테이블에 **페이지당 20건**, **항상 노출**(1페이지일 때도 `‹ 1 ›` 비활성). **윈도잉 규칙**: 총 페이지 ≤ 7이면 전체 노출, 8 이상이면 `1 ... cur-1 cur cur+1 ... last` 윈도우 + `...`. 필터·검색 변경 시 1페이지 자동 리셋 ([CEB-BO-012] §5-2 공통 페이지네이션 정합)
- **1일 기준 = KST 자정** (00:00 ~ 23:59:59). 회원당 1일 교환 한도는 매일 자정에 자동 리셋. 환율·한도 모두 [CEB-BO-RFT-301]에서 운영자가 조정
- **앱내 구매 운영 토글** — [CEB-BO-RFT-301] §3에서 운영자가 즉시 ON/OFF. OFF 시 앱 Buy-RF-Ticket-001 화면의 [구매하기] 버튼이 [점검중] 텍스트로 비활성. 변경 즉시 회원 앱에 반영(캐시 TTL 60초)

## 6. 라이프사이클

응모권은 다음 3상태를 거친다.

```
[정규 발급 출처 5종]
   ↓ 발급
발급 ───── 회원 보유 ─────→ 보유 ───── 래플 응모 ─────→ 사용
```

| 상태 | 의미 |
|---|---|
| 발급 | 정규 발급 출처 5종 중 하나가 트리거되어 회원에게 응모권이 지급된 시점 |
| 보유 | 발급 후 회원 잔액으로 누적되어 있는 상태 (만료 없음, 영구 유지) |
| 사용 | 회원이 래플 응모에 응모권을 차감한 시점 |

> **상태 머신 보조 정책**: 정규 발급·사용 모두 시스템 자동 트리거다. 운영자가 상태를 강제로 전환할 수 없다. 부정 사용 회수·CS 보정이 필요한 경우 백오피스에서 직접 처리하지 않고 데이터팀의 백엔드 직접 보정으로 처리한다 (§5 정책 참조).

## 7. 활동 로그

응모권 변동은 시스템 자동 기록과 운영자 토글 변경 로그 2종으로 구성된다. 회원 자산 변동 로그는 [CEB-BO-RFT-201] 응모권 변동 내역에서 시계열로 조회 가능.

### 7-1. 자산 변동 로그 (시스템 자동)

정규 발급 5종 + 사용 1종 모두 시스템 자동 트리거. 활동 로그는 동일한 변동 로그 스트림에 기록되며 [CEB-BO-RFT-201]에서 조회.

| 분류 | 메시지 형식 |
|---|---|
| 발급 (정규) | 퀘스트 보상으로 회원 '{닉네임}'에게 응모권 {N}장 발급 |
| 발급 (정규) | PM·ST 보상('{게임 타이틀}')으로 회원 '{닉네임}'에게 응모권 {N}장 발급 |
| 발급 (정규) | BIVE 보유혜택으로 회원 '{닉네임}'에게 응모권 {N}장 발급 |
| 발급 (정규) | 덕력 시즌 정산('{시즌명}')으로 회원 '{닉네임}'에게 응모권 {N}장 발급 |
| 발급 (정규) | 회원 '{닉네임}'이 GP {M}을(를) 사용해 응모권 {N}장 구매 |
| 사용 (정규) | 래플 '{타이틀}'에 회원 '{닉네임}'이 응모권 {N}장 사용 |

### 7-2. 정책 변경 로그 (운영자 액션, [CEB-BO-RFT-301] 트리거)

| 분류 | 메시지 형식 | 권한 |
|---|---|---|
| GP 환율 변경 | GP → 응모권 환율을 {기존} → {신규}로 변경했습니다. (변경자: {닉네임}) | RFT 영역 쓰기 권한 (SUPER 또는 운영자 정의 그룹) |
| 1일 교환 한도 변경 | 회원당 1일 교환 한도를 {기존} → {신규}로 변경했습니다. (변경자: {닉네임}) | RFT 영역 쓰기 권한 |
| 앱내 구매 토글 변경 | 앱내 응모권 구매를 {ON → OFF / OFF → ON}로 전환했습니다. (변경자: {닉네임}, 사유: {요약}) | RFT 영역 쓰기 권한 |
| PM·ST 게임 응모권 정책 변경 | PM·ST 게임별 응모권 지급 기본 정책을 변경했습니다. (변경 항목: {필드 목록}) | RFT 영역 쓰기 권한 |
