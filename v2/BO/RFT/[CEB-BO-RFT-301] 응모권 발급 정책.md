## 1. 화면 개요

운영자가 응모권 발급·구매·교환과 관련된 **운영 정책 4종**을 한 화면에서 직접 조정하는 설정 화면.

1. **GP → 응모권 교환 환율**
2. **회원당 1일 교환 한도**
3. **앱내 응모권 구매 운영 토글** — 점검·장애 시 앱의 [구매하기] 버튼을 즉시 [점검중]으로 비활성. 앱 측 화면(Buy-RF-Ticket-001) 자체의 구성 명세는 별도
4. **PM·ST 게임별 응모권 지급 기본 정책** — 게임유형별 기본 지급 여부·수량. 게임별 오버라이드는 [CEB-BO-RFT-302] 정정 명세 참조

변경 사항은 저장 즉시 회원 앱·운영자 화면에 반영된다. 사용량 통계 조회는 [CEB-BO-RFT-101] / [CEB-BO-RFT-201]에서 별도 제공.

## 2. 화면 구성

### 2-1. 헤더 + 1일 기준 안내 박스

- 페이지 제목 "응모권 발급 정책"
- 부제: "GP를 응모권으로 교환할 때 적용되는 환율과 회원당 1일 교환 한도를 운영자가 직접 조정합니다."
- 안내 박스 (인디고 톤): "**1일 기준**: KST 자정(00:00 ~ 23:59:59). 매일 자정에 회원별 한도가 자동 리셋됩니다."

### 2-2. 카드 1 — GP → 응모권 환율

| 요소 | 내용 |
|---|---|
| 라벨 | "GP → 응모권 환율" |
| 필수 표시 | 우측 상단 "필수" 텍스트 |
| 입력 | 숫자 인풋 + 고정 텍스트 "GP → 응모권 1장" |
| 검증 | 양의 정수 (≥ 1). 미충족 시 입력칸 rose 외곽선 + "최소 1 이상의 양의 정수만 입력 가능합니다." 에러 |
| 안내 | "변경 즉시 회원 앱의 GP 교환 화면에 반영됩니다." |

### 2-3. 카드 2 — 회원당 1일 교환 한도

| 요소 | 내용 |
|---|---|
| 라벨 | "회원당 1일 교환 한도" |
| 필수 표시 | 우측 상단 "필수" 텍스트 |
| **모드 라디오 그룹** | "한도 적용" / "무제한" **(기본)** |
| 입력 (한도 적용 모드만) | 숫자 인풋 + 고정 텍스트 "장 / 1일" |
| 입력 (무제한 모드) | 입력칸 회색·비활성 처리 |
| 검증 | 한도 적용 모드일 때 양의 정수 (≥ 1). 무제한 모드는 검증 우회 |
| 안내 | "회원 1명이 KST 자정 기준 1일에 교환받을 수 있는 응모권 최대 장수. 한도 초과 시 회원 앱에서 교환이 차단되고 '오늘 한도 도달. 내일 00:00에 리셋' 안내가 노출됩니다. **무제한 선택 시 한도 검증을 우회하여 회원이 GP만 충분하면 제한 없이 교환 가능**합니다." |
| 마지막 변경 정보 | 카드 하단에 "마지막 변경 (GP 환율·한도): **{닉네임}** · YYYY.MM.DD HH:mm" 표기 (카드 1과 카드 2는 같은 정책 묶음이므로 카드 2 하단에 통합) |

### 2-4. 저장 버튼

- 우측 정렬 [저장하기] 버튼 — 4개 카드 중 변경값이 있을 때 활성 (GP 환율 / 1일 한도 / 앱내 구매 토글 / PM·ST 게임 정책 중 하나라도 변경되면 활성)
- 각 카드별 마지막 변경 정보는 해당 카드 하단에 위치 (카드 1·2 묶음은 §2-3 카드 2 하단, 카드 3·4는 각 카드 하단)

### 2-5. 저장 확인 모달 (`md` 사이즈)

[저장하기] 클릭 시 노출. 변경된 항목만 메시지에 표시:

| 변경 유형 | 메시지 형식 |
|---|---|
| 환율만 변경 | "GP 환율을 {기존} GP → {신규} GP로 변경합니다." |
| 한도만 변경 (제한 → 제한) | "회원당 1일 교환 한도를 {기존}장 → {신규}장으로 변경합니다." |
| 한도만 변경 (제한 → 무제한) | "회원당 1일 교환 한도를 {기존}장 → **무제한**으로 변경합니다." |
| 한도만 변경 (무제한 → 제한) | "회원당 1일 교환 한도를 **무제한** → {신규}장으로 변경합니다." |
| 앱내 구매 토글 변경 | "앱내 응모권 구매를 **{기존 ON/OFF} → {신규 OFF/ON}**으로 전환합니다." (사유: {요약}) |
| PM·ST 정책 변경 | "PM·ST 게임별 응모권 지급 기본 정책을 변경합니다. (변경 항목: {필드 목록})" |
| 여러 항목 변경 | 변경된 모든 항목을 줄바꿈으로 나열 |

버튼: [취소] / [변경하기] (인디고)

### 2-6. 카드 3 — 앱내 응모권 구매 운영 토글

| 요소 | 내용 |
|---|---|
| 라벨 | "앱내 응모권 구매 (Buy-RF-Ticket-001)" |
| 부제 | "회원이 앱에서 GP를 사용해 응모권을 직접 구매할 수 있는 기능의 운영 상태." |
| 토글 | "[● ON] 정상 운영 / [○ OFF] 점검중" (라디오 또는 스위치, 기본 ON) |
| 점검 사유 입력 | OFF 선택 시 활성. 텍스트 200자 (선택 입력, 운영 로그 보존용). 예시 placeholder: "결제 게이트웨이 점검으로 일시 중단" |
| 마지막 변경 정보 | "마지막 변경: **{닉네임}** · YYYY.MM.DD HH:mm · 사유: {요약}" |
| 안내 박스 | "OFF 전환 시 앱 Buy-RF-Ticket-001 화면의 [구매하기] 버튼이 즉시 **[점검중]** 텍스트로 비활성됩니다 (탭 시 토스트 안내). 변경은 저장 즉시 회원 앱에 반영됩니다." |

### 2-7. 카드 4 — PM·ST 게임별 응모권 지급 기본 정책

게임 종료 시 응모권 자동 지급 여부와 기본 수량을 게임유형별로 설정. 개별 게임에서 다른 값을 사용하려면 [CEB-BO-RFT-302]에 정의된 게임 생성 폼(BO-GAM-201-CREATE) 토글에서 오버라이드.

| 게임유형 | 기본 지급 (토글) | 기본 수량 (입력) | 비고 |
|---|---|---|---|
| Prediction Market | [● ON] | 0매 (기본) | 게임별로 운영자가 수량 지정 |
| Survival Trivia (생존자) | [● ON] | 0매 (기본) | 게임별로 운영자가 수량 지정 |
| Survival Trivia (탈락자) | [● ON] | 0매 (기본) | 게임별로 운영자가 수량 지정 |

> 기본값은 모두 ON이지만 수량 0매이므로 실질 지급 0매. 게임 생성 시 운영자가 [CEB-BO-RFT-302]에서 수량을 지정해야 실제로 지급됨. ON 유지 + 수량 0 조합은 저장 시 안내 토스트 노출 ("0매 지급은 OFF와 동일합니다. 진행하시겠습니까?").

| 요소 | 내용 |
|---|---|
| 검증 | 토글 ON일 때 수량은 0 이상 정수. 수량 0 + ON 조합은 저장 시 안내 토스트 ("0매 지급은 OFF와 동일합니다. 진행하시겠습니까?") |
| 안내 박스 | "본 매트릭스는 **모든 PM·ST 게임의 기본값**입니다. 개별 게임에서 다른 값을 사용하려면 게임 생성 폼에서 오버라이드하세요 ([CEB-BO-RFT-302] 참조). 현재 PM·ST는 운영 0건 휴면 상태이므로 정책은 향후 운영 재개 시점에 적용됩니다." |
| 마지막 변경 정보 | "마지막 변경: **{닉네임}** · YYYY.MM.DD HH:mm" |

## 4. 상태별 동작 분기

본 화면은 **단일 설정 화면**이므로 화면 자체에 상태 머신이 없다. 데이터·입력 조건에 따른 분기만 존재.

| 조건 | 동작 |
|---|---|
| 환율·한도 양호 + 변경사항 있음 | [저장하기] 활성 |
| 환율 < 1 또는 정수 아님 | [저장하기] 비활성 + 환율 입력칸 에러 표시 |
| 한도 적용 모드 + 입력값 < 1 또는 정수 아님 | [저장하기] 비활성 + 한도 입력칸 에러 표시 |
| 무제한 모드 | 한도 숫자 입력 비활성·회색 처리. 검증 우회 |
| 앱내 구매 토글 OFF 선택 | 점검 사유 입력란 활성 (선택 입력) |
| 앱내 구매 토글 ON 선택 | 점검 사유 입력란 비활성·회색 처리 |
| PM·ST 게임 토글 ON + 수량 0 | 저장 시 안내 토스트 ("0매 지급은 OFF와 동일합니다. 진행하시겠습니까?") + 한 번 더 [확인]·[취소] 분기 |
| PM·ST 게임 토글 OFF | 수량 입력 비활성·회색 처리 |
| 변경사항 없음 (모든 입력 = 현재값) | [저장하기] 비활성 |
| 권한 없음 운영자 진입 | [관리자 > 권한관리] 설정에 따라 화면 미노출 또는 접근 차단 |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 환율 입력 | **양의 정수만 허용** (≥ 1). 소수·0·음수·문자 입력 불가 |
| 한도 입력 (제한 모드) | **양의 정수만 허용** (≥ 1). 동일 검증 |
| 한도 무제한 모드 | 숫자 입력 비활성. 한도 검증 우회 |
| 1일 기준 | **KST 자정 (00:00 ~ 23:59:59)** 고정. 매일 자정 자동 리셋. 시간대 변경 불가 |
| 한도 초과 시 회원 앱 동작 | GP 교환 시도 차단 + "오늘 한도 도달. 내일 00:00에 리셋" 안내 노출. 다음 자정에 회원별 카운터 자동 0 리셋 |
| 앱내 구매 토글 OFF 시 회원 앱 동작 | Buy-RF-Ticket-001 화면의 [구매하기] 버튼이 **[점검중]** 텍스트로 비활성 (회색). 탭 시 토스트 "현재 응모권 구매가 일시적으로 중단되었습니다." 노출. 점검 사유는 회원에게 노출하지 않음 (운영 로그 보존용) |
| 앱내 구매 토글 변경 권한 | 본 화면 진입 권한자라면 누구나 가능. 단 변경 즉시 활동 로그 자동 기록 (변경자·이전·신규·사유) |
| PM·ST 정책 적용 시점 | PM·ST 게임 종료 시 시스템이 본 매트릭스를 조회해 응모권 자동 지급. 게임별 오버라이드값이 있으면 우선 적용 ([CEB-BO-RFT-302]) |
| 변경 적용 시점 | 저장 확인 모달 [변경하기] 클릭 즉시 시스템 정책 갱신. 회원 앱에 실시간 반영 |
| 변경 이력 | 각 카드별 마지막 수정자·일시 화면 노출. 전체 변경 이력은 활동 로그 영역에서 별도 추적 ([CEB-BO-010] §7-2 정합) |
