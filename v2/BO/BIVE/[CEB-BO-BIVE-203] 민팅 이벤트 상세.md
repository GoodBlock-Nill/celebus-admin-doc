## 1. 화면 개요

특정 **민팅 이벤트**의 상세를 조회·관리한다. 4상태(초안/활성/중지/종료) × 4유형(Event/Ticket/Mix/Pick) 매트릭스 적용. 3탭(기본정보·BIVE 보상·보상내역) 구조이며, 상태에 따라 헤더 액션·필드 잠금이 분기된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/minting/{campaignId}` |
| Breadcrumb | 홈 / BIVE / 민팅 관리 / {Event 또는 Ticket} / 캠페인 상세 |
| 페이지 제목 | `{캠페인명}` |

## 2. 화면 구성

### 2-1. 헤더 (상태별 액션 분기)

- 좌측: 탭 네비게이션 "기본정보 | BIVE 보상 | 보상내역"
- 우측 액션 (상태별):

| 상태 | 헤더 액션 |
|---|---|
| **초안** | [삭제] (삭제 모달) + [저장] + [활성화] (활성화 모달) |
| **활성** | [일시중지] (일시중지 모달) |
| **중지** | [저장] + [활성화] (활성화 모달) |
| **종료** | 없음 (전체 읽기 전용) |

### 2-2. 기본정보 탭

| 필드 | 입력 방식 | 잠금 조건 |
|---|---|---|
| 캠페인명 | Text Input | 초안만 수정 가능. 활성/중지/종료 read-only |
| 연결 기능 | Select (개발 프리셋) | 초안만 수정 가능. 활성/중지/종료 read-only |

**관리 정보 카드** (우측):

| 라벨 | 값 |
|---|---|
| 상태 | 초안(노랑) / 활성(녹색) / 중지(빨강) / 종료(회색) 배지 |
| 생성 관리자 | 최초 생성 (수정 불가) |
| 생성 일시 | YYYY.MM.DD hh:mm (수정 불가) |
| 최근 수정자 | 저장/활성화/일시중지 시 갱신 |
| 최근 수정 일시 | (동일) |

### 2-3. BIVE 보상 탭

- 안내: "보상으로 지급되는 BIVE를 추가하고 각 항목에 가중치를 입력하세요."
- 우측 상단 카운터: **가중치 합: n**
- 우측 상단 액션 [+ 추가하기] (초안·중지에서만 활성): 클릭 시 [CEB-BO-BIVE-203-MD-ADD] BIVE 보상 추가 모달 오픈

**BIVE 보상 목록 테이블**:

| 컬럼 | 설명 |
|---|---|
| BIVE 명칭 | 자동 생성 `{아티스트명} {등급}-{등급번호}` |
| 아티스트 그룹 | 아티스트명 |
| 아티스트 | 아티스트명 |
| 등급 | Event/Normal/Special/Wicked/Next/Final |
| 등급번호 | 5자리 숫자 |
| 가중치 | 정수 1~999. 입력 가능 여부 상태에 따라 분기. placeholder "1-999" |
| 비중 | `가중치 / 가중치합` 분수 표기 (예: 10/55) — 읽기 전용, 실시간 재계산 |
| 관리 | X(삭제) 아이콘. 상태별 분기 |

**상태별 BIVE 보상 탭 동작**:

| 상태 | [+ 추가하기] | 가중치 입력 | 행 삭제(X) |
|---|---|---|---|
| 초안 | 활성 | 가능 | 가능 |
| 활성 | 비활성 | read-only | 비활성 |
| 중지 | 활성 | 가능 | 가능 |
| 종료 | 없음 | read-only | 없음 |

### 2-4. 보상내역 탭 (활성·중지·종료에서만 의미 있음)

활성 캠페인의 BIVE별 실제 민팅 현황 모니터링.

**필터/조회 영역**:

- 검색 입력 "BIVE명칭 입력": 부분 일치
- [조회] 버튼

**테이블**:

| 컬럼 | 설명 |
|---|---|
| BIVE 명칭 | 하이퍼링크 — 클릭 시 [CEB-BO-BIVE-202] BIVE 상세 (민팅이력 탭) 새 창. 제거된 항목은 우측에 회색 "제거됨" 배지 |
| 민팅 수 | 본 캠페인을 통해 발행된 누적 수량 |
| 가중치 | BIVE 보상 탭의 현재 가중치 |
| 가중치 비중 | BIVE 보상 탭의 현재 비중 |

**정렬**:

- 기본: 가중치 오름차순 → 등급/번호 순. 열 헤더 클릭 시 토글

**페이지네이션 하단 20행/페이지**.

빈 상태: "등록된 BIVE 보상이 없습니다."

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 활성화 선행 조건 | 기본정보 + BIVE 보상 모두 **저장 완료** + BIVE 보상 ≥ **1건** |
| 활성화 실패 (저장 안 함) | 토스트 "활성화 실패. 저장되지 않은 정보가 있습니다." — 저장 → 활성화 흐름으로 진행 필요 |
| 가중치 입력 | 정수만, 1~999 |
| 가중치 합 재계산 | 입력 즉시 가중치 합·각 행 비중 재계산 (분수 표기 예: 10/55) |
| BIVE 중복 방지 | 동일 BIVE 중복 등록 불가. 모달에서 이미 등록된 항목은 체크박스 비활성 |
| 종료 처리 | 종료는 운영자 직접 처리 불가 — 개발팀 수동 처리 |
| 권한 | [관리자 > 권한관리] BIVE 쓰기 권한 보유자만 헤더 액션 노출 |
