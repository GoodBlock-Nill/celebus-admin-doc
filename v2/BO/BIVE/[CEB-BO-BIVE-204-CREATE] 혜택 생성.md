## 1. 화면 개요

신규 **혜택**(BP 또는 RT)을 생성한다. 기본정보 + BIVE 추가 2탭 모두 입력 후 [생성하기] 클릭 시 **초안** 상태로 [CEB-BO-BIVE-104]에 추가된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/benefits/{type}/create` |
| Breadcrumb | 홈 / BIVE / 혜택 관리 / {Boost Point 또는 Raffle Ticket} / 혜택 생성 |
| 페이지 제목 | `{Boost point / Raffle Ticket} 혜택 생성` |
| 진입 경로 | [CEB-BO-BIVE-104] 혜택 리스트의 [+ 혜택 생성] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더 (공통)

- 좌측: 페이지 타이틀
- 우측 액션:
  - [취소하기]: [CEB-BO-BIVE-104]로 이동 (입력 폐기)
  - [생성하기] (Primary): 검증 통과 시 토스트 "생성 완료" → [CEB-BO-BIVE-104]로 이동 + 초안 상태로 행 추가
- 탭 네비게이션: "기본정보 | BIVE 추가"

### 2-2. 기본정보 탭

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 혜택 명칭 | Text Input | placeholder "혜택 명칭을 입력해 주세요". 공백 제외 1자 이상 | ★ |
| 수량 | Number Input | BP 탭: "BP 수량" 단위 "BP" / RT 탭: "Raffle ticket 수량". 정수 ≥ 1. 숫자 외 입력 제거 | ★ |
| 지급주기 | Select | **일일 / 주간 / 1회** | ★ |
| 지급 요일 (주간) | Select 단일선택 | 주간 선택 시에만 활성. 월~일 중 택1. 일일/1회는 비활성·회색 | ★ (주간일 때) |
| 시작일 | Date | YYYY.MM.DD. **오늘 이후** | ★ |
| 시작시간 | Time | hh:mm. 시작일 입력 후 활성. 시작일이 오늘이면 현재 시간 이후만 허용 | ★ |
| 종료일 | Date | YYYY.MM.DD. **시작일 + 1일 이후** | ★ |
| 종료시간 | Time | hh:mm. 종료일 입력 후 활성. 유효성 조건 없음 | — |

> 필드 배치: 주기/요일은 한 줄. 날짜/시간은 "시작일+시작시간" / "종료일+종료시간" 2열 구성.

### 2-3. BIVE 추가 탭

- 섹션 헤더: "BIVE 보상" + 가이드 "혜택을 적용할 BIVE를 추가해주세요."
- 우측 [+ 추가하기] 버튼: 클릭 시 [CEB-BO-BIVE-204-MD-ADD] BIVE 추가 모달 오픈

**테이블 컬럼**:

- BIVE 명칭 / 아티스트 / 멤버 / 등급 / 등급번호 / 관리 (X 삭제)

**정렬**: BIVE 생성일시 빠른순

**페이지네이션**: 테이블 영역 내 무한 스크롤

**빈 상태**: "추가하기 버튼을 눌러 BIVE를 등록하세요."

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 진입 직후 | 기본정보 탭 활성, 혜택 명칭 입력 포커스 |
| 지급주기 = 주간 선택 | 지급 요일 Select 활성 |
| 지급주기 = 일일/1회 선택 | 지급 요일 Select 비활성 (회색) |
| 시작일 입력 | 시작시간 Select 활성 |
| 종료일 입력 | 종료시간 Select 활성 |
| BIVE 추가 [+ 추가하기] 클릭 | [CEB-BO-BIVE-204-MD-ADD] 모달 오픈 |
| 모달에서 BIVE 선택 후 [추가] | 테이블에 행 추가 |
| 행 삭제 (X) | 즉시 행 제거 |
| [생성하기] 클릭 (모든 필수 충족) | 토스트 "생성 완료" → [CEB-BO-BIVE-104]로 이동 + 초안 상태로 행 추가 |
| [생성하기] 클릭 (필수 누락) | 토스트 "생성 실패: 설정되지 않은 정보가 있습니다." 화면 유지 |
| [생성하기] 클릭 (네트워크 오류) | 토스트 "생성 실패: 다시 시도해주세요." 화면 유지 |
| [취소하기] 클릭 | [CEB-BO-BIVE-104]로 이동 (입력 폐기) |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 필수 (기본정보) | 혜택 명칭·수량·지급주기·시작일·시작시간·종료일 모두 입력. 주간이면 지급 요일도 필수 |
| 수량 검증 | 정수만, 1 이상 |
| 시작일 검증 | 오늘 이후 |
| 시작시간 검증 | 시작일이 오늘인 경우 현재 시간 이후만 허용 |
| 종료일 검증 | 시작일 + 1일 이후 (즉 `종료일 ≥ 시작일 + 1일`) |
| 필수 (BIVE) | BIVE 목록 ≥ 1건 |
| BIVE 중복 방지 | 동일 혜택 내 동일 BIVE 중복 불가 (모달에서 비활성) |
| 권한 | [관리자 > 권한관리] BIVE 쓰기 권한 보유자만 진입 |
