## 1. 화면 개요

회원에게 BIVE NFT를 자동 민팅하는 **민팅 이벤트**을 유형별로 조회·생성·상태 관리하는 SSOT 화면. SQ/RFL/FQ 등 외부 영역은 본 화면에 등록된 "활성" 상태 캠페인을 드롭다운으로 참조한다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/minting` |
| Breadcrumb | 홈 / BIVE / 민팅 관리 |
| 페이지 제목 | 민팅 관리 |
| 진입 경로 | 사이드바 > BIVE > 민팅 관리 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| SSOT 정책 | 본 영역에 등록된 캠페인만 SQ/RFL/FQ에서 참조 가능 ([CEB-BO-004] §5 정합) |

## 2. 화면 구성

### 2-1. 상단 탭

4종 캠페인 유형별 탭으로 화면 분리. 본 명세는 공통 구조 정의 (각 탭별 차이는 §4-2에 명시).

| 탭 | 의미 | 연결 기능 예시 |
|---|---|---|
| **Event** (기본) | 일반 이벤트형 자동 민팅 | 운영 BO 실제 4종 — 회원가입 보상 / 출석체크 보상 / 래플 보상 / 팬퀘스트 보상 |
| **Ticket** | 티켓 검증 기반 민팅 | 공연·이벤트 티켓 검증 후 자동 에어드랍 (운영 프리셋 별도) |
| **Mix** | 회원이 BIVE를 합성(Mix)할 때 자동 민팅 | (개발 프리셋) |
| **Pick** | 회원이 BIVE를 픽(Pick)할 때 자동 민팅 | (개발 프리셋) |

### 2-2. 필터·검색 영역

- **정렬 드롭다운**: `생성일(빠른순) / 생성일(오래된순)` (운영 BO 실제 표기, 기본값 "생성일(빠른순)")
- **상태 필터** (드롭다운): `상태(전체) / 초안 / 활성 / 중지 / 종료` (운영 BO 실제 표기, 기본값 "상태(전체)")
- **검색 입력**: placeholder "캠페인명 입력" — 키워드 부분 일치
- **[조회] 버튼**: 운영 BO는 별도 [조회] 버튼 없이 필터·검색 입력 즉시 자동 갱신
- **[+ 캠페인 생성] 버튼** (우측 상단): 현재 탭의 캠페인 생성 페이지로 이동
  - Event 탭 → [CEB-BO-BIVE-203-CREATE] (Event 변형)
  - Ticket 탭 → [CEB-BO-BIVE-203-CREATE] (Ticket 변형)

### 2-3. 데이터 테이블

| 컬럼 | 설명 |
|---|---|
| ID | 내부 일련번호 |
| 상태 | 배지 — 초안(노랑) / 활성(녹색) / 중지(빨강) / 종료(회색) |
| 캠페인명 | 운영자가 설정한 캠페인 이름 |
| 보상 방식 (v1.2) | 배지 — 가중치(indigo) / 지정(emerald). 사이클 #1 보상 방식 이분화 정합 |
| 연결 기능 | 자동 민팅 트리거 (개발 프리셋, 캠페인:연결기능 = 1:1) |
| 등록된 BIVE 수 | 본 캠페인에 보상으로 연결된 BIVE 개수 |
| 발행 수 | 본 캠페인을 통한 누적 민팅 수 (없으면 "-") |
| 생성일 | `YYYY.MM.DD` |

**행 인터랙션**:

- 관리 컬럼 없음 (행 클릭으로만 진입)

## 4. 상태별 동작 분기

### 4-1. 캠페인 상태 정의 + 전이

| 상태 | 의미 | 전이 가능 |
|---|---|---|
| **초안** | 생성 직후. 기본정보·BIVE 보상 모두 수정 가능 | → 활성 (초안으로 되돌리기 불가), → 삭제 |
| **활성** | 활성화 후. 회원 액션 발생 시 자동 민팅 작동. 모든 정보 수정 불가 | → 중지 |
| **중지** | 일시중지. 회원 액션 발생해도 민팅 안 됨. BIVE 보상만 수정 가능 | → 활성 |
| **종료** | 완전 종료. 모든 정보 읽기 전용 | (전이 없음 — 개발팀 수동 처리) |

### 4-2. 화면 동작

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 테이블 |
| 데이터 없음 | "등록된 캠페인이 없습니다." 안내 |
| 검색·필터 결과 없음 | "조건에 맞는 캠페인이 없습니다." 안내 |
| 행 클릭 | [CEB-BO-BIVE-203] 캠페인 상세 진입 |
| [+ 캠페인 생성] 클릭 | 현재 탭에 맞는 캠페인 생성 페이지 진입 |
| 권한 없음 진입 | 화면 미노출 또는 [+ 캠페인 생성] 비활성 |

## 5. 검증 규칙

| 검증 | 내용 |
|---|---|
| 검색 | 캠페인명 키워드 부분 일치 |
| 정렬 | `생성일(빠른순)` 기본 또는 `생성일(오래된순)` (운영 BO 실제 표기) |
| 캠페인 활성화 선행 조건 (상세에서 검증) | 기본정보 + BIVE 보상 모두 저장 완료 + BIVE 보상 ≥ 1건 ([CEB-BO-BIVE-203]에서 검증) |
| 권한 | [관리자 > 권한관리] BIVE 쓰기 권한 보유자에게만 [+ 캠페인 생성] 노출 |
| SSOT 정책 | 외부 영역(SQ/RFL/FQ)은 본 화면에 등록된 "활성" 상태 캠페인만 참조. 캠페인 정의는 본 영역에서만 가능 |

## 6. 외부 영역 연동

| BIVE 관리 (BIVE-102) | 캠페인이 보상으로 등록한 BIVE의 활성 여부 검증 | BIVE 보상 추가 모달 [CEB-BO-BIVE-203-MD-ADD]에서 "활성" BIVE만 선택 가능 |
