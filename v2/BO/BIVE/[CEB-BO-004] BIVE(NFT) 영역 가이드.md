## 1. 영역 개요

CELEBUS 플랫폼의 **BIVE(NFT)** 영역은 회원에게 발행되는 디지털 자산(BIVE NFT)의 마스터·민팅 트리거·혜택 정책을 관리한다. SQ 미션 보상·RFL 당첨자 보상·앱 내 보상 이벤트 등에서 **자동 민팅의 SSOT**로 동작한다.

**핵심 계층** (v1 기준 그대로):

```
에디션 (=프로젝트) — 다국어 명칭만 보유
  └─ BIVE N개 — 아티스트 × 등급 × 등급번호 + 보내기/Mix/Pick 기능 ON/OFF
       └─ 민팅 이벤트 N:M — Event/Ticket/Mix/Pick 4유형
              └─ 회원 자동 민팅 (캠페인 "연결 기능" 트리거)

별도 흐름:
혜택 (Boost Point + Raffle Ticket) — BIVE 보유 조건 → 일일/주간/1회 자동 지급 (00:05 배치)
```

**운영 의도**:

- **에디션**은 BIVE 묶음의 큐레이션 단위 (예: V01D 시즌 1 에디션 등). 다국어 명칭만 등록
- **BIVE**는 실제 NFT 자체. 아티스트·등급·등급번호 조합으로 식별 (예: "하시모토 칸나 Normal-001"). 등록 후 활성화하면 민팅 이벤트에 보상으로 등록 가능
- **민팅 이벤트**은 회원의 특정 액션(연결 기능)이 발생하면 자동으로 등록된 BIVE 중 가중치 기반 추첨으로 1개를 발행
- **혜택**은 BIVE 보유자에게 별도로 지급되는 Boost Point(BP) 또는 Raffle Ticket(RT). 일일/주간/1회 자동 지급

```text
BIVE
├── 에디션 관리          → /bive/editions
│   ├── 에디션 리스트       → [CEB-BO-BIVE-101] /bive/editions
│   ├── 에디션 생성/수정 모달 → [CEB-BO-BIVE-201-CREATE] / [CEB-BO-BIVE-201-EDIT] (모달)
│   └── BIVE 관리          → [CEB-BO-BIVE-102] /bive/editions/{editionId}
│       ├── BIVE 등록      → [CEB-BO-BIVE-202-CREATE] /bive/editions/{editionId}/create
│       ├── BIVE 상세      → [CEB-BO-BIVE-202] /bive/editions/{editionId}/bives/{biveId} (기본·기능·민팅관리·민팅이력 4탭)
│       └── BIVE 삭제 모달  → [CEB-BO-BIVE-202-MD-DELETE] (모달)
│
├── 민팅 관리            → /bive/minting
│   ├── 민팅 이벤트 리스트   → [CEB-BO-BIVE-103] /bive/minting (Event/Ticket/Mix/Pick 4탭)
│   ├── 캠페인 생성        → [CEB-BO-BIVE-203-CREATE] /bive/minting/create?type=EVENT (기본정보·BIVE보상 2탭)
│   ├── 캠페인 상세        → [CEB-BO-BIVE-203] /bive/minting/{campaignId} (기본정보·BIVE보상·보상내역 3탭)
│   └── BIVE 보상 추가 모달 → [CEB-BO-BIVE-203-MD-ADD] (모달)
│
└── 혜택 관리            → /bive/benefits
    ├── 혜택 리스트         → [CEB-BO-BIVE-104] /bive/benefits (Boost Point/Raffle Ticket 2탭)
    ├── 혜택 생성          → [CEB-BO-BIVE-204-CREATE] /bive/benefits/create?type=boostPoint (기본정보·BIVE추가 2탭)
    ├── 혜택 상세          → [CEB-BO-BIVE-204] /bive/benefits/{benefitId} (기본정보·BIVE추가 2탭)
    └── BIVE 추가 모달      → [CEB-BO-BIVE-204-MD-ADD] (모달)
```

> **운영 BO 라우트 정합 (v1.5)**: 모든 화면 경로는 운영 BO 실제 형식과 1:1 일치. 캠페인·혜택 유형(EVENT/TICKET/MIX/PICK, boostPoint/ticket)은 생성 시 쿼리 파라미터(`?type=`)로 전달하고, 상세 진입 후에는 ID만 사용 (유형 구분은 헤더 페이지 제목으로 표시).

## 3. 권한

본 영역의 화면 접근 + 액션 권한은 **[관리자 > 권한관리]에서 설정된 BIVE 대메뉴의 읽기/쓰기 권한을 따른다**. 별도 역할 enum은 본 가이드에서 정의하지 않는다.

> v1에서 운영된 권한 매트릭스 (Super Admin / Admin 쓰기 / Admin 읽기 / 없음)는 [관리자 > 권한관리] 화면의 BIVE 항목 설정으로 일원화. 본 영역의 모든 화면에서 [수정/생성/삭제] 액션은 쓰기 권한 보유자에게만 노출.

## 5. 정책 상수

본 영역 운영 정책 및 한도는 다음과 같다.

| 정책 | 값·정의 |
|---|---|
| **에디션 다국어 명칭** | **한국어·영어·일본어 3종 모두 필수** (SQ 영역 §5 다국어 필수 표준과 정합). 에디션 자체는 명칭 외 추가 데이터 없음 |

| **BIVE 등급** | **6종 고정**: Event / Normal / Special / Wicked / Next / Final |
| **BIVE 등급번호** | 최대 5자리 숫자 (예: 001, 12345) |
| **BIVE 고유성** | 동일 에디션 내에서 (아티스트, 등급, 등급번호) 조합은 유니크 |
| **BIVE 상태** | **3상태**: Draft(초안) / Active(활성) / Inactive(비활성). Active 후 정보 수정 불가 |

| **BIVE 미디어 — 이미지 타입** (v1.12) | 메인 이미지(★) + 서브 이미지(선택). 둘 다 PNG·JPG·GIF·WebP, 최대 20MB |
| **BIVE 미디어 — 영상 타입** (v1.12) | 영상(★) + 썸네일(★). 영상은 MP4만, 최대 50MB. 썸네일은 PNG·JPG·GIF·WebP, 최대 20MB (리스트·앱 미리보기 표시용) |
| **BIVE 미디어 — 음성 타입** (v1.12) | 음성(★) + 썸네일(★). 음성은 MP3만, 최대 10MB. 썸네일은 PNG·JPG·GIF·WebP, 최대 20MB |

| **BIVE 기능 토글** | 보내기(Send) / Mix / Pick — 3종 모두 기본 ON. 운영자가 개별 ON/OFF |
| **민팅 이벤트 유형** | **4종 고정**: Event / Ticket / Mix / Pick |
| **민팅 이벤트 상태** | **4상태**: 초안 / 활성 / 중지 / 종료. 활성→초안 되돌리기 불가. 종료는 개발팀 수동 처리 |

| **민팅 이벤트 보상 방식** (v1.6 신규) | **2종 XOR** — `WEIGHTED`(가중치 보상) / `FIXED`(지정 보상). 캠페인당 1개만 선택. 초안·중지에서만 전환 가능. 활성·종료 read-only |
| **WEIGHTED (가중치 보상)** | 트리거 → 가중치 분포 추첨 → BIVE **1개** 발행. 가중치 1~999 정수, 비중 = 가중치/가중치합 |
| **FIXED (지정 보상)** | 트리거 → 등록된 모든 BIVE(1~10종) **동시** 발행. 가중치·비중 무관. **최대 10종** (`FIXED_REWARD_MAX = 10`) |
| **보상 방식 전환 정책** | 등록 BIVE 목록 그대로 유지. WEIGHTED→FIXED 전환 시 11종 이상이면 경고 + 상위 10종만 유지. 활성·종료 캠페인은 전환 불가 |
| **보상 방식 배지 노출 표준** (v1.11) | 가중치 보상 = `bg-indigo-100 text-indigo-700` "가중치" / 지정 보상 = `bg-emerald-100 text-emerald-700` "지정". 노출 위치 — **[103] 민팅 이벤트 리스트 테이블 + [203] 캠페인 상세 §BIVE 보상 탭 라디오 선택 UI** (캠페인 단위 속성이므로 캠페인 컨텍스트에서만 노출). [202] BIVE 상세 §민팅관리 탭은 BIVE 단위 인덱스라 캠페인 속성 미노출 — 캠페인 단위 정보는 캠페인명 클릭으로 캠페인 상세 새 창 |
| **민팅 이벤트 활성화 선행 조건** | 기본정보 + BIVE 보상 모두 저장 완료 + BIVE 보상 ≥ 1건 |
| **민팅 BIVE 보상 가중치** | **정수 1~999**. 비중 = 가중치 / 가중치합. 확률 분포 추첨 |
| **민팅 이벤트 보상 BIVE 제거 (활성 이력 존재)** | 보상 내역 테이블에서 행 유지 + 가중치/비중 "-" 표시 (민팅 수 0이면 행 제거) |
| **혜택 유형** | **2종**: Boost Point (BP) / Raffle Ticket (RT) |
| **혜택 상태** | **5상태**: 초안 / 대기 / 활성 / 중지 / 종료. 대기는 시작일시 도달 시 자동 활성화 |
| **혜택 지급주기** | **3종**: 일일 / 주간 / 1회(획득 시) (운영 BO 실제 표기, 기본값 "일일") |
| **혜택 시작·종료 시간 기본값** | 시작시간 기본 "00:00" / 종료시간 기본 "23:59" (운영 BO 실제) |

## 6. 상태 머신

### 6-1. 에디션 상태 — 2상태 (간단)

```
존재 ↔ 삭제 (BIVE 0건 + 민팅 0건일 때만)
```

에디션은 다국어 명칭만 가지므로 별도 상태 관리 없음. BIVE가 1건이라도 등록되면 수정·삭제 불가.

### 6-2. BIVE 상태 — 3상태

```
Draft(초안) → Active(활성) → Inactive(비활성)
   ↑                              ↓
   └─── (불가)                     └─→ Active (재활성 가능)
```

- **Draft**: 등록 직후. 기본·기능·이미지 모두 수정 가능. 삭제 가능
- **Active**: 활성화 후. 기본·기능 모두 수정 불가. 민팅 이벤트에 보상으로 등록 가능. 비활성 전환 가능 (연결된 캠페인이 없거나 모두 종료 상태일 때만)
- **Inactive**: 비활성. 민팅 이벤트에 보상으로 등록 불가. 활성 전환 가능

### 6-3. 민팅 이벤트 상태 — 4상태

```
초안 → 활성 ⇄ 중지 → 종료
  ↓                  ↑
  └─→ 삭제(초안만)    └── 개발팀 수동 처리
```

- **초안**: 생성 직후. 기본정보·BIVE 보상 모두 수정 가능. 삭제 가능. 활성화 시 초안으로 되돌릴 수 없음
- **활성**: 활성화 후. 기본정보·BIVE 보상 모두 수정 불가. 회원 액션에 따라 자동 민팅 발생. 일시중지 가능
- **중지**: 일시중지 상태. 회원 액션 발생해도 민팅 안 됨. BIVE 보상 설정만 수정 가능. 활성 재전환 가능
- **종료**: 캠페인 완전 종료. 모든 정보 읽기 전용. 개발팀에 요청해 수동 처리

### 6-4. 혜택 상태 — 5상태

```
초안 → 대기 → 활성 ⇄ 중지 → 종료
  ↓     ↓                    ↑
  삭제  대기취소(→초안)        └── 종료일시 자동 도달
```

- **초안**: 생성 직후. 기본정보 전체 수정 가능. 삭제 가능. 활성대기 전환 가능
- **대기**: 활성대기 후. 시작일시 도달 시 자동 활성. 모든 정보 수정 불가. 대기취소 시 초안 복귀
- **활성**: 지급 수행 중. 모든 정보 수정 불가. 일시중지 가능. 종료일시 도달 시 자동 종료
- **중지**: 지급 중단. 종료일·종료시간만 수정 가능. 활성 재전환 가능. 종료일시 도달 시 자동 종료
- **종료**: 읽기 전용. 모든 전환·수정 불가

## 7. 외부 영역 연동

| 연동 영역 | 운영 동작 | 본 영역의 역할 |
|---|---|---|
| SQ — 스토리 퀘스트 | 에피소드 완료 시 BIVE 자동 민팅 (`biveRewardYn` ON일 때 mintingEventId 참조) | 활성 상태 캠페인 목록 제공 → SQ-202-CREATE 드롭다운 옵션. SSOT [CEB-BO-BIVE-103] |
| RFL — 래플 | 당첨자에게 BIVE 자동 민팅 (추가 보상 BIVE 설정 시) | 활성 상태 캠페인 목록 제공 → RFL-201-CREATE 드롭다운 옵션 |
| FQ — 팬퀘스트 | Quest 보상으로 BIVE 자동 민팅 (SQ-302-CREATE 흡수) | 활성 상태 캠페인 목록 제공 |
| 회원 (USR) | BIVE 보유 회원 식별 + 지갑주소 매핑 | BIVE 상세 §민팅이력 탭에서 닉네임 → 회원상세 새 창 |
| 외부 블록체인 (BSC) | 실제 NFT 민팅 트랜잭션 발행 | BIVE 상세 §민팅이력 탭에서 지갑주소 → BSCScan 새 창 |
| 권한 관리 (SYS) | BIVE 대메뉴 읽기/쓰기 권한 부여 | [관리자 > 권한관리] 설정에 따른다 |
