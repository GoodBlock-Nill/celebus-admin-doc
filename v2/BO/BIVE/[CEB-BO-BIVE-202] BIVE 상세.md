## 2. 화면 구성

### 2-1. 헤더 (상태별 액션 분기)

- 좌측: 탭 네비게이션 "기본정보 | 기능설정 | 민팅관리 | 민팅이력"
- 우측 액션 (상태별 분기):

| 상태 | 헤더 액션 |
|---|---|
| **Draft (초안)** | [삭제] (BIVE 삭제 모달) + [저장] (저장 토스트) + [활성화] (활성화 모달) |
| **Active (활성)** | [비활성화] (조건부) |
| **Inactive (비활성)** | [활성화] |

### 2-2. 기본정보 탭 (Draft에서 수정 가능, Active/Inactive에서 read-only)

- **BIVE 명칭** (Read-only): `{멤버명} {등급}-{등급번호}` 자동 생성/갱신
- 나머지 필드는 [CEB-BO-BIVE-202-CREATE] 등록 화면과 동일 (아티스트·멤버·등급·등급번호·설명·미디어)

**관리 정보 카드** (우측 하단):

| 라벨 | 값 |
|---|---|
| 상태 | Draft / Active / Inactive 배지 |
| 생성 관리자 | 최초 생성 관리자 (수정 불가) |
| 생성 일시 | YYYY.MM.DD hh:mm (수정 불가) |
| 최근 수정자 | 저장/활성화 시 갱신 |
| 최근 수정 일시 | 저장/활성화 시 갱신 |

### 2-3. 기능설정 탭 (Draft에서 수정 가능, Active/Inactive에서 read-only)

- 보내기 (Send) ON/OFF
- Mix ON/OFF
- Pick ON/OFF

### 2-4. 민팅관리 탭 (모든 상태에서 조회만)

본 BIVE가 보상으로 연결된 민팅 캠페인 현황 조회.

**필터 바**:

- 상태 (전체/초안/활성/중지/종료)
- 민팅 타입 (전체/Event/Ticket/Mix/Pick/상품)
- 연결 일시 정렬 (빠른순 기본 / 오래된 순)
- 검색 입력 "캠페인명 입력" + [조회]

**테이블**:

| 컬럼 | 설명 |
|---|---|
| 상태 | 배지 — 초안(amber)/활성(green)/중지(red)/종료(gray) |
| 민팅 타입 | Event / Ticket / Mix / Pick / 상품 |
| 캠페인명 | 클릭 시 캠페인 상세 새 창 |
| 민팅 횟수 | 본 캠페인을 통한 BIVE 누적 발행 수 (없으면 "-") |
| 캠페인 연결 일시 | YYYY.MM.DD hh:mm (기본 정렬 내림차순) |

빈 상태: "연결된 캠페인이 없습니다."

### 2-5. 민팅이력 탭 (모든 상태에서 조회만)

본 BIVE의 실제 발행 결과(회원·지갑 단위) 조회.

**요약 영역**: "총 발행: {수량}" + 가이드 "연결된 캠페인에서 BIVE의 민팅 이력을 보여줍니다."

**필터 영역**:

- 검색 기준 드롭다운: 닉네임(기본) / 지갑주소
- 검색 입력: placeholder 선택값에 따라 변경
- 정렬: 민팅일시 최근 순 (기본)
- [조회] 버튼

**테이블**:

| 컬럼 | 설명 |
|---|---|
| Token ID | NFT 식별값 |
| 닉네임 | 파란색 하이퍼링크. 클릭 시 회원 상세 새 창 |
| 지갑주소 | 클릭 시 BSCScan 새 창 (`https://bscscan.com/address/{address}`) |
| 민팅일시 | YYYY.MM.DD hh:mm 로컬 |

빈 상태: "민팅 이력이 없습니다."

페이지네이션 하단 20행/페이지.

## 4. 상태별 동작 분기

| 상태 | 기본정보·기능 | 헤더 액션 | 민팅 캠페인 보상 등록 |
|---|---|---|---|
| **Draft** | ✏️ 수정 가능 | [삭제] + [저장] + [활성화] | 불가 (Active만 가능) |
| **Active** | 🔒 read-only | [비활성화] (조건부) | 가능 |
| **Inactive** | 🔒 read-only | [활성화] | 불가 (신규 등록 차단, 기존 활성 캠페인은 유지) |

**Active → Inactive 전환 조건**:

- 본 BIVE가 보상으로 연결된 민팅 캠페인이 **없거나 모두 종료 상태**여야 함
- 활성/중지 상태 캠페인에 연결되어 있으면 비활성 불가 (토스트로 안내)

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 활성화 검증 | Draft 상태에서 저장된 데이터와 입력 데이터가 다른 경우 활성화 실패 ("등급번호 001 저장 후 002로 수정 + 활성화" 등) → 토스트 "활성화 실패" |
| 등급/등급번호 고유성 | 동일 에디션 내 (멤버, 등급, 등급번호) 조합 유니크 (서버 검증) |
| 비활성화 조건 | 연결된 캠페인이 모두 종료 상태일 때만 비활성화 가능 |
| 권한 | [관리자 > 권한관리] BIVE 쓰기 권한 보유자만 헤더 액션 노출 |

## 6. 외부 영역 연동

| 영역 | 운영 동작 | 사용처 |
|---|---|---|
| 아티스트·멤버 관리 | 자동 채움 (기본정보 탭) | 등록 시 사용 |
| 민팅 캠페인 (BIVE-103/203) | 본 BIVE가 보상으로 연결된 캠페인 + 민팅 카운트 집계 | 민팅관리 탭 |
