## 1. 화면 개요

특정 **혜택**(BP 또는 RT)의 상세를 조회·수정한다. 5상태(초안/대기/활성/중지/종료) × 2탭(기본정보·BIVE 추가) 매트릭스 적용. 상태에 따라 헤더 액션·필드 잠금이 분기된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/benefits/{type}/{benefitId}` |
| Breadcrumb | 홈 / BIVE / 혜택 관리 / {Boost Point 또는 Raffle Ticket} / 혜택 상세 |
| 페이지 제목 | `{Boost point / Raffle Ticket} 혜택 상세` |
| 진입 경로 | [CEB-BO-BIVE-104] 혜택 리스트의 행 클릭 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더 (상태별 액션 분기)

- 좌측: 페이지 타이틀 + 탭 네비게이션 "기본정보 | BIVE 추가"
- 우측 액션 (상태별):

| 상태 | 헤더 액션 | 동작 |
|---|---|---|
| **초안** | [삭제] + [저장] + [활성대기] | 삭제 모달 / 저장 토스트 / 활성대기 모달 → 대기 전환 |
| **대기** | [대기취소] | 대기취소 모달 → 초안 복귀 |
| **활성** | [일시중지] | 일시중지 모달 → 중지 전환 |
| **중지** | [저장] + [활성화] | 저장 토스트 (※ 종료일·종료시간만) / 활성화 모달 → 활성 전환 |
| **종료** | 없음 | 모든 영역 read-only |

### 2-2. 기본정보 탭

**본문 좌측 — 캠페인 기본 정보**:

| 필드 | 입력 방식 | 검증·옵션 |
|---|---|---|
| 혜택 명칭 | Text Input | placeholder "혜택 명칭을 입력해 주세요" |
| 수량 | Number Input | BP 탭: "BP 수량" / RT 탭: "Raffle Ticket 수량". 정수 ≥ 1 |
| 지급주기 | Select | **일일 / 주간 / 1회** |
| 지급 요일 (주간) | Select 단일선택 | 주간 선택 시에만 활성. **월~일 중 1개** |
| 시작일 | Date | YYYY.MM.DD |
| 시작시간 | Time | hh:mm |
| 종료일 | Date | YYYY.MM.DD |
| 종료시간 | Time | hh:mm |

**본문 우측 — 관리 정보 패널**:

| 라벨 | 값 |
|---|---|
| 상태 | 초안(노랑) / 대기(파랑) / 활성(녹색) / 중지(빨강) / 종료(회색) 배지 |
| 생성 관리자 | 최초 생성 |
| 생성 일시 | YYYY.MM.DD hh:mm |
| 최근 수정자 | 저장/활성화 시 갱신 |
| 최근 수정 일시 | (동일) |

**상태별 기본정보 편집 범위**:

| 상태 | 편집 가능 범위 |
|---|---|
| **초안** | 전체 수정 가능 |
| **대기** | 전체 read-only |
| **활성** | 전체 read-only |
| **중지** | **종료일·종료시간만** 수정 가능 (그 외 read-only) |
| **종료** | 전체 read-only |

### 2-3. BIVE 추가 탭

- 섹션 헤더: "BIVE 보상" + 가이드 "혜택을 적용할 BIVE를 추가해주세요."
- 우측 [+ 추가하기] 버튼 (상태별 분기): 클릭 시 [CEB-BO-BIVE-204-MD-ADD] BIVE 추가 모달 오픈

**테이블 컬럼**:

| 컬럼 | 설명 |
|---|---|
| BIVE 명칭 | `{아티스트명} {등급}-{등급번호}` |
| 아티스트 그룹 | 아티스트명 |
| 아티스트 | 아티스트명 |
| 등급 | Event/Normal/Special/Wicked/Next/Final |
| 등급번호 | 5자리 숫자 |
| 관리 | X(삭제) 아이콘 (상태별 분기) |

**정렬**: BIVE 생성일시 빠른순

**페이지네이션**: 테이블 영역 내 무한 스크롤

**빈 상태**: "추가하기 버튼을 눌러 BIVE를 등록하세요."

**상태별 BIVE 추가 탭 동작**:

| 상태 | [+ 추가하기] | 관리(X) 삭제 |
|---|---|---|
| 초안 | ✅ 활성 | ✅ 가능 |
| 대기 | 🔒 비활성 | 🔒 비활성 |
| 활성 | 🔒 비활성 | 🔒 비활성 |
| 중지 | ✅ 활성 | ✅ 가능 |
| 종료 | 🔒 비활성 | 🔒 비활성 |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 활성대기 전환 선행 조건 | 등록된 BIVE ≥ 1 + 수량 ≥ 1 + 지급주기별 필수 필드 충족 |
| 일일 조건 | 시작일시 + 종료일시 필수 |
| 주간 조건 | 요일 1개 + 시작일시 + 종료일시 필수 |
| 1회 조건 | 시작일 필수 |
| 중지 상태 저장 | 종료일·종료시간만 저장 가능 (다른 필드 변경 무시) |
| 종료일 검증 | 종료일 ≥ 시작일 + 1일 |
| 권한 | [관리자 > 권한관리] BIVE 쓰기 권한 보유자만 헤더 액션 노출 |
