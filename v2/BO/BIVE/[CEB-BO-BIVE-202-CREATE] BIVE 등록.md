## 1. 화면 개요

에디션 하위에 신규 **BIVE**(NFT)를 등록한다. 등록 시 운영자가 사전에 생성한 아티스트 그룹·아티스트 리소스를 사용하며, 등록 직후 Draft 상태로 [CEB-BO-BIVE-102] 리스트에 추가된다. 등록은 2탭 구조(기본정보 / 기능설정)로 모두 입력 후 [등록] 클릭.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/editions/{editionId}/bive/create` |
| Breadcrumb | 홈 / BIVE / 에디션 관리 / {에디션명} / BIVE 등록 |
| 페이지 제목 | "BIVE 등록" |
| 진입 경로 | [CEB-BO-BIVE-102] BIVE 관리의 [+ BIVE 등록] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더 (공통)

- 좌측: 페이지 타이틀 "BIVE 등록"
- 우측 액션:
  - [취소]: 입력값 폐기 → [CEB-BO-BIVE-102] BIVE 관리로 이동
  - [등록]: 검증 통과 시 토스트 "등록 완료" → [CEB-BO-BIVE-102]로 이동 + Draft 상태로 리스트에 추가
- 탭 네비게이션: "기본정보 | 기능설정" (현재 탭 활성 표시)

### 2-2. 기본정보 탭 (왼쪽 폼 + 오른쪽 미디어 + 속성정보)

- 안내 문구: "기본정보 및 속성정보를 모두 입력해주세요" (운영 BO 실제)

**왼쪽 폼**:

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 아티스트 그룹 | Select | placeholder "아티스트 그룹을 선택해주세요". 아티스트 관리에서 "활성" 상태만 옵션 노출. 운영 BO 실제 옵션 5종 — **언더라이트 / V01D / CELEBUS / MADEIN / iKON** (운영 데이터 기준, 신규 그룹 등록 시 옵션 추가) | ★ |
| 아티스트 | Select | placeholder "아티스트를 선택해주세요". 선택한 아티스트 그룹 하위의 "활성" 멤버만 옵션 노출. 운영 BO 실제 V01D 그룹 5명 — **Mavin / Ji-A / Shaden / Misa / Lee** | ★ |
| 설명 (EN) | Textarea | 영문 설명 자유 입력 | — |

**속성정보 섹션**:

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 등급 | Select | placeholder "등급을 선택해주세요". **고정 6종**: Event / Normal / Special / Wicked / Next / Final (운영 BO 실제 표기) | ★ |
| 등급번호 | Number Input | placeholder "숫자 5자리 이내" (운영 BO 실제). 숫자만, 최대 5자리 (1~99999) | ★ |

> **운영 BO 실제 정합**: v1 Confluence 명세의 "아티스트 그룹/아티스트 KR/EN/JP 자동 채움 Read-only 필드"는 운영 BO에서 별도 표시되지 않음. Select 선택 결과만 단일 라벨로 노출됨. 또한 "에디션명 KR/EN/JP Read-only 자동 채움" 필드도 운영 BO 본 화면에 노출되지 않음 (Breadcrumb 컨텍스트로만 식별).

**오른쪽 패널 — 미디어 파일 업로드**:

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 미디어 파일 | 파일 등록 / 파일 제거 | 허용 확장자: **PNG, JPG, GIF, WebP, MP4** (운영 BO 실제 — WebP 포함). **최대 20MB** | ★ |

- [파일 등록] 버튼: 항상 활성. 클릭 시 파일 탐색기. 기존 파일 있으면 대체
- [파일 제거] 버튼: 파일 등록 시에만 활성. 클릭 시 미디어 제거

### 2-3. 기능설정 탭

- 안내: "기본정보 및 기능설정이 모두 입력·저장되어야 활성화할 수 있습니다."
- **기능 토글 3종 (기본 ON)**:
  - 보내기 (Send): ON/OFF
  - Mix: ON/OFF
  - Pick: ON/OFF
