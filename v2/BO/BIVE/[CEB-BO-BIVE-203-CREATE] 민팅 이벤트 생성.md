## 1. 화면 개요

신규 **민팅 이벤트**을 생성한다. 기본정보 + BIVE 보상 2탭 구조이며, 모두 입력 후 [생성하기] 클릭 시 **초안** 상태로 [CEB-BO-BIVE-103]에 추가된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/minting/create?type={EVENT / TICKET / MIX / PICK}` |
| Breadcrumb | 홈 / BIVE / 민팅 관리 / {Event 또는 Ticket} / 캠페인 생성 |
| 페이지 제목 | `{Event 또는 Ticket} 캠페인 생성` |
| 진입 경로 | [CEB-BO-BIVE-103] 민팅 이벤트 리스트의 [+ 캠페인 생성] 버튼 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 헤더 (공통)

- 좌측: 페이지 타이틀 "{Event/Ticket} 캠페인 생성"
- 우측 액션:
  - [취소하기]: [CEB-BO-BIVE-103]으로 이동 (입력 폐기)
  - [생성하기] (Primary): 검증 통과 시 토스트 "생성 완료" → [CEB-BO-BIVE-103]으로 이동 + 초안 상태로 행 추가
- 탭 네비게이션: "기본정보 | BIVE 보상" (현재 탭 활성)

### 2-2. 기본정보 탭

- 안내 문구: "캠페인 식별을 위한 캠페인 명을 입력하고, 연결기능을 선택해 주세요."

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 캠페인명 | Text Input | placeholder "캠페인 명을 입력하세요" (운영 BO 실제). 길이 2~30자. 선후 공백 trim. 중복 허용 | ★ |
| 연결 기능 | Select | placeholder "연결 기능을 선택해주세요" (운영 BO 실제). 개발 프리셋 4종 — **회원가입 보상 / 출석체크 보상 / 래플 보상 / 팬퀘스트 보상** (운영 BO 실제). **캠페인 ↔ 연결 기능 = 1:1** | ★ |

### 2-3. BIVE 보상 탭

- 안내: "보상으로 지급되는 BIVE를 추가하고 각 항목에 가중치를 입력하세요."
- 우측 상단 카운터: **가중치 합: n**
- 우측 상단 액션 [+ 추가하기]: 클릭 시 [CEB-BO-BIVE-203-MD-ADD] BIVE 보상 추가 모달 오픈

**테이블 컬럼** (CEB-BO-BIVE-203 §2-3과 동일):

- BIVE 명칭 / 아티스트 그룹 / 아티스트 / 등급 / 등급번호 / 가중치 (입력) / 비중 (읽기) / 관리 (X)

**가중치 입력**:

- 정수만, 1~999 허용 범위
- placeholder "1-999"
- 입력 즉시 가중치 합·각 행 비중 재계산
- 비중 표기: `가중치 / 가중치합` 분수 (예: 10/55)
- (선택) 백분율 보조 표기: `(가중치 ÷ 가중치합 × 100)%`

**행 삭제** (X 아이콘): 즉시 행 제거 + 가중치 합/비중 재계산

**빈 상태**: "+ 추가 버튼을 눌러 BIVE를 등록하세요."
