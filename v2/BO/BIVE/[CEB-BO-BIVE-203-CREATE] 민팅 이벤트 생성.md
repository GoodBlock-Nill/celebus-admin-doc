## 2. 화면 구성

### 2-1. 헤더 (공통)

- 좌측: 페이지 타이틀 "{Event/Ticket} 캠페인 생성"
- 우측 액션:
  - [취소하기]: [CEB-BO-BIVE-103]으로 이동 (입력 폐기)
  - [생성하기] (Primary): 검증 통과 시 토스트 "생성 완료" → [CEB-BO-BIVE-103]으로 이동 + 초안 상태로 행 추가
- 탭 네비게이션: "기본정보 | BIVE 보상" (현재 탭 활성)

### 2-2. 기본정보 탭

- 안내 문구: "캠페인 식별을 위한 캠페인 명을 입력하고, 연결기능을 선택해 주세요."

| 필드 | 입력 방식 | 검증·옵션 | 필수 |
|---|---|---|---|
| 캠페인명 | Text Input | placeholder "캠페인 명을 입력하세요" (운영 BO 실제). 길이 2~30자. 선후 공백 trim. 중복 허용 | ★ |
| 연결 기능 | Select | placeholder "연결 기능을 선택해주세요". 개발 프리셋 **공통 7종** (v1.6) — **회원가입 보상 / 출석체크 보상 / 래플 보상 / 팬퀘스트 보상 / 에피소드 보상 / 덕력랭킹 보상 / 아티스트 키우기 보상**. 모든 캠페인 유형(Event·Ticket·Mix·Pick) 동일 옵션. **캠페인 ↔ 연결 기능 = 1:1** | ★ |

### 2-3. BIVE 보상 탭

**보상 방식 선택 — 라디오 + 설명 박스** (v1.4, 탭 최상단):

- 카드 컨테이너 안에 "보상 방식 선택" 헤딩 + "두 가지 방식 중 한 가지를 선택하여 캠페인을 활성화합니다." 안내
- 라디오 옵션 2종 (각 옵션은 라디오 + 이름 + 한 줄 설명):
  - **⦿ 가중치 보상 (WEIGHTED)** — "등록된 BIVE 중 가중치 확률로 1개를 추첨하여 회원에게 발행합니다."
  - **◯ 지정 보상 (FIXED)** — "등록된 모든 BIVE를 회원에게 동시에 발행합니다. 최대 10종까지 등록 가능합니다."
- 캠페인당 1개 선택 (XOR). 생성 화면에서는 자유롭게 전환 가능 (캠페인 생성 후 [CEB-BO-BIVE-203] §2-3 상태별 매트릭스에 따라 잠금)

**안내 분기**:

- 가중치: "보상으로 지급되는 BIVE를 추가하고 각 항목에 가중치를 입력하세요."
- 지정: "캠페인 트리거 시 등록된 모든 BIVE가 동시에 지급됩니다. 최대 10종까지 등록 가능합니다."

**우측 상단**:

- 가중치 보상: 카운터 "가중치 합: n"
- 지정 보상: 카운터 "등록 BIVE: n / 10"
- **[+ 추가하기]** (v1.5, 운영 BO 정합): **생성 화면에서도 항상 활성**. 클릭 시 [CEB-BO-BIVE-203-MD-ADD] 모달 오픈. 선택된 BIVE는 보상 테이블에 행으로 추가되며, 캠페인 [생성하기] 시점에 함께 저장됨

**테이블 컬럼** (보상 방식에 따라 분기):

- 공통: BIVE 명칭 / 아티스트 그룹 / 아티스트 / 등급 / 등급번호 / 관리 (X)
- 가중치 보상에만 추가: 가중치 (입력) / 비중 (읽기)

**가중치 입력** (가중치 보상 전용):

- 정수만, 1~999 허용 범위
- placeholder "1-999"
- 입력 즉시 가중치 합·각 행 비중 재계산
- 비중 표기: `가중치 / 가중치합` 분수 (예: 10/55)
- (선택) 백분율 보조 표기: `(가중치 ÷ 가중치합 × 100)%`

**행 삭제** (X 아이콘): 즉시 행 제거 + 가중치 합/비중 재계산

**빈 상태**: "+ 추가 버튼을 눌러 BIVE를 등록하세요."
