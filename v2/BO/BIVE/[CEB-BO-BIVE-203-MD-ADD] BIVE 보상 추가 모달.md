## 1. 모달 개요

캠페인의 보상으로 지급할 **BIVE를 다중 선택**하여 추가한다. 에디션·아티스트·아티스트·등급 필터로 BIVE 목록을 좁혀가며, 이미 추가된 BIVE는 중복 방지된다.

| 항목 | 값 |
|---|---|
| 트리거 | [CEB-BO-BIVE-203] / [CEB-BO-BIVE-203-CREATE]의 BIVE 보상 탭 [+ 추가하기] 버튼 |
| 차단 조건 | 캠페인 상태가 활성·종료이면 트리거 버튼 자체 비활성 |
| 영향 범위 | 호출 화면의 BIVE 보상 테이블에 선택한 BIVE 행 추가 (가중치는 미입력 상태) |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |

## 2. 화면 구성

### 2-1. 모달 헤더

- 타이틀: "보상으로 지급될 BIVE를 추가해주세요."
- 우측 상단: 닫기 [X] 아이콘
- 우측 하단(고정): [추가하기] (Primary, 선택 0건이면 비활성)

### 2-2. 필터 영역 (상단)

| 필드 | 입력 방식 | 옵션 | 필수 |
|---|---|---|---|
| 에디션 | Select | 등록된 에디션 목록 | ★ |
| 아티스트 그룹 | Select (기본: 전체) | 선택한 에디션의 BIVE 아티스트 목록 (에디션 선택 전 비활성) | — |
| 아티스트 | Select (기본: 전체) | 선택한 에디션의 BIVE 아티스트 목록 (에디션 선택 전 비활성) | — |
| 등급 | Select (기본: 전체) | 선택한 에디션의 등급 목록 (에디션 선택 전 비활성) | — |

- 필터 변경 시 목록 즉시 갱신

### 2-3. 목록 영역 (테이블)

| 컬럼 | 설명 |
|---|---|
| [선택] | 체크박스. 이미 보상에 포함된 BIVE는 체크 + 비활성 (Disabled), 행 텍스트 60% 톤다운 |
| BIVE 명칭 | `{아티스트명} {등급}-{등급번호}` |
| 아티스트 그룹 | 아티스트명 |
| 아티스트 | 아티스트명 |
| 등급 | Event/Normal/Special/Wicked/Next/Final |
| 등급번호 | 5자리 숫자 |

**헤더 체크박스 (전체 선택)**:

- 현재 페이지의 **선택 가능 행**만 전체 토글
- 선택/미선택 혼재 시 중간(Indeterminate) 상태

**정렬**: BIVE 생성일 빠른순 (기본)

**페이지네이션**: 무한 스크롤 (테이블 영역 내에서만 스크롤 처리, 테이블 위 필터 영역은 고정)

### 2-4. 빈 상태

| 상황 | 메시지 |
|---|---|
| 에디션 미선택 | "에디션을 선택해 등록된 BIVE를 확인하세요." |
| 결과 0건 | "조건에 맞는 BIVE가 없습니다." |

## 4. 상태별 동작 분기

| 상황 | 동작 |
|---|---|
| 모달 오픈 (에디션 미선택) | 테이블 빈 상태 + 안내 |
| 에디션 선택 | 아티스트·아티스트·등급 필터 활성 + 테이블 데이터 로드 |
| 필터 변경 | 테이블 즉시 갱신 |
| 체크박스 선택 | [추가하기] 활성 (1건 이상 선택 시) |
| 이미 보상에 포함된 BIVE | 체크박스 비활성 (체크된 상태 + 회색) |
| [추가하기] 클릭 (1건 이상) | 호출 화면의 BIVE 보상 테이블에 선택 BIVE 행 추가 → 모달 닫힘 |
| [추가하기] 클릭 (0건) | 버튼 비활성으로 클릭 불가 |
| 닫기 [X] / ESC / 오버레이 클릭 | 모달 닫힘 (선택 폐기) |
| 네트워크 오류 | 토스트 "불러오기에 실패했습니다. 다시 시도해 주세요." |

## 5. 검증 규칙

| 규칙 | 내용 |
|---|---|
| 에디션 필수 | 에디션 미선택 시 테이블 미노출 + 가이드 문구 |
| 중복 방지 | 호출 화면의 BIVE 보상 테이블에 이미 추가된 BIVE는 체크박스 비활성 (선택 불가) |
| 선택 1건 이상 | [추가하기] 활성 조건 |
| 활성 상태 BIVE만 노출 | [CEB-BO-004] §5 — 비활성 상태 BIVE는 신규 캠페인 보상 추가 불가 (필터링) |
