## 1. 화면 개요

BIVE 보유 조건에 따라 회원에게 지급되는 **혜택**(Boost Point + Raffle Ticket)을 유형별로 조회·생성·상태 관리하는 화면. 일일·주간·1회(획득 시) 형태의 지급 정책을 운영자가 정의하고, 매일 00:05 배치로 자동 지급된다.

| 항목 | 값 |
|---|---|
| 화면 경로 | `/bive/benefits` |
| Breadcrumb | 홈 / BIVE / 혜택 관리 |
| 페이지 제목 | 혜택 관리 |
| 진입 경로 | 사이드바 > BIVE > 혜택 관리 |
| 접근 권한 | [관리자 > 권한관리] 설정에 따른다 |
| 지급 시간 안내 | 우측 상단 "일일/주간 혜택 지급시간: 00:05" (서버 설정 반영, 읽기 전용) |

## 2. 화면 구성

### 2-1. 상단 탭

| 탭 | 의미 |
|---|---|
| **Boost Point** (BP, 기본) | 회원에게 Boost Point 지급. SQ/RFL 등에서 보상 배수 강화 등에 사용 |
| **Raffle Ticket** (RT) | 회원에게 응모권(Raffle Ticket) 지급. RFL 영역에서 응모 시 사용 |

### 2-2. 필터·검색 영역

- **상태 필터** (드롭다운): `전체(기본) / 초안 / 대기 / 활성 / 중지 / 종료`
- **지급주기 필터** (드롭다운): `전체(기본) / 일일 / 주간 / 1회(획득 시)`
- **검색 입력**: placeholder "혜택 명칭 입력" — 키워드 부분 일치
- **[조회] 버튼**: 현재 탭 + 조건으로 재조회
- **[+ 혜택 생성] 버튼** (우측 상단): 현재 탭의 혜택 생성 페이지로 이동
  - Boost Point 탭 → [CEB-BO-BIVE-204-CREATE] (BP 변형)
  - Raffle Ticket 탭 → [CEB-BO-BIVE-204-CREATE] (RT 변형)

### 2-3. 데이터 테이블

| 컬럼 | 설명 |
|---|---|
| 상태 | 칩 — 초안(노랑) / 대기(파랑) / 활성(녹색) / 중지(빨강) / 종료(회색) |
| 혜택 명칭 | 운영자가 설정한 혜택 이름 |
| 등록된 BIVE | 본 혜택에 적용되는 BIVE 수 |
| 수량 | 정수 (BP 탭은 "BP 수량", RT 탭은 "Ticket 수량") |
| 지급 주기 | 일일 / 주간 / 1회 |
| 지급 요일 | 주간일 때 요일 (월~일 중 택1). 일일/1회는 "-" |
| 시작일 | `YYYY.MM.DD` |
| 종료일 | `YYYY.MM.DD` |

**행 인터랙션**:

- 행 전체 클릭 → [CEB-BO-BIVE-204] 혜택 상세 (기본정보 탭)

**정렬**:

- 기본: 생성일시 빠른순

### 2-4. 페이지네이션

## 4. 상태별 동작 분기

### 4-1. 혜택 상태 정의 + 전이

| 상태 | 의미 | 전이 가능 |
|---|---|---|
| **초안** | 생성 직후. 기본정보 전체 수정 가능 | → 대기 (활성대기 버튼), → 삭제 |
| **대기** | 활성대기 후. 시작일시 자동 도달 시 활성 전환. 모든 정보 수정 불가 | → 초안 (대기취소), → 활성 (자동) |
| **활성** | 지급 수행 중. 모든 정보 수정 불가. 종료일시 도달 시 자동 종료 | → 중지 (일시중지) |
| **중지** | 지급 중단. 종료일·종료시간만 수정 가능. 종료일시 도달 시 자동 종료 | → 활성 (재활성화) |
| **종료** | 읽기 전용. 모든 전환·수정 불가 | (전이 없음) |

### 4-2. 화면 동작

| 상황 | 동작 |
|---|---|
| 데이터 로딩 중 | Skeleton 테이블 |
| 데이터 없음 | "등록된 혜택이 없습니다." 안내 |
| 검색·필터 결과 없음 | "조건에 맞는 혜택이 없습니다." 안내 |
| 행 클릭 | [CEB-BO-BIVE-204] 혜택 상세 진입 |
| [+ 혜택 생성] 클릭 | 현재 탭의 혜택 생성 페이지 진입 |
| 권한 없음 진입 | 화면 미노출 또는 [+ 혜택 생성] 비활성 |

## 5. 검증 규칙

| 검증 | 내용 |
|---|---|
| 활성/대기 전환 선행 조건 (상세에서 검증) | 등록된 BIVE ≥ 1 + 수량 ≥ 1 정수 + 지급주기별 필수 필드 충족 |
| 일일 지급 조건 | 시작일시 + 종료일시 필수 |
| 주간 지급 조건 | 요일 1개 선택 + 시작일시 + 종료일시 필수 |
| 1회(획득 시) 조건 | 시작일 필수. 해당일~종료일까지 조건 충족 시 1회 지급 |
| 멱등성 | 동일 혜택/사용자/일(또는 지정 주기 키) 중복 지급 방지 |
| 권한 | [관리자 > 권한관리] BIVE 쓰기 권한 보유자에게만 [+ 혜택 생성] 노출 |

## 6. 외부 영역 연동

| 영역 | 운영 동작 | 사용처 |
|---|---|---|
| BIVE 관리 (BIVE-102) | 혜택에 적용할 BIVE 보유자 식별 | BIVE 추가 모달 [CEB-BO-BIVE-204-MD-ADD]에서 활성 BIVE만 선택 가능 |
| 회원 보유 NFT | BIVE 보유 회원 조회 | 일일/주간 배치 시 지급 대상 산출 |
| 회원 액션 (외부 트리거) | 1회(획득 시) 지급 — BIVE 획득 이벤트 발생 시 | 즉시 또는 짧은 지연 배치 |
| Boost Point 시스템 | BP 지급 처리 | BP 탭 혜택의 활성 시 |
| Raffle Ticket 시스템 | RT 지급 처리 | RT 탭 혜택의 활성 시. RFL 응모 시 사용 |
| 배치 스케줄러 | 매일 00:05 일일·주간 지급 트리거 | 활성 상태 혜택 |
