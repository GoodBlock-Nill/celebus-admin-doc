## 0. 감사 개요

팬덤레벨 메뉴에 대해 **앱이 요구하는 관리 항목을 BO가 올바르게 기획하고 프로토타입에 구현했는지** 교차 검증했다.

| 감사 축 | 대상 |
| --- | --- |
| 앱 요구 | [CEB-EVT-201] 팬덤 레벨 + 앱 프로토타입 (`prototype-app/src/app/fandom/page.tsx`) |
| BO 기획 | [CEB-BO-EVT-000/101/201/401] (2026-05-29 운영 정합 개정본) |
| BO 프로토타입 | `prototype/src/app/artists/fandom/` (리스트·생성모달·상세 3탭·mock) |
| 데이터 모델 | `Celebus_고도화_덕력_팬덤_기억저장소.sql` (팬덤 레벨 8개 테이블) |

### 종합 평가

| 항목 | 점수 | 결론 |
| --- | --- | --- |
| 기획(명세) 품질 | ⭐⭐⭐⭐☆ | 우수 — 4개 문서 내부 일관성·데이터 모델 95% 정합 |
| BO 프로토타입 정합 | ⭐⭐☆☆☆ | **약 45% — 개정 이전(v1.0) 기준 제작, 전면 개정 필요** |
| 앱 ↔ BO 커버리지 | ⭐⭐⭐⭐☆ | 약 80% — 핵심 관리 항목 충족, 일부 정책 텍스트 관리 공백 |

### 5.1 보상 모델 재설계 (기존 SSOT 연계·단순화) ✅

추가 심층 검토 결과, 보상 설정이 **기존 BO 보상 SSOT를 파악하지 못한 채 재발명**됐음이 확인됐다 (운영 플로우 = 🔴 곤란). 근본 원인:

- BIVE 민팅 이벤트의 "연결 기능"에 **"아티스트 키우기 보상" 트리거가 이미 존재** ([CEB-BO-BIVE-203]) → 팬덤 레벨업 → BIVE 자동 민팅 아키텍처가 인프라에 이미 있음. 그런데 EVT는 "참조 연결: 바이브"를 라벨로만 둠.
- RFL 래플 = 추첨 SSOT인데 EVT가 자체 [래플 추첨 실행]을 신설 → 추첨 이원화.

사용자 결정(2026-05-29)으로 보상 모델을 **대폭 단순화·SSOT 연계 재설계**:

| 정정 | 내용 |
| --- | --- |
| 보상 5종 → 활성 3종 | 디지털 굿즈(BIVE 캠페인 연계) / 래플 예고(텍스트) / 서포트 예고(텍스트). 독점 콘텐츠·다운로드·실물·이벤트 참여권은 **보류** |
| 디지털 굿즈 = BIVE 연계 | "아티스트 키우기 보상" 연결 기능 활성 캠페인 선택 → BIVE SSOT 자동 민팅 |
| EVT 자체 추첨 제거 | [래플 추첨 실행]·추첨 상태·재시도·6 지급 상태·배송·참조 4종 삭제 |
| 덕력 참조 정정 | 'CEB-BO-DUK-401' → **[CEB-BO-ART-401-HISTORY] 덕력 내역** (덕력관리는 ART 영역) |
| 디지털 굿즈 보상 타이틀 | BIVE 캠페인 연결과 별개로 **보상 타이틀(다국어 한·영·일)** 입력 — 앱 표시명 |
| 라이프사이클 단순화 | 보상 단순화로 정산 작업이 없어져 **[정산 시작]·정산중 단계 제거** → 상태 3단계(준비/진행중/종료). 운영자 일상 액션 [게시]만, 정상 종료 12.31 자동, 조기 종료 [강제 종료] |

→ EVT-000 v1.5 / EVT-201 v1.4 / EVT-101 v1.3 / EVT-401 v1.1로 반영 완료. 프로토타입도 보상 3종·상태 3단계로 재개정.
