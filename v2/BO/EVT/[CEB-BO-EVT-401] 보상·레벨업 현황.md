## 1. 화면 목적

전체 회원의 팬덤 레벨 **보상 지급 현황**과 **레벨업·추첨 작업 처리 현황**을 통합 조회하는 화면(읽기 전용). 운영자는 어느 그룹·시즌·레벨에서 누구에게 어떤 보상이 지급됐는지, 자동 지급·래플 추첨이 정상 처리됐는지(실패·재시도 포함)를 한 화면에서 확인한다. CS 대응·정산 점검·운영 모니터링에 사용한다.

> 회원의 덕력 변동(기여) 내역 자체는 덕력관리 영역([CEB-BO-DUK-401] 덕력 변동 내역)에서 조회한다. 본 화면은 팬덤 레벨 **보상·레벨업**에 한정한다.

### 진입 경로

- 사이드 메뉴 > 아티스트 > 팬덤레벨 > 보상·레벨업 현황
- [CEB-BO-EVT-101] 팬덤 레벨 리스트 / [CEB-BO-EVT-201] 상세에서 연계 진입

### 2.2 보상 지급 현황 (목록)

| 컬럼 | 설명 |
| --- | --- |
| 그룹 | 아티스트 그룹명 |
| 시즌 | "{아티스트} 팬덤 N기 (YYYY)" |
| 레벨 | 보상 대상 레벨 (Lv.{N}) |
| 회원 | 닉네임 |
| 보상명 | 지급 보상명 (한국어) |
| 보상 타입 | 독점 콘텐츠 / 디지털 굿즈 / 다운로드 콘텐츠 / 실물 보상 / 이벤트 참여권 |
| 지급 방식 | 전체 지급 / 추첨 |
| 당첨 여부 | 추첨 보상의 당첨/미당첨 (전체 지급은 "-") |
| 지급 상태 | 지급 대기 / 자동 지급 완료 / 추첨 대기 / 당첨 / 지급 완료 / 지급 실패 ([CEB-BO-EVT-000] §4.5) |
| 일시 | 지급·확정 일시 YYYY.MM.DD HH:mm (KST) |

- read-only 조회. 기본 정렬: 일시 내림차순. 페이지네이션 하단 중앙.

### 2.3 레벨업·추첨 작업 모니터링

레벨업·래플 추첨은 서버 자동 작업으로 처리된다. 작업 실패 시 운영 점검을 위해 별도 조회한다.

| 컬럼 | 설명 |
| --- | --- |
| 그룹 | 아티스트 그룹명 |
| 시즌 | 해당 시즌 |
| 작업 종류 | 레벨업 보상 지급 / 래플 추첨 |
| 레벨 | 대상 레벨 |
| 처리 상태 | 대기 / 처리중 / 완료 / 실패 |
| 재시도 횟수 | 실패 후 재시도 횟수 |
| 처리 관리자 | 래플 추첨을 실행한 운영자 (자동 작업은 "-") |
| 시작·완료 일시 | YYYY.MM.DD HH:mm (KST) |

- 실패 작업은 [CEB-BO-EVT-201] 상세에서 [재시도]로 재실행한다.

> 시각적 스타일은 피그마 참조

## 3. 규칙·로직

### 3.1 권한

- 권한 그룹별 읽기 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.
- 본 화면은 **읽기 전용**이다. 보상 재지급·추첨 재시도 등 액션은 [CEB-BO-EVT-201] 상세에서 수행한다.
- 외부 아티스트 권한 그룹은 자기 아티스트 데이터만 조회한다.

### 3.2 조회 기준

- 그룹·시즌·레벨·보상 타입·지급 상태·기간·닉네임 = AND 조합 필터.
- 보상 지급 현황은 레벨 달성 시점부터 누적 표시되며, 지급 상태는 실시간 반영한다.
