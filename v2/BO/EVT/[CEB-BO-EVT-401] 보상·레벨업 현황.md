## 1. 화면 목적

전체 회원의 팬덤 레벨 **레벨업 이력**과 **디지털 굿즈(BIVE) 지급 연계 현황**을 통합 조회하는 화면(읽기 전용). 운영자는 어느 그룹·시즌·레벨에서 레벨업이 발생했고, 연결된 BIVE 보상이 누구에게 민팅됐는지를 한 화면에서 확인한다. CS 대응·정산 점검·운영 모니터링에 사용한다.

> 회원의 덕력 변동(기여) 내역은 덕력관리 영역([CEB-BO-ART-401-HISTORY] 덕력 내역)에서, BIVE 민팅 상세는 [CEB-BO-BIVE-203] 민팅 이벤트 상세에서 조회한다. 래플·서포트 예고는 앱 안내 표시 전용이라 본 화면 지급 대상이 아니다(실제 운영은 RFL·SUP 영역).

### 진입 경로

- 사이드 메뉴 > 아티스트 > 팬덤레벨 > 보상·레벨업 현황
- [CEB-BO-EVT-101] 팬덤 레벨 리스트 / [CEB-BO-EVT-201] 상세에서 연계 진입

## 2. UI 설명

### 2.1 검색·필터

| 구분 | 요소 | 타입 | 설명 |
| --- | --- | --- | --- |
| 필터 | 그룹 | Dropdown | 전체 / 특정 그룹 |
| 필터 | 시즌 | Dropdown | 전체 / 시즌 연도별 |
| 필터 | 레벨 | Dropdown | 전체 / Lv.1 ~ |
| 필터 | 기간 | Date Picker | 레벨업 달성·민팅 일시 기준 |
| 입력 | 닉네임 | Text Input | 부분 일치 검색 (BIVE 지급 연계 조회) |
| 버튼 | 초기화 | Dark Button | 필터·검색어 리셋 |

### 2.2 레벨업 이력

| 컬럼 | 설명 |
| --- | --- |
| 그룹 | 아티스트 그룹명 |
| 시즌 | "{아티스트} 팬덤 N기 (YYYY)" |
| 레벨 | 달성 레벨 (Lv.{N}) |
| 달성 일시 | 레벨업 달성 일시 (KST) |
| 대상 회원 수 | 해당 레벨 기여 회원(1DUK 이상) 수 |
| 연결 보상 | 보상 요약 (디지털 굿즈 캠페인명 / 래플·서포트 예고 문구) |

- read-only 조회. 기본 정렬: 달성 일시 내림차순. 페이지네이션 하단 중앙.

### 2.3 디지털 굿즈(BIVE) 지급 연계 현황

디지털 굿즈 보상은 레벨 달성 시 BIVE 영역이 자동 민팅한다. 본 표는 그 연계 결과를 조회한다(상세는 [CEB-BO-BIVE-203]).

| 컬럼 | 설명 |
| --- | --- |
| 그룹 | 아티스트 그룹명 |
| 시즌 | 해당 시즌 |
| 레벨 | 보상 대상 레벨 |
| 회원 | 닉네임 |
| 캠페인 | 연결된 BIVE 민팅 캠페인명 |
| 지급 상태 | BIVE 영역 민팅 상태 (연계 표시) |
| 일시 | YYYY.MM.DD HH:mm (KST) |

- 래플·서포트 예고는 지급 대상이 아니므로 본 현황에 표시되지 않는다.

> 시각적 스타일은 피그마 참조

## 3. 규칙·로직

### 3.1 권한

- 권한 그룹별 읽기 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.
- 본 화면은 **읽기 전용**이다. 보상 설정 등 액션은 [CEB-BO-EVT-201] 상세에서 수행한다.
- 외부 아티스트 권한 그룹은 자기 아티스트 데이터만 조회한다.

### 3.2 조회 기준

- 그룹·시즌·레벨·기간·닉네임 = AND 조합 필터.
- 레벨업 이력은 달성 시점부터 누적 표시되며, BIVE 지급 연계 현황은 BIVE 영역 민팅 상태를 실시간 반영한다.
