## 1. 화면 목적

운영자가 아티스트(그룹)별 팬덤 레벨을 **수동 생성·관리**하는 화면. 운영자는 [생성]으로 아티스트·시즌 단위 팬덤레벨을 직접 생성하고(준비), 상태(준비/진행중/종료)별로 조회하며, 개별 팬덤레벨 상세로 진입한다. **자동 생성되지 않는다.**

### 진입 경로

- 사이드 메뉴 > 아티스트 > 팬덤레벨

---

## 2. UI 설명

### 2.1 상단 영역

- Breadcrumb: `홈 > 아티스트 > 팬덤레벨`
- 화면 타이틀: 팬덤레벨
- 우측 상단 액션: [생성] (Primary) — 팬덤레벨 생성 모달(§2.4)

### 2.2 상태 카운터 (3개)

| 카운터 | 의미 |
| --- | --- |
| 준비 | 생성 후 게시 전 (곡선·보상 설정 단계) |
| 진행중 | 앱 노출·운영 중 |
| 종료 | 시즌 마감 |

- 각 카운터 클릭 시 해당 상태로 필터링된다.

### 2.3 검색·필터 영역

| 구분 | 요소 | 타입 | 설명 |
| --- | --- | --- | --- |
| 필터 | 상태 | Dropdown | 상태(전체) / 준비 / 진행중 / 종료 |
| 필터 | 시즌 | Dropdown | 시즌(전체) / 시즌 연도별 |
| 입력 | 그룹명 | Text Input | 부분 일치 검색 (입력 안내: "그룹명 입력") |
| 버튼 | 초기화 | Dark Button | 상태·시즌 필터·검색어 리셋 |

### 2.4 생성 모달

운영자가 [생성]으로 팬덤레벨을 직접 만든다(자동 생성 없음).

| 항목 | 표시 |
| --- | --- |
| 아티스트 | Dropdown — 그룹 선택 |
| 시즌 연도 | Dropdown — 시즌 연도 선택 (기본: 올해). 기간은 해당 연도 1월 1일 ~ 12월 31일 자동 설정 |
| 시즌명 | Text Input — "{아티스트} 팬덤 N기 (YYYY)" 자동 제안(수정 가능) |
| 액션 | [취소] / [생성] — 생성 시 **준비** 상태로 리스트 추가 |

- 시즌 기간은 캘린더 연도(1.1~12.31)로 자동 설정된다. 동일 아티스트·동일 연도 중복 생성은 차단된다.
- 레벨 곡선·보상 등 상세 항목은 생성 후 [CEB-BO-EVT-201] 상세에서 설정한다.

### 2.5 목록 영역 (테이블)

| 컬럼 | 데이터 예시 | 설명 |
| --- | --- | --- |
| 상태 | 준비 / 진행중 / 종료 | 상태 뱃지 |
| 그룹 | V01D | 아티스트 그룹명 (한국어 우선) |
| 현재 시즌 | V01D 팬덤 2기 (2026) | "{아티스트} 팬덤 N기 (YYYY)" |
| 시즌 기간 | 2026.01.01 ~ 2026.12.31 | 캘린더 연도 |
| 레벨 수 | 10 | 설정된 레벨 단계 수 (미설정 시 "-") |
| 현재 팬덤 레벨 | Lv.4 | 현재 달성 레벨 (최고레벨 도달 시 "Lv.10 MAX", 미설정 시 "-") |
| 누적 덕력 | 1,240,000 | 시즌 내 팬덤 전체 누적 획득 덕력 |
| 참여 팬 수 | 8,210 | 시즌 내 1DUK 이상 기여 회원 수 |
| 업데이트 일시 | 2026.05.28 17:21 | YYYY.MM.DD HH:mm (KST) |

- Row 클릭 → [CEB-BO-EVT-201] 팬덤 레벨 상세 이동
- 페이지네이션: 하단 중앙

> 시각적 스타일은 피그마 참조

## 4. 규칙·로직

### 4.1 권한

- 권한 그룹별 읽기·쓰기 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.

### 4.2 정렬·필터

- 기본 정렬: 누적 덕력 내림차순.
- 상태 필터 + 시즌 필터 + 그룹명 검색 = AND 조합. 그룹명 검색은 한국어·영어 통합 부분 일치.

### 4.3 상태 정의

| 상태 | 의미 | 앱 노출 |
| --- | --- | --- |
| 준비 | 생성 후 게시 전 (상세에서 곡선·보상 설정) | 미노출 |
| 진행중 | 앱 노출·운영 중 | 노출 |
| 종료 | 시즌 마감 | 미노출 |

> 운영자 상태 전환은 [게시](준비→진행중)만 수행하며, 종료는 12.31 자동(또는 [강제 종료])이다. 상세 정의는 [CEB-BO-EVT-000] §4.1.1 참조.

## 6. 활동 로그

목록 조회는 로그를 기록하지 않는다. 단, 팬덤레벨 생성은 로그를 기록한다. 곡선·보상·추첨·상태 전환 액션은 [CEB-BO-EVT-201]에서 발생한다.

| 액션 | 로그 메시지 (한국어) |
| --- | --- |
| 팬덤레벨 생성 | '{시즌명}' 팬덤레벨을 생성했습니다. (그룹: {그룹명}) |
