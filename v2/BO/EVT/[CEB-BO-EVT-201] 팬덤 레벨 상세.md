## 1. 화면 목적

개별 아티스트(그룹)의 팬덤 레벨을 설정·관리하는 화면. 레벨 곡선(레벨 수·레벨별 목표 덕력), 레벨별 보상, 회원 변동 내역을 3탭으로 제공한다.

### 진입 경로

- [CEB-BO-EVT-101] 팬덤 레벨 리스트 > Row 클릭

## 2. UI 설명

### 2.1 상단 헤더

- Breadcrumb + 뒤로가기
- 상단 표기: "{그룹명}" + 현재 시즌("{아티스트} 팬덤 N기 (YYYY)") + **시즌 상태 뱃지(준비/진행중/종료)** + 현재 팬덤 레벨 뱃지(최고레벨 도달 시 "Lv.{N} MAX")

#### 상태 전환 액션 (상태별 노출)

| 현재 상태 | 노출 버튼 | 동작 | 전환 조건 |
| --- | --- | --- | --- |
| 준비 | [게시] | 준비 → 진행중. 앱 노출 시작 (확인 모달) | 레벨 곡선 1개 이상 설정 필수 |
| 진행중 | [강제 종료] | 진행중 → 종료. 예외적 조기 종료 (확인 모달) | — (정상 종료는 12.31 자동) |
| 종료 | (없음) | — | — |

- 운영자 수동 상태 전환은 **[게시]** 만 일상적으로 사용한다. 시즌은 매년 12월 31일 23:59:59 **자동 종료**되며, 종료 시 시스템이 최종 레벨 확정·통계 마감을 자동 처리한다(운영자 정산 작업 없음).
- [강제 종료]는 아티스트 활동 중단 등 예외적 조기 종료용이다. 상태 전환은 [CEB-BO-EVT-000] §4.1.1 라이프사이클을 따른다.

### 2.2 탭 메뉴 (3탭, 상호 배타)

| # | 탭명 | 내용 |
| --- | --- | --- |
| 1 | 레벨·곡선 설정 | 시즌 정보 + 레벨별 목표 덕력 설정 |
| 2 | 레벨별 보상 설정 | 레벨별 보상 매핑 |
| 3 | 보상·레벨업 현황 | 레벨업 이력 + 회원별 보상 지급 현황 조회 |

### 2.3 탭 1 — 레벨·곡선 설정

| 항목 | 표시 |
| --- | --- |
| 시즌 정보 | 시즌명·기간(1.1~12.31)·현재 팬덤 레벨·팬덤 전체 누적 덕력·최고레벨 도달 여부 (읽기) |
| 레벨 테이블 | 레벨(Lv.1~) · 레벨업 목표 덕력 · 권장값(참고 표시) · 최고레벨 표시 · 달성 여부·달성 일시 |
| 레벨 추가/삭제 | (수정 모드) 레벨 행 추가·삭제 (최대 10레벨). 이미 달성된 레벨은 삭제·목표 수정 불가 |
| [권장 곡선 적용] | (수정 모드) 권장 곡선값 일괄 채움 |

#### 곡선 조회·수정 모드

| 모드 | 표시 | 액션 |
| --- | --- | --- |
| 조회 (기본) | 레벨 테이블·달성 여부·MAX 읽기 전용 | 우측 상단 **[수정]** |
| 수정 | 목표 덕력 입력·레벨 추가/삭제·[권장 곡선 적용] 활성 | **[저장]**(확인 모달 → 완료 토스트 + 활동 로그 → 조회 복귀) · **[취소]**(변경 폐기 → 조회 복귀) |

- **최고레벨(MAX) 도달 시즌은 [수정] 비활성** — "최고 레벨 도달 — 곡선 확정, 수정 불가" 안내.
- **종료 시즌도 [수정] 비활성**(읽기 전용) — 마감된 시즌은 곡선·보상을 변경할 수 없다.
- 카운터 모델: 각 레벨 0부터 시작, 목표 도달 시 다음 레벨 진입 + 카운터 0 리셋 ([CEB-BO-EVT-000] §4.2).
- **신규 레벨 추가**는 운영중(최고레벨 미도달) 시즌의 미달성 레벨(현재 레벨 초과)에 대해서만 가능하다. 신규 레벨 보상은 추가 이후 달성 시점에 기여 회원에게 적용되며 기존 달성·지급분은 소급 변경되지 않는다.
- **최고레벨(MAX) 도달 시 시즌 곡선·보상이 확정되어 수정·신규 레벨 추가가 모두 불가하다** ([CEB-BO-EVT-000] §4.1.2). 추가 확장은 차기 시즌에서 진행한다.

### 2.4 탭 2 — 레벨별 보상 설정

| 항목 | 표시 |
| --- | --- |
| 레벨 선택 | Lv.1 ~ 최대 레벨 중 선택 |
| 보상 목록 | 레벨별 보상 행 리스트 (노출 순서대로) |
| 보상 추가/수정/삭제 | (수정 모드) 보상 행 추가·인라인 수정·삭제 + 노출 순서 조정 |

#### 보상 종류 (활성 3종)

레벨별로 다음 3종 중 선택해 보상 행을 추가한다 ([CEB-BO-EVT-000] §4.4). 실제 자동 지급은 디지털 굿즈만 발생하며, 래플·서포트는 앱 예고 표시 전용이다.

| 보상 종류 | 입력 | 동작 |
| --- | --- | --- |
| 디지털 굿즈 (BIVE) | **보상 타이틀(다국어)** + BIVE 민팅 캠페인 선택 (드롭다운) | 레벨 달성 시 BIVE 영역이 기여 회원(1DUK 이상)에게 자동 민팅. 보상 타이틀은 앱 보상 리스트 표시명 |
| 래플 예고 | 안내 텍스트(다국어) 입력 | 앱 예고 표시 전용 (실제 래플은 RFL 영역) |
| 서포트 예고 | 안내 텍스트(다국어) 입력 | 앱 예고 표시 전용 (실제 서포트는 SUP 영역) |

#### 보상 종류별 입력 규칙

| 보상 종류 | 입력 항목 | 비고 |
| --- | --- | --- |
| 디지털 굿즈 (BIVE) | **보상 타이틀(한국어·영어·일본어)** + BIVE 캠페인 선택 + 노출 순서 | 보상 타이틀은 앱 보상 리스트 표시명(다국어 필수). 드롭다운 데이터 소스 = **"아티스트 키우기 보상" 연결 기능을 가진 활성 BIVE 민팅 캠페인** ([CEB-BO-BIVE-203]). 캠페인명·등록 BIVE 미리보기 표시. 활성 캠페인이 없으면 "BIVE 영역에서 캠페인을 먼저 생성·활성화해주세요" 안내 |
| 래플 예고 | 안내 텍스트(한국어·영어·일본어) + 노출 순서 | 예: "Lv.5 달성 시 사인 굿즈 래플 진행 예정". 추첨·지급 로직 없음 |
| 서포트 예고 | 안내 텍스트(한국어·영어·일본어) + 노출 순서 | 예: "Lv.10 달성 시 서포트 이벤트 예정". 지급 로직 없음 |

- 디지털 굿즈 보상은 사전에 [CEB-BO-BIVE-203-CREATE]에서 "아티스트 키우기 보상" 연결 기능 캠페인을 생성·활성화해야 선택 가능하다.
- 래플·서포트 예고는 앱 안내 표시 전용 — EVT에서 추첨·지급·배송을 수행하지 않는다.

#### 보상 조회·수정 모드

| 모드 | 표시 | 액션 |
| --- | --- | --- |
| 조회 (기본) | 레벨별 보상 카드 읽기 전용 | 우측 상단 **[수정]** |
| 수정 | 보상 행 추가·인라인 수정(종류·보상 타이틀·BIVE 캠페인·예고 텍스트·노출 순서)·삭제 활성 | **[저장]**(확인 모달 "변경된 레벨 보상이 앱에 반영됩니다" → 완료 토스트 + 활동 로그 → 조회 복귀) · **[취소]**(변경 폐기 → 조회 복귀) |

- 진행중 시즌에도 미달성 레벨 보상은 자유 수정. **이미 달성된 레벨의 보상은 지급(BIVE 자동 민팅)이 완료됐으므로 수정·삭제 불가** (소급 없음, 곡선 달성 레벨 제한과 동일 원칙).
- **최고레벨(MAX) 도달 시즌은 [수정] 비활성** — 모든 레벨이 달성되어 보상 변경·신규 레벨 추가가 불가하다.
- **종료 시즌도 [수정] 비활성**(읽기 전용) — 마감된 시즌은 보상을 변경할 수 없다.
- 수정 모드에서 미저장 상태로 탭 이탈·뒤로가기 시 이탈 확인 안내.

### 2.5 탭 3 — 보상·레벨업 현황

해당 그룹의 레벨업 이력과 디지털 굿즈(BIVE) 지급 연계 현황을 조회한다(읽기 전용). 회원 덕력 변동은 [CEB-BO-ART-401-HISTORY] 덕력 내역에서, BIVE 지급 상세는 [CEB-BO-BIVE-203] 민팅 이벤트 상세에서 조회한다.

#### 레벨업 이력

| 컬럼 | 설명 |
| --- | --- |
| 레벨 | 달성 레벨 (Lv.{N}) |
| 달성 일시 | 레벨업 달성 일시 (KST) |
| 대상 회원 수 | 해당 레벨 기여 회원(1DUK 이상) 수 |
| 연결 보상 | 해당 레벨 보상 요약 (디지털 굿즈 캠페인명 / 래플·서포트 예고 문구) |

#### BIVE 지급 연계 현황

| 컬럼 | 설명 |
| --- | --- |
| 회원 | 닉네임 |
| 레벨 | 보상 대상 레벨 |
| 캠페인 | 연결된 BIVE 민팅 캠페인명 |
| 지급 상태 | BIVE 영역 민팅 상태 (연계 표시) |
| 일시 | YYYY.MM.DD HH:mm (KST) |

- read-only 조회. 레벨 필터 제공. 전체 통합 조회는 [CEB-BO-EVT-401].
- 래플·서포트 예고는 지급 대상이 아니므로 본 현황에 표시되지 않는다. 보상 미설정 레벨은 레벨업 이력에만 표시된다.

> 시각적 스타일은 피그마 참조

## 3. 규칙·로직

### 3.1 권한

- 권한 그룹별 읽기·쓰기·삭제 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.
- 곡선 저장·보상 설정·상태 전환은 쓰기 권한 보유 그룹만 수행한다.

### 3.2 곡선 검증

| 항목 | 규칙 |
| --- | --- |
| 레벨 수 | 1~10 |
| 목표 덕력 | 1 이상 정수. 레벨 순서대로 증가 권장(권장 곡선 기준) |
| 미설정 | 저장 전 권장값 자동 적용 |

### 3.3 보상 지급

- **디지털 굿즈(BIVE)**: 레벨 달성 시 BIVE 영역이 기여 회원(1DUK 이상)에게 자동 민팅한다 ([CEB-BO-BIVE-203] 연계). EVT는 캠페인 연결만 한다.
- **래플·서포트 예고**: 앱 안내 표시 전용. EVT에서 추첨·지급을 수행하지 않으며, 실제 래플·서포트는 각 영역([CEB-BO-RFL-101]·[CEB-BO-SUP-101])에서 별도 운영한다.
- 레벨별 보상 설정은 **선택**이다. 보상 미설정 레벨은 레벨업만 발생하고 지급 보상은 없다.

### 3.4 상태 전환 규칙

| 전환 | 조건 |
| --- | --- |
| 준비 → 진행중 ([게시]) | 레벨 곡선 1개 이상 설정 필수. 게시 후 앱 노출 시작 |
| 진행중 → 종료 (12.31 자동) | 매년 12월 31일 23:59:59 시스템 자동 종료 (최종 레벨 확정·통계 마감 자동 처리) |
| 진행중 → 종료 ([강제 종료]) | 운영자 예외적 조기 종료 (확인 모달). 아티스트 활동 중단 등 |

- 운영자 정산 작업은 없다 — 디지털 굿즈(BIVE)는 진행중 실시간 자동 민팅, 래플·서포트는 예고 표시 전용.
- 종료 시즌은 수정 불가(읽기 전용). 신규 시즌은 [CEB-BO-EVT-101]에서 생성.
