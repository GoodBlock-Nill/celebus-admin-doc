## 1. 화면 목적

개별 아티스트(그룹)의 팬덤 레벨을 설정·관리하는 화면. 레벨 곡선(레벨 수·레벨별 목표 덕력), 레벨별 보상, 회원 변동 내역을 3탭으로 제공한다.

### 진입 경로

- [CEB-BO-EVT-101] 팬덤 레벨 리스트 > Row 클릭

## 2. UI 설명

### 2.1 상단 헤더

- Breadcrumb + 뒤로가기
- 상단 표기: "{그룹명}" + 현재 시즌("{아티스트} 팬덤 N기 (YYYY)") + **시즌 상태 뱃지(준비/진행중/정산중/종료)** + 현재 팬덤 레벨 뱃지(최고레벨 도달 시 "Lv.{N} MAX")

#### 상태 전환 액션 (상태별 노출)

| 현재 상태 | 노출 버튼 | 동작 | 전환 조건 |
| --- | --- | --- | --- |
| 준비 | [게시] | 준비 → 진행중. 앱 노출 시작 (확인 모달) | 레벨 곡선 1개 이상 설정 필수 |
| 진행중 | [정산 시작] | 진행중 → 정산중. 보상 지급(자동 + 래플) 진행 단계로 전환 (확인 모달) | — (시즌 자동 종료 12.31 전 조기 정산 가능) |
| 정산중 | [종료] | 정산중 → 종료. 시즌 마감 (확인 모달) | 모든 레벨 보상 지급·추첨 완료 |
| 종료 | (없음) | — | — |

- 시즌은 매년 12월 31일 23:59:59 자동으로 **정산중**으로 전환된다. [정산 시작]은 그 이전 조기 전환용이다.
- 상태 전환은 [CEB-BO-EVT-000] §4.1.1 라이프사이클을 따른다.

### 2.2 탭 메뉴 (3탭, 상호 배타)

| # | 탭명 | 내용 |
| --- | --- | --- |
| 1 | 레벨·곡선 설정 | 시즌 정보 + 레벨별 목표 덕력 설정 |
| 2 | 레벨별 보상 설정 | 레벨별 보상 매핑 |
| 3 | 보상·레벨업 현황 | 레벨업 이력 + 회원별 보상 지급 현황 조회 |

### 2.3 탭 1 — 레벨·곡선 설정

| 항목 | 표시 |
| --- | --- |
| 시즌 정보 | 시즌명·기간(1.1~12.31)·현재 팬덤 레벨·팬덤 전체 누적 덕력·최고레벨 도달 여부 (읽기) |
| 레벨 테이블 | 레벨(Lv.1~) · 레벨업 목표 덕력(입력) · 권장값(참고 표시) · 최고레벨 표시 · 달성 여부·달성 일시(읽기) |
| 레벨 추가/삭제 | 레벨 행 추가·삭제 (최대 10레벨). 이미 달성된 레벨은 삭제·목표 수정 불가 |
| [권장 곡선 적용] | 권장 곡선값 일괄 채움 |
| [저장] | 곡선 저장 (확인 모달 → 완료 토스트) |

- 카운터 모델: 각 레벨 0부터 시작, 목표 도달 시 다음 레벨 진입 + 카운터 0 리셋 ([CEB-BO-EVT-000] §4.2).
- **최고레벨(MAX) 운영**: 팬덤이 마지막 레벨에 도달하면 최고레벨(MAX) 상태가 된다. 진행중 시즌에도 신규 레벨을 추가할 수 있으며, 추가 시 최고레벨이 갱신되어 다시 운영중으로 전환된다 ([CEB-BO-EVT-000] §4.1.2).
- 신규 레벨 추가는 미달성 레벨(현재 레벨 초과)에 대해서만 가능하다.
- 신규 레벨의 보상은 추가 이후 그 레벨 달성 시점에 기여 회원에게 적용되며, 기존 달성·지급분은 변경되지 않는다(소급 없음). [CEB-BO-EVT-000] §4.1.2 참조.

### 2.4 탭 2 — 레벨별 보상 설정

| 항목 | 표시 |
| --- | --- |
| 레벨 선택 | Lv.1 ~ 최대 레벨 중 선택 |
| 보상 목록 | 레벨별 보상 행 리스트 (노출 순서대로) |
| 보상 추가/삭제 | 보상 행 추가·삭제 + 노출 순서 조정 |
| [래플 추첨 실행] | 실물·이벤트 보상의 당첨자 추첨. 레벨 달성 후 활성 (§3.4) |

#### 보상 행 입력 항목

| 항목 | 입력 | 비고 |
| --- | --- | --- |
| 보상 타입 | Dropdown 5종 (독점 콘텐츠 / 디지털 굿즈 / 다운로드 콘텐츠 / 실물 보상 / 이벤트 참여권) | [CEB-BO-EVT-000] §4.4 |
| 보상명 (다국어) | 한국어·영어·일본어 | 앱 표시명 |
| 지급 방식 | 전체 지급 / 추첨 | 디지털 3종은 전체 지급, 실물·이벤트는 추첨 |
| 당첨 인원 | 추첨 시 당첨자 수 입력 | 추첨일 때만 |
| 참조 연결 | 보상 연결 대상 — 바이브 / 래플 / 이벤트 / 뱃지 | 디지털 굿즈(바이브·뱃지), 실물(래플), 이벤트(이벤트) |
| 배송지 입력 | 배송지 입력 마감일·입력 폼 | 실물 보상 한정 |
| 다운로드 파일 | 콘텐츠 파일 등록 | 다운로드 콘텐츠 한정 |
| 노출 순서 | 같은 레벨 내 표시 순서 | — |

#### 보상 타입별 입력 규칙

보상 타입을 선택하면 지급 방식이 자동 고정되고, 타입별로 허용되는 참조 연결·필수 입력이 달라진다.

| 보상 타입 | 지급 방식(고정) | 참조 연결(필수) | 타입별 필수 입력 |
| --- | --- | --- | --- |
| 독점 콘텐츠 | 전체 지급 | 없음 | 보상명(다국어) |
| 디지털 굿즈 | 전체 지급 | 바이브 또는 뱃지 | 보상명(다국어) + 참조 연결 |
| 다운로드 콘텐츠 | 전체 지급 | 없음 | 보상명(다국어) + 다운로드 파일 |
| 실물 보상 | 추첨 | 래플 | 보상명(다국어) + 당첨 인원 + 배송지 입력(마감·폼) |
| 이벤트 참여권 | 추첨 | 이벤트 | 보상명(다국어) + 당첨 인원 |

- 지급 방식은 보상 타입에 따라 자동 결정되며 운영자가 임의 변경할 수 없다(디지털 3종=전체 지급, 실물·이벤트=추첨).
- 타입에 맞지 않는 참조 연결 선택·필수 입력 누락 시 저장이 차단된다.

#### 래플 추첨 실행

| 항목 | 설명 |
| --- | --- |
| 활성 조건 | 해당 레벨 달성 후 + 추첨 미실행(추첨 대기) 상태 |
| 실행 | [래플 추첨 실행] → 확인 모달 → 기여도 비례 가중치로 당첨자 확정 → 완료 토스트 |
| 추첨 상태 | 미실행(레벨 미달성) / 추첨 대기(달성 후 실행 전) / 추첨 완료 3단계 표시. 처리 관리자·실행 일시 기록 |
| 실패·재시도 | 추첨 처리 실패 시 [재시도] 노출. 실패 사유 안내 |

### 2.5 탭 3 — 보상·레벨업 현황

해당 그룹의 레벨업 이력과 회원별 보상 지급 현황을 조회한다. 회원 덕력 변동 자체는 덕력관리([아티스트 > 덕력관리]) 영역에서 관리한다.

#### 레벨업 이력

| 컬럼 | 설명 |
| --- | --- |
| 레벨 | 달성 레벨 (Lv.{N}) |
| 달성 일시 | 레벨업 달성 일시 (KST) |
| 처리 상태 | 보상 지급 처리 상태 (대기/처리중/완료/실패) |
| 대상 회원 수 | 해당 레벨 기여 회원(1DUK 이상) 수 |

#### 보상 지급 현황

| 컬럼 | 설명 |
| --- | --- |
| 회원 | 닉네임 |
| 레벨 | 보상 대상 레벨 |
| 보상명 | 지급 보상명 |
| 지급 방식 | 전체 지급 / 추첨 |
| 지급 상태 | 지급 대기 / 자동 지급 완료 / 추첨 대기 / 당첨 / 지급 완료 / 지급 실패 ([CEB-BO-EVT-000] §4.5) |
| 일시 | YYYY.MM.DD HH:mm (KST) |

- read-only 조회. 레벨·지급 상태 필터 제공. 전체 통합 조회는 [CEB-BO-EVT-401].
- **보상 미설정 레벨**은 레벨업 이력에만 표시되며, 지급할 보상이 없으므로 보상 지급 현황에는 행이 생성되지 않는다([CEB-BO-EVT-201] §3.3).

> 시각적 스타일은 피그마 참조

## 3. 규칙·로직

### 3.1 권한

- 권한 그룹별 읽기·쓰기·삭제 범위는 [CEB-BO-SYS-102] 권한 관리 그룹 설정을 따른다.
- 곡선 저장·보상 설정·래플 추첨은 쓰기 권한 보유 그룹만 수행한다.

### 3.2 곡선 검증

| 항목 | 규칙 |
| --- | --- |
| 레벨 수 | 1~10 |
| 목표 덕력 | 1 이상 정수. 레벨 순서대로 증가 권장(권장 곡선 기준) |
| 미설정 | 저장 전 권장값 자동 적용 |

### 3.3 보상 지급

- 디지털 보상(독점 콘텐츠·디지털 굿즈·다운로드)은 레벨 달성 시 기여 회원(1DUK 이상)에게 자동 지급.
- 실물·이벤트 보상은 기여 회원 중 래플 추첨(기여도 비례 가중치). 운영자가 [래플 추첨 실행]으로 당첨자 확정 → 배송·안내.
- 레벨별 보상 설정은 **선택**이다. 보상 미설정 레벨은 레벨업만 발생하고 지급 보상은 없다.

### 3.4 래플 추첨 활성 조건

| 조건 | 규칙 |
| --- | --- |
| 활성 시점 | 해당 레벨 **달성 후** + 추첨 대기 상태 |
| 비활성 | 레벨 미달성 / 이미 추첨 완료 시 비활성 |
| 재시도 | 추첨 처리 실패 시에만 [재시도] 활성 |

### 3.5 상태 전환 규칙

| 전환 | 조건 |
| --- | --- |
| 준비 → 진행중 ([게시]) | 레벨 곡선 1개 이상 설정 필수. 게시 후 앱 노출 시작 |
| 진행중 → 정산중 ([정산 시작]) | 운영자 조기 정산. 시즌 자동 종료(12.31) 시에도 자동 전환 |
| 정산중 → 종료 ([종료]) | 모든 레벨 보상 지급·추첨 완료 시 |

- 종료 시즌은 수정 불가(읽기 전용). 신규 시즌은 [CEB-BO-EVT-101]에서 생성.
