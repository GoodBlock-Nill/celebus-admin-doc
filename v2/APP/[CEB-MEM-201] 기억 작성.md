## 1. 화면 개요

##### 목적

기억(팬 다이어리)을 작성하거나 기존 기억을 수정하는 폼 화면이다. 작성(create)과 수정(edit) 모드를 하나의 화면으로 통합한다.

> 핵심 원칙: 팬이 아티스트와의 소중한 순간을 쉽게 기록할 수 있어야 한다.

##### 주요 기능

- 감정 이모지 **1개 단일 선택** (필수)
- 이미지 최대 3장 첨부 (**카메라 촬영 + 갤러리 선택** 둘 다 가능, 영상 미지원)
- 텍스트 최대 1,000자
- 공개/비공개 설정
- 수정 모드: 기존 데이터 프리필

##### 접근 경로

- CEB-MEM-101 > [+] 플로팅 작성 버튼 탭 (작성 모드)
- CEB-MEM-202 > ⋯ 메뉴 > [수정하기] (수정 모드)

##### 레이아웃 참조

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 장소 입력 필드 1줄 (탭 시 장소 검색 화면 진입) |
| 플레이스홀더 | "장소를 추가해 보세요 (선택)" |

| 스타일 | 피그마 참조 |

##### 장소 검색 (서브 화면)

| 항목 | 내용 |
| ---- | ---- |
| 진입 | 위치 입력 필드 탭 시 |
| 구성 | ← + 검색 입력창 + [X] 초기화 + 검색 결과 리스트 |
| 결과 항목 | 📍 장소명 + 주소 (예: "고척스카이돔 / 서울특별시 구로구 경인로") |
| 동작 | 검색어 입력 → 실시간 결과 리스트 → 항목 선택 시 지도 확인으로 이동 |

##### 지도 확인 (서브 화면)

| 항목 | 내용 |
| ---- | ---- |
| 진입 | 장소 검색 결과 선택 시 |
| 구성 | 선택 장소 핀이 표시된 지도 + 장소명·주소 |
| 동작 | 위치 확정 → 기억 작성 화면으로 복귀, 선택 장소가 위치 필드에 반영 |

> 입력한 장소는 기억저장소 지도 뷰(CEB-MEM-101 §2.4 지도 뷰)에서 위치 기반으로 표시된다.

### 2.5 미디어 업로드

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 이미지 | 최대 3장. 각 10MB 이하 (JPG/PNG/WebP/HEIC) |
| 영상 | 미지원 |
| UI | 썸네일 프리뷰 가로 스크롤 + [+] 추가 버튼 + 각 썸네일 [X] 개별 삭제 |
| 업로드 중 | 썸네일 위 프로그레스 바 표시 |
| 순서 | 업로드 순서대로 표시. 드래그 정렬 없음 (MVP) |
| 스타일 | 피그마 참조 |

#### 동작

- [+] 버튼 탭 → **OS 기본 피커 호출 (카메라 촬영 / 갤러리 선택 둘 다 제공)**. 사용자가 즉시 촬영하거나 기존 사진을 선택할 수 있음
- 선택 후 업로드 시작 → 썸네일 위 프로그레스 바 표시
- 업로드 완료 → 프로그레스 제거, 썸네일 정상 표시
- 업로드 실패 → 해당 썸네일에 에러 아이콘 + 탭 시 재시도
- 수정 모드: 기존 이미지는 썸네일 유지. 개별 삭제 가능, 추가 가능 (최대 3장)

#### 에러/예외

| 상황 | 처리 |
| ---- | ---- |
| 이미지 3장 초과 | 안내 토스트 "이미지는 최대 3장까지 추가할 수 있어요" |
| 이미지 용량 초과 | 에러 토스트 "10MB 이하의 이미지만 업로드할 수 있어요" |
| 이미지 포맷 에러 | 에러 토스트 "지원하지 않는 파일 형식이에요 (JPG/PNG/WebP/HEIC)" |
| 업로드 실패 | 해당 썸네일에 에러 아이콘 + 탭 시 재시도. 에러 토스트 "업로드에 실패했어요. 다시 시도해주세요" |

### 2.8 저장 CTA

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 버튼 텍스트 | "저장하기" |
| 활성 조건 | 감정 이모지 1개 이상 선택 |
| 비활성 상태 | 회색 처리, 탭 불가 |
| 스타일 | 피그마 참조 |

##### 동작 - 작성 모드

- [+] 플로팅 작성 버튼 탭 → 빈 폼 진입
- 필수 필드 입력 → [저장하기] 활성
- 저장 완료 → 완료 토스트 "기억이 저장되었어요! 덕력 75DUK 획득"
- CEB-MEM-101로 자동 복귀 + 캘린더 데이터 재조회

##### 동작 - 수정 모드

- CEB-MEM-202 > ⋯ 메뉴 > [수정하기] 탭 → 기존 데이터 프리필 진입
- 저장 완료 → 완료 토스트 "기억이 수정되었어요"
- CEB-MEM-202로 자동 복귀 (수정된 내용 반영)
- **덕력(Fan Power / 推し力) 중복 미지급** (최초 작성 시에만 **75DUK** — T3 고관여, [CEB-000] §5.2 보상 매트릭스 참조)
- **귀속**: 기억의 아티스트에 자동 귀속 ([CEB-000] §5.2.2 멀티 아티스트 귀속 매트릭스 참조)
- **캡 적용**: 일일 500DUK·주간 3,000DUK 캡에 합산

##### 에러/예외

| 상황 | 처리 |
| ---- | ---- |
| 저장 실패 | 에러 토스트 "저장에 실패했어요. 다시 시도해주세요" |
| 월간 한도 초과 (서버) | 에러 토스트 "이번 달 업로드 한도에 도달했어요" |
| 네트워크 에러 | 공통정책 11.3 참조 |
