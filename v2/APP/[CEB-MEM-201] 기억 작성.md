## 1. 화면 목적

기억(팬 다이어리)을 작성하거나 기존 기억을 수정하는 폼 화면이다. 작성(create)과 수정(edit) 모드를 하나의 화면으로 통합한다.

> 핵심 원칙: 팬이 아티스트와의 소중한 순간을 사진, 편지, 메모로 쉽게 기록할 수 있어야 한다.

##### 주요 기능

- 유형 선택 (사진 기록 / 편지 / 메모)
- 감정 이모지 1~3개 선택
- 이미지 최대 3장 업로드 (영상 미지원)
- 텍스트 최대 1,000자
- 공개/비공개 설정
- 수정 모드: 기존 데이터 프리필, 유형 변경 불가

##### 접근 경로

- CEB-MEM-101 > [+] FAB 탭 (작성 모드)
- CEB-MEM-202 > ⋯ 메뉴 > [수정하기] (수정 모드)

## 3. 화면 내 기능설명

### 3.1 기억 작성 (mode=create)

- [+] FAB 탭 → 빈 폼 진입
- 필수 필드 입력 → [저장하기] 활성
- 저장 완료 → 토스트(Success) "기억이 저장되었어요! 덕력 30pt 획득"
- CEB-MEM-101로 자동 복귀 + 캘린더 데이터 재조회

### 3.2 기억 수정 (mode=edit)

- CEB-MEM-202 > ⋯ 메뉴 > [수정하기] 탭 → 기존 데이터 프리필 진입
- 유형은 변경 불가 (고정 표시)
- 기존 이미지는 썸네일 유지. 개별 삭제 가능, 추가 가능 (최대 3장)
- 저장 완료 → 토스트(Success) "기억이 수정되었어요"
- CEB-MEM-202로 자동 복귀 (수정된 내용 반영)
- **덕력 중복 미지급** (최초 작성 시에만 30pt)

### 3.3 뒤로가기 / 취소

- 입력 내용이 있는 상태에서 ← 탭 → 취소 확인 모달:
  - 텍스트: "작성을 취소할까요?"
  - 서브텍스트: "입력한 내용이 사라져요"
  - CTA: [취소] (모달 닫기) / [나가기] (폼 초기화 + 이전 화면 복귀)
- 입력 내용이 없으면 모달 없이 바로 복귀

### 3.4 미디어 업로드

- [+] 버튼 탭 → OS 이미지 Picker 호출
- 선택 후 업로드 시작 → 썸네일 위 프로그레스 바 표시
- 업로드 완료 → 프로그레스 제거, 썸네일 정상 표시
- 업로드 실패 → 해당 썸네일에 에러 아이콘 + 탭 시 재시도

### 3.5 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 (수정) | 기존 기억 데이터 로드 + 폼 프리필 |
| 저장 완료 | 이전 화면으로 복귀 + 해당 화면 데이터 재조회 |

| 서버 검증 | 저장 API 호출 시에도 서버에서 한도 검증. 초과 시 거절 + 토스트(Error) |

| 미디어 | 추가/삭제 가능 |
| 텍스트 | 변경 가능 |

## 5. 에러 및 예외상황

| 상태 | 처리 |
| ---- | ---- |
| Loading (수정 모드) | 폼 영역 Skeleton UI (기존 데이터 로드 중) |
| 뒤로가기 (입력 있음) | 취소 확인 모달: "작성을 취소할까요? 입력한 내용이 사라져요" + [취소]/[나가기] |
| 뒤로가기 (입력 없음) | 모달 없이 바로 이전 화면 복귀 |
| 이미지 포맷 에러 | 토스트(Error) "지원하지 않는 파일 형식이에요 (JPG/PNG/WebP/HEIC)" |
| 이미지 용량 초과 | 토스트(Error) "10MB 이하의 이미지만 업로드할 수 있어요" |
| 이미지 3장 초과 | 토스트(Info) "이미지는 최대 3장까지 추가할 수 있어요" |
| 텍스트 1,000자 초과 | 입력 차단 + 글자 수 카운터 빨간색 표시 |
| 업로드 실패 | 해당 썸네일에 에러 아이콘 + 탭 시 재시도. 토스트(Error) "업로드에 실패했어요. 다시 시도해주세요" |
| 저장 실패 | 토스트(Error) "저장에 실패했어요. 다시 시도해주세요" |
| 월간 한도 초과 (서버) | 토스트(Error) "이번 달 업로드 한도에 도달했어요" |
| 네트워크 에러 | 공통정책 11.3 참조 |
| 비로그인 | CEB-MEM-101에서 로그인 유도 처리됨 (공통정책 10.5 참조). 본 화면은 로그인 상태에서만 접근 가능 |
