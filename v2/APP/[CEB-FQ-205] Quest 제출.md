## 1. 화면 목적

유저가 퀘스트의 구체적인 수행 방법(가이드)을 확인하고, 요구사항에 맞는 이미지(인증샷)를 촬영하거나 업로드하여, 운영진에게 심사를 요청(제출)하는 화면이다.

##### 주요 기능

- 퀘스트 기본정보 조회 (타이틀, 설명)
- 이미지 촬영 또는 갤러리 업로드
- 업로드 사진 가이드 확인 (아코디언)
- 제출하기 플로우 (심사 요청)

##### 접근 경로

- CEB-FQ-101 Quest 메인 > 활성 장 > 미션 [참여하기] 탭 시 이동 (캡처 제출형 미션)
- CEB-FQ-101 Quest 메인 > 타임라인 > 반려 미션 → CEB-FQ-201-MD 반려 사유 모달에서 [재도전] 탭 시 본 화면으로 이동 (기존 이미지 클리어 상태)
- 진입 가능 조건: 해당 미션의 상태가 INCOMPLETE(미완료) 또는 REJECTED(반려)인 경우에만 진입

## 2. UI 설명

### 2.1 공통 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.2 영역별 구성요소

##### Header 영역

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | [<] 뒤로가기 버튼 (CEB-FQ-101 Quest 메인으로 복귀) |
| 타이틀 | "Quest" |

##### Quest 정보 영역

| 항목 | 내용 |
| ---- | ---- |
| 타이틀 | 백오피스 [CEB-BO-FQ-101-Create] 타이틀 (유저 언어 기준) |
| 설명 | 백오피스 [CEB-BO-FQ-101-Create] 설명 (유저 언어 기준) |
| 보상 정보 | 승인 시 즉시 지급될 보상 아이콘 및 수량 (예: 응모권 1개, 덕력 {N}pt). 보상 지급 정책은 공통정책 8.5 참조 |

##### 연관 링크 영역

| 영역 | 항목 | 설명 |
| ---- | ---- | ---- |
| 연관 링크 | 링크 버튼 (최대 2개) | 백오피스(BO)에서 등록한 연관 링크를 라벨 버튼 형태로 표시. 클릭 시 인앱 브라우저 또는 외부 브라우저로 해당 URL 이동 |

- 연관 링크가 등록되지 않은 Quest의 경우 해당 영역 미노출
- 라벨 텍스트는 유저의 언어 설정(KO/EN/JA)에 맞춰 표시
- 연관 링크 데이터 출처: [CEB-BO-FQ-101-Create] Quest 생성 시 등록

##### 업로드 영역

| 항목 | 내용 |
| ---- | ---- |
| 초기 상태 | 갤러리 아이콘, 사진 촬영하기 아이콘 노출 |
| 업로드 후 | 선택한 이미지 미리보기 노출. 우측 상단 [X] (삭제) 버튼 제공 |

##### 가이드 영역 (아코디언)

| 항목 | 내용 |
| ---- | ---- |
| 타이틀 | "업로드 사진 가이드" + 펼치기/접기 화살표 |
| 기본 상태 | 접힌 상태 (타이틀만 표시) |
| 내용 | 어드민 [CEB-BO-FQ-101] 유저 가이드 텍스트 (예: 흐린 사진 반려 등) |

##### 하단 고정 바

| 항목 | 내용 |
| ---- | ---- |
| [제출하기] 버튼 | 초기엔 비활성(Dimmed). 이미지 업로드 완료 시 활성화(Active) |

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 초기 진입 및 유효성 체크

##### 진입 조건

| 미션 상태 | 처리 |
| ---- | ---- |
| INCOMPLETE (미완료) | 본 화면 진입 가능 |
| REJECTED (반려) | 본 화면 진입 가능 (기존 반려 이미지는 초기화/삭제된 상태로 진입) |
| SUBMITTED (심사중) | 진입 불가 (타임라인에서 비활성 처리) |
| APPROVED (승인) | 진입 불가 (타임라인에서 완료 상태 표시) |
| AUTO_COMPLETED (자동완료) | 진입 불가 (타임라인에서 완료 상태 표시) |

### 3.2 이미지 업로드 (Upload)

- 소스 선택: 업로드 영역 클릭 시 OS 기본 피커(카메라/갤러리)를 호출

##### 파일 제한

| 항목 | 규칙 |
| ---- | ---- |
| 형식 | JPG, PNG, WebP, HEIC |
| 매수 | 1장 (공통정책 13.3의 10장 규격 대신 Quest 제출 화면 고유 규칙 적용) |
| 용량 | 최대 10MB (초과 시 에러 처리) |

##### 상태 변화

| 상태 | 표시 |
| ---- | ---- |
| 이미지 선택 전 | Placeholder 노출, [제출하기] 버튼 Disabled |
| 이미지 선택 후 | Preview 노출, [제출하기] 버튼 Active |
| 이미지 삭제 시 | 다시 초기 상태(Placeholder)로 복귀 |

### 3.3 제출하기 버튼 동작

- 클릭 시: 중복 제출 방지를 위해 버튼을 로딩 상태(Spinner)로 변경하고 클릭을 막음
- 이미지를 서버로 전송

| 결과 | 처리 |
| ---- | ---- |
| 성공 | "제출 완료! 결과는 빠르면 하루 안에 알려드릴게요" Success 토스트 노출 → 화면을 닫고 CEB-FQ-101 Quest 메인(타임라인)으로 복귀 → 해당 미션의 로컬 상태를 SUBMITTED(심사중)으로 업데이트 |
| 실패 | 에러 유형별 Error 토스트 노출 (5. 에러 및 예외상황 참조) |

> 제출 즉시 해당 건은 Admin의 검토 대기 목록([CEB-BO-FQ-202])에 등록된다.

## 4. 규칙/로직

### 4.1 데이터 노출 조건

| 항목 | 노출 조건 |
| ---- | ---- |
| 가이드 텍스트 | 어드민에서 유저 가이드를 설정한 경우에만 가이드 영역 표시 |
| [제출하기] 버튼 활성화 | 이미지가 업로드된 상태에서만 Active |

### 4.2 반려 후 재진입

- 기존에 반려된 이미지는 초기화(삭제)된 상태로 진입
- 유저는 새 이미지를 업로드하여 다시 제출 가능
- 진입 경로: CEB-FQ-101 타임라인 > 반려 미션 [재제출] 탭 (CEB-FQ-101 3.2 참조)

### 4.3 보상 지급 정책

- 보상(덕력, 응모권 등)은 심사 승인 시 즉시 지급
- 보상 지급 상세 정책은 공통정책 8.5 참조

### 4.4 심사 SLA

- 제출 완료 토스트에 "결과는 빠르면 하루 안에 알려드릴게요" 심사 완료 기준 안내
- 심사 SLA 상세 기준은 CEB-FQ-101 4.6 참조

### 4.5 다국어 처리

| 항목 | 다국어 범위 |
| ---- | ---- |
| 퀘스트 타이틀 | 서버 데이터 (다국어 지원 여부 어드민 설정에 따름) |
| 퀘스트 설명 | 서버 데이터 (다국어 지원 여부 어드민 설정에 따름) |
| 가이드 텍스트 | 서버 데이터 (다국어 지원 여부 어드민 설정에 따름) |
| 시스템 UI 텍스트 | KO / EN / JP (앱 다국어 리소스) |

## 5. 에러 및 예외상황

##### 업로드 실패

| 상황 | 처리 |
| ---- | ---- |
| Loading | Quest 정보 영역 Skeleton UI |
| Default | Quest 정보 + 업로드 영역 + 가이드 정상 표시 |
| 네트워크 불안정 또는 파일 손상으로 업로드 실패 | "앗, 사진이 올라가지 않았어요. 다시 시도해 주세요" Error 토스트 |

##### 용량 초과

| 상황 | 처리 |
| ---- | ---- |
| 제한 용량(10MB)을 초과하는 고화질 이미지 선택 | "10MB 이하의 이미지만 업로드 가능합니다." Error 토스트 |

##### 기기 권한 거부

| 상황 | 처리 |
| ---- | ---- |
| 유저가 카메라/갤러리 접근 권한을 거부한 상태 | "사진을 업로드하려면 권한 설정이 필요합니다." 안내 후 OS 설정 화면으로 이동 유도 |
| 파일 형식 불일치 | 지원하지 않는 파일 형식 선택 시 Error 토스트: "JPG, PNG, WebP, HEIC 형식만 올릴 수 있어요" |
| 제출 실패 복구 | 네트워크 에러로 제출 실패 시: Error 토스트 + 버튼 활성 상태로 복구 (재시도 가능) |
| 중복 제출 방지 | 진입 시 미션 상태 재확인. 이미 SUBMITTED이면 진입 차단 + 토스트(Info) "이미 제출된 미션이에요" |
