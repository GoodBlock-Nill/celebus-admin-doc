## 1. 화면 목적

팔로우할 아티스트를 탐색하고 추가/제거하는 화면이다. 런칭 초기 소수 아티스트부터 대규모 확장까지 대응한다.

> 핵심 원칙: 팬이 좋아하는 아티스트를 쉽게 찾고, 한 탭에 팔로우할 수 있어야 한다.

##### 주요 기능

- 추천 아티스트 카드로 빠른 탐색
- 전체 아티스트 리스트에서 검색/팔로우
- 팔로우 중인 아티스트 관리 (언팔로우)

##### 접근 경로

- 홈 > 아티스트 선택 영역 > [+] 추가 버튼
- 아티스트 0명 팔로우 시 자동 진입 (공통정책 10.3.1)

## 2. UI 설명

### 2.1 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.2 영역별 구성요소

##### 1. 헤더

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | ← 뒤로가기 (이전 화면 복귀) |
| 타이틀 | "아티스트" |
| 우측 | 🔍 검색 아이콘. 탭 시 검색바 노출 |
| 스타일 | 피그마 참조 |

##### 2. 검색바 (검색 아이콘 탭 시 노출)

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 헤더 아래 슬라이드 노출 |
| 구성 | 텍스트 입력 + [X] 초기화 버튼 |
| 동작 | 아티스트명 한글/영문 실시간 필터링 |
| 플레이스홀더 | "아티스트 검색" |
| 결과 없음 | "앗, '{검색어}'에 해당하는 아티스트가 없어요" |
| 스타일 | 피그마 참조 |

##### 3. 내 아티스트 (팔로우 중)

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 검색바 아래 |
| 노출 조건 | 팔로우 중인 아티스트 1명 이상일 때만 노출 |
| 구성 | 가로 스크롤 원형 로고 리스트. 각 로고에 ✓ 체크 뱃지 |
| 탭 동작 | 언팔로우 확인 모달 표시 |
| 스타일 | 피그마 참조 |

##### 4. 추천 아티스트

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 내 아티스트 아래 |
| 노출 조건 | BO에서 추천 설정한 아티스트가 1명 이상일 때만 노출 |
| 구성 | 가로 스크롤 카드. 카드: 배경 이미지 + 아티스트명 + 멤버 수 + [팔로우] 버튼 |
| 이미 팔로우 | 버튼 비활성 "팔로우 중" 표시 |
| 스타일 | 피그마 참조 |

##### 5. 전체 아티스트

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 추천 아래 |
| 구성 | 세로 리스트. 각 행: 원형 로고 + 아티스트명 + 영문명 + [팔로우]/[팔로우 중] 버튼 |
| 팔로우 버튼 | 미팔로우 시 바이올렛 [팔로우] 버튼. 팔로우 중이면 회색 [팔로우 중] |
| 정렬 | 가나다순 (한글 기준) |
| 스타일 | 피그마 참조 |

## 3. 화면 내 기능설명

### 3.1 아티스트 팔로우

- [팔로우] 버튼 탭 → 즉시 팔로우 완료
- 토스트(Success) "'{아티스트명}'을 팔로우했어요!"
- 홈으로 자동 이동 + 해당 아티스트로 전환
- 하단 탭 2번째 탭명이 해당 아티스트명으로 변경

> 팔로우 즉시 홈으로 이동하는 이유: 팬이 팔로우한 아티스트의 홈 경험을 바로 체험하도록 유도. 추가 팔로우가 필요하면 홈에서 [+] 버튼으로 재진입.

### 3.2 아티스트 언팔로우

- 내 아티스트 로고 탭 또는 전체 리스트 [팔로우 중] 버튼 탭
- 언팔로우 확인 모달 표시:
  - 텍스트: "'{아티스트명}'을 언팔로우할까요?"
  - 서브텍스트: "기존 활동 데이터는 보존됩니다."
  - CTA: [취소] / [언팔로우]
- 언팔로우 완료 → 토스트(Success) "언팔로우했어요. 데이터는 보존돼요"
- 데이터 처리: 공통정책 10.3.1 참조 (DB 유지, UI 미노출, 재팔로우 시 즉시 복원)

### 3.3 마지막 아티스트 언팔로우

- 언팔로우 완료 후 화면 유지 (새 아티스트를 팔로우하도록 유도)
- 홈/하단 탭은 "아티스트를 추가해보세요!" 안내 상태로 전환 (유저가 뒤로가기로 홈 복귀 시 반영)

### 3.4 검색

- 헤더 🔍 아이콘 탭 → 검색바 노출
- 아티스트명 한글/영문 모두 실시간 필터링
- 검색어 입력 시 전체 리스트가 필터링된 결과로 표시
- [X] 버튼 탭 → 검색어 초기화, 전체 리스트 복원

### 3.5 뒤로가기

- ← 탭 → 이전 화면(홈) 복귀

### 3.6 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 아티스트 리스트 + 팔로우 상태 조회 |
| 팔로우/언팔로우 | 해당 아티스트 상태 즉시 반영 |
| Pull-to-Refresh | 전체 재조회 (공통정책 11.2 참조) |

## 4. 규칙/로직

### 4.1 정렬 규칙

| 영역 | 정렬 |
| ---- | ---- |
| 내 아티스트 | 팔로우 순서 (최근 팔로우가 우측) |
| 추천 | BO 설정 순서 |
| 전체 | 가나다순 (한글 기준), 영문명 보조 표시 |

### 4.2 팔로우 제한

| 규칙 | 설명 |
| ---- | ---- |
| 최대 팔로우 수 | 제한 없음 |
| 최소 팔로우 수 | 0명 가능 (마지막 언팔로우 허용) |
| 중복 팔로우 | 불가 (이미 팔로우 중이면 버튼 비활성) |

### 4.3 비로그인 정책 (공통정책 10.5)

| 상태 | 정책 |
| ---- | ---- |
| 비로그인 | 아티스트 리스트 전체 열람 가능 (서비스 매력 체험) |
| [팔로우] 탭 | → 로그인 화면 이동 |
| 검색 | 비로그인에서도 사용 가능 |

### 4.4 멀티 아티스트 대응 (공통정책 10 참조)

| 상황 | 동작 |
| ---- | ---- |
| 첫 팔로우 | 팔로우 완료 → 홈 이동 + 해당 아티스트 자동 선택 |
| 추가 팔로우 | 팔로우 완료 → 홈 이동 + 해당 아티스트로 전환 |
| 언팔로우 (남은 1명+) | 내 아티스트에서 제거. 홈은 다른 아티스트로 자동 전환 |
| 전체 언팔로우 | 화면 유지. 홈은 "아티스트를 추가해보세요!" 표시 |

## 5. 에러 및 예외상황

| 상태 | 처리 |
| ---- | ---- |
| Loading | Skeleton UI (카드 3~4개 플레이스홀더) |
| Default | 추천 + 전체 리스트 정상 표시 |
| Error | "앗, 불러오지 못했어요" + [다시 불러오기] 버튼 |
| 아티스트 0개 (서비스 전체) | "곧 새로운 아티스트가 찾아올 거예요!" Empty 일러스트 |
| 추천 0건 | 추천 섹션 미노출 |
| 전체 모두 팔로우 | 모든 [팔로우] 버튼 "팔로우 중" 비활성. 상단 안내 "모든 아티스트를 팔로우하고 있어요!" |
| 검색 결과 없음 | "앗, '{검색어}'에 해당하는 아티스트가 없어요" |
| 네트워크 에러 | 공통정책 11.3 참조 |
| 팔로우 실패 | 토스트(Error) "팔로우에 실패했어요. 다시 시도해주세요" |
| 언팔로우 실패 | 토스트(Error) "언팔로우에 실패했어요. 다시 시도해주세요" |
