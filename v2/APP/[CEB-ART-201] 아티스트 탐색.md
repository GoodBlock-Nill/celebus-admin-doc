## 1. 화면 개요

##### 목적

팔로우할 아티스트를 탐색하고 추가/제거하는 화면이다. 런칭 초기 소수 아티스트부터 대규모 확장까지 대응한다.

> 핵심 원칙: 팬이 좋아하는 아티스트를 쉽게 찾고, 한 탭에 팔로우할 수 있어야 한다.

##### 주요 기능

- 추천 아티스트 카드로 빠른 탐색
- 전체 아티스트 리스트에서 검색/팔로우
- 팔로우 중인 아티스트 관리 (언팔로우)

##### 접근 경로

- 홈 > 아티스트 선택 영역 > [+] 추가 버튼
- 최초 가입 시 아티스트 선택 필수 (온보딩에서 진입)

##### 레이아웃 참조

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

## 2. 영역별 상세

### 2.1 서브페이지 헤더 + 검색

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | ← 뒤로가기 (이전 화면 복귀) |
| 타이틀 | "아티스트" |
| 우측 | 🔍 검색 아이콘. 탭 시 검색바 노출 |
| 스타일 | 피그마 참조 |

#### 검색바 (검색 아이콘 탭 시 노출)

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 헤더 아래 슬라이드 노출 |
| 구성 | 텍스트 입력 + [X] 초기화 버튼 |
| 플레이스홀더 | "아티스트 검색" |
| 스타일 | 피그마 참조 |

#### 동작

- ← 탭 → 이전 화면(홈) 복귀
- 🔍 아이콘 탭 → 검색바 노출
- 아티스트명 한글/영문 모두 실시간 필터링
- 검색어 입력 시 전체 리스트가 필터링된 결과로 표시
- [X] 버튼 탭 → 검색어 초기화, 전체 리스트 복원

#### 상태별 표시

| 상태 | 처리 |
| ---- | ---- |
| 검색 결과 없음 | "앗, '{검색어}'에 해당하는 아티스트가 없어요" |

#### 비로그인 (공통정책 10.5)

- 비로그인에서도 검색 사용 가능

### 2.2 내 아티스트

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 검색바 아래 |
| 노출 조건 | 팔로우 중인 아티스트 1명 이상일 때만 노출 |
| 구성 | 가로 스크롤 원형 로고 리스트. 각 로고에 ✓ 체크 뱃지 |
| 정렬 | 팔로우 순서 (최근 팔로우가 우측) |
| 스타일 | 피그마 참조 |

#### 동작

- 로고 탭 → 언팔로우 확인 모달 표시
- 팔로우 중인 아티스트가 1명뿐일 때 로고 탭 시: 에러 토스트 "최소 1명의 아티스트를 팔로우해야 해요". 언팔로우 모달 미표시

### 2.3 추천 아티스트

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 내 아티스트 아래 |
| 노출 조건 | BO에서 추천 설정한 아티스트가 1명 이상일 때만 노출. 추천 그룹도 **상태 Active + 탐색 노출 켬**이어야 노출 (탐색 노출 끔이면 추천에서도 제외) |
| 구성 | 가로 스크롤 카드. 카드: 배경 이미지 + 아티스트명 + 멤버 수 + [팔로우] 버튼 |
| 이미 팔로우 | 버튼 비활성 "팔로우 중" 표시 |
| 정렬 | BO 설정 순서 |
| 스타일 | 피그마 참조 |

#### 동작

- [팔로우] 버튼 탭 → 즉시 팔로우 완료
- 완료 토스트 "'{아티스트명}'을 팔로우했어요!"
- 홈으로 자동 이동 + 해당 아티스트로 전환
- 하단 탭 2번째 탭명이 해당 아티스트명으로 변경

> 팔로우 즉시 홈으로 이동하는 이유: 팬이 팔로우한 아티스트의 홈 경험을 바로 체험하도록 유도. 추가 팔로우가 필요하면 홈에서 [+] 버튼으로 재진입.

#### 상태별 표시

| 상태 | 처리 |
| ---- | ---- |
| 추천 0건 | 추천 섹션 미노출 |

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 팔로우 실패 | 에러 토스트 "팔로우에 실패했어요. 다시 시도해주세요" |

### 2.4 전체 아티스트

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 추천 아래 |
| 노출 조건 | BO에서 **상태 Active + 탐색 노출 켬**으로 설정된 그룹만 노출 (BO [CEB-BO-ART-201] 그룹 리스트 [탐색 노출] 컬럼 기준). Inactive 또는 탐색 노출 끔 그룹은 미노출 |
| 구성 | 세로 리스트. 각 행: 원형 로고 + 아티스트명 + 영문명 + [팔로우]/[팔로우 중] 버튼 |
| 팔로우 버튼 | 미팔로우 시 바이올렛 [팔로우] 버튼. 팔로우 중이면 회색 [팔로우 중] |
| 정렬 | 가나다순 (한글 기준), 영문명 보조 표시 |
| 스타일 | 피그마 참조 |

#### 동작

- [팔로우] 버튼 탭 → 즉시 팔로우 완료. 완료 토스트 + 홈 이동 + 해당 아티스트로 전환
- [팔로우 중] 버튼 탭 → 언팔로우 확인 모달 표시
- 팔로우 중 1명뿐일 때 [팔로우 중] 탭 시: 에러 토스트 "최소 1명의 아티스트를 팔로우해야 해요". 버튼 비활성 처리

#### 상태별 표시

| 상태 | 처리 |
| ---- | ---- |
| Loading | Skeleton UI (카드 3~4개 플레이스홀더) |

| Error | "앗, 불러오지 못했어요" + [다시 불러오기] 버튼 |
| 아티스트 0개 (서비스 전체) | "곧 새로운 아티스트가 찾아올 거예요!" Empty 일러스트 |
| 전체 모두 팔로우 | 모든 [팔로우] 버튼 "팔로우 중" 비활성. 상단 안내 "모든 아티스트를 팔로우하고 있어요!" |

## 3. 공통 규칙

### 3.1 팔로우 제한

| 규칙 | 설명 |
| ---- | ---- |
| 최대 팔로우 수 | 제한 없음 |
| 최소 팔로우 수 | **1명 필수** (마지막 아티스트 언팔로우 차단) |
| 중복 팔로우 | 불가 (이미 팔로우 중이면 버튼 비활성) |

### 3.2 멀티 아티스트 대응 (공통정책 10 참조)

| 상황 | 동작 |
| ---- | ---- |
| 첫 팔로우 | 팔로우 완료 → 홈 이동 + 해당 아티스트 자동 선택 |
| 추가 팔로우 | 팔로우 완료 → 홈 이동 + 해당 아티스트로 전환 |
| 언팔로우 (남은 1명+) | 내 아티스트에서 제거. 홈은 다른 아티스트로 자동 전환 |
| 마지막 아티스트 언팔로우 | 차단. 에러 토스트 "최소 1명의 아티스트를 팔로우해야 해요" |

### 3.3 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 아티스트 리스트 + 팔로우 상태 조회 |
| 팔로우/언팔로우 | 해당 아티스트 상태 즉시 반영 |
| Pull-to-Refresh | 전체 재조회 (공통정책 11.2 참조) |
