## 1. 화면 개요

아티스트 탭(CEB-ART-101)에서 "{아티스트명} 응원하기" 카드를 탭했을 때 진입하는 화면이다. (유저 대면 명칭: {아티스트명} 응원하기, 예: V01D 응원하기) 아티스트 서포트 이벤트 목록을 표시하고, 유저가 보유 덕력을 응원하여 팬덤 서포트에 참여할 수 있다.

##### 주요 기능

- 서포트 이벤트 카드 리스트 (상태별: 모집중 / 달성 / 집행중 / 완료 / 미달성종료 / 집행취소)
- 카드 탭 시 인라인 확장 (상세 정보 + 덕력 응원 UI + 진행률)
- 덕력 응원: 수량 입력 + 확인 모달 + 응원 완료
- 완료 이벤트 결과 확인 (사진/영상 인라인 표시)

##### 접근 경로

- CEB-ART-101 > 서포트 카드 탭
- CEB-HOM-001 > 바로가기 > {아티스트명} 응원하기 카드 탭

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.4 덕력 응원 UI

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 수량 입력 | [-] 버튼 + 수량 필드 (직접 입력 가능) + [+] 버튼 |
| 증감 단위 | [-]/[+] 탭 시 100pt 단위 증감 |
| 최소 | 1pt |
| 최대 | min(보유 덕력, 남은 목표량). 초과 입력 시 자동 조정 |
| 보유 표시 | "보유: {N}pt" (입력 필드 아래) |
| [응원하기] 버튼 | 수량 0이면 비활성. 1 이상이면 활성 |
| 달성 상태 | 응원 UI 전체 비활성 + "목표 달성! 감사합니다 💜" 텍스트 |

#### 동작

##### 덕력 응원 플로우

1. 수량 입력 (직접 입력 또는 [-]/[+] 버튼)
2. [응원하기] 탭 → 확인 모달 표시
3. 확인 모달에서 [응원] 탭 → 서버 처리
4. 성공 → 토스트(Success) "덕력 {N}pt 응원 완료!" + 프로그레스 바 갱신 + "내 응원" 갱신 + 보유 덕력 차감 반영
5. 달성 트리거: 응원으로 누적 >= 목표 시 서버가 즉시 달성 상태로 전환 → 응원 UI 비활성 + 토스트(Success) "🎯 목표 달성!"

##### 누적 응원

- 동일 이벤트에 여러 번 응원 가능
- "내 응원: {N}pt"는 누적 합산 표시
- 매 응원마다 확인 모달 표시

##### 초과 응원 방지

- UI에서 입력량이 남은 목표량을 초과 시 남은 목표량으로 자동 조정 ("남은 목표: {N}pt" 표시)
- 서버에서도 (현재 누적 + 응원량 > 목표) 확인하여 초과분은 보유 덕력에 유지
- 예: 목표 15,000pt, 현재 14,800pt, 500pt 응원 시 → 200pt만 차감, 300pt 유지

##### 덕력 응원 규칙

| 규칙 | 설명 |
| ---- | ---- |
| 응원 대상 | 보유 덕력 / Available Fan Power 에서만 차감 (획득 덕력/랭킹에 영향 없음). 응원 덕력은 **해당 서포트 이벤트의 아티스트**에 귀속 ([CEB-000] §5.2.2 멀티 아티스트 귀속 매트릭스 참조) |
| 최소 | 1pt |
| 최대 | min(보유 덕력, 남은 목표량) |
| 누적 | 동일 이벤트에 여러 번 응원 가능 |
| 취소 | 불가 (확인 모달에서 "한번 응원하면 돌이킬 수 없어요" 고지) |
| 0pt 입력 | [응원하기] 버튼 비활성 |
| 달성 후 | 응원 UI 전체 비활성 |

### 2.6 결과 표시

#### UI 구성

> 완료/미달성종료/집행취소 상태의 확장 영역 UI는 2.3 이벤트 카드 (확장) 참조.

#### 동작

##### 이벤트 상태 정의 (6종)

| 상태 | 코드 | 전이 트리거 | 유저 액션 |
| ---- | ---- | ---- | ---- |
| 모집중 | Active | BO에서 이벤트 생성 | 덕력 응원 가능 |
| 달성 | Achieved | 서버 자동 (누적 응원 >= 목표) | 응원 차단. 집행 대기 |
| 집행중 | Executing | BO 수동 전환 | 대기 (읽기 전용) |
| 완료 | Completed | BO 수동 전환 (결과물 업로드) | 결과 확인 |
| 미달성종료 | Expired | 서버 자동 (마감일시 경과 + 미달성) | 덕력 반환 확인 |
| 집행취소 | Cancelled | BO 수동 전환 (달성 후 집행 불가 시) | 덕력 반환 확인 |

##### 상태 전이 규칙

| 현재 상태 | 전이 조건 | 다음 상태 |
| ---- | ---- | ---- |
| 모집중 | 누적 응원 >= 목표 (서버 실시간 판정) | 달성 |
| 모집중 | 마감일시 경과 + 목표 미달성 (서버 자동) | 미달성종료 |
| 모집중 | BO 강제 종료 | 미달성종료 (전액 반환) |
| 달성 | BO에서 집행 시작 전환 | 집행중 |
| 달성 | BO에서 집행 취소 전환 | 집행취소 (전액 반환) |
| 집행중 | BO에서 결과물 업로드 + 완료 전환 | 완료 |
| 집행중 | BO에서 집행 취소 전환 | 집행취소 (전액 반환) |

##### 반환 규칙

| 상황 | 처리 |
| ---- | ---- |
| 미달성종료 | 모든 참여자에게 응원 덕력 전액 반환 (보유 덕력 복구) |
| 집행취소 | 모든 참여자에게 응원 덕력 전액 반환 |
| BO 강제 종료 | 전액 반환 |
| 달성 + 정상 집행 | 반환 없음 (서포트 집행에 사용) |
| 반환 시점 | 상태 전이 즉시 (서버 자동) |
| 반환 알림 | 푸시 알림 + 화면 진입 시 토스트(Info) |

## 3. 공통 규칙

### 3.1 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |

| 복귀 시 | 상태 변경 가능성 재조회 |

### 3.2 Pull-to-Refresh

- 공통정책 11.2 규격 적용

### 3.3 멀티 아티스트 대응

- 서포트 이벤트는 아티스트별 분리 (`artist_id` 기반)
- 아티스트 전환 시 해당 아티스트의 이벤트 목록으로 갱신

### 3.4 비로그인 공통

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역의 '비로그인' 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

- 비로그인도 열람 가능. 이벤트 목록/상세 표시
- 내 응원 "-" 표시
