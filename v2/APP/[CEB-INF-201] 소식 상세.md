## 1. 화면 개요

정보 피드(CEB-INF-101)에서 소식(뉴스) 항목을 탭했을 때 진입하는 상세 화면이다. 아티스트 관련 소식의 본문, 이미지, 관련 링크를 풍부하게 보여주고 공유할 수 있다.

##### 주요 기능

- 소식 본문 전체 표시 (제목 + 본문 텍스트 + 이미지)
- 공식 제공 뱃지 표시 (공식 출처 표기된 콘텐츠)
- 관련 링크 바로가기
- 소식 공유 (Web Share API)
- 이전/다음 소식 네비게이션

##### 접근 경로

- CEB-INF-101 > 소식(📰) 항목 탭

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입을 참조하세요. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

## 2. 영역별 상세

### 2.1 서브페이지 헤더

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | ← 뒤로가기 (이전 화면 복귀) |
| 타이틀 | "소식" |
| 우측 | 공유 버튼 (↗ 아이콘) |
| 스타일 | 피그마 참조 |

#### 동작

- ← 탭 → CEB-INF-101 복귀
- 공유 버튼 탭 → Web Share API 호출 (지원 안 될 경우 URL 클립보드 복사 + 토스트)

---

### 2.2 소식 헤더

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 공식 출처 뱃지 | BO에서 "공식 출처 표기" 플래그 설정된 소식에 "[공식 제공]" 태그 표시. 미설정 시 미노출 |
| 제목 | 소식 타이틀 (최대 2줄, 초과 시 말줄임) |
| 날짜 | YYYY.MM.DD 형식 |
| 아티스트 | {아티스트명} 아바타 + 이름 표시 |
| 스타일 | 피그마 참조 |

---

### 2.3 소식 본문

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 이미지 | 소식에 첨부된 이미지 (최대 10장). 세로 나열. 탭 시 전체화면 확대 |
| 본문 텍스트 | BO에서 등록한 본문 (HTML 또는 Markdown 렌더링). 링크 자동 감지 + 하이라이트 |
| 영상 | 외부 영상 링크가 있을 경우 인라인 임베드 (YouTube/Vimeo 등) |
| 스타일 | 피그마 참조 |

#### 동작

- 이미지 탭 → 전체화면 갤러리 (좌우 스와이프, 핀치 줌)
- 본문 내 링크 탭 → 외부 브라우저 열기

---

### 2.4 관련 링크

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 표시 조건 | BO에서 등록한 관련 링크가 있을 때만 표시. 0건이면 섹션 미노출 |
| 구성 | 링크 라벨 + URL 버튼 형태. 복수 링크 세로 나열 |
| 최대 건수 | **최대 3건**. BO 등록 시 3건 초과 불가 |
| 스타일 | 피그마 참조 |

#### 동작

- 링크 버튼 탭 → 외부 브라우저에서 URL 열기

---

### 2.5 이전/다음 네비게이션

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 본문 하단 |
| 구성 | [← 이전 소식] / [다음 소식 →] 버튼 |
| 표시 | 이전/다음이 없으면 해당 버튼 비활성 |
| 스타일 | 피그마 참조 |

#### 동작

- 이전/다음 탭 → 동일 아티스트의 인접 소식으로 이동 (날짜순)
- 최초 소식: [← 이전 소식] 비활성
- 최신 소식: [다음 소식 →] 비활성

---

### 2.6 하단 액션 바

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 좋아요 | ❤️ 아이콘 + 좋아요 수. 토글 가능 |
| 공유 | ↗ 공유 아이콘. 탭 시 Web Share API |
| 스타일 | 하단 고정. 피그마 참조 |

#### 동작

- 좋아요 탭 → 토글 (좋아요 추가/해제) + 카운트 갱신
- 공유 탭 → Web Share API (미지원 시: 클립보드 복사)

#### 비로그인

- 좋아요 탭 시 → 로그인 화면 이동 (공통정책 10.5 참조)
- 공유는 비로그인도 가능

## 3. 공통 규칙

### 3.1 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 소식 상세 데이터 조회 |
| 좋아요 토글 | 서버 반영 + 카운트 즉시 갱신 |
| Pull-to-Refresh | 소식 데이터 재조회 (공통정책 11.2 참조) |

### 3.2 멀티 아티스트 대응

- 소식은 아티스트별 분리 (`artist_id` 기반)
- 이전/다음 네비게이션도 동일 아티스트 내에서만 이동

### 3.3 비로그인 공통

- 비로그인도 소식 상세 전체 열람 가능
- 좋아요 CTA 탭 시 → 로그인 화면 이동

### 3.4 공통 에러 상태

| 상태 | 처리 |
| ---- | ---- |
| Loading | Skeleton UI (헤더 + 본문 영역) |
| Default | 소식 상세 정상 표시 |
| Error | 에러 안내 + [다시 불러오기] 버튼 |
| 삭제된 소식 | "삭제된 소식이에요" + [돌아가기] CTA |
