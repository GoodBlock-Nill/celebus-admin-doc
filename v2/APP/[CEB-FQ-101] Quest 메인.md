## 1. 화면 목적

아티스트 탭(CEB-ART-101)에서 "챌린지" 카드를 탭했을 때 진입하는 화면이다. 서사 Quest 5장의 진행 상황을 타임라인으로 보여주고, 5장 완료 후에는 반복 퀘스트 리스트로 전환된다.

##### 주요 기능

- 5장 서사 타임라인으로 진행 상황 한눈에 파악
- 활성 장의 미션 인라인 표시 (제출/심사중/승인/반려 상태)
- 제출 기반 임시 해금으로 심사 대기 중에도 다음 장 진행 가능
- 5장 완료 후 축소 배너 + 반복 퀘스트 리스트로 자동 전환

##### 접근 경로

- CEB-ART-101 > 챌린지 카드 탭
- CEB-HOM-001 > 추천 카드 [이어하기] / [퀘스트 보기] 탭
- CEB-HOM-001 > 오늘의 할 일 > 퀘스트 [이어서→] 탭

## 3. 화면 내 기능설명

### 3.1 타임라인 노드 탭

- 잠금 노드: 프리뷰 바텀시트 표시 (2.2 잠금 장 프리뷰 바텀시트 참조)
- 활성/임시해금/심사중/완료 노드: 해당 장의 인라인 미션 리스트 토글 (확장/축소)
- 동시에 1개 장만 확장 (다른 장 확장 시 이전 장 자동 축소)

### 3.2 미션 탭 액션 (타입별 상세)

#### 📸 캡처 제출형 (Ch1, Ch2, Ch5)

| 동작 | 설명 |
| ---- | ---- |
| [참여하기] 탭 | → CEB-FQ-205 Quest 제출 화면 이동. 연관 링크는 제출 화면(CEB-FQ-205)에서 표시 |
| 이미지 업로드 + 제출 | → 미션 상태 SUBMITTED (심사중)으로 전환 |
| 반려 시 [사유 확인] 탭 | → 반려 사유 모달(CEB-FQ-201-MD) 표시 → [재도전] → CEB-FQ-205 (기존 이미지 클리어) |
| 심사중 | 탭 불가 (비활성 상태) |
| 승인 완료 | 탭 시 보상 정보 인라인 표시 |

#### 🧠 Trivia 참여 (Ch3)

| 동작 | 설명 |
| ---- | ---- |
| [도전하기] 탭 | → CEB-GAM-204 Trivia 홈 딥링크 이동 |
| 조건 충족 (7/10 이상) | → 미션 AUTO_COMPLETED (서버 자동 연동). 복귀 시 상태 재조회로 반영 |
| 조건 미달 (7문제 미만) | → 미션 INCOMPLETE 유지. **즉시 재참여 가능 (횟수 제한 없음)** |
| 복귀 시 | 미션 상태 자동 재조회 (3.4 데이터 갱신 참조) |

#### 🎯 PM 참여 (Ch4)

| 동작 | 설명 |
| ---- | ---- |
| [참여하기] 탭 | → CEB-GAM-202 PM 상세 딥링크 이동. 각 미션 = PM 게임 1회 참여 |
| 참여 완료 | → 해당 미션 AUTO_COMPLETED (서버 자동 연동) |
| PM 게임 미가용 시 | → 토스트(Info) "현재 진행중인 PM 게임이 없습니다". 미션 항목에 "게임 준비중" 상태 표시 + [알림 설정] CTA. [알림 설정] 탭 시 토스트(Success) '새 PM 게임이 등록되면 알려드릴게요'. 서버에서 PM 게임 등록 시 푸시 알림 발송 (공통정책 15 참조) |
| 복귀 시 | 미션 상태 자동 재조회 (3.4 데이터 갱신 참조) |

### 3.3 디지털 굿즈 수령

- 장 전체 미션 승인 완료 시 해당 장 확장 영역 하단에 [디지털 굿즈 받기] 버튼 표시
- 탭 시 수령 애니메이션 + 컬렉션 추가 + 토스트(Success)
- 수령은 선택적 (나중에 수령 가능)
- 5장 전체 굿즈 수령 후 스페셜 굿즈 합성 CTA 표시 (→ CEB-COL-101)

### 3.4 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 전체 타임라인 + 미션 상태 서버 조회 |
| 서브 화면 복귀 | 변경 가능성 있는 미션 상태 재조회 |
| Pull-to-Refresh | 전체 데이터 재조회 (공통정책 11.2 참조) |
| 딥링크 복귀 (Trivia/PM) | 해당 미션 상태 자동 재조회. Trivia/PM 결과 서버 반영 후 미션 자동 완료 처리 |
| 푸시 알림 탭 진입 | 해당 미션/장으로 스크롤 + 상태 갱신 |

### 3.5 스토리 인트로 표시

- 첫 진입 시 전체 표시 (배경 이미지 + 텍스트)
- 스크롤 또는 탭으로 축소
- 이후 진입 시 1줄 요약만 표시 ("V01D 챌린지 — {N}장 진행중")
- 1회성 처리 (localStorage 기반)

### 3.6 Pull-to-Refresh

- 스크롤 최상단에서 아래로 당김 시 전체 데이터 재조회
- 공통정책 11.2 Pull-to-Refresh 규격 적용

## 4. 규칙/로직

### 4.1 제출 기반 임시 해금 규칙

| 규칙 | 설명 |
| ---- | ---- |
| 임시 해금 조건 | 현재 장의 모든 미션을 **제출** 완료 (승인 불필요) |
| 장 내 미션 수행 | 병렬 (순서 무관, 동시 진행 가능) |
| 보상 지급 시점 | 개별 미션 **승인 시** 즉시 지급 (덕력, 응모권) |
| 디지털 굿즈 | 해당 장 전체 미션 승인 후 수령 가능 |
| 반려 시 영향 | 해당 미션만 재제출 필요. 다른 장 진행에 영향 없음 |
| 5장 최종 보상 | 전 장 전체 미션 승인 완료 시 상시 보상 해금 (서명 포카 래플 자격, 독점 콘텐츠) |

### 4.2 장 상태 전이

| 현재 상태 | 전이 조건 | 다음 상태 |
| ---- | ---- | ---- |
| locked | 이전 장 전체 미션 제출 완료 | provisional |
| provisional | 유저가 해당 장의 미션 CTA를 최초 탭한 시점 | active |
| active | 해당 장 전체 미션 제출(SUBMITTED) 또는 자동완료(AUTO_COMPLETED) | reviewing |
| active | 해당 장 전체 미션이 APPROVED 또는 AUTO_COMPLETED (심사 불필요) | cleared (reviewing 생략) |
| reviewing | 해당 장 전체 미션 승인 완료 | cleared |
| reviewing | 미션 1개 이상 반려 | active (반려 미션 재제출 필요) |

> 1장은 초기 상태가 active (잠금 없음)
> 자동검증 미션(Trivia/PM)만으로 구성된 장은 전체 AUTO_COMPLETED 시 reviewing을 거치지 않고 바로 cleared로 전이
> 임시 해금(locked → provisional) 시 토스트(Success) "다음 장이 열렸습니다!" 표시

### 4.3 미션 상태 정의

| 상태 | 코드 | 설명 | 아이콘 |
| ---- | ---- | ---- | ---- |
| 미완료 | INCOMPLETE | 아직 제출하지 않음 | □ |
| 제출(확인 중) | SUBMITTED | 제출 완료, 심사 대기 | ⏳ |
| 승인 | APPROVED | 심사 승인, 보상 지급 완료 | ✅ |
| 재도전 필요 | REJECTED | 심사 반려, 재제출 필요 | ❌ |
| 자동완료 | AUTO_COMPLETED | 딥링크(Trivia/PM) 자동 검증 완료 | ✅ |

### 4.4 5장 완료 판정 및 축하 모달

##### 완료 조건

- 5개 장 × 모든 미션 전체 APPROVED 또는 AUTO_COMPLETED

##### 축하 모달

| 항목 | 내용 |
| ---- | ---- |
| 트리거 | 마지막 미션 승인/자동완료로 5장 완료 조건 충족 시 자동 표시 |
| 표시 횟수 | **최초 1회만 표시**. [나중에 받기] 탭 시에도 재표시하지 않음. 보상은 축소 배너에서 수령 가능 |
| 유형 | 화면 중앙 모달 (Dimmed 배경) |
| 타이틀 | "🎉 V01D 챌린지 완주!" |
| 내용 | "5장의 스토리를 모두 완료했습니다! 특별한 보상이 준비되어 있어요." |
| CTA 1 | [보상 받기] → 모달 닫기 + 완료 배너의 보상 수령 처리 |
| CTA 2 | [나중에 받기] → 모달 닫기. 다음 진입 시 미표시 (완료 배너에서 수령 가능) |
| 스타일 | 축하 이모지/애니메이션 (피그마 참조) |
| Dimmed 탭 | 닫히지 않음 (CTA 버튼으로만 닫기) |

##### 완료 후 화면 전환

- 모달 닫기 후 → 스토리 타임라인이 축소 배너([보상 받기] + [더보기])로 전환
- 축소 배너 아래에 반복 퀘스트 리스트 표시
- 보상 미수령 시에도 반복 퀘스트 표시 (보상 수령은 필수 아님)

### 4.5 반복 퀘스트 표시 규칙

| 항목 | 규칙 |
| ---- | ---- |
| 표시 조건 | 5장 전체 완료 후 |
| 정렬 | 진행중(Active) > 신규(Ready) > 종료(Ended), 최신순 |
| 카드 정보 | 타이틀 + 기간(MM.DD~MM.DD) + 미션 수 + 보상 요약 |
| 상태 뱃지 | 진행중(초록) / 신규(파란) / 종료(회색) |
| 탭 액션 | 해당 퀘스트 미션 리스트 인라인 확장 |

### 4.6 심사 SLA 안내

- 심사중 미션이 1개 이상 존재할 때 타임라인 하단에 안내 표시
- 문구: "결과는 빠르면 하루 안에 알려드릴게요!"
- 심사 운영 시간 (내부 기준): 평일 09:30~18:30 KST (유저에게 운영시간은 미공개, SLA만 안내)

### 4.7 멀티 아티스트 대응

- 이 화면은 현재 선택된 아티스트 기준으로 표시
- `artist_id` 기반 데이터 조회
- 아티스트 전환 시 해당 아티스트의 Quest 진행 상황으로 전환 (예: V01D 5장 Ch3 진행중 → iKON 전환 시 iKON Quest 진행 상황 표시)

### 4.8 미션 타입별 규칙

| 타입 | 검증 방식 | 재시도 | 특수 규칙 |
| ---- | ---- | ---- | ---- |
| 📸 캡처 제출형 | 수동 심사 (BO) | 반려 시 재제출 (기존 이미지 클리어) | 이미지 1장. JPG/PNG/WebP/HEIC, 10MB. 연관 링크 0~2개 (BO 등록) |
| 🧠 Trivia | 자동검증 (서버) | 조건 미달 시 즉시 재참여 (횟수 무제한) | 조건: 10문제 중 7문제 이상 정답. 실패 시 INCOMPLETE 유지 |
| 🎯 PM | 자동검증 (서버) | 게임 미가용 시 [알림 설정] CTA | 각 미션 = PM 게임 1회 참여. 게임 등록 시 자동 활성화 |
