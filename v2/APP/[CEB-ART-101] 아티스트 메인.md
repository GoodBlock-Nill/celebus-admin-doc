## 1. 화면 개요

유저가 하단 탭에서 아티스트 탭을 탭했을 때 진입하는 메인 화면이다. 아티스트 관련 모든 서비스의 진입점이자, 해당 아티스트의 현재 상태(키우기·응원·미션·기록·소식)를 한눈에 보여주는 **콘텐츠 피드** 역할을 한다.

> v.3 정합: 기존 "3그룹 2열 카드 그리드" → **세로 스크롤 콘텐츠 피드**로 전면 재설계.

##### 주요 기능

- 아티스트 헤더: 배경 이미지 + 아티스트 전환 드롭다운 + 내 덕력·키우기 레벨 + [팔로우]
- 키우기·응원 하이라이트 카드 (진행률)
- 오늘 스케줄 요약
- 미션 리스트 (퀘스트·일일 미션·응원·키우기 진행 상태)
- 내 기록 요약 (덕력 순위·응모권·래플)
- 소식·일정 피드 프리뷰 + 더보기
- 기억 저장소 프리뷰 그리드 + 더보기

##### 접근 경로

- 하단 탭 > {아티스트명} 탭 (예: V01D, iKON)

> 참고: 홈에서 아티스트를 선택하면 홈 화면 내 콘텐츠만 전환되며, 이 화면(CEB-ART-101)으로 이동하지 않는다. 이 화면은 하단 탭을 통해서만 접근한다.

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.2 아티스트 헤더

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 배경 이미지 | 아티스트별 대표 이미지 (상단 배경, 피그마 참조) |
| 아티스트 전환 | 아티스트명 + ▼ 드롭다운 (예: "V01D ▼"). 탭 시 팔로우 아티스트 전환 |
| 내 자산 | "내 덕력 ✦{N}DUK" + "키우기 Lv.{N}" 요약 표시 |
| 팔로우 | [+ 팔로우] / [✓ 팔로우] 토글 버튼 |

#### 동작

- 아티스트 ▼ 드롭다운 탭 → 팔로우 아티스트 목록에서 전환. 전환 시 화면 전체 데이터 갱신 (해당 아티스트 기준)
- [팔로우]/[팔로우 해제] 탭 → 팔로우 상태 토글 (비로그인 시 로그인 유도)
- "내 덕력"·"키우기 Lv" 탭 → 각각 CEB-DUK-101 / CEB-EVT-201로 이동 (피그마 참조)

### 2.3 콘텐츠 피드

> v.3 정합: 2열 카드 그리드 → **세로 스크롤 콘텐츠 피드**. 각 서비스 진입 화면은 §4 연관화면 참조. 각 항목의 상태 표기는 아래 "상태별 표시" 참조.

#### 2.3.1 키우기·응원 하이라이트

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 키우기 카드 + 응원 카드 (가로 캐러셀, 좌우 스와이프) |
| 키우기 카드 | "{아티스트명} 키우기 Lv.{N}" + 진행률 바 + "{달성률}%" → CEB-EVT-201 |
| 응원 카드 | 진행중 서포트(달성 임박 우선) + 달성률/[목표달성] 표시 → CEB-DUK-201. 서포트 없으면 "곧 열려요!" |

#### 2.3.2 오늘 스케줄

| 항목 | 내용 |
| ---- | ---- |
| 구성 | "오늘 일정을 확인해보세요" + 오늘 일정 요약 |
| 탭 액션 | → CEB-INF-101 정보 피드 |

#### 2.3.3 미션

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 미션 리스트(세로 행). 각 행: 아이콘 + 미션명 + 상태 + 진입 |
| 퀘스트 | "{아티스트명} 퀘스트" + 진행 뱃지(예: [진행중] EP.{N}) → CEB-FQ-101 |
| 일일 미션 | "오늘의 할 일" + 완료 상태(예: "{N}건 완료") → CEB-FQ-501 |
| 응원 | "{아티스트명} 응원하기" + 진행 상태 → CEB-DUK-201 |
| 키우기 | "{아티스트명} 키우기" + 레벨 → CEB-EVT-201 |

#### 2.3.4 래플 배너

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 진행중 래플 강조 배너 ("RAFFLE NOW") |
| 탭 액션 | → CEB-EVT-101 래플 리스트 |

#### 2.3.5 내 기록

| 항목 | 내용 |
| ---- | ---- |
| 덕력 | "시즌 {순위}위" + 시즌 D-{N} → CEB-DUK-101 |
| 응모권 | "응모권 {N}장" → CEB-EVT-301 응모권 |
| 래플 | "래플 {N}건" → CEB-EVT-101 |

#### 2.3.6 소식·일정 프리뷰

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 최신 소식·일정 리스트 프리뷰(공식 뱃지 포함) + "더보기" |
| 탭 액션 | 항목 탭 → 해당 소식 상세 / "더보기" → CEB-INF-101 정보 피드 |

#### 2.3.7 기억 저장소 프리뷰

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 최근 기억 그리드 프리뷰 + "더보기" |
| 탭 액션 | 기억 탭 → 기억 상세(CEB-MEM-202) / "더보기" → CEB-MEM-101 기억저장소 |

#### 동작

- 각 섹션의 카드·행·항목 탭 시 해당 서비스 화면으로 이동 (§4 연관화면 참조)
- 화면 전체 세로 스크롤
- Coming Soon 서비스: 항목에 "Coming Soon" 표시, 탭 시 "곧 만나요! 기대해 주세요" 토스트(Info) 노출 후 이동 없음 (런칭 시 BO 설정, 초기 기본값: 전체 오픈)

#### 상태별 표시

| 카드 | 상태 | 표시 |
| ---- | ---- | ---- |
| 퀘스트 | 5개 에피소드 미완료 | "{완료}/5 에피소드" (예: "2/5"). 완료 수 = cleared 상태 에피소드만 카운트 (reviewing/provisional 미포함) |
| 퀘스트 | 5개 에피소드 완료 | "완료 ✅" |
| 일일 미션 | 오늘 미완료 | "도전 중" (출석 또는 미션 중 하나라도 미완료) |
| 일일 미션 | 오늘 완료 | "완료! ✅" (출석 + 일일 미션 모두 완료 시) |
| {아티스트명} 응원하기 | 진행중 이벤트 1건 | "진행중 {달성률}%" |
| {아티스트명} 응원하기 | 진행중 이벤트 다건 | "진행중 {달성률}% 외 {N}건" (마감 임박 1건의 달성률 표시) |
| {아티스트명} 응원하기 | 진행중 이벤트 없음 | "곧 열려요!" |
| 덕력 | 시즌 진행중 | "시즌 {순위}위" |
| 덕력 | 시즌 미참여 | "참여하기" |
| 컬렉션 | 보유 굿즈 있음 | "{N}종" |
| 컬렉션 | 보유 굿즈 없음 | "수집 시작!" |
| Raffle | 진행중 래플 있음 | "래플 {N}건 진행중" |
| Raffle | 진행중 래플 없음 | "새 래플 준비 중!" |
| {아티스트명} 키우기 | 레벨 진행중 | "Lv.{N} ({현재DUK}/{목표DUK}DUK)" |
| {아티스트명} 키우기 | 데이터 없음 | "팬덤 레벨 확인" |
| 정보 | 새 소식 있음 | "새소식 {N}" (뱃지). **미읽음 기준** — 유저가 정보 피드(CEB-INF-101)에 진입하면 읽음 처리되어 카운트 감소 |
| 정보 | 새 소식 없음 | (뱃지 미표시) |
| 기억 저장소 | 기억 있음 | "기억 {N}개" 또는 최근 날짜 |
| 기억 저장소 | 기억 없음 | "첫 기억을 남겨보세요" |

##### 정렬 기준

- 피드 섹션 순서 (위→아래, 고정): 아티스트 헤더 → 키우기·응원 하이라이트 → 오늘 스케줄 → 미션 → 래플 배너 → 내 기록 → 소식·일정 프리뷰 → 기억 저장소 프리뷰
- 미션 리스트 행 순서: 퀘스트 → 일일 미션 → 응원 → 키우기
- 내 기록 항목 순서: 덕력 → 응모권 → 래플
- 소식·일정/기억 프리뷰: 최신순

#### 비로그인

- 이 화면은 비로그인 상태에서도 열람 가능 (카드 목록 표시)
- 개인화 데이터(Quest 진행률, 덕력 순위 등)는 "-" 표시
- 서비스 전체 통계(래플 진행 수 등)는 실제 데이터 표시
- **비로그인 개방 카드**(덕력/서포트/키우기/Raffle/정보) 탭 시 → 해당 화면으로 읽기 전용 이동
- **로그인 필수 카드**(Quest/일일미션/컬렉션/기억저장소) 탭 시 → 로그인 화면으로 이동 (공통정책 10.5 참조)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| Loading | 카드 영역에 Skeleton UI 표시 |
| Default | 각 카드에 상태 정보 표시 |
| Error | 개별 카드의 상태 정보 조회 실패 시 해당 카드만 "-" 표시. 전체 실패 시 에러 안내 + 재시도 버튼 |
| Coming Soon | 미오픈 서비스 카드 → "Coming Soon" 오버레이 표시. 탭 시 "곧 만나요! 기대해 주세요" 토스트(Info) 노출, 화면 이동 없음 (공통정책 11 참조) |
| Error 카드 탭 | 상태 정보가 "-"인 카드를 탭 시 → 해당 서비스 화면으로 정상 이동 (서브 화면에서 데이터 재조회) |

> 에러 상태 중 Loading/Default/Error는 상태별 표시와 중복되는 항목입니다. 상태별 표시 섹션을 우선 참조하세요.

## 3. 공통 규칙

### 3.1 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 각 카드의 상태 정보를 서버에서 조회하여 최신 데이터 표시 |
| Pull-to-Refresh | 전체 카드 상태 정보를 재조회 (공통정책 11.2 참조) |
| 서브 화면 복귀 | 서브 화면(카드 탭으로 이동한 화면)에서 이 화면으로 돌아올 때 모든 카드 상태 정보를 재조회하여 최신 상태 반영 |

- Quest 진행률, 덕력 순위, 래플 진행 수, 팬덤 레벨 등은 위 시점에 갱신

### 3.2 Pull-to-Refresh

- 유저가 화면을 아래로 당기면 전체 카드 상태 정보를 재조회
- 공통정책 11.2 규격 적용

### 3.3 멀티 아티스트 대응

- 이 화면은 현재 선택된 아티스트 기준으로 표시
- 아티스트 전환 시 이 화면의 모든 카드 상태 정보가 해당 아티스트 데이터로 갱신
- `artist_id` 기반으로 모든 데이터 조회

### 3.4 비로그인 공통

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역의 '비로그인' 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

- 비로그인 상태에서도 화면 열람 가능 (카드 목록 표시)
- 개인화 데이터는 "-" 표시, 서비스 전체 통계는 실제 데이터 표시
- 비로그인 개방 카드(덕력/서포트/키우기/Raffle/정보) 탭 시 → 읽기 전용 이동
- 로그인 필수 카드(Quest/일일미션/컬렉션/기억저장소) 탭 시 → 로그인 화면 이동 (공통정책 10.5 참조)
