## 1. 화면 목적

유저가 하단 탭에서 아티스트 탭을 탭했을 때 진입하는 메인 화면이다. 아티스트 관련 모든 서비스(미션/내기록/더보기)의 진입점 역할을 한다.

##### 주요 기능

- 3그룹(미션/내기록/더보기) 카드 그리드로 서비스 한눈에 파악
- 각 카드에 현재 상태 정보 표시 (Quest 진행률, 덕력 순위, 응모권 수, 팬덤 레벨 등)
- 카드 탭 시 해당 서비스 화면으로 이동

##### 접근 경로

- 하단 탭 > {아티스트명} 탭 (예: V01D, iKON)

> 참고: 홈에서 아티스트를 선택하면 홈 화면 내 콘텐츠만 전환되며, 이 화면(CEB-ART-101)으로 이동하지 않는다. 이 화면은 하단 탭을 통해서만 접근한다.

## 3. 화면 내 기능설명

### 3.1 카드 탭

- 각 카드 탭 시 해당 서비스의 메인 화면으로 이동
- 이동 화면 목록은 2.2 카드 구성 표의 "탭 액션" 참조
- 비로그인 유저가 카드 탭 시 → 로그인 화면으로 이동 (공통정책 10.5 참조)
- Coming Soon 서비스 카드 탭 시 → "곧 만나요! 기대해 주세요" 토스트(Info) 노출, 화면 이동 없음

### 3.2 상태 정보 갱신

- **진입 시**: 화면 진입 시 각 카드의 상태 정보를 서버에서 조회하여 최신 데이터 표시
- **Pull-to-Refresh**: 유저가 화면을 아래로 당기면 전체 카드 상태 정보를 재조회 (공통정책 11.2 참조)
- **복귀 시 재조회**: 서브 화면(카드 탭으로 이동한 화면)에서 이 화면으로 돌아올 때 모든 카드 상태 정보를 재조회하여 최신 상태 반영
- Quest 진행률, 덕력 순위, 래플 진행 수, 팬덤 레벨 등은 위 시점에 갱신

### 3.3 그룹 라벨

- "미션", "내기록", "더보기" 라벨은 정보 구분용으로만 사용
- 라벨 자체에 탭 이벤트 없음

### 3.4 스크롤

- 화면 전체가 세로 스크롤 가능 (카드가 많을 경우)
- 총 9개 카드(미션 3 + 내기록 4 + 더보기 2)이므로 디바이스에 따라 스크롤 필요

### 3.5 온보딩 안내 오버레이

- 신규 유저(활동 데이터가 전혀 없는 첫 진입 시) 대상으로 온보딩 안내 오버레이 표시
- 오버레이는 각 카드의 기능을 간략히 설명하는 가이드 UI (피그마 참조)
- 유저가 오버레이를 탭하거나 "닫기" 버튼을 탭하면 오버레이 해제
- 오버레이 해제 후 재진입 시에는 표시하지 않음 (1회성)

## 4. 규칙/로직

### 4.1 카드별 상태 정보 표시 규칙

| 카드 | 상태 | 표시 |
| ---- | ---- | ---- |
| 챌린지 | 5장 미완료 | "{완료장수}/5장" (예: "2/5장"). 완료장수 = cleared 상태 장만 카운트 (reviewing/provisional 미포함) |
| 챌린지 | 5장 완료 | "완료 ✅" |
| 일일 미션 | 오늘 미완료 | "도전 중" (출석 또는 미션 중 하나라도 미완료) |
| 일일 미션 | 오늘 완료 | "완료! ✅" (출석 + 일일 미션 모두 완료 시) |
| {아티스트명} 응원하기 | 진행중 이벤트 1건 | "진행중 {달성률}%" |
| {아티스트명} 응원하기 | 진행중 이벤트 다건 | "진행중 {달성률}% 외 {N}건" (마감 임박 1건의 달성률 표시) |
| {아티스트명} 응원하기 | 진행중 이벤트 없음 | "곧 열려요!" |
| 덕력 | 시즌 진행중 | "시즌 {순위}위" |
| 덕력 | 시즌 미참여 | "참여하기" |
| 컬렉션 | 보유 굿즈 있음 | "{N}종" |
| 컬렉션 | 보유 굿즈 없음 | "수집 시작!" |
| Raffle | 진행중 래플 있음 | "래플 {N}건 진행중" |
| Raffle | 진행중 래플 없음 | "새 래플 준비 중!" |
| {아티스트명} 키우기 | 레벨 진행중 | "Lv.{N} ({현재pt}/{목표pt}pt)" |
| {아티스트명} 키우기 | 데이터 없음 | "팬덤 레벨 확인" |
| 정보 | 새 소식 있음 | "새소식 {N}" (뱃지) |
| 정보 | 새 소식 없음 | (뱃지 미표시) |
| 기억 저장소 | 기억 있음 | "기억 {N}개" 또는 최근 날짜 |
| 기억 저장소 | 기억 없음 | "첫 기억을 남겨보세요" |

### 4.2 정렬 기준

- 그룹 순서: 미션 → 내기록 → 더보기 (고정)
- 그룹 내 카드 순서: 위 표 순서 그대로 (고정)
- 미션 그룹 카드 배치: Row 1(챌린지 | 일일미션), Row 2(서포트 — 1열 풀)
- 내기록 그룹 카드 배치: Row 1(덕력 | 컬렉션), Row 2(응모 | 팬덤 레벨)
- 더보기 그룹 카드 배치: Row 1(정보 | 기억저장소)

### 4.3 멀티 아티스트 대응

- 이 화면은 현재 선택된 아티스트 기준으로 표시
- 아티스트 전환 시 이 화면의 모든 카드 상태 정보가 해당 아티스트 데이터로 갱신
- `artist_id` 기반으로 모든 데이터 조회

## 5. 에러 및 예외상황

| 상태 | 처리 |
| ---- | ---- |
| Loading | 카드 영역에 Skeleton UI 표시 |
| Default | 각 카드에 상태 정보 표시 |
| Error | 개별 카드의 상태 정보 조회 실패 시 해당 카드만 "-" 표시. 전체 실패 시 에러 안내 + 재시도 버튼 |
| 비로그인 | 이 화면은 비로그인 상태에서도 열람 가능 (카드 목록 표시). 개인화 데이터(Quest 진행률, 덕력 순위 등)는 "-" 표시. 서비스 전체 통계(래플 진행 수 등)는 실제 데이터 표시. 카드 탭 시 → 로그인 화면으로 이동 (공통정책 10.5, 11.1 참조) |
| Coming Soon | 미오픈 서비스 카드 → "Coming Soon" 오버레이 표시. 탭 시 "곧 만나요! 기대해 주세요" 토스트(Info) 노출, 화면 이동 없음 (공통정책 11 참조) |
| Error 카드 탭 | 상태 정보가 "-"인 카드를 탭 시 → 해당 서비스 화면으로 정상 이동 (서브 화면에서 데이터 재조회) |
| 신규 유저 | 활동 데이터가 없는 첫 진입 시 온보딩 안내 오버레이 표시 (3.5 참조) |
