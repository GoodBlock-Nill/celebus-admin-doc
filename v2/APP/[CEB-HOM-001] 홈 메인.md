## 1. 화면 개요

앱을 열면 가장 먼저 보는 화면이다. "오늘 뭘 하면 되지?"를 즉시 알려주는 허브 역할을 한다.

> 핵심 원칙: "다음에 뭘 해야 하지?"를 유저가 절대 고민하지 않아야 한다.

##### 주요 기능

- 상단 [HOME] / [EVENTS] 2개 탭. HOME 탭은 본 화면(아래 영역), EVENTS 탭은 이벤트 전체 리스트(CEB-HOM-101)
- 캐러셀 배너로 진행 중인 이벤트를 한눈에 파악
- 아티스트 선택으로 콘텐츠 전환
- 오늘의 할 일(출석 + 일일 미션) — 개별 미션 인라인 노출
- 상태 기반 핵심 추천 1개
- 바로가기 그리드로 주요 서비스 빠른 진입

##### 접근 경로

- 하단 탭 > 홈 (기본 탭)

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 기본/완료/비로그인/Empty 등 다양한 상태를 전환하며 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

## 2. 영역별 상세

### 2.1 헤더 + 상단 탭

#### 헤더 UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | CELEBUS 로고 |
| 우측 1 | "응모 {N}" 응모권 잔고 pill. 탭 시 CEB-FQ-210 이동 |
| 우측 2 | 🔔 알림 아이콘. 미확인 N건 뱃지. 탭 시 CEB-NTF-510 알림으로 이동 |
| 스타일 | 피그마 참조 |

> 헤더 우측 배치 순서는 응모권 pill → 알림 벨 순 (피그마 참조).

#### 상단 탭 (HOME / EVENTS)

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 헤더 아래 |
| 구성 | [HOME] / [EVENTS] 2개 탭. 기본 선택: HOME |
| HOME 탭 | 본 화면(캐러셀·아티스트 선택·오늘의 할 일·추천·바로가기·BIVE 프리뷰) |
| EVENTS 탭 | 이벤트 전체 리스트 (CEB-HOM-101 영역). 탭 전환 시 동일 화면 내 콘텐츠 전환 (별도 페이지 이동 아님) |

- [EVENTS] 탭 탭 → 동일 화면 내 이벤트 전체 리스트로 전환 (CEB-HOM-101 §2 참조). 헤더는 유지
- [HOME] 탭 탭 → 홈 콘텐츠로 복귀

#### 비로그인

- 알림 뱃지 미표시 (미확인 건수 없음)
- 응모권 잔고 "-" 표시

### 2.2 캐러셀 배너

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 헤더 아래 |
| 콘텐츠 | 래플 / 서포트 아티스트 무관 통합 (퀘스트 미포함) |
| 필터 | [진행중] / [집계중] / [마감됨] 3개 탭. 기본: 진행중 |
| 자동 슬라이드 | 3~5초 간격. 수동 스와이프 가능 |
| 인디케이터 | 하단 도트 (● ○ ○) |
| 배너 탭 | 딥링크 URL로 이동 (이벤트 상세) |
| 배너 수 | BO 자유 등록 (권장 3~5개) |
| BO 관리 항목 | 배너 이미지 + 타이틀 + 딥링크 URL + 상태(진행중/집계중/마감됨) + 노출 기간 + 순서 |
| 전체 보기 | 캐러셀 우측 상단 "전체 보기 >" 텍스트 버튼. 탭 시 상단 EVENTS 탭(이벤트 전체, CEB-HOM-101)으로 전환 |
| 스타일 | 피그마 참조 |

#### 동작

- [전체 보기 >] 탭 → 상단 EVENTS 탭(CEB-HOM-101 이벤트 전체)으로 전환
- 배너 탭 → 딥링크 URL로 이동 (이벤트 상세)
- 다른 아티스트 이벤트 탭 시: 아티스트 선택 유지 (자동 전환 안 함). 이벤트 상세만 이동. 뒤로가기 → 홈 복귀, 기존 아티스트 유지 (공통정책 10.4 참조)
- 미팔로우 아티스트 이벤트 탭 시: 이벤트 상세 이동 + 하단 인라인 배너 '이 아티스트를 팔로우하시겠습니까?' 표시. [팔로우하기] / [닫기] CTA. 팔로우 완료 시 토스트(Success). 배너 닫기 시 이벤트 상세 유지

#### 상태별 표시

| 규칙 | 설명 |
| ---- | ---- |
| 필터 기본 | 진행중 |
| 진행중 정렬 | BO 설정 순서 |
| 집계중 정렬 | 기간 종료일 최신순 (최근 종료 먼저) |
| 집계중 상태 | 기간 종료 + 추첨 미완료인 추첨형 이벤트(래플 전용). 비추첨형은 종료 시 바로 마감됨으로 전이 |
| 마감됨 정렬 | 마감일 최신순 |
| 이벤트 0건 | 캐러셀 영역 축소 + "이벤트 준비 중" 플레이스홀더 |

#### 비로그인

- 배너 탭 → 이벤트 상세 열람 가능 (응모 CTA 탭 시 로그인)
- 전체 보기 탭 → 이벤트 전체 열람 가능 (개별 이벤트 응모 CTA 탭 시 로그인)

> 에러 상태는 상태별 표시 참조. 공통 에러(Loading/Error)는 섹션 3.5 참조.

### 2.4 오늘의 할 일 (카드형)

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 아티스트 | 선택된 아티스트 기준 |
| 구성 | 섹션 헤더("오늘의 할 일" + "전체 보기 >") + 출석 체크인 카드 + 일일 미션 개별 리스트 |
| 스타일 | 피그마 참조 |

> 섹션 우측 상단 "전체 보기 >" 탭 시 CEB-FQ-501 오늘의 할 일(일일 미션) 화면으로 이동.

##### 상단 — 출석 체크인 카드

| 상태 | 표시 |
| ---- | ---- |
| 미완료 | 보라 그라데이션 배경. 👋 아이콘 + "출석 체크하기" + "탭하여 출석 · 덕력 +5DUK" + [체크인] pill 버튼 |
| 완료 | 초록 그라데이션 배경. ✅ 아이콘 + "출석 완료!" + "🔥 {N}일 연속 · +5DUK 획득" |

##### 하단 — 일일 미션 (개별 리스트)

| 항목 | 내용 |
| ---- | ---- |
| 진행 헤더 | "일일 미션 {완료}/{전체} 완료" (예: "일일 미션 0/2 완료") + "매일 미션 모두 완료 시 +25DUK 지급" 안내 |
| 미션 행 | 오늘 할당된 일일 미션을 **개별 행으로 나열**. 각 행: 미션 아이콘 + 미션명(예: "게임존 방문", "기억저장소 방문") + 우측 [미션하기] 버튼 |
| 미션 수 | 현 **하루 2건** (CEB-FQ-501 §2.3 미션 풀에서 선택. 향후 BO 확장) |
| 행 완료 | 완료 미션 행은 [미션완료 ✓] 비활성 표시 |
| 전체 완료 보상 | 모든 미션 완료 시 +25DUK **일괄 지급** ([CEB-FQ-501] §2.3 참조) |
| 탭 액션 | 미션 행 [미션하기] 탭 → 해당 서비스 화면(딥링크) 이동. 섹션 "전체 보기 >" 탭 → CEB-FQ-501 이동 |

> 퀘스트 진행률은 하단 핵심 추천 카드(2.5)에서 표시하므로 이 섹션에서 중복 제거
> 개별 미션 완료 시 토스트 없음 (행 체크만 갱신). 전체 완료 시점에만 일괄 지급 토스트 ([CEB-FQ-501] §2.3 참조)

##### 전체 완료 상태

| 항목 | 내용 |
| ---- | ---- |
| 표시 | 초록 그라데이션 단일 카드. 🔥 아이콘 + "{N}일 연속 달성!" + "오늘 할 일을 모두 완료했어요" |
| 탭 동작 | → CEB-FQ-501 일일 미션 화면으로 이동 (상세 확인) |

#### 동작

- 출석 [체크인] 탭 → 즉시 출석 완료 + 토스트(Success) "출석 완료! 덕력 5DUK 획득" + 스트릭 +1 + 카드 초록 전환
- 일일 미션 행 [미션하기] 탭 → 해당 서비스 화면(딥링크) 이동. 복귀 시 미션 자동 완료 처리
- 섹션 "전체 보기 >" 또는 전체 완료 카드 탭 → CEB-FQ-501 이동 (상세 확인)
- CEB-FQ-501에서도 출석 처리 가능 (이중 진입점. 공통정책 6.1 참조)

#### 비로그인

- 비로그인 시 "로그인하고 시작하기 — 출석, 미션, 응모권이 기다리고 있어요!" CTA 표시
- 비로그인 출석 CTA 탭 → 로그인 완료 후 홈으로 자동 복귀. 출석은 유저가 다시 탭 필요 (자동 처리 아님)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 오늘의 할 일 모두 완료 | 초록 그라데이션 단일 카드로 접힘 (위 "전체 완료 상태" 참조) |

### 2.5 핵심 추천 (1개)

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 아티스트 | 선택된 아티스트 기준 |
| 표시 수 | 1개 (6단계 우선순위 중 최상위만) |
| 형태 | 풀 너비 카드 (아이콘 + 텍스트 + CTA 버튼) |
| 스타일 | 피그마 참조 |

##### 추천 우선순위 (6단계)

| 순위 | 유저 상태 | 추천 카드 | CTA |
| ---- | ---- | ---- | ---- |
| 1 | Quest 5개 에피소드 미완료 (cleared 아닌 에피소드 1개 이상) | "V01D 퀘스트 {N} 에피소드 진행중" | [이어하기] → CEB-FQ-101 |
| 2 | 1년 전 오늘 기억 존재 | "💜 1년 전 오늘의 기억" | [다시 보기] → CEB-MEM-101 |
| 3 | 디지털 굿즈 미수령 | "새 디지털 굿즈를 받을 수 있어요!" | [받으러 가기] → CEB-COL-101 |
| 4 | 덕력 시즌 진행중 | "이번 시즌 {순위}위 ({포인트}DUK)" | [랭킹 보기] → CEB-DUK-101 |
| 5 | 게임존 신규 게임 | "새 PM 오픈!" | [참여하기] → CEB-GAM-101 |
| 6 | 기본 상태 | "오늘의 퀘스트를 완료하세요" | [퀘스트 보기] → CEB-FQ-101 |

> 해당 상태가 없으면 스킵. 남은 것 중 최상위 1개만 표시. 6번(기본)은 항상 존재하므로 추천 카드는 반드시 표시된다.
> 래플 진행중 + 응모권 보유 → 바로가기 Raffle 카드에서 제공하므로 추천에서 제외. 팬덤 레벨 80%+ → 바로가기 키우기 카드에서 제공하므로 추천에서 제외.

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 추천 조회 실패 | 추천 카드 영역에 6번(기본) 표시: "오늘의 퀘스트를 완료하세요" |

| 카드 | 아이콘 | 타이틀 | 상태 정보 | 탭 액션 |

| 키우기 | 🏆 | {아티스트명} 키우기 | "Lv.{N}" + 프로그레스 바 + "{현재}/{목표}DUK · {달성률}%" | → CEB-EVT-201 |
| 응원 | 💜 | 달성 임박 서포트 이벤트명 | "달성 임박!" 뱃지 + 프로그레스 바 + "{현재}/{목표}DUK · {달성률}%" | → CEB-DUK-201 |

> 키우기 카드: 따뜻한 그라데이션(amber). 현재 레벨 + 다음 레벨까지 프로그레스를 직관적으로 표시.
> 응원 카드: 바이올렛~핑크 그라데이션. 달성 임박(80%+) 서포트를 우선 표시하여 참여 유도. 서포트 없으면 "서포트 이벤트 준비 중! 덕력을 모아두세요 💜" 표시.

### 2.7 BIVE 컬렉션 프리뷰

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 바로가기 섹션 아래 |
| 의도 | "활동하면 이런 디지털 굿즈를 얻을 수 있다" — 유저의 수집 욕구와 소유욕을 자극하여 서비스 참여 유도 |
| 로그인/비로그인 | **동일하게 표시** (비로그인 유저에게도 BIVE 매력을 보여줘 가입 전환 유도) |
| 아티스트 | 선택된 아티스트 기준 |
| 섹션 타이틀 | "{아티스트명} 바이브" (예: "V01D 바이브") |
| 구성 | 2열 그리드. BIVE 카드 4~6개 표시 |
| 카드 구성 | BIVE 이미지 + 등급 뱃지(Grade 1~5 / 스페셜) + BIVE 이름 |
| 보유 상태 (로그인) | 보유 BIVE: "보유 중" 뱃지 (성취감). 미보유 BIVE: 뱃지 없음 (소유욕 자극) |
| 보유 상태 (비로그인) | 전체 뱃지 없음 (보유 여부 판별 불가) |
| 카드 탭 | BIVE 상세 미리보기 바텀시트 표시 (아래 동작 참조) |
| 하단 CTA | 섹션 우측 상단 "도감 보기 >" 탭 → CEB-COL-101 컬렉션 이동 (비로그인 시 로그인 유도) |
| Empty (BIVE 0건) | "곧 첫 BIVE가 등록될 거예요!" 안내. 섹션은 표시하되 카드 없음 |
| 스타일 | 피그마 참조 |

> BIVE 이미지는 실제 이미지를 보여줘야 소유욕이 발생한다. 실루엣/잠금 처리 아님. 미보유 BIVE도 풀 이미지 노출.
> BO에서 홈 노출 BIVE를 선택 가능 (추천 BIVE 플래그). 미설정 시 등급순으로 자동 선택.

#### 동작

- BIVE 카드 탭 → 바텀시트 미리보기 표시:
  - BIVE 이미지 (대형)
  - 등급 뱃지 (Grade 1~5 / 스페셜)
  - BIVE 이름
  - 획득 방법 안내 (예: "Quest 1장 완료 시 획득", "합성으로 획득")
  - 보유 상태 (로그인 시): "보유 중" 또는 미보유
  - CTA: [컬렉션에서 보기] → CEB-COL-101 이동 (비로그인 시 로그인 유도)
- 바텀시트 Dimmed 영역 탭 시 닫기
- 비로그인 유저도 바텀시트 열람 가능 (CTA 탭 시만 로그인 유도)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| BIVE 0건 | "곧 첫 BIVE가 등록될 거예요!" 안내. 섹션 표시, 카드 없음 |

### 3.2 Pull-to-Refresh

- 공통정책 11.2 규격 적용

### 3.3 멀티 아티스트 대응

- 아티스트 선택 시 홈 아래 콘텐츠 전환 (캐러셀 제외)
- `artist_id` 기반으로 오늘의 할 일 / 추천 카드 / 바로가기 / BIVE 프리뷰 데이터 조회

### 3.4 비로그인 공통

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역(2.1~2.7)의 "비로그인" 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

- 비로그인 홈 열람: 전체 열람 가능 (캐러셀, 아티스트 선택, 오늘의 할 일, 추천, 바로가기 모두 노출)
- 비로그인 CTA 탭 → 로그인 완료 후 이전 화면으로 자동 복귀 (공통정책 10.5). CTA 동작은 유저가 다시 탭 필요 (자동 실행 아님).

### 3.5 공통 에러 상태

| 상태 | 처리 |
| ---- | ---- |
| Loading | 각 영역 Skeleton UI |
| Default | 전체 영역 정상 표시 |
| Error | "앗, 불러오지 못했어요" + [다시 불러오기] 버튼 |

### 3.6 하단 글로벌 내비게이션

모든 주요 화면에 공통 노출되는 하단 탭 바.
