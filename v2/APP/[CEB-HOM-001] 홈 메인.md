## 1. 화면 개요

앱을 열면 가장 먼저 보는 화면이다. "오늘 뭘 하면 되지?"를 즉시 알려주는 허브 역할을 한다.

> 핵심 원칙: "다음에 뭘 해야 하지?"를 유저가 절대 고민하지 않아야 한다.

##### 주요 기능

- 캐러셀 배너로 진행 중인 이벤트를 한눈에 파악
- 아티스트 선택으로 콘텐츠 전환
- 오늘의 할 일(출석+미션+퀘스트) 컴팩트 카드
- 상태 기반 핵심 추천 1개
- 바로가기 그리드로 주요 서비스 빠른 진입

##### 접근 경로

- 하단 탭 > 홈 (기본 탭)

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

> 오늘의 할 일 모두 완료 시 레이아웃도 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.2 캐러셀 배너

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 헤더 아래 |
| 콘텐츠 | 래플/서포트/이벤트 등 아티스트 무관 통합 |
| 필터 | [진행중] / [마감됨] 2개 탭. 기본: 진행중 |
| 자동 슬라이드 | 3~5초 간격. 수동 스와이프 가능 |
| 인디케이터 | 하단 도트 (● ○ ○) |
| 배너 탭 | 딥링크 URL로 이동 (이벤트 상세) |
| 배너 수 | BO 자유 등록 (권장 3~5개) |
| BO 관리 항목 | 배너 이미지 + 타이틀 + 딥링크 URL + 상태(진행중/마감됨) + 노출 기간 + 순서 |
| 전체 보기 | 캐러셀 우측 상단 "전체 보기 >" 텍스트 버튼. 탭 시 CEB-HOM-101 이동 |
| 스타일 | 피그마 참조 |

#### 동작

- [전체 보기 >] 탭 → CEB-HOM-101 이벤트 전체 화면 이동
- 배너 탭 → 딥링크 URL로 이동 (이벤트 상세)
- 다른 아티스트 이벤트 탭 시: 아티스트 선택 유지 (자동 전환 안 함). 이벤트 상세만 이동. 뒤로가기 → 홈 복귀, 기존 아티스트 유지 (공통정책 10.4 참조)
- 미팔로우 아티스트 이벤트 탭 시: 이벤트 상세 이동 + 하단 인라인 배너 '이 아티스트를 팔로우하시겠습니까?' 표시. [팔로우하기] / [닫기] CTA. 팔로우 완료 시 토스트(Success). 배너 닫기 시 이벤트 상세 유지

#### 상태별 표시

| 규칙 | 설명 |
| ---- | ---- |
| 필터 기본 | 진행중 |
| 진행중 정렬 | BO 설정 순서 |
| 마감됨 정렬 | 마감일 최신순 |
| 이벤트 0건 | 캐러셀 영역 축소 + "이벤트 준비 중" 플레이스홀더 |

#### 비로그인

- 배너 탭 → 이벤트 상세 열람 가능 (응모 CTA 탭 시 로그인)
- 전체 보기 탭 → 이벤트 전체 열람 가능 (개별 이벤트 응모 CTA 탭 시 로그인)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 캐러셀 0건 | 캐러셀 영역 축소 + "이벤트 준비 중" 플레이스홀더 |

### 2.3 아티스트 선택

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 캐러셀 아래 |
| 구성 | 팔로우한 아티스트 원형 로고 가로 나열 + [+] 추가 버튼 |
| 정렬 | 팔로우 순서 (최근 팔로우가 우측) |
| 선택 상태 | 선택 아티스트 하이라이트 (보더/배경). 스타일 피그마 참조 |
| 탭 동작 | 아티스트 로고 탭 → 하단 탭명 변경 + 홈 아래 콘텐츠 전환 |
| [+] 추가 | 아티스트 탐색/추가 화면(CEB-ART-201) 이동 |
| 초기 상태 | 가장 최근 활동한 아티스트 기본 선택 |
| 1명만 팔로우 | 로고 1개 + [+] 표시. 전환 불필요 |

##### 아티스트 선택 시 변경 항목

| 항목 | 변경 여부 |
| ---- | ---- |
| 캐러셀 배너 | 변경 없음 (아티스트 무관 통합) |
| 하단 탭명 | 변경됨 (2번째 탭이 선택 아티스트명으로) |
| 오늘의 할 일 | 변경됨 (선택 아티스트 기준) |
| 핵심 추천 | 변경됨 (선택 아티스트 기준) |
| 바로가기 | 변경됨 (선택 아티스트 기준) |

#### 동작

- 아티스트 로고 탭 → 하단 탭명 변경 + 홈 아래 콘텐츠(오늘의 할 일/추천/바로가기) 전환
- 전환 시 서버 재조회

#### 비로그인

- 아티스트 로고 탭 → 로그인 화면 이동 (섹션은 노출, 탭 시 로그인)
- [+] 추가 탭 → CEB-ART-201 이동 (아티스트 목록 열람 가능, [팔로우] 탭 시 로그인)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 아티스트 0명 | 발생하지 않음 (최소 1명 팔로우 필수, 온보딩에서 선택) |

### 2.4 오늘의 할 일 (컴팩트 카드)

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 아티스트 | 선택된 아티스트 기준 |
| 구성 | 2줄 컴팩트 카드 |
| 1줄 | "🔥 {N}일째 연속" + 출석 상태(☐/☑) + 미션 상태(☐/☑) |
| 2줄 | "☐ 이번 주 퀘스트: {진행률}" + [이어서→] CTA |
| 출석 탭 | ☐ 아이콘 탭 → 즉시 출석 완료 + 토스트(Success) "출석 완료! 덕력 10pt 획득" |
| 미션 탭 | ☐ 미션 아이콘 또는 "미션" 텍스트 영역 탭 → CEB-FQ-501 일일 미션 화면으로 이동 |
| 퀘스트 CTA | [이어서→] → CEB-FQ-101 이동 |
| 모두 완료 시 | "✅ 오늘 할 일 완료! 🔥{N}일째" 1줄로 접힘 |
| 스타일 | 피그마 참조 |

#### 동작

- 오늘의 할 일 카드에서 출석 ☐ 아이콘 탭 → 즉시 완료
- 토스트(Success) "출석 완료! 덕력 10pt 획득"
- 스트릭 +1 반영
- CEB-FQ-501에서도 동일하게 처리 가능 (이중 진입점. 공통정책 6.1 참조)

#### 비로그인

- 비로그인 시 "로그인하고 시작하기 — 출석, 미션, 응모권이 기다리고 있어요!" CTA 표시
- 비로그인 출석 CTA 탭 → 로그인 완료 후 홈으로 자동 복귀. 출석은 유저가 다시 탭 필요 (자동 처리 아님)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 오늘의 할 일 모두 완료 | "✅ 오늘 할 일 완료! 🔥{N}일째" 1줄로 접힘 |

### 2.5 핵심 추천 (1개)

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 아티스트 | 선택된 아티스트 기준 |
| 표시 수 | 1개 (8단계 우선순위 중 최상위만) |
| 형태 | 풀 너비 카드 (아이콘 + 텍스트 + CTA 버튼) |
| 스타일 | 피그마 참조 |

##### 추천 우선순위 (8단계)

| 순위 | 유저 상태 | 추천 카드 | CTA |
| ---- | ---- | ---- | ---- |
| 1 | Quest 5장 미완료 (cleared 아닌 장이 1개 이상) | "V01D 챌린지 {N}장 진행중" | [이어하기] → CEB-FQ-101 |
| 2 | 1년 전 오늘 기억 존재 | "💜 1년 전 오늘의 기억" | [다시 보기] → CEB-MEM-101 |
| 3 | 디지털 굿즈 미수령 | "새 디지털 굿즈를 받을 수 있어요!" | [받으러 가기] → CEB-COL-101 |
| 4 | 래플 진행중 + 응모권 보유 | "응모권 {N}장 — 래플 참여하기" | [응모하기] → CEB-EVT-101 |
| 5 | 덕력 시즌 진행중 | "이번 시즌 {순위}위 ({포인트}pt)" | [랭킹 보기] → CEB-DUK-101 |
| 6 | 팬덤 레벨 80%+ | "Lv.{N}까지 {남은}pt!" | [기여하기] → CEB-EVT-201 |
| 7 | 게임존 신규 게임 | "새 PM 오픈!" | [참여하기] → CEB-GAM-101 |
| 8 | 기본 상태 | "오늘의 퀘스트를 완료하세요" | [퀘스트 보기] → CEB-FQ-101 |

> 해당 상태가 없으면 스킵. 남은 것 중 최상위 1개만 표시. 8번(기본)은 항상 존재하므로 추천 카드는 반드시 표시된다.

#### 상태별 표시

| 규칙 | 설명 |
| ---- | ---- |
| 우선순위 | 8단계 (추천 우선순위 표 참조) |
| 표시 수 | 1개 (최상위만) |
| 해당 상태 없음 | 스킵 → 다음 순위 확인 |
| 전부 없음 | 8번(기본) 표시 (항상 해당됨) |

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 추천 조회 실패 | 추천 카드 영역에 8번(기본) 표시: "오늘의 퀘스트를 완료하세요" |

### 2.6 바로가기

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 아티스트 | 선택된 아티스트 기준 |
| 구성 | 1열(풀 너비) 카드 4개 세로 나열. 각 카드에 아이콘 + 타이틀 + 상태 정보 + 프로그레스(해당 시) + CTA |
| 탭 동작 | 해당 서브 화면 직접 이동 |

##### 리치 카드 (프로그레스 포함)

| 카드 | 아이콘 | 타이틀 | 상태 정보 | 탭 액션 |
| ---- | ---- | ---- | ---- | ---- |
| 키우기 | 🏆 | {아티스트명} 키우기 | "Lv.{N}" + 프로그레스 바 + "{현재}/{목표}pt · {달성률}%" | → CEB-EVT-201 |
| 응원 | 💜 | 달성 임박 서포트 이벤트명 | "달성 임박!" 뱃지 + 프로그레스 바 + "{현재}/{목표}pt · {달성률}%" | → CEB-DUK-201 |

> 키우기 카드: 따뜻한 그라데이션(amber). 현재 레벨 + 다음 레벨까지 프로그레스를 직관적으로 표시.
> 응원 카드: 바이올렛~핑크 그라데이션. 달성 임박(80%+) 서포트를 우선 표시하여 참여 유도. 서포트 없으면 "서포트 이벤트 준비 중! 덕력을 모아두세요 💜" 표시.

##### 콘텐츠 카드 (오늘의 일정/소식 + Raffle)

| 카드 | 아이콘 | 구성 | 탭 액션 |
| ---- | ---- | ---- | ---- |
| 오늘의 일정/소식 | 📅 | "오늘" 뱃지 + 오늘 첫 번째 일정 (시간+일정명) + 최신 소식 1줄 + "+N건 더보기 →". 오늘 일정 없으면 최신 소식 2줄 표시. 블루 그라데이션 배경. 일정·소식 모두 없을 때: '새 소식이 올 때 알려드릴게요' 표시 | → CEB-INF-101 |
| Raffle | 🎁 | "D-{N}" 뱃지(마감 임박 래플) + 래플명 + 참여자 수 + "응모하기 →". 래플 없으면 "새 래플 준비 중!" 표시. 핑크 그라데이션 배경 | → CEB-EVT-101 |

> 기존 "정보" 라벨 → 팬에게 의미 있는 실제 콘텐츠(오늘의 일정/소식) 미리보기로 변경. 유저가 "어, 오늘 음악중심 나오네?" 하고 관심을 가지도록 유도.

#### 비로그인

- 출석/미션/래플 등 CTA 탭 → 로그인 화면 이동

### 2.7 BIVE 컬렉션 프리뷰

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 위치 | 바로가기 섹션 아래 |
| 의도 | "활동하면 이런 디지털 굿즈를 얻을 수 있다" — 유저의 수집 욕구와 소유욕을 자극하여 서비스 참여 유도 |
| 로그인/비로그인 | **동일하게 표시** (비로그인 유저에게도 BIVE 매력을 보여줘 가입 전환 유도) |
| 아티스트 | 선택된 아티스트 기준 |
| 섹션 타이틀 | "🃏 이런 BIVE를 모을 수 있어요" |
| 구성 | 2열 그리드. BIVE 카드 4~6개 표시 |
| 카드 구성 | BIVE 이미지 + 등급 뱃지(Grade 1~5 / 스페셜) + BIVE 이름 |
| 보유 상태 (로그인) | 보유 BIVE: "보유 중" 뱃지 (성취감). 미보유 BIVE: 뱃지 없음 (소유욕 자극) |
| 보유 상태 (비로그인) | 전체 뱃지 없음 (보유 여부 판별 불가) |
| 카드 탭 | BIVE 상세 미리보기 바텀시트 표시 (아래 동작 참조) |
| 하단 CTA | "전체 컬렉션 보기 →" 탭 → CEB-COL-101 이동 (비로그인 시 로그인 유도) |
| Empty (BIVE 0건) | "곧 첫 BIVE가 등록될 거예요!" 안내. 섹션은 표시하되 카드 없음 |
| 스타일 | 피그마 참조 |

> BIVE 이미지는 실제 이미지를 보여줘야 소유욕이 발생한다. 실루엣/잠금 처리 아님. 미보유 BIVE도 풀 이미지 노출.
> BO에서 홈 노출 BIVE를 선택 가능 (추천 BIVE 플래그). 미설정 시 등급순으로 자동 선택.

#### 동작

- BIVE 카드 탭 → 바텀시트 미리보기 표시:
  - BIVE 이미지 (대형)
  - 등급 뱃지 (Grade 1~5 / 스페셜)
  - BIVE 이름
  - 획득 방법 안내 (예: "Quest 1장 완료 시 획득", "합성으로 획득")
  - 보유 상태 (로그인 시): "보유 중" 또는 미보유
  - CTA: [컬렉션에서 보기] → CEB-COL-101 이동 (비로그인 시 로그인 유도)
- 바텀시트 Dimmed 영역 탭 시 닫기
- 비로그인 유저도 바텀시트 열람 가능 (CTA 탭 시만 로그인 유도)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| BIVE 0건 | "곧 첫 BIVE가 등록될 거예요!" 안내. 섹션 표시, 카드 없음 |

## 3. 공통 규칙

### 3.1 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 전체 영역 데이터 조회 |

| Pull-to-Refresh | 전체 재조회 (공통정책 11.2 참조) |
| 서브 화면 복귀 | 변경 가능성 있는 데이터 재조회 |

### 3.2 Pull-to-Refresh

- 공통정책 11.2 규격 적용

### 3.3 멀티 아티스트 대응

- 아티스트 선택 시 홈 아래 콘텐츠 전환 (캐러셀 제외)
- `artist_id` 기반으로 오늘의 할 일 / 추천 카드 / 바로가기 / BIVE 프리뷰 데이터 조회

### 3.4 비로그인 공통

- 비로그인 홈 열람: 전체 열람 가능 (캐러셀, 아티스트 선택, 오늘의 할 일, 추천, 바로가기 모두 노출)
- 비로그인 CTA 탭 → 로그인 완료 후 이전 화면으로 자동 복귀 (공통정책 10.5). CTA 동작은 유저가 다시 탭 필요 (자동 실행 아님).

### 3.5 공통 에러 상태

| 상태 | 처리 |
| ---- | ---- |
| Loading | 각 영역 Skeleton UI |
| Default | 전체 영역 정상 표시 |
| Error | "앗, 불러오지 못했어요" + [다시 불러오기] 버튼 |
| 비로그인 | 전체 열람 가능. CTA 탭 시 로그인 이동 (공통정책 10.5 참조). 오늘의 할 일 카드에 "로그인하고 시작하기 — 출석, 미션, 응모권이 기다리고 있어요!" CTA 표시 |
