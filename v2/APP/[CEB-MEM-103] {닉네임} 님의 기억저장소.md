## 1. 화면 개요

[CEB-MEM-102] 모두의 기억저장소에서 특정 유저의 닉네임을 탭했을 때 진입하는 화면이다. 해당 유저가 **저장소 공개 + 게시글 공개** 두 조건을 모두 충족한 게시글만 그리드 형태로 노출된다 (공통정책 §7.1 참조).

### 주요 기능

- 특정 유저의 공개 기억 그리드 조회 (읽기 전용·최신순)
- 게시글 좋아요 (공통정책 §7.6 참조)
- 게시글 조회수 자동 카운팅 (브라우저 세션 + 24시간 쿨다운)
- 게시글 카드 탭 → [CEB-MEM-202] 기억 상세 진입

> 그 외 동작은 [CEB-MEM-102] 모두의 기억저장소와 동일하다. 본 문서는 차이점 위주로 기술한다.

### 접근 경로

- [CEB-MEM-102] 모두의 기억저장소 > 게시글 카드의 닉네임 탭
- 비로그인 유저도 진입 가능 (좋아요·조회수 카운트 미적립). MEM-102 자체가 비로그인 열람 허용 화면이므로 동일하게 적용

> MEM-101/102와 달리 본 화면은 **타인 저장소**이므로 본인 설정(⚙️)과 뷰 전환 버튼이 표시되지 않는다.

### 2.2 그리드 뷰

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 구성 | **3열 그리드**. 정사각형 카드 (MEM-102와 동일 규격) |
| 프로필 사진 | **본 화면에서는 표시 생략** (이미 닉네임 저장소이므로 작성자가 자명) |
| 닉네임 | 카드 내 닉네임 텍스트 미표시 (헤더 타이틀로 갈음) |
| 멀티 사진 아이콘 🖼️ | 등록 이미지 ≥2장일 때 표시 |
| 메인 사진 | 등록 사진 첫 장 OR **시스템 생성 SVG (글 20자 컷오프)** — 사진 미등록 게시글의 경우 (MEM-102 §2.2 동일 규격) |
| 조회수 | `👁 N` 천 단위 콤마 (예: 1,234) |
| 좋아요 | `❤ N` 1탭 토글 |
| 정렬 | **최신순 단일** (`created_at DESC`). 정렬 토글 없음 |
| 탭 동작 | 카드 탭 → [CEB-MEM-202] 기억 상세 |

> 카드 내부 구성·시스템 생성 SVG·좋아요 동작·조회수 동작은 [CEB-MEM-102] §2.2와 동일. 본 화면 고유의 차이점만 위에 명시.

#### 좋아요·조회수 동작

[CEB-MEM-102] §2.2 좋아요/조회수 동작 정책 동일 적용. 공통정책 §7.6 참조.

## 3. 상태별 표시

| 상태 | 처리 |
| ---- | ---- |
| Loading | 그리드 영역에 Skeleton UI (3열 × 4행) |
| Default | 정상 그리드 표시 |
| Empty | "아직 공개된 기억이 없어요" 안내 텍스트 (해당 유저가 저장소는 공개했으나 공개 게시글이 0건일 때) |
| 저장소 비공개 전환 감지 | 진입 시점에 해당 유저가 저장소를 비공개로 전환한 경우 → "이 유저는 저장소를 비공개로 전환했어요" 안내 + [돌아가기] CTA → MEM-102 |
| Error | "앗, 불러오지 못했어요" + [다시 불러오기] 버튼 (공통정책 §11.3) |
