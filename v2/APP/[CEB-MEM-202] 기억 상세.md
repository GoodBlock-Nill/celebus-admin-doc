## 1. 화면 개요

작성된 기억의 상세 내용을 확인하고, 수정/삭제/신고하는 화면이다. 내 기억과 타인의 공개 기억 모두 이 화면에서 열람한다.

> 핵심 원칙: 팬이 기록한 소중한 순간을 풍부하게 다시 체험할 수 있어야 한다.

##### 주요 기능

- 감정 이모지 + 유형 + 미디어 + 텍스트 + 위치 전체 표시
- 이미지 풀스크린 갤러리 (스와이프 + 핀치 줌)
- 이미지 최대 3장 표시
- 내 기억: 수정/삭제
- 타인 공개 기억: 신고
- 공개 기억 외부 공유 (Web Share API / 클립보드 복사)

##### 접근 경로

- CEB-MEM-101 > 기억 항목 탭 (캘린더/그리드/지도 뷰에서)
- 푸시 알림 "1년 전 오늘 기억" 탭 시 진입
- 외부 공유 링크 접속 (app.celebus.xyz/memory/{memoryId}) — 비로그인도 공개 기억 읽기 전용 열람 가능

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.5 메뉴 (수정/삭제/신고)

내 기억과 타인의 공개 기억에 따라 메뉴가 다르게 표시된다.

#### UI 구성

| 조건 | 메뉴 |
| ---- | ---- |
| 내 기억 | [수정하기] / [삭제하기] |
| 타인의 공개 기억 | [신고하기] |
| 타인의 비공개 기억 | 접근 불가 (리스트에 미노출) |
| 공유 링크 접속 (비로그인) | ⋯ 메뉴 미노출 |

#### 동작

##### 수정하기

- [수정하기] 탭 → CEB-MEM-201 (mode=edit, 기존 데이터 프리필)
- 유형은 변경 불가
- 수정 완료 후 본 화면으로 자동 복귀 + 수정된 내용 반영
- 덕력 중복 미지급

##### 삭제하기

- [삭제하기] 탭 → 확인 모달:
  - 텍스트: "삭제한 기억은 복구할 수 없어요. 삭제할까요?"
  - CTA: [취소] / [삭제]
- 삭제 완료 → 토스트(Success) "기억이 삭제되었어요"
- CEB-MEM-101로 자동 복귀 + 캘린더 데이터 재조회
- 덕력 환수 없음

##### 신고하기 (공개 기억, 타인 것만)

- [신고하기] 탭 → 신고 사유 선택 바텀시트:

| 신고 사유 | 코드 |
| ---- | ---- |
| 부적절한 콘텐츠 | INAPPROPRIATE |
| 스팸 | SPAM |
| 욕설·혐오 | HATE_SPEECH |
| 개인정보 노출 | PRIVACY |
| 기타 | OTHER |

- 사유 선택 + [신고] 탭 → 토스트(Success) "신고가 접수되었어요"
- 동일 기억 중복 신고 차단: 토스트(Info) "이미 신고한 기억이에요"
- 신고 접수 후 BO에서 검토 (별도 BO 화면)

##### 공개/비공개 접근 규칙

| 규칙 | 설명 |
| ---- | ---- |
| 내 기억 | 공개/비공개 모두 열람 가능 |
| 타인의 기억 | 공개 기억만 열람 가능. 비공개 기억은 리스트에 미노출 |
| 공개 조건 | 기억저장소 전체 공개 설정 + 개별 기억 공개 설정 모두 ON (공통정책 7.1 참조) |

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 삭제 실패 | 토스트(Error) "삭제에 실패했어요. 다시 시도해주세요" |
| 신고 실패 | 토스트(Error) "신고 접수에 실패했어요. 다시 시도해주세요" |
| 중복 신고 | 토스트(Info) "이미 신고한 기억이에요" |

### 2.7 공유 링크 접속

외부 공유 링크(`app.celebus.xyz/memory/{memoryId}`)를 통해 접속한 유저의 화면.

#### UI 구성

| 접속 상태 | 화면 |
| ---- | ---- |
| 로그인 유저 | 기존 CEB-MEM-202 동일 (⋯ 메뉴 포함) |
| 비로그인 유저 | 읽기 전용 상세 화면 (⋯ 메뉴 미노출) + 하단 가입 유도 배너 |
| 공개 기억 | 정상 표시 |
| 비공개 기억 | "이 기억은 비공개예요" 안내 + [CELEBUS 홈으로] CTA |
| 삭제된 기억 | "삭제된 기억이에요" 안내 + [CELEBUS 홈으로] CTA |

##### 비로그인 읽기 전용 화면 구성

| 영역 | 내용 |
| ---- | ---- |
| 상단 | CELEBUS 로고 (홈으로 이동) |
| 본문 | 기억 상세 전체 표시 (이모지, 유형, 미디어, 텍스트, 위치) |
| ⋯ 메뉴 | 미노출 (수정/삭제/신고 불가) |
| 공유 버튼 | 표시 (비로그인도 재공유 가능) |
| 하단 고정 배너 | "CELEBUS에서 나만의 기억을 남겨보세요 💜" + [시작하기] CTA → app.celebus.xyz/home |

##### OG 메타 태그 (SNS 미리보기)

| 태그 | 값 |
| ---- | ---- |
| og:title | "{닉네임}의 기억 💜" |
| og:description | 텍스트 앞 100자 (텍스트 없으면 "CELEBUS에서 기억을 공유했어요") |
| og:image | CELEBUS 로고 이미지 (사진 미포함 — 개인정보 보호) |
| og:url | `https://app.celebus.xyz/memory/{memoryId}` |
| og:type | article |

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 공유 링크 → 비공개 기억 | "이 기억은 비공개예요" 안내 + [CELEBUS 홈으로] CTA |

### 3.2 비로그인 공통

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역(2.1~2.7)의 "비로그인" 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

- 앱 내 접근 (MEM-101 → MEM-202): 로그인 상태에서만 접근 가능 (기존 유지). CEB-ART-101 카드 탭 시 로그인 유도 (공통정책 10.5 참조)
- 공유 링크 접근 (외부 URL): 비로그인도 공개 기억 읽기 전용 열람 가능. ⋯ 메뉴 미노출. 하단 가입 유도 배너 표시
- 비로그인 CTA: [시작하기] → app.celebus.xyz/home 이동
