## 1. 화면 목적

아티스트 탭(CEB-ART-101)에서 "Raffle" 카드를 탭했을 때 진입하는 화면이다. 해당 아티스트의 래플 이벤트 목록을 필터별로 표시하고, 각 래플 상세/참여/결과 확인으로 이동하는 허브 화면이다.

##### 주요 기능

- [진행중] / [마감] 2개 필터로 래플 목록 조회
- 래플 카드에 경품 이미지 + 경품명 + D-day + 참여자 수 + 내 응모 상태 표시
- 카드 탭 시 상태에 따라 상세 화면(진행중/당첨/미당첨)으로 이동
- 상단에 보유 응모권 수 표시

##### 접근 경로

- CEB-ART-101 > Raffle 카드 탭
- CEB-HOM-001 > 바로가기 > Raffle 카드 탭
- CEB-HOM-001 > 추천 카드 [응모하기] 탭

## 2. UI 설명

### 2.1 화면 레이아웃

> 화면 레이아웃은 프로토타입(localhost:3001)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.2 영역별 구성요소

##### 서브페이지 헤더

| 항목 | 내용 |
| ---- | ---- |
| 뒤로가기 | ← 탭 시 CEB-ART-101로 복귀 |
| 타이틀 | "Raffle" |
| 보유 응모권 | "응모권: {N}장" (우측 표시). 탭 시 CEB-FQ-210 응모권 내역 화면으로 이동 |
| 스타일 | 피그마 참조 |

##### 필터 탭

| 항목 | 내용 |
| ---- | ---- |
| 구성 | [진행중] [마감] 2개 탭 |
| 기본 선택 | 진행중 |
| 진행중 | 현재 응모 가능한 래플 (Active 상태) |
| 마감 | 응모 마감 및 추첨 완료된 래플 (당첨/미당첨 결과 포함) |
| 탭 전환 | 즉시 리스트 갱신 |
| 스타일 | 피그마 참조 |

##### 래플 카드

| 항목 | 내용 |
| ---- | ---- |
| 경품 이미지 | 좌측 썸네일 (정사각형, 피그마 참조) |
| 경품명 | 이미지 우측 상단 (Bold) |
| D-day | "D-{N}" 또는 "오늘 마감" (진행중 탭에서만 표시) |
| 참여자 수 | "참여 {N}명" |
| 내 응모 상태 | 응모함: "내 응모: {N}장" (보라색) / 미응모: "미응모" (회색) |
| 결과 뱃지 | 당첨: "🎉 당첨" (초록) / 미당첨: "미당첨" (회색) / 추첨 대기: "추첨 대기" (주황) |
| 탭 동작 | 상태에 따라 상세 화면 이동 (3.1 참조) |
| 스타일 | 피그마 참조 |

##### 래플 카드 상태별 표시

| 상태 | 표시 내용 | 탭 동작 |
| ---- | ---- | ---- |
| 진행중 (Active) | D-day + 참여자 수 + 내 응모 상태 | → CEB-FQ-202 Raffle 상세(진행중) |
| 추첨 대기 (Drawing) | "추첨 대기" 뱃지 + 참여자 수. 마감 탭에 표시 | 탭 시 토스트(Info) "추첨 결과를 기다려주세요" |
| 당첨 (Winner) | "🎉 당첨" 뱃지 + 참여자 수 | → CEB-FQ-203 Raffle 상세(당첨) |
| 미당첨 (Loser) | "미당첨" 뱃지 + 참여자 수 | → CEB-FQ-204 Raffle 상세(미당첨) |
| 미참여 마감 | "마감" 뱃지 + 참여자 수 | 탭 시 래플 정보 조회 (읽기 전용) |

## 3. 화면 내 기능설명

### 3.1 래플 카드 탭 액션

| 카드 상태 | 탭 동작 |
| ---- | ---- |
| 진행중 + 미응모 | → CEB-FQ-202 (응모 가능) |
| 진행중 + 응모함 | → CEB-FQ-202 (추가 응모 가능) |
| 추첨 대기 | 토스트(Info) "추첨 결과를 기다려주세요" |
| 당첨 | → CEB-FQ-203 (당첨 결과 및 수령 안내) |
| 미당첨 | → CEB-FQ-204 (위로 메시지) |
| 미참여 마감 | 래플 정보 조회 화면 (읽기 전용) |

### 3.2 필터 전환

- [진행중] 탭: Active 상태 래플만 표시. 정렬: D-day 임박순 (마감이 가까운 순)
- [마감] 탭: 추첨 완료 및 미참여 마감 래플 표시. 정렬: 마감일 최신순 (최근 마감 먼저)
- 탭 전환 시 리스트 즉시 갱신

### 3.3 보유 응모권 표시

- 헤더 우측에 "응모권: {N}장" 표시
- 래플 상세(CEB-FQ-202)에서 응모 후 본 화면으로 복귀 시 최신 수량으로 갱신

### 3.4 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 래플 리스트 및 보유 응모권 수 조회 |
| Pull-to-Refresh | 전체 데이터 재조회 (공통정책 11.2 참조) |
| 상세 화면 복귀 | 래플 상태 및 보유 응모권 수 재조회 |
| 필터 전환 | 해당 필터 리스트 재조회 |

## 4. 규칙/로직

### 4.1 래플 정렬 규칙

| 필터 | 정렬 기준 |
| ---- | ---- |
| 진행중 | D-day 임박순 (마감이 가까운 래플이 상단) |
| 마감 | 마감일 최신순 (가장 최근 마감된 래플이 상단) |

### 4.2 응모권 정책

| 규칙 | 설명 |
| ---- | ---- |
| 단위 | 1장 = 1회 응모 |
| 소비 | 응모 시 즉시 차감. 취소 및 반환 불가 |
| 당첨 확률 | 내 응모 수 / 전체 응모 수 = 기본 당첨 확률 |
| 획득 경로 | Quest 보상, ST 탈락 보상, PM 보상, GP 구매 (100GP = 1장, BO 설정 기준) |
| 아티스트별 분리 | 응모권은 아티스트별 분리 (artist_id 기반) |

### 4.3 D-day 표시 규칙

| 조건 | 표시 |
| ---- | ---- |
| 마감까지 2일 이상 | "D-{N}" (예: D-5, D-12) |
| 마감 당일 | "오늘 마감" |
| 마감일 지남 | D-day 미표시 (마감 탭으로 이동됨) |

### 4.4 멀티 아티스트 대응

- 래플 이벤트는 아티스트별 분리
- artist_id 기반으로 데이터 조회
- 아티스트 전환 시 CEB-ART-101 재진입 → 해당 아티스트의 래플 목록 표시

## 5. 에러 및 예외상황

| 상태 | 처리 |
| ---- | ---- |
| Loading | 카드 리스트 영역에 Skeleton UI 표시 |
| Default | 래플 리스트 정상 표시 |
| Error | 전체 조회 실패 시 에러 안내 메시지 + 재시도 버튼 |
| Empty (진행중 0건) | "현재 진행 중인 래플이 없습니다" 안내 + "새 래플이 열리면 알려드릴게요" + [알림 설정] CTA |
| Empty (마감 0건) | "아직 참여한 래플이 없습니다" 안내 + "진행중인 래플에 참여해보세요" |
| 응모권 0장 | 헤더에 "응모권: 0장" 표시. 응모 시도 시 CEB-FQ-202에서 응모 불가 처리 |
| 비로그인 | 본 화면은 로그인 상태에서만 접근 가능. CEB-ART-101에서 Raffle 카드 탭 시 로그인 유도 처리됨 (공통정책 10.5 참조) |
| 추첨 소요 시간 | 추첨 대기 탭 시 토스트: "추첨 결과를 기다려주세요. 결과는 마감 후 영업일 2일 이내 발표돼요" |
