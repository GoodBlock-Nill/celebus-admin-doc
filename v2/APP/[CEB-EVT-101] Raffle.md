## 1. 화면 개요

아티스트 탭(CEB-ART-101)에서 "Raffle" 카드를 탭했을 때 진입하는 화면이다. 해당 아티스트의 래플 이벤트 목록을 필터별로 표시하고, 각 래플 상세/참여/결과 확인으로 이동하는 허브 화면이다.

##### 주요 기능

- [전체] / [진행중] / [집계중] / [종료] 4개 필터로 래플 목록 조회
- "내 참여만 보기" 토글로 본인 응모 래플만 필터
- 래플 카드(풀 너비 이미지)에 경품명 + 응모 수 + 당첨인원 + D-day + 인라인 [응모하기]
- 카드 탭 시 상태에 따라 상세 화면(진행중/당첨/미당첨)으로 이동
- 헤더에 보유 응모권 수 표시

##### 접근 경로

- CEB-ART-101 > Raffle 카드 탭
- CEB-HOM-001 > 바로가기 > Raffle 카드 탭
- CEB-HOM-001 > 추천 카드 [응모하기] 탭

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

### 2.2 필터 탭

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 구성 | [전체] [진행중] [집계중] [종료] 4개 탭 |
| 기본 선택 | 전체 |
| 내 참여만 보기 | 필터 탭 아래 체크박스 토글. 체크 시 본인이 응모한 래플만 표시 |
| 전체 | 진행중 + 집계중 + 종료 모든 래플 |
| 진행중 | 현재 응모 가능한 래플 (Active 상태) |
| 집계중 | 응모 마감 후 추첨 대기 중인 래플 (Tallying 상태). 공통 정책 §4.7 참조 |
| 종료 | 추첨 완료된 래플 (Closed): 당첨 / 미당첨 / 미참여 마감 포함 |
| 탭 전환·토글 | 즉시 리스트 갱신 |
| 스타일 | 피그마 참조 |

#### 동작

- [전체] 탭: 모든 상태 래플 표시. 정렬: 진행중 → 집계중 → 종료 그룹 순, 그룹 내 마감 임박/최신순
- [진행중] 탭: Active 상태 래플만 표시. 정렬: D-day 임박순 (마감이 가까운 순)
- [집계중] 탭: Tallying 상태 래플만 표시 (응모 마감 + 추첨 미완료). 정렬: 마감일 최신순 (최근 마감 먼저)
- [종료] 탭: 추첨 완료 래플만 표시 (당첨 / 미당첨 / 미참여 마감). 정렬: 마감일 최신순 (최근 마감 먼저)
- "내 참여만 보기" 체크 → 현재 탭 결과에서 본인 응모 래플만 추가 필터
- 탭 전환·토글 시 리스트 즉시 갱신

##### 래플 정렬 규칙

| 필터 | 정렬 기준 |
| ---- | ---- |
| 전체 | 진행중 → 집계중 → 종료 그룹 순, 각 그룹 내 마감 임박/최신순 |
| 진행중 | D-day 임박순 (마감이 가까운 래플이 상단) |
| 집계중 | 마감일 최신순 (최근 마감된 래플이 상단) |
| 종료 | 마감일 최신순 (가장 최근 마감된 래플이 상단) |

### 2.3 래플 카드

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 경품 이미지 | 카드 상단 풀 너비 이미지 (피그마 참조) |
| 타입 뱃지 | "래플" 좌상단 뱃지 |
| 경품명 | 이미지 아래 아티스트명 + 경품명 (Bold) |
| 지표 | "응모 {N}" + "당첨인원 {N}명" |
| D-day | "D-{N}" 또는 "오늘 마감" (진행중에서만 표시) + 기간(YYYY.MM.DD ~ MM.DD) |
| 내 응모 상태 | 응모함: "내 응모: {N}장" (보라색) / 미응모: "미응모" (회색) |
| 결과 뱃지 | 당첨: "🎉 당첨" (초록) / 미당첨: "미당첨" (회색) / 집계 중: "집계 중" (노랑, 공통 정책 §4.7) / 종료: CLOSED 스탬프 |
| 인라인 CTA | 진행중 카드에 [응모하기] 버튼 (카드 내 직접 응모 진입 → CEB-FQ-202) |
| 스타일 | 피그마 참조 |

#### 동작

| 카드 상태 | 탭 동작 |
| ---- | ---- |
| 진행중 + 미응모 | → CEB-FQ-202 (응모 가능) |
| 진행중 + 응모함 | → CEB-FQ-202 (추가 응모 가능) |
| 집계 중 | 토스트(Info) "추첨 결과를 기다려주세요" |
| 당첨 | → CEB-FQ-203 (당첨 결과 및 수령 안내) |
| 미당첨 | → CEB-FQ-204 (위로 메시지) |
| 미참여 마감 | 래플 정보 조회 화면 (읽기 전용) |

#### 상태별 표시

| 상태 | 표시 내용 | 탭 동작 |
| ---- | ---- | ---- |
| 진행중 (Active) | D-day + 참여자 수 + 내 응모 상태 | → CEB-FQ-202 Raffle 상세(진행중) |
| 집계 중 (Tallying) | "집계 중" 뱃지 + 참여자 수. 집계중 탭에 표시 | 탭 시 토스트(Info) "추첨 결과를 기다려주세요" |
| 당첨 (Winner) | "🎉 당첨" 뱃지 + 참여자 수 | → CEB-FQ-203 Raffle 상세(당첨) |
| 미당첨 (Loser) | "미당첨" 뱃지 + 참여자 수 | → CEB-FQ-204 Raffle 상세(미당첨) |
| 미참여 마감 | "마감" 뱃지 + 참여자 수 | 탭 시 래플 정보 조회 (읽기 전용) |

##### D-day 표시 규칙

| 조건 | 표시 |
| ---- | ---- |
| 마감까지 2일 이상 | "D-{N}" (예: D-5, D-12) |
| 마감 당일 | "오늘 마감" |
| 마감일 지남 | D-day 미표시 (마감됨 탭으로 이동됨) |

#### 비로그인

- 비로그인도 래플 목록 열람 가능. 내 응모 상태 "-" 표시
- 래플 카드 탭(응모) 시 → 로그인 화면 이동 (공통정책 10.5 참조)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| Empty (전체 0건) | "현재 진행 중인 래플이 없습니다" 안내 + "새 래플이 열리면 알려드릴게요" + [알림 설정] CTA + "퀘스트를 완료하고 응모권을 모아두세요!" + [V01D 퀘스트] CTA → CEB-FQ-101 |
| Empty (진행중 0건) | "지금 응모할 수 있는 래플이 없어요" + [알림 설정] CTA |
| Empty (집계중 0건) | "집계 중인 래플이 없어요" Empty 메시지 |
| Empty (종료 0건) | "종료된 래플이 없어요" Empty 메시지 |
| Empty (내 참여만 보기 0건) | "아직 참여한 래플이 없습니다" 안내 + "진행중인 래플에 참여해보세요" |
| 추첨 소요 시간 | 집계중 탭 진입 또는 집계 중 래플 카드 탭 시 토스트: "추첨 결과를 기다려주세요. 결과는 마감 후 영업일 2일 이내 발표돼요" |

### 2.4 보유 응모권

##### 응모권 정책

| 규칙 | 설명 |
| ---- | ---- |
| 단위 | 1장 = 1회 응모 |
| 소비 | 응모 시 즉시 차감. 취소 및 반환 불가 |
| 당첨 확률 | 내 응모 수 / 전체 응모 수 = 기본 당첨 확률 |
| 획득 경로 | Quest 보상, ST 탈락 보상, PM 보상, GP 구매 (100GP = 1장, BO 설정 기준) |
| 범위 | **아티스트 무관 통합** — 어떤 아티스트 활동으로 획득한 응모권이든 모든 래플에 사용 가능 |

## 3. 공통 규칙

### 3.1 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 래플 리스트 및 보유 응모권 수 조회 |
| Pull-to-Refresh | 전체 데이터 재조회 (공통정책 11.2 참조) |
| 상세 화면 복귀 | 래플 상태 및 보유 응모권 수 재조회 |
| 필터 전환 | 해당 필터 리스트 재조회 |

### 3.2 멀티 아티스트 대응

- 래플 이벤트는 아티스트별 분리 (artist_id 기반 조회)
- 아티스트 전환 시 CEB-ART-101 재진입 → 해당 아티스트의 래플 목록 표시
- **응모권은 아티스트 무관 통합** — 어떤 아티스트 활동으로 획득한 응모권이든 모든 래플에 사용 가능

### 3.3 비로그인 공통

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역(2.1~2.3)의 "비로그인" 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

- 비로그인도 열람 가능. 래플 목록 표시
- 내 응모 상태 "-", 응모권 "-" 표시
- 비로그인 CTA 탭 → 로그인 화면 이동 (공통정책 10.5 참조)

### 3.4 공통 에러 상태

| 상태 | 처리 |
| ---- | ---- |
| Loading | 카드 리스트 영역에 Skeleton UI 표시 |
| Default | 래플 리스트 정상 표시 |
| Error | 전체 조회 실패 시 에러 안내 메시지 + 재시도 버튼 |

### 3.5 v1 스펙 재활용

> v1 스펙(CEB-FQ-202~204, 210, CEB-FQ-202-MD)은 접근 경로를 CEB-EVT-101 기준으로 수정하여 재활용합니다.
> **v1 스펙 마이그레이션 체크리스트**: 각 v1 문서의 접근 경로(Breadcrumb)를 CEB-EVT-101 기준으로 업데이트, 연관화면에 CEB-EVT-101 추가 필요.
