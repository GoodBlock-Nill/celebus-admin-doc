## 1. 화면 개요

진행 중인(Active) 래플의 상세 정보·혜택·내 응모 현황을 확인하고, 보유 응모권으로 응모하는 화면.

##### 주요 기능
- Raffle 기본정보 + 보상 + D-day
- 누적 참여자·누적 응모권 통계
- 내 누적 응모권 표시
- 응모 수량 입력(Stepper) + [응모하기]
- 상태 분기: Active / Closed / Ended

##### 접근 경로
- [CEB-EVT-101] Raffle 메인 > 진행중 카드 탭
- [CEB-HOM-001] 홈 메인 > 이벤트 카드 탭
- 딥링크

## 2. UI 설명

```text
[<] RAFFLE
─────────────────────────────────
{아티스트명}
{타이틀}
D-{n}  YYYY.MM.DD ~ MM.DD
{설명} (2줄 초과 시 더보기)
─────────────────────────────────
[보상 정보]  팬싸인회 입장권 1장 …
─────────────────────────────────
[참여 통계]  누적 참여 {N}명 · 누적 응모권 {M}장
[내 참여 현황]  내 누적 응모권 {n}장
─────────────────────────────────
[응모 영역] (하단 고정)
사용 가능 {N}장 · [-] 1 [+] · [응모하기]
─────────────────────────────────
```

| 영역 | 항목 | 표시 |
| --- | --- | --- |
| 헤더 | [<] / 타이틀 "RAFFLE" | |
| 기본 정보 | 아티스트, 타이틀, D-day, 기간, 설명 | BO [CEB-BO-FQ-102-CREATE] 연동 |
| 보상 | 보상 내역 | BO 보상 상세 |
| 통계 | 누적 참여자 · 누적 응모권 | 중복 제외 / 합계 |
| 내 현황 | 내 누적 응모권 | 로그인 시만 |
| 응모 | 사용 가능 응모권 + Stepper + [응모하기] | 보유 0장 시 비활성 |

> 시각적 스타일은 피그마 참조

## 3. 규칙·로직

### 3.1 상태 분기

| Raffle 상태 | 처리 |
| --- | --- |
| Active | 본 화면 유지 |
| Closed (집계 중) | "현재 집계 진행중입니다" 토스트 → 이전 화면 복귀 |
| Ended + 당첨 | [CEB-EVT-103] Raffle 상세 (당첨) replace |
| Ended + 미당첨 | [CEB-EVT-104] Raffle 상세 (미당첨) replace |

### 3.2 응모 수량 (Stepper)

| 항목 | 규칙 |
| --- | --- |
| 기본값 | 1 (보유 ≥ 1) |
| 최소 | 1 (0 입력 시 1로 보정) |
| 최대 | 보유 응모권 |
| 1인당 한도 | 없음 (Max Cap 정책 없음) |
| 비활성 조건 | 보유 0장 → Stepper 0 fixed + Disabled |

보유 부족 시 "보유 응모권이 부족합니다" 인라인 에러.

### 3.3 [응모하기] 동작

- 입력 수량 유효 시 활성
- 클릭 → [CEB-EVT-102-MD] Raffle 응모 모달 호출 (수량 n 전달)

### 3.4 체류 중 마감 처리

- 상세 체류 중 서버에서 Active → Closed 전환된 경우, [응모하기] 클릭 시 `EVENT_CLOSED` 응답 → "이벤트가 종료되었습니다" 안내 후 이전 화면 복귀.
- 예: D-day가 다음날 00시로 넘어간 직후.

### 3.5 응모권 거래 코드 (BO-006 §7.6)

응모 시 차감 트랜잭션은 **`TICKET_RAFFLE_USE`** 코드로 BO [CEB-BO-FQ-301] 응모권 거래 내역에 기록.

### 3.6 다국어

- 아티스트명·타이틀·설명: 서버 데이터 (BO 다국어 설정 의존)
- 시스템 UI: 앱 i18n (KO/EN/JP)
