## 1. 화면 개요

아티스트 탭(CEB-ART-101)에서 "{아티스트명} 키우기" 카드를 탭했을 때 진입하는 화면이다. (유저 대면 명칭: {아티스트명} 키우기, 예: V01D 키우기) 팬덤 전체의 공동 레벨업 진행 상황과 레벨별 보상을 확인하고, 덕력 획득 활동으로 유도하는 화면이다.

> 참고: 팬덤 레벨은 개인 경쟁이 아닌 팬덤 전체의 공동 달성 목표이다. 개인 순위 경쟁은 CEB-DUK-101 덕력 랭킹에서 제공한다.

##### 주요 기능

- 현재 팬덤 레벨 + 프로그레스 바 + 내 기여도 확인
- 레벨별 보상 리스트 확인 (해금 완료 / 진행중 / 미해금 "???" 실루엣)
- 팬덤 활동 요약 (참여 팬 수 + 이번 달 합계 + 주요 활동 유형)
- 덕력 획득 활동 유도 CTA (출석 체크 / 퀘스트 / 랭킹)
- 레벨업 달성 시 축하 모달 표시

##### 접근 경로

- CEB-ART-101 > 팬덤 레벨 카드 탭
- CEB-HOM-001 > 바로가기 > {아티스트명} 키우기 카드 탭
- CEB-HOM-001 > 추천 카드 [기여하기] 탭

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

> MAX 상태 레이아웃(Lv.5 달성 + 추가 레벨 미등록)도 프로토타입에서 확인할 수 있습니다.

##### 2.2.1 권장 레벨링 곡선 (v2.1 신규 / v2.2 리셋 모델 명문화)

> **카운터 모델 (v2.2 명시)**: 각 레벨에서 **0부터 시작**하여 아래 "레벨업 목표 덕력" 도달 시 다음 레벨 진입과 동시에 **카운터 0으로 리셋**. 즉, 각 레벨의 목표값은 **이전 레벨 누적이 아니라 해당 레벨 단독 목표값**. 진행률은 `현재 레벨 카운터 / 해당 레벨 목표 덕력 × 100 (%)`.

| 레벨 | 레벨업 목표 덕력 | 비고 |
| ---: | ---: | ---- |
| Lv 1 | 10,000 | 첫 레벨업 — 작은 팬덤도 빠른 체감 |
| Lv 2 | 30,000 | |
| Lv 3 | 70,000 | |
| Lv 4 | 150,000 | |
| Lv 5 | 300,000 | **6~12개월 Lv5 도달 설계 기준점** |
| Lv 6 | 550,000 | |
| Lv 7 | 1,000,000 | |
| Lv 8 | 1,800,000 | |
| Lv 9 | 3,200,000 | |
| Lv 10 | 6,000,000 | 최종 — 12개월+ 활성 팬덤 도달 |

> **권장 곡선 설계 원칙**: 초기 빠른 레벨업 → 후반 점진 증가. 동일 활성도 기준 레벨업 간격이 점차 길어져 도전감 유지.
> **BO 자유 변경**: 운영자(BO 관리자)는 위 권장값을 자유롭게 조정 가능. 미설정 시 권장값 자동 적용 (fallback).

##### 2.2.2 0DUK 기여 유저 처리 (v2.1 신규 — 디자이너 피드백 반영)

레벨업 발생 시 해당 아티스트에 **획득 덕력 0DUK** 기여한 유저는 다음과 같이 처리한다 ("조용 제외 + 유도" 정책).

| 항목 | 처리 |
| ---- | ---- |
| 레벨업 푸시 알림 | **수신 X** (조용 제외) |
| 레벨업 축하 모달 | **표시 X** |
| 보상 지급 | **제외** (기존 정책 유지) |
| 다음 앱 진입 시 안내 배너 | **1회 노출** — "V01D의 레벨업 축하해주세요. 이번 턴 참여하면 다음 보상에서 만나요!" |
| 배너 닫기 | 유저 직접 닫기 가능. 닫은 후 재노출 X |
| 활성 유저(1DUK 이상) | 기존 정책대로 푸시 + 모달 + 보상 모두 정상 진행 |

> **취지**: 0DUK 유저에게 레벨업 알림을 강제 노출하면 "왜 나는 보상 못 받지?" 이탈 유발 우려. 대신 다음 방문 시 자연스럽게 다음 라운드 참여 유도하여 차별화·이탈 방지 균형.

| 보상 종류 | BO에서 자유 설정 (독점 콘텐츠 / 래플 / 디지털 굿즈 / 이벤트 등). 레벨 수와 각 레벨별 보상은 BO에서 자유 설정 가능 (최대 10레벨) |
| 보상 수령 | 디지털 보상: 서버 자동 지급 (기여 1DUK 이상 유저). 실물 보상: 래플 추첨 (기여도 비례 가중치). 상세는 2.2 보상 유형별 지급 방식 참조 |
| 미해금 탭 | 탭 불가 (비활성) |
| 스타일 | 피그마 참조 |

#### 동작

- 해금 완료 레벨: 보상명 + ✅ 수령 완료 표시
- 현재 진행 레벨: 보상명 + 레벨 내 진행률 프로그레스 바 표시
- 미해금 레벨: "???" 실루엣 표시 (탭 불가)
- 디지털 보상: 레벨 달성 즉시 기여 유저(1DUK 이상)에게 자동 지급
- 실물 보상(사인앨범 등): 기여 유저 중 래플 추첨으로 지급 (기여도 비례 가중치)

#### 비로그인

- 비로그인도 열람 가능. 보상 리스트 전체 표시

### 2.4 팬덤 활동 요약

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 참여 팬 수 | "👥 참여 팬: {N}명" (이번 달 1DUK 이상 획득한 유저 수. 월간 스냅샷) |
| 이번 달 합계 | "⭐ 이번 달 합계: {N}DUK" (이번 월 팬덤 전체 획득 덕력 합계. **월간 스냅샷** — 매월 1일 수치 리셋) |
| 주요 활동 유형 | "🎯 가장 많은 활동: {유형}" (Quest / 게임 / 스트리밍 등 BO 데이터 기반. 월간 스냅샷) |
| 목적 | 팬덤 공동 달성 감각 강화. 개인 순위는 CEB-DUK-101에서 제공 (이 섹션에 TOP 기여자 리스트 없음) |
| 스타일 | 피그마 참조 |

> **레벨 진행률 vs 활동 요약 구분**: 팬덤 레벨 프로그레스 바는 **상시 누적** (리셋 없음, 공통정책 5.5 참조). 팬덤 활동 요약의 "이번 달 합계/참여 팬/주요 활동"은 **월간 스냅샷** (매월 1일 수치만 리셋). 즉 월이 바뀌어도 레벨 진행률은 그대로이고, 활동 요약 수치만 새 달 기준으로 초기화됩니다.

#### 동작

- 참여 팬 수, 이번 달 합계, 주요 활동 유형 표시 (모두 월간 스냅샷)
- 매월 1일에 활동 요약 수치만 리셋 (레벨 진행률은 유지)
- TOP 기여자 개인 리스트는 표시하지 않음 (CEB-DUK-101 덕력 랭킹에서 제공)

#### 비로그인

- 비로그인도 열람 가능. 활동 요약 수치 전체 표시

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역(2.1~2.6)의 "비로그인" 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.
