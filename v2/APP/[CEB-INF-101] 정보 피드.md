## 1. 화면 개요

아티스트 탭(CEB-ART-101)에서 "정보" 카드를 탭했을 때 진입하는 화면이다. 아티스트의 소식(뉴스)과 일정을 시간순 통합 타임라인으로 보여주는 정보 허브 역할을 한다.

##### 주요 기능

- 소식(뉴스)과 일정을 시간순 통합 타임라인으로 표시
- "오늘 / 이번 주 / 지난 주 / 이전" 날짜 그룹핑
- **아티스트별** 🔔 알림 ON/OFF 설정 기능 (개별 일정 알림 아님, 아티스트 단위)
- IX 선공개 일정/소식에 "CELEBUS 단독" 뱃지 표시
- **아티스트 공지**는 상단 고정 배너 (등록된 공지가 있을 때만 표시). 서비스(플랫폼) 공지는 알림센터(CEB-NTF-510)에서 처리

##### 접근 경로

- CEB-ART-101 > 정보 카드 탭
- CEB-HOM-001 > 바로가기 > 오늘의 일정/소식 카드 탭

##### 화면 레이아웃

> 화면 레이아웃은 프로토타입(https://celebus-app.vercel.app/home)을 참조하세요. 플로팅 디버그 버튼으로 다양한 상태를 확인할 수 있습니다. 프로토타입은 구조/플로우 확인용이며, 시각적 스타일은 Figma 시안을 따릅니다.

## 2. 영역별 상세

### 2.1 서브페이지 헤더

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 뒤로가기 | ← 탭 → 이전 화면으로 복귀 (네비게이션 스택 기반) |
| 타이틀 | "{아티스트명} 정보" (예: V01D 정보) |
| 우측 | 🔔 **아티스트별** 알림 ON/OFF 토글 버튼. ON: 🔔 보라색 / OFF: 🔕 회색. 현재 선택된 아티스트의 일정/소식 알림을 제어 |
| 스타일 | 피그마 참조 |

#### 동작

- 🔔 토글 ON → 토스트(Success) "{아티스트명} 알림이 설정되었어요. 새로운 일정과 소식을 알려드릴게요"
- 🔔 토글 OFF → 토스트(Info) "{아티스트명} 알림이 해제되었어요"
- 알림 ON 시: **해당 아티스트의** 모든 새 일정/소식에 대해 푸시 발송 (D-1 20:00 KST + 당일 00:00 KST)
- 알림 OFF 시: **해당 아티스트의** 푸시 미발송 (다른 아티스트 알림 설정에는 영향 없음)
- 아티스트 전환 시: 전환된 아티스트의 알림 ON/OFF 상태 반영

### 2.2 공지 배너

#### UI 구성

| 항목 | 내용 |
| ---- | ---- |
| 표시 조건 | BO에서 등록한 **아티스트 공지**가 있을 때만 표시. 없을 시 배너 영역 미노출 |
| 구성 | ⚠️ 아이콘 + 공지 제목 + 날짜 |
| 복수 공지 | 최신 1건만 배너에 표시. 이전 공지는 바텀시트 내 "이전 공지 보기"로 접근 |
| 위치 | 스크롤해도 항상 상단 고정 (sticky) |
| 스타일 | 피그마 참조 |

#### 동작

- 공지 배너 탭 시 공지 상세 바텀시트 표시 (공지 본문 + 날짜)
- BO에서 아티스트별 공지 등록/삭제로 관리
- **아티스트 관련 공지만** 표시 (예: "V01D 팬미팅 일정 변경", "V01D 앨범 발매 연기")
- 서비스(플랫폼) 공지 (예: 서버 점검, 앱 업데이트)는 이 화면에 미표시

> **공지 노출 위치 정리**
> - 정보 피드(INF-101): **아티스트 공지만** — 해당 아티스트 탭 진입 시 즉시 인지용 상단 배너
> - 알림센터(CEB-NTF-510): **아티스트 공지(팔로우 기준) + 서비스 공지 모두** — 전체 알림 통합

#### 상태별 표시

| 상태 | 처리 |
| ---- | ---- |
| 공지 0건 | 해당 아티스트에 등록된 공지 없음: 공지 배너 영역 미노출 |

### 2.3 통합 타임라인

#### UI 구성

##### 날짜 그룹 구분선

| 항목 | 내용 |
| ---- | ---- |
| 그룹 종류 | "오늘 {MM.DD}" / "이번 주" / "지난 주" / "이전" |
| 스타일 | 구분선 + 그룹 라벨 (좌측 정렬, 회색 텍스트). 오늘 그룹은 하이라이트 배경 (피그마 참조) |
| 빈 그룹 | 해당 그룹에 항목이 없으면 구분선 자체 미표시 |

##### 일정 항목 (📅)

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 📅 아이콘 + 시간/날짜 + 일정명 + 장소(선택) |
| 선공개 뱃지 | BO에서 "선공개" 플래그가 설정된 일정에 "⭐ CELEBUS 단독" 뱃지 표시 |
| 알림 | 개별 일정 알림 없음. **아티스트별 알림**은 헤더 🔔 토글로 제어 |
| 탭 동작 | 일정 상세 바텀시트 표시 (상세 설명 + 장소 + 시간 + 관련 링크 + [기억 남기기] CTA) |
| [기억 남기기] CTA | 바텀시트 하단에 표시. 탭 시 CEB-MEM-201 기억 작성 화면으로 이동. 일정 정보(날짜/장소)가 자동 프리필됨 |
| 스타일 | 피그마 참조 |

##### 소식 항목 (📰)

| 항목 | 내용 |
| ---- | ---- |
| 구성 | 📰 아이콘 + 헤드라인 + 썸네일 이미지(선택) + 날짜 |
| 이미지 | BO에서 등록한 경우만 표시. 미등록 시 텍스트만 |
| 선공개 뱃지 | BO에서 "선공개" 플래그가 설정된 소식에 "[CELEBUS 단독]" 헤드라인 앞 태그 표시 |
| 탭 동작 | 소식 상세 화면 이동 (풀 텍스트 + 이미지 갤러리 + 게시일 + 공유 버튼) |
| 공유 버튼 | 소식 상세 화면에서 SNS 공유 시트 표시 (OS 기본 공유) |
| 스타일 | 피그마 참조 |

#### 동작

- 소식과 일정을 날짜 기준으로 통합 정렬하여 하나의 타임라인으로 표시
- 일정: 해당 날짜/시간 기준으로 배치
- 소식: 게시일 기준으로 배치
- 동일 날짜 내 정렬 기준: 일정(시간순 오름차순) → 소식(최신순)

##### 타임라인 정렬 규칙

| 규칙 | 설명 |
| ---- | ---- |
| 그룹 순서 | 오늘 → 이번 주 → 지난 주 → 이전 |
| 그룹 내 정렬 | 일정(시간순 오름차순) → 소식(최신순) |
| 빈 그룹 | 해당 그룹에 항목 없으면 구분선 미표시 |
| 과거 일정 표시 범위 | 지난 주까지 표시. "이전" 그룹은 소식만 표시 (지나간 일정 미표시) |
| 미래 일정 | "이번 주" 이후의 미래 일정도 모두 표시. 날짜 구분선에 날짜 직접 표기 (예: "4/25 금요일") |
| 미래 일정 그룹 | "다음 주" / "이후" 그룹 추가. 등록된 미래 일정 전체 노출 |

##### 날짜 그룹 기준

| 그룹명 | 기준 |
| ---- | ---- |
| 오늘 | 서버 기준 오늘 날짜 (KST 00:00:00 ~ 23:59:59) |
| 이번 주 | 오늘을 제외한 이번 주 월~일 (미래 포함) |
| 다음 주 | 다음 주 월~일 |
| 이후 | 다음 주 이후의 미래 일정 (날짜별 구분선에 날짜 직접 표기) |
| 지난 주 | 지난 주 월~일 |
| 이전 | 지난 주 이전 (소식만 표시) |

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| Empty | 소식/일정 모두 없을 때 "V01D의 첫 소식이 곧 올 거예요!" + [알림 설정] CTA + [V01D 챌린지 시작하기] CTA → CEB-FQ-101 |
| 일정만 Empty | 오늘 그룹에 일정이 없을 때 "등록된 일정이 없습니다. 다음 일정: {날짜} {일정명}" 표시 |

### 2.4 아티스트별 알림

> 개별 일정/소식별 알림이 아닌, **아티스트 단위** 알림 ON/OFF 방식입니다. 팔로우한 아티스트마다 독립적으로 설정됩니다.

#### 알림 규칙

| 규칙 | 설명 |
| ---- | ---- |
| 토글 위치 | 헤더 우측 🔔 버튼 |
| 범위 | ON 시 **해당 아티스트의** 모든 새 일정/소식에 푸시 발송 (다른 아티스트에 영향 없음) |
| 발송 시점 (일정) | D-1 20:00 KST, 당일 00:00 KST |
| 발송 시점 (소식) | 새 소식 등록 즉시 |
| 푸시 내용 (D-1) | "내일 {아티스트명} {일정명}이 있어요!" |
| 푸시 내용 (당일) | "오늘 {아티스트명} {일정명}!" |
| 푸시 내용 (소식) | "{아티스트명}의 새 소식이 올라왔어요!" |
| 권한 미허용 | "알림 권한을 허용해주세요" 안내 후 OS 알림 설정 화면으로 이동 |

##### 선공개 뱃지 규칙

| 규칙 | 설명 |
| ---- | ---- |
| 대상 | BO에서 "선공개" 플래그가 설정된 일정 또는 소식 |
| 일정 표시 | "⭐ CELEBUS 단독" 뱃지를 일정 항목 내 별도 행에 표시 |
| 소식 표시 | "[CELEBUS 단독]" 태그를 헤드라인 앞에 인라인으로 표시 |
| 목적 | 타 플랫폼 대비 차별점 시각화 (해당 플랫폼에서 먼저 공개) |

#### 비로그인

- 비로그인도 타임라인 전체 열람 가능
- [알림 설정] CTA 탭 시 → 로그인 화면 이동 (공통정책 10.5 참조)

#### 에러/예외

| 상태 | 처리 |
| ---- | ---- |
| 알림 권한 미허용 | 🔔 탭 시 알림 권한 미허용 상태: "알림 권한을 허용해주세요" 안내 + OS 설정 이동 |

## 3. 공통 규칙

### 3.1 데이터 갱신

| 트리거 | 동작 |
| ---- | ---- |
| 화면 진입 | 타임라인 데이터 + 공지 조회 |
| Pull-to-Refresh | 전체 데이터 재조회 (공통정책 11.2 참조) |
| 복귀 시 | 상태 재조회 (알림 ON/OFF 상태 반영) |

### 3.2 멀티 아티스트 대응

- 정보 피드는 아티스트별로 분리하여 표시 (artist_id 기반 조회)
- 아티스트 전환 → CEB-ART-101 재진입 → 해당 아티스트의 정보 피드 표시

### 3.3 비로그인 공통

> 아래는 화면 전체에 적용되는 공통 원칙입니다. 각 영역(2.1~2.5)의 "비로그인" 섹션에는 해당 영역 고유의 구체적 동작만 기술합니다.

- 비로그인도 열람 가능. 타임라인 전체 표시
- 비로그인 CTA 탭 → 로그인 화면 이동 (공통정책 10.5 참조)

### 3.4 공통 에러 상태

| 상태 | 처리 |
| ---- | ---- |
| Loading | 타임라인 영역 Skeleton UI 표시 |
| Default | 통합 타임라인 정상 표시 |
| Error | 전체 조회 실패 시 에러 안내 메시지 + 재시도 버튼 표시 |
