# CELEBUS APP - 게임존 화면 구조 설계

---

## 1. 설계 배경 및 목적

### 서비스 개요

CELEBUS는 K-pop 팬 엔터테인먼트 플랫폼이며, **게임존(Game Zone)**은 GP(Game Point)를 활용하여 게임에 참여하고 보상을 획득하는 모바일 앱 기능입니다.

### 게임 유형

| 게임 유형 | 설명 | Phase |
| --------- | ---- | ----- |
| Prediction Market | YES/NO 예측에 참여하고 결과에 따라 GP를 획득하는 참여형 게임 | Phase 1 |
| Survival Trivia | 실시간 생존형 퀴즈 게임 (10문제, 4지선다, 10초 제한) | Phase 2 |

### 본 문서의 목적

- **디자이너**: 게임존 전체 화면 구조와 플로우를 파악하여 일관된 UX/UI를 설계
- **개발자**: 화면 간 네비게이션 관계, 상태 관리, 데이터 흐름을 이해하여 개발 구조 설계

### 관련 문서

| 문서 | 설명 |
| ---- | ---- |
| APP-GAM 화면설명서 (20개) | 각 화면의 상세 기획 (UI, 기능, 규칙, 에러 처리) |
| CELEBUS ADMIN - 게임존 메뉴 구조 설계 | 백오피스 측 구조설계 문서 |
| BO-GAM 화면설명서 (23개) | 백오피스 각 화면의 상세 기획 |

## 2. 설계 원칙

| 원칙 | 설명 |
| ---- | ---- |
| 실시간 인터랙션 우선 | Survival Trivia는 라이브 퀴즈 + 채팅, Prediction Market은 실시간 투표 비율 표시 |
| GP 경제 투명성 | 모든 화면에서 GP 잔액 표시, 참여 비용/보상 명확 안내 |
| 게임 상태별 UI 자동 전환 | 서버 상태 변경에 따라 버튼, 카드, 화면이 자동으로 전환 |
| 에러 안전성 | GP 트랜잭션은 서버에서 검증, 실패 시 롤백 보장 |
| 다국어 지원 | KO/EN/JP 3개 언어, 브랜드명(Survival Trivia, Prediction Market)은 영문 고정 |
| 다크/라이트 테마 | 시스템 설정 또는 앱 내 설정에 따라 자동 전환 |

## 7. 화면 Flow 다이어그램

### 7.1 공통 플로우 (게임존 메인 기준)

```text
[CEB-GAM-101] 게임존 메인
    │
    ├─→ Survival Trivia 카드 탭 ─→ [CEB-GAM-204] 트리비아 홈
    │
    ├─→ Prediction Market 카드 탭 ─→ [CEB-GAM-201] PM 리스트
    │
    ├─→ 유저 랭킹 카드 탭 ─→ [CEB-GAM-401] 랭킹
    │
    ├─→ GP 히스토리 카드 탭 ─→ [CEB-GAM-501] GP 히스토리
    │
    └─→ Header GP 잔액 탭 ─→ [CEB-GAM-501] GP 히스토리
```

### 7.2 Prediction Market 플로우

```text
[CEB-GAM-201] PM 리스트
    │
    └─→ 게임 카드 탭
        │
        └─→ [CEB-GAM-202] PM 상세
                │
                ├─→ [진행중 + 미참여] YES/NO 선택
                │       └─→ [CEB-GAM-202-MD-CONFIRMED] 참여확정 모달
                │               ├─→ [참여하기] 버튼
                │               │       ├─→ 성공 → 토스트 + [CEB-GAM-202-MD-COMPLETED] 부스팅 유도
                │               │       │       ├─→ [부스팅 참여] → 부스팅 반영
                │               │       │       └─→ [다음에 할게요] → 모달 닫기
                │               │       └─→ GP 부족 → [CEB-GAM-202-MD-NOBALANCE] 잔고 부족
                │               │               ├─→ [GP 교환소로 이동] → [CEB-GAM-301]
                │               │               └─→ [닫기] → 모달 닫기
                │               └─→ [취소] → 모달 닫기
                │
                ├─→ [진행중 + 참여완료] 부스팅 참여
                │       └─→ 부스팅 확인 → 부스팅 반영 → 토스트
                │
                ├─→ [결과 대기] 결과 대기 안내 표시
                │
                └─→ [결과 발표] 결과 확인
                        └─→ [결과 자세히 보기 >] → [CEB-GAM-203] 결과 상세
```

### 7.3 Survival Trivia 플로우

```text
[CEB-GAM-204] 트리비아 홈
    │
    ├─→ [입장하기 버튼] (시작 10분 전 ~ 시작 시점)
    │       ├─→ GP 부족 → [CEB-GAM-202-MD-NOBALANCE] 잔고 부족 모달
    │       └─→ 성공 → GP 차감
    │
    └─→ [CEB-GAM-205] Pre-start (대기/카운트다운)
            │
            ├─→ 카운트다운 완료 (5초) → 자동 전환
            │
            └─→ [CEB-GAM-206] 트리비아 상세 (게임 진행)
                    │
                    ├─→ [<] 뒤로가기 → [CEB-GAM-206-MD-LEAVE] 나가기 확인
                    │       ├─→ [나가기] → [CEB-GAM-204] 트리비아 홈
                    │       └─→ [계속하기] → 모달 닫기
                    │
                    ├─→ 정답 → 다음 스테이지 계속
                    │
                    ├─→ 오답 + 하트 ≥ 1 → 하트 -1, 다음 스테이지 계속
                    │
                    ├─→ 오답 + 하트 = 0 → [CEB-GAM-206-MD-ELIMINATE] 탈락 모달
                    │       ├─→ [관전하고 하트 받기] → [CEB-GAM-206-SPECTATE] 관전 모드
                    │       │       └─→ 게임 종료 → [CEB-GAM-206-RESULT] (C타입, 하트 보상)
                    │       └─→ [나가기] → [CEB-GAM-204] 트리비아 홈
                    │               └─→ 게임 종료 이벤트 → [CEB-GAM-206-RESULT] (D타입, 보상 없음)
                    │
                    ├─→ 미응답 (시간 초과) → [CEB-GAM-206-MD-ELIMINATE] 탈락 모달
                    │       └─→ (위와 동일)
                    │
                    └─→ 마지막 스테이지 (10/10)
                            ├─→ 정답 (생존) → [CEB-GAM-206-RESULT] (A 또는 B타입, GP 보상)
                            └─→ 오답/미응답 (탈락) → [CEB-GAM-206-RESULT] (D타입, 보상 없음)
```

### 7.4 GP 교환소 / 히스토리 플로우

```text
[CEB-GAM-301] GP 교환소
    │
    ├─→ [GP 가져오기] (CELB→GP)
    │       ├─→ CELB 잔액 부족 → [CEB-GAM-302] CELB 입금주소 (바텀시트)
    │       └─→ 교환 실행 → 성공 토스트 / 실패 에러
    │
    ├─→ [CELB으로 보내기] (GP→CELB)
    │       └─→ 교환 실행 → 성공 토스트 / 실패 에러
    │
    └─→ [GP 히스토리] → [CEB-GAM-501]

[CEB-GAM-501] GP 히스토리
    │
    ├─→ [참여내역 탭] → ROW 탭 → [CEB-GAM-502] 참여내역 상세
    │
    └─→ [교환내역 탭] → ROW 탭 → [CEB-GAM-503] 교환내역 상세
```

## 8. 게임 유형별 특성 비교

| 항목 | Prediction Market | Survival Trivia |
| ---- | ----------------- | --------------- |
| 게임 방식 | YES/NO 이진 예측 | 4지선다 퀴즈 (10문제) |
| 참여 시점 | 진행중(Active) 동안 언제든 | 시작 10분 전 ~ 시작 시점 |
| 참여 비용 | 게임별 설정 (기본 1GP) | 게임별 설정 |
| 부스팅 | 있음 (참여 후 추가 GP 투입, 2~10배) | 없음 |
| 하트 시스템 | 없음 | 있음 (오답 시 소진, 최대 3개) |
| 정원 | 게임별 설정 (무제한 가능) | 최대 500명 |
| 진행 시간 | 투표 종료일시까지 (수일~수주) | 실시간 (약 5~10분) |
| 결과 결정 | 관리자가 결과 입력 (YES/NO) | 서버 자동 판정 (정답/오답) |
| 보상 방식 | 정답자에게 비율 배분 (참여+부스팅 가중) | 최종 생존자에게 균등 분배 |
| 참여GP 환급 | 정답/오답 무관 전액 환급 | 환급 없음 |
| 관전 | 없음 | 있음 (탈락 후 관전 → 하트 보상) |
| 채팅 | 없음 | 있음 (게임 진행 중 실시간) |
| 동시 진행 | 여러 게임 동시 진행 가능 | 한 번에 1개 게임만 진행 가능 |
| 결과 화면 | CEB-GAM-203 결과 상세 | CEB-GAM-206-RESULT 게임 결과 (A/B/C/D 4개 타입) |

### Prediction Market

| BO 상태 | 코드 | 앱 노출 | 카드 표시 (201) | 상세 화면 (202) | 사용자 액션 |

| 임시저장 | Draft | 노출 안됨 | - | - | - |

| 결과 대기 | Pending | 노출됨 | "발표대기중" 배지 | 결과 대기 안내 표시 | 조회만 |

| 임시저장 | Draft | 노출 안됨 | - | - | - |
| 게시대기 | Ready | 노출됨 | 일정 표시 (날짜/시간) | 게임 정보 표시, 버튼 비활성화 | 조회만 |
| 진행중 | Active | 노출됨 | LIVE 배지 + 진행중 표시 | 입장 가능 (시작 10분 전~시작) | 입장 |
| 결과 확정 | Closed | - | - | - | - |
| 종료 | Ended | - | "다음 라이브 일정 준비중" | 다음 게임 정보 또는 일정 없음 안내 | - |

> Survival Trivia는 실시간 진행이므로 결과 대기(Pending) / 결과 확정(Closed) 상태가 사용자에게 별도 노출되지 않습니다. 게임 종료 즉시 결과 화면(CEB-GAM-206-RESULT)으로 전환됩니다.

## 10. GP 시스템

### GP 변동 유형

| 유형 | 코드 | 설명 | GP 변동 | 적용 게임 |
| ---- | ---- | ---- | ------- | --------- |
| 참여 | PARTICIPATION | 게임 참여 시 | - (차감) | PM, ST |
| 부스팅 | BOOSTING | 부스팅 사용 시 | - (차감) | PM |
| 환급 | REFUND | 결과 발표 후 참여 GP 반환 | + (증가) | PM |
| 보상 | REWARD | 정답/생존 시 보상 지급 | + (증가) | PM, ST |
| 환불 | REFUND_CANCEL | 게임 취소 시 환불 | + (증가) | PM, ST |
| GP 가져오기 | EXCHANGE_IN | CELB → GP 교환 | + (증가) | - |
| CELB으로 보내기 | EXCHANGE_OUT | GP → CELB 교환 | - (차감) | - |

### 보상 계산 로직

#### Prediction Market

```text
정답자 보상 = 총 상금 GP × (개인 배분 값 / 전체 합)
기본: 개인 배분 값 = 참여 GP
- 부스팅 구매시: 개인 배분 값 = 참여 GP × 부스팅 배수
전체 합 = ∑(각 참여자의 배분 값)
```

- 참여 GP는 정답/오답 무관 **전액 환급**
- 부스팅 GP는 환급 없음 (배분 가중치로만 사용)
- 부스팅 배수: 게임별 설정 (기본 2배, 최대 10배)
- 소수점 8자리 이하 절삭, 단수는 시스템 귀속

#### Survival Trivia

```text
생존자 개인 보상 = floor(총 상금 GP / 최종 생존자 수)
```

- 총 상금 GP를 최종 생존자에게 **균등 분배**
- 소수점 이하 절사 (나머지는 플랫폼 귀속)
- 참여 GP는 환급 없음 (입장 비용)
- 부스팅 없음

| 항목 | 규칙 |
| ---- | ---- |
| beforeunload | 게임 진행 중 브라우저 기본 경고 표시 |
| pagehide | 서버에 이탈 알림 전송 시도 |
| 서버 측 | WebSocket 끊김 후 30초 내 재연결 없으면 이탈 처리 |

### 인증 만료 처리 (공통)

| 항목 | 규칙 |
| ---- | ---- |
| 감지 | API 응답 401 Unauthorized 또는 WebSocket 인증 실패 |
| 처리 | 토큰 자동 갱신 시도 (Refresh Token). 성공 시 원래 요청 재시도. 실패 시 로그인 화면으로 리다이렉트 |
| 게임 진행 중 (206/SPECTATE) | 토큰 갱신 실패 시 "세션이 만료되었습니다. 다시 로그인해주세요." 안내 모달 → 로그인 화면 이동. 게임 진행 데이터는 서버에 유지 (재로그인 후 세션 복구 가능) |
| GP 트랜잭션 중 | 인증 만료로 트랜잭션 실패 시 서버에서 롤백. 재로그인 후 재시도 |
