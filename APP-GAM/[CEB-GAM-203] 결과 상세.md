## 1. 화면 목적

Prediction Market 게임의 최종 결과를 상세하게 조회하는 화면입니다. 게임의 YES/NO 결과, 결과 제목, 결과 설명, 판정 기준, 공식 소스 링크 등을 사용자에게 명확하게 전달합니다.

### 주요 기능
- 게임 결과 (YES/NO) 표시
- 결과 제목 및 상세 설명 표시
- 판정 기준일시 및 기준 표시
- 외부 공식 소스 링크 제공
- 결과 확인 완료 안내 배너 표시

### 접근 경로
- Prediction Market 상세(CEB-GAM-202) > 결과발표 상태 > [결과 자세히 보기 >] 버튼 탭

### 결과 Badge 색상

| 결과 | Badge 색상 | Badge 텍스트 |
| ---- | ---------- | ------------ |
| YES | 파란색 | YES |
| NO | 빨간색 | NO |

### 캐릭터 일러스트

##### 구성
- YES/NO 칠판 앞 별 캐릭터 (CELEBUS 마스코트)
- 칠판에 "YES | NO" 텍스트 표시
- 정답 쪽에 초록색 체크(✓) 표시
- 오답 쪽에 빨간색 X(✗) 표시

##### 예시
```
결과가 NO인 경우:
  ✗ YES | NO ✓
     [별 캐릭터]

결과가 YES인 경우:
  ✓ YES | NO ✗
     [별 캐릭터]
```

> 상세 스타일 및 일러스트는 피그마 참조

## 3. 화면 내 기능설명

### 뒤로가기

##### 트리거
- Header 좌측 [<] 버튼 탭

##### 동작
1. Prediction Market 상세(CEB-GAM-202) 화면으로 복귀
2. 네비게이션 스택에서 현재 화면 제거
3. 상세 화면의 결과/보상 탭 영역으로 자동 스크롤

### 결과 Badge

##### 표시 규칙
- 라벨: "결과:" (회색 텍스트)
- Badge: [YES] 또는 [NO] (파란색 또는 빨간색)
- 위치: Content 최상단
- 정렬: 좌측 정렬

### 결과 제목

##### 표시 규칙
- 타이포그래피: H2 (Bold, 큰 크기)
- 다국어 지원: KO/EN/JP (사용자 언어 설정에 따라 표시)
- 최대 줄 수: 3줄
- 초과 처리: 말줄임(...) 처리
- 예시:
  - "지드래곤은 MAMA 대상을 수상했어요" (YES 결과)
  - "지드래곤은 MAMA 대상을 수상하지 못했어요" (NO 결과)

### 결과 설명

##### 표시 규칙
- 타이포그래피: Body Text (Regular, 중간 크기)
- 다국어 지원: KO/EN/JP (사용자 언어 설정에 따라 표시)
- Rich Text 지원: 볼드, 이탤릭, 링크 등
- 최대 줄 수: 제한 없음 (세로 스크롤)
- 예시: "2024 MAMA AWARDS에서 지드래곤은 베스트 남자 아티스트 부문 후보로 올랐지만, 대상은 다른 아티스트에게 수여되었습니다."

### 기준일시

##### 표시 규칙
- 아이콘: 📅 (캘린더)
- 형식: YYYY. MM. DD · HH:mm (KST, UTC+9)
- 예시: "2026. 02. 22 · 12:30 (KST, UTC+9)"
- 타임존: 항상 KST 기준 표시
- 색상: 회색 (보조 정보)

### 판정 기준

##### 표시 규칙
- 아이콘: 🏛 (건물/기관)
- 텍스트: 판정 기준 설명 (다국어 지원)
- 외부링크 아이콘: → (우측)
- 예시: "MAMA 공식 시상 결과"

##### 동작
- 판정 기준 영역 탭 시 외부 URL로 이동
- 브라우저 또는 인앱 웹뷰로 열기
- URL이 없는 경우 탭 불가 (비활성화)

### 안내 배너

##### 표시 규칙
- 배경색: 초록색 (연한 초록 배경)
- 아이콘: ✅ (체크)
- 텍스트: "공식 소스를 통해 결과가 확인되었습니다" (고정)
- 위치: Content 하단 (고정)
- 항상 표시 (조건 없음)

## 4. 규칙/로직

### 데이터 매핑

##### 백오피스 → 앱 매핑

| 백오피스 필드 (CEB-BO-GZ-202) | 앱 화면 표시 |
| ---- | ---- |
| 결과 (YES/NO) | 결과 Badge |
| 결과제목 (KO/EN/JP) | 결과 제목 (H2) |
| 결과설명 (KO/EN/JP) | 결과 설명 (Body Text) |
| 결과링크 (URL) | 판정 기준 외부링크 |
| 결과링크 텍스트 (KO/EN/JP) | 판정 기준 텍스트 |
| 결과 입력일 | 기준일시 (YYYY. MM. DD · HH:mm) |

### 다국어 처리

##### 지원 언어
- 한국어 (KO)
- 영어 (EN)
- 일본어 (JP)

##### 적용 대상
- 결과 제목
- 결과 설명
- 판정 기준 텍스트

##### 표시 규칙
- 사용자 앱 언어 설정에 따라 자동 선택
- 해당 언어 데이터가 없는 경우 기본 언어(KO) 표시
- 번역되지 않은 고유명사는 원문 유지

### Rich Text 지원

##### 지원 태그
- `<b>` 또는 `<strong>`: 볼드
- `<i>` 또는 `<em>`: 이탤릭
- `<a href="">`: 링크 (인라인)
- `<br>`: 줄바꿈

##### 표시 규칙
- 링크는 파란색 밑줄 표시
- 링크 탭 시 인앱 브라우저로 열기
- 이미지, 비디오 등 미지원 태그는 무시

### 캐릭터 일러스트 표시 규칙

##### 정답/오답 표시
- 정답 쪽: 초록색 체크(✓) 표시
- 오답 쪽: 빨간색 X(✗) 표시

##### 예시
- 결과가 YES: YES 쪽에 ✓, NO 쪽에 ✗
- 결과가 NO: NO 쪽에 ✓, YES 쪽에 ✗

### 외부 링크 처리

##### 열기 방식
- 인앱 브라우저 (WebView) 또는 외부 브라우저
- iOS: SFSafariViewController
- Android: Custom Tabs

##### URL 검증
- http:// 또는 https://로 시작하는 유효한 URL만 허용
- 잘못된 URL인 경우 에러 토스트 표시: "링크를 열 수 없습니다."

### 화면 진입 시 동작

##### 데이터 로딩
1. 게임 ID를 파라미터로 전달받음
2. 서버에서 게임 결과 정보 조회
3. Loading 상태 표시 (Skeleton UI)
4. 데이터 로딩 완료 후 화면 렌더링

##### 결과 확인 처리
- 화면 진입 시 서버에 "결과 확인 완료" 이벤트 전송
- 리스트 화면(CEB-GAM-201)의 "결과 발표(N)" 배지 카운트 감소 (N-1)
- 같은 게임의 결과를 다시 확인해도 카운트 변화 없음

## 5. 에러 및 예외상황

### 데이터 로딩 실패

##### 상황
- API 호출 실패 또는 네트워크 오류

##### 처리
1. Error 상태 표시
2. 에러 메시지: "결과 정보를 불러오는 데 실패했습니다."
3. [다시 시도] 버튼 제공
4. 버튼 탭 시 데이터 재로딩

### 결과 정보 없음

##### 상황
- 게임은 존재하지만 결과가 아직 입력되지 않은 경우

##### 처리
1. Error 상태 표시
2. 에러 메시지: "아직 결과가 발표되지 않았습니다."
3. [뒤로가기] 버튼 제공
4. 버튼 탭 시 PM 상세(CEB-GAM-202) 화면으로 복귀

### 게임이 존재하지 않음

##### 상황
- 잘못된 게임 ID 또는 삭제된 게임

##### 처리
1. Error 상태 표시
2. 에러 메시지: "존재하지 않는 게임입니다."
3. 에러 토스트 표시 후 자동으로 리스트 화면으로 이동 (2초 후)

### Loading 상태

##### 상황
- 데이터 로딩 중

##### 처리
1. Skeleton UI 표시
   - 결과 Badge: 회색 박스
   - 결과 제목: 회색 박스 (3줄)
   - 캐릭터 일러스트: 회색 사각형
   - 결과 설명: 회색 박스 (5줄)
   - 기준일시: 회색 박스 (1줄)
   - 판정 기준: 회색 박스 (1줄)
2. 로딩 완료 시 실제 데이터로 전환 (페이드인 애니메이션)

### 외부 링크 오류

##### 상황
- 판정 기준 외부링크 탭 시 URL이 유효하지 않거나 열 수 없는 경우

##### 처리
1. 에러 토스트: "링크를 열 수 없습니다."
2. 화면 유지 (이동하지 않음)

### 네트워크 연결 끊김

##### 상황
- 오프라인 상태에서 화면 진입 또는 데이터 로딩 시도

##### 처리
1. Error 상태 표시
2. 에러 메시지: "네트워크 연결을 확인해주세요."
3. [다시 시도] 버튼 제공
4. 온라인 복구 시 자동 재로딩
