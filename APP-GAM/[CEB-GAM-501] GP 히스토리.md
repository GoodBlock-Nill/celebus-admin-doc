## 1. 화면 목적

사용자 본인의 모든 GP 변동 내역을 조회하고 추적할 수 있는 화면입니다. 게임 참여/부스팅/보상/환급/환불과 GP 교환(충전/출금) 내역을 통합 조회하며, 각 내역의 상세 정보를 확인할 수 있습니다.

### 주요 기능

- 게임 참여 내역 조회 (참여/부스팅/보상/환급/환불)
- GP 교환 내역 조회 (충전/출금)
- 날짜별 그룹핑 및 변동 유형별 필터링
- 참여내역 상세 및 교환내역 상세 화면 진입

### 접근 경로

```text
Game 탭 > 게임존 메인 > GP 히스토리
```

## 3. 화면 내 기능설명

### 3.1 참여내역 탭

#### 변동 유형 정의

| 유형 | 코드 | 아이콘 | GP 변동 | Badge 색상 |
| ---- | ---- | ---- | ---- | ---- |
| 참여 | PARTICIPATION | ⚡ (파란) | - (차감) | - |
| 부스팅 | BOOSTING | ⚡ (파란) | - (차감) | - |
| 보상 | REWARD | ⚡ (파란) | + (증가) | - |
| 환급 | REFUND | ⚡ (파란) | + (증가) | - |
| 환불 | REFUND_CANCEL | ⚡ (파란) | + (증가) | - |

#### 필터 옵션

| 옵션 | 표시 내역 |
| ---- | ---- |
| 전체 | 모든 참여내역 |
| 참여 | PARTICIPATION만 |
| 부스팅 | BOOSTING만 |
| 보상 | REWARD만 |
| 환급 | REFUND만 |
| 환불 | REFUND_CANCEL만 |

#### 게임유형별 변동 유형 규칙

| 게임유형 | 발생 가능 유형 |
| ---- | ---- |
| Prediction Market | PARTICIPATION, BOOSTING, REWARD, REFUND, REFUND_CANCEL |
| Survival Trivia | PARTICIPATION, REWARD, REFUND_CANCEL |

> Survival Trivia는 부스팅(BOOSTING) 기능을 지원하지 않으므로 BOOSTING, REFUND 유형이 발생하지 않습니다.

#### 내역 아이템 구성

**A. 참여/부스팅**

```
⚡ [참여] 게임 타이틀              - {GP} >
    HH:mm (UTC+9)        {게임유형}
```

- 1행: 아이콘 + [유형] + 게임 타이틀 (최대 1줄, 말줄임) + GP 변동량(빨간)
- 2행: 시간 + 게임유형: Prediction Market 또는 Survival Trivia (영문 고정)

**B. 보상**

```
⚡ [보상] 게임 타이틀 (길면 말줄임...)
    + {GP} >
    HH:mm (UTC+9)        {게임유형}
```

- 1행: 아이콘 + [유형] + 게임 타이틀 (최대 1줄, 말줄임)
- 2행: GP 변동량(파란)
- 3행: 시간 + 게임유형: Prediction Market 또는 Survival Trivia (영문 고정)

> 보상 금액이 크기 때문에 2행 레이아웃 사용
> 보상 = 상금풀내에서 받은 GP (돌려받는 GP는 제외)

**C. 환급**

```
⚡ [환급] 게임 타이틀              + {GP} >
    HH:mm (UTC+9)        {게임유형}
```

- 1행: 아이콘 + [환급] + 게임 타이틀 (최대 1줄, 말줄임) + GP 변동량(파란)
- 2행: 시간 + 게임유형: Prediction Market 또는 Survival Trivia (영문 고정)

**D. 환불**

```
⚡ [환불] 게임 타이틀              + {GP} >
    HH:mm (UTC+9)        {게임유형}
```

- 1행: 아이콘 + [환불] + 게임 타이틀 (최대 1줄, 말줄임) + GP 변동량(파란)
- 2행: 시간 + 게임유형: Prediction Market 또는 Survival Trivia (영문 고정)

#### 정렬 기준

- 기본 정렬: 변동일시 내림차순 (최신순)
- 날짜 그룹 내에서도 최신순

#### 탭 동작

- 각 내역 아이템 탭 → CEB-GAM-502 참여내역 상세 이동
- 전달 파라미터: 변동 ID, 변동 유형

### 3.2 교환내역 탭

#### 변동 유형 정의

| 유형 | 코드 | 아이콘 | GP 변동 |
| ---- | ---- | ---- | ---- |
| GP 가져오기 (가져오기) | EXCHANGE_IN | ⬇ (파란) | + (증가) |
| CELB으로 보내기 (보내기) | EXCHANGE_OUT | ⬆ (파란) | - (차감) |

#### 필터 옵션

| 옵션 | 표시 내역 |
| ---- | ---- |
| 전체 | 모든 교환내역 |
| GP 가져오기 | EXCHANGE_IN만 |
| CELB으로 보내기 | EXCHANGE_OUT만 |

#### 내역 아이템 구성

**A. GP 가져오기 (성공)**

```
⬇ GP 가져오기                    + {GP} >
    HH:mm (UTC+9)
```

- 1행: 아이콘 + "GP 가져오기" + GP 변동량(파란)
- 2행: 시간

**B. GP 가져오기 (실패)**

```
⬇ GP 가져오기        [실패]  2,000 >
    HH:mm (UTC+9)
```

- 1행: 아이콘 + "GP 가져오기" + [실패] Badge(빨간) + 금액(취소선)
- 2행: 시간

**C. CELB으로 보내기 (성공)**

```
⬆ CELB으로 보내기                - {GP} >
    HH:mm (UTC+9)
```

- 1행: 아이콘 + "CELB으로 보내기" + GP 변동량(빨간)
- 2행: 시간

**D. CELB으로 보내기 (실패)**

```
⬆ CELB으로 보내기      [실패]  1,000 >
    HH:mm (UTC+9)
```

- 1행: 아이콘 + "CELB으로 보내기" + [실패] Badge(빨간) + 금액(취소선)
- 2행: 시간

#### 정렬 기준

- 기본 정렬: 거래일시 내림차순 (최신순)
- 날짜 그룹 내에서도 최신순

#### 탭 동작

- 각 내역 아이템 탭 → CEB-GAM-503 교환내역 상세 이동
- 전달 파라미터: 교환 ID, 교환 유형, 상태

### 3.3 날짜 그룹핑

- 형식: YYYY. MM. DD
- 그룹 헤더: 회색 텍스트 + 우측 구분선
- 동일 날짜 내역을 그룹으로 묶어 표시
- 날짜별 내림차순 (최신 날짜가 위)

### 3.4 필터 바텀시트

- 위치: 우측 상단, "전체 {건수}건" 우측의 필터 버튼 (전체 ▼)
- 트리거: 필터 버튼 탭 시 바텀시트 오픈
- 기본값: "전체"
- 참여내역 탭 옵션: 전체 / 참여 / 부스팅 / 보상 / 환급 / 환불
- 교환내역 탭 옵션: 전체 / GP 가져오기 / CELB으로 보내기
- 선택 시: 바텀시트 닫힘, 필터링된 내역만 표시, 건수 업데이트

### 3.5 GP 변동량 표시

| 변동 방향 | 표시 형식 | 색상 |
| ---- | ---- | ---- |
| 증가 (+) | + {숫자} > | 파란색 (#1E90FF) |
| 감소 (-) | - {숫자} > | 빨간색 (#FF4D4F) |

- 천 단위 콤마 적용
- 우측에 ">" 화살표 표시 (탭 가능 표시)

### 3.6 상태별 UI

| 상태 | 설명 | 표시 |
| ---- | ---- | ---- |
| Loading | 데이터 로딩 중 | 스켈레톤 UI (3개 항목) |
| Default | 데이터 정상 조회 | 내역 리스트 |
| Empty | 내역 없음 | "내역이 없습니다" 중앙 표시 |
| Error | 조회 실패 | "내역을 불러올 수 없습니다" + 재시도 버튼 |

### 3.7 사용자 시나리오

#### 시나리오 1: 게임 참여내역 확인

1. 사용자가 게임존 메인에서 "GP 히스토리" 버튼 탭
2. GP 히스토리 화면 진입 (기본: 참여내역 탭)
3. 날짜별로 그룹핑된 참여내역 확인
4. 필터 드롭다운에서 "보상" 선택
5. 보상 내역만 필터링되어 표시
6. 특정 보상 내역 탭 → CEB-GAM-502 참여내역 상세 이동

#### 시나리오 2: GP 교환내역 확인

1. 사용자가 GP 히스토리 화면에서 "교환내역" 탭 탭
2. 교환내역 리스트 표시
3. "GP 가져오기" 필터 선택
4. 충전 내역만 필터링되어 표시
5. 특정 교환 내역 탭 → CEB-GAM-503 교환내역 상세 이동

#### 시나리오 3: 실패한 교환 확인

1. 교환내역 탭에서 [실패] Badge가 있는 내역 확인
2. 금액에 취소선 표시
3. 해당 내역 탭 → CEB-GAM-503 교환내역 상세에서 실패 사유 확인

- 날짜 그룹 헤더는 회색 텍스트 + 우측 구분선

### 비즈니스 규칙

1. GP 변동 내역은 삭제할 수 없습니다 (이력 보존)
2. 실패한 교환 내역도 표시됩니다 ([실패] Badge + 금액 취소선)
3. 게임 관련 변동은 게임 타이틀과 게임 유형이 표시됩니다
4. 교환 관련 변동은 교환 방향과 금액만 표시됩니다
5. 변동일시는 GP가 실제로 증감된 시각입니다 (KST)
6. 게임 삭제 시에도 참여내역은 유지되며 게임 타이틀은 표시됩니다
7. 예측 성공한 유저는 보상 내역과 환급 내역이 각각 별도로 생성됩니다
