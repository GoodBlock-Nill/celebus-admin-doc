## 1. 화면 목적

GP 교환 내역의 상세 정보를 보여주는 화면입니다. GP 가져오기(가져오기) 및 CELB으로 보내기(보내기) 내역의 블록체인 트랜잭션 정보를 확인하고, 성공/실패 상태에 따른 상세 정보를 제공합니다.

### 주요 기능

- GP 교환 상세 정보 조회 (충전/출금)
- 블록체인 트랜잭션 정보 확인
- BscScan 트랜잭션 페이지 연결
- 교환 성공/실패 상태 표시

### 진입 경로

```text
CEB-GAM-501 GP 히스토리 > 교환내역 탭 > 내역 항목 탭
```

### 영역별 구성요소

#### Header 영역

- 좌측: < 뒤로가기 버튼 (GP 히스토리로 복귀)
- 중앙: "{유형} 상세" 타이틀
  - 가져오기 상세 (EXCHANGE_IN)
  - 보내기 상세 (EXCHANGE_OUT)

#### 상단 카드 영역

- 상태 텍스트 (가져오기 완료 / 가져오기 실패 / 보내기 완료 / 보내기 실패)
- GP 변동량 또는 금액 (성공: +/- 표시, 실패: 취소선)

#### 구분선

- 그라데이션 또는 단순 구분선

#### 상세 정보 영역

2컬럼 레이아웃 (Label : Value)

- 라벨: 왼쪽 정렬, 회색 텍스트
- 값: 오른쪽 정렬, 검정 텍스트 (mono 폰트 적용)

#### CTA 버튼

- [BscScan에서 보기 ↗] 버튼
- 그라데이션 배경 (핑크/보라)
- 외부 링크 아이콘 (↗)

> 상세 스타일은 피그마 참조

### 3.2 GP 변동량 및 금액 표시

#### 성공 시

| 교환 유형 | 표시 형식 | 색상 |
| ---- | ---- | ---- |
| 가져오기 (가져오기) | + {숫자} GP | 파란색 (#1E90FF) |
| 보내기 (보내기) | - {숫자} GP | 빨간색 (#FF4D4F) |

#### 실패 시

| 교환 유형 | 표시 형식 | 색상 |
| ---- | ---- | ---- |
| 가져오기 (가져오기) | {숫자} GP (취소선) | 회색 (#999999) |
| 보내기 (보내기) | {숫자} GP (취소선) | 회색 (#999999) |

- 천 단위 콤마 적용
- 폰트 크기: 본문보다 큰 사이즈

### 3.3 지갑주소 및 트랜잭션 해시 표시

#### 형식

- 전체 길이: 0x로 시작하는 40자리 16진수 (주소) 또는 64자리 (해시)
- 표시 형식: 축약 (0x + 앞 8자리 + ... + 뒤 8자리)
- 예시: 0x6B175474...95271d0F
- 폰트: mono 폰트 (고정폭)

#### 복사 버튼 (📋)

- 위치: 주소/해시 우측
- 아이콘: 📋 (클립보드)
- 탭 동작: 전체 주소/해시를 클립보드에 복사
- 복사 완료 시: Toast "복사되었습니다" (1초 표시)

### 3.4 네트워크 표시

- 고정값: "BSC (BNB Smart Chain)"
- Phase 1에서는 BSC만 지원
- Phase 2에서 다른 네트워크 추가 시 동적 표시

### 3.5 BscScan에서 보기 버튼

- 위치: 화면 하단
- 스타일: 그라데이션 배경 (핑크 → 보라)
- 라벨: "BscScan에서 보기 ↗"
- 탭 동작: 외부 브라우저에서 BscScan 트랜잭션 페이지 열기
- 링크 형식: `https://bscscan.com/tx/{txid}`
- 전체 트랜잭션 해시를 URL에 사용

### 3.6 뒤로가기 동작

- Header의 < 버튼 탭 → CEB-GAM-501 GP 히스토리로 복귀
- 복귀 시: 교환내역 탭 유지, 스크롤 위치 복원

### 3.7 사용자 시나리오

#### 시나리오 1: GP 가져오기 성공 상세 확인

1. GP 히스토리 > 교환내역 탭에서 "⬇ GP 가져오기" (성공) 내역 탭
2. 가져오기 상세 화면 진입
3. Header: "가져오기 상세"
4. 상단 카드: "가져오기 완료" + "+ 1,000 GP" (파란색)
5. 거래일시: 2026.01.01. 09:53:32 (UTC+9)
6. 보낸 주소: 0x6B175474...95271d0F 📋
7. 네트워크: BSC (BNB Smart Chain)
8. 트랜잭션 해시: 0x6b26e0a3...a367e8d8 📋
9. [BscScan에서 보기 ↗] 버튼 표시

#### 시나리오 2: GP 가져오기 실패 상세 확인

1. GP 히스토리 > 교환내역 탭에서 "⬇ GP 가져오기 [실패]" 내역 탭
2. 가져오기 상세 화면 진입
3. Header: "가져오기 상세"
4. 상단 카드: "가져오기 실패" + "2,000 GP" (회색, 취소선)
5. 거래일시: 2026.01.05. 09:12:30 (UTC+9)
6. 보낸 주소, 네트워크, 트랜잭션 해시 확인
7. [BscScan에서 보기 ↗] 버튼 표시 (실패 사유 확인 가능)

#### 시나리오 3: CELB으로 보내기 성공 상세 확인

1. GP 히스토리 > 교환내역 탭에서 "⬆ CELB으로 보내기" (성공) 내역 탭
2. 보내기 상세 화면 진입
3. Header: "보내기 상세"
4. 상단 카드: "보내기 완료" + "- 1,000 GP" (빨간색)
5. 거래일시: 2026.01.05. 14:22:45 (UTC+9)
6. 받는 주소: 0x7A3BCD12...4F9E6A21 📋
7. 네트워크: BSC (BNB Smart Chain)
8. 트랜잭션 해시: 0x8d5c2b1a...f3e7c9d4 📋

#### 시나리오 4: 지갑주소 복사

1. 교환내역 상세 화면에서 "보낸 주소" 또는 "받는 주소" 필드의 📋 버튼 탭
2. 전체 지갑주소가 클립보드에 복사됨
3. Toast "복사되었습니다" 표시 (1초)
4. 복사된 주소를 외부에서 사용 가능

#### 시나리오 5: BscScan에서 트랜잭션 확인

1. 교환내역 상세 화면에서 [BscScan에서 보기 ↗] 버튼 탭
2. 외부 브라우저에서 BscScan 트랜잭션 페이지 열림
3. 블록체인 상의 실제 트랜잭션 정보 확인
  - 블록 번호, 가스비, 컨펌 수, 실패 사유(실패 시) 등
4. 브라우저 뒤로가기로 앱으로 복귀

#### 시나리오 6: 트랜잭션 해시 복사

1. 교환내역 상세 화면에서 "트랜잭션 해시" 필드의 📋 버튼 탭
2. 전체 트랜잭션 해시가 클립보드에 복사됨
3. Toast "복사되었습니다" 표시 (1초)
4. 복사된 해시를 외부 블록체인 탐색기에서 조회 가능

### 표시 규칙

1. 지갑주소 및 트랜잭션 해시: 축약 형태 표시, mono 폰트
2. GP 변동량: 성공 시 +/- 색상 적용, 실패 시 취소선 + 회색
3. 복사 버튼: 📋 아이콘, 탭 시 전체 값 복사
4. BscScan 버튼: 항상 표시 (성공/실패 무관)
5. 네트워크: Phase 1에서는 "BSC (BNB Smart Chain)" 고정

### 상태 매핑

| 교환 유형 | 성공 상태 | 실패 상태 |
| ---- | ---- | ---- |
| EXCHANGE_IN (가져오기) | 가져오기 완료 | 가져오기 실패 |
| EXCHANGE_OUT (보내기) | 보내기 완료 | 보내기 실패 |

### 주소 필드명

| 교환 유형 | 필드명 |
| ---- | ---- |
| EXCHANGE_IN (가져오기) | 보낸 주소 |
| EXCHANGE_OUT (보내기) | 받는 주소 |

### 복사 기능

- 복사 대상: 지갑주소 전체, 트랜잭션 해시 전체
- 복사 완료 Toast: "복사되었습니다" (1초 표시)
- 복사 실패 시: Toast "복사할 수 없습니다" (1초 표시)

### BscScan 링크

- URL 형식: `https://bscscan.com/tx/{txid}`
- 열기 방식: 외부 브라우저 (인앱 브라우저 아님)
- 전체 트랜잭션 해시를 URL에 사용 (축약 아님)

### 성능 요구사항

| 항목 | 요구사항 |
| ---- | ---- |
| 화면 로딩 시간 | 1초 이내 |
| 복사 버튼 반응 시간 | 즉시 (0.1초 이내) |
| BscScan 버튼 탭 반응 시간 | 0.3초 이내 |

### 접근성

| 항목 | 요구사항 |
| ---- | ---- |
| 터치 영역 | 최소 44x44pt (복사 버튼, BscScan 버튼) |
| 색상 대비 | WCAG AA 준수 (취소선 회색도 대비 확보) |
| 포커스 표시 | 탭 가능 영역 하이라이트 |

### 비즈니스 규칙

1. 교환내역 상세는 수정할 수 없습니다 (조회 전용)
2. 실패한 교환도 트랜잭션 해시가 존재할 수 있습니다 (블록체인 상 실패)
3. BscScan 버튼은 성공/실패 무관하게 항상 표시됩니다
4. 트랜잭션 해시는 블록체인 상의 고유 식별자이며 변경 불가합니다
5. 지갑주소는 BSC 메인넷 주소이며 0x로 시작합니다
6. 복사 기능은 전체 값을 복사하며 축약형이 아닙니다

## 5. 에러 및 예외상황

### 5.1 데이터 검증

| 검증 항목 | 검증 규칙 | 오류 처리 |
| ---- | ---- | ---- |
| 교환 ID 존재 | 유효한 교환 ID | Toast: "교환 정보를 찾을 수 없습니다." + 뒤로가기 |
| 교환 유형 | EXCHANGE_IN/EXCHANGE_OUT | Toast: "잘못된 교환 유형입니다." |
| 트랜잭션 해시 | 0x로 시작하는 64자리 16진수 | Toast: "트랜잭션 정보가 없습니다." |
| 지갑주소 | 0x로 시작하는 40자리 16진수 | 기본값: "-" |
| GP 변동량 | 숫자 타입 | 기본값: 0 |
| 거래일시 | 유효한 날짜/시간 | 기본값: "-" |
