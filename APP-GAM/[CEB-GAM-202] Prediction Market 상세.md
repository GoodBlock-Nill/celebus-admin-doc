## 1. 화면 목적

Prediction Market 게임의 상세 정보를 조회하고, 게임 상태와 유저 참여 상태의 조합에 따라 적절한 UI와 인터랙션을 제공하는 화면입니다.

##### 주요 기능
- 게임 기본정보 조회 (타이틀, 상세설명, 총 상금 GP, 참여 정원)
- 게임 상태별 분기 화면 (진행중 / 발표대기 / 결과발표)
- 유저 참여 상태별 분기 화면 (미참여 / 참여완료 / 부스팅완료)
- YES/NO 투표 참여 플로우
- 투표 현황 조회 (비율 바)
- 결과 확인 및 보상 확인
- 부스팅 참여

##### 접근 경로
- PM 리스트(CEB-GAM-201) > 게임 카드 탭 시 이동
- 게임존 메인(CEB-GAM-101) > 추천 게임 카드 탭 시 이동
- 딥링크 / 공유 링크를 통한 직접 진입

### 2.2 영역별 구성요소

##### Header 영역

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | [<] 뒤로가기 버튼 (PM 리스트로 이동) |
| 우측 | [공유] 아이콘 버튼 (게임 공유 링크 복사/공유) |

##### 게임 정보 영역

| 항목 | 내용 |
| ---- | ---- |
| 게임 타이틀 | {타이틀} (다국어, 사용자 언어 설정에 따라 KO/EN/JP 중 표시) |
| 상태 배지 | 게임 상태에 따른 배지 (진행중 / 발표대기중 / 결과발표) |
| 전체상금 | "전체상금 {숫자} GP" (천 단위 콤마) |
| 참여 정원 | "선착순 {숫자} 명" (정원 제한 시), 무제한인 경우 비표시 |

##### 참여 상태 배너 (참여완료 시에만 표시)

| 조건 | 배너 내용 |
| ---- | ---- |
| 참여완료 (부스팅 미사용) | [참여완료] "내 선택 **{YES/NO}**" |
| 참여완료 + 부스팅 사용 | [참여완료] "내 선택 **{YES/NO}** . 부스팅 사용" |

> 배너 배경: 다크 톤의 카드 형태. 상세 스타일은 피그마 참조

##### 본문 영역

| 항목 | 내용 |
| ---- | ---- |
| 캐릭터 일러스트 | YES/NO 칠판 앞 별 캐릭터 이미지 (미참여 상태에서만 표시) |
| 상세설명 | 리치텍스트 (HTML 렌더링), 다국어 지원 |
| 결과일정 | "결과는 {YYYY}년 {M}월 {D}일, {결과 확인 기준} 기준으로 확정됩니다." |
| 힌트 링크 | "예측을 좀 더 잘하고 싶다면?" + [힌트 보기 >] 버튼 (힌트 링크 설정 시에만 표시) |

##### 투표 현황 영역 (참여완료 또는 발표대기/결과발표 시 표시)

| 항목 | 내용 |
| ---- | ---- |
| 섹션 타이틀 | "전체 유저 투표 현황" (진행중), "전체 유저 투표" + "투표마감" (발표대기/결과발표) |
| 업데이트 시각 | "최근 업데이트 {HH:mm} (+UTC9)" + 새로고침 아이콘 (진행중 참여완료 시) |
| 비율 바 | YES/NO 비율을 시각적 바로 표시 (파란색: YES, 빨간색: NO) |
| YES 비율 | "YES {n}%" + [내 선택] 배지 (YES 선택 시) |
| NO 비율 | "NO {n}%" + [내 선택] 배지 (NO 선택 시) |
| WIN 표시 | 결과발표 시 정답 쪽에 "WIN!" 텍스트 (빨간색) 표시 |
| 결과 자세히 보기 | [결과 자세히 보기 >] 링크 (결과발표 시에만 표시) |

##### 결과 카드 (결과발표 시에만 표시)

| 조건 | 카드 내용 |
| ---- | ---- |
| 미참여 | "참여하신 내역이 없네요" + "참여했다면, 정답 시 최대 {n} GP를 받을 수 있었어요" + [다른 예측 게임 둘러보기 >] |
| 정답 + 참여완료 (부스팅 미사용) | "예측 성공!" + "총 보상 **{n} GP** 획득" |
| 정답 + 참여완료 + 부스팅 사용 | "예측 성공!" + "총 보상 **{n} GP** 획득" + [부스팅 효과] 배지 |
| 오답 + 참여완료 | "아쉽게도 정답을 맞추지 못하셨네요" + "정답 여부와 관계없이 예측에 사용한 GP는 모두 환불돼요" + [다른 예측 게임 둘러보기 >] |
| 오답 + 참여완료 + 부스팅 사용 | 오답 + 참여완료와 동일 |

##### 부스팅 유도 카드 (진행중 + 참여완료 + 사용자 부스팅 미사용 시)

| 항목 | 내용 |
| ---- | ---- |
| 아이콘 | 타겟 아이콘 |
| 타이틀 | "부스팅 = 내 당첨 지분을 키우는 선택" |
| 설명 | "같은 정답자여도, 부스팅 여부에 따라 보상 비중이 달라져요." |

##### Prediction Market 이용 안내 (아코디언)

| 항목 | 내용 |
| ---- | ---- |
| 기본 상태 | 접힌 상태 (타이틀만 표시) |
| 타이틀 | "Prediction Market 이용 안내" + 펼치기/접기 화살표 |
| 참여 방식 | GP를 사용해 YES/NO 중 하나를 선택합니다 / 참여 완료 후에는 선택 변경이 불가능합니다 / 참여에 사용된 GP는 예측 결과와 관계없이 반환됩니다 |
| GP & 부스팅 | 참여에 사용한 GP는 결과 발표 후 반환됩니다 / 부스팅은 제시된 상금풀에서 추가 배분 값을 높이기 위한 선택 옵션입니다 / 부스팅에 사용한 GP는 반환되지 않습니다 |
| 보상 기준 | 예측이 적중하면 사전 안내된 상금풀 내에서 개인 배분 값에 따라 추가 보상이 지급됩니다 / 보상 규모는 참여 GP 및 부스팅 배수에 따라 달라질 수 있습니다 / 예측에 실패한 경우 추가 보상은 지급되지 않습니다 |
| 유의사항 | 부정 참여가 확인될 경우 보상이 제한될 수 있습니다 / 서비스 정책에 따라 보상 방식 및 기준은 변경될 수 있습니다 / 참여 후 탈퇴할 경우, 해당 참여에 사용된 GP는 소멸됩니다 |

> 상세 스타일은 피그마 참조

### 3.2 투표 참여 플로우

##### Step 1: YES/NO 선택

- 하단 고정 바의 [YES] 또는 [NO] 버튼 탭
- 하프 모달(Bottom Sheet) 표시:
  - 게임 타이틀
  - "보유 GP: {n}" (GP 아이콘 + 잔액)
  - "내 선택 **{YES/NO}** . 참여 {n} GP"
  - [취소] 버튼 (좌측), [**{YES/NO}로 참여하기**] 버튼 (우측, 파란색)

##### Step 2: 참여 확정 모달 (CEB-GAM-202-MD-CONFIRMED)

- "예측 참여를 확정할까요?"
- "선택: **{YES/NO}** / 사용 GP: **{n}**"
- "예측에 참여하면, 선택은 변경할 수 없어요. 결과에 따라 GP를 획득하거나 소모해요"
- [취소] 버튼, [참여 확정] 버튼

##### Step 3: 참여 완료 (CEB-GAM-202-MD-COMPLETED)

- 화면 중앙에 "참여가 완료되었어요" 토스트/배너 표시
- 부스팅 유도 바텀시트 자동 표시:
  - "부스팅 = 내 당첨 지분을 키우는 선택"
  - "같은 정답자여도, 부스팅 여부에 따라 보상 비중이 달라져요."
  - [나중에] 버튼 (좌측), [{n}GP로 부스팅] 버튼 (우측, 파란색)

##### Step 4: 부스팅 완료 (부스팅 선택 시)

- "부스팅이 적용되었어요! 정답자 보상 분배에서 내 비중이 높아졌어요" 토스트 표시
- 참여 배너에 "부스팅 사용" 추가 표시

### 3.3 GP 잔고 부족 처리 (CEB-GAM-202-MD-NOBALANCE)

- 참여 또는 부스팅 시 GP 잔고가 부족한 경우 모달 표시
- "**GP가 부족해요**"
- "예측에 참여하려면 {n}GP가 필요해요. GP를 충전한 뒤 다시 시도해 주세요."
- "보유 GP: {n}" (빨간색)
- [취소] 버튼, [GP 가져오기하기] 버튼
- [GP 가져오기하기] 탭 시 GP 교환소(CEB-GAM-301)로 이동

### 3.4 인원 마감 처리

- 참여 정원이 가득 찬 경우
- 참여 배너 위치에 알림 배너 표시: "선착순 인원 마감으로 투표가 종료되었어요"
- [YES] [NO] 버튼 비표시 (하단 타이머만 유지)
- 캐릭터 일러스트 및 상세설명은 그대로 표시

### 3.5 투표 카운트다운 타이머

- "투표 종료까지 [DD]일 [HH]:[mm]:[ss]"
- 실시간 카운트다운 (1초 단위 갱신)
- DD: 일, HH: 시, mm: 분, ss: 초
- 투표 종료 시간 도달 시 자동으로 발표대기 상태로 전환

### 3.6 투표 현황 갱신

- 진행중 + 참여완료 상태: "최근 업데이트 {HH:mm} (+UTC9)" + 새로고침 아이콘
- 새로고침 아이콘 탭 시 투표 비율 최신화
- 발표대기 / 결과발표 상태: 투표마감 체크 아이콘 표시, 새로고침 불가

### 3.7 결과 자세히 보기

- 결과발표 상태에서 투표 현황 아래 [결과 자세히 보기 >] 링크 표시
- 탭 시 결과 상세(CEB-GAM-203) 화면으로 이동

### 3.8 공유 기능

- Header 우측 공유 아이콘 탭
- OS 기본 공유 시트 호출 (딥링크 URL 포함)

### 3.9 축하 애니메이션

- 결과발표 시 정답 유저가 최초 진입할 때 재생
- "축하합니다!" 텍스트 + "예측에 성공하셨어요" 서브텍스트
- 별 캐릭터 (기쁜 표정) + 색종이/리본 파티클 효과
- 보상 카드에 "기본 보상 {n} GP" + "부스팅으로 +{n} GP" (부스팅 사용 시) 상세 표시
- 화면 탭 또는 일정 시간 후 자동 닫힘
- 최초 1회만 재생 (재진입 시 미재생)

## 4. 규칙/로직

### 4.1 상태 배지 매핑 (앱)

| 게임 상태 | 앱 배지 텍스트 | 배지 색상 |
| ---- | ---- | ---- |
| 진행중 (Active) | 진행중 | 초록색 |
| 결과 대기 (Pending) | 발표대기중 | 회색 |
| 결과 확정 (Closed) | 결과발표 | 초록색 |
| 종료 (Ended) | 결과발표 | 초록색 |

> 앱에서는 임시저장(Draft), 게시대기(Ready) 상태의 게임은 노출되지 않습니다.
> Closed와 Ended 상태는 앱에서 동일하게 "결과발표"로 표시됩니다.

### 4.2 투표 현황 비율 표시 규칙

- YES/NO 비율은 참여자 수 기준으로 산출 (GP 가중치 아님)
- 소수점 이하 반올림하여 정수(%)로 표시
- YES + NO = 100% 보장
- 참여자가 0명인 경우 비율 바 비표시

### 4.3 보상 계산 로직

- 정답자 보상 = 총 상금 GP × (개인 배분 값 / 전체 합)
- 기본: 개인 배분 값 = 참여 GP
  - 부스팅 구매시: 개인 배분 값 = 참여 GP × 부스팅 배수
- 전체 합 = ∑(각 참여자의 배분 값)
- 참여 GP는 정답/오답 관계없이 전액 환급
- 부스팅 GP는 환급 없음, 배분 가중치로만 사용
- 소수점 8자리 이하 절삭, 단수는 시스템 귀속

### 4.4 데이터 노출 조건

| 항목 | 노출 조건 |
| ---- | ---- |
| 참여 정원 ("선착순 {n} 명") | 참여 정원 제한이 설정된 경우에만 표시 |
| 힌트 링크 | 백오피스에서 힌트 링크 "사용"으로 설정한 경우에만 표시 |
| 부스팅 CTA | 진행중 + 참여완료 + 유저 미부스팅 상태 |
| 부스팅 유도 카드 | 진행중 + 참여완료 + 사용자 부스팅 미사용 |
| 결과일정 텍스트 | 항상 표시 (결과 발표 예정일 + 결과 확인 기준 텍스트 조합) |

### 4.5 다국어 처리

| 항목 | 다국어 범위 |
| ---- | ---- |
| 게임 타이틀 | KO / EN / JP (사용자 언어 설정에 따라 자동 전환) |
| 상세설명 | KO / EN / JP |
| 결과 확인 기준 | KO / EN / JP |
| 힌트 링크 URL | 단일 URL (언어 무관) |
| 시스템 UI 텍스트 | KO / EN / JP (앱 다국어 리소스) |

### 4.6 백오피스 설정 반영 범위

| 백오피스 설정 | 앱 반영 |
| ---- | ---- |
| 게임 상태 변경 (게시/종료) | 앱 상태 배지 및 화면 분기 즉시 반영 |
| 결과 입력 (YES/NO) | 앱 투표 현황에 WIN 표시 + 결과 카드 표시 |
| 보상 지급 | 앱 보상 카드에 보상 GP 표시 |
| 게임 정보 수정 | 타이틀, 설명, 일정 등 실시간 반영 |
| 강제 종료 | 앱에서 참여 불가 처리 + 참여 GP 환불 |

### 4.7 선택 변경 불가 규칙

- YES/NO 선택 확정 후 변경 불가 (서버에서 강제)
- 참여 확정 모달에서 명확히 안내
- 부스팅은 참여 후 별도 액션으로, 1회만 가능

## 5. 에러 및 예외상황

##### 데이터 로딩

| 상태 | 표시 |
| ---- | ---- |
| Loading | Skeleton UI (타이틀, 배지, 본문 영역) |
| Error | "게임 정보를 불러올 수 없습니다." + [재시도] 버튼 |
| 404 (삭제된 게임) | "존재하지 않는 게임입니다." + [게임존으로 이동] 버튼 |

##### 참여 실패

| 상황 | 처리 |
| ---- | ---- |
| GP 잔고 부족 | CEB-GAM-202-MD-NOBALANCE 모달 표시 |
| 인원 마감 (참여 시점에 마감) | "선착순 인원이 마감되었습니다." Error 토스트 + 화면 새로고침 |
| 투표 종료 (참여 시점에 종료) | "투표가 종료되었습니다." Error 토스트 + 화면 새로고침 |
| 네트워크 오류 | "네트워크 오류가 발생했습니다. 다시 시도해주세요." Error 토스트 |
| 이미 참여한 게임 | "이미 참여한 게임입니다." Error 토스트 (중복 방지) |

##### 부스팅 실패

| 상황 | 처리 |
| ---- | ---- |
| GP 잔고 부족 | CEB-GAM-202-MD-NOBALANCE 모달 표시 (부스팅 비용 기준) |
| 이미 부스팅 완료 | "이미 부스팅이 적용되었습니다." Error 토스트 |
| 투표 종료 | "투표가 종료되어 부스팅을 사용할 수 없습니다." Error 토스트 |

- YES/NO 버튼 또는 부스팅 버튼 탭 시 로그인 유도 화면으로 이동
- 로그인 완료 후 현재 게임 상세 화면으로 복귀
