## 1. 화면 목적

팬퀘스트(Fan Quest) 검수 진행 시, 운영자가 선택할 수 있는 반려 사유(Rejection Reason) 데이터베이스를 조회하고 관리합니다.

### 주요 기능

- 운영자가 식별하기 쉬운 관리용 명칭과 유저에게 실제로 발송되는 안내 메시지를 한눈에 파악할 수 있도록 리스트 형태로 제공
- 사유의 사용 여부(사용/미사용)를 필터링하여 유효한 항목만 빠르게 확인
- 신규 반려사유 추가 및 기존 사유 수정

### 진입 경로

- [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > [반려사유 설정] 버튼 클릭

---

## 2. UI 설명

### 2.1 상단 네비게이션

- Breadcrumb: 홈 > 게임존 > Fan Quest 관리 > 반려사유 설정
- 화면 타이틀: 반려사유 설정

### 2.2 검색 및 필터 영역

- 상태 필터 (Select Box)
  - 옵션: 상태(전체) (Default), 사용, 미사용
  - 동작: 옵션 선택 시 리스트가 즉시 갱신
- [+ 새 사유 추가] 버튼
  - 동작: 신규 사유 등록을 위한 모달(CEB-BO-FQ-101-MD-ADD)을 호출합니다.

### 2.3 목록 영역 (List Table)

데이터는 테이블(Grid) 형태로 노출되며, 각 컬럼의 정의는 다음과 같습니다.

| 컬럼명 | 데이터 예시 | 설명 |
| ------ | ----------- | ---- |
| 상태 | 사용 / 미사용 | 해당 사유가 현재 검수 메뉴에서 노출되는지 여부 (Badge 형태) |
| 운영자 노출명 (Admin Label) | 식별불가, 대상 불일치 | 관리자가 내부적으로 식별하기 위한 요약된 사유명 |
| 유저 노출 메시지 (Client Message) | 이미지가 흐릿하거나... | 유저의 앱(App) 알림 또는 반려 내역에 실제로 노출되는 상세 텍스트 |

### 2.4 페이지네이션

- 위치: 목록 최하단 중앙
- 구성: < Previous 1 2 3 ... Next >
- 기능: 페이지 당 10개(또는 20개) 항목 노출, 페이지 번호 클릭 시 이동

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 버튼 동작

- **[+ 새 사유 추가]**: 클릭 시 신규 사유 등록을 위한 모달(CEB-BO-FQ-101-MD-ADD)을 호출합니다.

### 3.2 네비게이션

- Row Click (행 클릭): 리스트의 특정 행(영역 전체)을 클릭하면 해당 사유의 상세 정보를 수정할 수 있는 수정 모달(CEB-BO-FQ-101-MD-EDIT)이 호출됩니다.

---

## 4. 규칙/로직

### 4.1 리스트 출력 및 정렬

- 기본 정렬: 최근에 등록(생성)된 사유가 상단에 노출되는 최신순(Created Date Desc)을 기본으로 합니다.
- 텍스트 말줄임: 유저 노출 메시지가 컬럼 너비를 초과할 경우 1줄 까지만 노출하고 말줄임(...) 처리합니다. Mouse Over 시 툴팁으로 전체 내용을 보여줍니다.

### 4.2 상태(Status) 정의 및 표시

| 상태 | 의미 |
| ---- | ---- |
| 사용 (Active) | 현재 퀘스트 검수 반려 팝업 리스트에 노출되는 항목 |
| 미사용 (Inactive) | 과거에는 사용했으나 현재는 사용하지 않는 항목으로, 검수 반려 팝업 리스트에 노출되지 않음 |

---

## 5. 에러 및 예외상황

### 5.1 데이터 없음

- 초기 진입 시: 등록된 사유가 하나도 없을 경우 테이블 영역 중앙에 "등록된 반려 사유가 없습니다." 안내 문구 표시
- 검색 결과 없음: 필터(예: 미사용) 선택 후 해당하는 데이터가 없을 경우 "해당 상태의 반려 사유가 없습니다." 안내 문구 표시
