## 1. 화면 목적

Quest 수정 작업 중, 사용자가 [취소하기] 버튼을 눌렀을 때 수정 중이던 데이터 손실에 대한 주의를 주고, 최종적인 수정 취소 의사를 확인합니다.

### 주요 기능

- 수정 데이터 손실 경고 안내
- 수정 취소 최종 확인

### 진입 경로

- Quest 수정 화면(CEB-BO-FQ-201-EDIT)에서 [취소하기] 버튼 클릭 시 호출
- 임시저장(Draft) 상태 및 진행중(Active) 상태 모두 동일한 모달 사용

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 경고 심볼(Warning Icon) 배치
- 타이틀: "Quest 수정 취소"
- 안내 메시지:
  - "수정된 데이터는 저장되지 않습니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [취소]: 수정 취소를 철회하고, 기존 수정 화면으로 돌아가 작업을 계속
- [확인]: 수정 중이던 모든 변경사항을 폐기하고, Quest 상세 화면으로 이동

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 버튼 동작

| 버튼 | 동작 |
| ---- | ---- |
| [취소] | 수정 취소를 철회하고 기존 수정 화면으로 복귀 |
| [확인] | 수정 중이던 모든 변경사항을 폐기하고 Quest 상세(CEB-BO-FQ-201)로 이동 |
| [X] 닫기 | 모달만 닫히며 수정 중이던 화면 상태 유지 |

---

## 4. 규칙/로직

### 4.1 수정 취소 프로세스

- **진입**: Quest 수정 화면(CEB-BO-FQ-201-EDIT)에서 [취소하기] 버튼 클릭 시 호출. Quest 상태(임시저장/진행중)에 관계없이 동일한 모달이 표시됩니다.
- **확인(Confirm)**: [확인] 버튼 클릭 시, 클라이언트(브라우저) 상에서 변경된 수정 데이터를 모두 폐기합니다. Quest 상세 화면(CEB-BO-FQ-201)으로 리다이렉트됩니다.
- **철회(Cancel)**: [취소] 또는 우측 상단 [X] 버튼 클릭 시 모달만 닫히며, 수정 중이던 화면 상태는 그대로 유지됩니다.
