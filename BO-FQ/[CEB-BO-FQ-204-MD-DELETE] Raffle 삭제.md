## 1. 화면 목적

아직 게시되지 않은 임시저장(Draft) 상태의 Raffle을 시스템에서 영구적으로 제거하기 위해 관리자 확인을 수행합니다.

삭제는 되돌릴 수 없음을 인지시킵니다.

### 주요 기능

- 임시저장(Draft) 상태 Raffle 영구 삭제 확인
- 삭제 후 Raffle 목록 화면으로 이동

### 진입 경로

- Raffle 상세(CEB-BO-FQ-204)에서 임시저장(Draft) 상태일 때 [삭제하기] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 경고 심볼(Warning Icon) 배치
- 타이틀: "Raffle 삭제"
- 안내 메시지:
  - "확인 버튼을 누르면 Raffle이 삭제 됩니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [취소]: 삭제 프로세스를 취소하고 모달을 닫음
- [삭제]: Raffle 데이터를 시스템에서 영구 삭제 (빨간색 Danger 버튼)

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 버튼 동작

| 버튼 | 동작 |
| ---- | ---- |
| [취소] | 삭제 프로세스를 취소하고 모달 닫기 |
| [삭제] | 서버로 삭제 요청 전송, Raffle 데이터 영구 제거 후 목록 화면으로 이동 |

---

## 4. 규칙/로직

### 4.1 삭제 프로세스

- **진입**: 임시저장(Draft) 상태의 Raffle 상세 화면에서 [삭제하기] 버튼 클릭 시 호출 (게시된 Raffle은 삭제 불가)
- **확인(Confirm)**:
  - [삭제] 버튼 클릭 시 서버로 삭제 요청(Delete API)을 전송
  - 서버에서 해당 Raffle 데이터가 완전히 제거됩니다
  - 삭제 완료 후 사용자는 Raffle 목록(CEB-BO-FQ-102)으로 자동 이동합니다

---

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

**성공 (Success)**

- 메시지: "요청성공 - Raffle이 삭제되었습니다."
- 토스트 타입: Success
- 후행 동작: 모달이 닫히고, 사용자는 상위 화면인 Raffle 목록(CEB-BO-FQ-102)으로 자동 이동합니다.

**실패 (Failure)**

- 메시지: "요청실패 - Raffle 삭제에 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 원인 예시: 네트워크 오류, 이미 삭제된 데이터인 경우, 또는 그 사이 게시(Active)되어 삭제가 불가능해진 경우 등
- 후행 동작: 모달이 닫히지 않고 유지됩니다.
