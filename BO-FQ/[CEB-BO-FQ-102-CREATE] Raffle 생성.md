## 1. 화면 목적

유저가 응모권을 사용하여 응모 후 추첨(Draw)을 통해 보상을 받는 래플(Raffle)형 퀘스트를 신규 생성합니다.

### 주요 기능

- 응모 기간, 당첨 인원, 보상 지급 방식 등 추첨 이벤트에 특화된 정보를 설정
- 좌측(항목 리스트)과 우측(입력 폼)이 상호작용하는 Split View UI를 통해 단계별 입력 지원
- Raffle 기본 정보 입력 (아티스트, 마감일시, 대표 이미지, 타이틀, 설명) - 다국어 지원 (한국어/영어/일본어)
- Raffle 보상 정보 입력 (당첨 추첨 수, 보상지급 타입, 보상 상세, 수령 가이드, 유의사항)
- 생성된 Raffle은 Draft 상태로 저장되며, 이후 상세 화면에서 게시할 수 있습니다.

### 진입 경로

- [CEB-BO-FQ-102] 팬퀘스트 메인 (Raffle 탭) > [생성하기] 버튼 클릭

## 3. 화면 내 기능설명

### 3.1 버튼 동작

- **[취소하기]**: 클릭 시 생성 취소 확인 모달(CEB-BO-FQ-102-MD-CANCEL)을 호출합니다.
- **[생성하기]**: 모든 필수 입력 항목이 입력완료 상태일 때만 활성화됩니다. 클릭 시 생성 완료 확인 모달(CEB-BO-FQ-102-MD-COMPLE)을 호출합니다.

### 3.2 네비게이션

- 생성 취소 시: [CEB-BO-FQ-102] 팬퀘스트 메인 (Raffle 탭)으로 이동
- 생성 완료 시: [CEB-BO-FQ-204] 해당 Raffle 상세 화면으로 이동

## 4. 규칙/로직

### 4.1 입력 상태 관리

| 상태 | 설명 |
| ---- | ---- |
| 입력하기 | 아직 데이터가 없는 항목 |
| 입력중... | 현재 우측 패널에 활성화된 항목 |
| 입력완료 | 데이터가 입력되고 유효성 검사를 통과한 항목 |

- 조건: 좌측 상단의 4가지 상세 항목(이미지, 타이틀, 설명, 보상상세)이 모두 입력완료 상태여야 최종 생성이 가능합니다.

### 4.2 이미지 업로드 정책

| 항목 | 제한 |
| ---- | ---- |
| 최대 용량 | 50MB |
| 지원 포맷 | jpeg, png, jpg |
| 업로드 수량 | 1장 |

- 실패 처리: 업로드 실패 시 Toast 메시지("이미지 업로드에 실패하였습니다") 노출 및 재시도 유도

### 4.3 기간 및 추첨 로직

- 마감일: 현재 시간보다 미래로 설정해야 합니다.
- 추첨 수: 1 이상의 정수만 입력 가능합니다.

### 4.4 보상지급 타입별 수령 가이드 입력 항목

| 보상지급 타입 | 수령 가이드 입력 항목 |
| ------------- | --------------------- |
| 현장 수령 | 수령 기간(시작일/마감일), 운영 시간(시작/마감), 수령 장소(다국어), 지참물(다국어) |
| 배송 수령 | 배송지 입력 마감일시, 배송지 입력 폼 URL |
| 온라인 수령 | (별도 입력 항목 없음 또는 추후 정의) |
