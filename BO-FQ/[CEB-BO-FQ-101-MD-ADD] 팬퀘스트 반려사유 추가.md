## 1. 화면 목적

팬퀘스트(Fan Quest) 반려 처리 시 사용할 새로운 반려 사유(Rejection Reason)를 시스템에 등록합니다.

운영자용 식별 명칭과 유저에게 발송될 다국어 메시지를 설정합니다.

### 주요 기능

- 운영자 노출명 등록
- 다국어(KO/EN/JP) 유저 노출 메시지 입력
- 신규 반려 사유 시스템 등록

### 진입 경로

- Quest 반려사유 설정 화면(CEB-BO-FQ-101-REJETSET)에서 [+ 새 사유 추가] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

**타이틀**: 반려사유 추가

**닫기 버튼 (X)**: 우측 상단 위치. 클릭 시 작성 내용을 저장하지 않고 모달을 닫습니다.

**입력 폼**

- **운영자 노출명** (Input Text)
  - Placeholder: "운영자 노출명 입력"
  - 글자수 제한: 100자
  - 설명: 관리자 페이지 내 리스트 및 드롭다운 등에서 운영자가 식별하기 위한 짧은 명칭

- **유저 노출 메시지** (Multi-Language Input)
  - 언어 선택 탭/라디오 (Radio Group): 한국어(기본값) / 영어 / 일본어
  - 메시지 입력창 (Textarea)
    - Placeholder: "유저 노출 메시지 입력"
    - 글자수 제한: 200자
    - 설명: 선택된 언어(Radio)에 해당하는 유저 안내 메시지를 입력

**하단 액션 버튼**

- [취소하기]: 모달을 닫고 이전 화면(목록)으로 복귀
- [추가하기]: 모든 필드의 입력여부를 확인 후 서버에 전송하여 신규 사유를 등록

> 상세 스타일은 피그마 참조

## 4. 규칙/로직

### 4.1 초기값 및 상태 설정

- 기본 상태(Status): 신규 추가되는 모든 반려 사유는 기본적으로 '사용(Active)' 상태로 등록됩니다. (별도의 상태 선택 UI 없음)
- 기본 언어: 모달 진입 시 언어 선택 라디오 버튼은 한국어(기본값)이 선택되어 있어야 합니다.

### 4.2 다국어 입력 로직

- **데이터 유지**: 한국어를 선택하고 메시지를 입력한 뒤, 영어 탭을 선택하면 입력창은 비워집니다(또는 영어 데이터 로드). 다시 한국어 탭으로 돌아오면 이전에 입력했던 한국어 메시지가 유지되어 있어야 합니다.
- **필수값**: 모든 언어에 대한 메시지는 필수 입력 사항입니다.

### 4.3 유효성 검사 (Validation)

- 운영자 노출명, 유저 노출 메시지가 모두 입력완료 → 추가하기 버튼 활성화

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

작업 결과는 모달 위 또는 상위 화면에 Toast 팝업 형태로 피드백을 제공합니다.

**성공 (Success)**

- 메시지: "요청성공 - 새로운 반려사유 추가를 완료하였습니다."
- 토스트 타입: Success
- 후행 동작: 모달이 닫히고, 목록이 새로고침되어 추가된 항목이 최상단에 노출됩니다.

**실패 (Failure)**

- 메시지: "요청실패 - 새로운 반려사유 추가를 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 후행 동작: 모달이 닫히지 않고 유지되며, 사용자가 재시도할 수 있습니다.
