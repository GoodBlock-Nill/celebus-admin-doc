## 1. 화면 목적

진행 중인(Active) Raffle을 관리자가 수동으로 조기 종료(Early Termination)하고자 할 때, 실수로 인한 종료를 방지하기 위해 최종 확인을 수행합니다.

종료 시 Raffle 상태가 추첨대기(Closed)로 전환되어 신규 응모가 즉시 마감됨을 인지시킵니다.

### 주요 기능

- Raffle 조기 종료 최종 확인
- 상태 변경 (진행중(Active) → 추첨대기(Closed))

### 진입 경로

- Raffle 상세(CEB-BO-FQ-204) 또는 응모내역 탭(CEB-BO-FQ-205)에서 진행중(Active) 상태일 때 [종료하기] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 경고 심볼(Warning Icon) 배치
- 타이틀: "Raffle 종료"
- 안내 메시지:
  - "확인 버튼을 누르면 Raffle이 조기 종료됩니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [취소]: 종료 프로세스를 취소하고 모달을 닫음
- [종료]: Raffle을 즉시 추첨대기(Closed) 상태로 변경 (빨간색 Danger 버튼)

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 버튼 동작

| ---- | ---- |
| [취소] | 종료 프로세스를 취소하고 모달 닫기 |
| [종료] | 서버로 종료 요청 전송, Raffle 상태를 진행중(Active) → 추첨대기(Closed)로 변경 |

---

## 4. 규칙/로직

### 4.1 종료 프로세스

- **진입**: 진행중(Active) 상태의 Raffle 상세 또는 응모내역 탭에서 [종료하기] 버튼 클릭 시 호출
- **확인(Confirm)**:
  - [종료] 버튼 클릭 시 서버로 종료 요청을 전송
  - 서버에서는 해당 Raffle의 상태를 진행중(Active) → 추첨대기(Closed)로 업데이트
  - 종료 시점(Timestamp)이 마감(종료) 일시로 기록됩니다 (기존 예정일보다 앞당겨짐)
  - 더 이상 유저가 해당 Raffle에 응모할 수 없게 됩니다

---

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

**성공 (Success)**

- 메시지: "요청성공 - Raffle이 종료되었습니다."
- 토스트 타입: Success
- 후행 동작: 모달이 닫히고, 상세 화면이 새로고침되어 상태 뱃지가 추첨대기(Closed)로 변경됩니다. 액션 버튼은 추첨대기 상태에 맞게 변경됩니다.

**실패 (Failure)**

- 메시지: "요청실패 - Raffle 종료에 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 원인 예시: 네트워크 오류, 이미 종료된 Raffle인 경우 등
- 후행 동작: 모달이 닫히지 않고 유지됩니다.
