## 1. 화면 목적

신규 Quest를 생성하기 위한 입력 폼 화면입니다.

### 주요 기능

- Quest 기본 정보 입력 (타이틀, 설명, 유저 가이드, 대표이미지) - 다국어 지원 (한국어/영어/일본어)
- Quest 설정 정보 입력 (아티스트, 타입, 마감일시)
- Quest 보상 정보 입력 (보상 타입, 응모권 수량, NFT 이벤트)
- Quest 연관 링크 입력 (최대 2개, 선택) - 앱에서 라벨 버튼으로 표시
- 생성된 Quest는 임시저장(Draft) 상태로 저장되며, 이후 상세 화면에서 게시할 수 있습니다.

### 진입 경로

- [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > [생성하기] 버튼 클릭

## 2. UI 설명

### 2.1 상단 헤더 (Header)

| 구분 | 요소명 | 타입 | 설명 |
| ---- | ------ | ---- | ---- |
| 텍스트 | Quest 생성 | Title | 화면 타이틀 |
| 버튼 | 취소하기 | Secondary Button | 생성 취소 확인 모달 호출 |
| 버튼 | 생성하기 | Primary Button | 모든 필수 항목 입력 완료 시 활성화. 생성 완료 확인 모달 호출 |

### 2.2 화면 레이아웃 (3-Column Layout)

화면은 3개 섹션이 좌->중->우 순서로 배치된 3-column 레이아웃입니다.

```text
+----------------------------+----------------------------+----------------------------+
|        기본 정보           |        퀘스트 설정          |        보상 설정            |
|                            |                            |                            |
|  타이틀  *  [KO/EN/JA]    |  아티스트  *    [Dropdown]  |  보상 종류  *   [Dropdown]  |
|  설명    *  [KO/EN/JA]    |  퀘스트 타입 * [Dropdown]  |  응모권 수량    [Dropdown]  |
|  유저 가이드 * [KO/EN/JA] |  마감 일시  *  [DatePicker] |  NFT 이벤트    [Dropdown]  |
|  대표 이미지 *  [Upload]   |  연관 링크     [+ 링크 추가]|                            |
|                            |                            |                            |
|                            |  * 시작 일시는 게시 시점에  |                            |
|                            |    자동으로 설정됩니다.     |                            |
|                            |    마감일은 오늘 기준 최소  |                            |
|                            |    1일 이후부터 선택 가능   |                            |
+----------------------------+----------------------------+----------------------------+
```

> 상세 스타일은 피그마 참조

### 2.3 기본 정보 섹션 (좌측)

| 요소명 | 타입 | 필수 | 글자수 제한 | 설명 |
| ------ | ---- | ---- | ----------- | ---- |
| 타이틀 | Text Input + 언어 탭(KO/EN/JA) | Y | 100자 | 다국어 퀘스트 타이틀 입력 |
| 설명 | Textarea + 언어 탭(KO/EN/JA) | Y | 200자 | 다국어 퀘스트 설명 입력 |
| 유저 가이드 | Textarea + 언어 탭(KO/EN/JA) | Y | 500자 | 다국어 유저 가이드 입력 |
| 대표 이미지 | Image Upload (Drag & Drop) | Y | - | 권장: 1920x1080px (16:9), jpeg/png/jpg, 최대 50MB |

- 각 텍스트 필드 우측에 글자수 카운터 표시 (예: 0/100)
- 언어 탭: KO(기본 선택) / EN / JA - 미입력 언어는 빨간 점(dot) 표시, 입력 완료 시 체크마크(V)

### 2.4 퀘스트 설정 섹션 (중앙)

| 요소명 | 타입 | 필수 | 설명 |
| ------ | ---- | ---- | ---- |
| 아티스트 | Dropdown | Y | 등록된 아티스트 목록에서 선택 (Placeholder: "아티스트 선택") |
| 퀘스트 타입 | Dropdown | Y | 미션 유형 선택 (기본값: "이미지 촬영 및 업로드") |
| 마감 일시 | Date Picker + Time Picker | Y | 마감 날짜 및 시간 설정 (Time 기본값: 23:59) |

- 안내 문구: "* 시작 일시는 게시 시점에 자동으로 설정됩니다. 마감일은 오늘 기준 최소 1일 이후부터 선택 가능합니다."

#### 연관 링크 (선택)

유저가 퀘스트를 수행할 때 참고할 수 있는 외부 링크를 등록합니다. 등록된 링크는 앱에서 라벨 버튼 형태로 노출됩니다. 최대 2개까지 등록 가능하며, 미입력 시 앱에 링크 버튼이 노출되지 않습니다.

| 요소명 | 타입 | 필수 | 글자수 제한 | 설명 |
| ------ | ---- | ---- | ----------- | ---- |
| 링크 라벨 | Text Input + 언어 탭(KO/EN/JA) | 조건부 | 각 20자 | 앱에서 버튼 텍스트로 표시되는 라벨. URL 입력 시 3개 언어 모두 필수 |
| 링크 URL | Text Input | 조건부 | 500자 | 연결할 외부 URL. 라벨 입력 시 필수. https:// 또는 http:// 형식 |
| [+ 링크 추가] | Text Button | - | - | 새 링크 입력 행 추가 (최대 2개까지, 2개 도달 시 버튼 비활성화) |
| [삭제] | Icon Button (X) | - | - | 해당 링크 행 삭제 |

- 초기 상태: 링크 입력 행 없음, [+ 링크 추가] 버튼만 표시
- 링크 추가 시: 라벨(KO/EN/JA) + URL 입력 필드가 한 세트로 추가됨

### 2.5 보상 설정 섹션 (우측)

| 요소명 | 타입 | 필수 | 설명 |
| ------ | ---- | ---- | ---- |
| 보상 종류 | Dropdown | Y | 응모권 / 응모권 + NFT / NFT 중 선택 (기본값: 응모권) |
| 응모권 수량 | Dropdown | 조건부 | 보상 종류에 "응모권"이 포함될 때 노출 (기본값: 1장) |
| NFT 이벤트 | Dropdown | 조건부 | 보상 종류에 "NFT"가 포함될 때 노출 (BIVE NFT 민팅 이벤트 목록) |

- 보상 종류 선택에 따른 동적 UI:
  - "응모권" -> 응모권 수량만 노출
  - "응모권 + NFT" -> 응모권 수량 + NFT 이벤트 모두 노출
  - "NFT" -> NFT 이벤트만 노출

## 3. 화면 내 기능설명

### 3.1 버튼 동작

- **[취소하기]**: 클릭 시, 작성 취소 확인 모달인 [CEB-BO-FQ-101-MD-CANCEL] Quest 생성취소 화면을 호출합니다.
- **[생성하기]**: 모든 필수 입력 항목이 입력완료 상태가 되고, 하단 규칙 설정이 완료되었을 때만 활성화됩니다. 클릭 시, 생성 완료 안내 모달인 [CEB-BO-FQ-101-MD-COMPLE] Quest 생성완료 화면을 호출합니다.

### 3.2 네비게이션

- 생성 취소 시: [CEB-BO-FQ-101] 팬퀘스트 메인으로 이동
- 생성 완료 시: [CEB-BO-FQ-201] 해당 Quest 상세 화면으로 이동

## 4. 규칙/로직

### 4.1 다국어 입력 유효성 검사

| 상태 | 표시 |
| ---- | ---- |
| 미입력 언어 존재 | 해당 언어 탭에 빨간 점(dot) 표시 |
| 모든 언어 입력 완료 | 모든 언어 탭에 체크마크(V) 표시 |

### 4.2 기간 설정 로직

- 시작 일시: 관리자가 [생성하기] 버튼을 눌러 생성이 완료한 이후 [게시하기] 버튼을 눌러 게시한 서버 시간(Timestamp)을 시작 일시로 자동 설정합니다.
- 종료 일시: 반드시 현재 시간(생성 시점)보다 미래의 시간으로만 설정 가능합니다.
  - 마감일 설정 필드를 누르면 캘린더 표시
  - 현재일자 +1일 부터 설정 가능
  - 시간 설정 필드를 누르면 정수만 입력 가능
    - 시: 00~23
    - 분: 00~59

### 4.3 보상 설정 로직

- 응모권 수량 선택: 직접 입력이 아닌 Select Box를 통해 1~10장 중 하나를 선택합니다.
- 동적 UI: 선택한 보상 종류(응모권+NFT, 응모권, NFT)에 따라 필요한 Select Box(수량 또는 이벤트 선택)만 노출합니다.

### 4.4 데이터 연동

- NFT 민팅 이벤트 리스트: NFT 관련 옵션 선택 시, 연동된 BIVE 시스템의 '진행 가능한 NFT 민팅 이벤트' 목록을 API로 조회하여 바인딩합니다.

### 4.5 연관 링크 유효성 검사

- 링크는 0~2개 입력 가능 (선택 사항)
- 링크 입력 시 라벨(3개 언어 모두) + URL 쌍으로 입력 필수
  - 라벨만 입력하고 URL 미입력 시: "URL을 입력해주세요" 인라인 에러
  - URL만 입력하고 라벨 미입력 시: "라벨을 입력해주세요" 인라인 에러
- URL 형식 검증: https:// 또는 http:// 로 시작해야 함
  - 형식 불일치 시: "올바른 URL 형식을 입력해주세요 (예: https://example.com)" 인라인 에러
- 연관 링크 미입력 시에도 [생성하기] 버튼 활성화에 영향 없음

### 4.6 생성하기 버튼 활성화 조건

다음 조건을 모두 충족해야 [생성하기] 버튼이 활성화됩니다:

- 타이틀: 3개 언어 모두 입력완료
- 설명: 3개 언어 모두 입력완료
- 유저 가이드: 3개 언어 모두 입력완료
- 대표 이미지: 업로드 완료
- 아티스트: 선택 완료
- 퀘스트 타입: 선택 완료
- 마감일/시간: 설정 완료
- 보상 설정: 선택 완료 (보상 종류에 따른 하위 항목 포함)
- 연관 링크: 선택 항목이므로 활성화 조건에 포함되지 않음. 단, 입력한 경우 유효성 검사를 통과해야 함
