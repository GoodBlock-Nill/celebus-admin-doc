## 1. 화면 목적

응모가 마감(Closed)된 Raffle에 대해 추첨을 실행하고, 추첨 완료 후 당첨자 명단과 보상 지급 상태를 관리합니다.

### 주요 기능

- 추첨 실행: 시스템 알고리즘을 통해 당첨자를 선발(Draw)
- 결과 확인: 추첨 완료(Ended) 후 당첨자(Winner) 명단과 전체 참여자들의 당락 여부(Win/Lose) 조회
- 보상 관리: 당첨자에게 보상이 정상적으로 지급되었는지 상태 모니터링

### 진입 경로

- [CEB-BO-FQ-204] Raffle 상세 (기본정보 탭) > 추첨내역 탭 클릭

---

## 2. UI 설명

### 2.1 상단 영역

- 탭 메뉴: 기본정보, 응모내역, **추첨내역 (Selected)**

- 상태별 화면 분기:
  - **추첨 전 (Active / Draft)**: 데이터 없음. "Raffle의 상태가 Closed 인 상태에서만 추첨이 가능합니다." 안내 문구 노출.
  - **추첨 대기 (Closed)**: 데이터 없음. "Closed 상태인 경우 하단 추첨하기 버튼을 눌러 추첨을 진행해주세요" 안내 문구. [추첨하기] 버튼 노출.
  - **추첨 완료 (Ended)**: 통계 대시보드(보상지급(n/m), 총 응모 유저, Win(당첨자수), Lose(낙첨자수)), 필터(상태: Win/Lose), 검색(닉네임, 전화번호) 기능 제공.

### 2.2 목록 영역 (Ended 상태 시)

추첨 완료 후 결과 데이터가 테이블로 제공됩니다.

| 컬럼명 | 데이터 예시 | 설명 |
| ------ | ----------- | ---- |
| 상태 | Win / Lose | 당첨 여부 뱃지 |
| 유저닉네임 | {유저닉네임} | 참여자 닉네임 |
| 전화번호 | 821012345678 | 연락처 정보 |
| 누적 사용 응모권 | 10 | 해당 유저가 사용한 총 티켓 수 |
| 추첨 일시 | 2025.09.12 15:23 | 추첨이 실행된 시간 |
| 보상 지급여부 | 지급완료 / 미지급 | 당첨자에 대한 보상 지급 상태 아이콘 |

### 2.3 하단 액션 (Closed 상태 시)

- [추첨하기] 버튼: 중앙 하단에 위치하며, 클릭 시 추첨 모달을 호출합니다.

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 추첨 실행

- [추첨하기] 버튼 클릭 시, [CEB-BO-FQ-206-MD-DRAW] Raffle 추첨하기 모달이 호출됩니다.
- 모달 내에서 추첨 결과를 확인하고 [추첨결과 확정하기]를 누르면, Raffle 상태가 Ended로 변경되고 유저에게 추첨 결과를 전달합니다.
- 당첨 여부에 따라 유저에게 표시되는 결과화면에서 보상수령 방법을 안내 받습니다.

### 3.2 네비게이션

- 기본정보 탭 클릭: [CEB-BO-FQ-204] Raffle 상세 (기본정보 탭) 화면으로 이동
- 응모내역 탭 클릭: [CEB-BO-FQ-205] Raffle 상세 (응모내역 탭) 화면으로 이동

## 4. 규칙/로직

### 4.1 추첨 실행 프로세스

- 진입 조건: Raffle 상태가 반드시 Closed여야 합니다.
- 실행: [추첨하기] 버튼 클릭 시, [CEB-BO-FQ-206-MD-DRAW] Raffle 추첨하기 모달이 호출됩니다.
- 확정: 모달 내에서 추첨 결과를 확인하고 [추첨결과 확정하기]를 누르면, Raffle 상태가 Ended로 변경되고 유저에게 추첨 결과를 전달합니다.

### 4.2 보상 지급 로직

- 추첨 확정 시, Win 상태의 유저에게 사전에 설정된 보상(NFT 등)이 시스템에 의해 자동 지급 요청됩니다.
- 지급 결과는 리스트의 보상 지급여부 컬럼에 실시간으로 업데이트됩니다.
