## 1. 화면 목적

진행 중인(Active) 퀘스트를 관리자가 수동으로 조기 종료(Early Termination)하고자 할 때, 실수로 인한 종료를 방지하기 위해 최종 확인을 수행합니다.

### 주요 기능

- 퀘스트 조기 종료 최종 확인
- 상태 변경 (Active → Ended)

### 진입 경로

- Quest 상세 화면(CEB-BO-FQ-201)에서 Active 상태일 때 [종료하기] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 경고 심볼(Warning Icon) 배치
- 타이틀: "Quest 종료"
- 안내 메시지:
  - "확인 버튼을 누르면 Quest가 조기 종료됩니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [다시 돌아가기]: 종료 프로세스를 취소하고 모달을 닫음
- [확인]: 퀘스트를 즉시 종료 상태(Ended)로 변경

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 버튼 동작

| 버튼 | 동작 |
| ---- | ---- |
| [다시 돌아가기] | 종료 프로세스를 취소하고 모달 닫기 |
| [확인] | 서버로 종료 요청 전송, 퀘스트 상태를 Active → Ended로 변경 |

---

## 4. 규칙/로직

### 4.1 종료 프로세스

- **진입**: Active 상태의 퀘스트 상세 화면에서 [종료하기] 버튼 클릭 시 호출
- **확인(Confirm)**:
  - [확인] 버튼 클릭 시 서버로 종료 요청을 전송
  - 서버에서는 해당 퀘스트의 상태를 Active → Ended로 업데이트
  - 종료 시점(Timestamp)이 마감(종료)일시로 기록됩니다 (기존 예정일보다 앞당겨짐)
  - 더 이상 유저가 해당 퀘스트에 참여(제출)할 수 없게 됩니다

---

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

**성공 (Success)**

- 메시지: "요청성공 - Quest가 종료 되었습니다."
- 토스트 타입: Success
- 후행 동작: 모달이 닫히고, 상세 화면이 새로고침되어 상태가 Ended로 변경되고 [종료하기] 버튼이 사라집니다.

**실패 (Failure)**

- 메시지: "요청실패 - Quest 종료에 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 원인 예시: 네트워크 오류, 이미 종료된 퀘스트인 경우 등
- 후행 동작: 모달이 닫히지 않고 유지됩니다.
