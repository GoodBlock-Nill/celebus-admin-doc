## 1. 화면 목적

기존에 등록된 반려 사유의 운영자 식별 명칭이나 유저 안내 메시지를 수정합니다.

해당 사유의 사용 여부(상태)를 변경하여, 실제 검수 화면에서의 노출 여부를 제어합니다.

### 주요 기능

- 운영자 노출명 수정
- 다국어(KO/EN/JP) 유저 노출 메시지 수정
- 사용/미사용 상태 변경

### 진입 경로

- 반려사유 설정 목록(CEB-BO-FQ-101-REJETSET)에서 Row 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

**타이틀**: 반려사유 수정

**닫기 버튼 (X)**: 우측 상단. 클릭 시 수정 사항을 저장하지 않고 닫습니다.

**입력 폼**

- **운영자 노출명** (Input Text)
  - 기존에 등록된 데이터가 바인딩되어 노출
  - 수정 가능
  - 글자수 제한: 100자

- **유저 노출 메시지** (Multi-Language Input)
  - 언어 선택 탭/라디오 (Radio Group): 한국어(기본값) / 영어 / 일본어
  - 메시지 입력창 (Textarea)
    - 선택한 언어에 해당하는 기존 메시지가 바인딩
    - 수정 가능
    - 글자수 제한: 200자

- **상태설정** (Select Box)
  - Default: 현재 해당 사유의 상태값 (예: 사용)
  - Options:
    - 사용: 검수 화면 리스트에 노출됨 (Active)
    - 미사용: 검수 화면 리스트에서 숨김 처리됨 (Inactive)
  - 미사용 처리 된 반려사유는 과거 데이터에 영향을 주지 않습니다

**하단 액션 버튼**

- [취소하기]: 수정 내역을 저장하지 않고 모달을 닫음
- [저장하기]: 변경된 내용을 유효성 검사 후 서버에 저장. 수정된 반려사유는 과거 데이터에 영향을 주지 않습니다

> 상세 스타일은 피그마 참조

## 4. 규칙/로직

### 4.1 데이터 바인딩 (Data Binding)

- 목록 화면(REJETSET)에서 클릭한 항목(Row)의 상세 데이터를 API로 조회하여 각 필드에 매핑합니다.
- 다국어 메시지의 경우, 등록된 모든 언어 데이터를 불러와야 하며, 사용자가 언어 탭(Radio) 변경 시 해당 언어의 데이터를 보여줍니다.

### 4.2 상태 변경 로직

- **사용 → 미사용 변경 시**: 저장 즉시 퀘스트 검수/반려 팝업의 사유 목록에서 해당 항목이 제거됩니다. (기존에 이미 반려 처리된 퀘스트 내역에는 영향 없음)
- **미사용 → 사용 변경 시**: 다시 검수/반려 팝업 목록에 노출됩니다.

### 4.3 유효성 검사

- 필수값 체크: 운영자 노출명, 유저 노출 메시지(모든 언어)가 비어있지 않아야 저장 가능합니다.

---

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

**성공 (Success)**

- 메시지: "요청성공 - 반려사유 수정을 완료하였습니다."
- 토스트 타입: Success
- 후행 동작: 모달이 닫히고, 목록 화면이 새로고침(변경된 내용 반영)됩니다.

**실패 (Failure)**

- 메시지: "요청실패 - 반려사유 수정을 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 후행 동작: 모달이 유지되며 사용자가 다시 저장을 시도할 수 있습니다.
