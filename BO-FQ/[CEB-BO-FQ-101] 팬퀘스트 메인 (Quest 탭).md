## 1. 화면 목적

팬퀘스트 서비스의 Quest 목록을 조회하고 관리하는 어드민 메인 화면입니다.

### 주요 기능

- 등록된 전체 Quest 목록 조회 및 검색
- 상태별 Quest 필터링 (진행중 / 임시저장 / 종료)
- 신규 Quest 생성 (생성완료/생성취소 모달 포함)
- 유저 제출물 반려 시 사용할 반려사유 설정/관리
- Quest 상세 화면으로 진입하여 개별 Quest 관리

### 진입 경로

- 사이드 메뉴 > 팬퀘스트 > Fan Quest 관리 진입 시 기본 노출 화면

## 2. UI 설명

### 2.1 상단 영역 (Top Area)

- Breadcrumb: 홈 > 팬퀘스트 > Fan Quest 관리
- Tab Menu:
  - **Quest (진행중)**: 현재 화면
  - **Raffle**: 래플 목록 화면 (CEB-BO-FQ-102)으로 이동

### 2.2 검색 및 필터 영역

| 구분 | 요소명 | 타입 | 설명 |
| ---- | ------ | ---- | ---- |
| 필터 | 상태(전체) | Dropdown | 상태값 필터 선택 (전체 / 진행중 / 임시저장 / 종료) |
| 필터 | 타이틀 | Dropdown | 검색 대상 필드 선택 |
| 입력 | 타이틀 입력 | Text Input | 검색할 Quest 타이틀 키워드 입력. 검색은 Enter 키 또는 입력 필드 내 검색 아이콘으로 실행 |
| 버튼 | 초기화 | Dark Button | 검색 조건을 초기 상태로 리셋 |
| 버튼 | 반려사유 설정 | Secondary Button | 반려사유 설정 화면으로 이동 |
| 버튼 | 생성하기 | Primary Button | 신규 Quest 생성 화면으로 이동 |

- 상태 필터 Default: 전체
- 검색 조건 Default: 타이틀
- 검색어 Placeholder: 타이틀 선택 시 "타이틀 입력", 아티스트 선택 시 "아티스트 입력"

### 2.3 목록 영역 (List Table)

| 컬럼명 | 데이터 타입 | 설명 |
| ------ | ----------- | ---- |
| 상태 | Badge | Quest 상태 표시 (진행중 / 임시저장 / 종료) |
| 타이틀 | Text | Quest 타이틀 (한국어 기준 노출) |
| 아티스트 | Text | Quest에 연결된 아티스트명 |
| 기간 | Date Range | Quest 진행 기간 (YYYY.MM.DD ~ YYYY.MM.DD) |
| 검토 필요 | Number | 검토 대기 중인 유저 제출물 수량 |

- 리스트 Row 클릭 시 해당 Quest 상세화면(CEB-BO-FQ-201)으로 이동
- 기간: 시작(게시)일 ~ 마감(종료)일
  - 마감일 전에 수동으로 종료되면 해당 일자로 변경됨
- 검토 필요: 유저가 제출한 미검토(Pending) 건수 표시 (예: 234)

### 2.4 페이지네이션

| 구분 | 요소명 | 설명 |
| ---- | ------ | ---- |
| 버튼 | Previous | 이전 페이지로 이동 |
| 버튼 | 1, 2, 3, ..., N | 해당 페이지로 직접 이동 |
| 버튼 | Next | 다음 페이지로 이동 |

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 버튼 동작

- **[초기화]**: 검색 조건을 초기 상태로 리셋합니다.
- **[반려사유 설정]**: 클릭 시 CEB-BO-FQ-101-REJETSET 화면으로 이동합니다.
- **[생성하기]**: 클릭 시 CEB-BO-FQ-101-Create 화면으로 이동합니다.

### 3.2 네비게이션

- 리스트 항목 클릭: 해당 항목의 영역을 클릭하면 [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) 화면으로 이동합니다.
- Raffle 탭 클릭: [CEB-BO-FQ-102] 팬퀘스트 (Raffle 메인) 화면으로 이동합니다.

## 4. 규칙/로직

### 4.1 권한별 기능 분기

| 권한 | 화면 접근 | 조회 | [반려사유 설정] | [생성하기] | ROW 클릭 (상세 이동) |
| ---- | --------- | ---- | --------------- | ---------- | -------------------- |
| Super Admin | ✅ | ✅ | ✅ | ✅ | ✅ |
| Admin (팬퀘스트 쓰기 권한 有) | ✅ | ✅ | ✅ | ✅ | ✅ |
| Admin (팬퀘스트 읽기 권한만) | ✅ | ✅ | ❌ | ❌ | ✅ |
| 권한 없음 | ❌ | ❌ | ❌ | ❌ | ❌ |

- 권한은 [CEB-BO-ADST-202] 권한관리에서 설정된 팬퀘스트 대메뉴의 읽기/쓰기 권한에 따라 분기됩니다.

### 4.2 정렬 기준

| 정렬 기준 | 정렬 방식 | 비고 |
| --------- | --------- | ---- |
| 마감(종료)일 | 오름차순 (임박순) | 기본 정렬 |

### 4.3 검토 필요 카운트

- 상태가 임시저장(Draft)인 경우 참여가 불가능하므로 '0'으로 표시됩니다.
- 상태가 진행중(Active) 또는 종료(Ended)인 경우, 관리자의 승인/반려 처리가 되지 않은 대기 건수를 실시간으로 표시합니다.

### 4.4 상태(Status) 정의

| 상태 | 설명 | 검토 필요 컬럼 |
| ---- | ---- | -------------- |
| 임시저장(Draft) | Quest가 생성되었으나 아직 게시되지 않은 상태 | 0 (고정) |
| 진행중(Active) | Quest가 게시되어 유저가 참여 가능한 상태 | 실시간 카운트 |
| 종료(Ended) | Quest가 종료된 상태 (마감일 도래 또는 수동 종료) | 실시간 카운트 |

### 4.5 검색 조건

- 상태 필터와 타이틀 검색은 AND 조건으로 적용
- 타이틀 검색은 부분 일치(LIKE) 검색 지원
- 검색어 미입력 시 해당 필터 조건 무시

| 상황 | 메시지/처리 |
| ---- | ----------- |

| 네트워크 오류 | 공통 에러 처리 로직 적용 (로딩 실패 안내) |
