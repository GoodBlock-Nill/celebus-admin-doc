## 1. 화면 목적

관리자가 수정한 Raffle 정보를 최종적으로 시스템에 반영하기 전, 저장 의사를 다시 한번 확인합니다.

### 주요 기능

- Raffle 수정사항 최종 저장 확인
- 서버 데이터 업데이트 요청

### 진입 경로

- Raffle 수정 화면(CEB-BO-FQ-204-EDIT)에서 [수정하기] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 번개 모양 아이콘 배치
- 타이틀: "Raffle 수정"
- 안내 메시지:
  - "확인 버튼을 누르면 수정사항이 적용됩니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [다시 돌아가기]: 저장 프로세스를 잠시 보류하고, 수정 화면으로 돌아가 내용을 재검토
- [확인]: 변경된 데이터를 서버로 전송하여 최종 저장(Update)

> 상세 스타일은 피그마 참조

## 4. 규칙/로직

### 4.1 저장 프로세스

- **진입**: 수정 화면([CEB-BO-FQ-204-EDIT])에서 [저장하기] 버튼 클릭 시 호출
- **확인(Confirm)**: [확인] 버튼 클릭 시 서버로 데이터 업데이트 요청을 전송합니다. 성공 시, 수정 성공 토스트(Toast) 메시지를 노출하고 상세 화면으로 이동합니다.

---

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

**성공 (Success)**

- 메시지: "요청성공 - Raffle 수정이 완료되었습니다."
- 토스트 타입: Success
- 후행 동작: 모달 닫힘 → 상세 화면(CEB-BO-FQ-204)으로 이동

**실패 (Failure)**

- 메시지: "요청실패 - Raffle 수정에 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 후행 동작: 모달 닫힘 → 수정 화면 유지 (재시도 가능)
