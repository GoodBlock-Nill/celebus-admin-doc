## 1. 화면 목적

등록된 개별 퀘스트의 기본 정보와 설정 상태를 확인하고, 퀘스트의 생애 주기(Life Cycle)를 관리합니다.

### 주요 기능

- 퀘스트 기본 정보 및 설정 상태 확인
- 유저들의 퀘스트 참여(제출) 현황을 대시보드(수치) 형태로 모니터링
- 퀘스트의 생애 주기 관리 (게시, 수정, 종료, 삭제)

### 진입 경로

- [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > Quest 리스트 ROW 클릭

---

## 2. UI 설명

### 2.1 상단 네비게이션 및 헤더

- Breadcrumb: 홈 > 팬퀘스트 > Fan Quest 관리 > Quest > {Quest 타이틀}
- 뒤로가기 화살표 (<) 아이콘 (상위 목록으로 이동)
- 타이틀: Quest 이름 + Status Badge (예: iKON 굿즈 인증 [진행중(Active)])
- 탭 메뉴 (Tab Menu):
  - **기본정보 (Selected)**: 현재 화면
  - **대기내역**: 검수 대기 중인 제출 건 리스트로 이동
  - **처리내역**: 승인/반려 처리된 제출 건 리스트로 이동

### 2.2 상태별 액션 버튼

화면 우측 상단에 위치하며, 퀘스트의 현재 상태(Status)에 따라 노출되는 버튼이 달라집니다.

| 상태 | 버튼 구성 | 기능 설명 |
| ---- | --------- | --------- |
| 임시저장(Draft) | [삭제하기] [수정하기] [게시하기] | 삭제/수정/게시 |
| 진행중(Active) | [수정하기] [종료하기] | 수정/종료 |
| 종료(Ended) | (버튼 없음) | 변경 불가 |

### 2.3 현황 대시보드 (Dashboard Cards)

상단 영역에 4개의 카드로 구성되어 실시간 제출 현황을 표시합니다.

- **총 제출 수** (people 아이콘): 전체 참여 건수
- **승인** (체크마크 아이콘): 검수 완료 및 보상 지급 확정된 건수
- **반려** (X 아이콘): 검수 결과 반려된 건수
- **대기** (시계 아이콘): 아직 검수가 진행되지 않은 대기 건수

### 2.4 상세 정보 그리드 (Info Grid)

#### A. 대표 이미지 (좌측 상단)

- Quest의 대표 이미지를 큰 썸네일로 표시
- 이미지 클릭 시 원본 이미지 보기 가능

#### B. Quest 정보 (우측 상단)

| 항목 | 설명 |
| ---- | ---- |
| 아티스트 | 연결된 아티스트명 |
| 타입 | 퀘스트 미션 타입 (예: 이미지 촬영 및 업로드) |
| 보상 | 보상 아이콘 및 수량 (예: 응모권 1개 + NFT (팬퀘스트 iKON)) |
| 마감일시 | 마감(종료) 예정일시 (임시저장 상태에서만 표시) |
| 시작일시 | 게시 시점 일시 (진행중/종료 상태에서만 표시) |
| 마감일시 | 마감(종료)일시 (진행중/종료 상태에서 표시) |
| 연관 링크 | 등록된 링크의 라벨(KO 기준)과 URL을 표시. 최대 2개. 미등록 시 "-" 표시 |

> 임시저장(Draft)에서는 마감일시만 표시. 진행중(Active)/종료(Ended)에서는 시작일시와 마감일시 모두 표시.

#### C. 관리 정보 (우측 하단)

| 항목 | 설명 |
| ---- | ---- |
| 상태 | Status Badge (임시저장/진행중/종료) |
| 생성 관리자 | 생성자 이름 |
| 생성 일시 | YYYY.MM.DD HH:mm |
| 최근 수정자 | 수정자 이름 |
| 최근 수정 일시 | YYYY.MM.DD HH:mm |

#### D. Quest 타이틀 (하단)

- KO / EN / JA 각 언어별 타이틀 텍스트 표시

#### E. Quest 설명 (하단)

- KO / EN / JA 각 언어별 설명 텍스트 표시

#### F. Quest 유저 가이드 (하단)

- KO / EN / JA 각 언어별 유저 가이드 텍스트 표시

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 버튼 동작

- **[삭제하기]** (임시저장): 퀘스트를 영구 삭제합니다.
- **[수정하기]** (임시저장/진행중): 퀘스트 정보를 수정합니다. Quest 수정 화면(CEB-BO-FQ-201-EDIT)으로 이동합니다.
- **[게시하기]** (임시저장): 퀘스트를 유저에게 공개하고 진행중(Active) 상태로 전환합니다.
- **[종료하기]** (진행중): 퀘스트를 조기 종료하여 종료(Ended) 상태로 전환합니다.

### 3.2 네비게이션

- 대기내역 탭 클릭: [CEB-BO-FQ-202] 대기내역 화면으로 이동
- 처리내역 탭 클릭: [CEB-BO-FQ-203] 처리내역 화면으로 이동

---

## 4. 규칙/로직

### 4.1 상태 전환 로직

- **게시하기 (임시저장 → 진행중)**: 임시저장(Draft) 상태에서 [게시하기] 버튼 클릭 시, 현재 서버 시간을 시작 일시로 기록하며 상태가 진행중(Active)으로 변경됩니다. 유저 앱(App)에 퀘스트가 즉시 노출됩니다.
- **종료하기 (진행중 → 종료)**: 진행중(Active) 상태에서 [종료하기] 버튼 클릭 시, 퀘스트 참여가 중단되며 상태가 종료(Ended)로 변경됩니다. 마감일시가 도래하면 시스템에 의해 자동으로 종료(Ended) 처리 됩니다.

### 4.2 데이터 표시 규칙

- 대기 카운트: 종료(Ended) 상태가 되어도 미처리된 건이 있다면 숫자가 남아있을 수 있습니다.
- 날짜 포맷: YYYY.MM.DD hh:mm 형태를 준수합니다.

---

## 5. 에러 및 예외상황

- **삭제 불가**: 이미 참여자가 발생한 진행중(Active) 퀘스트는 삭제가 불가능하며, 종료하기만 가능합니다. (임시저장(Draft) 상태에서만 삭제 가능)
- **수정 제한**: 진행중(Active) 상태에서는 핵심 정보(보상 종류, 아티스트 등)의 수정이 제한될 수 있으며, 오타 수정 등 단순 정보 변경만 허용됩니다.
