## 1. 화면 목적

해당 퀘스트에서 관리자가 검수 완료(승인 또는 반려)한 모든 제출 건들의 처리 이력(History)을 조회합니다.

### 주요 기능

- 승인/반려 처리된 제출 건의 상태, 처리 일시, 처리자(관리자) 정보 확인
- 이미지 컬럼의 아이콘 버튼 클릭으로 유저 제출 이미지 확인
- 상태별 필터링 및 검색 기능

### 진입 경로

- [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) > 처리내역 탭 클릭

---

## 2. UI 설명

### 2.1 상단 영역

- 탭 메뉴: 기본정보, 대기내역, **처리내역 (Selected)**
- 검색 및 필터:
  - 상태 필터 (Select Box): 상태(전체), Approved, Rejected
  - 검색 조건 (Select Box): 닉네임, 제출ID
  - 검색어 입력창 (Input): 검색 키워드 입력
  - [초기화] 버튼: 필터 및 검색 조건 적용

### 2.2 목록 영역 (List Table)

데이터는 리스트(Table) 형태로 노출되며, 주요 컬럼은 다음과 같습니다.

| 컬럼명 | 설명 |
| ------ | ---- |
| 상태 | Approved(Badge) 또는 Rejected(Badge) |
| 제출ID | 유일한 제출 식별 코드 (예: Q00000003A) |
| 유저닉네임 | 제출한 유저의 닉네임 |
| 반려사유 | 반려된 경우 선택된 사유명 노출 (승인 시 -) |
| 이미지보기 | 아이콘 버튼 형태로 제공. 클릭 시 유저가 제출한 원본 이미지를 팝업(오버레이)으로 표시 |
| 제출일시 | 유저가 제출한 시간 (YYYY.MM.DD hh:mm) |
| 처리일시 | 관리자가 처리 완료한 시간 (YYYY.MM.DD hh:mm) |
| 처리자 | 처리를 수행한 관리자명 |

- 데이터 없음: 테이블 중앙에 "처리내역이 없습니다." 문구 노출
- 페이지네이션: 하단 중앙 10페이지 단위 이동

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 이미지 보기

- 이미지보기 컬럼의 아이콘 버튼을 클릭하면, 해당 제출 건의 유저가 업로드한 원본 이미지를 팝업(오버레이) 형태로 표시합니다.
- Row 자체를 클릭하는 동작은 제공하지 않습니다.

### 3.2 네비게이션

- 기본정보 탭 클릭: [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) 화면으로 이동
- 대기내역 탭 클릭: [CEB-BO-FQ-202] 퀘스트 상세 (대기내역 탭) 화면으로 이동

---

## 4. 규칙/로직

### 4.1 정렬 및 필터링

- 정렬: 기본적으로 최신 처리일시 순(Desc)으로 정렬하여 최근 작업 내역을 상단에 보여줍니다.

---

## 5. 에러 및 예외상황

- 검색 결과 없음: 조건에 맞는 데이터가 없을 경우 리스트 영역에 "검색 결과가 없습니다." 안내 문구를 노출합니다.
