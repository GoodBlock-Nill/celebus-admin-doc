## 1. 화면 목적

특정 Raffle 이벤트에 참여한 모든 응모자(Participant)의 정보와 응모권 사용 내역(History)을 조회합니다.

### 주요 기능

- 유저별 누적 응모 횟수, 최근 응모 시점 등을 파악하여 이벤트 참여 현황 모니터링
- 특정 유저의 상세 이력 검증
- 추첨은 본 화면이 아닌 추첨내역 탭에서 진행되므로, 이곳은 순수하게 데이터 조회 및 모니터링에 집중

### 진입 경로

- [CEB-BO-FQ-204] Raffle 상세 (기본정보 탭) > 응모내역 탭 클릭

---

## 2. UI 설명

### 2.1 상단 탭 및 통계

- 탭 메뉴 (Tab Menu):
  - **기본정보**: Raffle 설정 정보
  - **응모내역 (Selected)**: 현재 화면
  - **추첨내역**: 당첨 결과 확인 및 추첨 실행

- 통계 대시보드 (Summary Card):
  - 누적 참여자 수 (Unique Participants): 중복을 제거한 순수 참여 유저 수 (단위: 명)
  - 누적 사용 응모권 합계 (Total Tickets Used): 모든 유저가 해당 Raffle에 사용한 응모권(Ticket)의 총합 (단위: 개/장)

### 2.2 검색 및 필터 영역

- 정렬 조건 (Sort Select Box):
  - 응모일시 최근 순 (Default): 가장 최근에 응모한 내역이 상단에 노출
  - 누적 응모권 많은 순: 가장 많이 응모한 '헤비 유저' 순으로 정렬
- 검색 조건 (Search Select Box): 닉네임, 전화번호
- 검색어 입력 (Input): Placeholder: "검색어를 입력하세요"
- [조회] 버튼: 설정된 정렬 및 검색 조건을 적용하여 리스트를 갱신합니다.

### 2.3 목록 영역 (Data Grid)

응모 내역은 테이블 형태로 제공되며, 각 행(Row)은 한 명의 유저(Participant) 정보를 나타냅니다.

| 컬럼명 | 데이터 예시 | 상세 설명 |
| ------ | ----------- | --------- |
| 유저닉네임 | Nill Yoo | 응모한 유저의 닉네임 텍스트 |
| 전화번호 | 821012345678 | 유저의 인증된 휴대전화 번호(국가번호 포함) |
| 최근 사용 응모권 | 5 | 유저가 가장 마지막으로 응모 버튼을 눌렀을 때 한 번에 사용한 티켓 수량 |
| 최근 응모일시 | 2025.09.12 15:23:01 | 마지막 응모가 이루어진 서버 시간 (YYYY.MM.DD HH:mm:ss) |
| 누적 사용 응모권 | 50 | 이 유저가 해당 Raffle에 투입한 총 티켓 수량 |

- Empty State: 검색 결과나 응모 내역이 없을 경우, 리스트 영역 중앙에 "응모내역이 없습니다." 메시지를 노출합니다.
- Pagination: 하단 중앙에 < Prev 1 2 3 ... 10 Next > 형태의 페이지 이동 컨트롤을 제공합니다.

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 상세 보기 (Row Click)

- Row Click: 리스트의 행 영역을 클릭하면 [CEB-BO-FQ-205-MD-DETAIL] 응모내역 상세 모달이 팝업됩니다.

### 3.2 네비게이션

- 기본정보 탭 클릭: [CEB-BO-FQ-204] Raffle 상세 (기본정보 탭) 화면으로 이동
- 추첨내역 탭 클릭: [CEB-BO-FQ-206] Raffle 상세 (추첨내역 탭) 화면으로 이동

---

## 4. 규칙/로직

### 4.1 리스트 조회 및 갱신

- 실시간성: Active 상태의 Raffle인 경우, 유저의 응모가 계속 발생하므로 관리자가 새로고침하거나 탭을 재진입할 때마다 통계 수치와 리스트가 최신화되어야 합니다.
- 중복 처리: 목록은 유저 단위로 Grouping 되어 보여집니다. 즉, 한 유저가 10번 나누어 응모했더라도 리스트에는 1개의 행(Row)으로 나오며, 누적 사용 응모권에 합산되어 표시됩니다.
