## 1. 화면 목적

Quest 생성 작업 중, 사용자가 [취소하기] 버튼을 눌렀을 때 작성 중이던 데이터 손실에 대한 주의를 주고, 최종적인 취소 의사를 확인합니다.

### 주요 기능

- 데이터 손실 경고 안내
- 생성 취소 최종 확인

### 진입 경로

- Quest 생성 화면(CEB-BO-FQ-101-Create)에서 [취소하기] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 경고 심볼(Warning Icon) 배치
- 타이틀: "Quest 생성 취소"
- 안내 메시지:
  - "입력된 데이터는 모두 사라집니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [다시 돌아가기]: 생성 취소를 철회하고, 기존 작성 화면으로 돌아가 작업을 계속
- [확인]: 작성 중이던 모든 데이터를 초기화하고, 상위 목록 화면으로 이동

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 버튼 동작

| 버튼 | 동작 |
| ---- | ---- |
| [다시 돌아가기] | 생성 취소를 철회하고 기존 작성 화면으로 복귀 |
| [확인] | 작성 중이던 모든 데이터를 초기화하고 Quest 목록(CEB-BO-FQ-101)으로 이동 |
| [X] 닫기 | 모달만 닫히며 작성 중이던 화면 상태 유지 |

---

## 4. 규칙/로직

### 4.1 취소 프로세스

- **진입**: Quest 생성 화면에서 [취소하기] 버튼 클릭 시 호출
- **확인(Confirm)**: [확인] 버튼 클릭 시, 클라이언트(브라우저) 상에 입력되어 있던 임시 데이터(타이틀, 설명, 가이드 등)를 모두 삭제합니다. Quest 목록 화면(CEB-BO-FQ-101)으로 리다이렉트됩니다.
- **철회(Cancel)**: [다시 돌아가기] 또는 우측 상단 [X] 버튼 클릭 시 모달만 닫히며, 작성 중이던 화면 상태는 그대로 유지됩니다.
