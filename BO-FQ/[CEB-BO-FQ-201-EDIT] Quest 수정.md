## 1. 화면 목적

이미 생성된 Quest의 정보를 변경합니다. Quest의 현재 진행 상태(임시저장(Draft) 또는 진행중(Active))에 따라 수정 가능한 항목의 범위를 엄격히 제어하여 운영의 유연성과 데이터 무결성을 동시에 확보합니다.

### 주요 기능

- Quest의 현재 진행 상태(임시저장(Draft) 또는 진행중(Active))에 따라 수정 가능한 항목 범위를 제어
- 임시저장(Draft) 상태: 기본 정보, 퀘스트 설정, 보상 설정 전체 수정 가능
- 진행중(Active) 상태: 기본 정보(타이틀, 설명, 유저 가이드)만 수정 가능, 나머지 항목은 비활성화
- 다국어(한국어/영어/일본어) 입력 패널을 통한 다국어 콘텐츠 수정 지원

### 진입 경로

- [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) > [수정하기] 버튼 클릭

### 2.2 화면 레이아웃

화면은 3개 섹션이 좌→중→우 순서로 배치된 3-column 레이아웃입니다.

```
+----------------------------+----------------------------+----------------------------+
|        기본 정보           |        퀘스트 설정          |        보상 설정            |
|                            |                            |                            |
|  타이틀  *      [입력완료]  |  아티스트  *    [Dropdown]  |  보상 종류  *   [Dropdown]  |
|  설명    *      [입력완료]  |  퀘스트 타입 * [Dropdown]  |  응모권 수량    [Dropdown]  |
|  유저 가이드 * [입력완료]  |  마감 일시  *  [DatePicker] |  NFT 이벤트    [Dropdown]  |
|  대표 이미지 *  [썸네일]   |  연관 링크     [+ 링크 추가]|                            |
|              [변경] [제거] |                            |                            |
|                            |  * 시작 일시는 게시 시점에  |                            |
|                            |    자동으로 설정됩니다.     |                            |
+----------------------------+----------------------------+----------------------------+
```

> 상세 스타일은 피그마 참조

### 2.3 기본 정보 섹션 (좌측)

Quest의 콘텐츠 정보를 표시하고 수정하는 영역입니다. 진행중(Active) 상태에서도 수정 가능합니다.

| 요소명 | 타입 | 필수 | 설명 |
| ------ | ---- | ---- | ---- |
| 타이틀 | Expandable Button (다국어 입력 패널) | Y | 입력 상태 버튼 클릭 시 다국어 입력 패널 전개. 최대 100자 |
| 설명 | Expandable Button (다국어 입력 패널) | Y | 입력 상태 버튼 클릭 시 다국어 입력 패널 전개. 최대 200자 |
| 유저 가이드 | Expandable Button (다국어 입력 패널) | Y | 입력 상태 버튼 클릭 시 다국어 입력 패널 전개. 최대 500자 |
| 대표 이미지 | Image Uploader | Y | 현재 등록된 이미지 썸네일 표시. [변경] 버튼으로 이미지 교체, [제거] 버튼으로 이미지 삭제 |

- 각 입력 항목(타이틀, 설명, 유저 가이드) 우측에 입력 상태 버튼이 표시됩니다.
  - 입력완료: 기존 데이터가 존재하거나 수정이 완료된 상태
  - 입력중...: 현재 입력 패널이 열려 편집 중인 상태

### 2.4 퀘스트 설정 섹션 (중앙)

Quest의 운영 규칙을 설정하는 영역입니다. 진행중(Active) 상태에서는 비활성화됩니다.

| 요소명 | 타입 | 필수 | 설명 |
| ------ | ---- | ---- | ---- |
| 아티스트 | Dropdown | Y | 등록된 아티스트 목록에서 선택 |
| 퀘스트 타입 | Dropdown | Y | Quest 참여 타입 선택 (이미지 촬영 및 업로드) |
| 마감 일시 | Date Picker + Time Picker | Y | Quest 마감 날짜 및 시간 설정 |

- 마감 일시 하단에 안내 문구를 표시합니다: "* 시작 일시는 게시 시점에 자동으로 설정됩니다."

#### 연관 링크 (선택)

유저가 퀘스트를 수행할 때 참고할 수 있는 외부 링크를 등록합니다. 등록된 링크는 앱에서 라벨 버튼 형태로 노출됩니다. 최대 2개까지 등록 가능하며, 미입력 시 앱에 링크 버튼이 노출되지 않습니다.

| 요소명 | 타입 | 필수 | 글자수 제한 | 설명 |
| ------ | ---- | ---- | ----------- | ---- |
| 링크 라벨 | Text Input + 언어 탭(KO/EN/JA) | 조건부 | 각 20자 | 앱에서 버튼 텍스트로 표시되는 라벨. URL 입력 시 3개 언어 모두 필수 |
| 링크 URL | Text Input | 조건부 | 500자 | 연결할 외부 URL. 라벨 입력 시 필수. https:// 또는 http:// 형식 |
| [+ 링크 추가] | Text Button | - | - | 새 링크 입력 행 추가 (최대 2개까지, 2개 도달 시 버튼 비활성화) |
| [삭제] | Icon Button (X) | - | - | 해당 링크 행 삭제 |

- 초기 상태: 링크 입력 행 없음, [+ 링크 추가] 버튼만 표시
- 링크 추가 시: 라벨(KO/EN/JA) + URL 입력 필드가 한 세트로 추가됨

## 3. 화면 내 기능설명

### 3.1 버튼 동작

- **[취소하기]**: 수정 작업을 중단하고 이전 화면으로 돌아갑니다. 클릭 시 [CEB-BO-FQ-201-MD-CANCEL] Quest 수정취소 확인 모달을 호출합니다.
- **[수정하기]**: 변경된 내용을 서버에 반영합니다. 수정 가능한 필수 항목이 모두 유효한 상태일 때만 활성화됩니다. 클릭 시 [CEB-BO-FQ-201-MD-COMPLE] Quest 수정완료 확인 모달을 호출합니다.

### 3.2 네비게이션

- 취소 시: [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭)으로 복귀
- 수정 완료 시: [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭)으로 복귀

## 4. 규칙/로직

### 4.1 상태별 수정 권한 (Permission Logic)

시스템은 화면 진입 시 Quest의 상태 코드를 확인하여 각 섹션 및 필드의 활성/비활성 여부를 결정합니다.

| 항목 | 임시저장(Draft) | 진행중(Active) | 비고 |
| ---- | --------------- | -------------- | ---- |
| 타이틀 | 수정 가능 | 수정 가능 | 오타 수정, 추가 안내 등 운영 대응 필요 |
| 설명 | 수정 가능 | 수정 가능 | 오타 수정, 추가 안내 등 운영 대응 필요 |
| 유저 가이드 | 수정 가능 | 수정 가능 | 오타 수정, 추가 안내 등 운영 대응 필요 |
| 대표 이미지 | 수정 가능 | 수정 불가 | 진행 중 이미지 변경 시 혼란 방지 |
| 아티스트 | 수정 가능 | 수정 불가 | 진행 주체 변경 불가 |
| 퀘스트 타입 | 수정 가능 | 수정 불가 | 진행 중 참여 방식 변경 불가 |
| 마감 일시 | 수정 가능 | 수정 불가 | 참여 기간 임의 변경 방지 (공정성) |
| 보상 종류 | 수정 가능 | 수정 불가 | 참여 후 보상 조건 변경 불가 |
| 응모권 수량 | 수정 가능 | 수정 불가 | 참여 후 보상 수량 변경 불가 |
| NFT 이벤트 | 수정 가능 | 수정 불가 | 참여 후 보상 이벤트 변경 불가 |
| 연관 링크 | 수정 가능 | 수정 가능 | 운영 중 링크 변경/추가 필요 가능 |

### 4.2 다국어 입력 버튼 상태

| 상태 | 버튼 텍스트 | 조건 |
| ---- | ----------- | ---- |
| 미입력 | 입력하기 | 해당 항목 입력을 시작하지 않은 상태 |
| 입력중 | 입력중... | 입력 패널이 열려있고 입력 진행 중인 상태 |
| 입력완료 | 입력완료 | 3개 언어 모두 입력 및 저장 완료된 상태 |

### 4.3 다국어 입력 패널 진행률 표시

| 표시 | 의미 |
| ---- | ---- |
| (0/3) | 아무 언어도 입력되지 않음 |
| (1/3) | 1개 언어 입력 완료 |
| (2/3) | 2개 언어 입력 완료 |
| (3/3) | 3개 언어 모두 입력 완료 |

### 4.4 다국어 입력 저장 시 유효성 검사

| 상태 | 메시지 |
| ---- | ------ |
| 미입력 언어 존재 | "*모든 언어를 빠짐 없이 입력해주세요" |
| 모든 언어 입력 완료 | "*모든 언어가 성공적으로 저장되었습니다" |

### 4.5 보상 설정 동적 UI 로직

- 선택한 보상 종류에 따라 하위 항목이 동적으로 노출됩니다.

| 보상 종류 선택 | 응모권 수량 노출 | NFT 이벤트 노출 |
| -------------- | --------------- | --------------- |
| 응모권 + NFT | O | O |
| 응모권 | O | X |
| NFT | X | O |

### 4.6 마감 일시 수정 로직 (임시저장(Draft) 한정)

- 마감 일시는 현재 시간(수정 시점)보다 미래의 시간으로만 설정 가능합니다.
  - 현재일자 +1일 부터 선택 가능
  - 시간: 00~23, 분: 00~59

### 4.7 연관 링크 유효성 검사

- 링크는 0~2개 입력 가능 (선택 사항)
- 링크 입력 시 라벨(3개 언어 모두) + URL 쌍으로 입력 필수
  - 라벨만 입력하고 URL 미입력 시: "URL을 입력해주세요" 인라인 에러
  - URL만 입력하고 라벨 미입력 시: "라벨을 입력해주세요" 인라인 에러
- URL 형식 검증: https:// 또는 http:// 로 시작해야 함
  - 형식 불일치 시: "올바른 URL 형식을 입력해주세요 (예: https://example.com)" 인라인 에러
- 연관 링크 미입력 시에도 [수정하기] 버튼 활성화에 영향 없음

### 4.8 수정하기 버튼 활성화 조건

- 수정 가능한 필수 항목이 모두 유효한 값을 가져야 [수정하기] 버튼이 활성화됩니다.

| 상태 | 활성화 조건 |
| ---- | ----------- |
| 임시저장(Draft) | 타이틀/설명/유저 가이드(3개 언어 모두), 대표 이미지, 아티스트, 퀘스트 타입, 마감 일시, 보상 종류 및 관련 항목 모두 입력 완료 |
| 진행중(Active) | 타이틀/설명/유저 가이드(3개 언어 모두) 입력 완료 |

- 연관 링크: 선택 항목이므로 활성화 조건에 포함되지 않음. 단, 입력한 경우 유효성 검사를 통과해야 함

### 4.9 데이터 연동

- NFT 민팅 이벤트 리스트: NFT 관련 옵션 선택 시, 연동된 BIVE 시스템의 '진행 가능한 NFT 민팅 이벤트' 목록을 API로 조회하여 바인딩합니다.
