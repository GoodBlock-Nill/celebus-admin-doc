## 1. 화면 목적

유저가 해당 퀘스트 미션을 수행하고 제출한 결과물(이미지 등) 중, 아직 검수되지 않은 대기(Pending) 상태의 건들을 확인하고 검수합니다.

### 주요 기능

- 대기(Pending) 상태의 유저 제출물 목록 조회
- 제출물에 대한 개별 승인(Approve) 또는 반려(Reject) 처리
- 현재 페이지 전체 일괄 승인 처리

### 진입 경로

- [CEB-BO-FQ-201] 퀘스트 상세 (기본정보 탭) > 대기내역 탭 클릭

---

## 2. UI 설명

### 2.1 상단 영역

- 탭 메뉴: 기본정보, **대기내역 (Selected)**, 처리내역
- [현재 페이지 모두 승인] 버튼:
  - 위치: 우측 상단
  - 기능: 현재 화면에 노출된(Pagination된) 모든 대기 건을 일괄 승인 처리합니다.
  - 내역이 없을 경우 비활성화

### 2.2 목록 영역 (Grid List)

제출된 결과물은 카드(Card) 형태의 그리드로 나열됩니다.

- 정렬방식: 오래된순
- 페이지당 최대 10건 노출
- 데이터 없음: "제출내역이 없습니다" 문구 노출

데이터 있음 시 각 카드에 다음 정보가 표시됩니다:

- 이미지 썸네일: 유저가 업로드한 원본 이미지 (클릭 시 확대/원본 보기 가능)
- 유저 정보: {유저닉네임}, User ID 등 식별 정보
- 제출 정보: Submit ID, 제출일시(YYYY.MM.DD hh:mm)
- 액션 버튼: [반려], [승인]

### 2.3 페이지네이션

- 하단 중앙에 위치하며 10페이지 단위로 이동 가능합니다.

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 승인 처리 (Approve)

- 개별 승인: 카드 내 [승인] 버튼 클릭 시, 별도의 확인 절차 없이 즉시 승인 처리됩니다.
  - 상태가 Approved로 변경되며, 해당 카드는 대기내역 탭에서 사라지고 처리내역 탭으로 이동합니다.
  - 유저에게 보상이 즉시 지급됩니다.
  - 알림메시지 및 푸시알림 전송
  - 알림 내용은 피그마 참조
  - Fan Quest 내역에 응모권 지급
  - 콜렉션에 NFT 지급
- 일괄 승인: [현재 페이지 모두 승인] 클릭 시, 현재 페이지에 있는 모든 건을 한 번에 승인합니다.

### 3.2 반려 처리 (Reject)

- 개별 반려: 카드 내 [반려] 버튼 클릭 시, 즉시 처리되지 않고 반려 사유 입력 모달([CEB-BO-FQ-202-MD-REJECT])을 호출합니다.
  - 모달에서 사유 선택 및 입력 후 최종 확인 시 반려 처리됩니다.
  - 상태가 Rejected로 변경되며 처리내역 탭으로 이동합니다.
