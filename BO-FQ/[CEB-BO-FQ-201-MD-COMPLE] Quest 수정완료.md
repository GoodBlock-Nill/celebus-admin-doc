## 1. 화면 목적

Quest 수정 화면에서 수정 작업이 완료된 후, 관리자의 최종 수정 확정 의사를 확인하고 실제 데이터 업데이트를 요청합니다.

### 주요 기능

- Quest 수정 최종 확인
- 서버 데이터 업데이트 요청

### 진입 경로

- Quest 수정 화면(CEB-BO-FQ-201-EDIT)에서 [수정하기] 버튼 클릭 시 호출

---

## 2. UI 설명

### 모달 구조

- 아이콘: 상단 중앙에 번개 모양 아이콘 배치
- 타이틀: "Quest 수정"
- 안내 메시지:
  - "확인 버튼을 누르면 수정사항이 적용됩니다"
  - "계속 진행하시겠습니까?"

**하단 액션 버튼**

- [취소]: 수정 프로세스를 보류하고, 수정 화면으로 돌아가 내용을 재검토하거나 수정
- [확인]: 변경된 모든 데이터를 서버로 전송하여 Quest 정보를 업데이트

> 상세 스타일은 피그마 참조

---

## 3. 화면 내 기능설명

### 3.1 버튼 동작

| 버튼 | 동작 |
| ---- | ---- |
| [취소] | 수정 프로세스를 보류하고 수정 화면으로 복귀 |
| [확인] | 변경된 모든 데이터를 서버로 전송하여 Quest 정보 업데이트 |

---

## 4. 규칙/로직

### 4.1 수정 프로세스

- **진입**: Quest 수정 화면(CEB-BO-FQ-201-EDIT)에서 [수정하기] 버튼 클릭 시 호출
- **확인(Confirm)**: [확인] 버튼 클릭 시 서버로 수정 요청을 전송합니다. 수정 성공 시 Quest 상세 화면(CEB-BO-FQ-201)으로 이동합니다.
- **철회(Cancel)**: [취소] 버튼 클릭 시 모달만 닫히며, 수정 중이던 화면 상태는 그대로 유지됩니다.

---

## 5. 에러 및 예외상황

### 5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

**성공 (Success)**

- 메시지: "요청성공 - Quest 수정이 완료되었습니다."
- 토스트 타입: Success
- 후행 동작: 모달이 닫히고, Quest 상세 화면(CEB-BO-FQ-201)으로 이동

**실패 (Failure)**

- 메시지: "요청실패 - Quest 수정에 실패하였습니다. 다시 시도하세요"
- 토스트 타입: Error
- 원인 예시: 네트워크 오류, 필수 데이터 누락 등
- 후행 동작: 모달이 닫히고 수정 화면이 유지되어 재시도할 수 있습니다.
