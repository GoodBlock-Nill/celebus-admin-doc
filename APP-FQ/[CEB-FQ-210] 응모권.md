## 1. 화면 목적

유저가 보유한 응모권(티켓) 수량을 확인하고, 적립/사용 거래 이력을 조회하는 화면입니다.

##### 주요 기능
- 보유 응모권 수량 실시간 확인
- 응모권 구매 기능 (GP 사용)
- 적립 이력 조회 (Fan Quest 보상, Survival Trivia 탈락 보상, Prediction Market 보상, 응모권 구매)
- 사용 이력 조회 (Raffle 응모)

##### 접근 경로

- 홈 메인(CEB-HOM-501-FQ) > 환영 배너 영역에서 진입

## 2. UI 설명

### 2.1 공통 레이아웃

```text
┌──────────────────────────────────┐
│ Status Bar                       │
├──────────────────────────────────┤
│ [<] 응모권                       │
├──────────────────────────────────┤
│ 🎫 보유 응모권  {N}  [구매하기]  │
│ (강조 카드)                       │
├──────────────────────────────────┤
│ [적립] [사용]                    │
├──────────────────────────────────┤
│ 이력 리스트 (무한 스크롤)         │
│                                  │
│ [{아티스트}] QUEST               │
│ {Quest title}               +{N} │
│ YYYY.MM.DD hh:mm:ss              │
│──────────────────────────────────│
│ Survival Trivia                  │
│ {Game title}                +{N} │
│ YYYY.MM.DD hh:mm:ss              │
│──────────────────────────────────│
│ {N} Game point 사용              │
│ 응모권 구매                  +{N} │
│ YYYY.MM.DD hh:mm:ss              │
└──────────────────────────────────┘
```

### 2.2 영역별 구성요소

##### Header 영역

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | [<] 뒤로가기 버튼 (이전 화면으로 복귀) |
| 타이틀 | "응모권" |

##### 보유 응모권 카드

| 항목 | 내용 |
| ---- | ---- |
| 아이콘 | 🎫 응모권 아이콘 |
| 수량 | 서버에서 호출한 유저의 보유 응모권 수량 (실시간 표시) |
| 표시 형식 | "보유 응모권" 레이블 + 수량 (Big Font) |
| 구매하기 버튼 | 카드 우측에 위치. 탭 시 응모권 구매 화면(Buy_RF_Ticket)으로 이동 |

> 상세 스타일은 피그마 참조

##### 탭 메뉴

| 항목 | 내용 |
| ---- | ---- |
| 구조 | 상단 고정 탭 (2개) |
| Tab 1 | 적립 (+ 변동만) |
| Tab 2 | 사용 (- 변동만) |
| 초기 상태 | 적립 탭이 기본 선택된 상태로 진입 |

##### 이력 리스트

| 요소 | 설명 |
| ---- | ---- |
| 서브 타이틀 | 출처/유형 정보 (예: [{아티스트}] QUEST, Survival Trivia, Prediction Market, {N} Game point 사용, [{아티스트}] RAFFLE). 회색, 소문자 |
| 메인 타이틀 | 콘텐츠명 또는 이벤트명. 흰색, 굵게 |
| 날짜/시간 | YYYY.MM.DD hh:mm:ss 형식 |
| 우측 수량 | 적립 +{N} / 사용 -{N} (보라색) |
| 구분선 | 아이템 간 얇은 수평 구분선 |
| 탭 이벤트 | 없음 (상세 페이지 없음) |

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 [<] 뒤로가기

- 이전 화면(홈 메인)으로 복귀

### 3.2 구매하기 버튼

- 보유 응모권 카드 내 [구매하기] 버튼 탭 시 응모권 구매 화면(Buy_RF_Ticket)으로 이동
- GP를 사용하여 응모권 구매 (교환 비율은 추후 결정)

### 3.3 탭 전환

- 적립 / 사용 탭 전환 시 해당 조건에 맞는 이력만 필터링하여 표시
- 탭 전환 시 리스트 최상단으로 스크롤 초기화

### 3.4 리스트 스크롤

- 무한 스크롤 (20건씩 추가 로드)
- 스크롤 바닥 도달 시 추가 데이터 로딩

---

## 4. 규칙/로직

### 4.1 탭별 필터

| 탭 | 표시 내용 |
| ---- | ---- |
| 적립 | + 변동만 (Quest 보상, ST 탈락 보상, PM 보상, 응모권 구매) |
| 사용 | - 변동만 (Raffle 응모) |

### 4.2 적립 탭 이력 항목 유형

| 유형 | 서브 타이틀 | 메인 타이틀 | 날짜/시간 | 금액 |
| ---- | ---- | ---- | ---- | ---- |
| Quest | [{아티스트}] QUEST | {Quest title} | YYYY.MM.DD hh:mm:ss | +{N} (보라색) |
| Survival Trivia | Survival Trivia | {Game title} | YYYY.MM.DD hh:mm:ss | +{N} (보라색) |
| Prediction Market | Prediction Market | {Game title} | YYYY.MM.DD hh:mm:ss | +{N} (보라색) |
| 응모권 구매 | {N} Game point 사용 | 응모권 구매 | YYYY.MM.DD hh:mm:ss | +{N} (보라색) |

- Quest 유형의 서브 타이틀에는 아티스트명 포함
- 응모권 구매 유형: GP를 사용하여 응모권을 구매한 경우. 서브 타이틀에 사용된 GP 수량 표시

### 4.3 사용 탭 이력 항목 유형

| 유형 | 서브 타이틀 | 메인 타이틀 | 날짜/시간 | 금액 |
| ---- | ---- | ---- | ---- | ---- |
| Raffle 응모 | [{아티스트}] RAFFLE | {Raffle title} | YYYY.MM.DD hh:mm:ss | -{N} (보라색) |

- 현재 Raffle 응모 시 차감되는 케이스만 존재

### 4.4 정렬 기준

- 최신 일시순 (desc)

### 4.5 검색

- 없음

### 4.6 페이지네이션

- 무한 스크롤 (20건씩 추가 로드)
