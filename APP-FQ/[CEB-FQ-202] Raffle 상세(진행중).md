## 1. 화면 목적

진행 중인(Active) 래플 이벤트의 상세 정보와 혜택을 유저에게 매력적으로 전달하고, 보유한 티켓을 사용하여 원하는 수량만큼 응모하도록 유도하는 화면입니다.

##### 주요 기능
- 래플 기본정보 조회 (아티스트, 타이틀, D-day, 설명, 보상 내역)
- 참여 통계 제공 (누적 참여 유저 수, 누적 응모권)
- 내 참여 현황 조회 (내 누적 응모권)
- 응모권 수량 입력 및 응모하기 플로우
- 래플 상태별 분기 처리 (Active / Closed / Ended)

##### 접근 경로
- 홈 메인(CEB-HOM-501-FQ) > Event > 래플 카드 탭 시 이동
- 딥링크를 통한 직접 진입

+------------------------------------------------------------------+
| 참여 통계 영역                                                    |

### 2.2 영역별 구성요소

##### Header 영역

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | [<] 뒤로가기 버튼 (이전 화면으로 이동) |

##### 기본 정보 영역

| 항목 | 내용 |

| 타이틀 | 어드민 [CEB-BO-FQ-102-CREATE] 타이틀 |

| 설명 | 어드민 [CEB-BO-FQ-102-CREATE] 설명 (2줄 초과 시 더보기/접기 버튼 제공) |

##### 보상 정보 영역

| 항목 | 내용 |
| ---- | ---- |

| 항목 | 내용 |
| ---- | ---- |
| 누적 참여 유저 수 | 현재까지 응모한 순수 유저 수 (중복 제외) |
| 누적 응모권 | 전체 유저가 사용한 총 응모권 수 |

##### 내 참여 현황 영역

| 항목 | 내용 |
| ---- | ---- |
| 내 누적 응모권 | 유저 본인이 해당 Raffle에 사용한 총 응모권 수 (없으면 0장) |

##### 응모 영역 (하단 고정)

| 항목 | 내용 |
| ---- | ---- |
| 사용 가능한 응모권 | 현재 유저가 보유한 잔여 응모권 수량 |
| 수량 입력 (Stepper) | 숫자(Input) + [-]/[+] 컨트롤 (디폴트값: 1, 최소: 1). 사용 가능 응모권이 0장인 경우: 값 0, disabled 상태 (회색 처리) |
| [응모하기] 버튼 | 응모 API 호출. 보유량 부족 시 비활성화 또는 경고 |

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 초기 진입 시 상태 체크

서버에서 해당 래플의 상태(status)를 확인하여 분기 처리합니다.

| 래플 상태 | 처리 |
| ---- | ---- |
| Active (진행중) | 본 화면 유지 |
| Closed (마감, 집계중) | MyTicket 리스트에서 "현재 집계 진행중입니다" 토스트 표시 |
| Ended (발표완료, 당첨) | [CEB-FQ-203] Raffle 상세(당첨) 화면 표시 |
| Ended (발표완료, 미당첨) | [CEB-FQ-204] Raffle 상세(미당첨) 화면 표시 |

##### 데이터 바인딩

- 유저의 `current_entry_count` (내 응모 수)를 불러와 [내 참여 현황] 영역에 표시
- 유저의 `available_ticket_balance` (보유 티켓)를 불러와 [응모 영역] 하단에 표시

### 3.2 응모 수량 입력 (Input)

| 항목 | 규칙 |
| ---- | ---- |
| 기본값 | 1 (유저가 보유한 티켓이 1장 이상일 때) |
| 최소값 | 1 (0 입력 시 자동으로 1로 보정) |
| 최대값 | 유저의 사용 가능한 응모권 수량 |
| 입력 방식 | [+]/[-] 버튼 터치로 증감 / 숫자 영역 터치 시 키패드 활성화되어 직접 입력 가능 |
| 비활성 조건 | 사용 가능한 응모권이 0장일 경우, Input은 0으로 고정 + Disabled 처리, [응모하기] 버튼도 비활성화 |
| 1인당 최대 응모 한도 | 없음 (Max Cap 정책 없음) |

##### 유효성 검사 (Validation)

- 보유 티켓이 0장이거나, 입력값이 보유량을 초과하면 [응모하기] 버튼을 비활성화

### 3.3 응모하기 버튼 동작

- 조건: 입력된 수량이 유효할 때 활성화
- 클릭 시: [CEB-FQ-202-MD] Raffle 응모 모달을 호출
- 파라미터로 입력된 수량(n)을 전달

## 4. 규칙/로직

### 4.1 데이터 노출 조건

| 항목 | 노출 조건 |
| ---- | ---- |
| 설명 더보기/접기 | 설명 텍스트가 2줄을 초과할 경우에만 더보기 버튼 표시 |
| 내 참여 현황 | 로그인 유저에게만 표시 |
| 응모 영역 | Active 상태에서만 활성화 |

### 4.2 상세 화면 체류 중 이벤트 마감 처리

유저가 상세 화면에 머무는 동안 서버에서 Raffle 상태가 진행중(Active) → 추첨대기(Closed)로 변경된 경우:

- 유저가 [응모하기] 버튼 클릭 시 서버에서 EVENT_CLOSED 응답 반환
- 알림 모달 표시: "이벤트가 종료되었습니다"
- [확인] 버튼 클릭 시 이전 탭(Event)으로 복귀
- 예시: 상세화면 진입했는데 D-day 에서 다음날 00시로 넘어갔을 때

### 4.3 반응형 고려사항

- 최대 화면 너비: 768px
- 768px 초과 시 중앙 정렬, 좌우 여백 자동 조절

### 4.4 다국어 처리

| 항목 | 다국어 범위 |
| ---- | ---- |
| 아티스트명 | 서버 데이터 (다국어 지원 여부 어드민 설정에 따름) |
| 타이틀 | 서버 데이터 (다국어 지원 여부 어드민 설정에 따름) |
| 설명 | 서버 데이터 (다국어 지원 여부 어드민 설정에 따름) |
