## 1. 화면 목적

Raffle 추첨 결과, 당첨된 유저(Winner)에게 명확한 축하 메시지와 당첨 사실을 전달하는 화면입니다. 경품 수령 방식(현장/배송/온라인)에 따라 유저가 수행해야 할 필수 행동(Action)을 가이드하며, 긍정적인 경험(WOW Moment)을 제공하여 서비스 만족도를 높입니다.

##### 주요 기능
- 당첨 축하 메시지 및 컨페티 애니메이션 표시
- 당첨 경품 정보(경품명, 이미지, 수량) 표시
- 수령 방식별(현장/배송/온라인) 안내 및 폼 제공
- 배송 수령 시 배송지 정보 입력 폼 제공

##### 접근 경로
- Event(CEB-FQ-201) > Raffle 탭 > '당첨' 리스트 클릭
- 당첨 Push 알림 클릭

## 3. 화면 내 기능설명

### 뒤로가기

##### 트리거
- Header 좌측 [<] 버튼 탭

##### 동작
1. 이전 화면으로 복귀
2. 네비게이션 스택에서 현재 화면 제거

> **주의**: Type B(배송) 당첨자가 배송지 미입력 상태로 뒤로가기 시 컨펌 팝업 노출 (3.5절 참조)

### 컨페티 애니메이션 (Confetti)

##### 동작
- 화면 진입(OnLoad) 시점에 1회성으로 재생
- 화려한 컨페티 효과로 당첨의 기쁨을 시각적으로 강조
- 재진입 시에도 매번 1회 재생

### 당첨 경품 정보

##### 표시 규칙
- 경품명: 운영자가 등록한 상품명 표시 (다국어 지원)
- 썸네일: 경품 이미지 (없을 경우 기본 이미지)
- 수량: 기본 1개

### 수령 방식별 UI 분기

##### 분기 기준
- 서버에서 내려주는 `receive_type` 값에 따라 UI 구성

##### Case 1: 현장 수령 (ONSITE)
- Guide 영역: 운영자가 등록한 수령 가이드 텍스트를 줄바꿈하여 그대로 노출
- 표시 항목: 수령 장소, 일시, 지참물(신분증 등)
- Action 버튼: "확인" → Event(CEB-FQ-201) > Raffle 탭으로 이동

##### Case 2: 배송 수령 (DELIVERY)
- Guide 영역: "경품 배송을 위해 주소를 입력해주세요." 문구 노출
- Form 영역: 활성화 (배송 정보 입력 폼)
- 기존에 저장된 배송지가 있다면 Default Value로 바인딩
- Action 버튼: "배송지 입력 완료"
  - 클릭 시 배송 정보 저장 API 호출
  - 성공 시 "저장되었습니다" Success 토스트 후 이동
- 입력 기한 만료 시: 폼 비활성화 + "입력 기간이 종료되어 당첨이 취소되었어요." 에러 메시지 노출

##### Case 3: 온라인 지급 (ONLINE)
- Guide 영역: 추후 정의 예정 (Phase 2 범위, 추후 정의 예정)
- Action 버튼: "확인"

### 배송 정보 입력 폼 (Type B 전용)

##### 필드 구성

| 필드명 | 필수 여부 | 입력 타입 | 비고 |
| ------ | --------- | --------- | ---- |
| 수령인 성함 | 필수 | 텍스트 | |
| 연락처 | 필수 | 전화번호 | |
| 주소 | 필수 | 주소검색 | |
| 상세주소 | 필수 | 텍스트 | |
| 배송메모 | 선택 | 텍스트 | |

##### 폼 검증
- 필수 필드 미입력 시 "배송지 입력 완료" 버튼 비활성화
- 모든 필수 필드 입력 완료 시 버튼 활성화

### 확인/제출 버튼

##### Type별 동작

| receive_type | 버튼 텍스트 | 동작 |
| ------------ | ----------- | ---- |
| ONSITE | 확인 | Event(CEB-FQ-201) > Raffle 탭으로 이동 |
| DELIVERY | 배송지 입력 완료 | 배송 정보 저장 API 호출 → 성공 시 이동 |
| ONLINE | 확인 | 추후 정의 예정 |

## 4. 규칙/로직

### 진입 조건 및 상태 체크

##### 유효성 검증
1. 서버에서 해당 유저의 `is_winner` 값이 `true`인지 반드시 재검증
2. `false`인 경우 미당첨 화면(CEB-FQ-204)으로 즉시 이동(Replace)

### 화면 진입 시 동작

##### 데이터 로딩
1. Raffle ID를 파라미터로 전달받음
2. 서버에서 당첨 정보 조회 (`is_winner`, `receive_type`, 경품 정보)
3. Loading 상태 표시 (Skeleton UI)
4. 데이터 로딩 완료 후 화면 렌더링
5. 컨페티 애니메이션 재생

### 다국어 처리

##### 지원 언어
- 한국어 (KO)
- 영어 (EN)
- 일본어 (JP)

##### 적용 대상
- 축하 문구
- 경품명
- 수령 안내 텍스트
- 버튼 텍스트

##### 표시 규칙
- 사용자 앱 언어 설정에 따라 자동 선택
- 해당 언어 데이터가 없는 경우 기본 언어(KO) 표시

### 배송지 입력 기한

##### 규칙
- 입력 기한: 결과 발표 후 운영자 설정 기간 (예: 7일)
- 기한 내: 폼 활성화, 입력 및 수정 가능
- 기한 만료 후: 폼 비활성화, "입력 기간이 종료되어 당첨이 취소되었어요." 안내 문구 노출, "배송지 입력 완료" 버튼 비활성화

## 5. 에러 및 예외상황

### 배송지 미입력 이탈

##### 상황
- 배송형(DELIVERY) 경품 당첨자가 정보를 입력하지 않고 뒤로가기를 누를 때

##### 처리
1. 컨펌 팝업 노출
2. 팝업 메시지: "기한내 배송지 정보를 입력해야 상품수령이 가능해요. 입력하지 않고 나가시겠어요?"
3. [나가기] 버튼: 이전 화면으로 이동
4. [계속 입력] 버튼: 팝업 닫기, 폼으로 복귀

### 입력 기한 만료

##### 상황
- 당첨자 정보 입력 기한이 지난 후 접속

##### 처리
1. 입력 폼 대신 안내 문구 표시: "입력 기간이 종료되어 당첨이 취소되었어요."
2. "배송지 입력 완료" 버튼 비활성화
3. "확인" 버튼만 활성 → Event으로 이동

### 데이터 로딩 실패

##### 상황
- API 호출 실패 또는 네트워크 오류

##### 처리
1. Error 상태 표시
2. 에러 메시지: "정보를 불러오는 데 실패했습니다."
3. [다시 시도] 버튼 제공
4. 버튼 탭 시 데이터 재로딩

### 배송 정보 저장 실패

##### 상황
- 배송 정보 저장 API 호출 실패

##### 처리
1. Error 토스트: "저장에 실패했습니다. 다시 시도해주세요."
2. 화면 유지 (폼 데이터 보존)

### 당첨자 검증 실패

##### 상황
- 진입 시 `is_winner` API 재조회 결과 `false`인 경우

##### 처리
1. 미당첨 화면(CEB-FQ-204)으로 즉시 이동(Replace)
2. 에러 토스트 없음 (자연스러운 화면 전환)

### Loading 상태

##### 처리
1. Skeleton UI 표시
   - 축하 이미지: 회색 사각형
   - 축하 문구: 회색 박스 (1줄)
   - 경품 정보: 회색 박스 (썸네일 + 텍스트)
   - 수령 안내: 회색 박스 (3줄)
2. 로딩 완료 시 실제 데이터로 전환 (페이드인 애니메이션)

### 네트워크 연결 끊김

##### 상황
- 오프라인 상태에서 화면 진입 또는 데이터 로딩 시도

##### 처리
1. Error 상태 표시
2. 에러 메시지: "네트워크 연결을 확인해주세요."
3. [다시 시도] 버튼 제공
4. 온라인 복구 시 자동 재로딩
