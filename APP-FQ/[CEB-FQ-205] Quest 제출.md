## 1. 화면 목적

유저가 퀘스트의 구체적인 수행 방법(가이드)을 확인하고, 요구사항에 맞는 이미지(인증샷)를 촬영하거나 업로드하여, 운영진에게 심사를 요청(제출)하는 화면입니다.

##### 주요 기능
- 퀘스트 기본정보 조회 (타이틀, 설명)
- 이미지 촬영 또는 갤러리 업로드
- 업로드 사진 가이드 확인 (아코디언)
- 제출하기 플로우 (심사 요청)

##### 접근 경로
- 홈 메인(CEB-HOM-501-FQ) > Event > Quest 카드 탭 시 이동
- 홈 메인(CEB-HOM-501-FQ) > QUESTS 섹션 카드 탭 시 이동
- 상태가 Available(참여가능) 또는 Rejected(반려)인 퀘스트만 진입 가능

## 2. UI 설명

### 2.1 공통 레이아웃

```text
+------------------------------------------------------------------+
| Status Bar                                                        |
+------------------------------------------------------------------+
| [<] Quest                                                         |
+------------------------------------------------------------------+
| Quest 정보 영역                                                   |
|   타이틀                                                          |
|   설명                                                            |
|   보상 정보 (아이콘 + 수량)                                        |
+------------------------------------------------------------------+
| 연관 링크 영역 (등록된 경우에만 노출)                                |
|   [라벨 버튼1]  [라벨 버튼2]                                       |
+------------------------------------------------------------------+
| 업로드 영역                                                       |
|   +------------------------+  +------------------------+          |
|   | [갤러리 아이콘]         |  | [카메라 아이콘]         |          |
|   |  갤러리에서 선택        |  |  사진 촬영하기          |          |
|   +------------------------+  +------------------------+          |
|                                                                   |
|   (업로드 후: 이미지 미리보기 + [X] 삭제 버튼)                     |
+------------------------------------------------------------------+
| 가이드 영역 (아코디언, 기본: 접힘)                                 |
|   [v] 업로드 사진 가이드                                          |
|   (펼침 시: 가이드 텍스트 내용)                                    |
+------------------------------------------------------------------+
| 하단 고정 바                                                      |
|   [제출하기] (이미지 미업로드 시 Disabled)                         |
+------------------------------------------------------------------+
```

### 2.2 영역별 구성요소

##### Header 영역

| 항목 | 내용 |
| ---- | ---- |
| 좌측 | [<] 뒤로가기 버튼 (이전 화면으로 이동) |
| 타이틀 | "Quest" |

##### Quest 정보 영역

| 항목 | 내용 |
| ---- | ---- |
| 타이틀 | 백오피스 [CEB-BO-FQ-101-Create] 타이틀 (유저 언어 기준) |
| 설명 | 백오피스 [CEB-BO-FQ-101-Create] 설명 (유저 언어 기준) |
| 보상 정보 | 승인 시 지급될 보상 아이콘 및 수량 (예: 응모권 1개, NFT) |

##### 연관 링크 영역

| 영역 | 항목 | 설명 |
| ---- | ---- | ---- |
| 연관 링크 | 링크 버튼 (최대 2개) | 백오피스(BO)에서 등록한 연관 링크를 라벨 버튼 형태로 표시. 클릭 시 인앱 브라우저 또는 외부 브라우저로 해당 URL 이동 |

- 연관 링크가 등록되지 않은 Quest의 경우 해당 영역 미노출
- 라벨 텍스트는 유저의 언어 설정(KO/EN/JA)에 맞춰 표시
- 연관 링크 데이터 출처: [CEB-BO-FQ-101-Create] Quest 생성 시 등록

##### 업로드 영역

| 항목 | 내용 |
| ---- | ---- |
| 초기 상태 | 갤러리 아이콘, 사진 촬영하기 아이콘 노출 |
| 업로드 후 | 선택한 이미지 미리보기 노출. 우측 상단 [X] (삭제) 버튼 제공 |

##### 가이드 영역 (아코디언)

| 항목 | 내용 |
| ---- | ---- |
| 타이틀 | "업로드 사진 가이드" + 펼치기/접기 화살표 |
| 기본 상태 | 접힌 상태 (타이틀만 표시) |
| 내용 | 어드민 [CEB-BO-FQ-101] 유저 가이드 텍스트 (예: 흐린 사진 반려 등) |

##### 하단 고정 바

| 항목 | 내용 |
| ---- | ---- |
| [제출하기] 버튼 | 초기엔 비활성(Dimmed). 이미지 업로드 완료 시 활성화(Active) |

> 상세 스타일은 피그마 참조

## 3. 화면 내 기능설명

### 3.1 초기 진입 및 유효성 체크

##### 진입 조건

| 퀘스트 상태 | 처리 |
| ---- | ---- |
| Available (참여가능) | 본 화면 진입 가능 |
| Rejected (반려) | 본 화면 진입 가능 (기존 반려 이미지는 초기화/삭제된 상태로 진입) |
| Pending (검토중) | 진입 불가 (리스트에서 토스트 처리) |
| Completed (완료) | 진입 불가 (리스트에서 토스트 처리) |

### 3.2 이미지 업로드 (Upload)

- 소스 선택: 업로드 영역 클릭 시 OS 기본 피커(카메라/갤러리)를 호출

##### 파일 제한

| 항목 | 규칙 |
| ---- | ---- |
| 형식 | JPG, PNG, HEIC |
| 용량 | 최대 10MB (초과 시 에러 처리) |

##### 상태 변화

| 상태 | 표시 |
| ---- | ---- |
| 이미지 선택 전 | Placeholder 노출, [제출하기] 버튼 Disabled |
| 이미지 선택 후 | Preview 노출, [제출하기] 버튼 Active |
| 이미지 삭제 시 | 다시 초기 상태(Placeholder)로 복귀 |

### 3.3 제출하기 버튼 동작

- 클릭 시: 중복 제출 방지를 위해 버튼을 로딩 상태(Spinner)로 변경하고 클릭을 막음
- API 호출: 이미지를 서버로 전송

| 결과 | 처리 |
| ---- | ---- |
| 성공 | "제출이 완료되었습니다. 심사 결과는 Event에서 확인해주세요." Success 토스트 노출 → 화면을 닫고 이전 리스트 화면으로 복귀 → 해당 퀘스트의 로컬 상태를 [심사 중]으로 업데이트 |
| 실패 | 에러 유형별 Error 토스트 노출 (5. 에러 및 예외상황 참조) |

> 제출 즉시 해당 건은 Admin의 검토 대기 목록([CEB-BO-FQ-202])에 등록됩니다.

| 항목 | 노출 조건 |
| ---- | ---- |

## 5. 에러 및 예외상황

##### 업로드 실패

| 상황 | 처리 |
| ---- | ---- |
| 네트워크 불안정 또는 파일 손상으로 업로드 실패 | "이미지 업로드에 실패했습니다. 다시 시도해주세요." Error 토스트 |

##### 용량 초과

| 상황 | 처리 |
| ---- | ---- |
| 제한 용량(10MB)을 초과하는 고화질 이미지 선택 | "10MB 이하의 이미지만 업로드 가능합니다." Error 토스트 |

##### 기기 권한 거부

| 상황 | 처리 |
| ---- | ---- |
| 유저가 카메라/갤러리 접근 권한을 거부한 상태 | "사진을 업로드하려면 권한 설정이 필요합니다." 안내 후 OS 설정 화면으로 이동 유도 |
