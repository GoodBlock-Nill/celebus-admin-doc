## 1. 화면 목적

GP와 CELB 간 교환 내역을 조회하고 관리하는 화면입니다. 관리자는 사용자들의 GP 충전(CELB→GP) 및 GP 출금(GP→CELB) 내역을 실시간으로 모니터링하고, 교환 현황을 통계로 확인할 수 있습니다.

### 주요 기능

- GP↔CELB 교환 내역 조회
- 교환 방향별(충전/출금) 필터링
- 오늘의 교환 현황 요약 (충전/출금/건수)
- 기간별, 사용자별 검색
- 교환 상세 정보 확인
- 교환 설정 페이지로 이동

### 접근 경로

```text
홈 > 게임존 > GP 교환소 > [교환 내역] 탭
```

## 3. 화면 내 기능설명

### 3.1 요약 카드

#### 오늘 GP 충전

- 계산식: SUM(CELB→GP 교환의 GP 수량) WHERE 교환일시 = TODAY AND 상태 = '완료'
- 실패한 교환은 제외
- 예시: "12,500 GP"

#### 오늘 GP 출금

- 계산식: SUM(GP→CELB 교환의 GP 수량) WHERE 교환일시 = TODAY AND 상태 = '완료'
- 실패한 교환은 제외
- 예시: "8,300 GP"

#### 오늘 교환 건수

- 계산식: COUNT(*) WHERE 교환일시 = TODAY AND 상태 = '완료'
- 충전 + 출금 총합
- 예시: "156건"

> ※ 요약 카드는 기간 필터 설정과 무관하게 항상 오늘(TODAY) 데이터 기준으로 표시됩니다.

### 3.2 필터 기능

#### 교환 방향 필터

| 옵션 | 동작 |
| ---- | ---- |
| 전체 | 모든 교환 내역 표시 |
| GP 충전 | CELB→GP 교환만 표시 |
| GP 출금 | GP→CELB 교환만 표시 |

#### 기간 필터

| 옵션 | 기간 범위 |
| ---- | ---- |
| 전체 | 전체 기간 (제한 없음) |
| 오늘 | 2025.02.03 00:00 ~ 23:59 |
| 최근 7일 | 2025.01.28 00:00 ~ 2025.02.03 23:59 |
| 최근 30일 | 2025.01.05 00:00 ~ 2025.02.03 23:59 |
| 커스텀 | DateRangePicker (최대 90일) |

#### 검색 기능

- 입력: 텍스트 입력 (닉네임)
- 동작: 부분 일치 검색 (LIKE '%keyword%')
- Enter 키 또는 검색 버튼 클릭 시 검색 실행

### 3.3 교환 내역 테이블

#### 컬럼 정의

| 컬럼 | 설명 | 정렬 |
| ---- | ---- | ---- |
| No | 순번 (페이지 내) | - |
| 교환일시 | YYYY.MM.DD HH:mm | 내림차순 기본 |
| 닉네임 | 사용자 닉네임 | 오름차순 |
| Txid | 블록체인 트랜잭션 해시 (truncate 표시: 0x7a3b...8a9b) | - |
| 교환 방향 | GP 충전 / GP 출금 | - |
| GP 수량 | {숫자} GP | 내림차순 |
| CELB 수량 | {숫자} CELB | 내림차순 |
| 교환 비율 | 1 GP = {숫자} CELB | - |
| 상태 | Badge (완료/실패) | - |

#### 상태 표시

| 상태 | Badge 색상 | 설명 |
| ---- | ---- | ---- |
| 완료 | 초록색 | 정상 처리됨 |
| 실패 | 빨간색 | 처리 실패 (잔액 부족, 한도 초과 등) |

#### 교환 방향 표시

- GP 충전: "GP 충전" (CELB→GP) - 파란색 텍스트
- GP 출금: "GP 출금" (GP→CELB) - 주황색 텍스트

#### Row 클릭 동작

- 클릭 시: CEB-BO-GZ-501-MD-DETAIL 모달 오픈
- 전달 데이터: Txid

#### 정렬 기능

- 기본 정렬: 교환일시 내림차순 (최신순)
- 정렬 가능 컬럼: 교환일시, 닉네임, GP 수량, CELB 수량
- 클릭 시 오름차순 ↔ 내림차순 토글

#### 페이지네이션

- 페이지당 20건 고정
- 페이지 번호 클릭으로 이동
- ◀ ▶ 버튼으로 이전/다음 페이지 이동

### 3.5 사용자 시나리오

#### 시나리오 1: 오늘의 교환 현황 확인

1. 관리자가 [게임존 > GP 교환소] 메뉴 클릭
2. 교환 내역 화면 진입 (기본: 전체 기간)
3. 요약 카드에서 오늘의 충전/출금/건수 확인
4. 테이블에서 최근 교환 내역 확인

#### 시나리오 2: 특정 사용자 교환 내역 조회

1. 검색창에 닉네임 "user1" 입력 후 Enter
2. user1의 교환 내역만 필터링되어 표시
3. 특정 Row 클릭하여 상세 정보 확인
4. 모달에서 교환 전/후 GP 잔액 확인

#### 시나리오 3: GP 충전 내역만 조회

1. [교환 방향] 필터에서 "GP 충전" 선택
2. CELB→GP 교환 내역만 표시
3. [기간 필터]에서 "최근 7일" 선택
4. 최근 7일간의 GP 충전 내역 확인

#### 시나리오 4: 실패한 교환 건 확인

1. 테이블에서 상태가 "실패" (빨간색 Badge)인 Row 확인
2. 실패 건 Row 클릭
3. 모달에서 실패 사유 확인
4. 필요 시 사용자 문의 응대 또는 수동 처리

#### 시나리오 5: 교환 비율 변경

1. Header 우측 [교환 설정] 버튼 클릭
2. CEB-BO-GZ-502 화면으로 이동
3. 교환 비율 수정 후 저장
4. 교환 내역 화면으로 돌아와 새 비율 적용 확인

## 5. 에러 및 예외상황

### 5.1 입력 검증

| 항목 | 검증 규칙 | 오류 메시지 |
| ---- | ---- | ---- |
| 커스텀 기간 | 최대 90일 | "조회 기간은 최대 90일까지 선택할 수 있습니다." |
| 검색어 | 1자 이상 | "검색어를 입력해주세요." |
| 페이지 번호 | 1 이상, 최대 페이지 이하 | "유효하지 않은 페이지입니다." |

### 5.2 데이터 검증

| 상황 | 처리 방법 |
| ---- | ---- |
| 교환 내역 없음 | Empty State 표시 |
| API 호출 실패 | Toast: "교환 내역을 불러오는 중 오류가 발생했습니다." |
| 네트워크 오류 | Retry 버튼 표시 |
| 권한 없음 | 403 에러, "접근 권한이 없습니다." |

### 5.3 예외 처리
