## 1. 화면 목적

새로운 게임을 생성하는 폼 화면입니다. 관리자는 게임 유형(Prediction Market 또는 Survival Trivia)을 선택하고, 기본 정보, 보상 설정, 참여 설정, 일정 설정, 결과 설정을 입력하여 게임을 임시저장하거나 즉시 게시할 수 있습니다. 선택한 게임 유형에 따라 폼 항목이 조건부로 표시됩니다.

## 3. 화면 내 기능설명

### 다국어 입력

##### 타이틀 입력
- KO/EN/JP 탭으로 언어 전환
- 각 언어별 50자 제한
- 실시간 글자 수 표시: "25/50"
- 필수 입력: 모든 언어 필수 (KO/EN/JP) (권장: 모든 언어)

##### 상세설명 입력
- KO/EN/JP 탭으로 언어 전환
- WYSIWYG 에디터 제공 (Toolbar + 편집 영역)
- Toolbar 기능: Bold, Italic, Underline, 링크 삽입, 순서 없는 리스트, 순서 있는 리스트, 이미지 삽입, 좌측/중앙/우측 정렬
- 각 언어별 500자 제한
- 필수 입력: 모든 언어 필수 (KO/EN/JP)

### 힌트 링크 입력 (Prediction Market 전용)

##### 힌트 링크
- Segmented Button: 사용/미사용 (기본값: 미사용)
- "사용" 선택 시 URL 입력 필드 표시
- "미사용" 선택 시 URL 입력 필드 숨김
- Placeholder: "https://example.com/hint"
- 도움말 텍스트: "게임 관련 힌트 페이지 URL을 입력하세요."
- URL 형식 검증 (입력 시)

### 퀴즈 입력 (Survival Trivia 선택 시)

##### 문제당 제한시간
- Number Input 사용
- 기본값: 10초
- 최소값: 5초, 최대값: 60초

##### 문제 텍스트
- KO/EN/JP 탭으로 언어 전환
- 각 언어별 200자 제한
- 실시간 글자 수 표시
- 필수 입력: 모든 언어 필수 (KO/EN/JP)

##### 선택지 입력
- 선택지 4개 고정 (추가/삭제 불가)
- 각 선택지: KO/EN/JP 탭으로 언어 전환
- 각 언어별 100자 제한
- 필수 입력: 모든 언어 필수 (KO/EN/JP)

##### 정답 지정
- 각 문제별 Radio 버튼으로 4개 선택지 중 정답 1개 지정
- 정답 미지정 시 저장/생성 불가

##### 문제 순서 재정렬
- 드래그앤드롭으로 문제 순서 변경 가능
- 문제 수 10개 고정 (추가/삭제 불가)

### 보상설정 입력

##### 총 상금 GP
- Number Input 사용
- 양수만 입력 가능
- 천 단위 콤마 자동 표시
- 최소값: 1 GP

### 참여설정 입력

##### 참여 정원 (Prediction Market)
- 체크박스 '참여 정원 제한' 표시
- 체크 해제 시: 무제한 (Number Input 숨김)
- 체크 시: Number Input 표시
- 최소값: 1명

##### 참여 비용 (Prediction Market)
- Number Input 사용
- 기본값: 1 GP
- 최소값: 1 GP

##### 부스팅 비용 (Prediction Market)
- Number Input 사용
- 기본값: 1 GP
- 최소값: 1 GP

##### 부스팅 배수 (Prediction Market)
- Number Input 사용
- 기본값: 2배
- 최소값: 1배
- 최대값: 10배

##### 참여 정원 (Survival Trivia)
- Number Input 사용 (필수)
- 기본값: 100명
- 최소값: 1명
- 최대값: 500명

##### 참여 비용 (Survival Trivia)
- Number Input 사용
- 기본값: 10 GP
- 최소값: 1 GP

### 일정설정 입력

##### 투표 시작일시 (Prediction Market)
- 게시 시점에 자동 설정됨
- 사용자 입력 불가
- 정보성 텍스트로만 표시

##### 투표 종료일시 (Prediction Market)
- Date Picker로 날짜 선택
- Time Picker로 시간 선택 (분 단위)
- 현재 시간 이후만 선택 가능
- 형식: YYYY.MM.DD HH:mm

##### 결과 발표 예정일 (Prediction Market)
- Date Picker로 날짜 선택
- 참여 종료일 이후만 선택 가능
- 형식: YYYY.MM.DD

##### 게임 시작일시 (Survival Trivia)
- Date Picker로 날짜 선택
- Time Picker로 시간 선택 (분 단위)
- 현재 시간 + 30분 이후만 선택 가능
- 형식: YYYY.MM.DD HH:mm
- 게임 시작 10분 전부터 참여자가 입장 가능하며, 시작 시점에 자동으로 게임 진행

### 결과설정 입력 (Prediction Market 전용)

##### 결과 확인 기준 (다국어)
- KO/EN/JP 탭으로 언어 전환
- 각 언어별 500자 제한
- Textarea 사용
- 예시 텍스트 제공: "예: FIFA 공식 홈페이지 발표 기준"
- 필수 입력: 모든 언어 필수 (KO/EN/JP)

> Survival Trivia는 서버에서 자동으로 결과를 판정하므로 결과설정 섹션이 표시되지 않습니다.

### 취소

##### 트리거
- [취소] 버튼 클릭

##### 동작
1. CEB-BO-GZ-201-MD-CANCEL (취소 확인 모달) 오픈
2. 사용자가 [취소하기] 선택 시:
   - 입력된 모든 데이터 폐기
   - CEB-BO-GZ-201 (게임 리스트)로 이동

### 임시저장

##### 트리거
- [임시저장] 버튼 클릭

##### 동작
1. 필수 항목 Validation (필수 아닌 항목은 체크 안 함)
2. 통과 시:
   - 게임 상태: Draft
   - 서버에 저장
   - Success 토스트: "게임이 임시저장되었습니다."
   - CEB-BO-GZ-201 (게임 리스트)로 이동
3. 실패 시:
   - 인라인 에러 메시지 표시
   - 첫 번째 에러 필드로 스크롤

### 생성하기

##### 트리거
- [생성하기] 버튼 클릭

##### 동작

| 항목 | 조건 |

| 선택지 텍스트 (4개×10문제) | 모든 선택지 모든 언어 입력 필수 (KO/EN/JP), 각 100자 이내 |

##### 날짜/시간 관계 검증 (Prediction Market)

```text
현재 시간 < 투표 종료일시 < 결과 발표 예정일
(투표 시작일시는 게시 시점에 자동 설정)
```

- 투표 종료일시가 현재보다 과거일 경우: "투표 종료일시는 현재 시간 이후여야 합니다."
- 결과 발표일이 종료일보다 이전일 경우: "결과 발표 예정일은 투표 종료일 이후여야 합니다."

##### 날짜/시간 검증 (Survival Trivia)

```text
현재 시간 + 30분 ≤ 게임 시작일시
```

- 게임 시작일시가 현재 시간 + 30분 이내일 경우: "게임 시작일시는 현재 시간으로부터 30분 이후여야 합니다."

## 5. 에러 및 예외상황

### 필수 항목 미입력

##### 상황
- 필수 항목이 비어있는 상태로 저장 시도

##### 처리
1. 인라인 에러 메시지 표시: "{항목명}을(를) 입력해주세요."
2. 해당 입력 필드 빨간색 테두리 표시
3. 첫 번째 에러 필드로 자동 스크롤
4. 버튼 비활성화 유지

### 날짜/시간 Validation 실패

##### 상황
- 투표 종료일시 < 현재 시간 (Prediction Market)

##### 처리
- 인라인 에러 메시지: "투표 종료일시는 현재 시간 이후여야 합니다."
- 투표 종료일시 필드 빨간색 테두리

##### 상황
- 결과 발표 예정일 < 참여 종료일 (Prediction Market)

##### 처리
- 인라인 에러 메시지: "결과 발표 예정일은 참여 종료일 이후여야 합니다."
- 결과 발표 예정일 필드 빨간색 테두리

##### 상황
- 게임 시작일시가 현재 시간 + 30분 이내 (Survival Trivia)

##### 처리
- 인라인 에러 메시지: "게임 시작일시는 현재 시간으로부터 30분 이후여야 합니다."
- 게임 시작일시 필드 빨간색 테두리

### 서버 저장 실패

##### 상황
- API 호출 실패 또는 네트워크 오류

##### 처리
1. 에러 토스트 표시: "게임 저장에 실패했습니다. 다시 시도해주세요."
2. 입력된 데이터 유지
3. 사용자가 다시 저장 시도 가능

### 권한 오류

##### 상황
- 쓰기 권한이 없는 관리자가 화면 접근 시도

##### 처리
1. 접근 차단
2. 권한 오류 페이지로 리다이렉트
3. 에러 메시지: "게임을 생성할 권한이 없습니다."

### 취소 시 미저장 데이터

##### 상황
- 입력된 데이터가 있는 상태에서 [취소] 버튼 클릭

##### 처리
1. CEB-BO-GZ-201-MD-CANCEL (취소 확인 모달) 오픈
2. 사용자 선택 대기:
   - [계속 작성]: 모달 닫기, 폼 유지
   - [취소하기]: 데이터 폐기, 리스트로 이동

### 중복 제출 방지

##### 상황
- [임시저장] 또는 [생성하기] 버튼 연속 클릭

##### 처리
1. 첫 번째 클릭 후 버튼 비활성화
2. 로딩 인디케이터 표시
3. 서버 응답 후 버튼 재활성화 또는 화면 이동
