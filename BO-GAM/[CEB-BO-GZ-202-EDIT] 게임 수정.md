## 1. 화면 목적

기존 게임의 정보를 수정하는 폼 화면입니다. 생성 폼과 동일한 구조이나 기존 값이 채워져 있으며, 일부 필드는 수정이 제한됩니다.

##### 주요 기능
- 게임 기본정보 수정 (타이틀, 설명, 힌트 링크)
- 보상/참여/일정 설정 수정
- 결과 확인 기준 수정
- 변경사항 저장 또는 취소

##### 접근 경로
- 게임 상세(CEB-BO-GZ-202) > [수정하기] 버튼 클릭 시 이동

##### 수정 가능 조건
- 게임 상태가 Draft, Ready, Active, Pending일 때 수정 가능
- 상태에 따라 수정 가능한 필드가 제한됨 (상세는 4. 규칙/로직 참조)
- Closed, Ended 상태에서는 수정 불가 (수정 페이지 접근 시 상세 화면으로 리다이렉트)

### 영역별 구성요소

##### 기본정보 섹션

###### 게임유형 (읽기 전용)
- 타입: 선택 (비활성화)
- 값: "Prediction Market" (고정)
- 회색 배경 표시
- 설명: "게임유형은 수정할 수 없습니다." 안내 텍스트

###### 타이틀 (다국어)
- 타입: 텍스트 입력 (다국어 3개 필드)
- 최대 길이: 각 50자
- 필수: Yes
- 실시간 글자 수 카운터 표시 (예: 25/50)
- Validation:
  - 공백만 입력 불가
  - 특수문자 제한 (이모지 허용)
  - 최소 1자 이상

###### 상세설명 (다국어)
- 타입: WYSIWYG 에디터 (다국어 3개 필드)
- Toolbar: Bold, Italic, Underline, 링크, 리스트, 이미지, 정렬
- 최대 길이: 각 500자
- 필수: Yes
- 실시간 글자 수 카운터 표시

###### 힌트 링크
- 타입: URL 입력
- 필수: No
- Placeholder: "https://example.com/hint"
- 도움말: "게임 관련 힌트 페이지 URL 입력"
- URL 형식 검증 (입력 시)

##### 보상설정 섹션

###### 총 상금 GP
- 타입: 숫자 입력
- 필수: Yes
- 최소값: 1 GP
- 입력 형식: 천 단위 콤마 자동 적용
- Validation: 양수만 가능

##### 참여설정 섹션

###### 참여 정원
- 타입: 체크박스 + 숫자 입력
- 필수: Yes
- 무제한 체크 시: 숫자 입력 비활성화
- 무제한 체크 해제 시: 숫자 입력 활성화
  - 최소값: 1명

###### 참여 비용
- 타입: 숫자 입력
- 필수: Yes
- 기본값: 1 GP
- 최소값: 1 GP

###### 부스팅 비용

- 타입: 숫자 입력
- 필수: Yes
- 기본값: 1 GP
- 최소값: 1 GP

###### 부스팅 배수

- 타입: 숫자 입력
- 필수: Yes
- 기본값: 2배
- 최소값: 1배, 최대값: 10배

##### 일정설정 섹션

###### 투표 시작일시
- 타입: 읽기 전용 텍스트
- 게시 시점에 자동 설정 (사용자 수정 불가)
- 게시 전: "(게시 시 설정)" 표시
- 게시 후: 게시 시점 일시 표시

###### 투표 종료일시
- 타입: 날짜/시간 선택
- 필수: Yes
- 형식: YYYY.MM.DD HH:mm
- 제약: 현재 시간 이후여야 함

###### 결과 발표 예정일
- 타입: 날짜 선택
- 필수: Yes
- 형식: YYYY.MM.DD
- 제약: 투표 종료일시 이후여야 함

- 필수: Yes
- KO/EN/JP 탭으로 언어 전환
- 실시간 글자 수 카운터 표시
- 필수 입력: 모든 언어 필수 (KO/EN/JP)

## 3. 화면 내 기능설명

### 3.1 버튼 동작

##### 취소 버튼
- 위치: 하단 좌측
- 스타일: Secondary
- 동작: CEB-BO-GZ-202-MD-EDIT-CANCEL 모달 표시
- 단축키: ESC

##### 저장 버튼
- 위치: 하단 우측
- 스타일: Primary
- 동작:
  1. 수정 가능한 필드에 대한 유효성 검사
  2. 통과 시 API 호출
  3. 성공 시:
     - 현재 상태가 Draft이고 모든 필수 항목 입력 완료:
       - 게임 상태 Draft → Ready로 변경
       - Success 토스트: "게임이 저장되었습니다. 게시대기 상태로 변경되었습니다."
     - 현재 상태가 Ready, Active, Pending:
       - 상태 유지
       - Success 토스트: "게임 정보가 수정되었습니다."
     - 게임 상세(CEB-BO-GZ-202) 화면으로 복귀
  4. 실패 시:
     - 오류 메시지 토스트 표시
     - 폼 유지
- 단축키: Ctrl+S (Cmd+S on Mac)

## 4. 규칙/로직

### 권한 구분

| 권한 | 접근 가능 | 수정 가능 |
| ---- | ---- | ---- |
| 쓰기 | Yes | Yes |
| 읽기 | No | No |
| 없음 | No | No |

### 제약사항

- Closed, Ended 상태에서는 수정 불가 (접근 시 상세 화면으로 리다이렉트)
- 게임유형은 변경 불가 (읽기 전용)
- 참여자가 1명 이상 있을 경우 참여 비용, 부스팅 비용/배수 수정 불가

| ---- | ----- | ----- | ------ | ------- |

| 투표 종료일시 | ✅ | ✅ | ✅ | ❌ |

| 결과 확인 기준 | ✅ | ✅ | ✅ | ✅ |

> *참여자가 1명 이상인 경우 수정 불가

### 저장 시 상태 변경 규칙

| 현재 상태 | 저장 시 변경 | 조건 |
| -------- | ---------- | ---- |
| Draft | → Ready | 모든 필수 항목 입력 완료 + Validation 통과 |
| Ready | Ready 유지 | - |
| Active | Active 유지 | - |
| Pending | Pending 유지 | - |

## 5. 에러 및 예외상황

##### 필수 입력 검사
- 모든 필수(*) 필드 미입력 시:
  - 해당 필드 하단에 빨간색 오류 메시지 표시
  - 예: "타이틀을 입력해주세요."
  - 저장 버튼 비활성화

##### 날짜/시간 검증
- 투표 종료일시가 현재 시간 이전일 경우: "투표 종료일시는 현재 시간 이후여야 합니다."
- 결과 발표 예정일이 종료일시 이전: "결과 발표 예정일은 투표 종료일시 이후여야 합니다."

##### 숫자 범위 검증
- 총 상금 GP가 0 이하: "총 상금 GP는 1 이상이어야 합니다."
- 참여 정원이 0 이하: "참여 정원은 1명 이상이어야 합니다."

##### 네트워크 오류
- API 호출 실패 시: "게임 수정에 실패했습니다. 다시 시도해주세요." 토스트 표시
- 타임아웃: "요청 시간이 초과되었습니다. 다시 시도해주세요."

##### 권한 오류
- 수정 권한 없음: "게임을 수정할 권한이 없습니다." 토스트 표시 후 상세 화면으로 복귀
- 게임 상태가 Closed 또는 Ended: "종료된 게임은 수정할 수 없습니다." 토스트 표시 후 상세 화면으로 리다이렉트
