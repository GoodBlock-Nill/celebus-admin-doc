## 1. 화면 목적

운영용 블록체인 지갑(충전용/출금용)을 등록, 수정, 삭제하고 잔액을 조회하는 화면입니다. GP 교환소 운영에 필요한 지갑을 관리할 수 있습니다.

### 주요 기능

- 운영 지갑 목록 조회 (충전용/출금용)
- 지갑 등록 (지갑 유형, 지갑 주소)
- 지갑 수정 (지갑 유형, 지갑 주소)
- 지갑 삭제
- 지갑 활성화/비활성화
- 잔액 새로고침
- 대표 지갑 지정 (유형별 1개)

### 접근 경로

```text
홈 > 게임존 > GP 교환소 > [지갑 관리] 버튼
또는
CEB-BO-GZ-501 화면 우측 [지갑 관리] 버튼
```

## 3. 화면 내 기능설명

### 3.1 지갑 유형 정의

| 유형 | 영문 코드 | 설명 | Badge 색상 |
| ---- | ---- | ---- | ---- |
| 충전용 | CHARGE | GP 충전 시 사용되는 지갑 (CELB 수신) | 파란색 |
| 출금용 | WITHDRAW | GP 출금 시 사용되는 지갑 (CELB 송금) | 주황색 |

### 3.2 지갑 상태 정의

| 상태 | 설명 | Badge 색상 |
| ---- | ---- | ---- |
| 활성 | 교환에 사용 가능한 상태 | 초록색 |
| 비활성 | 교환에 사용 불가한 상태 | 회색 |

### 3.3 잔액 새로고침 버튼

- 위치: Header 우측
- 라벨: [잔액 새로고침]
- 스타일: 회색 (Secondary)
- 동작: 모든 지갑의 잔액을 블록체인에서 조회하여 갱신
- 성공 시: Toast "잔액이 갱신되었습니다."
- 실패 시: Toast "잔액 조회 중 오류가 발생했습니다."

### 3.4 지갑 등록 버튼

- 위치: Header 우측
- 라벨: [+ 지갑 등록]
- 스타일: 파란색 (Primary)
- 동작: 지갑 등록 모달 오픈
- 권한: 쓰기 권한 필요

### 3.5 지갑 등록/수정 모달

#### 모달 정보

- 모달 제목: "지갑 등록" 또는 "지갑 수정"
- 모달 너비: max-width 400px

#### 입력 필드

| 필드 | 입력 타입 | 필수 | 설명 |
| ---- | ---- | ---- | ---- |
| 지갑 유형 | Toggle Button (충전용/출금용) | O | 충전용 또는 출금용 선택 |
| 지갑 주소 | Text Input | O | 블록체인 지갑 주소 (0x...) |

#### 버튼

- [취소]: 모달 닫기
- [등록] 또는 [수정]: 저장 후 모달 닫기

#### 성공/실패 처리

- 등록 성공: Toast "지갑이 등록되었습니다."
- 수정 성공: Toast "지갑이 수정되었습니다."
- 실패: Toast "지갑 저장 중 오류가 발생했습니다."

### 3.6 지갑 삭제

#### 삭제 확인 모달

- 모달 제목: "지갑 삭제 확인"
- 메시지: "{지갑유형} 지갑을 삭제하시겠습니까?"
- 경고 문구: "삭제된 지갑은 복구할 수 없습니다."
- 버튼: [취소] [삭제] (빨간색)

> 대표 지갑은 삭제할 수 없습니다. 먼저 다른 지갑을 대표로 지정한 후 삭제해야 합니다. 대표 지갑의 [삭제] 버튼은 비활성화 처리됩니다.

#### 성공/실패 처리

- 삭제 성공: Toast "지갑이 삭제되었습니다."
- 실패: Toast "지갑 삭제 중 오류가 발생했습니다."

### 3.7 지갑 활성화/비활성화

- 위치: 각 Row의 액션 컬럼
- 라벨: [비활성화] (활성 상태일 때) / [활성화] (비활성 상태일 때)
- 동작: 즉시 상태 변경 (Confirm 없음)
- 성공 시: Toast "지갑이 활성화되었습니다." 또는 "지갑이 비활성화되었습니다."

### 3.8 대표 지갑 지정

- 위치: 각 Row의 액션 컬럼 (대표가 아닌 지갑에만 표시)
- 라벨: [대표 지정]
- 스타일: 노란색 테두리 텍스트 버튼
- 동작: 해당 지갑을 해당 유형의 대표 지갑으로 지정
- 규칙:
  - 동일 유형(충전용/출금용) 내에서 대표 지갑은 1개만 가능
  - 새 대표 지정 시 기존 대표는 자동 해제
  - Confirm 모달 없음 (즉시 적용)
- 성공 시: Toast "{유형명} 대표 지갑이 변경되었습니다."

### 3.9 지갑 주소 클릭

- 동작: BSCScan 새 탭에서 열기
- URL 형식: https://bscscan.com/address/{지갑주소}

### 3.10 잔액 표시 형식

- 형식: {숫자} CELB
- 소수점: 8자리까지 표시
- 예시: 1,234.56789012 CELB

### 3.11 Empty State

```text

│                                 │
│   등록된 지갑이 없습니다.        │
│                                 │
└─────────────────────────────────┘
```

## 4. 규칙/로직

### 권한

| 역할 | 읽기 | 쓰기 | 삭제 |
| ---- | ---- | ---- | ---- |
| 슈퍼 관리자 | O | O | O |
| 게임 운영자 | O | O | O |
| 일반 관리자 | O | X | X |

> 쓰기/삭제 권한 없으면 버튼 미노출

### 로그 정책

| 이벤트 | 로그 메시지 | 레벨 |
| ---- | ---- | ---- |
| 화면 진입 | 지갑 관리 화면을 조회했습니다. | INFO |
| 지갑 등록 | 지갑을 등록했습니다. (유형: {유형}, 주소: {주소}) | INFO |
| 지갑 수정 | 지갑을 수정했습니다. (지갑ID: {id}) | INFO |
| 지갑 삭제 | 지갑을 삭제했습니다. (지갑ID: {id}) | INFO |
| 상태 변경 | 지갑 상태를 변경했습니다. (지갑ID: {id}, 상태: {상태}) | INFO |
| 대표 지정 | 대표 지갑을 변경했습니다. (유형: {유형}, 지갑ID: {id}) | INFO |
| 잔액 갱신 | 지갑 잔액을 갱신했습니다. | INFO |
| 조회 실패 | 지갑 목록 조회 중 오류가 발생했습니다. (에러: {error}) | ERROR |

### 비즈니스 규칙

1. 지갑 유형(충전용/출금용)은 중복 등록 가능합니다
2. 비활성화된 지갑은 GP 교환에 사용되지 않습니다
3. 지갑 잔액은 블록체인에서 직접 조회합니다
4. 지갑 삭제 시 관련 교환 내역은 유지됩니다
5. 잔액 새로고침은 모든 지갑을 일괄 갱신합니다
6. 대표 지갑은 유형(충전용/출금용)별로 1개만 지정할 수 있습니다
7. 대표 지갑은 삭제할 수 없습니다 (다른 지갑으로 대표 변경 후 삭제)
8. 대표 지갑의 잔액이 CEB-BO-GZ-501 운영 현황 대시보드에 표시됩니다
9. 신규 등록된 지갑은 비대표 상태입니다
10. 지갑 유형 변경 시 대표 상태는 자동 해제됩니다

### 용어 정의

| 용어 | 정의 |
| ---- | ---- |
| 충전용 지갑 | GP 충전(CELB→GP) 시 사용자로부터 CELB를 수신하는 지갑 |
| 출금용 지갑 | GP 출금(GP→CELB) 시 사용자에게 CELB를 송금하는 지갑 |
| 대표 지갑 | 각 유형(충전용/출금용)별로 1개만 지정 가능한 대표 운영 지갑. 대시보드에 잔액이 표시됨 |
| 잔액 | 지갑에 보유한 CELB 수량 |

## 5. 에러 및 예외상황

### 5.1 입력 검증

| 항목 | 검증 규칙 | 오류 메시지 |
| ---- | ---- | ---- |
| 지갑 주소 | 필수 입력 | "지갑 주소를 입력하세요." |
| 지갑 주소 형식 | 0x로 시작하는 42자 | "올바른 지갑 주소 형식이 아닙니다." |

### 5.2 데이터 검증

| 상황 | 처리 방법 |
| ---- | ---- |
| 지갑 목록 없음 | Empty State 표시 |
| API 호출 실패 | Toast: "지갑 목록을 불러오는 중 오류가 발생했습니다." |
| 네트워크 오류 | Retry 버튼 표시 |
| 권한 없음 | 403 에러, 버튼 미노출 |

### 5.3 예외 처리

| 예외 상황 | 처리 방법 |
| ---- | ---- |
| 데이터 로딩 중 | 테이블 영역에 Skeleton 표시 |
| 잔액 조회 실패 | 해당 지갑 잔액에 "-" 표시 |
| 블록체인 네트워크 오류 | Toast: "블록체인 네트워크 오류가 발생했습니다." |
