## 1. 화면 목적

사용자의 모든 GP 증감 내역을 통합 조회하는 화면입니다. 게임 참여, 부스팅, 환급, 보상, GP 충전, GP 출금 등 모든 GP 변동 유형을 한눈에 확인하고 추적할 수 있습니다.

### 주요 기능

- 모든 GP 변동 내역 통합 조회
- 변동 유형별 필터링 (참여/부스팅/환급/보상/GP충전/GP출금/환불)
- 오늘의 GP 증감 현황 요약
- 기간별, 사용자별 검색
- GP 변동 상세 정보 확인
- 관련 게임 또는 교환 이력 추적

### 접근 경로

```text
홈 > 게임존 > GP 변동 내역
```

## 3. 화면 내 기능설명

### 3.1 변동 유형 정의

| 유형 | 영문 코드 | 설명 | 변동 방향 | Badge 색상 |
| ---- | ---- | ---- | ---- | ---- |
| 참여 | PARTICIPATION | 게임 참여 시 GP 차감 | - (차감) | 파란색 |
| 부스팅 | BOOSTING | 부스팅 사용 시 GP 차감 | - (차감) | 보라색 |
| 환급 | REFUND | 결과 발표 후 참여 GP 반환 | + (증가) | 초록색 |
| 보상 | REWARD | 정답 시 보상 GP 지급 | + (증가) | 금색 |
| GP 충전 | EXCHANGE_IN | CELB → GP 교환 | + (증가) | 청록색 |
| GP 출금 | EXCHANGE_OUT | GP → CELB 교환 | - (차감) | 주황색 |
| 환불 | REFUND_CANCEL | 게임 취소 등으로 인한 GP 환불 | + (증가) | 회색 |

### 3.4 사용자 시나리오

#### 시나리오 1: 전체 GP 변동 현황 확인

1. 관리자가 [게임존 > GP 변동 내역] 메뉴 클릭
2. GP 변동 내역 화면 진입 (기본: 전체 기간)
3. 테이블에서 최근 변동 내역 확인

#### 시나리오 2: 특정 사용자 GP 변동 추적

1. 검색창에 닉네임 "starlight23" 입력 후 Enter
2. starlight23의 GP 변동 내역만 필터링되어 표시
3. 변동일시 순으로 정렬하여 시간순 변동 추적
4. 특정 Row 클릭하여 상세 정보 확인
5. 모달에서 변동 전/후 GP 잔액 확인

#### 시나리오 3: 보상 지급 내역만 조회

1. [변동 유형] 필터에서 "보상" 선택
2. REWARD 유형만 표시
3. [기간 필터]에서 "최근 7일" 선택
4. 최근 7일간의 보상 지급 내역 확인
5. GP 변동량 컬럼으로 정렬하여 큰 보상부터 확인

#### 시나리오 4: 교환 관련 GP 변동 확인

1. [변동 유형] 필터에서 "GP 충전" 선택
2. EXCHANGE_IN 유형만 표시
3. 관련 게임 컬럼이 "-"로 표시됨
4. 비고 컬럼에 "CELB 교환" 표시됨
5. Row 클릭 시 모달에서 관련 교환 정보 확인 가능

#### 시나리오 5: GP 변동 상세 확인

1. 테이블에서 특정 Row 클릭
2. CEB-BO-GZ-601-MD-DETAIL 모달 오픈
3. 변동 상세 정보 확인 (변동 전후 GP 잔액, 관련 게임/교환 등)
4. 모달 닫기

### 비즈니스 규칙

1. GP 변동 내역은 삭제할 수 없습니다 (이력 보존)
2. 모든 GP 증감은 반드시 기록됩니다
3. 변동 전/후 잔액은 변동 시점의 실제 잔액입니다
4. 게임 관련 변동은 게임 ID와 연결됩니다
5. 교환 관련 변동은 교환 ID와 연결됩니다
6. 변동일시는 GP가 실제로 증감된 시각입니다
7. 게임 삭제 시에도 변동 내역은 유지됩니다 (게임 타이틀만 표시)

### GP 변동량 계산 예시

#### 게임 참여 (PARTICIPATION)

```text
변동 전 GP 잔액: 5,000 GP
참여 GP: 500 GP
변동 후 GP 잔액: 4,500 GP (5,000 - 500)
GP 변동량: -500 GP (빨간색)
```

#### 보상 지급 (REWARD)

```text
변동 전 GP 잔액: 4,500 GP
보상 GP: 1,000 GP
변동 후 GP 잔액: 5,500 GP (4,500 + 1,000)
GP 변동량: +1,000 GP (초록색)
```

#### GP 충전 (EXCHANGE_IN)

```text
변동 전 GP 잔액: 5,000 GP
충전 GP: 2,000 GP
변동 후 GP 잔액: 7,000 GP (5,000 + 2,000)
GP 변동량: +2,000 GP (초록색)
```

### 용어 정의

| 용어 | 정의 |
| ---- | ---- |
| GP 변동 내역 | 사용자의 GP 잔액이 증가 또는 감소한 모든 이력 |
| 변동 유형 | GP 증감의 원인 (참여, 부스팅, 환급, 보상, GP충전, GP출금, 환불) |
| 변동일시 | GP가 실제로 증감된 시각 |
| GP 변동량 | 증가(+) 또는 감소(-) GP 수량 |
| 변동 후 잔액 | GP 증감 직후 사용자의 GP 잔액 |
| 관련 게임 | GP 변동과 연결된 게임 (교환의 경우 "-") |
| 관련 교환 | GP 변동과 연결된 교환 (게임의 경우 null) |

## 5. 에러 및 예외상황

### 5.1 입력 검증

### 5.2 데이터 검증

| 상황 | 처리 방법 |
| ---- | ---- |
| GP 변동 내역 없음 | Empty State 표시 |
| API 호출 실패 | Toast: "GP 변동 내역을 불러오는 중 오류가 발생했습니다." |
| 네트워크 오류 | Retry 버튼 표시 |

| 예외 상황 | 처리 방법 |
| ---- | ---- |
