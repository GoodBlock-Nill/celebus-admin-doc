## 1. 화면 목적

교환 내역의 상세 정보를 보여주는 모달입니다. Txid, 교환일시, 사용자 정보, 교환 수량, 비율, 교환 전후 GP 잔액, 처리 상태 및 실패 사유 등을 확인할 수 있습니다.

### 주요 기능

- 교환 상세 정보 조회
- 교환 전/후 GP 잔액 확인
- 실패 사유 확인 (실패 건)
- 회원 상세 화면으로 이동
- 교환 이력 추적

### 진입 경로

- CEB-BO-GZ-501 교환 내역 테이블 Row 클릭

## 3. 화면 내 기능설명

### 3.1 표시 정보

| 필드 | 형식 | 설명 | 예시 |
| ---- | ---- | ---- | ---- |
| Txid | 텍스트 (mono) + 링크 + 복사 | 블록체인 트랜잭션 해시 (전체 표시), 클릭 시 BSCScan 새 탭 열기, 복사 버튼 | 0x7a3b...f8a9b (전체) |
| 교환일시 | YYYY.MM.DD HH:mm:ss | 교환 완료/실패 시각 | 2025.02.03 14:23:15 |
| 닉네임 | 링크 | 사용자 닉네임 (클릭 시 회원상세 이동) | user1 |
| 지갑주소 | 텍스트 (mono) + 링크 + 복사 | BSC 지갑주소 (전체 표시), 클릭 시 BSCScan 새 탭 열기, 복사 버튼 | 0x1234...5678 (전체) |
| 교환 방향 | Badge | GP 가져오기 / CELB으로 보내기 | [GP 가져오기] |
| GP 수량 | {숫자} GP | 교환된 GP 수량 (천 단위 콤마) | 1,000 GP |
| CELB 수량 | {숫자} CELB | 교환된 CELB 수량 (천 단위 콤마) | 1,000 CELB |
| 적용 비율 | 1 GP = {숫자} CELB | 교환 시점의 비율 | 1 GP = 1.0 CELB |
| 교환 전 GP 잔액 | {숫자} GP | 교환 전 사용자 GP 잔액 | 5,000 GP |
| 교환 후 GP 잔액 | {숫자} GP | 교환 후 사용자 GP 잔액 | 6,000 GP |
| 상태 | Badge | 완료 / 실패 | [완료] |
| 실패 사유 | 텍스트 | 실패 시만 표시 | 잔액 부족 |

### 3.2 교환 방향 Badge

| 방향 | Badge 색상 | 텍스트 |
| ---- | ---- | ---- |
| GP 가져오기 | 파란색 | GP 가져오기 |
| CELB으로 보내기 | 주황색 | CELB으로 보내기 |

### 3.3 상태 Badge

| 상태 | Badge 색상 | 설명 |
| ---- | ---- | ---- |
| 완료 | 초록색 | 정상 처리됨 |
| 실패 | 빨간색 | 처리 실패 |

### 3.4 실패 사유

완료 상태일 경우 "-" 표시, 실패 상태일 경우 사유 텍스트 표시

| 실패 사유 코드 | 사유 메시지 |
| ---- | ---- |
| INSUFFICIENT_BALANCE | 잔액 부족 |
| DAILY_LIMIT_EXCEEDED | 1일 한도 초과 |
| MIN_AMOUNT_NOT_MET | 최소 교환 수량 미만 |
| MAX_AMOUNT_EXCEEDED | 최대 교환 수량 초과 |
| EXCHANGE_DISABLED | 교환 기능 비활성화 |
| SYSTEM_ERROR | 시스템 오류 |

### 3.5 닉네임 링크

- 새 탭에서 열림
- 파라미터: 지갑주소 전달

### 3.6 닫기 버튼

- 라벨: [닫기]
- 동작: 모달 닫기

### 3.7 사용자 시나리오

### 3.8 Txid/지갑주소 링크 및 복사

#### Txid
- 전체 표시: truncate 없이 0x + 64자 hex 전체 표시
- 클릭 시: https://bscscan.com/tx/{txid} 새 탭에서 열기
- 복사 버튼: 클릭 시 Txid 전체 값 클립보드 복사

#### 지갑주소
- 전체 표시: truncate 없이 0x + 40자 hex 전체 표시
- 클릭 시: https://bscscan.com/address/{walletAddress} 새 탭에서 열기
- 복사 버튼: 클릭 시 지갑주소 전체 값 클립보드 복사

#### 복사 완료 피드백
- 클립보드 복사 성공 시: Toast "복사되었습니다." (Success)

### 3.9 사용자 시나리오

#### 시나리오 1: 정상 교환 내역 확인

1. 교환 내역 화면에서 상태가 "완료"인 Row 클릭
2. 교환 상세 모달 오픈
3. Txid, 일시, 사용자 정보 확인
4. GP 수량, CELB 수량, 적용 비율 확인
5. 교환 전/후 GP 잔액 변동 확인
6. 실패 사유는 "-"로 표시됨
7. [닫기] 버튼으로 모달 닫기

#### 시나리오 2: 실패한 교환 내역 확인

1. 교환 내역 화면에서 상태가 "실패"인 Row 클릭
2. 교환 상세 모달 오픈
3. 상태가 [실패] Badge로 표시됨 (빨간색)
4. 실패 사유 필드에 "잔액 부족" 등의 사유 표시
5. 교환 전/후 GP 잔액 확인 (실패 시 잔액 변동 없음)
6. 사용자 문의 시 실패 사유 안내

#### 시나리오 3: 사용자 정보 확인

1. 교환 상세 모달에서 닉네임 클릭
2. 새 탭에서 CEB-BO-USR-201 회원 상세 화면 열림
3. 사용자의 전체 정보, GP 잔액, 교환 이력 확인
4. 모달은 유지됨 (닫히지 않음)

#### 시나리오 4: 교환 비율 확인

1. 교환 상세 모달에서 "적용 비율" 필드 확인
2. 교환 시점의 비율이 기록되어 있음 (예: 1 GP = 1.0 CELB)
3. 현재 설정 비율과 다를 수 있음 (과거 비율 유지)
4. 교환 전/후 잔액 계산이 맞는지 검증

| 항목 | 요구사항 |
| ---- | ---- |

| 스크린 리더 | 모든 필드에 aria-label 적용 |
| 포커스 트랩 | 모달 오픈 시 포커스가 모달 내부에만 유지 |

### 비즈니스 규칙

1. 교환 상세는 수정할 수 없습니다 (조회 전용)
2. 교환 시점의 비율이 기록됩니다 (현재 설정과 다를 수 있음)
3. 실패한 교환도 이력으로 남습니다
4. 교환 전/후 GP 잔액은 교환 시점의 실제 잔액입니다
5. GP 가져오기 시: 교환 후 잔액 = 교환 전 잔액 + GP 수량
6. CELB으로 보내기 시: 교환 후 잔액 = 교환 전 잔액 - GP 수량
7. 실패 시: 교환 전 잔액 = 교환 후 잔액 (변동 없음)

### 계산 예시

#### GP 가져오기 (CELB → GP)

```text
교환 전 GP 잔액: 5,000 GP
GP 수량: 1,000 GP
교환 후 GP 잔액: 6,000 GP (5,000 + 1,000)
```

#### CELB으로 보내기 (GP → CELB)

```text
교환 전 GP 잔액: 5,000 GP
GP 수량: 500 GP
교환 후 GP 잔액: 4,500 GP (5,000 - 500)
```

#### 실패 케이스

```text
교환 전 GP 잔액: 300 GP
GP 수량: 500 GP (출금 시도)
교환 후 GP 잔액: 300 GP (변동 없음)
상태: 실패
실패 사유: 잔액 부족
```

### 용어 정의

| 용어 | 정의 |
| ---- | ---- |
| Txid | 블록체인 트랜잭션 해시 (형식: 0x + 64자 hex, BSC 네트워크) |
| 교환일시 | 교환이 완료되거나 실패한 시각 (요청 시각 아님) |
| 적용 비율 | 교환 시점에 적용된 GP↔CELB 교환 비율 |
| 교환 전 GP 잔액 | 교환 실행 직전 사용자의 GP 잔액 |
| 교환 후 GP 잔액 | 교환 완료/실패 직후 사용자의 GP 잔액 |

## 5. 에러 및 예외상황

### 5.1 데이터 검증

| 검증 항목 | 검증 규칙 | 오류 처리 |
| ---- | ---- | ---- |
| Txid 존재 | 유효한 Txid | Toast: "교환 정보를 찾을 수 없습니다." |
| 지갑주소 | 지갑주소 유효성 | Toast: "사용자 정보를 불러올 수 없습니다." |
| 교환 방향 | CHARGE 또는 WITHDRAW | 기본값: "-" |
| 교환 전후 잔액 | 숫자 타입 | 기본값: 0 |

### 5.2 예외 처리

| 예외 상황 | 처리 방법 |
| ---- | ---- |
