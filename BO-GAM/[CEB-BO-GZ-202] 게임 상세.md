## 1. 화면 목적

게임의 상세 정보를 조회하고 게임 상태에 따른 액션(수정, 게시, 결과 입력, 보상 지급 등)을 수행하는 화면입니다. Prediction Market(PM)과 Survival Trivia(ST) 두 가지 게임유형을 지원합니다.

##### 주요 기능
- 게임 기본정보 조회 (타이틀, 설명, 보상/참여/일정 설정)
- 퀴즈 문제/정답 조회 (Survival Trivia)
- 실시간 게임 진행 현황 조회 (Survival Trivia)
- 참여자 내역 조회 (PM: 선택, 참여 GP, 부스팅 GP / ST: 생존 스테이지, 하트 사용, 참여 GP)
- 결과 및 보상 정보 조회
- 게임 상태별 액션 실행 (게시, 수정, 강제 종료, 결과 입력(PM), 보상 지급(PM))

##### 접근 경로
- 게임 목록(CEB-BO-GZ-201) > 게임 행 클릭 시 이동

##### 상태별 액션 버튼 배치 (Header 우측)

###### Prediction Market (PM)
```text
Draft(임시저장):     [삭제하기] [수정하기]
Ready(게시대기):     [삭제하기] [수정하기] [게시하기]
Active(진행중):      [수정하기] [강제 종료]
Pending(결과 대기):  [수정하기] [결과 입력] [강제 종료]
Closed(결과 확정):   [결과 수정] [보상 지급]
Ended(종료):         (버튼 없음)
```

###### Survival Trivia (ST)
```text
Draft(임시저장):     [삭제하기] [수정하기]
Ready(게시대기):     [삭제하기] [수정하기] [게시하기]
Active(진행중):      [수정하기] [강제 종료]
Closed(결과 확정):   (버튼 없음)             ← 결과/보상 자동처리
Ended(종료):         (버튼 없음)
```

> ST는 Pending 상태 없음 (게임 종료 즉시 결과 자동 확정). ST는 Active 상태에서 타이틀/상세설명만 수정 가능 (퀴즈, 보상, 참여설정, 게임 시작일시 수정 불가). ST는 "결과 입력", "보상 지급" 버튼 없음 (서버 자동 처리).

## 3. 화면 내 기능설명

### 3.1 게임 상태 및 액션

##### Prediction Market 상태별 액션

| 상태 | 코드 | Badge | 가능 액션 |
| ---- | ---- | ---- | ---- |
| 임시저장 | Draft | 회색 | 수정, 삭제 |
| 게시대기 | Ready | 파란색 | 수정, 게시, 삭제 |
| 진행중 | Active | 초록색 | 수정, 강제 종료 |
| 결과 대기 | Pending | 주황색 | 수정, 결과 입력, 강제 종료 |
| 결과 확정 | Closed | 보라색 | 결과 수정, 보상 지급 |
| 종료 | Ended | 검정색 | 조회만 가능 |

##### Survival Trivia 상태별 액션

| 상태 | 코드 | Badge | 가능 액션 |
| ---- | ---- | ---- | ---- |
| 임시저장 | Draft | 회색 | 수정, 삭제 |
| 게시대기 | Ready | 파란색 | 수정, 게시, 삭제 |
| 진행중 | Active | 초록색 | 수정, 강제 종료 |
| 결과 확정 | Closed | 보라색 | 조회만 가능 (자동 처리) |
| 종료 | Ended | 검정색 | 조회만 가능 |

> ST는 Pending(결과 대기) 상태 없음. 게임 종료 시 서버가 결과를 즉시 자동 확정하고 보상을 자동 지급. "결과 입력", "보상 지급" 버튼 없음. Active 상태에서도 타이틀/상세설명 항목 수정 가능 (퀴즈, 보상, 참여설정, 게임 시작일시 수정 불가).

### 3.2 탭 구성

###### 참여내역 탭 동작
- 기본 정렬: 참여일시 내림차순
- 페이지네이션: 20건/페이지
- ROW 클릭: 회원 상세(CEB-BO-USR-201) 화면으로 이동
- Empty 상태: "참여 내역이 없습니다." 메시지 표시

###### 보상 상세 테이블 동작 (PM)
- 기본 정렬: 보상 GP 내림차순
- 결과가 입력되지 않았거나 정답자가 없을 경우 테이블 비표시

###### 보상 상세 테이블 동작 (ST)
- 기본 정렬: 보상 GP 내림차순
- 게임 종료 전(Active 상태)에는 테이블 비표시
- 전원 탈락 시 보상 상세 테이블 비표시, "전원 탈락으로 보상이 지급되지 않았습니다." 메시지 표시

> **보상 계산 로직**: 상세 계산 공식은 **[CEB-BO-GZ-000] 공통 정책** "5. 보상 계산 로직"을 참조하세요.

### 3.3 액션 버튼별 동작

##### 수정하기
- 조건 (PM): 상태가 Draft, Ready, Active, 또는 Pending
- 조건 (ST): 상태가 Draft, Ready, 또는 Active (Active 시 타이틀/상세설명만 수정 가능)
- 동작: CEB-BO-GZ-202-EDIT 화면으로 이동

##### 결과 수정
- 조건: PM만 해당, 상태가 Closed
- 동작: CEB-BO-GZ-202-MD-RESULT 모달 표시 (결과 수정 모드)

##### 게시하기
- 조건: 상태가 Ready (PM, ST 공통)
- 동작: CEB-BO-GZ-202-MD-PUBLISH 모달 표시

##### 삭제하기
- 조건: 상태가 Draft 또는 Ready (PM, ST 공통)
- 동작: CEB-BO-GZ-202-MD-DELETE 모달 표시

##### 강제 종료
- 조건 (PM): 상태가 Active 또는 Pending
- 조건 (ST): 상태가 Active
- 동작: CEB-BO-GZ-202-MD-CLOSE 모달 표시

##### 결과 입력
- 조건: PM만 해당, 상태가 Pending
- 동작: CEB-BO-GZ-202-MD-RESULT 모달 표시

##### 보상 지급
- 조건: PM만 해당, 상태가 Closed
- 동작: CEB-BO-GZ-202-MD-REWARD 모달 표시

## 4. 규칙/로직

### 권한 구분

| 권한 | 접근 가능 영역 | 실행 가능 액션 |
| ---- | ---- | ---- |
| 읽기 | 모든 탭 조회 | 조회만 가능 |
| 쓰기 | 모든 탭 조회 | 수정, 게시, 결과 입력, 보상 지급 |
| 삭제 | 모든 탭 조회 | 삭제 추가 |

### 제약사항

- Closed 또는 Ended 상태에서는 수정 불가
- Ended 상태에서는 모든 액션 불가 (조회만 가능)
- 결과는 Closed 상태에서 수정 가능
- 보상 지급은 1회만 가능 (취소 불가)

## 5. 에러 및 예외상황

##### 접근 권한
- 읽기 권한 없음: 게임 목록으로 리디렉션, "접근 권한이 없습니다." 토스트 표시
- 쓰기 권한 없음: 액션 버튼 비활성화 또는 비표시

##### 데이터 로딩 실패
- 게임 정보 조회 실패 시 "게임 정보를 불러올 수 없습니다." 오류 메시지 표시
- [재시도] 버튼 제공

##### 네트워크 오류
- "네트워크 오류가 발생했습니다. 다시 시도해주세요." 토스트 표시
