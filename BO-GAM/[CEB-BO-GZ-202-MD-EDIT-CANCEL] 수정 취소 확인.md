## 1. 화면 목적

게임 수정 화면에서 사용자가 [취소] 버튼을 클릭했을 때, 변경 사항이 저장되지 않음을 경고하고 최종 확인을 받는 모달입니다. 실수로 인한 변경사항 손실을 방지합니다.

### 레이아웃

```text

│                      [배경 Dim 처리]                         │
│                                                             │
```

| 요소 | 타입 | 설명 |

## 3. 화면 내 기능설명

### 모달 오픈

##### 트리거
- CEB-BO-GZ-202-EDIT (게임 수정) 화면에서 [취소] 버튼 클릭

##### 동작
1. 배경 화면 Dim 처리
2. 모달 중앙 정렬하여 표시
3. 모달 외부 스크롤 비활성화

### 계속 수정

##### 트리거
- [계속 수정] 버튼 클릭

##### 동작
1. 모달 닫기
2. CEB-BO-GZ-202-EDIT (게임 수정) 화면 유지
3. 변경된 데이터 모두 유지

### 취소하기

##### 트리거
- [취소하기] 버튼 클릭

##### 동작
1. 모달 닫기
2. 변경된 모든 데이터 폐기
3. CEB-BO-GZ-202 (게임 상세) 화면으로 이동

### 모달 닫기 (취소)

##### 트리거
- ESC 키 입력
- 모달 배경 클릭
- X 버튼 클릭 (있는 경우)

##### 동작
- 모달 닫기 (= [계속 수정]과 동일)

## 4. 규칙/로직

### 모달 표시 조건

| 조건 | 모달 표시 여부 |
| ---- | ------------- |
| 변경 사항이 있음 | 표시 (O) |
| 변경 사항이 없음 | 표시 안 함 (X), 바로 상세 화면으로 이동 |

##### 변경 사항 판단 기준
- 수정 화면의 모든 입력 필드가 기존값과 동일한지 확인
- 하나라도 변경된 경우 "변경 사항 있음"으로 간주

### 버튼 스타일

| 버튼 | 스타일 | 의미 |
| ---- | ------ | ---- |
| [계속 수정] | Secondary | 안전한 선택, 데이터 유지 |
| [취소하기] | Danger | 위험한 선택, 데이터 폐기 |

### 포커스 관리

- 모달 오픈 시 [계속 수정] 버튼에 기본 포커스
- Tab 키로 버튼 간 포커스 이동
- Enter 키로 포커스된 버튼 실행

### 키보드 단축키

| 키 | 동작 |
| --- | ---- |
| ESC | 모달 닫기 (계속 수정) |
| Enter | 포커스된 버튼 실행 |
| Tab | 다음 버튼으로 포커스 이동 |
