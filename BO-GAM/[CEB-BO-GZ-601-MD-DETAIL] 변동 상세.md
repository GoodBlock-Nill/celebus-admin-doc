## 1. 화면 목적

GP 변동 내역의 상세 정보를 보여주는 모달입니다. 변동 ID, 변동일시, 사용자 정보, 변동 유형, GP 변동량, 변동 전후 GP 잔액, 관련 게임 또는 교환 정보를 확인할 수 있습니다.

### 주요 기능

- GP 변동 상세 정보 조회
- 변동 전/후 GP 잔액 확인
- 관련 게임 또는 교환 이력 확인
- 회원 상세 화면으로 이동
- 게임 상세 화면으로 이동
- 교환 상세 모달 오픈

### 진입 경로

- CEB-BO-GZ-601 GP 변동 내역 테이블 Row 클릭

### 3.1 표시 정보

| 필드 | 형식 | 설명 | 예시 |

| 변동 ID | 텍스트 | GP 변동 고유 식별자 | GPT202502031435001 |
| 변동일시 | YYYY.MM.DD HH:mm:ss | GP 증감 시각 | 2025.02.03 14:35:20 |
| 닉네임 | 링크 | 사용자 닉네임 (클릭 시 회원상세 이동) | starlight23 |
| 지갑주소 | 텍스트 (mono) | BSC 지갑주소 | 0x1234...5678 |
| 변동 유형 | Badge | 7가지 유형 중 1개 | [보상] |
| GP 변동량 | +{숫자} GP / -{숫자} GP | 증가/감소 GP 수량 (천 단위 콤마) | +1,000 GP |
| 변동 전 잔액 | {숫자} GP | GP 증감 직전 사용자 GP 잔액 | 5,000 GP |
| 변동 후 잔액 | {숫자} GP | GP 증감 직후 사용자 GP 잔액 | 6,000 GP |
| 관련 게임 | 링크 또는 "-" | 게임 타이틀 (클릭 시 게임상세 이동) 또는 "-" | 오늘의 퀴즈 |
| 관련 교환 | 텍스트 (mono) + 링크 또는 "-" | Txid (BSCScan 링크, 교환 유형만 표시) 또는 "-" | 0x7a3b... |
| 비고 | 텍스트 | 추가 정보 | 정답, CELB 교환 등 |

### 3.3 지갑주소 표시

- 형식: 0x로 시작하는 40자리 16진수
- 표시: 축약형 (0x1234...5678)
- 폰트: mono 폰트 (고정폭)
- 전체 주소는 호버 시 툴팁으로 표시

### 3.4 GP 변동량 표시

| 변동 방향 | 표시 형식 | 색상 |
| ---- | ---- | ---- |
| 증가 | +1,000 GP | 초록색 |
| 감소 | -500 GP | 빨간색 |

### 3.5 관련 게임/교환 표시

#### 게임 관련 변동 (참여, 부스팅, 환급, 보상, 환불)

- 관련 게임: 게임 타이틀 (링크)
- 관련 교환: 표시 안됨 (필드 자체가 보이지 않음)
- 클릭 시: CEB-BO-GZ-202 게임 상세 화면으로 이동 (새 탭)

> **ST 게임 관련 변동**: 참여(PARTICIPATION), 보상(REWARD), 환불(REFUND_CANCEL)이 발생합니다. 부스팅(BOOSTING)과 환급(REFUND)은 PM 게임에서만 발생합니다.

#### 교환 관련 변동 (GP 가져오기, CELB으로 보내기)

- 관련 게임: "-"
- 관련 교환: Txid (BSCScan 링크)
  - 형식: 0x로 시작하는 트랜잭션 해시 전체 표시
  - 폰트: mono 폰트 (고정폭)
  - 클릭 시: 새 탭에서 BSCScan 트랜잭션 페이지 열림
  - 링크 형식: `https://bscscan.com/tx/{txid}`

### 비즈니스 규칙

1. GP 변동 상세는 수정할 수 없습니다 (조회 전용)
2. 변동 전/후 GP 잔액은 변동 시점의 실제 잔액입니다
3. 게임 관련 변동은 게임 ID와 연결되며, 게임 삭제 시에도 타이틀은 표시됩니다
4. 교환 관련 변동은 Txid와 연결되며, 관련 교환 필드에 BSCScan 링크로 표시됩니다
5. 증가 변동: 변동 후 잔액 = 변동 전 잔액 + GP 변동량
6. 감소 변동: 변동 후 잔액 = 변동 전 잔액 - GP 변동량 (절댓값)
7. 관련 게임과 관련 교환은 동시에 존재할 수 없습니다 (둘 중 하나만)
8. 지갑주소는 BSC 메인넷 주소이며, mono 폰트로 표시됩니다
9. 관련 교환 필드는 교환 관련 변동(GP 가져오기/CELB으로 보내기)에만 표시되며, Txid가 BSCScan 링크로 제공됩니다
10. 게임 관련 변동(참여/부스팅/환급/보상/환불)의 경우 관련 교환 필드가 표시되지 않습니다

### 계산 예시

#### 게임 참여 (PARTICIPATION)

```text
변동 전 GP 잔액: 5,000 GP
GP 변동량: -500 GP
변동 후 GP 잔액: 4,500 GP (5,000 - 500)
```

#### 보상 지급 (REWARD)

```text
변동 전 GP 잔액: 4,500 GP
GP 변동량: +1,000 GP
변동 후 GP 잔액: 5,500 GP (4,500 + 1,000)
```

#### 부스팅 (BOOSTING)

```text
변동 전 GP 잔액: 5,500 GP
GP 변동량: -100 GP
변동 후 GP 잔액: 5,400 GP (5,500 - 100)
비고: "2배"
```

#### GP 가져오기 (EXCHANGE_IN)

```text
변동 전 GP 잔액: 5,000 GP
GP 변동량: +2,000 GP
변동 후 GP 잔액: 7,000 GP (5,000 + 2,000)
관련 교환: 0x7a3b... (Txid, BSCScan 링크)
비고: "CELB 교환"
```

#### CELB으로 보내기 (EXCHANGE_OUT)

```text
변동 전 GP 잔액: 7,000 GP
GP 변동량: -1,000 GP
변동 후 GP 잔액: 6,000 GP (7,000 - 1,000)
관련 교환: 0x8c4d... (Txid, BSCScan 링크)
비고: "CELB 교환"
```

### 모달과 외부 링크

- 관련 게임 클릭: 새 탭에서 게임 상세 화면 열림 (모달 유지)
- 관련 교환(Txid) 클릭: 새 탭에서 BSCScan 트랜잭션 페이지 열림 (모달 유지)
- 닉네임 클릭: 새 탭에서 회원 상세 화면 열림 (모달 유지)
- 모달 닫기: Esc 키 또는 닫기 버튼으로 GP 변동 내역 화면으로 복귀

### 용어 정의

| 용어 | 정의 |
| ---- | ---- |
| 변동 ID | GP 변동 고유 식별자 (형식: GPT + YYYYMMDD + HHmmss + 일련번호) |
| 변동일시 | GP가 실제로 증감된 시각 |
| 지갑주소 | BSC 메인넷 지갑주소 (0x로 시작하는 40자리 16진수) |
| GP 변동량 | 증가(+) 또는 감소(-) GP 수량 |
| 변동 전 GP 잔액 | GP 증감 직전 사용자의 GP 잔액 |
| 변동 후 GP 잔액 | GP 증감 직후 사용자의 GP 잔액 |
| 관련 게임 | GP 변동과 연결된 게임 (게임 관련 변동의 경우) |
| 관련 교환 | 블록체인 트랜잭션 해시(Txid)를 BSCScan 링크로 표시 (교환 관련 변동의 경우만 필드 표시) |
