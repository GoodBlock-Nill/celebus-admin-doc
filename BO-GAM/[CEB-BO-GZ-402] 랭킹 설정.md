## 1. 화면 목적

게임존 랭킹 시스템의 설정을 관리하는 화면입니다. TOP 10 공개 여부, 랭킹 집계 기준, 갱신 주기, 최소 참여 횟수 등을 설정합니다.

### 주요 기능

- TOP 10 랭킹 앱 공개 여부 설정
- 랭킹 집계 기준 선택 (누적 GP / 승률 / 참여 횟수)
- 랭킹 갱신 주기 설정 (실시간 / 1시간 / 1일)
- 랭킹 등록 최소 참여 횟수 설정

### 앱 반영

- **TOP 10 공개 설정**: 즉시 앱에 반영
- **집계 기준/갱신 주기**: 다음 갱신 주기부터 적용

## 3. 화면 내 기능설명

### 3.1 TOP 10 공개 Toggle

**ON → OFF**:

- 앱에서 TOP 10 랭킹 숨김
- 전체 랭킹은 유지 (백오피스에서만 조회 가능)

**OFF → ON**:

- 앱에서 TOP 10 랭킹 공개

**즉시 반영**: 저장 시 즉시 앱에 반영

### 3.2 랭킹 집계 기준 변경

**트리거**: 라디오 버튼 선택

**옵션**:

| 기준 | 정렬 방식 | 설명 |
| ---- | ---- | ---- |
| 누적 GP | 누적 GP 내림차순 | 보상-참여 순 수익 기준 |
| 승률 | 승률 내림차순 | (정답 게임 / 참여 게임) × 100% |
| 참여 횟수 | 참여 게임 수 내림차순 | 게임 참여 활동성 기준 |

**적용 시점**: 다음 갱신 주기부터 적용

### 3.3 랭킹 갱신 주기 변경

**트리거**: 드롭다운 선택

**옵션**:

| 주기 | 설명 | 부하 |
| ---- | ---- | ---- |
| 실시간 | 매 게임 완료 시 즉시 갱신 | 높음 |
| 1시간 | 매시 정각에 일괄 갱신 | 중간 |
| 1일 | 매일 00:00 KST에 갱신 | 낮음 |

**권장**: 1시간 (성능과 실시간성 균형)

**적용 시점**: 저장 후 다음 갱신 시점부터 적용

### 3.4 최소 참여 횟수 변경

**트리거**: Input 필드 값 변경

**검증**:

- 최솟값: 1
- 최댓값: 100
- 정수만 입력 가능

**효과**: 최소 참여 횟수 미달 사용자는 랭킹에서 제외

**예시**:

- 최소 참여 횟수 = 5
- 4회 참여 사용자: 랭킹 제외
- 5회 이상 참여 사용자: 랭킹 포함

### 3.5 초기화

**트리거**: [초기화] 버튼 클릭

**확인 다이얼로그**:

```text
모든 설정을 기본값으로 복원하시겠습니까?
변경사항은 저장되지 않습니다.

[취소] [복원]
```

**기본값**:

| 항목 | 기본값 |
| ---- | ---- |
| TOP 10 공개 | ON |
| 랭킹 집계 기준 | 누적 GP |
| 랭킹 갱신 주기 | 1시간 |
| 최소 참여 횟수 | 1 |

**동작**: [복원] 클릭 시 모든 필드를 기본값으로 설정 (저장은 하지 않음)

### 3.6 저장

**트리거**: [저장] 버튼 클릭

**검증**:

1. 최소 참여 횟수 범위 확인 (1~100)
2. 모든 필드 유효성 검사

**성공 시**:

- 토스트 메시지: "랭킹 설정이 저장되었습니다."
- 활동 로그 기록
- 화면 유지

**실패 시**:

- 에러 메시지: "설정 저장에 실패했습니다. 입력값을 확인해주세요."

## 4. 규칙/로직

### 4.1 TOP 10 공개 정책

**ON 상태**:

- 앱 게임존 메인에 "TOP 10 랭킹" 섹션 표시
- 1~3위: 메달 아이콘과 함께 하이라이트
- 4~10위: 일반 스타일

**OFF 상태**:

- 앱에서 랭킹 섹션 전체 숨김
- 백오피스에서는 항상 조회 가능

**반영 시점**: 저장 즉시 앱에 반영 (캐시 갱신)

### 4.2 랭킹 집계 기준

**누적 GP**:

- 공식: (보상 GP + 환급 GP) - (참여 GP + 부스팅 GP)
- 정렬: 누적 GP 내림차순

**승률**:

- 공식: (정답 게임 수 / 참여 완료 게임 수) × 100
- 정렬: 승률 내림차순
- 동률 시: 참여 게임 수 많은 순

**참여 횟수**:

- 집계: 결과 확정된 게임 참여 횟수
- 정렬: 참여 횟수 내림차순
- 동률 시: 누적 GP 많은 순

### 4.3 랭킹 갱신 주기

**실시간**:

- 게임 완료 이벤트 발생 시 즉시 랭킹 재계산
- 높은 부하 가능성
- 소규모 서비스에 적합

**1시간** (권장):

- 매시 정각 (00분)에 배치 작업 실행
- 성능과 실시간성 균형
- 중규모 서비스에 적합

**1일**:

- 매일 00:00 KST에 배치 작업 실행
- 낮은 부하
- 대규모 서비스에 적합

### 4.4 최소 참여 횟수

**필터링 로직**:

```sql
WHERE game_participation_count >= {최소 참여 횟수}
```

**효과**:

- 최소 참여 횟수 미달 사용자 랭킹 제외
- 우연한 1~2회 참여자 제외하여 랭킹 품질 향상

**권장값**: 1~5회

### 4.5 권한 및 제약사항

#### 권한 매트릭스

| 역할 | 읽기 (조회) | 쓰기 (설정 변경) |
| ---- | ---- | ---- |
| Super Admin | ✅ | ✅ |
| Admin | ✅ | ✅ |
| Viewer | ✅ | ❌ |

#### 제약사항

1. **앱 반영 시점**: TOP 10 공개는 즉시 반영, 집계 기준/갱신 주기는 다음 갱신부터
2. **성능 고려**: 실시간 갱신은 트래픽 증가 시 성능 저하 가능
3. **동시 변경 불가**: 여러 관리자가 동시에 설정 변경 시 마지막 저장이 우선

#### 활동 로그

**로그 형식**:

```text
랭킹 설정을 변경했습니다. (TOP 10 공개: {ON/OFF}, 집계 기준: {기준}, 갱신 주기: {주기}, 최소 참여 횟수: {횟수})
```

**예시**:

```text
랭킹 설정을 변경했습니다. (TOP 10 공개: ON, 집계 기준: 누적 GP, 갱신 주기: 1시간, 최소 참여 횟수: 5)
```

## 5. 에러 및 예외상황

### 데이터 로딩 실패

##### 상황
- API 호출 실패 또는 네트워크 오류

##### 처리
1. 에러 토스트 표시
2. [다시 시도] 버튼 제공

### 권한 오류

##### 상황
- 읽기 권한이 없는 관리자가 화면 접근 시도

##### 처리
1. 접근 차단
2. 권한 오류 페이지로 리다이렉트

### 설정 저장 실패

##### 상황
- 입력값 검증 오류 (최소 참여 횟수 범위 초과)
- 서버 오류

##### 처리
1. 에러 토스트 표시: "설정 저장에 실패했습니다. 입력값을 확인해주세요."
2. 잘못된 입력 필드에 빨간색 테두리 표시
3. 에러 메시지 필드 하단에 표시

### 실시간 갱신 성능 저하

##### 상황
- 실시간 갱신 모드에서 트래픽 과다로 성능 저하

##### 처리
1. 경고 메시지: "실시간 갱신으로 인해 서버 부하가 증가하고 있습니다. 1시간 갱신 주기로 변경을 권장합니다."
2. 관리자 알림 발송
