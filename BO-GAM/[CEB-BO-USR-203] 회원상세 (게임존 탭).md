## 1. 화면 목적

회원상세 페이지 내 "게임존" 탭으로, 특정 회원의 게임존 활동 전체를 한눈에 파악할 수 있는 조회 전용 화면입니다.

##### 주요 기능
- 회원의 GP 보유/획득/사용 현황 요약
- 게임존 내 랭킹 순위 확인
- 게임 참여 내역 테이블 조회 (필터, 정렬, 페이지네이션)
- GP 변동 내역 테이블 조회 (필터, 정렬, 페이지네이션)

##### 접근 경로

| 진입 화면 | 경로 |
| ---- | ---- |
| CEB-BO-USR-201 | 회원상세 페이지에서 "게임존" 탭 클릭 |
| CEB-BO-GZ-401 | 랭킹 조회 테이블 ROW 클릭 시 이동 |
| CEB-BO-GZ-202 | 게임 상세 참여내역 탭에서 닉네임 클릭 시 이동 |
| CEB-BO-GZ-601 | GP 변동 내역에서 닉네임 클릭 시 이동 |
| CEB-BO-GZ-501 | 교환 내역에서 닉네임 클릭 시 이동 |

| Card | Label | Value 형식 | 보조 텍스트 | 설명 |
| ---- | ---- | ---- | ---- | ---- |
| Card 1 | 현재 GP 잔액 | {숫자} GP | - | 해당 회원의 현재 GP 보유량 |
| Card 2 | 누적 획득 GP | {숫자} GP | 보상+환급+환불+충전 | REWARD + REFUND + EXCHANGE_IN + REFUND_CANCEL 합산 |

## 3. 화면 내 기능설명

### 3.1 탭 진입

##### 트리거
- 회원상세 페이지에서 "게임존" 탭 클릭
- 외부 화면(랭킹, GP 변동, 교환 내역)에서 닉네임 클릭 시 `tab=gamezone` 파라미터로 직접 진입

##### 동작
1. GP 요약 카드, 게임 성과 요약 카드, 게임 참여 내역(1페이지), GP 변동 내역(1페이지) 동시 로딩
2. 각 영역 독립 API 호출 (한 영역 실패가 다른 영역에 영향 없음)
3. 로딩 중 각 영역 Skeleton UI 표시

### 3.2 게임 참여 내역 테이블

##### 게임유형 Segmented Control 동작

- Segmented Control 변경 시 즉시 적용
- 테이블 컬럼, 데이터, 게임상태 필터 옵션 모두 교체
- 페이지 1로 리셋, 기간 필터는 유지
- 게임상태 필터는 "전체"로 리셋

##### 게임 타이틀 클릭
- 클릭 시: CEB-BO-GZ-202 (게임 상세) 화면 새 탭 오픈
- 전달 데이터: gameId
- 삭제된 게임: 타이틀 텍스트만 표시, 링크 비활성화 (회색 텍스트)

##### 필터 동작
- 필터 변경 시 즉시 적용 (페이지 1로 리셋)
- 다중 필터 AND 조건 적용

##### 정렬 동작
- 기본 정렬: 참여일시 내림차순 (최신순)
- 컬럼 헤더 클릭 시 오름차순 ↔ 내림차순 토글
- 정렬 변경 시 현재 페이지 유지

### 3.3 GP 변동 내역 테이블

##### Row 클릭
- 클릭 시: CEB-BO-GZ-601-MD-DETAIL (GP 변동 상세 모달) 오픈
- 전달 데이터: GP 변동 ID
- 모달은 기존 CEB-BO-GZ-601에서 사용하는 것과 동일

## 4. 규칙/로직

### 데이터 집계 기준

##### 누적 획득 GP

```text
누적 획득 GP = SUM(REWARD) + SUM(REFUND) + SUM(EXCHANGE_IN) + SUM(REFUND_CANCEL)
```

##### 누적 사용 GP

```text
누적 사용 GP = SUM(PARTICIPATION) + SUM(BOOSTING) + SUM(EXCHANGE_OUT)
```

> ST 게임은 부스팅이 없으므로 BOOSTING 항목은 PM 게임에서만 발생합니다.
> ST 게임은 참여 GP 환급이 없으므로 REFUND 항목은 PM 게임에서만 발생합니다.

##### GP 정합성 규칙

```text
현재 GP 잔액 = 누적 획득 GP - 누적 사용 GP
```

- GP 변동 내역 테이블의 최신(첫 번째) Row의 "변동 후 잔액" = GP 요약의 "현재 GP 잔액"
- 모든 시점에서 잔액은 0 이상

##### 랭킹 순위

CEB-BO-GZ-401 랭킹 조회의 집계 기준과 동일:
```text
누적 GP = (보상 GP + 환급 GP) - (참여 GP + 부스팅 GP)
```
- 최소 참여 횟수: CEB-BO-GZ-402 랭킹 설정 값 참조
- 최소 참여 횟수 미달 시: "-" 표시

##### PM 승률

```text
PM 승률 = (정답 수 / PM 결과 확정 게임 수) × 100
- 정답: 회원 선택 = 게임 결과
```

- 결과 확정 게임: Closed 또는 Ended 상태의 PM 게임
- PM 결과 확정 게임 0건 시: "-" 표시

##### ST 생존율

```text
ST 생존율 = (최종 생존 수 / ST 종료 게임 수) × 100
- 최종 생존: 10스테이지 모두 완료
```

- ST 종료 게임: Ended 상태의 ST 게임
- ST 종료 게임 0건 시: "-" 표시

### 제약사항

- 임시저장(Draft), 게시대기(Ready) 상태의 게임은 참여 불가하므로 게임 참여 내역에 표시되지 않음
- GP 변동 내역은 CEB-BO-GZ-601의 데이터를 해당 회원으로 필터링하여 표시
- 게임이 삭제되어도 참여 내역과 GP 변동 내역은 유지됨 (게임 타이틀만 표시, 링크 비활성화)
- 탈퇴 예정 또는 정지 상태의 회원도 게임존 탭 데이터는 정상 조회됨
- 랭킹 순위는 CEB-BO-GZ-402 설정의 갱신 주기에 따라 지연될 수 있음
- 커스텀 기간 필터는 최대 90일까지 선택 가능
- 각 게임 참여는 반드시 대응하는 GP 변동 엔트리가 생성됨:
  - PARTICIPATION: 참여 시 (-참여 GP)
  - BOOSTING: 부스팅 사용 시 (-부스팅 GP, 부스팅 GP > 0인 경우만)
  - REFUND: 결과 확정 후 참여 GP 환급 (+환급 GP, Closed/Ended 게임만)
  - REWARD: 정답 시 보상 지급 (+보상 GP, 정답인 경우만)
- 게임 참여 내역의 참여 GP/부스팅 GP 금액 = GP 변동 내역의 대응 엔트리 금액
- ST 게임의 "환급 GP" 컬럼은 항상 "-"으로 표시됨 (ST는 참여 GP를 환급하지 않음)
- ST 게임의 "부스팅 GP" 컬럼은 "하트 사용"으로 변경되어 하트 사용 횟수를 표시
- ST 게임의 GP 변동 엔트리:
  - PARTICIPATION: 참여 시 (-참여 GP)
  - REWARD: 최종 생존 시 보상 지급 (+보상 GP)
  - BOOSTING, REFUND: 발생하지 않음

## 5. 에러 및 예외상황

##### 게임존 활동 이력 없음
- GP 요약 카드: "0 GP", "0 GP", "0 GP", 랭킹 "-"
- 게임 성과 요약: "0회", "-", "-"
- 게임 참여 내역: "게임 참여 내역이 없습니다." Empty State
- GP 변동 내역: "GP 변동 내역이 없습니다." Empty State

##### 요약 카드 데이터 조회 실패
- 카드에 "-" 표시
- 하단에 "재시도" 링크 제공

##### 게임 참여 내역 조회 실패
- 에러 토스트: "게임 참여 내역을 불러오는 중 오류가 발생했습니다."
- 테이블 영역에 [재시도] 버튼 표시

##### GP 변동 내역 조회 실패
- 에러 토스트: "GP 변동 내역을 불러오는 중 오류가 발생했습니다."
- 테이블 영역에 [재시도] 버튼 표시

##### GP 변동 상세 모달 조회 실패
- 에러 토스트: "상세 정보를 불러올 수 없습니다."

##### 삭제된 게임 참조
- 게임 타이틀은 텍스트로 표시
- 링크 비활성화 (회색 텍스트, 클릭 불가)

##### 커스텀 기간 범위 초과
- 인라인 에러: "조회 기간은 최대 90일까지 선택할 수 있습니다."

##### 네트워크 오류
- 에러 토스트: "네트워크 오류가 발생했습니다. 다시 시도해주세요."

##### 권한 없음
- 회원 읽기 권한 없음: 회원상세 페이지 전체 접근 불가
- 게임존 읽기 권한 없음: 탭 비활성화 또는 "접근 권한이 없습니다." 표시
