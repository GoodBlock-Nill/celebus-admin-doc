1. Prediction Market 게임 생성 하기 화면 수정 요청사항

**필수값**
게임유형 선택 (Prediction Market, Survival Trivia)
타이틀 입력 (한/영/일) [기존유지, 모든 언어가 입력되어야함]
상세 입력 (한/영/일) [에디터모드 기존유지, 모든 언어가 입력되어야함]
보상설정 [기존 유지]
참여정원 [무제한/제한 선택, 제한 선택시 참여정원 입력가능]
참여비용 [기존 유지]
투표 종료일시 [기존 유지]
결과 발표일자 [기존 유지]
결과 확인기준 설명 (한/영/일) [기존 유지, 모든 언어가 입력되어야함]


**옵션값**
힌트 링크 사용여부 [힌트링크 사용시 힌트링크 입력필드 입력가능]
힌트 링크 (사용 시 활성화)
부스팅 사용여부 [부스팅 사용시 부스팅 비용, 부스팅 배수 입력필드 입력가능]
부스팅 비용 
부스팅 배수

**삭제필드**
썸네일
투표 시작일시 > 투표시작일시는 게시일시로 자동적용

**CTA**
취소 > 게임관리화면으로 복귀
임시저장 > 임시저장 상태로 게임생성 (필수값 미입력시에도 활성화)
생성하기 > 게시하기 버튼을 생성하기 버튼으로 변경, 게시대기 상태로 게임생성 (필수값을 모두 입력해야 활성화)

**참고사항**
게임유형에 따라 입력필드가 다르게 보여야함
현재는 Prediction Market 게임만 기획된 상태이므로 Survival Trivia는 선택불가 처리

3. Prediction Market 게임상세 화면 수정 요청사항

**기본정보 탭**
게임생성화면의 변경사항에 맞춰 화면 재구성 필요 (운영자 가독성을 최우선)

**참여내역 탭** 
게임상태에 따라 적절한 화면이 표시되어야함
1) 임시저장 : 참여 내역이 없습니다
2) 게시대기 : 참여 내역이 없습니다
3) 진행중 : 닉네임만 영어소문자로 변경 나머지는 기존 유지
4) 결과대기 : 닉네임만 영어소문자로 변경 나머지는 기존 유지
5) 결과확정 : 닉네임만 영어소문자로 변경 나머지는 기존 유지
6) 종료 : 닉네임만 영어소문자로 변경 나머지는 기존 유지

**결과/보상 탭**
게임상태에 따라 적절한 화면이 표시되어야함
1) 임시저장 : 기존 유지
2) 게시대기 : 임시저장 상태과 동일한 화면으로 표시
3) 진행중 : 기존 유지
4) 결과대기 : 기존 유지
5) 결과확정 : 
   - 결과정보 섹션에서 결과 확인링크 항목 추가
   - 보상현황 섹션에서 미지급보상, 미지급보상자(탈퇴회원) 수, 지분당 보상 GP 추가
   - 보상상세 섹션에서 최대 20명까지 표시(닉네임은 영문소문자)하고 이후로는 페이지네이션 처리, 지분 컬럼추가

1) 종료 : 보상상세 섹션에서 최대 20명까지 표시(닉네임은 영문소문자)하고 이후로는 페이지네이션 처리 그외에는 기존 유지

**CTA**
게임상태에 따라 적절한 버튼이 표시되어야함
1) 임시저장 : 삭제하기(GZ-202-MD-DELETE), 수정하기(GZ-202-EDIT)
2) 게시대기 : 삭제하기(GZ-202-MD-DELETE), 수정하기(GZ-202-EDIT), 게시하기(GZ-202-MD-PUBLISH)
3) 진행중 : 수정하기(GZ-202-EDIT), 강제종료(GZ-202-MD-CLOSE)
4) 결과대기 : 수정하기(GZ-202-EDIT), 결과입력(GZ-202-MD-RESULT)
5) 결과확정 : 결과수정(GZ-202-MD-RESULT), 보상지급(GZ-202-MD-REWARD)
6) 종료 : 버튼없음

4. Prediction Market 게임상세의 GZ-202-EDIT 및 GZ-202-MD-RESULT 수정 요청사항
게임상태에 따라 수정가능한 필드를 다르게 지정되어야함
1) 임시저장 > 수정하기 클릭(게임수정(GZ-202-EDIT)) : 전체필드
2) 게시대기 > 수정하기 클릭(게임수정(GZ-202-EDIT)) : 전체필드
3) 진행중 > 수정하기 클릭(게임수정(GZ-202-EDIT)) : 타이틀, 상세설명, 힌트링크, 투표종료일시, 결과발표 예정일자, 결과 확인기준 설명, 결과확인 링크
4) 결과대기 > 수정하기 클릭(게임수정(GZ-202-EDIT)) : 결과발표 예정일자, 결과 확인기준 설명
5) 결과확정 > 결과수정 클릭(GZ-202-MD-RESULT) : 결과 선택, 결과 확인 링크 입력 [신규 추가]
6) 종료 : 버튼없음

5. [CEB-BO-GZ-202-MD-PUBLISH] 게시 확인 모달 수정 요청
**수정요청**
결과발표예정일자 항목 추가

6. [CEB-BO-GZ-202-MD-RESULT] 결과 입력 모달 수정요청
**수정요청**
결과 확인 링크 입력 필드 추가

- 대시보드 수정: 운영지갑잔액 표시(출금,충전), 전체유저들의 보유한 총 GP 수량 표시(탈퇴유저제외), 탈퇴유저들의 총 GP 수량 표시

**수정요청**

**수정요청**

비고사항: 부스팅 환불, 참여GP 환불
