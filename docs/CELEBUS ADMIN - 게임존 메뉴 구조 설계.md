# CELEBUS ADMIN - 게임존 메뉴 구조 설계

## 1. 클라이언트 문서 분석 요약

| 문서명 | 주요 기능 | ADMIN에서 관리해야 할 항목 |
| ------ | --------- | -------------------------- |
| game.pdf | Game Zone 메인, Prediction Market 상세 | 게임 생성/수정/삭제, 상태 관리, 결과 입력 |
| GP활동내역.pdf | 게임 참여내역, 교환 내역 | 유저별 참여/부스팅/환급/보상 내역 조회 |
| 랭킹.pdf | 누적 GP 기준 전체 랭킹, TOP 10 | 랭킹 조회, TOP 10 공개 설정 |
| 교환소.pdf | GP↔CELB 교환 | 교환 설정(최소 수량, 비율), 교환 내역 조회 |

## 2. 설계 원칙

| 원칙 | 설명 |
| ---- | ---- |
| 확장성 확보 | 새 게임 유형 추가 시 최소한의 개발로 대응 (Phase 2에서 Survival Trivia 등 확장) |
| 공통/개별 분리 | 랭킹/교환소/GP 변동 내역은 공통, 게임 관리는 유형별 분기 |
| 통합 리스트 | 게임별 메뉴 분리 대신 통합 관리 + 유형 필터 |
| Phase 구분 | Sprint 1은 Prediction Market만, 이후 게임 유형 확장 |

## 3. 용어 통일 가이드

| 구분 | 통일 용어 | 설명 |
| ---- | --------- | ---- |
| GP 증가 | 보상, 환급, 충전 | + GP |
| GP 감소 | 참여, 부스팅, 출금 | − GP |
| 교환 방향 | GP 충전 (CELB→GP), GP 출금 (GP→CELB) | 유저 관점 기준 |
| 상금 | 총 상금 GP | "상금풀" 사용 금지, 통일 |
| 게임 상태 | 임시저장/게시대기/진행중/결과 대기/결과 확정/종료 | 한글 표기 통일 |
| GP 이력 | GP 변동 내역 | "거래" 용어 사용 금지 |

## 8. 게임 상태 정의

| 상태 | 코드 | 설명 | Badge 색상 | 앱 반영 |
| ---- | ---- | ---- | ---------- | ------- |
| 임시저장 | Draft | 작성 완료, 미게시 | 회색 | 노출 안됨 |
| 게시대기 | Ready | 생성 완료, 게시 대기 중 | 파란색 | 노출 안됨 |
| 진행중 | Active | 참여 가능 | 초록색 | 참여 가능 |
| 결과 대기 | Pending | 참여 마감, 결과 입력 대기 | 주황색 | 결과 대기 표시 |
| 결과 확정 | Closed | 결과 확정, 보상 지급 전 | 보라색 | 결과 확인 가능 |
| 종료 | Ended | 보상 지급 완료 | 검정색 | 종료 표시 |

### 상태별 가능 액션

| 현재 상태 | 가능 액션 |
| --------- | --------- |
| 임시저장 (Draft) | 삭제하기, 수정하기 |
| 게시대기 (Ready) | 삭제하기, 수정하기, 게시하기 |
| 진행중 (Active) | 수정하기, 강제 종료 |
| 결과 대기 (Pending) | 수정하기, 결과 입력, 강제 종료 |
| 결과 확정 (Closed) | 결과 수정, 보상 지급 |
| 종료 (Ended) | 버튼 없음 |

## 9. GP 변동 유형 정의

| 유형 | 코드 | 설명 | GP 변동 | 비고 |
| ---- | ---- | ---- | ------- | ---- |
| 참여 | PARTICIPATION | 게임 참여 시 | − (차감) | - |
| 부스팅 | BOOSTING | 부스팅 사용 시 | − (차감) | - |
| 환급 | REFUND | 결과 발표 후 참여 GP 반환 | + (증가) | - |
| 보상 | REWARD | 정답 시 보상 지급 | + (증가) | - |
| 환불 | REFUND_CANCEL | 게임 취소 시 환불 | + (증가) | 부스팅 환불, 참여GP 환불 |
| GP 충전 | EXCHANGE_IN | CELB → GP 교환 | + (증가) | - |
| GP 출금 | EXCHANGE_OUT | GP → CELB 교환 | − (차감) | - |
