## 상태별 변형

### 버튼 동작
- **취소**: 모달 닫기, 게임 상세 화면 복귀
- **삭제하기**: 게임 데이터 완전 삭제, 리스트 화면으로 이동

### 성공 시
- 토스트 메시지: "게임이 삭제되었습니다."
- 모달 닫기
- 게임 리스트 화면(CEB-BO-GZ-201)으로 이동

### 실패 시
- 에러 토스트 표시
- 모달 유지 (재시도 가능)

## 디자이너 참고사항

### 모달 크기
- 너비: 480px (작은 모달)
- 높이: auto

### 경고 문구
- 색상: 경고 색상 (주황색 또는 빨간색)
- 위치: 테이블 위

### 요약 테이블
- 2개 행: 타이틀, 상태
- 상태: "임시저장" (Draft 상태에서만 삭제 가능)

### 버튼 스타일
- **취소**: Secondary
- **삭제하기**: Danger (빨간색) - 영구 삭제 강조

### 삭제 가능 조건
- 게임 상태 = Draft (임시저장)만 삭제 가능
- 다른 상태에서는 버튼 비활성화

### 삭제 불가 상태
| 상태 | 사유 |
|------|------|
| Scheduled | 이미 게시됨, 게시 취소 후 삭제 가능 |
| Active | 진행 중, 강제 종료 후 삭제 불가 (보관) |
| Pending | 결과 입력 대기 중, 삭제 불가 (보관) |
| Closed | 보상 지급 대기 중, 삭제 불가 (보관) |
| Ended | 종료됨, 삭제 불가 (이력 보관) |

### 에러 케이스
- 게임 상태가 Draft가 아님: "임시저장 상태의 게임만 삭제할 수 있습니다."
- 이미 삭제됨: "이미 삭제된 게임입니다."
