## 디자이너 참고사항

### Card 간격
- 카드 간 세로 간격: 16px
- 카드 내부 필드 간 간격: 24px
- Label과 Input 간격: 8px

### Card 스타일
- 배경: 흰색 (#FFFFFF)
- 테두리: 1px 연한 회색 (#E5E7EB)
- 모서리 라운드: 8px
- 패딩: 24px (상하좌우)

### Input Number 스타일
- 높이: 40px
- 우측에 ▲▼ 버튼
- 단위 표시: Input 우측 외부에 "GP", "CELB" 텍스트
- 비율 표시: "1 GP = [Input] CELB" 형식

### Toggle Switch 스펙
- 너비: 44px
- 높이: 24px
- 슬라이더 크기: 20×20px
- ON 색상: 파란색 (#3B82F6)
- OFF 색상: 회색 (#D1D5DB)
- Label과 Toggle 배치: 같은 행, Toggle 우측 정렬

### Footer 배치
- 버튼 간 간격: 12px
- 우측 정렬
- 상단 여백: 32px

### 인터랙션
- [초기화] 클릭 → Confirm 모달 표시
- [저장] 클릭 → 유효성 검증 → 비대칭 비율 경고 → API 호출 → Toast
- Toggle OFF → 즉시 반영 (저장 버튼 클릭 필요)

### 초기화 Confirm 모달
- 제목: "모든 설정을 기본값으로 복원하시겠습니까?"
- 메시지: "변경사항은 저장되지 않습니다."
- 버튼: [취소] (Outline), [복원] (Solid/Red)

### 비대칭 비율 경고 모달
조건: GP→CELB 비율 × CELB→GP 비율 ≠ 1.0

```
⚠️ 교환 비율이 비대칭입니다.

GP→CELB: 1 GP = 1.5 CELB
CELB→GP: 1 CELB = 0.8 GP

이 설정은 사용자가 GP ↔ CELB를 반복 교환 시
손실 또는 이득이 발생할 수 있습니다.

계속하시겠습니까?

[취소] [계속 저장]
```

### 기본값 목록 (초기화 시)
- GP → CELB 비율: 1.0
- CELB → GP 비율: 1.0
- 최소 충전 수량: 10 CELB
- 최대 충전 수량: 10,000 CELB
- 1일 충전 한도: 100,000 CELB
- 최소 출금 수량: 10 GP
- 최대 출금 수량: 10,000 GP
- 1일 출금 한도: 100,000 GP
- GP 충전 활성화: ON
- GP 출금 활성화: ON

### 접근성
- Label과 Input 연결 (for, id)
- 에러 메시지에 aria-live="polite"
- Toggle에 aria-label
- 설명 텍스트에 aria-describedby
- 키보드 네비게이션: Tab 순서 (상→하)
