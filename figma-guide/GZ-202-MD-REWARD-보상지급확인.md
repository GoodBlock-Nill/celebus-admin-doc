## 상태별 변형

### 버튼 동작
- **취소**: 모달 닫기, 게임 상세 화면 복귀
- **보상 지급**: 보상 배분 + 참여 GP 환급 실행, 게임 상태 Closed → Ended

### 성공 시
- 토스트 메시지: "보상이 지급되었습니다."
- 모달 닫기
- 게임 상세 화면 갱신 (상태 변경, 액션 버튼 제거)

### 실패 시
- 에러 토스트 표시
- 모달 유지 (재시도 가능)

## 디자이너 참고사항

### 모달 크기
- 너비: 640px (테이블과 안내 박스 포함으로 넓게)
- 높이: auto

### 보상 요약 테이블
- 5개 행으로 구성
- 필드명 열: 좌측 정렬, 굵게
- 값 열: 좌측 정렬

### 보상 계산 안내 박스
- 배경색: 연한 파란색 (Info)
- 패딩: 16px
- 아이콘: 💡 (전구, 20×20px)
- 리스트 스타일: Bullet (•)
- 3개 항목 포함

### 보상 계산 로직 (디자이너 이해용)

#### 기본 원칙
```
총 상금 GP = 게임 생성 시 설정된 고정 금액
정답자 보상 = 총 상금 GP × (개인 배분 가중치 / 전체 정답자 가중치 합)
참여 GP 환급 = 각 참여자가 베팅한 참여 GP 전액
부스팅 GP 환급 = 없음 (환급되지 않음)
```

#### 배분 가중치
```
개인 배분 가중치 = 참여 GP + (부스팅 GP × 2)
```

#### 예시
| 정답자 | 참여 GP | 부스팅 GP | 배분 가중치 | 보상 GP |
|--------|---------|-----------|-------------|---------|
| A | 1,000 | 500 | 2,000 | 2,000 GP |
| B | 2,000 | 1,000 | 4,000 | 4,000 GP |
| 합계 | - | - | 10,000 | 10,000 GP |

### 특수 케이스

#### 정답자가 0명인 경우
- 총 상금 GP: 시스템으로 회수 또는 소멸
- 참여 GP: 모든 참여자에게 전액 환급
- 부스팅 GP: 환급 없음 (소멸)
- 안내 메시지: "정답자가 없습니다. 참여 GP만 환급됩니다."

### 경고 표시
- 아이콘 + 텍스트 조합
- 색상: 주황색 (Warning)
- 위치: 안내 박스 아래
- 1회만 가능, 취소 불가 강조

### 버튼 스타일
- **취소**: Secondary
- **보상 지급**: Primary (파란색)

### GP 지급 실행
1. 각 정답자에게 보상 GP 지급
2. 각 참여자에게 참여 GP 환급
3. GP 변동 내역에 기록 생성
   - 보상 GP: "게임 '{타이틀}' 보상"
   - 환급 GP: "게임 '{타이틀}' 참여 GP 환급"

### 보상 지급 후 처리
- 게임 상태: Closed → Ended
- 보상 지급 완료 플래그 설정
- 더 이상 관리 액션 불가 (조회만 가능)

### 에러 케이스
- 게임 상태가 Closed가 아님: "보상을 지급할 수 없는 상태의 게임입니다."
- 이미 보상 지급됨: "이미 보상이 지급된 게임입니다."
- GP 지급 실패: 트랜잭션 롤백, 게임 상태 유지
