## 1. 게임 상태 Badge (6종)

Preline Badge > Soft 기반으로 커스터마이즈

| 상태 | 코드 | 텍스트 | 배경색 | 텍스트색 | Tailwind Class |
|------|------|--------|--------|---------|---------------|
| 임시저장 | Draft | 임시저장 | gray-100 | gray-600 | bg-gray-100 text-gray-600 |
| 예약 | Scheduled | 예약 | blue-100 | blue-600 | bg-blue-100 text-blue-600 |
| 진행중 | Active | 진행중 | green-100 | green-600 | bg-green-100 text-green-600 |
| 결과 대기 | Pending | 결과 대기 | orange-100 | orange-600 | bg-orange-100 text-orange-600 |
| 결과 확정 | Closed | 결과 확정 | purple-100 | purple-600 | bg-purple-100 text-purple-600 |
| 종료 | Ended | 종료 | gray-800 | white | bg-gray-800 text-white |

**스타일 속성:**
- 패딩: 4px 12px
- 모서리: 4px 라운드
- 폰트: 12px Medium
- 테두리: 없음

**Figma 생성 방법:**
1. Preline Badge > Soft 컴포넌트 복제
2. 6개 Variant 생성 (Draft, Scheduled, Active, Pending, Closed, Ended)
3. 각 Variant별 배경색/텍스트색 위 표대로 적용
4. Component 이름: `GameStatus/Badge`

## 3. 랭킹 Badge (4종)

원형 Badge 커스텀 생성

| 순위 | 배경색 | 텍스트색 | 아이콘 | Tailwind Class |
|------|--------|---------|--------|---------------|
| 1위 | amber-400 (금색) | white | 왕관 또는 "1" | bg-amber-400 text-white |
| 2위 | gray-300 (은색) | gray-700 | "2" | bg-gray-300 text-gray-700 |
| 3위 | amber-600 (동색) | white | "3" | bg-amber-600 text-white |
| 4~10위 | gray-100 | gray-600 | 순위 숫자 | bg-gray-100 text-gray-600 |

**스타일 속성:**
- 크기: 32×32px 원형
- 폰트: 14px Bold
- 중앙 정렬 (가로/세로)
- 텍스트: 순위 숫자 표시

**Figma 생성 방법:**
1. 32×32px 원형 오브젝트 생성
2. 4개 Variant 생성 (Gold, Silver, Bronze, Default)
3. 텍스트 레이어 중앙 정렬
4. Component 이름: `Ranking/Badge`

**사용 예:**
- 1~3위: 해당 Variant 사용
- 4~10위: Default Variant 사용 후 텍스트 Override

## 4. GP 표시 컴포넌트

| 속성 | 값 |
|------|-----|
| 포맷 | "{숫자} GP" (천 단위 콤마) |
| 예시 | "1,234 GP" |
| 증가 표시 | +{숫자} GP, 초록색 (green-600) |
| 감소 표시 | -{숫자} GP, 빨간색 (red-600) |
| 폰트 | 숫자부: Semi Bold, "GP" 접미사: Regular |

**스타일 속성:**
- 숫자부: 16px Semi Bold
- "GP" 텍스트: 14px Regular, gray-500
- 간격: 숫자와 "GP" 사이 4px

**Variant:**
1. Default (gray-900)
2. Increase (green-600, "+" 접두사)
3. Decrease (red-600, "-" 접두사)

**Figma 생성 방법:**
1. Auto Layout (가로 배치, 4px gap)
2. 텍스트 레이어 2개: 숫자부 + "GP"
3. 3개 Variant 생성
4. Component 이름: `GP/Display`

## 5. GP 요약 카드

Preline Card 기반으로 커스터마이즈

| 속성 | 값 |
|------|-----|
| 크기 | 높이 120px, 너비 flex (1fr) |
| 내부 패딩 | 20px |
| 배경 | 흰색 (white) |
| 테두리 | 1px gray-200 |
| 모서리 | 8px 라운드 |
| 상단 라벨 | 14px, gray-500, Regular |
| 하단 수치 | 28px, Bold |
| 수치 색상 | GP=blue-600, CELB=purple-600, 건수=gray-900 |

**구조:**
```
[카드 컨테이너]
  ├── [라벨 텍스트] (예: "총 충전 GP")
  └── [수치 텍스트] (예: "1,234,567 GP")
```

**Variant:**
1. GP (blue-600)
2. CELB (purple-600)
3. Count (gray-900)

**Figma 생성 방법:**
1. Preline Card > Simple 복제
2. Auto Layout (세로 배치, 8px gap)
3. 라벨 + 수치 텍스트 레이어
4. 3개 Variant 생성 (색상 다르게)
5. Component 이름: `Stats/Card`

## 6. 게임 유형 카드 (GZ-101용)

| 요소 | 속성 |
|------|------|
| 카드 크기 | 높이 auto, 너비 50% (2열 그리드) |
| 헤더 | 유형명 (Bold 18px) + 바로가기 링크 (우측) |
| 항목 행 | 라벨 (gray-500, 14px) + 값 (gray-900, 14px), 행 높이 36px |
| 비활성 카드 | 전체 opacity 50%, "준비중" Badge 표시 |
| 테두리 | 1px gray-200, 8px 라운드 |
| 패딩 | 24px |

**구조:**
```
[카드 컨테이너]
  ├── [헤더]
  │     ├── [유형명] (예: "Prediction Market")
  │     └── [바로가기 링크] (예: "바로가기 →")
  ├── [항목 행 1] 진행중: {n}개
  ├── [항목 행 2] 결과 대기: {n}개
  ├── [항목 행 3] 총 참여자: {n}명
  └── [항목 행 4] 총 참여 GP: {n} GP
```

**Variant:**
1. Active (일반 상태)
2. Disabled (비활성, opacity 50%, "준비중" Badge)

**Figma 생성 방법:**
1. Auto Layout (세로 배치)
2. 헤더 행 (유형명 + 링크)
3. 4개 항목 행 (라벨:값 쌍)
4. 2개 Variant 생성
5. Component 이름: `GameType/Card`

## 컴포넌트 네이밍 규칙

모든 커스텀 컴포넌트는 다음 형식으로 명명:

```
{Category}/{Type}
```

**예시:**
- `GameStatus/Badge` (게임 상태 Badge)
- `GPType/Badge` (GP 변동 유형 Badge)
- `Ranking/Badge` (랭킹 Badge)
- `GP/Display` (GP 표시)
- `Stats/Card` (요약 카드)
- `GameType/Card` (게임 유형 카드)
- `Admin/Sidebar` (관리자 사이드바)

## 사용 가이드

1. **컴포넌트 생성 우선순위**
   - 1순위: 게임 상태 Badge (가장 많이 사용)
   - 2순위: GP 표시 컴포넌트
   - 3순위: 관리자 사이드바 (모든 Page에 공통)
   - 4순위: GP 변동 유형 Badge
   - 5순위: 나머지 (필요시)

2. **Figma 라이브러리 등록**
   - 생성한 커스텀 컴포넌트를 로컬 라이브러리로 등록
   - 팀원들과 공유하여 일관성 유지

3. **Preline과의 조화**
   - 커스텀 컴포넌트도 Preline 디자인 시스템 준수
   - 폰트, 간격, 모서리 등 Preline 스타일 가이드 따르기

4. **문서화**
   - 각 컴포넌트 사용 예시 스크린샷 첨부
   - 속성 Override 방법 명시
