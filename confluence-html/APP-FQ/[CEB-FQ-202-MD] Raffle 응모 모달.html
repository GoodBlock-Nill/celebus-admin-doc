1. 화면 목적

Raffle 응모를 위한 최종 확인 단계(Confirmation) 모달입니다. 유저에게 차감될 티켓 수량을 명확히 고지하여 실수로 인한 재화 소모를 방지하고, 응모 확정 시 API를 호출하여 결과를 피드백합니다.

2. UI 설명

2.1 모달 레이아웃

+--------------------------------------------------+
|                                                    |
|  (Dimmed 배경)                                     |
|                                                    |
|     +--------------------------------------+       |
|     |                                      |       |
|     |  응모하기                    (Title)  |       |
|     |                                      |       |
|     |  응모권 {N}장이 차감됩니다.           |       |

2.2 영역별 구성요소

항목 내용

Title "응모하기"
안내 문구 "응모권 {N}장이 차감됩니다.\n진행하시겠습니까?"
부연설명 "사용된 응모권은 취소/반환되지 않습니다." (작은 글씨)

[확인] 버튼 우측 위치

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 호출 조건

  • 호출처: [CEB-FQ-202] Raffle 상세(진행중) 화면
  • 조건:
    • 유저가 응모 수량을 입력하고 [응모하기] 버튼을 클릭했을 때
    • 입력된 수량이 유효 (보유량 이하, 1 이상)할 때

3.2 버튼 동작

[취소] 버튼 클릭 시
  • 모달을 닫고 상세 화면으로 돌아간다 (아무 일도 일어나지 않음)
[확인] 버튼 클릭 시
  • 상태 변경: 버튼 비활성화 및 로딩 인디케이터 표시 (중복 클릭 방지)
결과 처리
성공 (Success) 모달 닫힘 → "응모가 완료되었습니다!" Success 토스트 노출 → 상세 화면 데이터(내 참여 수, 보유 티켓) 갱신
실패 (Fail) 모달 닫힘 → 에러 메시지 Error 토스트 노출 (예: "마감된 이벤트입니다")

4. 규칙/로직

4.1 중복 클릭 방지

  • [확인] 버튼 클릭 시 즉시 버튼 비활성화 + 로딩 인디케이터 표시
  • API 응답 수신 전까지 재클릭 불가

4.2 Dimmed 배경 처리

  • 배경 클릭 시 모달이 닫히지 않음 (의도치 않은 닫힘 방지)

5. 에러 및 예외상황

이벤트 마감
상황 처리
모달이 떠 있는 동안 이벤트 시간이 종료된 경우 [확인] 클릭 시 서버 에러(EVENT_CLOSED) 반환 → "이벤트가 마감되었습니다." Error 토스트 후 상세 화면 새로고침
이벤트 종료됨 (EVENT_CLOSED)

응모 API 호출 시 서버에서 이벤트가 이미 종료(Closed)된 것으로 응답한 경우:

  • 모달 내용 변경: "이벤트가 종료되었습니다"
  • [확인] 버튼만 표시 (응모 버튼 제거)
  • [확인] 클릭 시 모달 닫힘 → 이전 탭(Event)으로 복귀
네트워크 오류
상황 처리
API 호출 중 네트워크 오류 "네트워크 오류가 발생했습니다. 다시 시도해주세요." Error 토스트
