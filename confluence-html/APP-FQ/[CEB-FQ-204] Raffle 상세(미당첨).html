1. 화면 목적

Raffle 추첨 결과, 당첨되지 않은 유저(Loser)에게 결과를 정중하게 안내하는 화면입니다. 부정적인 감정을 최소화하기 위해 위로의 메시지를 전달하며, 서비스 이탈을 방지하고 Fan Quest 메인으로 돌아가 다른 활동을 이어가도록 유도합니다.

주요 기능
  • 미당첨 결과 안내 (위로 이미지 + 메시지)
  • 위로 문구 랜덤 노출 (Variation)
  • Event으로 복귀 유도
접근 경로
  • Event(CEB-FQ-201) > Raffle 탭 > '미당첨' 리스트 클릭
  • 결과 발표 Push 알림 클릭

3. 화면 내 기능설명

뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
동작
  1. 이전 화면으로 복귀
  2. 네비게이션 스택에서 현재 화면 제거

위로 이미지

표시 규칙
  • 아쉬움을 표현하는 감성적인 일러스트 또는 아티스트의 응원 이미지
  • 채도가 낮거나 차분한 톤앤매너 적용
  • 당첨 화면(CEB-FQ-203)의 화려한 축하 이미지와 대비되는 차분한 분위기

위로 문구 (Variation)

표시 규칙
  • 단조로움을 피하기 위해 위로 문구를 랜덤하게 노출 (서버 또는 클라이언트 로직)
  • 매 진입 시 아래 문구 세트 중 하나를 랜덤 선택
문구 세트
# 메인 문구 서브 문구
1 아쉽게도 당첨되지 않았어요. 다음 기회에 다시 도전해주세요! 더 좋은 이벤트로 찾아올게요.
2 이번엔 아쉽지만, 다음엔 꼭 당첨될 거예요! (서브 문구 없음)
3 다음 기회엔 꼭 좋은 소식 들려드릴게요 :) (서브 문구 없음)
4 오늘의 운은 잠시 쉬는 중… 내일은 다시 올라갈 거예요!
5 이번에는 놓쳤지만, 기회는 또 올 거예요! 더 재미있는 이벤트로 다시 찾아올게요
6 헉! 아쉽게도 떨어졌어요. 다음엔 우리 같이 꼭 잡아봐요

확인 버튼

트리거
  • Footer 하단 고정 [확인] 버튼 탭
동작
  1. Event(CEB-FQ-201) > Raffle 탭 화면으로 이동
  2. 네비게이션 스택에서 현재 화면 제거
스타일
  • Secondary(Line) 또는 Text Button (너무 강조하지 않음)
  • 미당첨 화면의 차분한 톤에 맞춰 과도하게 눈에 띄지 않는 스타일 적용

4. 규칙/로직

진입 조건 및 상태 체크

유효성 검증
  1. 서버에서 해당 유저의 is_winner 값이 false인지 확인
  2. 래플 상태가 Ended(종료)인지 확인
  3. 두 조건이 모두 충족되어야 미당첨 화면 표시

화면 진입 시 동작

데이터 로딩
  1. Raffle ID를 파라미터로 전달받음
  2. 서버에서 래플 결과 정보 조회 (is_winner, 래플 상태)
  3. Loading 상태 표시 (Skeleton UI)
  4. 데이터 로딩 완료 후 화면 렌더링
  5. 위로 문구 랜덤 선택 및 표시

다국어 처리

지원 언어
  • 한국어 (KO)
  • 영어 (EN)
  • 일본어 (JP)
적용 대상
  • 메인 위로 문구
  • 서브 위로 문구
  • 버튼 텍스트
표시 규칙
  • 사용자 앱 언어 설정에 따라 자동 선택
  • 해당 언어 데이터가 없는 경우 기본 언어(KO) 표시

5. 에러 및 예외상황

당첨자 접근 오류

상황
  • 실제 당첨자가 시스템 오류로 미당첨 화면을 보게 되는 경우 (치명적)
처리
  1. 진입 시 is_winner API 재조회
  2. true일 경우 즉시 당첨 화면(CEB-FQ-203)으로 강제 이동(Replace)
  3. 에러 토스트 없음 (자연스러운 화면 전환)

데이터 로딩 실패

상황
  • API 호출 실패 또는 네트워크 오류
처리
  1. Error 상태 표시
  2. 에러 메시지: "정보를 불러오는 데 실패했습니다."
  3. [다시 시도] 버튼 제공
  4. 버튼 탭 시 데이터 재로딩

Loading 상태

처리
  1. Skeleton UI 표시
    • 위로 이미지: 회색 사각형
    • 메인 문구: 회색 박스 (1줄)
    • 서브 문구: 회색 박스 (2줄)
  2. 로딩 완료 시 실제 데이터로 전환 (페이드인 애니메이션)

네트워크 연결 끊김

상황
  • 오프라인 상태에서 화면 진입 또는 데이터 로딩 시도
처리
  1. Error 상태 표시
  2. 에러 메시지: "네트워크 연결을 확인해주세요."
  3. [다시 시도] 버튼 제공
  4. 온라인 복구 시 자동 재로딩
