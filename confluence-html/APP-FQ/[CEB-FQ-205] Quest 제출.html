1. 화면 목적

유저가 퀘스트의 구체적인 수행 방법(가이드)을 확인하고, 요구사항에 맞는 이미지(인증샷)를 촬영하거나 업로드하여, 운영진에게 심사를 요청(제출)하는 화면입니다.

주요 기능
  • 퀘스트 기본정보 조회 (타이틀, 설명)
  • 이미지 촬영 또는 갤러리 업로드
  • 업로드 사진 가이드 확인 (아코디언)
  • 제출하기 플로우 (심사 요청)
접근 경로
  • 홈 메인(CEB-HOM-501-FQ) > Event > Quest 카드 탭 시 이동
  • 홈 메인(CEB-HOM-501-FQ) > QUESTS 섹션 카드 탭 시 이동
  • 상태가 Available(참여가능) 또는 Rejected(반려)인 퀘스트만 진입 가능

3. 화면 내 기능설명

3.1 초기 진입 및 유효성 체크

진입 조건
퀘스트 상태 처리
Available (참여가능) 본 화면 진입 가능
Rejected (반려) 본 화면 진입 가능 (기존 반려 이미지는 초기화/삭제된 상태로 진입)
Pending (검토중) 진입 불가 (리스트에서 토스트 처리)
Completed (완료) 진입 불가 (리스트에서 토스트 처리)

3.2 이미지 업로드 (Upload)

  • 소스 선택: 업로드 영역 클릭 시 OS 기본 피커(카메라/갤러리)를 호출
파일 제한
항목 규칙
형식 JPG, PNG, HEIC
용량 최대 10MB (초과 시 에러 처리)
상태 변화
상태 표시
이미지 선택 전 Placeholder 노출, [제출하기] 버튼 Disabled
이미지 선택 후 Preview 노출, [제출하기] 버튼 Active
이미지 삭제 시 다시 초기 상태(Placeholder)로 복귀

3.3 제출하기 버튼 동작

  • 클릭 시: 중복 제출 방지를 위해 버튼을 로딩 상태(Spinner)로 변경하고 클릭을 막음
  • API 호출: 이미지를 서버로 전송
결과 처리
성공 "제출이 완료되었습니다. 심사 결과는 Event에서 확인해주세요." Success 토스트 노출 → 화면을 닫고 이전 리스트 화면으로 복귀 → 해당 퀘스트의 로컬 상태를 [심사 중]으로 업데이트
실패 에러 유형별 Error 토스트 노출 (5. 에러 및 예외상황 참조)

제출 즉시 해당 건은 Admin의 검토 대기 목록([CEB-BO-FQ-202])에 등록됩니다.
