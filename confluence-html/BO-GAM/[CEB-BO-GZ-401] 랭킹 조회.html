1. 화면 목적

게임존 참여자들의 누적 GP 기준 랭킹을 조회하는 화면입니다. TOP 10 사용자를 하이라이트하여 표시하고, 전체 랭킹 목록을 제공합니다.

주요 기능

  • GP 기준 전체 랭킹 조회
  • TOP 10 하이라이트 섹션 (5열 그리드 카드형)
  • 닉네임 검색
  • 회원 상세 화면 연동

랭킹 집계 기준

  • 누적 GP: 보상 GP + 환급 GP - 참여 GP - 부스팅 GP (순 수익)
  • 승률: (정답 게임 수 / 참여 완료 게임 수) × 100%

통합 랭킹: 랭킹은 Prediction Market과 Survival Trivia를 포함한 모든 게임유형의 누적 GP를 통합하여 산정합니다.

  • PM: 보상 GP(비율 배분) + 환급 GP - 참여 GP - 부스팅 GP
  • ST: 보상 GP(균등 분배) - 참여 GP (부스팅, 환급 없음)
  • 승률: PM 정답 + ST 최종 생존 = 성공 게임으로 통합 집계

4. 규칙/로직

4.1 누적 GP 집계

  • 공식:
누적 GP = (보상 GP + 환급 GP) - (참여 GP + 부스팅 GP)
  • 구성 요소:
항목 설명 부호
보상 GP 정답 맞힌 게임의 보상 +
환급 GP 게임 취소/자동 환급 시 환급액 +
참여 GP 게임 참여 시 지불한 GP -
부스팅 GP 부스팅으로 지불한 GP -

게임유형별 구성 요소

  • Prediction Market: 보상 GP, 환급 GP, 참여 GP, 부스팅 GP 모두 적용
  • Survival Trivia: 보상 GP, 참여 GP만 적용 (부스팅 GP = 0, 환급 GP = 0)
  • 누적 GP는 모든 게임유형을 통합하여 산정
  • 예시:
  • 보상 GP: 50,000
  • 환급 GP: 10,000
  • 참여 GP: 8,000
  • 부스팅 GP: 2,000
  • 누적 GP = (50,000 + 10,000) - (8,000 + 2,000) = 50,000 GP

4.2 승률 계산

  • 공식:
승률 = (정답 게임 수 / 참여 완료 게임 수) × 100
  • 정답 게임: 사용자가 선택한 선택지가 최종 결과와 일치한 게임
  • 참여 완료 게임: 결과가 확정된 게임 (완료 상태)
  • 소수점: 반올림하여 정수 표시 (예: 85%)

게임유형별 정답 판정

  • PM 정답: 사용자가 선택한 선택지(YES/NO)가 최종 결과와 일치
  • ST 정답: 최종 생존 (10스테이지 모두 완료)
  • 승률은 PM + ST 통합 집계

4.3 참여 게임 수

  • 집계 기준: 결과가 확정된 게임 (완료 상태)
  • 제외: 대기, 모집중, 진행중, 취소 상태 게임

4.4 최근 참여일

  • 집계 기준: 사용자가 마지막으로 게임에 참여한 날짜
  • 형식: YYYY.MM.DD

4.5 랭킹 갱신 주기

  • 기본: 1시간마다 집계 (CEB-BO-GZ-402 설정에서 변경 가능)
  • 옵션: 실시간 / 1시간 / 1일

4.6 최소 참여 횟수

  • 기준: CEB-BO-GZ-402 설정에서 정의 (기본값: 1회)
  • 적용: 최소 참여 횟수 미달 사용자는 랭킹에서 제외

4.7 닉네임 형식

  • 허용 문자: 영문 소문자(a-z), 숫자(0-9), 밑줄(_)
  • 정규식: ^[a-z0-9_]+$
  • 적용: 앱에서 회원가입 시 적용되는 규칙이며, 백오피스에서는 표시만 함
