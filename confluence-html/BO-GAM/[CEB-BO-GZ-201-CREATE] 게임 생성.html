공통 정책 참조: 게임 상태/액션, 보상 계산, GP 변동 유형, 권한, UI 상태, 토스트/포맷 규격은 [CEB-BO-GZ-000] 공통 정책을 참조하세요.

1. 화면 목적

새로운 게임을 생성하는 폼 화면입니다. 관리자는 게임 유형(Prediction Market 또는 Survival Trivia)을 선택하고, 기본 정보, 보상 설정, 참여 설정, 일정 설정, 결과 설정을 입력하여 게임을 임시저장하거나 즉시 게시할 수 있습니다. 선택한 게임 유형에 따라 폼 항목이 조건부로 표시됩니다.

3. 화면 내 기능설명

다국어 입력

타이틀 입력
  • KO/EN/JP 탭으로 언어 전환
  • 각 언어별 50자 제한
  • 실시간 글자 수 표시: "25/50"
  • 필수 입력: 모든 언어 필수 (KO/EN/JP) (권장: 모든 언어)
상세설명 입력
  • KO/EN/JP 탭으로 언어 전환
  • WYSIWYG 에디터 제공 (Toolbar + 편집 영역)
  • Toolbar 기능: Bold, Italic, Underline, 링크 삽입, 순서 없는 리스트, 순서 있는 리스트, 이미지 삽입, 좌측/중앙/우측 정렬
  • 각 언어별 500자 제한
  • 필수 입력: 모든 언어 필수 (KO/EN/JP)

힌트 링크 입력 (Prediction Market 전용)

힌트 링크
  • Segmented Button: 사용/미사용 (기본값: 미사용)
  • "사용" 선택 시 URL 입력 필드 표시
  • "미사용" 선택 시 URL 입력 필드 숨김
  • Placeholder: "https://example.com/hint"
  • 도움말 텍스트: "게임 관련 힌트 페이지 URL을 입력하세요."
  • URL 형식 검증 (입력 시)

퀴즈 입력 (Survival Trivia 선택 시)

문제당 제한시간
  • Number Input 사용
  • 기본값: 10초
  • 최소값: 5초, 최대값: 60초
문제 텍스트
  • KO/EN/JP 탭으로 언어 전환
  • 각 언어별 200자 제한
  • 실시간 글자 수 표시
  • 필수 입력: 모든 언어 필수 (KO/EN/JP)
선택지 입력
  • 최소 2개, 최대 4개 (기본 4개)
  • [+ 선택지 추가] 버튼으로 선택지 추가 가능 (4개 도달 시 버튼 Disabled)
  • 각 선택지 우측 [삭제] 버튼으로 삭제 가능 (2개 남으면 버튼 Disabled)
  • 각 선택지: KO/EN/JP 탭으로 언어 전환
  • 각 언어별 100자 제한
  • 필수 입력: 모든 언어 필수 (KO/EN/JP)
정답 지정
  • 각 문제별 Radio 버튼으로 등록된 선택지 중 정답 1개 지정
  • 정답으로 지정된 선택지 삭제 시 첫 번째 선택지(선택지 1)로 정답 자동 초기화
  • 정답 미지정 시 저장/생성 불가
정답결과 노출시간
  • 각 문제별 정답 결과화면 카운트다운 시간 설정
  • 기본값: 5초, 범위: 1~10초
  • Number Input으로 입력
문제 순서 재정렬
  • 드래그앤드롭으로 문제 순서 변경 가능
문제 추가 및 삭제
  • 기본 10문제로 시작
  • [+ 문제 추가] 버튼으로 문제 추가 가능 (최대 10문제, 초과 시 버튼 Disabled)
  • 각 문제 헤더의 [삭제] 버튼으로 문제 삭제 가능 (최소 1문제, 1개 남으면 버튼 Disabled)
  • 문제 삭제 후 문제 번호 자동 재정렬 (Q1~QN 순서 유지)

보상설정 입력

총 상금 GP (Prediction Market)
  • Number Input 사용
  • 양수만 입력 가능
  • 천 단위 콤마 자동 표시
  • 최소값: 1 GP
최대 상금풀 (Survival Trivia)
  • Number Input 사용
  • 양수만 입력 가능
  • 천 단위 콤마 자동 표시
  • 최소값: 1 GP
배수 (Survival Trivia)
  • Number Input 사용
  • 기본값: 1.25배
  • 최소값: 1.01배 (1배 초과)
  • 소수점 2자리까지 입력 가능
참여비 (Survival Trivia, 자동 계산)
  • Readonly 필드 (수동 편집 불가)
  • 계산 공식: floor(최대 상금풀 / 최대 모집인원 × 배수)
  • 최대 상금풀, 최대 모집인원, 배수 중 하나라도 변경 시 실시간 재계산
  • 최대 상금풀 또는 최대 모집인원이 미입력 시 "—" 표시
  • 천 단위 콤마 자동 표시, GP 단위
보상 유형 (Survival Trivia)
  • Segmented Button: 단계별 보상 (기본) / 비율별 보상
  • 기본값: 단계별 보상
  • "단계별 보상" 선택 시: 모집인원별 상금 단계 테이블 표시
  • "비율별 보상" 선택 시: 테이블 숨기고 공식 안내 표시
    • 적용 상금풀 = 실제 모집인원 / 최대 모집인원 × 최대 상금풀
    • 안내 텍스트: "게임 시작 시 실제 모집인원에 비례하여 상금풀이 자동 결정됩니다."
모집인원별 상금 단계 (Survival Trivia, 단계별 보상 선택 시에만 표시)
  • 테이블 형태: 모집률(%) | 상금 비율(%) | 삭제 버튼
  • 기본 템플릿: 100%→100%, 80%→80%, 50%→50%, 20%→20% (4행)
  • [+ 단계 추가] 버튼으로 행 추가
  • 각 행의 [X] 버튼으로 삭제 (최소 1개 행 필수)
  • 모집률: 0~100%, Number Input
  • 상금 비율: 0~100%, Number Input
  • 모집률 기준 내림차순 자동 정렬
  • 중복 모집률 입력 불가 (인라인 에러: "동일한 모집률이 이미 존재합니다.")
  • 단계 사이 모집률은 하위 단계 적용 (보수적 적용 원칙)
탈락자 응모권 수량 (Survival Trivia)

  • 기본값: 1장
  • 최소값: 1장
  • 팬퀘스트 응모권으로 지급 (모든 탈락자 대상, 관전 여부 무관)
