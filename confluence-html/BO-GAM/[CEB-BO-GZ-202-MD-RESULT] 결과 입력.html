1. 화면 목적

게임이 종료되어 Pending 상태일 때 관리자가 게임의 최종 결과(YES 또는 NO)를 입력하는 모달입니다. 결과는 Closed 상태에서 수정 가능하며, 입력된 결과를 기반으로 정답자를 판정하여 보상 지급 대상을 확정합니다. 이 모달은 게임존에서 가장 중요한 관리 액션 중 하나입니다.

게임유형별 적용 범위

  • Prediction Market: 이 모달을 사용하여 관리자가 수동으로 결과(YES/NO)를 입력합니다.
  • Survival Trivia: 이 모달은 표시되지 않습니다. ST는 서버에서 실시간으로 각 스테이지의 정답/오답을 자동 판정하며, 게임 종료 시 최종 생존자가 자동으로 확정됩니다. 관리자의 수동 결과 입력이 필요하지 않습니다.

결과제목 입력

  1. 다국어 입력 필드 (KO/EN/JP 탭 전환)
  2. 각 언어별 50자 제한
  3. 실시간 글자 수 표시: "25/50"
  4. 필수 입력 항목
  5. Placeholder: "결과 제목을 입력하세요."
  6. 모든 언어(KO/EN/JP) 입력 필수

결과 선택 동작

YES 버튼 클릭

  1. YES 선택 상태로 변경
  2. YES 버튼: 초록색 테두리 + 초록색 배경 적용
  3. NO 버튼: 회색 테두리 복원
  4. confirmed 상태 초기화 (false)
  5. "결과 확정" 버튼 활성화

NO 버튼 클릭

  1. NO 선택 상태로 변경
  2. NO 버튼: 빨간색 테두리 + 빨간색 배경 적용
  3. YES 버튼: 회색 테두리 복원
  4. confirmed 상태 초기화 (false)
  5. "결과 확정" 버튼 활성화

결과설명 입력

  1. 다국어 입력 필드 (KO/EN/JP 탭 전환)
  2. 각 언어별 500자 제한
  3. 실시간 글자 수 표시: "25/500"
  4. 필수 입력 항목
  5. Placeholder: "결과에 대한 설명을 입력하세요."
  6. 모든 언어(KO/EN/JP) 입력 필수

버튼 동작

취소 버튼

  1. 모달 닫기
  2. 게임 상세 화면으로 복귀
  3. 상태 변경 없음

결과 확정 버튼

결과 미선택 시
  • 버튼 비활성화 (클릭 불가)
  • 스타일: 회색 배경, 커서 not-allowed
  • 버튼 텍스트: "결과 확정"
결과 선택 시 (1차 클릭)
  1. confirmed 상태를 true로 변경
  2. 주황색 경고 박스 표시:
    • 배경: 주황색 (bg-orange-50)
    • 테두리: 주황색 (border-orange-200)
    • 메시지: "결과 확정 후에는 변경할 수 없습니다. 정말로 "{YES/NO}"(으)로 확정하시겠습니까?"
  3. 버튼 텍스트 변경: "결과 확정" → "최종 확정"
결과 선택 시 (2차 클릭 - 최종 확정)
  1. onConfirm(selected, resultTitle, resultDescription, resultLinkText, resultLinkUrl) 호출
    • selected: 선택된 결과 (YES 또는 NO)
    • resultTitle: 결과제목 (다국어, 필수)
    • resultDescription: 결과설명 (다국어, 필수)
    • resultLinkText: 결과링크 텍스트 (다국어, 선택)
    • resultLinkUrl: 결과링크 URL (다국어, 선택)
  2. 성공 시:
    • 게임 상태: Pending → Closed
    • 게임 결과 저장 (결과값 + 결과설명 + 결과 확인 링크)
    • 정답자/오답자 판정 완료
    • 토스트 메시지: "게임 결과가 확정되었습니다." (Success)
    • 모달 닫기
    • 상태 초기화 (selected=null, resultTitle={ko:'', en:'', jp:''}, resultDescription={ko:'', en:'', jp:''}, resultLinkText={ko:'', en:'', jp:''}, resultLinkUrl={ko:'', en:'', jp:''}, confirmed=false)
    • 게임 상세 화면 갱신 (결과 표시, 보상 지급 버튼 활성화)
  3. 실패 시:
    • 에러 토스트 표시
    • 결과 입력 모달 유지 (재시도 가능)

결과링크 입력

  1. 다국어 입력 필드 (KO/EN/JP 탭 전환)
  2. 각 언어별 링크텍스트 + 링크 URL 쌍으로 구성
  3. 링크텍스트: 링크에 표시할 텍스트 (예: "FIFA 공식 결과 페이지")
  4. 링크 URL: URL 형식 검증
  5. 선택 입력 항목 (필수 아님)
  6. Placeholder (링크텍스트): "링크 텍스트를 입력하세요."
  7. Placeholder (링크 URL): "https://example.com/result-proof"
  8. 도움말: "결과를 확인할 수 있는 링크를 입력하세요. (선택)"

닫기(X) 버튼

  • 취소 버튼과 동일한 동작
  • 상태 초기화: selected=null, resultTitle={ko:'', en:'', jp:''}, resultDescription={ko:'', en:'', jp:''}, resultLinkText={ko:'', en:'', jp:''}, resultLinkUrl={ko:'', en:'', jp:''}, confirmed=false

4. 규칙/로직

결과 입력 가능 조건

조건 검증 시점 실패 시 처리
게임유형 = Prediction Market 모달 오픈 전 버튼 비표시 (ST일 경우)
게임 상태 = Pending 또는 Closed 모달 오픈 전 버튼 비활성화
참여 종료일시 경과 모달 오픈 전 상태 확인 필요
관리자 권한 확인 API 호출 시 403 에러

결과 수정 가능 조건

조건 설명
게임 상태 = Closed 결과 확정 후 Closed 상태에서 "결과 수정" 버튼으로 재오픈 가능
상태별 가능 액션 constants.ts: Closed: ['editResult', 'distributeReward']
경고 문구 UX적 주의 안내로 유지 ("결과 확정 후에는 변경할 수 없습니다")
실제 동작 Closed 상태에서도 결과 수정 가능하나, 신중한 사용 유도

참여자 통계 계산

총 참여자 수

총 참여자 수 = YES 선택자 수 + NO 선택자 수

예상 정답자 수

  • YES 버튼 카드: YES를 선택한 참여자 수
  • NO 버튼 카드: NO를 선택한 참여자 수
  • 비율(%)은 표시하지 않음

정답자 판정

YES가 결과일 때

  • 정답자: YES를 선택한 모든 참여자
  • 오답자: NO를 선택한 모든 참여자

NO가 결과일 때

  • 정답자: NO를 선택한 모든 참여자
  • 오답자: YES를 선택한 모든 참여자

결과 확정 후 처리

  1. 게임 결과 저장 (YES 또는 NO)
  2. 결과제목 저장 (다국어, 필수)
  3. 결과설명 저장 (다국어, 필수)
  4. 결과링크 저장 (링크텍스트 + URL, 다국어, 선택사항)
  5. 각 참여자에게 정답/오답 플래그 저장
  6. 게임 상태: Pending → Closed
  7. 보상 계산 준비 (실제 지급은 보상 지급 모달에서 수행)
