1. 화면 목적

사용자가 CELB 토큰과 GP(Game Point)를 상호 교환할 수 있는 화면입니다. GP 충전(CELB→GP)과 GP 출금(GP→CELB) 기능을 제공하며, 사용자의 GP 잔액을 실시간으로 확인하고 교환 내역으로 이동할 수 있습니다.

주요 기능

  • GP 잔액 조회 (실시간)
  • GP 충전 (CELB → GP)
  • GP 출금 (GP → CELB)
  • 교환 비율 안내 (1 GP = 1 CELB)
  • CELB 잔액 조회 (가용 잔액)
  • CELB 입금주소 확인
  • GP 히스토리 이동
  • 본인 인증 연동

접근 경로

Game 탭 > 게임존 메인 (CEB-GAM-101) > [GP 교환소] 버튼
또는
Game 탭 > 하단 플로팅 버튼 > [GP 교환소]

3. 화면 내 기능설명

3.1 GP 잔액 카드

GP 수량 표시

  • 사용자의 현재 GP 잔액을 실시간으로 표시
  • 천 단위 콤마 적용 (예: 2,914 GP)
  • 교환 완료 시 즉시 갱신

카드 탭 동작

  • 카드 영역 탭 시: CEB-GAM-501 (GP 히스토리) 화면으로 이동
  • GP 변동 내역 전체 조회 가능

3.2 GP 가져오기 탭 (CELB → GP)

가용 CELB 잔액 조회

  • 표시 형식: "가능: {숫자} CELB"
  • 소수점 8자리까지 표시 가능 (예: 2,500.12345678 CELB)
  • 네트워크에서 실시간 조회 (초기 로딩 시)

새로고침 버튼

  • 위치: 가용 잔액 우측
  • 아이콘: 🔄 (회전 아이콘)
  • 동작:
    1. 버튼 탭 시 아이콘 회전 애니메이션 (360도, 0.5초)
    2. CELB 잔액 재조회
    3. 조회 완료 시 애니메이션 종료
    4. 실패 시: Toast "잔액 조회에 실패했습니다. 다시 시도해주세요."

CELB 입력

  • 입력 타입: Number Input (정수 only)
  • 최솟값: 100 CELB (백오피스 설정값, 기본값)
  • 최댓값: 10,000 CELB (1회 충전 한도, 백오피스 설정값)
  • 입력 제한: 가용 잔액 이하
  • 입력 시 GP 출력 필드 자동 계산 (1:1 비율)

GP 출력

  • 계산식: 입력 CELB × 교환 비율 (기본 1:1)
  • 표시 형식: "{숫자} GP"
  • 자동 계산 (입력 즉시 반영)

CELB 입금주소 버튼

  • 라벨: [CELB 입금주소]
  • 스타일: 회색 테두리 버튼 (Secondary)
  • 동작: CEB-GAM-302 (CELB 입금주소 상세) 화면으로 이동
  • 설명: CELB를 입금할 수 있는 지갑 주소 안내

GamePoint 가져오기 버튼

  • 라벨: [GamePoint 가져오기]
  • 위치: 하단 고정 CTA
  • 스타일:
    • 비활성: 회색 (입력 미완료)
    • 활성: 파란색 Primary (입력 완료)
  • 활성화 조건:
    • CELB 입력값 ≥ 최소 충전 수량 (100 CELB)
    • CELB 입력값 ≤ 최대 충전 수량 (10,000 CELB)
    • CELB 입력값 ≤ 가용 잔액
    • CELB 입력값 ≤ 1일 충전 한도 잔여 금액
  • 동작:
    1. 버튼 탭 시 본인 인증 모달 (CEB-AUTH-115) 오픈
    2. 인증 성공 시 확인 모달 (CEB-GAM-301-MD-CONFIRM-IN) 오픈
    3. 확인 시 교환 요청 (API 호출)
    4. 성공 시: Toast "GP 충전이 완료되었습니다." + GP 잔액 갱신
    5. 실패 시: Toast + 실패 사유

3.3 CELB으로 보내기 탭 (GP → CELB)

보유 GP 잔액 표시

  • 표시 형식: "보유: {숫자} GP"
  • 천 단위 콤마 적용
  • 실시간 잔액 표시

GP 입력

  • 입력 타입: Number Input (정수 only)
  • 최솟값: 100 GP (백오피스 설정값, 기본값)
  • 최댓값: 10,000 GP (1회 출금 한도, 백오피스 설정값)
  • 입력 제한: 보유 GP 이하
  • 입력 시 CELB 출력 필드 자동 계산 (1:1 비율)

CELB 출력

  • 계산식: 입력 GP × 교환 비율 (기본 1:1)
  • 표시 형식: "{숫자} CELB"
  • 자동 계산 (입력 즉시 반영)

지갑 주소 입력

  • 입력 타입: Text Input
  • 형식: 블록체인 지갑 주소 (예: 0x7a3b...8a9b)
  • 검증: 주소 형식 검증 (16진수, 42자)
  • Placeholder: "지갑 주소 입력"

CELB으로 보내기 버튼

  • 라벨: [CELB으로 보내기]
  • 위치: 하단 고정 CTA
  • 스타일:
    • 비활성: 회색 (입력 미완료)
    • 활성: 파란색 Primary (입력 완료)
  • 활성화 조건:
    • GP 입력값 ≥ 최소 출금 수량 (100 GP)
    • GP 입력값 ≤ 최대 출금 수량 (10,000 GP)
    • GP 입력값 ≤ 보유 GP
    • GP 입력값 ≤ 1일 출금 한도 잔여 금액
    • 지갑 주소 입력 완료 + 형식 유효
  • 동작:
    1. 버튼 탭 시 본인 인증 모달 (CEB-AUTH-115) 오픈
    2. 인증 성공 시 확인 모달 (CEB-GAM-301-MD-CONFIRM-OUT) 오픈
    3. 확인 시 교환 요청 (API 호출)
    4. 성공 시: Toast "GP 출금이 완료되었습니다." + GP 잔액 갱신
    5. 실패 시: Toast + 실패 사유

3.4 본인 인증 연동

인증 시점

  • GP 충전 실행 시 (버튼 탭 후)
  • GP 출금 실행 시 (버튼 탭 후)

인증 방식

  • CEB-AUTH-115 (본인 인증) 화면 호출
  • 생체 인증 / 비밀번호 선택
  • 인증 성공 시: 확인 모달로 진행
  • 인증 실패 시: Toast "본인 인증에 실패했습니다."

3.5 사용자 시나리오

시나리오 1: GP 충전 (CELB → GP)

  1. 사용자가 [Game 탭 > 게임존 메인] 진입
  2. [GP 교환소] 버튼 탭
  3. GP 교환소 화면 진입 (기본: GP 가져오기 탭)
  4. "가능: 2,500 CELB" 확인
  5. CELB 입력란에 "1,000" 입력
  6. 자동 계산: "가져올 GamePoint: 1,000 GP"
  7. [GamePoint 가져오기] 버튼 활성화
  8. 버튼 탭 → 본인 인증 모달 (생체 인증 선택)
  9. 인증 성공 → 확인 모달 표시
  10. [확인] 탭 → 교환 요청
  11. Toast "GP 충전이 완료되었습니다."
  12. GP 잔액: 2,914 → 3,914 GP (자동 갱신)

시나리오 2: GP 출금 (GP → CELB)

  1. GP 교환소 화면에서 [CELB으로 보내기] 탭 선택
  2. "보유: 2,914 GP" 확인
  3. GP 입력란에 "500" 입력
  4. 자동 계산: "받을 Celeb: 500 CELB"
  5. 지갑 주소 입력란에 "0x7a3b...8a9b" 입력
  6. [CELB으로 보내기] 버튼 활성화
  7. 버튼 탭 → 본인 인증 모달 (비밀번호 선택)
  8. 인증 성공 → 확인 모달 표시
  9. [확인] 탭 → 교환 요청
  10. Toast "GP 출금이 완료되었습니다."
  11. GP 잔액: 2,914 → 2,414 GP (자동 갱신)

시나리오 3: 최소 금액 미달 에러

  1. GP 가져오기 탭에서 CELB 입력란에 "50" 입력
  2. 입력란 하단에 빨간색 에러 메시지: "최소 충전 금액은 100 CELB입니다."
  3. [GamePoint 가져오기] 버튼 비활성 상태 유지
  4. "150" 입력으로 수정
  5. 에러 메시지 사라지고 버튼 활성화

시나리오 4: 잔액 초과 에러

  1. CELB으로 보내기 탭에서 GP 입력란에 "5,000" 입력
  2. 보유 GP: 2,914 GP (부족)
  3. 입력란 하단에 빨간색 에러 메시지: "보유 GP가 부족합니다. (보유: 2,914 GP)"
  4. [CELB으로 보내기] 버튼 비활성 상태 유지

시나리오 5: 1일 한도 초과

  1. 사용자가 오늘 이미 80,000 CELB 충전함 (1일 한도: 100,000 CELB)
  2. 추가로 30,000 CELB 충전 시도
  3. 입력 시 에러 메시지: "1일 충전 한도를 초과했습니다. (80,000 / 100,000)"
  4. 최대 입력 가능 금액: 20,000 CELB

시나리오 6: 교환 기능 비활성화

  1. 백오피스에서 "GP 충전 활성화" OFF 설정
  2. 앱에서 GP 교환소 진입 시도
  3. Toast "현재 GP 충전 기능이 점검 중입니다. 잠시 후 다시 시도해주세요."
  4. GP 가져오기 탭 비활성화 (CELB으로 보내기 탭은 정상 노출)

시나리오 7: CELB 입금주소 확인

  1. GP 가져오기 탭에서 [CELB 입금주소] 버튼 탭
  2. CEB-GAM-302 화면으로 이동

비즈니스 규칙

  1. 교환 비율은 백오피스 설정값을 따릅니다
  2. 1일 한도는 금액 한도와 횟수 한도 모두 충족해야 합니다
  3. GP 충전 시 트랜잭션 완료 후 GP가 즉시 반영됩니다
  4. GP 출금 시 GP는 즉시 차감되고 CELB는 비동기로 전송됩니다
  5. 교환 실패 시 GP는 원복됩니다
  6. 교환 내역은 CEB-GAM-501 (GP 히스토리)에 실시간 기록됩니다
  7. 본인 인증은 교환 실행 시점에 필수입니다
  8. CELB 잔액은 새로고침 버튼으로 수동 갱신 가능합니다
