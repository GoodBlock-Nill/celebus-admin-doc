3. 화면 내 기능설명

3.1 게임 유형 선택

기능 동작
Survival Trivia 카드 탭 Phase 2 준비중: 비활성화, 활성화 시: Survival Trivia 화면 이동
Prediction Market 카드 탭 Prediction Market 리스트 화면 (CEB-GAM-201) 이동

3.2 바로가기 기능

클릭 요소 이동 화면 비고
GP 잔액 배지 GP 히스토리 (CEB-GAM-501) Header 영역
프로필 아이콘 My 페이지 Header 영역
알림 아이콘 알림 페이지 Header 영역
유저 랭킹 카드 랭킹 화면 (CEB-GAM-401) -
GP 히스토리 카드 GP 히스토리 화면 (CEB-GAM-501) -

3.3 실시간 데이터 표시

데이터 갱신 방식 표시 위치
GP 잔액 화면 진입 시 조회, Pull-to-Refresh 지원 Header GP 잔액 배지
진행중 게임 수 화면 진입 시 조회, Pull-to-Refresh 지원 Prediction Market 카드
Survival Trivia 일정 화면 진입 시 조회, LIVE 상태는 실시간 폴링 Survival Trivia 카드

4. 규칙/로직

4.1 데이터 조회 기준

항목 기준
GP 잔액 사용자의 현재 GP 잔액 (실시간 조회)
진행중 게임 수 상태가 Active인 Prediction Market 게임 수
Survival Trivia 일정 오늘 예정된 다음 게임 시작 시간 (KST, UTC+9)
Survival Trivia LIVE 상태 현재 진행중인 게임 존재 시 LIVE 배지 + 진행중 게임 수 표시

4.2 게임 유형 카드 노출 조건

게임 유형 Phase 카드 노출 클릭 가능 표시 상태
Survival Trivia Phase 2 (준비중) 노출 불가 "준비중" 배지 + 회색 처리
Survival Trivia Phase 2 (활성화) 노출 가능 일정 또는 LIVE 상태 표시
Prediction Market Phase 1 노출 가능 진행중 게임 수 표시

4.3 Survival Trivia 카드 상태별 표시

상태 표시 내용 스타일
준비중 (Phase 2) "준비중" 배지 + "Phase 2에서 제공 예정입니다" 회색 처리, 클릭 불가
다음 게임 예정 "오늘 12:00 (UTC+9)" 일반 스타일
진행중 "[LIVE] 1개 퀴즈 진행중" LIVE 배지 빨간색, 애니메이션 효과
오늘 일정 없음 "다음 게임을 기다려주세요" 회색 텍스트

4.4 숫자 포맷

항목 포맷 예시
GP 잔액 천 단위 콤마 + " GP" "1,234 GP"
진행중 게임 수 "{숫자}개 예측 진행중" "3개 예측 진행중"
퀴즈 진행중 "{숫자}개 퀴즈 진행중" "1개 퀴즈 진행중"

4.5 상태별 표시

상태 조건 표시
Loading 데이터 로딩 중 카드별 Skeleton UI
Default 데이터 정상 조회 조회된 값 표시
Empty 진행중 게임 없음 "진행중인 게임이 없습니다" (Prediction Market 카드)
Error 조회 실패 토스트: "데이터를 불러오는데 실패했습니다." + 재시도 버튼

4.6 Pull-to-Refresh

동작 결과
화면 상단에서 아래로 드래그 모든 데이터 재조회 (GP 잔액, 진행중 게임 수, Survival Trivia 일정)
새로고침 완료 토스트 없이 조용하게 데이터 갱신

5. 에러 및 예외상황

상황 처리
네트워크 에러 토스트: "네트워크 상태를 확인해주세요." + 재시도 버튼
GP 잔액 조회 실패 Header GP 잔액 배지에 "-" 표시
게임 데이터 조회 실패 해당 카드에 "조회 실패" 메시지 + 재시도 링크
Survival Trivia 준비중 카드 클릭 토스트: "준비중입니다. Phase 2에서 만나요!"
진행중 게임 없을 때 카드 클릭 Prediction Market: 리스트 화면 이동 (빈 상태 표시)
