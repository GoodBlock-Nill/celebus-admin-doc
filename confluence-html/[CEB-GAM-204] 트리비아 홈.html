1. 화면 목적

Survival Trivia 게임의 진입점 화면입니다. 사용자는 게임 정보(일정, 상금, 규칙)를 확인하고, GP를 지불하여 라이브 게임에 입장할 수 있습니다. 입장 가능 시간(시작 10분 전 ~ 시작 시점)에 따라 버튼 상태가 변경됩니다.

3. 화면 내 기능설명

뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
동작
  1. 게임존 메인(CEB-GAM-101)으로 이동
  2. 네비게이션 스택에서 현재 화면 제거

GP 잔액 표시

표시 규칙
  • Header 우측에 현재 GP 잔액 표시
  • 형식: "{숫자} GP" (천 단위 콤마)
  • 실시간 갱신
동작
  • GP 잔액 영역 탭 시 GP 히스토리(CEB-GAM-501)로 이동

입장하기 버튼

트리거
  • 하단 "{N}GP로 입장하기" 버튼 탭
사전 조건
  • 게임 상태: 시작 10분 전 ~ 시작 시점
  • GP 잔액 ≥ 참여 비용 (서버에서 탭 시 검증)
  • 정원 미달 (최대 500명)
  • 해당 게임에 이미 입장한 이력이 없을 것 (중복 입장 불가)
동작
  1. 버튼 로딩 상태로 전환 (중복 탭 방지)
  2. 서버에 입장 요청 (GP 차감 포함)
  3. GP 차감 처리 (첫 참여인 경우 하트 1개 동시 지급)
  4. 성공 시: CEB-GAM-205(Pre-start) 화면으로 전환
  5. 실패 시: 에러 처리 (하단 에러 섹션 참조)
트랜잭션 보증 GP 차감은 서버에서 처리. 최초 참여 시에는 하트 1개 지급이 동일 트랜잭션에 포함 (원자성 보장)
다중 탭/기기 검증
항목 설명
다중 탭 감지 입장 버튼 탭 시 BroadcastChannel로 다른 탭의 게임 진행 여부 확인. 진행 중 탭 있으면 입장 차단 + "이미 다른 탭에서 게임이 진행 중입니다" 토스트
BC 타임아웃 BroadcastChannel 메시지 전송 후 500ms 내 응답 없으면 "다른 탭 없음"으로 판단하여 입장 진행. 서버 검증이 최종 게이트
BC 미지원 환경 BroadcastChannel API 미지원 시 → BC 검증 건너뛰고 서버 검증만 진행
서버 검증 입장 API 호출 시 서버에서 동일 유저 활성 세션 확인. 있으면 입장 거부 + "이미 다른 기기에서 게임이 진행 중입니다" 에러 토스트
로딩 타임아웃
항목 설명
로딩 타임아웃 입장 버튼 탭 후 10초 내 서버 응답 없으면 버튼 자동 복구 + "일시적 오류가 발생했습니다" 에러 토스트
재시도 타임아웃 후 사용자가 직접 버튼 재탭으로 수동 재시도. 자동 재시도 없음. GP는 서버 응답 없었으므로 차감되지 않은 상태
GP 부족 시
  • 보유 GP < 참여 비용인 경우
  • CEB-GAM-202-MD-NOBALANCE 모달 표시
  • 모달 내 안내 문구: "입장하려면 {N}GP가 필요해요."

카운트다운 표시

표시 규칙
항목 설명
시간 기준 서버 시간 기준. 화면 진입 시 서버 시간과 동기화하여 클라이언트-서버 오프셋 보정
  • 시작까지 남은 시간에 따라 툴팁 메시지 실시간 갱신
  • 10분 이상: "10분 전 부터 입장할 수 있어요"
  • 10분 ~ 1분: "{N}분 뒤 게임이 시작돼요 시작 후에는 입장할 수 없어요"
  • 1분 미만: "곧 게임이 시작돼요 시작 후에는 입장할 수 없어요"
  • 시작 후: "이미 시작된 게임입니다"

상금 분배 설명

표시 규칙
  • 입장 불가 상태: "생존자 수에 따라 분배돼요"
  • 입장 가능 상태 (참여자 수 확정 단계): "생존자 수에 따라 n분의 1로 분배"

미확인 결과 자동 전환

감지 방식
항목 설명
감지 방식 204 화면 진입/복귀(visibilitychange) 시 서버 API로 미확인 결과 조회
조회 시점 화면 초기 렌더링 완료 후 (useEffect/onMounted 시점). UI 골격 먼저 표시 → API 응답 수신 → 미확인 결과 있으면 RESULT(D)로 전환
전환 UX 로딩 스피너 없이 자연스러운 페이지 전환 (SPA 라우팅). 미확인 결과 조회 동안 기존 204 화면 정상 표시
전환 동작 미확인 결과 있으면 CEB-GAM-206-RESULT(D타입)으로 자동 전환
체류 중 동작 204 체류 중 갑작스러운 전환 없음. 게임이 종료되더라도 화면 전환하지 않음. 다른 탭/앱 갔다가 204로 복귀(visibilitychange) 시에만 미확인 결과 재조회 → 있으면 RESULT(D)로 전환
트리거 (재방문)
  • 사용자가 204 화면에 진입 시 미확인 게임 결과가 존재
동작
  1. 화면 로딩 완료 후 CEB-GAM-206-RESULT(D타입)으로 자동 전환
대상
  • 탈락 모달에서 "나가기"를 선택하여 게임을 떠난 사용자만 해당
  • 완주자, 관전 완료자는 이미 RESULT 화면을 본 상태이므로 대상 아님

4. 규칙/로직

입장 가능 시간

구간 설명 버튼 상태
시작 10분 전 이전 사전 정보 확인 구간 비활성화
시작 10분 전 ~ 시작 시점 입장 가능 구간 활성화
시작 후 입장 마감 비활성화

서버 시간(KST) 기준으로 판단하며, 클라이언트는 서버 시간을 동기화하여 표시합니다.

정원 제한

  • 동시 접속자 최대 500명 (운영 정책에 따라 변동 가능)
  • 정원 초과 시 입장 불가
  • 서버에서 정원 검증 (클라이언트는 즉시 차단하지 않음)

중복 입장 차단

  • 이미 입장한 게임에 재입장 불가 (서버에서 검증)
  • 동일 게임에 대한 중복 GP 차감 방지

5. 에러 및 예외상황

데이터 로딩 실패

상황
  • 게임 정보 API 호출 실패 또는 네트워크 오류
처리
  1. Error 상태 표시
  2. 에러 메시지: "게임 정보를 불러오는 데 실패했습니다."
  3. [다시 시도] 버튼 제공

GP 잔액 부족

상황
  • 보유 GP < 참여 비용 상태에서 입장하기 버튼 탭
처리
  1. CEB-GAM-202-MD-NOBALANCE 모달 표시
  2. 모달 내 안내: "입장하려면 {N}GP가 필요해요."

정원 초과

상황
  • 입장 요청 시 서버에서 정원 초과 응답
처리
  1. 에러 토스트: "정원이 마감되었습니다."
  2. 버튼 비활성화 상태로 전환
  3. 버튼 텍스트: "입장 마감"

시작 후 입장 시도

상황
  • 입장 가능 상태에서 시작 시점이 지난 후 버튼 탭 (클라이언트/서버 시간 차이)
처리
  1. 에러 토스트: "이미 시작된 게임입니다."
  2. 화면 상태 갱신 (입장 불가 상태로 전환)

네트워크 연결 끊김

상황
  • 오프라인 상태에서 화면 진입 또는 입장 시도
처리
  1. Error 상태 표시
  2. 에러 메시지: "네트워크 연결을 확인해주세요."
  3. [다시 시도] 버튼 제공

입장 중 서버 오류

상황
  • 입장 요청 후 서버 오류 응답
처리
  1. 버튼 원래 상태로 복구 (로딩 해제)
  2. 에러 토스트: "입장에 실패했습니다. 다시 시도해주세요."
  3. GP 차감 없음 (서버에서 트랜잭션 롤백)

중복 입장 시도

상황

처리
  1. 에러 토스트: "이미 입장한 게임입니다."
  2. 버튼 상태 변경 없음 (입장 완료 상태로 전환)
