1. 화면 목적

Prediction Market 유형 게임들의 리스트를 조회하는 화면입니다. 사용자는 상태별 필터 탭을 통해 진행중/결과 발표/발표대기중 게임을 구분하여 볼 수 있으며, 게임 카드를 탭하여 상세 화면으로 이동할 수 있습니다.

3. 화면 내 기능설명

뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
동작
  1. 게임존 메인(CEB-GAM-101) 화면으로 이동
  2. 네비게이션 스택에서 현재 화면 제거

GP 잔액 표시

표시 규칙
  • 실시간 GP 잔액 표시
  • 형식: "{숫자} GP" (예: "300 GP")
  • 천 단위 콤마 적용
  • 1초마다 자동 갱신
동작
  • 탭 시 GP 변동 내역 화면으로 이동 (추후 구현)

필터 탭 전환

트리거
  • 필터 탭 버튼 탭
동작
  1. 선택된 탭 하이라이트 표시 (밑줄 또는 색상 변경)
  2. 해당 상태의 게임만 필터링하여 표시
  3. 리스트 최상단으로 스크롤 이동
  4. Loading 상태 표시 후 데이터 로딩
필터 탭별 표시 규칙
  • 전체: 모든 상태(Active, Pending, Closed, Ended)의 게임 표시
  • 진행중: 상태가 Active인 게임만 표시
  • 결과 발표(N): 상태가 Closed 또는 Ended인 게임 표시
    • N = 사용자가 참여하고 아직 확인하지 않은 새로운 결과 발표 건수
    • N = 0 이면 배지 숫자 표시 안함
  • 발표대기중: 상태가 Pending인 게임만 표시

게임 카드 탭

트리거
  • 게임 카드 영역 탭
동작
  1. 해당 게임의 ID를 파라미터로 전달
  2. Prediction Market 상세(CEB-GAM-202) 화면으로 이동
  3. 네비게이션 스택에 현재 화면 추가 (뒤로가기 가능)

[투표 참여하기] 버튼

트리거
  • 진행중이고 미참여 게임 카드의 [투표 참여하기] 버튼 탭
동작
  1. 게임 상세 화면(CEB-GAM-202)으로 이동
  2. 자동으로 투표 참여 영역으로 스크롤 이동
활성화 조건
  • 게임 상태: Active
  • 참여 상태: 미참여
  • 인원 마감: 정원 미달 또는 무제한

[결과 확인하기] 버튼

트리거
  • 결과 발표 상태 게임 카드의 [결과 확인하기] 버튼 탭
동작
  1. 게임 상세 화면(CEB-GAM-202)으로 이동
  2. 자동으로 결과 영역으로 스크롤 이동
  3. 새로운 결과인 경우 배지 카운트 감소 (N-1)

스크롤 및 새로고침

스크롤
  • 세로 스크롤로 게임 카드 리스트 탐색
  • 무한 스크롤 또는 페이지네이션 (구현 방식에 따라 결정)
Pull-to-Refresh
  • 리스트 최상단에서 아래로 드래그 시 새로고침
  • Loading 인디케이터 표시
  • 데이터 갱신 후 첫 페이지로 이동

4. 규칙/로직

정렬 기준

기본 정렬
  • 우선순위 1: 게임 상태
    • Active (진행중) > Pending (발표대기중) > Closed (결과 확정) > Ended (종료)
  • 우선순위 2: 상태별 세부 정렬
    • Active: 투표 종료일시 오름차순 (곧 종료되는 게임 먼저)
    • Pending: 결과 발표 예정일 오름차순 (곧 발표되는 게임 먼저)
    • Closed: 결과 확정일시 내림차순 (최근 확정된 게임 먼저)
    • Ended: 종료일시 내림차순 (최근 종료된 게임 먼저)
필터 탭별 정렬
  • 전체: 기본 정렬 적용
  • 진행중: 투표 종료일시 오름차순
  • 결과 발표: 결과 확정일시 내림차순
  • 발표대기중: 결과 발표 예정일 오름차순

게임 상태 정의

상태 코드 표시 조건 앱 표시
진행중 Active 투표 시작일시 ≤ 현재 시간 ≤ 투표 종료일시 [진행중] Badge
발표대기중 Pending 투표 종료일시 < 현재 시간 AND 결과 미확정 [발표대기중] Badge
결과 확정 Closed 결과 확정됨 AND 보상 지급 전 [결과발표] Badge
종료 Ended 보상 지급 완료 [결과발표] Badge

YES/NO 비율 계산

계산 방식
  • YES 비율 = (YES 선택 참여자 수 + YES 부스팅 가중치) / 전체 가중치 × 100
  • NO 비율 = (NO 선택 참여자 수 + NO 부스팅 가중치) / 전체 가중치 × 100
  • 전체 가중치 = (YES 참여자 수 + YES 부스팅 가중치) + (NO 참여자 수 + NO 부스팅 가중치)
부스팅 가중치
  • 부스팅 사용 시 부스팅 배수만큼 가중치 증가
  • 예: 부스팅 2배 사용 시, 해당 참여자의 가중치 = 1 + 2 = 3
표시 규칙
  • 소수점 첫째 자리까지 표시 (예: 45.2%)
  • 비율 바는 좌측 YES, 우측 NO
  • 비율에 따라 바 너비 조정
  • 참여자의 선택에 따라 강조 표시 (볼드 또는 색상 변경)

종료 카운트다운 계산

계산 방식
  • 남은 시간 = 투표 종료일시 - 현재 시간
  • 형식: "종료까지 X일 XX:XX:XX" (일, 시, 분, 초)
  • 1초마다 자동 갱신
표시 규칙
  • 1일 이상: "종료까지 X일 XX:XX:XX"
  • 1일 미만 24시간 이상: "종료까지 23:XX:XX"
  • 1시간 미만: "종료까지 00:XX:XX" (빨간색 강조)
  • 1분 미만: "종료까지 00:00:XX" (빨간색 강조)
  • 종료 시: "종료됨" 표시 후 상태 자동 전환 (Active → Pending)

참여 현황 표시

정원 제한 있을 때
  • 형식: "참여 {현재참여자수}/{정원}명"
  • 예: "참여 320/500명"
  • 인원 마감 시: "선착순 인원 마감" (빨간색 또는 강조 표시)
  • 마감 기준: 현재참여자수 ≥ 정원
정원 무제한일 때
  • 형식: "참여 {현재참여자수}명"
  • 예: "참여 1,234명"

새로운 결과 발표 건수 (N) 계산

계산 방식
  • 사용자가 참여한 게임 중 결과가 확정되었으나 사용자가 아직 결과 상세를 확인하지 않은 게임 수
  • 서버에서 "결과 확인 여부" 플래그 관리
표시 규칙
  • N > 0: "결과 발표(N)" 배지 표시
  • N = 0: "결과 발표" (배지 숫자 없음)
  • 사용자가 결과 상세 화면(CEB-GAM-202)에서 결과를 확인하면 N-1

테마 지원

라이트/다크 테마
  • 시스템 설정 또는 앱 내 설정에 따라 테마 자동 전환
  • 배경색, 텍스트 색상, 카드 색상 등 테마별 색상 적용
  • Badge, 버튼, 비율 바 등 모든 UI 요소에 테마 적용

5. 에러 및 예외상황

데이터 로딩 실패

상황
  • API 호출 실패 또는 네트워크 오류
처리
  1. Error 상태 표시
  2. 에러 메시지: "게임 목록을 불러오는 데 실패했습니다."
  3. [다시 시도] 버튼 제공
  4. 버튼 탭 시 데이터 재로딩

Empty 상태

상황
  • 게임이 없거나 필터 조건에 맞는 게임이 없는 경우
처리
  1. 리스트 대신 Empty 상태 메시지 표시
  2. 필터 탭별 메시지:
    • 전체: "현재 진행중인 게임이 없습니다."
    • 진행중: "현재 진행중인 게임이 없습니다."
    • 결과 발표: "결과가 발표된 게임이 없습니다."
    • 발표대기중: "발표를 기다리는 게임이 없습니다."

Loading 상태

상황
  • 데이터 로딩 중
처리
  1. Skeleton UI 표시 (게임 카드 형태의 스켈레톤 3개)
  2. 필터 탭 비활성화
  3. 로딩 완료 시 실제 데이터로 전환 (페이드인 애니메이션)

참여 불가 상태

상황
  • 인원 마감된 게임 카드 탭
처리
  1. 게임 상세 화면으로 이동
  2. 참여 버튼 비활성화 및 "인원 마감" 안내 표시

종료된 게임 참여 시도

상황
  • 카운트다운 종료 후 실시간으로 상태가 전환되기 전에 [투표 참여하기] 버튼 탭
처리
  1. 에러 토스트: "투표가 종료된 게임입니다."
  2. 리스트 자동 갱신
  3. 해당 게임 카드 상태 업데이트 (Active → Pending)

GP 잔액 부족

상황
  • GP 잔액이 참여 비용보다 적은 상태에서 [투표 참여하기] 버튼 탭
처리
  1. 게임 상세 화면으로 이동
  2. 참여 시도 시 GP 잔액 부족 모달 표시 (CEB-GAM-202-MD-NOBALANCE)

네트워크 연결 끊김

상황
  • 오프라인 상태에서 화면 진입 또는 데이터 로딩 시도
처리
  1. Error 상태 표시
  2. 에러 메시지: "네트워크 연결을 확인해주세요."
  3. [다시 시도] 버튼 제공
  4. 온라인 복구 시 자동 재로딩
