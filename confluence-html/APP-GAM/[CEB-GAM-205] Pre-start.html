1. 화면 목적

Survival Trivia 라이브 게임 시작 전 대기 화면입니다. 게임 규칙을 최종 안내하고, 서버 시간 기준 정시가 되면 5초 카운트다운 후 게임 진행 화면으로 자동 전환됩니다. CEB-GAM-204에서 입장 성공 후 이 화면으로 진입하며, 카운트다운 완료 시 CEB-GAM-206으로 자동 전환됩니다.

3. 화면 내 기능설명

뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
  • 브라우저 뒤로가기 버튼 및 제스처 (Android HW 백 버튼, popstate 이벤트)도 동일하게 CEB-GAM-206-MD-LEAVE 모달 표시
사전 조건
  • 카운트다운 진행 중이 아닌 경우에만 활성화
동작
  1. CEB-GAM-206-MD-LEAVE 모달 표시 (dim 영역 탭/백 제스처로 닫기 가능, 취소 처리)
  2. [취소] 선택 시: 모달 닫힘, 대기 화면 유지
  3. [나가기] 선택 시: CEB-GAM-204로 이동, GP 환불 없음

iOS Safari에서 스와이프 백 제스처는 완전 차단 불가 → 세션 복구(페이지 새로고침 처리 참조)로 대응

LIVE 타이머

표시 규칙
  • 게임 시작까지 남은 시간 카운트다운 (HH:MM:SS)
  • 서버 기준 시작 시각을 참조하여 클라이언트에서 남은 시간 계산
  • 클라이언트는 서버 시간과의 오프셋을 보정하여 정확도 유지
항목 설명
시간 기준 서버 시간 기준. 화면 진입 시 서버 시간과 동기화하여 클라이언트-서버 오프셋 보정
탭 복귀 시 visibilitychange 이벤트 감지 → 서버 시간 재조회 → 남은 시간 재계산. 게임 시작되었으면 즉시 CEB-GAM-206으로 전환
10초 이하 강조 타이머가 10초 이하일 때 타이머 텍스트 강조 애니메이션 적용 (피그마 참조)

하트 표시

표시 규칙
  • 현재 계정에 보유 중인 하트 수를 ❤️ 아이콘으로 표시
  • 하트는 계정 레벨에서 영구 보유 (게임별 초기화 없음)
  • 최소 0개 ~ 최대 3개
  • 상태 바 좌측에 배치

참여자 수

표시 규칙
  • 현재 접속 중인 참여자 수 실시간 표시
  • 형식: 👥 {숫자} (천 단위 콤마)
  • 상태 바 우측에 배치
  • 1초 간격으로 업데이트, 숫자 변경 시 fade transition 적용

카운트다운 전환

트리거
  • 서버 시간 기준 게임 시작 정시 도달
동작
  1. 대기 화면 → 카운트다운 화면으로 자동 전환 (카운트다운 레이어 오버레이 ease in, 150ms)
  2. 5→4→3→2→1 숫자 애니메이션 (각 1초, 파란색 대형 숫자)
  3. 카운트다운 중 [<] 뒤로가기 버튼 비활성화: 회색 처리, 불투명도 50%, 탭 무반응. 브라우저 뒤로가기(popstate)도 무시
  4. 카운트다운 완료(1 이후) 시: CEB-GAM-206(트리비아 상세)으로 자동 전환

구현 가이드 (개발자 참고): popstate 이벤트 발생 시 history.pushState()로 현재 entry를 재생성하여 뒤로가기 무효화. iOS Safari 스와이프 백 제스처는 완전 차단 불가 → 세션 복구로 대응 (페이지 새로고침 처리 참조). Android HW 백 버튼은 popstate와 동일 처리. 카운트다운은 5초로 짧아 사용자 불편 최소.

입장 후 GP 처리

  • LIVE 타이머 및 카운트다운 전환 모두 서버 시간 기준으로 동작

5. 에러 및 예외상황

재연결중 공통 UI

아래 모든 재연결 케이스에서 동일한 재연결중 UI가 표시됩니다:

  • "연결중..." 토스트(상단) + 화면 중앙 로딩 스피너
  • [재시도] 버튼 제공 (사용자가 수동으로 재연결 시도 가능)
  • 대기 중에는 [<] 뒤로가기 가능 (재연결 취소 + GP 환불 후 CEB-GAM-204 이동)

해당 케이스: 최초 WS 연결 실패 / 대기 중 네트워크 끊김 / 카운트다운 중 네트워크 끊김 / visibilitychange 후 WS 재연결 중 / 새로고침 후 서버 상태 조회 중

라이브 서버 연결 실패 / 대기 중 네트워크 끊김

상황
  • 최초 WebSocket 연결 실패 또는 대기 화면에서 네트워크 연결 끊김
처리
  1. 재연결중 공통 UI 표시
  2. 자동 재연결 시도 (최대 3회, 3초 간격)
  3. 재연결 성공 시: 대기 화면 정상 복귀
  4. 3회 실패 시: 에러 모달 표시 후 GP 자동 환불 (서버 처리) → CEB-GAM-204로 이동

에러 모달 문구: "연결이 불안정하여 게임에 참여할 수 없습니다. 참여에 사용된 GP가 환불됩니다." [확인]

카운트다운 중 네트워크 끊김

상황
  • 5초 카운트다운 진행 중 네트워크 연결 끊김
처리
  1. 클라이언트 카운트다운은 UX 목적으로 계속 표시
  2. 카운트다운 완료 후 서버 자동 재연결 시도 (최대 3회, 3초 간격)
  3. 재연결 성공 시 즉시 서버 게임 상태 조회:
서버 상태 처리
아직 Stage 1 진행 중 (놓친 스테이지 없음) CEB-GAM-206 합류 (정상 복귀)
놓친 스테이지 존재 (이미 탈락 처리) CEB-GAM-205-MD-RECONNECT 모달 표시 (나가기/관전 선택)
게임 종료 CEB-GAM-206-RESULT 이동

놓친 스테이지가 있으면 네트워크 끊김으로 인한 별도 탈락 처리가 완료된 상태입니다. 합류가 아닌 탈락 안내 후 관전/나가기 선택을 제공합니다.

  1. 3회 실패 시: 탈락 처리 안내 모달 표시 → CEB-GAM-204 이동 (GP 환불 없음)

에러 모달 문구: "게임 연결이 끊어졌어요. 재연결에 실패하여 이번 게임은 탈락 처리되었습니다."

WebSocket 재연결 UI "연결 중..." 토스트(상단) + 화면 중앙 로딩 스피너 표시. 대기 중에도 [<] 뒤로가기 가능 (재연결 취소 + GP 환불 후 CEB-GAM-204 이동)

재연결 중 뒤로가기 상세 처리

항목 설명
재연결 중 뒤로가기 동작 [<] 탭 시 CEB-GAM-206-MD-LEAVE 모달 표시 (기존 뒤로가기 플로우와 동일)
나가기 선택 시 (1) 진행 중인 WebSocket 재연결 시도 즉시 중단 (2) 서버에 이탈 요청 전송 (실패 시 서버에서 타임아웃으로 처리) (3) GP 환불은 서버 자동 처리 (별도 환불 API 호출 불필요) (4) CEB-GAM-204로 이동

6. 연관화면ID

화면ID 화면명 관계
CEB-GAM-204 트리비아 홈 뒤로가기(나가기 선택) 시 이동
CEB-GAM-205-MD-RECONNECT 재연결 안내 카운트다운 중 네트워크 끊김 → 재연결 시 놓친 스테이지 존재 시 모달 표시
CEB-GAM-206 트리비아 상세 카운트다운 완료 시 자동 전환
CEB-GAM-206-MD-LEAVE 나가기 확인 대기 중 뒤로가기 탭 시 모달 표시
CEB-GAM-206-RESULT 게임 결과 재연결 시 게임 종료된 경우 이동
