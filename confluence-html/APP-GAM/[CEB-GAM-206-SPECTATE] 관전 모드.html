1. 화면 목적

탈락 후 "관전하고 하트 받기"를 선택한 사용자가 다른 참여자의 게임 진행을 실시간으로 관전하는 화면입니다. 문제와 선택지를 볼 수 있지만 직접 응답할 수는 없습니다. 채팅은 계속 참여 가능합니다. 게임 종료 시까지 관전을 완료하면 하트 1개를 보상으로 받습니다.

이 문서는 탈락 후 관전 모드 진행 화면을 다룹니다. 게임 진행(CEB-GAM-206), 탈락 모달(CEB-GAM-206-MD-ELIMINATE), 나가기 모달(CEB-GAM-206-MD-LEAVE), 게임 결과(CEB-GAM-206-RESULT)는 별도 문서입니다.

3. 화면 내 기능설명

3.1 문제/선택지 관전

동작
  • 문제 텍스트, 선택지 4개가 실시간으로 표시됨
  • 선택지는 비활성화 상태 (회색 처리, 탭 불가)
  • 타이머 바도 동일하게 작동 (시각적으로 감소)
규칙
  • 관전자는 선택지를 탭해도 응답이 전송되지 않음. 선택지는 시각적으로 비활성화 (회색 처리, 탭 피드백 없음 (pointer-events: none 적용. 리플 효과, 하이라이트, 애니메이션 모두 제거))
  • 타이머 바는 표시만 하며 탈락 이벤트와 무관

3.2 정답 공개 관전

트리거
  • 서버에서 정답 + 선택지별 선택 비율(%) 수신
동작
  1. 정답 선택지: 파란 테두리 + ⭕ 마크
  2. 모든 선택지에 선택 비율(%) 표시
  3. "내 선택" 배지 없음 (관전자이므로 응답 내역 없음)
  4. 공개 후 5초 대기 → 자동으로 다음 스테이지 전환

3.3 채팅 기능

메시지 표시
  • 화면 하단 채팅 영역에 실시간 메시지 스트림 표시
  • 닉네임 + 메시지 형태로 좌측 정렬
  • 새 메시지 수신 시 자동 하단 스크롤
메시지 입력/전송
  • 💬 버튼 탭 → 키보드 오픈 + 입력 필드 활성화
  • ▶ 전송 버튼 탭 → 메시지 서버 전송
제한 규칙
규칙 내용
욕설 필터링 금칙어 목록 기반 자동 마스킹 처리 ("***")
발송 제한 초당 1회 (rate limit), 초과 시 발송 무시
문자 수 제한 최대 100자, 초과 입력 불가
키보드 활성화 시
  • 상태 바([관전중] 배지/스테이지/참여자) 숨김
  • 타이머 바와 문제/선택지는 축소 표시 (비활성화 상태 유지)
  • 키보드 닫기: 빈 영역 탭 또는 전송 완료 후

3.4 뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
동작
  1. CEB-GAM-206-MD-LEAVE (나가기 확인) 모달 표시
  2. 모달 안내 문구에 "관전 보상 하트도 받을 수 없습니다" 문구 추가
  3. 나가기 확인 시: CEB-GAM-204(트리비아 홈)로 이동, 하트 미지급

3.5 게임 종료 시 자동 전환

트리거
  • 마지막 스테이지 정답 공개 완료
동작
  1. CEB-GAM-206-RESULT (게임 결과) 화면으로 자동 전환
  2. 관전자 결과 화면: 탈락자 결과 표시 + 하트 1개 지급 안내
항목 설명
마지막 스테이지 RESULT 전환 마지막 스테이지 정답 공개 5초 표시 후 CEB-GAM-206-RESULT로 자동 전환 (다른 스테이지와 동일 패턴). 하트는 정답 공개 이벤트 수신 시점에 서버에서 지급 (화면 전환과 무관)

4. 규칙/로직

관전 모드 진입 조건

  • CEB-GAM-206-MD-ELIMINATE 모달에서 "관전하고 하트 받기" 버튼 선택

하트 보상 규칙

조건 하트 지급 비고
끝까지 관전 완료 +1개 마지막 스테이지 정답 공개 이벤트를 서버로부터 수신한 시점에 자동 지급
중도 이탈 (나가기) 미지급 나가기 모달에서 "나가기" 선택 시
네트워크 끊김 후 미복귀 미지급 재연결 실패 시 이탈로 처리
이미 최대 보유(3개) 미지급 하트 보유 상한(3개) 초과 불가

관전 완료 판정 기준: 마지막 스테이지의 정답 공개 이벤트를 서버로부터 수신한 시점을 기준으로 합니다. 클라이언트 상태가 아닌 서버 이벤트 수신 여부로 판단합니다.

항목 설명
하트 지급 기준 서버가 마지막 스테이지 정답 공개 이벤트 발생 시점에 관전 세션이 활성(WebSocket 연결 중)이면 하트 지급. 세션이 끊어져 있으면 미지급
재연결 후 게임 종료 재연결 성공 시 서버 로그 기준: 마지막 이벤트 시점에 세션이 활성이었으면 지급 완료 상태, 끊겨 있었으면 미지급

관전 재진입 불가

  • CEB-GAM-206-MD-ELIMINATE 모달에서 [나가기]를 선택한 사용자는 해당 게임의 관전 모드에 재진입 불가
  • 한 번 나가기를 선택하면 관전 기회는 영구 소멸
  • 재진입 시도 경로 자체가 존재하지 않음 (CEB-GAM-204에서 해당 게임은 이미 "참여 완료" 상태)

관전 기회는 1회

시나리오 관전 가능 여부
ELIMINATE 모달에서 [관전하고 하트 받기] 선택 관전 모드 진입
ELIMINATE 모달에서 [나가기] 선택 관전 불가, 영구 소멸
ELIMINATE 모달 5초 유예 만료 (미선택) CEB-GAM-204로 자동 이동, 관전 불가, 영구 소멸
관전 중 LEAVE 모달에서 [나가기] 선택 관전 종료, 재진입 불가

5초 유예 만료로 자동 이동된 경우도 [나가기]와 동일하게 취급

5. 에러 및 예외상황

에러 상황 에러 메시지 처리 방법
실시간 스트림 끊김 "연결이 끊어졌습니다." 자동 재연결 시도 (최대 3회), 실패 시 CEB-GAM-204 이동, 하트 미지급
게임 비정상 종료 (서버) "게임이 비정상 종료되었습니다. 참여 GP가 환불됩니다." 에러 안내 + CEB-GAM-204 이동. 하트 일괄 미지급. GP 전액 환불
비정상 종료 조건 - 운영자 강제 취소, 서버 장애 등으로 게임이 정상 진행되지 못한 경우
채팅 전송 실패 - 전송 실패 표시 (빨간색 느낌표), 재전송 없음 (다시 입력)
브라우저 탭 비활성화 / 화면 잠금 - WebSocket 끊김 감지 시 자동 재연결 최대 3회 (3초 간격). 성공 → 서버에서 현재 게임 상태 조회 후 관전 계속. 실패 → CEB-GAM-204 이동 (하트 미지급). 게임 종료 → RESULT(C) (하트는 서버 판단)

페이지 새로고침 처리

항목 처리
새로고침 방지 CSS overscroll-behavior: none으로 pull-to-refresh 차단
beforeunload 관전 중 새로고침/탭 닫기 시 브라우저 기본 경고 표시
세션 복구 페이지 로드 시 서버 API로 활성 세션 확인. 있으면 관전 모드로 복귀 (WebSocket 재연결). 게임 종료 시 RESULT(C)로 이동
