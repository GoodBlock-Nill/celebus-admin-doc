1. 화면 목적

게임 진행 중 또는 관전 모드에서 사용자가 뒤로가기([<]) 버튼을 탭할 때 표시되는 확인 모달입니다. 게임을 나가면 이번 회차의 결과를 포기하게 되며 참여 GP가 환불되지 않음을 안내합니다.

3. 화면 내 기능설명

모달 트리거

트리거 조건 동작
CEB-GAM-205 대기 화면에서 [<] 뒤로가기 탭 본 모달 오픈
CEB-GAM-206 게임 진행 중 [<] 뒤로가기 탭 본 모달 오픈
CEB-GAM-206-SPECTATE 관전 모드에서 [<] 뒤로가기 탭 본 모달 오픈
브라우저 뒤로가기 버튼 및 제스처 (Android HW 백 버튼, popstate 이벤트) 위 3개 화면 모두에서 동일하게 본 모달 표시

버튼 동작

취소 버튼

  1. 본 모달 닫기
  2. 기존 화면(게임 진행/관전)으로 복귀
  3. 상태 변경 없음

나가기 버튼

  1. 본 모달 닫기
  2. 서버에 게임 이탈 요청
  3. CEB-GAM-204 (트리비아 홈)로 즉시 이동
  4. GP 환불 없음
  5. 관전 모드에서 나간 경우: 하트 보상 미지급
  6. 하트 처리: 사용한 하트는 환불 불가

모달 닫기 동작

  • 취소 버튼 클릭 시
  • 모달 외부 영역(딤) 탭 시
  • 뒤로가기 제스처: 모달 닫기 (취소와 동일)

iOS Safari에서 스와이프 백 제스처는 완전 차단 불가 → 세션 복구로 대응

CEB-GAM-206-MD-ELIMINATE 모달과의 차이: LEAVE 모달은 딤 탭으로 닫기 가능 (취소 동작). ELIMINATE 모달은 딤 탭으로 닫히지 않음 (5초 내 강제 선택 필요). LEAVE는 사용자가 자발적으로 열었으므로 쉽게 닫을 수 있어야 하고, ELIMINATE는 시스템이 강제 표시하여 선택이 필요하므로 실수로 닫히면 안 됨.
