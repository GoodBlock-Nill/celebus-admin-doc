1. 화면 목적

Survival Trivia 게임의 진입점 화면입니다. 사용자는 게임 정보(일정, 상금, 규칙)를 확인하고, GP를 지불하여 라이브 게임에 입장할 수 있습니다. 입장 가능 시간(시작 10분 전 ~ 시작 시점)에 따라 버튼 상태가 변경됩니다.

3. 화면 내 기능설명

3.1 뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
동작
  1. 게임존 메인(CEB-GAM-101)으로 이동
  2. 네비게이션 스택에서 현재 화면 제거

3.2 GP 잔액 표시

표시 규칙
  • Header 우측에 현재 GP 잔액 표시
  • 형식: "{숫자} GP" (천 단위 콤마)
  • 실시간 갱신
동작
  • GP 잔액 영역 탭 시 GP 히스토리(CEB-GAM-501)로 이동

3.3 입장하기 버튼

트리거
  • 하단 "{N}GP로 입장하기" 버튼 탭
사전 조건
  • 게임 상태: 시작 10분 전 ~ 시작 시점
  • GP 잔액 ≥ 참여 비용 (서버에서 탭 시 검증)
  • 정원 미달 (최대 모집인원 이내)
  • 해당 게임에 이미 입장한 이력이 없을 것 (중복 입장 불가)
동작
  1. 버튼 → "확인중..." 로딩 상태로 전환 (중복 탭 방지)
  2. 네트워크 품질 확인 (서버 측정)
  3. 네트워크 안정: 자동 입장 처리 진행
  4. 네트워크 불안정: 경고 문구 노출 + "그래도 입장하기" 버튼으로 전환
  5. 네트워크 실패: "다시 시도" 버튼으로 전환
  6. 입장 진행 시: 서버에 입장 요청 (GP 차감 포함)
  7. GP 차감 처리 (첫 참여인 경우 하트 1개 동시 지급)
  8. 성공 시: CEB-GAM-205(Pre-start) 화면으로 전환
  9. 실패 시: 에러 처리 (하단 에러 섹션 참조)
트랜잭션 보증 GP 차감은 서버에서 처리. 최초 참여 시에는 하트 1개 지급이 동일 트랜잭션에 포함 (원자성 보장)
네트워크 확인 UX
단계 버튼/상태 동작
확인중 "확인중..." (로딩 스피너, 비활성화) 네트워크 품질 측정 중. "네트워크 연결 확인 중..." 안내 문구 표시
안정 자동 진행 별도 확인 없이 입장 처리 즉시 진행
불안정 "그래도 입장하기" (활성화, 파란색) "네트워크 상태가 불안정해요. 게임 중 연결이 끊길 수 있어요." 경고 표시. 유저가 직접 입장 여부 선택
실패 "다시 시도" (활성화, 파란색) "네트워크 연결을 확인해주세요." 안내 표시. 탭 시 네트워크 재확인
다중 탭/기기 검증
항목 설명
다중 탭 감지 입장 버튼 탭 시 BroadcastChannel로 다른 탭의 게임 진행 여부 확인. 진행 중 탭 있으면 입장 차단 + "이미 다른 탭에서 게임이 진행 중입니다" 토스트
BC 타임아웃 BroadcastChannel 메시지 전송 후 500ms 내 응답 없으면 "다른 탭 없음"으로 판단하여 입장 진행. 서버 검증이 최종 게이트
BC 미지원 환경 BroadcastChannel API 미지원 시 → BC 검증 건너뛰고 서버 검증만 진행
서버 검증 입장 API 호출 시 서버에서 동일 유저 활성 세션 확인. 있으면 입장 거부 + "이미 다른 기기에서 게임이 진행 중입니다" 에러 토스트
로딩 타임아웃
항목 설명
로딩 타임아웃 입장 버튼 탭 후 10초 내 서버 응답 없으면 버튼 자동 복구 + "일시적 오류가 발생했습니다" 에러 토스트
재시도 타임아웃 후 사용자가 직접 버튼 재탭으로 수동 재시도. 자동 재시도 없음. GP는 서버 응답 없었으므로 차감되지 않은 상태
GP 부족 시
  • 보유 GP < 참여 비용인 경우
  • CEB-GAM-202-MD-NOBALANCE 모달 표시
  • 모달 내 안내 문구: "참가하려면 {N} GP가 필요해요. 현재 보유 GP: {N}"

3.4 친구 초대하기 버튼

트리거
  • 바이럴 메시지 영역의 [친구 초대하기] 버튼 탭
동작
  1. 해당 게임의 공유 링크(딥링크)를 클립보드에 복사
  2. "초대 링크가 복사되었어요" 토스트 표시 (Success 타입)
노출 조건
  • 바이럴 메시지(🔥)와 함께 노출
  • 바이럴 메시지 노출 조건은 서버에서 결정 (예: 일정 모집률 이상 달성 시)

3.5 게임규칙 보기 버튼

트리거
  • 이용 안내 섹션 하단 [게임규칙 보기] 버튼 탭
동작
  • 게임 규칙 바텀시트(bottom sheet) 오픈
바텀시트 내용 (게임규칙)
  • 참여 비용 안내
  • 하트 시스템 상세 설명
  • 탈락 조건 (오답 시 하트 소진, 하트 0개 시 탈락)
  • 마지막 문제 틀리면 탈락 (하트 유무와 상관없이)
  • 탈락 후 관전 방법
  • 응모권 지급 조건
  • 하트 보너스 획득 방법

3.6 카운트다운 표시

표시 규칙
항목 설명
시간 기준 서버 시간 기준. 화면 진입 시 서버 시간과 동기화하여 클라이언트-서버 오프셋 보정
  • 시작까지 남은 시간에 따라 툴팁 메시지 실시간 갱신
  • 10분 이상: "10분 전 부터 입장할 수 있어요"
  • 10분 ~ 1분: "{N}분 뒤 게임이 시작돼요 시작 후에는 입장할 수 없어요"
  • 1분 미만: "곧 게임이 시작돼요 시작 후에는 입장할 수 없어요"
  • 시작 후: "이미 시작된 게임입니다"

3.7 상금 표시 규칙

최대 상금풀
  • 어드민이 설정한 최대 상금풀 값 (고정)
  • 항상 "🏆 최대 상금풀 {N} GP"로 표시
게이지 바
  • 현재 모집인원 / 최대 모집인원 비율로 게이지 채움
  • 첫 진입 시 0 → 현재 값으로 채워지는 글로우 애니메이션 재생
  • 실시간 갱신 (참여자 증가에 따라 게이지 변화)
현재 상금풀
  • 현재 참여자 수 기반으로 백오피스 게임 생성 시 설정한 보상 유형에 따라 실시간 계산
    • 단계별 보상: 모집인원별 상금 단계 테이블 기준 (단계 사이 모집률은 하위 단계 적용)
    • 비율별 보상: 실제 모집인원 / 최대 모집인원 × 최대 상금풀
  • 모든 상태: "현재 상금풀 {N} GP / 생존자 수에 따라 분배" (실시간 갱신)
  • 게임 시작 후: 적용 상금풀 확정 (변동 없음)
참여자 수
  • 게이지 바 하단에 "{현재 모집인원} / {최대 모집인원} 명 참여" 별도 표시
  • 실시간 갱신
참여 비용
  • 자동 계산된 값 표시 (어드민이 수동 입력하지 않음)
  • 공식: floor(최대 상금풀 / 최대 모집인원 × 배수)

4. 규칙/로직

4.1 입장 가능 시간

구간 설명 버튼 상태
시작 10분 전 이전 사전 정보 확인 구간 비활성화
시작 10분 전 ~ 시작 시점 입장 가능 구간 활성화
시작 후 입장 마감 비활성화

서버 시간(UTC+9) 기준으로 판단하며, 클라이언트는 서버 시간을 동기화하여 표시합니다.

4.2 정원 제한

  • 최대 모집인원은 어드민이 게임 생성 시 설정 (상한 없음, 운영 정책에 따라 조정)
  • 정원 초과 시 입장 불가
  • 서버에서 정원 검증 (클라이언트는 즉시 차단하지 않음)

4.3 중복 입장 차단

  • 이미 입장한 게임에 재입장 불가 (서버에서 검증)
  • 동일 게임에 대한 중복 GP 차감 방지
  • 재진입 시 버튼 상태가 "참여 완료" (비활성화)로 표시됨

4.4 게임 규칙 (바텀시트 항목)

  • 오답 시 하트 1개 소진
  • 하트가 0개인 상태에서 오답이면 탈락
  • 마지막 문제를 틀리면 탈락 (하트 유무와 상관없이)
  • 탈락 후 관전 가능, 관전 완료 시 하트 +1개 지급
  • 게임 완주(생존) 성공 시 하트 +1개 지급
  • 하트 최대 3개까지 보유 가능
  • 탈락 시 팬퀘스트 응모권 {N}장 지급
  • 최초 참여 시 하트 1개 자동 지급

4.5 하트 획득 방법

방법 획득 하트
게임 끝까지 관전 완료 +1 하트
게임 완주 성공 (생존) +1 하트
최대 보유 개수 3개

4.6 다국어 적용

요소 다국어 적용 비고

버튼 텍스트 O (KO/EN/JP) 고정 텍스트

영역 상태

  • 시스템 설정 또는 앱 내 설정에 따라 자동 전환
  • PDF 디자인: 다크 테마 기준, 라이트 테마도 동일 구조

5. 에러 및 예외상황

5.1 데이터 로딩 실패

상황
  • 게임 정보 API 호출 실패 또는 네트워크 오류
처리
  1. Error 상태 표시
  2. 에러 메시지: "게임 정보를 불러오는 데 실패했습니다."
  3. [다시 시도] 버튼 제공

5.2 GP 잔액 부족

상황
  • 보유 GP < 참여 비용 상태에서 입장하기 버튼 탭
처리
  1. CEB-GAM-202-MD-NOBALANCE 모달 표시
  2. 모달 내 안내: "참가하려면 {N} GP가 필요해요. 현재 보유 GP: {N}"

5.3 정원 초과

상황
  • 입장 요청 시 서버에서 정원 초과 응답
처리
  1. 에러 토스트: "정원이 마감되었습니다."
  2. 버튼 비활성화 상태로 전환
  3. 버튼 텍스트: "입장 마감"

5.4 시작 후 입장 시도

상황
  • 입장 가능 상태에서 시작 시점이 지난 후 버튼 탭 (클라이언트/서버 시간 차이)
처리
  1. 에러 토스트: "이미 시작된 게임입니다."
  2. 화면 상태 갱신 (입장 불가 상태로 전환)

5.5 입장 중 서버 오류

상황
  • 입장 요청 후 서버 오류 응답
처리
  1. 버튼 원래 상태로 복구 (로딩 해제)
  2. 에러 토스트: "입장에 실패했습니다. 다시 시도해주세요."
  3. GP 차감 없음 (서버에서 트랜잭션 롤백)

5.6 중복 입장 시도

상황
  • 이미 입장한 게임에 다시 입장 시도
처리
  1. 에러 토스트: "이미 입장한 게임입니다."
  2. 버튼 "참여 완료" 상태로 전환
