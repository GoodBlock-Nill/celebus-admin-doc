1. 화면 목적

실시간 Survival Trivia 퀴즈 게임 진행 화면입니다. 각 스테이지(문제)별로 4지선다 선택지를 제공하고, 제한 시간(10초) 내에 정답을 선택해야 합니다. 정답/오답 결과가 실시간으로 공개되며, 참여자 간 라이브 채팅을 통해 소통할 수 있습니다.

이 문서는 Stage별 문제 풀이 진행 화면을 다룹니다. 탈락 모달(CEB-GAM-206-MD-ELIMINATE), 나가기 모달(CEB-GAM-206-MD-LEAVE), 관전 모드(CEB-GAM-206-SPECTATE), 게임 결과(CEB-GAM-206-RESULT)는 별도 문서입니다.

3. 화면 내 기능설명

3.1 선택지 탭 (응답)

트리거
  • 4개 선택지 중 하나를 탭
동작
  1. 선택 즉시 서버로 전송
  2. 선택한 선택지 하이라이트 (선택 확정 표시)
  3. 나머지 선택지 탭 비활성화 (변경 불가)
  4. 다른 참여자 응답 대기
  5. 제한시간 종료 시 서버에서 정답 공개 이벤트 수신
선택 확정 표시 UI
항목 설명
선택 확정 표시 선택된 선택지: 파란색 반투명 배경 (opacity 20%) + 파란색 테두리 (2px solid). 좌측 번호: 파란색 원형 배경 + 흰색 숫자
미선택 선택지 기본 스타일 유지, 탭 비활성화 (중복 선택 불가)
전환 애니메이션 선택 시 0.2초 페이드인
규칙
  • 선택 후 변경 불가 (실시간 게임 특성)

3.2 타이머 바

표시 규칙
  • 서버 기준 {N}초(기본 10초) 카운트다운
  • 프로그레스 바: 좌→우 파란색, 시간 경과에 따라 감소
  • 서버 시간 기준 (클라이언트 시간 보정)
항목 설명
타이머 시작 매 문제 시작 시 서버에서 시작 시각 전달, 클라이언트는 오프셋 적용하여 카운트다운 표시
동작
  • 시간 종료 시 자동으로 정답 공개 화면 전환

3.3 정답/오답 공개

트리거
  • 서버에서 정답 + 선택지별 선택 비율(%) 수신
동작
  1. 정답 선택지: 파란 테두리 + ⭕ 마크
  2. 오답 선택지 (내가 선택한 경우): 빨간 테두리 + "내 선택" 배지
  3. 정답 선택지 (내가 선택한 경우): 파란 테두리 + ⭕ 마크 + "내 선택" 배지
  4. 나머지 선택지: 기본 테두리 + 비율만 표시
  5. 공개 후 5초 대기 → 자동으로 다음 스테이지 전환
항목 설명
선택 비율 갱신 정답 공개 시점에 서버에서 최종 비율 1회 전달. 5초 표시 동안 비율 변경 없음 (고정)

3.4 하트 소진

오답 시
  • 하트 1개 자동 소진 + 하트 소진 애니메이션 (❤️→💔 효과)
  • 하트 0개에서 오답 시: 정답 공개 5초 표시 → (5초 종료) → 탈락 모달(CEB-GAM-206-MD-ELIMINATE) 표시 → 5초 유예 → 미선택 시 CEB-GAM-204 이동
    • ※ 정답 공개와 탈락 모달은 연속 표시 (총 10초). 생존자는 정답 공개 5초 후 다음 스테이지로 이동
  • 단, 마지막 스테이지(10/10)에서 하트 0 + 오답 시 → 정답 공개 5초 표시 후 탈락 모달 생략 → CEB-GAM-206-RESULT 직행 (D타입: 탈락 미관전)
미응답(시간 초과) 시
  • 타이머 만료 시 키보드가 열려 있다면 자동 닫힘
  • 미응답(시간 초과) 시 하트 분기:
    • 하트 있음: 하트 1개 소진 → 다음 스테이지 진행 (탈락 모달 미표시)
    • 하트 없음: 정답 공개 5초 표시 → (5초 종료) → 탈락 모달(CEB-GAM-206-MD-ELIMINATE) 표시 → 5초 유예 → 미선택 시 CEB-GAM-204 이동 (오답 탈락과 동일 플로우, 동일 타이밍)
  • 단, 마지막 스테이지(10/10)에서 미응답 시 → 하트 소진 후 정답 공개 5초 표시 후 탈락 모달 생략 → CEB-GAM-206-RESULT 직행 (D타입: 탈락 미관전)
미응답(시간 초과) 후 정답 공개
  • 오답 시 정답 공개 UI와 동일한 레이아웃
  • 차이점: 선택지에 "오답" 배지 대신 "미선택" 배지(회색) 표시
  • 선택 비율(%)은 동일하게 표시 (미응답자는 비율 계산에서 제외)
  • 정답 공개 후 탈락 플로우도 오답과 동일 (정답 공개 5초 → 탈락 모달 5초 유예)

3.5 채팅 기능

메시지 표시
  • 화면 하단 채팅 영역에 실시간 메시지 스트림 표시
  • 닉네임 + 메시지 형태로 좌측 정렬
  • 새 메시지 수신 시 자동 하단 스크롤
메시지 입력/전송
  • 💬 버튼 탭 → 키보드 오픈 + 입력 필드 활성화
  • ▶ 전송 버튼 탭 → 메시지 서버 전송
제한 규칙
규칙 내용
욕설 필터링 금칙어 목록 기반 자동 마스킹 처리 ("***")
발송 제한 초당 1회 (rate limit), 초과 시 발송 무시
문자 수 제한 최대 100자, 초과 입력 불가
키보드 활성화 시
  • 상태 바(하트/스테이지/참여자) 숨김
  • 타이머 바와 문제/선택지는 축소 표시
  • 채팅 중에도 선택지 탭 가능 (키보드가 선택지를 가릴 수 있지만 스크롤로 접근 가능). 선택지 탭 시 키보드 자동 닫힘
  • 키보드 닫기: 빈 영역 탭, 전송 완료 후, 선택지 탭 시, 또는 타이머 만료 시 자동 닫힘
키보드 열림 중 타이머 만료 시 전환 순서
항목 설명
타이머 만료 시 전환 순서 (1) 키보드 자동 닫힘 (blur 처리) (2) 상태 바 복원 (하트/스테이지/참여자 재표시) (3) 정답 공개 화면 전환 (서버 이벤트 수신 즉시). 키보드 닫힘 애니메이션 완료를 기다리지 않음 (동시 진행)

3.6 뒤로가기

트리거
  • Header 좌측 [<] 버튼 탭
  • 브라우저 뒤로가기 버튼 및 제스처 (Android HW 백 버튼, popstate 이벤트)도 동일하게 CEB-GAM-206-MD-LEAVE 모달 표시

iOS Safari에서 스와이프 백 제스처는 완전 차단 불가 → 세션 복구(페이지 새로고침 처리 참조)로 대응

동작
  1. CEB-GAM-206-MD-LEAVE (나가기 확인) 모달 표시
  2. 나가기 확인 시: 게임 포기 + CEB-GAM-204(트리비아 홈)로 이동
  3. GP 환불 없음

3.7 스테이지 전환

트리거
  • 정답/오답 공개 후 5초 대기
동작 (자동 전환 시 화면 초기화)
  1. 타이머 바 리셋 (풀 파란색)
  2. 선택지 활성화 (선택 가능 상태)
  3. 결과 텍스트(정답/오답) 제거
  4. 새 문제 텍스트 + 선택지 표시
  5. 스테이지 번호 +1 (예: 1/10 → 2/10)
마지막 스테이지 완료 시
  • 마지막 스테이지(10/10) 정답 후 → CEB-GAM-206-RESULT (완주 성공)
  • 마지막 스테이지(10/10) 오답/미응답 시:
    • → 정답 공개 5초 표시 (유저가 마지막 문제의 정답 확인 가능)
    • → 탈락 모달 생략
    • → CEB-GAM-206-RESULT 직행 (D타입: 탈락 미관전)

4. 규칙/로직

게임 진행 규칙

규칙 설명
총 문제 수 10문제 (서버에서 설정, 변동 가능)
문제당 제한 시간 10초 (서버 기준)
선택지 수 4개 (객관식)
응답 변경 불가 (선택 즉시 확정)
미응답 (시간 초과) + 하트 있음 하트 1개 소진 → 다음 스테이지 진행
미응답 (시간 초과) + 하트 없음 즉시 탈락
오답 하트 1개 소진
하트 0개 + 오답 즉시 탈락
정답 하트 변화 없음, 다음 스테이지 진출

하트 시스템

항목 값
최초 지급 트리비아 최초 참여 시 1개 지급
보너스 획득 게임 끝까지 관전 시 +1, 게임 완주 성공 시 +1
최대 보유 3개
보유 방식 계정 레벨 영구 보유 (게임별 초기화 없음)
오답 소진 1개
미응답 하트 있음: 하트 1개 소진 → 다음 스테이지 진행 / 하트 없음: 즉시 탈락
재획득 보너스 조건 달성으로 재획득 (관전 완료 시 +1, 완주 성공 시 +1)

정답/오답 시각 표시

상태 테두리 색상 아이콘 배지 비율 표시
정답 (내 선택) 파란색 ⭕ "내 선택" O
정답 (타인 선택) 파란색 ⭕ 없음 O
오답 (내 선택) 빨간색 없음 "내 선택" O
일반 선택지 기본 (회색) 없음 없음 O

선택 비율 계산

  • 각 선택지별 선택 비율 = (해당 선택지 선택 인원 / 전체 응답 인원) × 100
  • 소수점 없이 정수로 표시 (반올림)
  • 미응답자는 비율 계산에서 제외 (응답자 기준 비율 계산. 예: 총 100명 중 80명 응답 → 응답자 80명 기준으로 각 선택지 비율 계산)

항목 설명

5. 에러 및 예외상황

네트워크 끊김 중 응답

상황
  • 선택지 탭 후 서버 전송 실패
처리
  1. 자동 재전송 시도 (최대 3회)
  2. 성공 시: 정상 진행
  3. 실패 시: 미응답 처리 → 하트 있음: 하트 1개 소진 → 다음 스테이지 / 하트 없음: 즉시 탈락
응답 전송 사용자 피드백 UI
항목 설명
응답 전송 선택지 탭 시 즉시 선택 확정 UI 표시 (낙관적 업데이트). 백그라운드에서 서버 전송 시도 (최대 3회, 1초 간격)
전송 성공 정상 진행 (추가 UI 변화 없음)
전송 실패 에러 토스트: "응답 전송에 실패했습니다." (Error 타입). 미응답 처리 → 하트 0이면 탈락

서버 오류

상황
  • 문제 수신 또는 정답 수신 실패
처리
  1. 에러 토스트: "일시적 오류가 발생했습니다."

라이브 서버 다운

상황
  • 게임 진행 중 서버 전체 장애
처리
  1. "서버 장애로 게임이 중단되었습니다." 안내
  2. GP 100% 환불 처리 (서버 자동)
  3. 공지 발송

채팅 전송 실패

상황
  • 채팅 메시지 전송 실패
처리
  1. 전송 실패 표시 (빨간색 느낌표)
  2. 재전송 없음 (다시 입력)

채팅 금칙어 감지

상황
  • 욕설/금칙어가 포함된 메시지 전송 시도
처리
  1. 메시지 내 금칙어 부분 "***" 마스킹 후 전송 (차단이 아닌 마스킹)

운영자 강제 종료

상황
  • 운영자가 게임을 강제 종료한 경우
처리
  1. 안내 모달: "운영자에 의해 게임이 종료되었습니다."
  2. GP 100% 환불 처리 (서버 자동)
  3. 모달 확인 시 CEB-GAM-204(트리비아 홈)로 이동

브라우저 탭 비활성화/활성화 전환

상황
  • 게임 진행 중 브라우저 탭이 비활성화된 후 다시 활성화될 때 (visibilitychange 이벤트 기반)
처리
항목 처리
탭 복귀 동작 WebSocket 재연결 성공 → 서버에서 현재 게임 상태 조회
놓친 스테이지 없음 현재 스테이지에 합류 (정상 복귀)
놓친 스테이지 존재 네트워크 끊김으로 인한 별도 탈락 처리 완료 (일반 미응답 하트 규칙과 무관) → CEB-GAM-205-MD-RECONNECT 모달 표시 (나가기/관전 선택)
게임 종료된 경우 CEB-GAM-206-RESULT 화면으로 이동

브라우저 탭 닫기 / 종료

상황
  • 게임 진행 중 브라우저 탭을 닫거나 브라우저를 종료한 경우
처리
  • beforeunload 이벤트로 경고 표시. 닫힘 시 서버에서 WebSocket 끊김 감지 → 30초 타임아웃 후 이탈 처리 (네트워크 끊김으로 인한 별도 탈락 처리)

페이지 새로고침 처리

항목 처리
새로고침 방지 CSS overscroll-behavior: none으로 pull-to-refresh 차단
beforeunload 게임 진행 중 새로고침/탭 닫기 시 브라우저 기본 경고 표시
세션 복구 페이지 로드 시 서버 API로 활성 세션 확인. 있으면 현재 스테이지로 복귀 (WebSocket 재연결). 놓친 문제는 미응답 처리. 게임 종료 시 RESULT로 이동
