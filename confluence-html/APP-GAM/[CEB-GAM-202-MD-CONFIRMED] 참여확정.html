1. 화면 목적

사용자가 Prediction Market 게임에서 YES 또는 NO를 선택한 후, 참여를 최종 확정하기 전에 선택 내용과 참여 비용을 확인하고 참여 의사를 재확인하는 모달입니다. 참여 후에는 선택을 변경할 수 없다는 점을 명확히 안내하여 실수를 방지합니다.

3. 화면 내 기능설명

모달 트리거

트리거 조건 동작
게임 상세(CEB-GAM-202)에서 YES 버튼 탭 "선택 항목: YES" 표시, 본 모달 오픈
게임 상세(CEB-GAM-202)에서 NO 버튼 탭 "선택 항목: NO" 표시, 본 모달 오픈

버튼 동작

취소 버튼

  1. 모달 닫기
  2. 게임 상세 화면으로 복귀
  3. 상태 변경 없음

참여하기 버튼

1. 참여 API 호출

  • 게임 ID
  • 선택 항목 (YES 또는 NO)
  • 사용자 ID
  • 참여 비용 (GP 차감)

2. 성공 시:

  • 사용자 GP 차감
  • 참여 내역 생성
  • 본 모달 닫기
  • CEB-GAM-202-MD-COMPLETED (참여완료 모달) 표시

3. 실패 시:

  • GP 부족: CEB-GAM-202-MD-NOBALANCE (GP 잔고 부족 모달) 표시
  • 기타 에러: 에러 토스트 표시, 본 모달 유지

모달 닫기 동작

  • 취소 버튼 클릭 시
  • 모달 외부 영역(딤) 탭 시
  • 뒤로가기 제스처

4. 규칙/로직

참여 가능 조건

조건 검증 시점 실패 시 처리
게임 상태 = Active API 호출 시 "참여 가능한 게임이 아닙니다." 에러 토스트
사용자 GP 잔액 >= 참여 비용 API 호출 시 CEB-GAM-202-MD-NOBALANCE 모달 표시
참여 정원 미달 (정원 설정 시) API 호출 시 "참여 정원이 마감되었습니다." 에러 토스트
중복 참여 방지 API 호출 시 "이미 참여한 게임입니다." 에러 토스트
투표 종료일시 미경과 API 호출 시 "참여 기간이 종료되었습니다." 에러 토스트

참여 비용 표시

  • 게임 설정에서 정의된 참여 비용 값을 표시
  • 기본값: 1 GP
  • 형식: "{숫자} GP" (천 단위 콤마)

선택 항목 표시

  • YES: "YES" 텍스트 표시
  • NO: "NO" 텍스트 표시
  • 색상: 검정색 (강조 없음)

5. 에러 및 예외상황

클라이언트 에러

에러 상황 에러 메시지 처리 방법
게임 상태가 Active가 아님 "참여 가능한 게임이 아닙니다." 에러 토스트, 모달 닫기, 게임 상세 갱신
GP 잔액 부족 "GP가 부족합니다." CEB-GAM-202-MD-NOBALANCE 모달 표시
이미 참여한 게임 "이미 참여한 게임입니다." 에러 토스트, 모달 닫기, 게임 상세 갱신

서버 에러

HTTP 상태 에러 메시지 처리 방법
400 "유효하지 않은 요청입니다." 에러 토스트, 모달 유지
403 "참여 권한이 없습니다." 에러 토스트, 모달 닫기
404 "게임을 찾을 수 없습니다." 에러 토스트, 게임 리스트로 이동
409 "이미 참여한 게임입니다." 에러 토스트, 모달 닫기, 게임 상세 갱신
422 "참여 정원이 마감되었습니다." 에러 토스트, 모달 닫기, 게임 상세 갱신
500 "서버 오류가 발생했습니다." 에러 토스트, 모달 유지

네트워크 에러

에러 상황 처리 방법
타임아웃 "요청 시간이 초과되었습니다. 다시 시도해주세요." 에러 토스트
연결 실패 "네트워크 연결을 확인해주세요." 에러 토스트
