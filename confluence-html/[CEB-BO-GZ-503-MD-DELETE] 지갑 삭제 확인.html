1. 화면 목적

등록된 운영 지갑을 삭제하기 전 최종 확인하는 모달입니다. 삭제된 지갑은 복구할 수 없으므로 신중한 확인이 필요합니다. 지갑 삭제 후 해당 지갑으로 처리된 기존 교환 내역은 유지됩니다.

3. 화면 내 기능설명

모달 트리거

트리거 조건 동작
CEB-BO-GZ-503(지갑 관리) 목록의 지갑 Row [삭제] 버튼 클릭 본 모달 오픈

버튼 동작

취소 버튼

  1. 모달 닫기
  2. 지갑 관리 화면으로 복귀
  3. 상태 변경 없음
  4. 목록 갱신 없음

삭제하기 버튼

  1. 지갑 삭제 요청 실행
  2. 성공 시:
    • 지갑 데이터 완전 삭제
    • 토스트 메시지: "지갑이 삭제되었습니다."
    • 모달 닫기
    • 지갑 목록 갱신 (해당 지갑 제거)
    • 실패 시:
      • 에러 토스트 표시
      • 모달 유지 (재시도 가능)

닫기(X) 버튼

  • 취소 버튼과 동일한 동작

4. 규칙/로직

삭제 가능 조건

조건 검증 시점 실패 시 처리
지갑이 존재함 모달 오픈 전 모달 미표시, 에러 토스트
관리자 삭제 권한 API 호출 시 403 에러

삭제 가능 상태

상태 삭제 가능 여부
활성 (Active) O
비활성 (Inactive) O

지갑의 활성/비활성 상태와 관계없이 삭제 가능합니다.

삭제 시 처리 로직

데이터 삭제

  1. 지갑 기본 정보 삭제 (지갑 ID, 유형, 주소, 잔액 등)
  2. 관련 설정 정보 삭제
  3. 지갑의 활성/비활성 상태 정보 삭제

관련 데이터 유지

  • 해당 지갑으로 처리된 교환 내역(CEB-BO-GZ-501)은 유지
  • 교환 내역의 지갑 정보(유형, 주소)는 그대로 표시
  • GP 변동 내역(CEB-BO-GZ-601)은 그대로 유지

관리자 로그

지갑을 삭제했습니다. (지갑ID: {id})
로그 필드 값
액션 타입 WALLET_DELETE
지갑 ID {walletId}
지갑 유형 {walletType: CHARGE 또는 WITHDRAW}
지갑 주소 {walletAddress}
관리자 ID {adminId}
타임스탬프 {ISO 8601}

  • Tab 키로 버튼 간 포커스 이동
  • Enter 키로 포커스된 버튼 실행

키 동작

5. 에러 및 예외상황

클라이언트 에러

에러 상황 에러 메시지 처리 방법
지갑이 존재하지 않음 "지갑을 찾을 수 없습니다." 모달 미표시, 리스트 갱신
이미 삭제된 지갑 "이미 삭제된 지갑입니다." 모달 닫기, 리스트 갱신

서버 에러

HTTP 상태 에러 메시지 처리 방법
400 "삭제할 수 없는 지갑입니다." 에러 토스트, 모달 닫기, 화면 갱신
403 "지갑 삭제 권한이 없습니다." 에러 토스트, 모달 닫기
404 "지갑을 찾을 수 없습니다." 에러 토스트, 리스트로 이동
409 "이미 삭제 처리 중인 지갑입니다." 에러 토스트, 리스트로 이동
500 "서버 오류가 발생했습니다." 에러 토스트, 모달 유지

네트워크 에러

에러 상황 처리 방법
타임아웃 "요청 시간이 초과되었습니다. 다시 시도해주세요."
연결 실패 "네트워크 연결을 확인해주세요."
