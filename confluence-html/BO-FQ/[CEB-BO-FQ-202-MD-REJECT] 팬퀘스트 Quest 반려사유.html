1. 화면 목적

제출된 퀘스트 결과물이 미션 조건에 부합하지 않는다고 판단될 경우, 구체적인 반려 사유(Rejection Reason)를 지정하여 반려 처리를 수행합니다.

지정된 사유는 유저에게 전달되어 재참여를 위한 가이드가 됩니다.

주요 기능

  • 반려 사유 선택 (드롭다운)
  • 반려 처리 확정 및 유저 알림

진입 경로

  • 대기내역 탭(CEB-BO-FQ-202)에서 특정 제출 건의 [반려] 버튼 클릭 시 호출

2. UI 설명

모달 구조

타이틀: "Quest 반려사유"

닫기 버튼 (X): 우측 상단. 클릭 시 반려 처리를 취소하고 모달을 닫습니다.

입력 폼

  • 반려사유 선택 (Select Box)
    • Label: 반려사유 선택
    • 기능: 클릭 시 [CEB-BO-FQ-101-REJETSET] 메뉴에서 사용(Active) 상태로 설정된 반려 사유 목록이 드롭다운으로 노출
    • 예시: "식별불가", "대상 불일치", "중복 참여 의심" 등
    • 드롭다운 정렬: 이름순 정렬 (A-Z)
    • Default: 선택되지 않은 상태 (Placeholder: "사유를 선택해주세요")

하단 액션 버튼

  • [취소하기]: 반려 프로세스를 중단하고 대기내역 화면으로 돌아감
  • [저장하기]: 선택한 사유로 반려 처리를 확정

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[취소하기] 반려 프로세스를 중단하고 대기내역 화면으로 복귀
[저장하기] 선택한 사유로 반려 처리 확정, 서버로 반려 요청 전송
[X] 닫기 반려 처리를 취소하고 모달 닫기

4. 규칙/로직

4.1 반려 프로세스

  • 진입: [CEB-BO-FQ-202] 대기내역 화면에서 특정 제출 건의 [반려] 버튼 클릭 시 호출
  • 선택(Select): 관리자는 드롭다운 목록에서 가장 적절한 사유 하나를 선택해야 합니다.
  • 확정(Confirm):
    • [저장하기] 클릭 시 서버로 반려 요청을 전송
    • 해당 제출 건의 상태는 Pending → Rejected로 변경되며, 선택된 사유 코드가 함께 저장
    • 유저에게는 해당 사유에 매핑된 유저 노출 메시지가 전달

4.2 유효성 검사

  • 필수값 체크: 사유를 선택하지 않고 [저장하기] 클릭 시, 경고 메시지를 노출하거나 진행되지 않아야 합니다.

5. 에러 및 예외상황

5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

성공 (Success)

  • 메시지: "요청성공 - 선택한 퀘스트의 반려 처리가 완료되었습니다"
  • 토스트 타입: Success
  • 후행 동작: 모달이 닫히고, 대기내역 목록이 새로고침되어 해당 건이 사라집니다 (처리내역으로 이동).

실패 (Failure)

  • 메시지: "요청실패 - 선택한 퀘스트의 반려 처리에 실패하였습니다"
  • 토스트 타입: Error
  • 후행 동작: 모달이 닫히지 않고 유지됩니다.
