1. 화면 목적

작성 완료된 Draft(작성 중) 상태의 퀘스트를 유저에게 공개(Active)하기 전, 관리자의 최종 확인을 수행합니다.

게시 시점부터 퀘스트가 즉시 시작됨을 인지시킵니다.

주요 기능

  • 퀘스트 게시 최종 확인
  • 상태 변경 (Draft → Active)

진입 경로

  • Quest 상세 화면(CEB-BO-FQ-201)에서 Draft 상태일 때 [게시하기] 버튼 클릭 시 호출

2. UI 설명

모달 구조

  • 아이콘: 상단 중앙에 번개 모양 아이콘 배치
  • 타이틀: "Quest 게시"
  • 안내 메시지:
    • "확인 버튼을 누르면 Quest가 게시됩니다"
    • "계속 진행하시겠습니까?"

하단 액션 버튼

  • [다시 돌아가기]: 게시 프로세스를 취소하고 모달을 닫음
  • [확인]: 퀘스트 상태를 Active로 변경하고 유저에게 공개

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[다시 돌아가기] 게시 프로세스를 취소하고 모달 닫기
[확인] 서버로 상태 변경 요청 전송, 퀘스트 상태를 Draft → Active로 변경

4. 규칙/로직

4.1 게시 프로세스

  • 진입: Draft 상태의 퀘스트 상세 화면에서 [게시하기] 버튼 클릭 시 호출
  • 확인(Confirm):
    • [확인] 버튼 클릭 시 서버로 상태 변경 요청을 전송
    • 시작 일시 설정: 게시가 완료되는 시점의 서버 시간(Timestamp)이 해당 퀘스트의 시작(게시) 일시로 자동 기록
    • 상태가 Draft → Active로 변경되어 유저 앱(App)에 즉시 노출

5. 에러 및 예외상황

5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

성공 (Success)

  • 메시지: "요청성공 - Quest가 게시 되었습니다."
  • 토스트 타입: Success
  • 후행 동작: 모달이 닫히고, 상세 화면이 새로고침되어 상태 뱃지가 Active로 변경됩니다. (버튼은 [종료하기]로 변경됨)

실패 (Failure)

  • 메시지: "요청실패 - Quest 게시에 실패하였습니다. 다시 시도하세요"
  • 토스트 타입: Error
  • 원인 예시: 필수 정보 누락(유효성 검사 실패), 네트워크 오류 등
  • 후행 동작: 모달이 닫히지 않고 유지됩니다.
