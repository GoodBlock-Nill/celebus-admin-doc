1. 화면 목적

Raffle 추첨이 완료된(Ended) 후, 당첨자(Winner)에 한해 특이사항(배송 정보, CS 기록 등)을 관리자가 수동으로 기록하고 저장합니다.

주의: 낙첨자(Lose) 항목 클릭 시에는 이 모달이 호출되지 않습니다.

주요 기능

  • 당첨자 기본 정보 확인 (닉네임, 전화번호)
  • 비고사항 입력 및 저장

진입 경로

  • 추첨내역 탭(CEB-BO-FQ-206) 목록에서 상태가 Win(당첨)인 Row 클릭 시 호출

2. UI 설명

모달 구조

타이틀: 당첨자 비고 사항

닫기 버튼 (X): 우측 상단

참여자 정보 영역 (User Info)

선택한 당첨자의 기본 식별 정보를 표시합니다.

  • 닉네임: {유저닉네임}
  • 전화번호: {전화번호}

비고사항 입력 영역 (Memo Input)

  • 라벨: 비고사항 입력
  • 입력창 (Textarea)
    • Placeholder: "비고사항을 입력해주세요"
    • 글자수 제한: 우측 하단에 (현재글자수 / 200) 카운터 표시. 최대 200자까지 입력 가능
    • 기본값: 이전에 저장된 비고 내용이 있다면 불러와서 표시

하단 액션 버튼

  • [취소하기]: 작성 중인 내용을 저장하지 않고 모달을 닫음
  • [저장하기]: 입력한 비고 내용을 서버에 저장하고 모달을 닫음

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[취소하기] 작성 중인 내용을 저장하지 않고 모달 닫기
[저장하기] 입력한 비고 내용을 서버에 저장하고 모달 닫기
[X] 닫기 모달 닫기

4. 규칙/로직

4.1 모달 호출 조건 (Trigger)

  • 화면: [CEB-BO-FQ-206] Raffle 상세(추첨내역) 탭
  • 조건: 리스트의 행(Row) 중 상태가 Win(당첨)인 항목을 클릭했을 때만 호출
  • 예외: Lose(낙첨) 항목 클릭 시에는 아무런 동작을 하지 않습니다

4.2 저장 프로세스

  • [저장하기] 클릭 시 해당 당첨자의 데이터에 비고 내용을 업데이트(Update)합니다.

5. 에러 및 예외상황

5.1 시스템 응답 처리 (Toast Message)

성공 (Success)

  • 메시지: "비고사항이 저장되었습니다"
  • 토스트 타입: Success

실패 (Failure)

  • 메시지: "비고사항 저장에 실패하였습니다. 다시 시도하세요"
  • 토스트 타입: Error
