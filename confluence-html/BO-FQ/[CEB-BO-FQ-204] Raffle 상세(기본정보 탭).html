1. 화면 목적

등록된 Raffle(추첨형 퀘스트)의 전체적인 설정 정보(기본 정보, 규칙, 설명, 보상 등)를 한눈에 파악하고 관리합니다.

주요 기능

  • Raffle의 생애 주기(Lifecycle) 모니터링 및 상태 변경 액션 수행 (임시저장(Draft) -> 진행중(Active) -> 추첨대기(Closed) -> 종료(Ended))
  • 실제 응모자 관리나 추첨 실행을 위한 하위 탭(응모내역, 추첨내역)으로 진입하는 관문 역할

진입 경로

  • [CEB-BO-FQ-102] 팬퀘스트 메인 (Raffle 탭) > Raffle 리스트 ROW 클릭

2. UI 설명

2.1 상단 네비게이션 및 헤더

  • Breadcrumb: 홈 > 게임존 > Fan Quest 관리 > Raffle > {Raffle Title}
  • 현재 위치를 계층적으로 표시하며, 상위 메뉴 클릭 시 이동 가능합니다.
  • 타이틀: 현재 보고 있는 Raffle의 제목을 대제목으로 표시합니다. (예: {Raffle Title})
  • 탭 메뉴 (Tab Menu):
    • 기본정보 (Selected): 현재 화면. Raffle의 설정값 및 상태 정보를 보여줍니다.
    • 응모내역: 유저들의 응모 리스트 표시합니다.
    • 추첨내역: 추첨(Draw) 및 추첨 완료 후 당첨자 명단을 확인하는 화면으로 이동합니다.

2.2 상태별 액션 버튼

화면 우측 상단에 위치하며, 현재 Raffle의 상태(Status)에 따라 노출되는 버튼과 기능이 동적으로 변경됩니다.

현재 상태 버튼명 기능 설명 호출 화면 ID
임시저장(Draft) [삭제하기] 해당 Raffle 데이터를 영구적으로 삭제 CEB-BO-FQ-204-MD-DELETE
[수정하기] 정보 수정 화면으로 이동 CEB-BO-FQ-204-EDIT
[게시하기] Raffle을 공개하고 응모 시작 CEB-BO-FQ-204-MD-OPEN

카드명 아이콘 설명
총 응모자 사람 아이콘 전체 응모 참여자 수
총 응모권 티켓 아이콘 사용된 총 응모권 수량

2.4 상세 정보 그리드 (Info Grid)

화면 중앙 영역을 4분할(2x2) 카드 형태로 구성하여 정보를 그룹화해 보여줍니다.

A. Raffle 정보 (좌상단)

Raffle의 핵심 식별 정보와 규칙 설정값을 표시합니다.

항목 설명
아티스트 연결된 아티스트명
당첨 인원 최종 선발 당첨자 인원 (예: 10명)
배송 타입 택배 배송 / 현장 수령
대표 이미지 이미지 보기 링크
마감일시 YYYY.MM.DD HH:mm

B. 관리 정보 (우상단)

시스템 관리 차원의 메타 데이터를 표시합니다.

  • 상태 (Status Badge): 임시저장(Draft) / 진행중(Active) / 추첨대기(Closed) / 종료(Ended)
  • 생성 정보: 생성 관리자 (이름/ID), 생성 일시
  • 수정 정보: 최근 수정자, 최근 수정 일시

C. Raffle 타이틀 (하단)

  • 유저 앱(App)에 노출되는 타이틀 텍스트
  • 각 언어별(KO/EN/JA) 타이틀 입력값이 표시됩니다.

D. Raffle 설명 (하단)

  • 유저 앱(App) 상세 화면에 노출되는 설명 텍스트
  • 각 언어별(KO/EN/JA) 설명 입력값이 표시됩니다.

E. 경품 상세 (하단)

  • 경품에 대한 구체적인 스펙이나 상세 내용 텍스트
  • 각 언어별(KO/EN/JA) 경품 상세 텍스트가 표시됩니다.

F. 배송 가이드 (하단)

  • 배송 타입이 "택배 배송"인 경우에만 노출
  • 배송지 입력 마감일시: YYYY.MM.DD HH:mm:ss
  • 배송지 입력 폼 URL: 외부 링크

G. 유의사항 (하단)

  • 유저 앱(App)에 노출되는 유의사항 텍스트
  • 각 언어별(KO/EN/JA) 유의사항 텍스트가 표시됩니다.

3. 화면 내 기능설명

3.1 버튼 동작

  • [삭제하기] (임시저장(Draft)): 해당 Raffle 데이터를 영구 삭제합니다. 삭제 확인 모달(CEB-BO-FQ-204-MD-DELETE)을 호출합니다.
  • [수정하기] (임시저장(Draft)/진행중(Active)): 정보 수정 화면(CEB-BO-FQ-204-EDIT)으로 이동합니다.
  • [게시하기] (임시저장(Draft)): 유저에게 Raffle을 공개합니다. 게시 확인 모달(CEB-BO-FQ-204-MD-OPEN)을 호출합니다.
  • [종료하기] (진행중(Active)): 응모를 조기 마감합니다. 종료 확인 모달(CEB-BO-FQ-204-MD-CLOSE)을 호출합니다.

3.2 네비게이션

  • 응모내역 탭 클릭: [CEB-BO-FQ-205] 응모내역 화면으로 이동
  • 추첨내역 탭 클릭: [CEB-BO-FQ-206] 추첨내역 화면으로 이동

4. 규칙/로직

4.1 상태 전환 및 생애 주기 (Lifecycle)

상태 설명
임시저장(Draft) 최초 생성 시 상태. 자유롭게 수정/삭제 가능
진행중(Active) [게시하기] 수행 시 전환. 응모 시작
추첨대기(Closed) 마감일시 경과 시 자동 전환, 또는 [종료하기]로 강제 전환. 추첨(Draw) 가능
종료(Ended) 추첨 실행 후 시스템 자동 전환

4.2 정보 수정 정책

  • 진행중(Active) 상태 수정: 이미 응모자가 있을 수 있으므로, 마감일시를 앞당기거나 당첨 수를 줄이는 등 유저에게 불리한 변경은 신중해야 합니다. (시스템상으로는 허용하되 경고 모달 등을 통해 주의를 줄 수 있음)
  • 추첨대기(Closed)/종료(Ended) 상태 수정 불가: 이미 마감된 이벤트의 정보는 데이터 무결성을 위해 수정할 수 없습니다.
