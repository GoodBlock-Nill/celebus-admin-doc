1. 화면 목적

이미 생성되어 운영 중이거나 작성 중인 Raffle(추첨형 퀘스트)의 정보를 변경합니다.

주요 기능

  • Raffle의 현재 진행 상태(임시저장(Draft) 또는 진행중(Active))에 따라 수정 가능한 데이터의 범위를 엄격하게 제어
  • 운영의 유연성과 데이터 무결성(공정성)을 동시에 확보

진입 경로

  • [CEB-BO-FQ-204] Raffle 상세 (기본정보 탭) > [수정하기] 버튼 클릭

2. UI 설명

2.1 상단 헤더 (Header)

  • 상태 뱃지 (Status Badge): 타이틀 좌측에 현재 Raffle의 상태를 시각적으로 표시합니다. (임시저장(Draft) / 진행중(Active))
  • 타이틀: Raffle 수정 텍스트 표시

2.2 좌측 네비게이션 및 상태 (Left Panel)

수정해야 할 데이터 항목들을 리스트업하고, 각 항목의 입력 상태를 표시합니다.

  • 항목 리스트: 대표 이미지, 타이틀, 설명, 보상 상세, 수령 가이드
  • 상태 버튼:
    • 입력완료: 기존 데이터가 존재하거나 수정이 완료된 상태
    • 입력중...: 현재 우측 패널에서 편집 중인 상태
    • 입력하기: 데이터가 비어있는 상태 (임시저장(Draft)의 경우)

2.3 우측 편집 영역 (Right Panel)

좌측에서 선택한 항목의 상세 내용을 수정하는 폼(Form)입니다.

  • 진행중(Active) 상태일 때: 모든 텍스트 필드 및 이미지 업로더가 활성화되어 수정 가능합니다.
  • 안내 메시지: 우측 영역 하단 또는 대기 화면에 상태별 수정 가능 범위를 명시합니다.
    • "1. 임시저장(Draft) → 모든 항목 수정 가능"
    • "2. 진행중(Active) → 대표이미지, 타이틀, 설명, 보상설명 만 수정 가능"

2.4 하단 기본 설정 영역 (Bottom Setting Area)

Raffle의 핵심 규칙을 설정하는 영역입니다. 상태에 따라 제어가 달라집니다.

  • 항목: 아티스트 선택, 마감일 설정(날짜/시간), 당첨 추첨 수, 배송 타입
  • 임시저장(Draft) 상태: 모든 필드가 Enable 상태로, 값 변경이 가능합니다.
  • 진행중(Active) 상태: 모든 필드가 Disabled (비활성화) 되어, 값을 조회할 수는 있으나 변경할 수 없습니다.

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

  • [취소하기]: 수정 작업을 중단하고 이전 화면으로 돌아갑니다. 클릭 시 수정 취소 확인 모달을 호출합니다.
  • [저장하기]: 변경된 내용을 서버에 반영합니다. 클릭 시 수정 완료 확인 모달을 호출합니다.

3.2 네비게이션

  • 취소/저장 후: [CEB-BO-FQ-204] Raffle 상세 (기본정보 탭)으로 복귀

4. 규칙/로직

시스템은 페이지 진입 시 Raffle의 상태 코드를 확인하여 UI의 활성/비활성 여부를 결정합니다.
