1. 화면 목적

등록된 전체 Raffle(래플)의 현황을 목록 형태로 조회하고 관리합니다.

주요 기능

  • 상태별(진행중(Active), 임시저장(Draft), 추첨대기(Closed), 종료(Ended)) 필터링 및 검색 기능을 통해 원하는 래플을 빠르게 검색
  • 신규 래플을 등록하기 위한 생성 화면(Create)으로 진입

진입 경로

  • [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > Raffle 탭 클릭

2. UI 설명

2.1 상단 영역 (Top Area)

  • Breadcrumb: Home > 팬퀘스트 > Fan Quest 관리
  • Tab Menu:
    • Quest: 팬 퀘스트 목록 (형제 화면)
    • Raffle (Active): 현재 화면 (래플 목록)

2.2 검색 및 필터 영역

  • 상태 필터 (Select Box)
    • 옵션: 전체(Default), 진행중(Active), 임시저장(Draft), 추첨대기(Closed), 종료(Ended)
  • 검색 조건 (Select Box): 검색 기준(타이틀, 아티스트) 선택 (Default: 타이틀)
  • 검색어 입력 (Input)
    • Placeholder: 타이틀 선택 시 "타이틀 입력", 아티스트 선택 시 "아티스트 입력"
  • [초기화] 버튼: 설정된 검색 조건을 초기 상태로 되돌립니다.
  • [생성하기] 버튼 (우측 상단): 신규 래플 등록 화면(CEB-BO-FQ-102-CREATE)으로 이동합니다.

2.3 목록 영역 (List Table)

데이터는 테이블 형태로 노출되며, 주요 컬럼은 다음과 같습니다.

컬럼명 설명 비고
상태 (Status) 래플의 현재 진행 상태를 배지(Badge) 형태로 표시 진행중(Active) / 임시저장(Draft) / 추첨대기(Closed) / 종료(Ended)
타이틀 래플의 제목
아티스트 해당 래플이 속한 아티스트명
기간 YYYY.MM.DD ~ YYYY.MM.DD 형태의 게시 기간 시작(게시)일 ~ 마감(종료)일. 임시저장(Draft) 상태일 경우 마감(종료)일만 표시 됨
누적 참여자 현재 응모한 총 참여자 수
누적 응모권 현재 응모한 총 응모권 수
  • ROW 클릭 시 상세화면으로 이동
  • 페이지네이션: 목록 하단에 페이지 이동 컨트롤(이전, 다음, 페이지 번호) 제공

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

  • [초기화]: 설정된 검색 조건을 초기 상태로 되돌립니다.
  • [생성하기]: 클릭 시 신규 생성 화면(CEB-BO-FQ-102-CREATE)으로 이동합니다.

3.2 네비게이션

  • 리스트 Row 클릭: 선택한 래플의 상세 화면(CEB-BO-FQ-204) 기본정보 탭으로 이동합니다.
  • Quest 탭 클릭: [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) 화면으로 이동합니다.

4. 규칙/로직

4.1 권한별 기능 분기

권한 화면 접근 조회 [생성하기] ROW 클릭 (상세 이동)
Super Admin ✅ ✅ ✅ ✅
Admin (팬퀘스트 쓰기 권한 有) ✅ ✅ ✅ ✅
Admin (팬퀘스트 읽기 권한만) ✅ ✅ ❌ ✅
권한 없음 ❌ ❌ ❌ ❌
  • 권한은 [CEB-BO-ADST-202] 권한관리에서 설정된 팬퀘스트 대메뉴의 읽기/쓰기 권한에 따라 분기됩니다.

4.2 정렬 기준

정렬 기준 정렬 방식 비고
마감일 (End Date) 오름차순 (임박순) 기본 정렬

4.3 상태 정의

상태 설명
임시저장(Draft) 작성 중이며 아직 유저에게 공개되지 않은 상태
진행중(Active) 게시되어 유저가 응모 가능한 상태
추첨대기(Closed) 모집 기간이 끝났으나 아직 추첨이 확정되지 않은 상태 (이 상태에서만 추첨 가능)
종료(Ended) 추첨이 완료되고 결과가 확정된 종료 상태

4.4 상태별 데이터 노출

  • 임시저장(Draft) 상태의 래플은 관리자 화면에만 노출되며, 앱(유저 화면)에는 노출되지 않습니다.
  • 추첨대기(Closed) 상태는 모집 기간이 지났거나 관리자가 강제 종료한 경우 진입하며, 추첨(Draw) 프로세스 대기 중임을 의미합니다.

5. 에러 및 예외상황

5.1 데이터 없음

  • 검색 결과 없음: 리스트 영역 중앙에 "검색 결과가 없습니다." 안내 문구를 노출합니다.
  • 등록된 데이터 없음: 초기 진입 시 데이터가 없을 경우 "등록된 래플이 없습니다. 신규 래플을 생성해보세요." 등의 안내를 노출합니다.

5.2 시스템 오류

  • 네트워크 지연/오류: 데이터 로딩 실패 시 "데이터를 불러오지 못했습니다. 잠시 후 다시 시도해주세요." 문구와 함께 [새로고침] 버튼을 제공합니다.
  • 권한 없음: 해당 메뉴 접근 권한이 없는 관리자가 접근 시, "접근 권한이 없습니다." Toast 팝업을 띄우고 대시보드 또는 이전 화면으로 리다이렉트합니다.
