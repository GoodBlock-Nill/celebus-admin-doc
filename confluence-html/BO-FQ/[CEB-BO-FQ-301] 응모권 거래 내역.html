1. 화면 목적

관리자가 유저별 응모권(티켓) 거래 이력을 조회하는 화면입니다. Fan Quest 보상, 게임존(ST/PM) 보상, Raffle 응모/반환 등 모든 적립/사용 내역을 통합 조회합니다.

주요 기능

  • 전체 유저의 응모권 변동 내역 통합 조회
  • 유형별 필터링 (적립/사용/반환)
  • 적립경로별 필터링 (Fan Quest/Game Zone)
  • 기간 검색 및 유저 검색 (닉네임/유저ID)

진입 경로

  • 사이드 메뉴 > 팬퀘스트 > 응모권 거래 내역

2. UI 설명

2.1 상단 영역 (Top Area)

  • Breadcrumb: 홈 > 팬퀘스트 > 응모권 거래 내역

2.2 검색 및 필터 영역

구분 요소명 타입 설명
필터 유형 Dropdown 전체 / 적립 / 사용 / 반환
필터 적립경로 Dropdown 전체 / Fan Quest / Game Zone
필터 기간 (시작) Date Picker 조회 시작일
필터 기간 (종료) Date Picker 조회 종료일
입력 검색 Text Input 닉네임 또는 유저ID 입력. 검색은 Enter 키 또는 입력 필드 내 검색 아이콘으로 실행
버튼 초기화 Dark Button 검색 조건을 초기 상태로 리셋
  • 유형 필터 Default: 전체
  • 적립경로 필터 Default: 전체
  • 기간 Default: 없음 (전체 기간)
  • 검색어 Placeholder: "닉네임 또는 유저ID 입력"

2.3 목록 영역 (List Table)

컬럼명 데이터 타입 설명
유형 Badge 적립 (Green) / 사용 (Red) / 반환 (Blue)
적립경로 Text Fan Quest / Game Zone (ST) / Game Zone (PM) / Raffle
유저 Text 닉네임 + 유저ID (2행 표시)
관련 이벤트 Text 해당 Quest/Game/Raffle 타이틀
변동 수량 Number +{N} (적립/반환) / -{N} (사용)
변동 후 잔여 Number 거래 후 해당 유저의 응모권 잔액
일시 DateTime YYYY.MM.DD HH:mm:ss
  • 리스트 Row 클릭 시 별도 동작 없음 (읽기 전용)

2.4 페이지네이션

구분 요소명 설명
버튼 Previous 이전 페이지로 이동
버튼 1, 2, 3, ..., N 해당 페이지로 직접 이동
버튼 Next 다음 페이지로 이동
  • 페이지당 표시 건수: 10건

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

  • [초기화]: 검색 조건을 초기 상태로 리셋합니다.

3.2 필터 동작

  • 유형, 적립경로, 기간, 검색어는 AND 조건으로 적용
  • 검색어 미입력 시 해당 필터 조건 무시
  • 닉네임/유저ID 검색은 부분 일치(LIKE) 검색 지원

4. 규칙/로직

4.1 권한별 기능 분기

권한 화면 접근 조회
Super Admin ✅ ✅
Admin (팬퀘스트 읽기 권한 有) ✅ ✅
권한 없음 ❌ ❌
  • 읽기 전용 화면 (수정/삭제 기능 없음)
  • 권한은 [CEB-BO-ADST-202] 권한관리에서 설정된 팬퀘스트 대메뉴의 읽기 권한에 따라 분기됩니다.

4.2 정렬 기준

정렬 기준 정렬 방식 비고
일시 내림차순 (최신순) 기본 정렬

4.3 유형별 분류

유형 설명 Badge 색상
적립 Fan Quest 보상, Game Zone(ST/PM) 보상 Green
사용 Raffle 응모 Red
반환 Raffle 취소 시 반환 Blue

4.4 적립경로별 분류

적립경로 설명
Fan Quest Quest 참여 보상으로 적립된 응모권
Game Zone Survival Trivia 탈락 보상 또는 Prediction Market 보상으로 적립된 응모권
Raffle Raffle 응모(사용) 또는 Raffle 취소(반환)

4.5 검색 조건

  • 유형 필터와 적립경로 필터, 기간, 검색어는 AND 조건으로 적용
  • 검색어 미입력 시 해당 필터 조건 무시
