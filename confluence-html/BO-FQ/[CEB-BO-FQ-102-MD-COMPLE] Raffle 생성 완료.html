1. 화면 목적

모든 필수 정보 입력이 완료된 Raffle에 대해, 관리자의 최종 생성 의사를 확인하고 실제 데이터 생성을 요청합니다.

주요 기능

  • Raffle 생성 최종 확인
  • 서버 데이터 생성 요청 (Draft 상태)

진입 경로

  • Raffle 생성 화면(CEB-BO-FQ-102-CREATE)에서 모든 필수값 입력 완료 후 [생성하기] 버튼 클릭 시 호출

2. UI 설명

모달 구조

  • 아이콘: 상단 중앙에 번개 모양 아이콘 배치
  • 타이틀: "Raffle 생성"
  • 안내 메시지:
    • "확인 버튼을 누르면 새로운 Raffle이 생성됩니다"
    • "계속 진행하시겠습니까?"

하단 액션 버튼

  • [다시 돌아가기]: 생성 프로세스를 보류하고, 작성 화면으로 돌아가 내용을 재검토하거나 수정
  • [확인]: 입력된 모든 데이터를 서버로 전송하여 새로운 Raffle을 생성(Draft 상태)

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[다시 돌아가기] 생성 프로세스를 보류하고 작성 화면으로 복귀
[확인] 입력된 모든 데이터를 서버로 전송하여 Raffle 생성 (Draft 상태)

4. 규칙/로직

4.1 생성 프로세스

  • 진입: Raffle 생성 화면에서 모든 필수값이 입력된 상태(입력완료)에서 [생성하기] 버튼 클릭 시 호출
  • 확인(Confirm): [확인] 버튼 클릭 시 서버로 생성 요청을 전송합니다. 생성된 Raffle은 초기 상태인 Draft(작성 중) 상태로 저장됩니다. (즉시 게시되지 않음)

5. 에러 및 예외상황

5.1 시스템 응답 처리 (Toast Message)

작업 결과는 상단 Toast 팝업으로 피드백을 제공합니다.

성공 (Success)

  • 메시지: "요청성공 - Raffle 생성이 완료되었습니다."
  • 토스트 타입: Success
  • 후행 동작: 모달이 닫히고, 생성된 Raffle의 상세 화면(CEB-BO-FQ-204)으로 이동

실패 (Failure)

  • 메시지: "요청실패 - Raffle 생성에 실패하였습니다. 다시 시도하세요"
  • 토스트 타입: Error
  • 원인 예시: 네트워크 오류, 필수 데이터 누락 등
  • 후행 동작: 모달이 닫히고 작성 화면이 유지되어 재시도할 수 있습니다.
