1. 화면 목적

마감(Closed)된 Raffle의 응모자 풀(Pool)에서 설정된 인원만큼 당첨자를 무작위 선발하고, 확정 전 당첨자 명단을 미리 검토(Preview)합니다. 필요 시 재추첨을 수행하며, 검토 완료된 결과를 시스템에 저장하여 Raffle 상태를 Ended로 변경하고 보상 지급 프로세스를 트리거합니다.

주요 기능

  • 추첨(Draw) 실행 및 가결과 미리보기
  • 재추첨 수행
  • 추첨결과 최종 확정

진입 경로

  • 추첨내역 탭(CEB-BO-FQ-206)에서 [추첨하기] 버튼 클릭 시 호출

2. UI 설명

모달 구조

추첨 결과 확인 영역 (Summary)

  • 당첨 추첨 수: 기획 단계에서 설정된 목표 당첨 인원 (예: 2명)
  • 당첨자 수: 실제 알고리즘에 의해 선발된 인원 (예: 2명)
  • 예외 케이스: 응모자가 목표 인원보다 적을 경우, 실제 당첨자 수는 응모자 수와 동일하게 표시

당첨자 리스트 (Preview List)

가추첨된 결과를 테이블로 보여줍니다.

  • 상태: Win / Lose
  • 유저 정보: {유저닉네임}, {전화번호}
  • 정렬 로직: Win 상태의 유저가 최상단에 고정 노출되며, 그 아래로 Lose 유저가 나열

하단 액션 버튼

  • [재추첨 하기]: 현재 가결과를 폐기하고, 추첨 알고리즘을 다시 실행하여 새로운 리스트를 불러옴
  • [추첨결과 확정하기]: 현재 화면에 표시된 당첨자 명단을 최종 결과로 DB에 저장

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[재추첨 하기] 현재 가결과를 폐기하고 추첨 알고리즘을 다시 실행하여 새로운 리스트 로드
[추첨결과 확정하기] 현재 당첨자 명단을 최종 결과로 DB에 저장, Raffle 상태를 Closed → Ended로 변경

4. 규칙/로직

4.1 추첨 실행 및 재추첨

  • 모달 진입 시점 또는 [재추첨 하기] 클릭 시점에 서버 API를 호출하여 난수 기반 추첨을 수행합니다.
  • 중복 당첨 방지: 동일 유저는 1회만 당첨 가능합니다.

4.2 확정 프로세스

  • [추첨결과 확정하기] 클릭 시, 서버는 해당 가결과를 영구 저장(Commit)합니다.
  • Raffle 상태는 Closed → Ended로 변경됩니다.
  • 확정 후에는 번복(재추첨)이 불가능합니다.

4.3 가중치 추첨 로직 가이드

데이터 표준 포맷 (Data Standard)

추첨 시 유저를 구분하는 고유 식별자(Unique Key)는 전화번호이며, 포맷은 다음 규칙을 엄격히 따릅니다.

  • 포맷 규칙: \[국가코드\]\[휴대전화번호\] (특수문자 제외, 공백 없음)
  • 자료형: String (문자열)
  • 예시:
    • 한국(82) 10-1234-5678 → 821012345678
    • 일본(81) 90-1234-5678 → 819012345678
  • 주의사항: 휴대전화번호 앞자리 '0' 미포함

로직 상세 프로세스 (Step-by-Step)

Step 1: 데이터 준비 (Data Preparation)

응모한 유저의 표준화된 전화번호와 응모권 개수를 매핑하여 로드합니다.

식별자 (Phone Number) 사용 응모권 (Weight)
821011112222 (User A) 1장
821033334444 (User B) 5장
819055556666 (User C) 2장
총 합계 (Total Weight) 8장

Step 2: 구간 설정 (Range Allocation)

각 유저에게 응모권 개수만큼의 확률 구간을 누적하여 할당합니다.

  • User A (8210...): 1 ~ 1 (1개 구간)
  • User B (8210...): 2 ~ 6 (5개 구간) ← 당첨 확률 높음
  • User C (8190...): 7 ~ 8 (2개 구간)

Step 3: 랜덤 추첨 및 중복 방지 (Draw & Deduplication)

  • 시스템은 1부터 전체 응모권 수(8) 사이의 숫자를 랜덤 생성합니다.
  • Random(4) 발생 → User B (821033334444) 당첨.
  • 중복 제거: 당첨된 821033334444는 남은 추첨 풀(Pool)에서 즉시 제외(Remove)합니다.
  • 반복: 목표 당첨자 수(N명)를 채울 때까지 남은 유저들로 위 과정을 반복합니다.

5. 에러 및 예외상황

5.1 추첨 실행 실패 (Toast)

모달 진입 시 또는 [재추첨 하기] 클릭 시 오류가 발생한 경우.

  • 메시지: "요청실패 - Raffle 추첨에 실패하였습니다. 다시 시도하세요"
  • 토스트 타입: Error
  • 원인: 일시적 서버 오류, 응모자 데이터 로딩 실패 등
  • 동작: 모달은 유지되며, 사용자는 다시 재추첨 버튼을 누를 수 있습니다.

5.2 추첨 결과 확정 실패 (Toast)

[추첨결과 확정하기] 클릭 시 오류가 발생한 경우.

  • 메시지: "요청실패 - Raffle 추첨결과 확정에 실패하였습니다. 다시 시도하세요"
  • 토스트 타입: Error
  • 원인: 네트워크 타임아웃, DB 트랜잭션 오류, 이미 확정된 건 등
  • 동작: 모달은 유지되며, 사용자는 다시 확정 버튼을 누를 수 있습니다.

5.3 성공 피드백 (Success Toast)

확정이 정상적으로 완료된 경우.

  • 메시지: "요청성공 - Raffle 추첨결과가 확정 되었습니다."
  • 토스트 타입: Success
  • 동작: 모달이 닫히고, 상위 화면(추첨내역 탭)이 갱신되어 확정된 리스트를 보여줍니다.
