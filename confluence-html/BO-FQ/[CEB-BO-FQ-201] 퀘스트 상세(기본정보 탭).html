1. 화면 목적

등록된 개별 퀘스트의 기본 정보와 설정 상태를 확인하고, 퀘스트의 생애 주기(Life Cycle)를 관리합니다.

주요 기능

  • 퀘스트 기본 정보 및 설정 상태 확인
  • 유저들의 퀘스트 참여(제출) 현황을 대시보드(수치) 형태로 모니터링
  • 퀘스트의 생애 주기 관리 (게시, 수정, 종료, 삭제)

진입 경로

  • [CEB-BO-FQ-101] 팬퀘스트 메인 (Quest 탭) > Quest 리스트 ROW 클릭

2. UI 설명

2.1 상단 네비게이션 및 헤더

  • Breadcrumb: 홈 > 게임존 > Fan Quest 관리 > Quest > {Quest Title}
  • 타이틀: 퀘스트 제목 (예: iKON 굿즈 인증)
  • 탭 메뉴 (Tab Menu):
    • 기본정보 (Selected): 현재 화면
    • 대기내역: 검수 대기 중인 제출 건 리스트로 이동
    • 처리내역: 승인/반려 처리된 제출 건 리스트로 이동

2.2 상태별 액션 버튼

화면 우측 상단에 위치하며, 퀘스트의 현재 상태(Status)에 따라 노출되는 버튼이 달라집니다.

상태 버튼 구성 기능 설명
Draft (작성 중) [삭제하기] 퀘스트를 영구 삭제합니다.
[수정하기] 퀘스트 정보를 수정합니다.
[게시하기] 퀘스트를 유저에게 공개하고 즉시 시작(Active) 상태로 전환합니다.
Active (진행 중) [수정하기] 일부 정보(텍스트 등)를 수정합니다.
[종료하기] 퀘스트를 조기 종료하거나 마감 처리하여 Ended 상태로 전환합니다.
Ended (종료) (버튼 없음) 종료된 퀘스트는 수정 및 상태 변경이 불가능합니다.

2.3 현황 대시보드 (Dashboard Cards)

상단 영역에 4개의 카드로 구성되어 실시간 제출 현황을 표시합니다.

  • 총 제출 수: 전체 참여 건수
  • Approved (승인): 검수 완료 및 보상 지급 확정된 건수
  • Rejected (반려): 검수 결과 반려된 건수
  • Pending (대기): 아직 검수가 진행되지 않은 대기 건수

2.4 상세 정보 그리드 (Info Grid)

화면 중앙을 4분할(2x2)하여 정보를 제공합니다.

A. Quest 정보 (좌상단)

  • 아티스트: 연결된 아티스트 명
  • 타입: 퀘스트 미션 타입 (예: 이미지 촬영 및 업로드)
  • 보상: 설정된 보상 아이콘 및 수량 (예: 1 Ticket, {NFT이벤트명})
  • 일시 표시 (상태별 상이):
    • Draft: 마감일시만 표시
    • Active / Ended: 시작(게시)일시와 마감(종료)일시를 모두 표시

B. 관리 정보 (우상단)

  • 상태 (Badge): Draft / Active / Ended
  • 이력 정보: 생성 관리자 / 생성 일시, 최근 수정자 / 최근 수정 일시

C. Quest 타이틀 (하단)

  • 유저 앱(App)에 노출되는 타이틀 텍스트
  • 각 언어별 타이틀 입력값이 표시됩니다.

D. Quest 설명 (하단)

  • 유저 앱(App) 상세 화면에 노출되는 설명 텍스트
  • 각 언어별 설명 입력값이 표시됩니다.

E. Quest 유저 가이드 (하단)

  • 유저 앱(App) 상세 화면에 노출되는 유저가이드 텍스트
  • 각 언어별 유저가이드 입력값이 표시됩니다.

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

  • [삭제하기] (Draft): 퀘스트를 영구 삭제합니다.
  • [수정하기] (Draft/Active): 퀘스트 정보를 수정합니다. [CEB-BO-FQ-101-Create] 수정 모드로 이동합니다.
  • [게시하기] (Draft): 퀘스트를 유저에게 공개하고 Active 상태로 전환합니다.
  • [종료하기] (Active): 퀘스트를 조기 종료하여 Ended 상태로 전환합니다.

3.2 네비게이션

  • 대기내역 탭 클릭: [CEB-BO-FQ-202] 대기내역 화면으로 이동
  • 처리내역 탭 클릭: [CEB-BO-FQ-203] 처리내역 화면으로 이동

4. 규칙/로직

4.1 상태 전환 로직

  • 게시하기 (Draft -> Active): Draft 상태에서 [게시하기] 버튼 클릭 시, 현재 서버 시간을 시작 일시로 기록하며 상태가 Active로 변경됩니다. 유저 앱(App)에 퀘스트가 즉시 노출됩니다.
  • 종료하기 (Active -> Ended): Active 상태에서 [종료하기] 버튼 클릭 시, 퀘스트 참여가 중단되며 상태가 Ended로 변경됩니다. 마감일시가 도래하면 시스템에 의해 자동으로 Ended 처리 됩니다.

4.2 데이터 표시 규칙

  • Pending(대기) 카운트: Ended 상태가 되어도 미처리된 건이 있다면 숫자가 남아있을 수 있습니다.
  • 날짜 포맷: YYYY.MM.DD hh:mm 형태를 준수합니다.

5. 에러 및 예외상황

  • 삭제 불가: 이미 참여자가 발생한 Active 퀘스트는 삭제가 불가능하며, 종료하기만 가능합니다. (Draft 상태에서만 삭제 가능)
  • 수정 제한: Active 상태에서는 핵심 정보(보상 종류, 아티스트 등)의 수정이 제한될 수 있으며, 오타 수정 등 단순 정보 변경만 허용됩니다.
