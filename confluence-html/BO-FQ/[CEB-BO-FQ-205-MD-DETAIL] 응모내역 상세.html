1. 화면 목적

특정 유저가 해당 Raffle에 응모한 모든 상세 이력(Log)을 시계열로 확인합니다.

유저가 언제, 몇 개의 응모권을 사용하여 참여했는지 구체적인 데이터를 파악하여 CS 대응이나 어뷰징 분석에 활용합니다.

주요 기능

  • 참여자 기본 정보 확인 (닉네임, 전화번호, 누적 사용 응모권)
  • 응모 이력 시계열 조회
  • 페이지네이션을 통한 대량 이력 탐색

진입 경로

  • 응모내역 탭(CEB-BO-FQ-205) 목록에서 Row 클릭 시 호출

2. UI 설명

모달 구조

타이틀: 응모내역 상세

닫기 버튼 (X): 우측 상단

참여자 정보 영역 (User Info)

상단에 위치하며, 선택한 유저의 기본 식별 정보를 요약하여 보여줍니다.

  • 닉네임: {유저닉네임}
  • 전화번호: 821012345678 (국가코드 포함 또는 하이픈 포맷팅)
  • 누적 사용 응모권: 10 (해당 유저의 총 응모 횟수 합계)

응모내역 리스트 영역 (History List)

하단에 위치하며, 유저의 응모 행위를 시간순으로 나열한 테이블입니다.

컬럼명 데이터 예시 설명
사용 응모권 1 해당 회차에 사용한 티켓 수량
응모일시 (최근 순) YYYY.MM.DD hh:mm 응모 버튼을 누른 서버 시간
  • 정렬: 가장 최근 응모 이력이 최상단에 위치합니다 (내림차순)
  • 페이지네이션: 내역이 많을 경우 하단에 페이지 이동 컨트롤(1, 2, ...)을 제공하여 스크롤 압박을 방지

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[X] 닫기 모달을 닫고 응모내역 목록으로 복귀

4. 규칙/로직

4.1 데이터 바인딩

  • CEB-BO-FQ-205 목록 화면에서 클릭한 유저의 ID를 기준으로, 해당 유저의 응모 로그 테이블을 조회하여 데이터를 매핑합니다.
  • 누적 합계 검증: 리스트에 나열된 모든 사용 응모권의 합은 상단 누적 사용 응모권 수치와 일치해야 합니다.

4.2 페이징 (Pagination)

  • 한 페이지당 노출할 내역 수(예: 10건)를 초과할 경우 페이징 처리됩니다.
  • 모달 내부에서만 동작하며, 모달을 닫았다가 다시 열면 1페이지로 초기화됩니다.
