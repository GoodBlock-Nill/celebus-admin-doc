1. 화면 목적

Raffle 정보 수정 작업 도중 사용자가 [취소하기] 버튼을 눌렀을 때, 현재까지 변경한 내용이 저장되지 않고 사라짐을 경고하고 최종 취소 의사를 확인합니다.

주요 기능

  • 수정 데이터 유실 경고 안내
  • 수정 취소 최종 확인

진입 경로

  • Raffle 수정 화면(CEB-BO-FQ-204-EDIT)에서 [취소하기] 버튼 클릭 시 호출

2. UI 설명

모달 구조

  • 아이콘: 상단 중앙에 경고 심볼(Warning Icon) 배치
  • 타이틀: "Raffle 수정 취소"
  • 안내 메시지:
    • "수정된 데이터는 저장되지 않습니다"
    • "계속 진행하시겠습니까?"

하단 액션 버튼

  • [다시 돌아가기]: 취소 요청을 철회하고, 현재 수정 화면을 유지
  • [확인]: 수정 내역을 모두 버리고(Revert), 이전 상세 화면으로 이동

상세 스타일은 피그마 참조

3. 화면 내 기능설명

3.1 버튼 동작

버튼 동작
[다시 돌아가기] 취소 요청을 철회하고 수정 화면 유지
[확인] 수정 내역을 모두 버리고 Raffle 상세(CEB-BO-FQ-204)로 이동
[X] 닫기 모달만 닫히며 수정 작업 계속 가능

4. 규칙/로직

4.1 적용 상태

  • 본 모달은 임시저장(Draft) 및 진행중(Active) 상태 모두에서 수정 화면 진입 후 [취소하기] 클릭 시 동일하게 호출됩니다.

4.2 취소 프로세스

  • 진입: 수정 화면([CEB-BO-FQ-204-EDIT])에서 [취소하기] 버튼 클릭 시 호출
  • 확인(Confirm): [확인] 버튼 클릭 시, 브라우저에 임시 저장되어 있던 변경 사항(Input 값)을 모두 초기화합니다. 화면은 [CEB-BO-FQ-204] Raffle 상세 화면으로 리다이렉트됩니다.
  • 철회(Cancel): [다시 돌아가기] 또는 우측 상단 [X] 버튼 클릭 시 모달만 닫히며, 사용자는 이어서 수정 작업을 계속할 수 있습니다.

5. 연관화면ID

화면 ID 화면명 관계
