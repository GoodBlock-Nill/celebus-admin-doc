1. 화면 목적

운영용 블록체인 지갑(충전용/출금용)을 새로 등록하거나 기존 지갑을 수정하는 모달입니다. 지갑 유형과 주소를 입력하여 저장합니다. 등록과 수정 모두 동일한 모달 레이아웃을 사용하며, 타이틀과 버튼 레이블이 변경됩니다.

3. 화면 내 기능설명

3.1 모달 트리거

등록 모드 오픈

  • 트리거:
  • CEB-BO-GZ-503 (지갑 관리) 화면의 [+ 지갑 등록] 버튼 클릭
  • 초기 상태:
  • 타이틀: "지갑 등록"
  • 지갑 유형: "충전용" 기본 선택
  • 지갑 주소: 빈 값
  • 등록 버튼 라벨: "등록"
  • 등록 버튼: 활성화 상태 (입력값에 따라 비활성화 가능)

수정 모드 오픈

  • 트리거:
  • CEB-BO-GZ-503 (지갑 관리) 화면의 지갑 목록 Row 우측 [수정] 버튼 클릭
  • 초기 상태:
  • 타이틀: "지갑 수정"
  • 지갑 유형: 기존 지갑의 유형 바인딩 (충전용/출금용)
  • 지갑 주소: 기존 지갑 주소 바인딩
  • 수정 버튼 라벨: "수정"
  • 수정 버튼: 활성화 상태

3.2 지갑 유형 선택

  • 트리거:
  • Toggle Button ([충전용] 또는 [출금용]) 클릭
  • 동작:
  • 선택된 버튼이 활성 상태로 변경
  • 선택되지 않은 버튼은 비활성 상태로 변경
  • 선택값이 즉시 변경됨 (별도 확인 없음)
  • 지갑 유형 정의:
  • 충전용 (CHARGE): GP 충전 시 사용되는 지갑 (CELB 수신)
  • 출금용 (WITHDRAW): GP 출금 시 사용되는 지갑 (CELB 송금)

3.3 지갑 주소 입력

  • 입력 필드:
  • 타입: Text Input
  • 플레이스홀더: "0x..."
  • 최대 길이: 42자 (0x + 40자 hex)
  • 입력 유효성 검사 (실시간):
  • 필드가 비어있음: 경고 없음
  • 입력값이 있음:
    • 0x로 시작하지 않음 → "올바른 지갑 주소 형식이 아닙니다." (에러 메시지 표시)
    • 0x로 시작하지만 길이가 42자가 아님 → "올바른 지갑 주소 형식이 아닙니다."
    • 0x로 시작하고 길이가 42자 → 에러 메시지 제거 (정상 상태)

3.4 취소 버튼

  • 트리거:
  • [취소] 버튼 클릭
  • [X] 닫기 버튼 클릭
  • ESC 키 입력
  • 모달 배경 클릭
  • 동작:
  • 모달 닫기
  • 입력된 모든 데이터 폐기
  • CEB-BO-GZ-503 (지갑 관리) 화면으로 복귀
  • 기존 목록 상태 유지 (필터/정렬 유지)

3.5 등록 버튼 (등록 모드)

  • 트리거:
  • [등록] 버튼 클릭
  • 포커스된 상태에서 Enter 키 입력
  • 사전 검증:
  • 지갑 주소 필수 입력 확인
  • 지갑 주소 형식 검증 (0x로 시작, 42자)
  • 검증 실패 시:
    • 에러 토스트 메시지 표시
    • 모달 유지, 재입력 가능
  • 검증 성공 시 처리:
  • "로딩" 상태로 버튼 비활성화
  • 지갑 등록 API 호출
  • API 응답 대기 중 중복 클릭 방지
  • 등록 성공 (HTTP 201/200):
  • 토스트 메시지: "지갑이 등록되었습니다."
  • 모달 자동 닫기 (500ms 지연 후)
  • CEB-BO-GZ-503 (지갑 관리) 화면의 목록 자동 갱신
  • 관리자 활동 로그 기록
  • 등록 실패:
  • HTTP 400: "올바른 지갑 주소 형식이 아닙니다." (또는 서버 에러 메시지)
  • HTTP 403: "지갑 등록 권한이 없습니다."
  • HTTP 409: "이미 등록된 지갑입니다."
  • HTTP 500: "지갑 등록 중 오류가 발생했습니다."
  • 네트워크 에러: "네트워크 연결을 확인해주세요."
  • 실패 시 처리:
  • 에러 토스트 메시지 표시
  • 모달 유지 (재시도 가능)
  • 입력값 유지

3.6 수정 버튼 (수정 모드)

  • 트리거:
  • [수정] 버튼 클릭
  • 포커스된 상태에서 Enter 키 입력
  • 사전 검증:
  • 지갑 주소 필수 입력 확인
  • 지갑 주소 형식 검증 (0x로 시작, 42자)
  • 변경사항 확인 (기존값과 다른지 확인, 선택사항)
  • 검증 실패 시:
    • 에러 토스트 메시지 표시
    • 모달 유지
  • 검증 성공 시 처리:
  • "로딩" 상태로 버튼 비활성화
  • 지갑 수정 API 호출
  • API 응답 대기 중 중복 클릭 방지
  • 수정 성공 (HTTP 200):
  • 토스트 메시지: "지갑이 수정되었습니다."
  • 모달 자동 닫기 (500ms 지연 후)
  • CEB-BO-GZ-503 (지갑 관리) 화면의 목록 자동 갱신
  • 관리자 활동 로그 기록
  • 수정 실패:
  • HTTP 400: "올바른 지갑 주소 형식이 아닙니다." (또는 서버 에러 메시지)
  • HTTP 403: "지갑 수정 권한이 없습니다."
  • HTTP 404: "지갑을 찾을 수 없습니다."
  • HTTP 409: "이미 등록된 지갑입니다."
  • HTTP 500: "지갑 수정 중 오류가 발생했습니다."
  • 네트워크 에러: "네트워크 연결을 확인해주세요."
  • 실패 시 처리:
  • 에러 토스트 메시지 표시
  • 모달 유지 (재시도 가능)
  • 입력값 유지

4.8 비즈니스 규칙

  1. 지갑 유형(충전용/출금용)은 중복 등록 가능 (여러 개의 충전용 지갑 등록 가능)
  2. 등록된 지갑 주소는 중복 불가 (같은 주소의 지갑을 2개 이상 등록할 수 없음)
  3. 지갑 유형과 주소를 모두 변경할 수 있음 (수정 모드)
  4. 모달 닫기 시 입력값은 완전히 폐기됨 (로컬 저장소 미사용)

5. 에러 및 예외상황

5.1 입력 검증 에러

상황 에러 메시지 처리 방법
지갑 주소 미입력 "지갑 주소를 입력하세요." 토스트 표시, 모달 유지
지갑 주소가 0x로 시작하지 않음 "올바른 지갑 주소 형식이 아닙니다." 실시간 에러 표시, 입력 필드 하단
지갑 주소 길이가 42자가 아님 "올바른 지갑 주소 형식이 아닙니다." 실시간 에러 표시, 입력 필드 하단

5.2 서버 에러 (등록/수정 실패)

HTTP 상태 에러 메시지 처리 방법
400 "올바른 지갑 주소 형식이 아닙니다." 또는 "잘못된 요청입니다." 에러 토스트 표시, 모달 유지
403 "지갑 등록/수정 권한이 없습니다." 에러 토스트 표시, 모달 닫기
404 "지갑을 찾을 수 없습니다." (수정 모드만) 에러 토스트 표시, 모달 닫기, 리스트 화면 갱신
409 "이미 등록된 지갑입니다." 에러 토스트 표시, 모달 유지
500 "서버 오류가 발생했습니다. 다시 시도해주세요." 에러 토스트 표시, 모달 유지

상황 에러 메시지 처리 방법

연결 실패 "네트워크 연결을 확인해주세요." 에러 토스트 표시, 모달 유지
