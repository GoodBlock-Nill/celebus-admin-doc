1. 화면 목적

운영용 블록체인 지갑(충전용/출금용)을 새로 등록하거나 기존 지갑을 수정하는 모달입니다. 지갑 유형과 주소를 입력하며, 출금용 지갑의 경우 토큰 전송에 필요한 Private Key도 함께 등록합니다. 등록과 수정 모두 동일한 모달 레이아웃을 사용하며, 타이틀과 버튼 레이블이 변경됩니다.

3. 화면 내 기능설명

3.1 모달 트리거

등록 모드 오픈

  • 트리거:
  • CEB-BO-GZ-503 (지갑 관리) 화면의 [+ 지갑 등록] 버튼 클릭
  • 초기 상태:
  • 타이틀: "지갑 등록"
  • 지갑 유형: "충전용" 기본 선택
  • 지갑 주소: 빈 값
  • 등록 버튼 라벨: "등록"
  • 등록 버튼: 활성화 상태 (입력값에 따라 비활성화 가능)

수정 모드 오픈

  • 트리거:
  • CEB-BO-GZ-503 (지갑 관리) 화면의 지갑 목록 Row 우측 [수정] 버튼 클릭
  • 초기 상태:
  • 타이틀: "지갑 수정"
  • 지갑 유형: 기존 지갑의 유형 바인딩 (충전용/출금용)
  • 지갑 주소: 기존 지갑 주소 바인딩
  • 수정 버튼 라벨: "수정"
  • 수정 버튼: 활성화 상태

3.2 지갑 유형 선택

  • 트리거:
  • Toggle Button ([충전용] 또는 [출금용]) 클릭
  • 동작:
  • 선택된 버튼이 활성 상태로 변경
  • 선택되지 않은 버튼은 비활성 상태로 변경
  • 선택값이 즉시 변경됨 (별도 확인 없음)
  • 출금용 선택 시: Private Key 입력 필드 및 BNB 안내 박스 노출
  • 충전용 선택 시: Private Key 입력 필드 및 BNB 안내 박스 숨김 (기존 입력값 폐기)
  • 지갑 유형 정의:
  • 충전용 (CHARGE): GP 충전 시 사용되는 지갑 (CELB 수신)
  • 출금용 (WITHDRAW): GP 출금 시 사용되는 지갑 (CELB 송금)

3.3 지갑 주소 입력

  • 입력 필드:
  • 타입: Text Input
  • 플레이스홀더: "0x..."
  • 최대 길이: 42자 (0x + 40자 hex)
  • 입력 유효성 검사 (실시간):
  • 필드가 비어있음: 경고 없음
  • 입력값이 있음:
    • 0x로 시작하지 않음 → "올바른 지갑 주소 형식이 아닙니다." (에러 메시지 표시)
    • 0x로 시작하지만 길이가 42자가 아님 → "올바른 지갑 주소 형식이 아닙니다."
    • 0x로 시작하고 길이가 42자 → 에러 메시지 제거 (정상 상태)

3.4 Private Key 입력 (출금용 전용)

  • 노출 조건:
  • 지갑 유형이 "출금용 (WITHDRAW)"으로 선택된 경우에만 노출
  • 충전용 선택 시 숨김
  • 입력 필드:
  • 타입: Password Input (마스킹 표시)
  • 플레이스홀더: "Private Key를 입력하세요"
  • 최대 길이: 66자 (0x 포함 시) 또는 64자 (hex only)
  • 등록 모드:
  • 필수 입력 (출금용일 때)
  • 도움말: "토큰 전송 트랜잭션 서명에 필요합니다. 서버에 암호화하여 저장됩니다."
  • 수정 모드:
  • 선택 입력 (빈 값이면 기존 PK 유지)
  • 도움말: "변경이 필요한 경우에만 입력하세요."
  • 기존 PK 값은 보안상 표시하지 않음 (빈 필드로 표시)
  • 보안 규칙:
  • 화면에서 입력값은 항상 마스킹 (●●●) 표시
  • 서버 전송 시 HTTPS 필수
  • 서버에 암호화 저장 (평문 저장 금지)
  • 클라이언트 메모리에서 저장 후 즉시 폐기

3.5 BNB 안내 박스 (출금용 전용)

  • 노출 조건:
  • 지갑 유형이 "출금용 (WITHDRAW)"으로 선택된 경우에만 노출
  • 표시 내용:
  • 스타일: 주황색 배경 안내 박스 (Warning Info Box)
  • 아이콘: ⚠ (경고)
  • 텍스트: "출금용 지갑은 가스비(BNB) 잔고가 있어야 출금이 가능합니다."

3.6 취소 버튼

  • 트리거:
  • [취소] 버튼 클릭
  • [X] 닫기 버튼 클릭
  • ESC 키 입력
  • 모달 배경 클릭
  • 동작:
  • 모달 닫기
  • 입력된 모든 데이터 폐기
  • CEB-BO-GZ-503 (지갑 관리) 화면으로 복귀
  • 기존 목록 상태 유지 (필터/정렬 유지)

3.7 등록 버튼 (등록 모드)

  • 트리거:
  • [등록] 버튼 클릭
  • 포커스된 상태에서 Enter 키 입력
  • 사전 검증:
  • 지갑 주소 필수 입력 확인
  • 지갑 주소 형식 검증 (0x로 시작, 42자)
  • 출금용인 경우 Private Key 필수 입력 확인
  • 검증 실패 시:
    • 에러 토스트 메시지 표시
    • 모달 유지, 재입력 가능
  • 검증 성공 시 처리:
  • "로딩" 상태로 버튼 비활성화
  • 지갑 등록 API 호출
  • API 응답 대기 중 중복 클릭 방지
  • 등록 성공 (HTTP 201/200):
  • 토스트 메시지: "지갑이 등록되었습니다."
  • 모달 자동 닫기 (500ms 지연 후)
  • CEB-BO-GZ-503 (지갑 관리) 화면의 목록 자동 갱신
  • 관리자 활동 로그 기록
  • 등록 실패:
  • HTTP 400: "올바른 지갑 주소 형식이 아닙니다." (또는 서버 에러 메시지)
  • HTTP 403: "지갑 등록 권한이 없습니다."
  • HTTP 409: "이미 등록된 지갑입니다."
  • HTTP 500: "지갑 등록 중 오류가 발생했습니다."
  • 네트워크 에러: "네트워크 연결을 확인해주세요."
  • 실패 시 처리:
  • 에러 토스트 메시지 표시
  • 모달 유지 (재시도 가능)
  • 입력값 유지

4.8 비즈니스 규칙

  1. 지갑 유형(충전용/출금용)은 중복 등록 가능 (여러 개의 충전용 지갑 등록 가능)
  2. 등록된 지갑 주소는 중복 불가 (같은 주소의 지갑을 2개 이상 등록할 수 없음)
  3. 지갑 유형과 주소를 모두 변경할 수 있음 (수정 모드)
  4. 모달 닫기 시 입력값은 완전히 폐기됨 (로컬 저장소 미사용)
  5. 출금용 지갑 등록 시 Private Key는 필수이며, 서버에 암호화하여 저장한다
  6. Private Key는 등록 후 화면에 전체 노출하지 않으며, 마스킹 처리한다
  7. 수정 모드에서 Private Key를 입력하지 않으면 기존 값을 유지한다
  8. 출금용 지갑은 BNB 잔고가 있어야 출금 트랜잭션 실행이 가능하다

4.9 토스트 메시지 규격

5. 에러 및 예외상황

5.1 입력 검증 에러

상황 에러 메시지 처리 방법
지갑 주소 미입력 "지갑 주소를 입력하세요." 토스트 표시, 모달 유지
지갑 주소가 0x로 시작하지 않음 "올바른 지갑 주소 형식이 아닙니다." 실시간 에러 표시, 입력 필드 하단
지갑 주소 길이가 42자가 아님 "올바른 지갑 주소 형식이 아닙니다." 실시간 에러 표시, 입력 필드 하단
Private Key 미입력 (출금용) "Private Key를 입력하세요." 토스트 표시, 모달 유지

5.2 서버 에러 (등록/수정 실패)

HTTP 상태 에러 메시지 처리 방법
400 "올바른 지갑 주소 형식이 아닙니다." 또는 "잘못된 요청입니다." 에러 토스트 표시, 모달 유지
403 "지갑 등록/수정 권한이 없습니다." 에러 토스트 표시, 모달 닫기
404 "지갑을 찾을 수 없습니다." (수정 모드만) 에러 토스트 표시, 모달 닫기, 리스트 화면 갱신
409 "이미 등록된 지갑입니다." 에러 토스트 표시, 모달 유지
500 "서버 오류가 발생했습니다. 다시 시도해주세요." 에러 토스트 표시, 모달 유지

5.3 네트워크 에러

상황 에러 메시지 처리 방법
타임아웃 "요청 시간이 초과되었습니다. 다시 시도해주세요." 에러 토스트 표시, 모달 유지
연결 실패 "네트워크 연결을 확인해주세요." 에러 토스트 표시, 모달 유지

5.4 중복 클릭 방지

상황 처리 방법
[등록]/[수정] 버튼 연속 클릭 첫 번째 클릭 후 버튼 비활성화, API 응답 대기
버튼 비활성화 상태에서 클릭 무시됨 (이벤트 발생 안 함)

5.5 기타 예외 상황

상황 처리 방법
모달이 열려있는 상태에서 부모 화면 새로고침 모달 닫기, 부모 화면 갱신
모달 오픈 중 권한 변경 다음 모달 오픈 시 권한 재확인
모달 외부 클릭 모달 닫기 (취소와 동일)
