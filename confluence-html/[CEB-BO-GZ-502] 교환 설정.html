1. 화면 목적

GP와 CELB 간 교환에 관련된 모든 설정을 관리하는 화면입니다. 교환 비율, 최소/최대 수량, 1일 한도, 교환 기능 활성화/비활성화를 설정할 수 있습니다.

주요 기능

  • GP↔CELB 교환 비율 설정
  • GP 가져오기(CELB→GP) 최소/최대 수량 설정
  • GP 가져오기 1일 한도 설정 (금액/횟수)
  • CELB으로 보내기(GP→CELB) 최소/최대 수량 설정
  • CELB으로 보내기 1일 한도 설정 (금액/횟수)
  • GP 가져오기/출금 기능 ON/OFF
  • 설정 초기화 (기본값 복원)
  • 설정 저장

접근 경로

홈 > 게임존 > GP 교환소 > [교환 설정] 탭
또는
CEB-BO-GZ-501 화면 우측 [교환 설정] 버튼

3. 화면 내 기능설명

3.1 교환 비율 설정

GP → CELB 비율

  • 라벨: "GP → CELB 비율"
  • 입력: Number Input
  • 형식: "1 GP = {숫자} CELB"
  • 기본값: 1.0
  • 최솟값: 0.1
  • 최댓값: 100.0
  • 소수점: 1자리
  • 설명: GP를 CELB로 교환할 때 적용되는 비율

CELB → GP 비율

  • 라벨: "CELB → GP 비율"
  • 입력: Number Input
  • 형식: "1 CELB = {숫자} GP"
  • 기본값: 1.0
  • 최솟값: 0.1
  • 최댓값: 100.0
  • 소수점: 1자리
  • 설명: CELB를 GP로 교환할 때 적용되는 비율

3.7 사용자 시나리오

시나리오 1: 교환 비율 변경

  1. 관리자가 [게임존 > GP 교환소 > 교환 설정] 메뉴 클릭
  2. 교환 설정 화면 진입
  3. "GP → CELB 비율" 입력란에 "1.5" 입력 (1 GP = 1.5 CELB)
  4. "CELB → GP 비율" 입력란에 "0.8" 입력 (1 CELB = 0.8 GP)
  5. [저장] 버튼 클릭
  6. Toast "교환 설정이 저장되었습니다." 표시
  7. 새로운 교환부터 변경된 비율 적용 (기존 교환은 소급 적용 안됨)

시나리오 2: GP 가져오기 한도 설정

  1. "1일 충전 한도" 입력란에 "50,000" 입력
  2. [저장] 버튼 클릭
  3. 사용자 A가 오늘 이미 30,000 CELB 충전함
  4. 사용자 A가 추가로 25,000 CELB 충전 시도
  5. 앱에서 에러 메시지: "1일 충전 한도를 초과했습니다. (30,000 / 50,000)"

시나리오 3: CELB으로 보내기 기능 비활성화

  1. "CELB으로 보내기 활성화" Toggle을 OFF로 전환
  2. [저장] 버튼 클릭
  3. 앱에서 CELB으로 보내기 버튼 미노출
  4. 사용자가 직접 API 호출 시도 시 403 에러 반환
  5. 긴급 상황 해결 후 다시 ON으로 전환하여 정상화

시나리오 4: 최소/최대 수량 설정 오류

  1. "최소 충전 수량" 입력란에 "1,000" 입력
  2. "최대 충전 수량" 입력란에 "500" 입력 (최소보다 작음)
  3. [저장] 버튼 클릭
  4. Toast "최대 충전 수량은 최소 충전 수량보다 커야 합니다."
  5. "최대 충전 수량"을 "2,000"으로 수정 후 다시 저장

시나리오 5: 설정 초기화

  1. 여러 설정을 변경한 후 [초기화] 버튼 클릭
  2. Confirm 모달: "모든 설정을 기본값으로 초기화하시겠습니까?"
  3. [확인] 클릭
  4. 모든 입력란이 기본값으로 변경됨
  5. [저장] 버튼 클릭하여 초기화 완료

비즈니스 규칙

  1. 교환 비율 변경은 새 교환부터 적용됩니다 (기존 교환 소급 적용 안됨)
  2. GP 가져오기/출금 기능 비활성화 시 앱에서 버튼 미노출 및 API 호출 차단
  3. 1일 한도는 00:00~23:59 기준입니다 (UTC+9 기준)
  4. 1일 한도 0은 무제한을 의미합니다
  5. 최소 수량은 반드시 최대 수량 이하여야 합니다
  6. 1일 한도는 최대 수량 이상이어야 합니다 (0 제외)
  7. 교환 비율은 소수점 1자리까지 허용됩니다
  8. 설정 변경 시 관리자 로그가 기록됩니다
  9. 교환 기능 비활성화는 긴급 상황에만 사용됩니다
  10. 초기화는 Confirm 후 실행됩니다 (실수 방지)

교환 비율 비대칭 경고

GP→CELB 비율과 CELB→GP 비율의 곱이 1.0이 아닐 경우 경고 모달 표시:

예시: GP→CELB = 1.5, CELB→GP = 0.8 (곱 = 1.2)

교환 비율이 비대칭입니다.

GP→CELB: 1 GP = 1.5 CELB
CELB→GP: 1 CELB = 0.8 GP

이 설정은 사용자가 GP ↔ CELB를 반복 교환 시
손실 또는 이득이 발생할 수 있습니다.
